UKG Ready to Acumatica Employee Synchronization Using Scripts

← Back to index

This guide covers the end-to-end tasks in setting up a script synchronization to sync employees from UKG Ready to Acumatica.

The following tasks are involved in setting up the synchronization from UKG Ready to Acumatica:

  • Initial Preparation
  • Configure Applications in Aquera Admin Portal
  • Synchronization Setup
  • Run Script Synchronization

Initial Preparation

Aquera Admin Portal

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configure Applications in Aquera Admin Portal

You must add and configure the following applications in the Aquera Admin portal:

  • UKG Ready: Refer to the UKG Ready Configuration Guide for details.
  • Acumatica: Refer to the Acumatica Configuration Guide for details.

Synchronization Setup

This section details how the synchronization must be configured to allow data push from UKG Ready to Acumatica.

Create Script Synchronization

Perform the following steps to create a new script synchronization:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations on the left panel which displays the list of synchronizations created by the user. 
    Note: The number of synchronization records displayed on this page is based on the display size of the user's screen.
  3. Click Create Synchronization.
  4. Select Script.
  5. Enter a name and a description for the synchronization.
  6. Click Continue. A new page is displayed to configure the synchronization.
  7. Click Create to create the script synchronization.

Configure Script Synchronization

This document describes how to set up the synchronization using the Script Synchronization method.

Connections

  • Set UKG Ready as an application from the Connections tab with the variable ukg.
  • Set Acumatica as an application from the Connections tab with the variable acumatica.

Refer to the Synchronization Setup Using Scripts and Working With Synchronization Scripts guides for details on the configurations.

Script

This section allows you to write a script to set up the synchronization. The script is based on JavaScript, so all the functions and language constructs are available in the script.

Note: The following script covers the synchronization of an employee from UKG Ready to Acumatica. During the synchronization, the script checks the existence of the UKG Ready employee in Acumatica based on the employee ID and creates a new employee in Acumatica if a matching user does not exist. You must modify the script based on your use case.

Configure the following script in the Script tab to synchronize an employee from UKG Ready to Acumatica:

let ukgEmployees = [];
let ukgEmployeesObj = ukg.getEmployee("17307179861");
ukgEmployees.push(ukgEmployeesObj);
let reportOnly = false;
for (var ukgEmployee of ukgEmployees) {
    if (acumatica.getEmployeeID({
            "EmployeeID": ukgEmployee.employeeId
        })?.statusCode !== 200) {
        let createObj = {
            "ContactInfo": {
                "FirstName": {
                    "value": ukgEmployee.firstName
                },
                "LastName": {
                    "value": ukgEmployee.lastName
                },
                "Email": {
                    "value": ukgEmployee.primaryEmail
                },
                "Phone1Type": {
                    "value": "Business 1"
                },
                "Phone1": {
                    "value": ukgEmployee?.phones?.workPhone
                }
            },
            "EmployeeSettings": {
                "CurrencyID": {
                    "value": "USD"
                },
                "DepartmentID": {
                    "value": "EXECUTIVE"
                },
                "EmployeeClass": {
                    "value": "SALARIED"
                },
                "ReportsTo": {
                    "value": (ukg.getEmployee((ukgEmployee?.managers.length  1 ? (ukgEmployee?.managers[1]?.emplRef?.accountId) : (ukgEmployee?.managers[0]?.emplRef?.accountId))))?.employeeId
                }
            },
            "Status": {
                "value": "Active"
            },
            "EmployeeID": {
                "value": ukgEmployee.employeeId
            },
            "EmployeeName": {
                "value": ukgEmployee.firstName + " " + ukgEmployee.lastName
            }
        };
        if (!reportOnly) {
            log.info("The user " + ukgEmployee.firstName + " " + ukgEmployee.lastName + " does not exist in Acumatica hence creating the user");
            log.info("User JSON before create " + JSON.stringify(createObj));
            let createdUser = acumatica.postEmployee(createObj);
            log.info("Response from Acumatica after creating the user " + JSON.stringify(createdUser));
            log.info("================================================================================================");
        } else {
            log.info("ReportOnly Not Created \nThe user " + ukgEmployee.firstName + " " + ukgEmployee.lastName + " does not exist in Acumatica hence user will be created");
            log.info("User JSON before create " + JSON.stringify(createObj));
            log.info("================================================================================================");
        }
    } else {
        log.info("The user " + ukgEmployee.firstName + " " + ukgEmployee.lastName + " Already exists in Acumatica hence not created");
        log.info("================================================================================================");
    }
}
