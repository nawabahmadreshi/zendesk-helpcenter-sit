Paylocity to Brivo Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Paylocity to Brivo integration, which synchronizes user data from Paylocity to Brivo to support user lifecycle and group management.

To identify user records across both systems, the integration uses the Employee ID attribute from Paylocity and matches it to the External ID in Brivo as a correlation attribute.

Key Functions

The Paylocity to Brivo integrations provide the following functions:

  • User Lifecycle Management: Handles user creation, updates, and deactivation (suspend/delete) from Paylocity to Brivo.

  • Group Rules and Assignment: Assigns users to specific groups in Brivo based on work location attributes like location, country, state, and city. Group assignments are ignored if the conditions are not met or if work location details are missing.

The following are the prerequisites:

Integration Setup

This section outlines how the integration must be configured to push user data from Paylocity to Brivo.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Paylocity to Brivo Integration.
  5. Enter a name and a description for the integration.
  6. Select your Paylocity and Brivo connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User Lifecycle Management: Select to enable user provisioning and perform incremental updates.
  • Enable Group Management: Select to assign users to groups based on their work location.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Log Insights: Select to visualize the integration-related metrics.

Manage Users

The Manage Users section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Paylocity Filter Settings

Configure the following source filter settings:

  • Filter by Company ID: Select one or more company IDs to limit which users are synced. Only users associated with the selected company IDs will be included in the integration run. For example, selecting AQUERADM1 and AQUERADM2 ensures only users from those companies are synced. 
  • Statuses to be Considered as Active: Select the user statuses that should be considered as active during the sync. Users whose current status matches any of the selected values will be handled as active records in the integration. For example, selecting Active and Leave of Absence treats users in those statuses as active during the sync.
  • Statuses to be Considered as Deactivated: Select the user statuses that should be considered deactivated during the sync. Users whose current status matches any of the selected values will be handled as deactivated records in the integration. For example, selecting Terminated, Retired, or Deceased ensures users in those statuses are processed as deactivated records.
  • Rehire Employee: Select to automatically reactivate a user’s Brivo account when they become active again in Paylocity.

Note: Statuses to be Considered as Active and Statuses to be Considered as Deactivated selections must be mutually exclusive. Overlapping values will cause the integration to fail validation.

User Credential Settings

Configure the following user credential settings:

  • Enable Mobile Pass: Select to enable users to use the mobile pass.
  • Mobile Pass Email Source: Select the mobile pass email source from the dropdown menu.
  • Credentials Assignment Condition: Select the condition that decides when credentials are assigned for a user from the dropdown menu.

User Provisioning Settings

Configure the following provisioning settings:

  • Create Condition: Select a condition from the dropdown menu to determine when users are created.
    • Custom Create Condition: Enter the JavaScript expression to create a new user in the target application. 
  • Update Condition: Select a condition from the dropdown menu to determine when users are updated.
    • Custom Update Condition: Enter the JavaScript expression to update a user in the target application. 
  • Delete Condition: Select a condition from the dropdown menu to determine when users are deleted.
    • Custom Delete Condition: Enter the JavaScript expression to terminate a user in the target application. 
  • Termination Action: Select an action from the dropdown that determines how user accounts are handled when the delete condition is met. The available options are Suspend and Delete.
  • Delete Credentials from User: Select to remove all access from Brivo users when they are suspended.

Correlation Settings

Configure the following correlation settings:

  • Source Correlation Attribute: Select the source attribute for correlation from the dropdown menu.
  • Target Correlation Attribute: Select the target attribute for correlation from the dropdown menu.

User Provisioning Threshold Settings

Configure the following provisioning threshold settings:

  • Enable Create Limit: Select to restrict the number of new users created during sync.
  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Enable Update Limit: Select to restrict the number of new users updated during sync.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single integration run.
  • Enable Delete Limit: Select to restrict the number of new users deleted during sync.
  • Terminate Limit: Specify the maximum number of user records that can be terminated from the target application during a single integration run.

Group Rules

The Group Rules tab defines how Paylocity users are assigned to Brivo security groups during user creation, updates, and termination. Each rule evaluates user attributes such as City, Country, Division, Location, and State.

When a user meets a rule’s conditions, they are automatically added to or removed from the specified Brivo groups. This ensures Paylocity user records are accurately mapped to the appropriate security groups in Brivo.

Perform the following steps to create a new grouprule:

  1. Click Add New Rule to create a new group rule. 
    The following fields are displayed:
  2. Click Choose Attribute to view the list of available Paylocity location attributes.
  3. Use the search bar to locate the attribute you want to use as a condition for the rule.
  4. Select the required attribute and configure its value to define the rule condition.
  5. Only users who meet this condition are included in the group rule.
  6. Click to delete a rule by using the delete icon.
  7. Click Switch to Expression to write a JavaScript expression and set the condition.
