Greenhouse Onboarding to Okta Integration Guide

← Back to index

Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic identity provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end steps for setting up an integration to sync pre-hire and employee records from Greenhouse Onboarding to the Users resource in Okta.

To identify records across both systems, the integration uses a correlation attribute from Greenhouse Onboarding (for example, email) and matches it to a corresponding user profile attribute in Okta. This correlation attribute is used to accurately create, update, and delete user records in Okta.

Key Functions

The Greenhouse Onboarding to Okta integration automates identity lifecycle management with the following primary capabilities:

  • Syncs pre-hire and employee records from Greenhouse Onboarding into Okta.
  • Creates new Okta users for pre-hires and employees.
  • Updates the Okta user records when employee attributes change in Greenhouse.
  • Deletes Okta users when corresponding employee records no longer exist in Greenhouse.
  • Allows records to be synchronized using full or incremental sync.
  • Supports filtering employees based on employment status.
  • It utilises Okta Identity Source (XaaS) to perform bulk user creation, update, and deletion.
  • Handles rate limiting, retries, and batching automatically.
  • Generates execution reports and logs.
  • Supports report-only mode for dry-run validation.
  • Sends email notifications after integration execution.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Greenhouse Onboarding: Refer to the Greenhouse Onboarding Configuration Guide for details.
    • Okta: Refer to the Okta Configuration Guide for details.

Source Priority

If your use case requires updating user attributes, such as email or username, in Okta, ensure that the source priority for these attributes is configured correctly.

Perform the following steps to configure the source priority:

  1. In the Okta Admin Console, navigate to Directory > Profile Editor.

  2. Select the User Profile.

  3. Scroll to the Attributes section.

  4. Select the attribute (for example, email).

  5. Set the Source Priority:

    • Select Inherit from Profile Source if Greenhouse Recruiting should be the source of truth.

    • Select Inherit from Okta if the attribute should be managed in Okta.

Repeat the same steps for other attributes as required.

Integration Setup

This section details the configuration and mapping process for integrating Greenhouse Recruiting with Okta. It outlines the steps to create the integration, set up integration preferences, and define how employee data is mapped from Greenhouse to Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.

  2. Navigate to Integrations from the left side panel.

  3. Click Create Integration.

  4. Select Greenhouse Onboarding to Okta.

  5. Enter a name and a description for the integration.

  6. Select your Greenhouse Onboarding and Okta connector instances.

  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenario settings:

  • Enable Okta Email Writeback: Select to update the email ID from Okta to Greenhouse Onboarding.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize the integration-related metrics.

Employee Management

The Employee Management section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Create/Update Employment Status in Okta: Select Greenhouse employees whose status changes will create or update Okta records. The available options are full-time, contract, intern, part-time, and temporary.
  • Delete Condition: Enter the expression used to validate the deletion of users.
  • Enable Incremental Mode: Select to run the integration in incremental mode.

Debug Mode

The Debug Mode section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Source Filter: Select the Greenhouse Recruiting filter to filter the users while retrieving from Greenhouse Recruiting.
  • Import Limit: Specify the maximum number of employee records that can be imported from the source application during a single integration run.

Log Settings

The Log Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Report Settings

The Report Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.

Notifications

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

Notification Report Settings

Configure the following notification report settings:

  • Send Integration Report via Email: Select to receive email notification with a CSV report for all integration runs.
  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.

Log Insights

The Log Insights section enables you to customize the integration settings according to your specific use case. It has the following settings:

Analytics Settings

Configure the following analytics settings:

  • Select the Log Insights Detail Level: Select the desired log insights detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the log insights.
      • Excluded Attributes: Select the attributes to be excluded from the log insights.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

Unified Log Insights

Configure the following unified log insights settings:

  • Unified Log Insights Detail: Select the unified log insights detail from the dropdown menu. The following are the available options:
    • Minimum – Operations only; no GUIDs or field values.
    • Metadata – Operations and field names; no GUIDs.
    • Non-PII Data – Operations, field names, and selected non-PII values; no GUIDs.
    • GUID with Non-PII Data – GUIDs, operations, field names, and selected non-PII values.

Attribute Map

The Attribute Map tab in Greenhouse Onboarding to Okta Synchronization defines the default Greenhouse Onboarding attributes mapped to Okta attributes.

Schedule

Integrations can be scheduled to run automatically at your preferred time. Refer to the Integration Schedule Setup Guide for further details.
