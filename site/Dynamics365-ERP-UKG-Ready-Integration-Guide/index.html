Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any users created, updated, or terminated in UKG Ready are automatically updated in Microsoft Dynamics 365 based on the defined schedule.

To identify user records across both systems from UKG Ready to Microsoft Dynamics 365, the integration uses the attribute configured in the UKG Ready Correlation Attribute parameter in UKG Ready and matches it to the attribute configured in the Microsoft Dynamics 365 Finance and Operations Correlation Attribute parameter in Microsoft Dynamics 365 as a correlation attribute.

Key Functions

The Dynamics 365 / UKG Ready Integration performs the following functions:

  • Synchronizes user records from UKG Ready to Microsoft Dynamics 365.
  • Creates new employees and system users in Microsoft Dynamics 365 when they are added in UKG Ready.
  • Updates existing employees and system user records in Microsoft Dynamics 365 when the corresponding user information changes in UKG Ready.
  • Terminates the employees and system user records in Microsoft Dynamics 365 when the corresponding user records are deleted from the UKG Ready.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Integration Setup

This section outlines how the integration must be configured to push user-related data from UKG Ready to Microsoft Dynamics 365.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dynamics365-ERP / UKG Ready.
  5. Enter a name and a description for the integration.
  6. Select your Microsoft Dynamics 365 and UKG Ready connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

  • Enable Log Insights: Select to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.

Configure the following settings:

Manage Employees

This section enables you to customize the integration settings for managing employees according to your specific use cases.

Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  •  Source Correlation Attribute*: Select the UKG Ready attribute to distinctly find the associated Microsoft Dynamics 365 employee record. For example, EmployeeId.
  • Target Correlation Attribute*: Select the Microsoft Dynamics 365 attribute to distinctly find the associated UKG Ready employee record. For example, PersonnelNumber.

Source Filter

Configure the following settings to filter employee records in UKG Ready.

  • Source Filter: Select an attribute to filter UKG Ready users for provisioning to the Microsoft Dynamics 365 application. For example, if you select EmployeeId, UKG Ready users will be filtered based on this attribute.
  • Source filter Value: Enter the value by which UKG Ready users must be filtered for the attribute selected in the Source Filter parameter. For example, if you enter EMP001, UKG Ready users with this employee number will be filtered and provisioned to the Microsoft Dynamics 365 application.

Create Settings

Configure the following settings to apply a condition for creating employee records from UKG Ready to Microsoft Dynamics 365.

  • Create Condition Type: Select a condition to specify which UKG Ready employee records must be created in the Microsoft Dynamics 365 application. Only employee records that satisfy this condition will be created in the Microsoft Dynamics 365 application. The available options are None and Basic. Select None if you do not want to apply any conditions. Selecting the None option will create all users from UKG Ready in Microsoft Dynamics 365. Select Basic to filter UKG Ready users by status before creating them in the Microsoft Dynamics 365 application. The following parameter is displayed when the Basic option is selected:
    • Eligible Worker Statuses for Create: Select one or more worker statuses from the dropdown menu. Only users with the selected statuses in UKG Ready will be created in Microsoft Dynamics 365. For example, if you select Active, only users with the Active status are filtered and created in the Microsoft Dynamics 365 application.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.
