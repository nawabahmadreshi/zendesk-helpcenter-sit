Overview

When used in the context of system administration, orchestration is defined as the automated configuration, coordination, and management of computer systems and software. As opposed to the term automation which generally means completing a single task or function without human intervention, orchestration arranges tasks to optimize a workflow. 

How Aquera uses SCIM to normalize Applications

Aquera provides a large set of application connectors that facilitate synchronization between identity management providers and popular applications. By themselves, these connectors can greatly improve IT productivity and reliability. Aquera's orchestrations take this functionality to the next level by enabling workflows between two or more applications. 

Types of Orchestration

Aquera supports two types, or categories of orchestrations, scheduled and event. Both of these can be invoked manually at any time, but scheduled orchestrations facilitate automatic periodic orchestration runs, and event-based orchestrations are invoked automatically when there is a change in a managed source attribute. Event orchestrations require configuration in the application and are initiated by the source application itself outside of the Aquera platform. Not all applications support asynchronous event processing, but all applications with an Aquera connector can be included in a scheduled orchestration.

Step-by-step procedure - ADP to AD

Create the 2 entries - ADP Workforce Now and Active Directory

Click Orchestration menu

Enter a name to the Orchestration - ADP to AD

Apply SCIMToSCIM

Select Source (ADP) and Target (AD)

  1. Report Only - Checkbox is left unchecked. 
  2. Update Limit - Unlimited.
  3. Termination Delay - Unlimited.
  4. Match Expression - [ENT].employeeNumber.

Click Create

Orchestration  is successful between ADP as Source and AD as target
