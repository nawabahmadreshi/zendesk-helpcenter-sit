ADP Workforce Now / HaloITSM Ticket Management Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide explains how the hook integration works with the user integration to create tickets in HaloITSM based on employee lifecycle events in ADP Workforce Now, including onboarding, updates, transfers, and offboarding.

To use this integration, first create a User Integration in the Aquera Admin portal that retrieves employee data from ADP Workforce Now. When a new employee is onboarded, the user integration creates the corresponding user account in the target system and triggers the hook integration to generate an onboarding ticket in HaloPSA.

If an employee’s general information is updated in ADP Workforce Now, the integration creates an update ticket. If the Business Unit field changes, the integration generates a mover (transfer) ticket to reflect the employee’s new business unit. When an employee is offboarded, the user integration deprovisions the user account in the target system and triggers the hook integration to create an offboarding ticket in HaloPSA.

To enable this behavior, configure the ADP Workforce Now–HaloPSA integration and specify the integration name in the New User Integration Hook and Deprovision User Integration Hook fields in the user integration settings.

Key Functions

The ADP Workforce Now to HaloITSM integrations provide the following functions:

  • Monitors user updates triggered by the user integration.
  • Automatically triggers when changes are detected in user information.
  • Creates a ticket in ADP when a user’s information is modified.
  • Creates a mover (transfer) ticket when the Business Unit field is updated.
  • Processes the changes and creates the appropriate ticket in ADP based on the update type.
  • Creates integration reports based on the integration configuration 
  • Sends email notifications for integration runs, along with the integration report, based on the integration configuration.

Integration Setup

This section outlines how the integration must be configured to push user data from ADP Workforce Now to HaloITSM.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / HaloITSM integration.
  5. Enter a name and a description for the integration.
  6. Select both application connector instances from the dropdown menus.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable Onboarding Ticket Management: Select this checkbox to create onboarding tickets in HaloITSM for users provisioned from ADP Workforce Now. When enabled, the Onboarding section is displayed for configuring additional integration settings.
  • Enable Offboarding Ticket Management: Select this checkbox to create offboarding tickets in HaloITSM for users deprovisioned from ADP Workforce Now. When enabled, the Offboarding section is displayed for configuring additional integration settings.
  • Enable Mover Ticket: Select this checkbox to enable mover tickets in HaloITSM. When enabled, the Mover Ticket section is displayed for configuring additional integration settings.
  • Enable Transfer: Select this checkbox to enable transfer tickets in HaloITSM. When enabled, the Transfer Ticket section is displayed for configuring additional integration settings.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insight Settings section is displayed for further configuration.

Onboarding Ticket

The Onboarding Ticket Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Onboarding Ticket Management is enabled in the Integration Scenarios section.

Onboarding User Configuration

Configure the following onboarding user configuration settings:

  • Agent Required: Select this checkbox to assign an agent to the onboarding ticket. When selected, the Agent dropdown list appears, allowing you to select an agent. If this checkbox is not selected, the ticket remains unassigned, and the agent name is shown as Unassigned.
    • Agents: Select the agent to whom the onboarding ticket will be assigned for further action. This field is displayed only when the Assign Agent checkbox is selected.
  • End User Required: Select this checkbox to assign an end user to the onboarding ticket. When selected, the following field is displayed:
    • Mark current user a end user: Select this checkbox to set the request submitter as the end user for the onboarding ticket. 
      • Source Correlation Attribute for End User: Select the ADP Workforce Now attribute to uniquely identify and correlate users between both systems. For example, select Worker ID.
    • End User Details:  Select the end user to whom the onboarding ticket will be assigned for further action. This field is displayed only when the End User Required checkbox is selected.
      Note: If the Source Correlation Attribute for End User checkbox is selected, this field will not appear. 

Onboarding Settings

Configure the following onboarding settings:

  • New Joiner Ticket Action*: Select the action to perform in HaloITSM when the integration receives details of a newly onboarded user from ADP Workforce Now. For example, select Create to create an onboarding ticket in HaloITSM.
  • ITIL Onboarding Ticket Type*: Enter the Jira project key where onboarding tickets should be created.
  • Onboarding Ticket Type: Select the HaloITSM ticket type to use when creating onboarding tickets. For example, HR Onboarding. 
    Note: If the ticket type is changed, the attribute mapper must be refreshed.

Mover Tickets

Configure the following mover settings:

  • Mover Source Action: Select an option to determine when a mover ticket should be created based on updates received from ADP Workforce Now. The following are the available options:
    • All Update: Select this option to create a mover ticket for all users whose records are updated in ADP Workforce Now.
    • Except Division Transfer: Select this option to create mover tickets for all user updates except when the change is a division transfer.
    • Custom: Select this option to define a custom condition to create mover tickets. When enabled, the following field is displayed:
      • Custom Mover Expression: Enter the JavaScript expression to determine when a mover ticket should be created. 
  • Mover Ticket Action: Select the action to perform in HaloITSM when the integration receives updated user details from ADP Workforce Now. For example, select Create to create a mover ticket in HaloITSM.
  • ITIL Mover Ticket Type*: Enter the Jira project key where mover tickets should be created.
  • Mover Ticket Type: Select the HaloITSM ticket type to use when creating mover tickets. For example, Payroll Request. 
    Note: If the ticket type is changed, the attribute mapper must be refreshed.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
