Overview

This guide provides information on the application logs. The Application tab in the Logs module displays the application logs that show all the transaction details for an application.

Tracing

You must enable tracing to view the details of all application method logs. Tracing mode is disabled by default.

Note: Users with roles Readonly or Synchronization Editor cannot view the Enable Tracing button. However, they can view the Application Logs. 
Users with roles Admin, Superadmin, or Owner of the tenant can only enable or disable the tracing mode. 

Perform the following steps to enable the tracing mode:

  1. Log in to the Aquera Admin portal and navigate to Logs.
  2. Click Enable Tracing to launch the Tracing Duration popup window.
  3. Select the duration for which logs will be created. The available options are 10 Minutes and 1 Hour.
    Note: Once this duration is exhausted, only the log entries will be generated but you cannot see the details of those logs.
  4. Click OK to save the changes.

Notes:

  • The options for tracing duration are limited to 10 minutes and 1 hour to ensure that your user data is secured. Aquera does not capture any user details in the logs unless the application-level tracing is enabled.
  • You will not be able to view the details of the log entries that were created when the tracing mode was disabled. If you click on any of those log entries to view the details, the message No Data is displayed.
  • If you want to enable tracing for a customized time duration (for example, 4 hours), contact the Aquera support team.

Application Log Columns

The Application tab displays logs with the following columns:

  • DATE: The date and time when the application log was generated. Example: 12/07/23 13:38:46.
  • EXEC TIME: The time taken to complete the operation. Example: 00:00:01.406.
  • REQUEST ID: The request ID for the operation. Example: 1970.
  • METHOD: The HTTP method used in the operation. Example: GET, POST.
  • DESTINATION: The application (connector) name on which the operation was performed. Example: Dayforce.
  • APP TYPE: Indicates if the application is deployed in the cloud or on-prem. Example: cloud.
  • REQ SOURCE: The source that initiated the operation. Example: Aquera-Console, Postman
  • START INDEX: The starting index to consider for requests. Example: 1
  • TOTAL RESULTS: The total results of the operation. Example: 1
  • PAGE SIZE: The page size of the operation results. Example: 25
  • HTTP STATUS: The HTTP status code. Example: 200
  • RESOURCE: The resource path. Example: /scim/5213a9777210a/e9c8d519-d44b-d6e8-1d8f-7c7d761d3ba9/harvest.greenhouse.io/Users?startIndex=1&count=25

Filtering Logs

You can search for application logs by entering the value for any of the following options in the Filter events search box:

  • applicationType: The type of the application such as cloud or on-prem.
  • destination: The display name of the connector provided by the user on the connector configuration page. 
  • verb: The HTTP method used in the operation. Example: GET, POST.
  • requestSource: The source that initiated the operation. Example: Aquera-Console, Postman
  • requestSourceId: The runId of the integration or the workflow that initiated the operation. You can find the runId in the header of the detailed synchronization logs. When you filter the logs by requestSourceId, all the application logs that were created as part of a particular integration/workflow execution are filtered and displayed. Example: 912520
  • responseCode: The HTTP status code of the operation. Example: 200, 400.
  • executionTime: The time taken to complete the operation as mentioned in the header of the detailed application logs. Example: 1333
  • startIndex: The starting index to consider for requests. Example: 1
  • totalResults: The total results of the operation. Example: 1
  • itemsPerPage: The page size of the operation results. Example: 25
  • requestId: The request ID for the operation. Example: 1970.
  • tenantId: The ID of your tenant.
  • startTime: The start time of the execution of the application operation.
  • endTime: The end time of the execution of the application operation.
  • id: The ID of the application log.
  • applicationId: The ID of the application (connector) created in the Aquera Admin portal. You can find this ID in the browser URL of your application. Example: 7cdb75cc-c3ba-0490-3adb-7595915934d3
  • applicationName: The name of the application (connector). You can find this application name in the browser URL of your application. Example: ad5
  • timestamp: The date and time when the application log was generated.
  • SequenceNumber: The sequence number of the application log.

Note: You can find the values of these options in the header of the detailed application logs.

You can also filter the logs based on the time the application operations were executed. A dropdown at the top of the page allows you to filter logs by time range, including All, 5 minutes, 1 hour, 6 hours, 1 day, or a Custom range. Click Custom to filter the logs within a specific date and time.

The date and time of each log are displayed in the Data column on the page. Click button to choose if you want to display the log time in your local time or UTC. The date and time will be displayed based on the selection.

Attribute Filtering

You can click Switch to attribute filter to use the advanced filtering options. You can set the filter condition to filter the logs using the following options:

  • Exec Time
  • Request ID
  • Method
  • Destination
  • Application Type
  • Request Source
  • Start Index
  • Total Results
  • Page Size
  • HTTP Status
  • User ID*
  • User Name*
  • Request Source ID*

Notes:

  • All attributes marked with a red asterisk (*) are not visible in the column view. You can view them in the detailed logs. Click on a log to view the detailed logs.
  • If the User ID or User Name attribute value is not available in the log detail, the filter will not show the result.
  • You can navigate to the Application Logs page from an application configuration page by clicking the vertical ellipsis icon and clicking Goto Logs. The application logs are then filtered by that specific application destination.
  • After setting a custom filter, if you switch to the basic or advanced filter mode, the selected custom filter dates will be cleared/reset.

Deleting Logs

A user having the tenant Owner, SuperAdmin, or Admin role can delete the application logs. Perform the following steps to delete logs:

  1. Click the delete icon at the top right corner of the page. The following pop-up window is displayed:
  2. Select All to delete all the transaction logs from the page, or select All Transactions for last to delete the transactions for a specific number of days.
  3. If you have selected All Transactions for last, then enter the number of days for which the transactions have to be deleted in the Days field. For example, if you enter 1 in the Days field, the transaction logs in the last 24 hours will be deleted.
  4. Click Delete.

Note: The delete icon will remain disabled until the current deletion process is completed.

Retention Period of Logs

The retention period of application logs is 1 year; the logs will be automatically deleted after a year.
