Entra ID to Awardco Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the integration that retrieves all users or filtered users from Entra ID and uploads the mapped data to Awardco.

To identify user records across both systems, the integration uses the User Principal Name attribute from Entra ID  and matches it to the Corresponding attribute in the Awardco as a correlation attribute.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for details.
    • Awardco: Refer to the Awardco Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Entra ID to Awardco.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID to Awardco Integration.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and Awardco connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Incremental Mode: Select to run the integration in incremental mode.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Logging: Select this checkbox to generate logs outlining data changes
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.

Source Settings

Configure the following source settings:

  • Source Filter: Select the Entra ID filter to filter users while retrieving from Entra ID. The available option is User Principal Name.

Logging

The Logging section enables you to customize the integration settings according to your specific use case. It has the following settings:

Log Settings

Configure the following log settings:

  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

Attribute Map

The Attribute Map tab in Entra ID to Awardco Synchronization defines the default Entra ID attributes that are mapped to the Awardco attributes.
