Overview

Aquera provides a new version for the Script orchestration to add new functionality, enhance the performance of the orchestration, or resolve any issues with the existing version. 

This guide explains the general steps to be followed to upgrade from an existing version of a Script orchestration to its latest version. 

Note: You can upgrade a Script orchestration to only its latest version and not to an intermediate version.

Script Orchestration Upgrade

The following is the general procedure to upgrade from an existing version of a Script orchestration to its latest version:

  1. Login to the Aquera Admin portal.
  2. Navigate to Orchestrations. 
  3. Click Create Orchestration. The Add Orchestration pop-up window is displayed.
  4. Enter a name for the orchestration in the Orchestration Name field, for example, Script_Orch_V3.
  5. Select Script V3 from the Scheduled tab of the pop-up window. 
    Note: Currently, version 3 is the latest version of the Script orchestration.
  6. Click Done. A new script orchestration page is displayed with the fields and tabs to configure the orchestration.
  7. Select the same connectors and use the same variable names that are configured in your existing version of the Script orchestration.
  8. Copy the script manually to the new version of the orchestration.
  9. Save the new orchestration.
  10. If you have configured tables, copy them manually to the new version of the orchestration. Save the new orchestration and reload the orchestration to reflect the new tables in the script.
  11. Execute the orchestration manually. After you verify the logs of the new version of the Script orchestration and confirm that the orchestration is set up correctly, set the schedule of the new orchestration in such a way that it does not overlap with your existing orchestration runs. For example, if your existing orchestration runs with a frequency of 1 hour at 12.00 am, 1.00 am, 2.00 am, and so on, then you set your new orchestration to run at 12.30 am, 1.30 am, 2.30 am, and so on.
  12. After the new orchestration runs multiple times without any issues, set the schedule frequency of your existing orchestration to Never to disable the orchestration run.
