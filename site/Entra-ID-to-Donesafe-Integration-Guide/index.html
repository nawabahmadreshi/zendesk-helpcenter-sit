Entra ID to Donesafe Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any employee created, updated, or terminated in Entra ID is automatically updated in Donesafe based on the defined schedule.

To identify user records across both systems, the integration uses the User Principal Name attribute from Entra ID  and matches it to the Email in Donesafe as a correlation attribute.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for details.
    • Donesafe: Refer to the Donesafe Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to Donesafe.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID to Donesafe Integration.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and Donesafe connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User Lifecycle Management: Select this checkbox to enable user provisioning and perform incremental updates.
    Note: It is mandatory to enable this checkbox to customize the user's lifecycle management.

General Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Full Sync: Select this checkbox to enable full sync.
  • Enable Logging: Select this checkbox to generate logs outlining data changes
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

Manage User

The Manage User section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Manage User Settings

Configure the following manager user settings:

  • Source Filter Attribute: Select the Entra ID filter to filter users while retrieving from Entra ID. The available options are Mail and User Principal Name.
  • Source Filter Value: Enter the value of the selected source filter.
  • Create User Limit: Specify the maximum number of employee records that can be created in the target application during a single integration run.
    Note: If an inactive user is passed from the source application during an integration run, then that user will not be found in the target application and will be ignored.
  • Create Condition: Select a create condition to determine whether a user has to be created in the target application. The available options are UserType(Member), UserType(Guest), and Custom Create Condition. 
  • Custom Create Condition: Enter the JavaScript expression to create a new user in the target application. 
    Note: This field is displayed only when the Custom Create Condition is selected.
  • Update User Limit: Specify the maximum number of employee records that can be updated in the target application during a single integration run.
  • Update Condition: Select the update condition to determine whether a user has to be updated in the target application. The available options are UserType(Member), UserType(Guest), User Company Name Update Condition, User Department Update Condition, and Custom Update Condition. 
  • Custom Update Condition: Enter the JavaScript expression to update a user in the target application. 
    Note: This field is displayed only when the Custom Update Condition is selected.
  • Terminate User Limit: Specify the maximum number of employee records that can be terminated from the target application during a single integration run.
  • Terminate Condition: Select the terminate condition to determine whether a user has to be terminated in the target application. The available options are UserType(Member), UserType(Guest), User Company Name Terminate Condition, User Department Terminate Condition, and Custom Terminate Condition. 
  • Custom Terminate Condition: Enter the custom terminate condition to terminate a user in the target application. 
    Note: This field is displayed only when the Custom Terminate Condition is selected.
  • Import Limit: Specify the maximum number of employee records that can be imported from the source application during a single integration run.
  • Map Manager ID from Donesafe: Select this checkbox to retrieve the user's manager ID from Donesafe when creating and updating a user.
    Notes:
    • When a user is created in Entra ID, the manager IDs in Entra ID and Donesafe may differ, but the email ID remains the same. This checkbox uses the manager's email ID to map the manager ID from Entra ID to Donesafe.
    • If the checkbox is not selected, the integration uses the attribute mapper to verify and map the manager ID.
    • If the attribute mapper does not contain manager details, the user will be created without a manager ID.

Logging

The Logging section enables you to customize the integration settings according to your specific use case. It has the following settings:

Log Settings

Configure the following settings to customize the logs:

  • Select the Log Detail Level*: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter* drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the log.
      • Excluded Attributes: Select the attributes to be excluded from the log.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

Notifications

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

Notification Settings

Configure the following settings for notifications:

  • Notify Admin via Email: Check to send email notifications to all admins. When this checkbox is selected, the Admin Notification Settings section is displayed.
  • Send Email on User Creation: Check to send email notifications when a new user is created. When this checkbox is selected, the User Creation Notification Settings section is displayed.
  • Send Email on User Termination: Check to send email notifications when a user is terminated. When this checkbox is selected, the User Termination Notification Settings section is displayed.

Admin Notification Settings

Configure the following settings for admin notifications:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Send Integration Report via Email: Select to receive email notification with a CSV report for all integration runs. When this checkbox is selected, the Notification Report Settings section is displayed, allowing you to customize the notification report. If the checkbox is not selected, the email is sent without a CSV report attachment.
    Notes:
    • Click User Creation CSV Report to view a sample CSV report that is attached to the email notification when a user is created.
    • Click User Update CSV Report to view a sample CSV report that is attached to the email notification when a user is updated.
    • Click User Termination CSV Report to view a sample CSV report that is attached to the email notification when a user is terminated.
    • Click No Update CSV Report to view a sample CSV report that is attached to the email notification when there is no update.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.

User Creation Notification Settings

Configure the following settings for creation notifications:

  • Notify Employee via Email: Check to send email notifications to the user created.
  • Notify Manager via Email: Check to send email notifications to the user's manager.
  • Notify Distribution List via Email: Check to send email notifications to the distribution list. When this checkbox is selected, the Distribution List's Email Addresses field is displayed.
  • Email Subject: Enter the content that must be used as the email subject for sending email notifications.
  • Email Template: Enter the content that must be used as the email body for all user email notifications.
  • URL Base Path: Enter the URL base path of Donesafe. For example, in the URL https://autosportsbuild.pb1.donesafe.com/admin/settings/users/123, the URL base path is https://autosportsbuild.pb1.donesafe.com/admin/.
  • Distribution List's Email Addresses: Enter the email addresses of the distribution lists to whom the email notifications will be sent.
