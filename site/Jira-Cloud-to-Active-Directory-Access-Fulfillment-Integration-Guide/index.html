Jira Cloud to Active Directory Access Fulfillment Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes how to configure the integration so that a user is assigned to a group in Active Directory by synchronizing the user’s group name from Jira Service Management Cloud.

To identify the matching user records across both systems, the integration reads one of the custom fields in the tickets from Jira Service Management Cloud and matches it against sAMAccountName, userPrincipalName, or employeeNumber in Active Directory.

Key Functions

The Jira Cloud to Active Directory Access Fulfillment integration performs the following functions:

  • Retrieves all the tickets in a specific status that belong to a specific project and work type from Jira Service Management Cloud.
  • Reads the custom user field from each ticket to identify the associated user.
  • Identifies a corresponding user with the matching field in Active Directory, as per your configuration.
  • Reads the group name(s) associated with the user from the Jira ticket's custom field and updates the user's group memberships in Active Directory accordingly, ensuring they receive the appropriate privileges.
  • Transition the ticket to a specific status, based on your configuration.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for details.
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured so that users are automatically added to Active Directory groups and granted access, based on the group name(s) provided in Jira Service Management Cloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Active Directory / Jira Cloud.
  5. Enter a name and a description for the integration.
  6. Select your Active Directory and Jira Service Management Cloud connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenario:

  • Enable Access Fulfillment Management: Select this checkbox to manage access fulfillment. 
    Note: The Access Management section is displayed only when you select this checkbox.

General Settings

Configure the following general settings, based on your requirements:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
    Note: The Log Insights Settings section will be displayed only when this checkbox is selected.

Access Management

Set the configurations related to the access fulfillment.

Note: Make sure that you click the Refresh icon next to each parameter field in the Access Management section to refresh the values populated in those fields. Otherwise, you will see only the values that are available by default in the integration template.

Access Fulfillment Settings

Configure the following access fulfillment settings:

  • Project Name: Select the Jira project from which the tickets need to be retrieved. For example, My Team.
  • Work Type: Select the type of tickets that need to be retrieved. For example, select Task to retrieve only the task tickets.
  • Ticket Pickup Status for Access Fulfillment: Select the status of the tickets that must be retrieved by the integration for processing. For example, select Open to retrieve all the tickets of Open status.

Correlation Field Settings

Configure the following fields to define how users from Jira tickets are matched with user accounts in Active Directory:

  • Source Correlation Attribute: Select the Jira ticket field that must be used for identifying a user. For example, ADCorrelation.
  • Target Correlation Attribute: Select the corresponding target attribute in Active Directory against which the Source Correlation Attribute value will be matched to find the user. For example, userPrincipalName.

Group Settings

Configure the following group settings:

  • Group Names Field: Select the custom field in the Jira ticket that contains the user's group information.

Post Access Fulfillment Settings

Configure the following post-access fulfillment settings:

  • Post Access Fulfillment Ticket Status: Select the status to which a ticket must be transitioned after the user's group assignment is successfully updated in Active Directory. Select None if you do not want the ticket status to be changed.
    Note: The user group assignment is not updated, and the ticket status will not be changed in the following scenarios:
    • No matching user is found in Active Directory
    • Multiple matching user records are found in Active Directory

Log Settings

Configure the following settings to customize the integration logs:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following is the available option:
    • Full: In addition to the count of operations, the logs also display the transaction status, the ticket details, the groups assigned and removed from users, and the ticket transition status.

Report Settings

Configure the following report settings:

  • Enable Integration Report Download: Select this checkbox to allow downloading the integration report as a CSV file.

Notification Settings

Configure the following settings for email notifications:

Email Notification Settings On Successful Integration

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent upon successful integration.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select to receive email notification with a report for all integration runs.
    Note: This option will be displayed only when the Enable Report Generation checkbox is selected under Parameters > General Settings.

Email Notification Settings On Integration Failure

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent in case the integration fails.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.

Attribute Map

The Attribute Map determines which attributes and their values should be updated in Jira when the ticket status changes after the successful assignment of the user to the group. Depending on your workflow, sometimes when you change the ticket status after the users' group assignment is completed, the system prompts you to provide additional details, such as Resolution, Comment, etc. You can update those attributes through the integration.
