The following is the behavior of groups in Synchronization Integration version 3:

  • Only when the checkbox Enable Group Management is selected in the configuration of integration, Aquera's integration will be able to manage the groups in the target application through Attribute Mapping or Group Rules.
  • When the Group Management Restriction List is empty and you have not set any group rules and have not set any attribute mapping for groups, the users will not be added to/removed from groups. However, users will be retained in their existing groups in the target application. [Refer Scenario 1A,1B]
  • When you have set valid group rules, users will be added/retained in the groups specified in the group rules. [Refer Scenario 2,4A,4B]
    Note: The Group Management Restriction List should be left empty when you have set valid group rules. 
  • When the Group Management Restriction List is empty and you are managing groups via attribute mapping, if a group is part of the attribute mapping evaluation, the users are added/retained in that group. However, if a group is not part of the attribute mapping evaluation, the users are removed from that group. [Refer Scenario 5]
  • When the Group Management Restriction List is not empty and you are managing groups via attribute mapping, then if a group is part of the attribute mapping evaluation and the group is mentioned in the restriction list, the users are added/retained in that group. However, if a group is not part of the attribute mapping evaluation and the group is mentioned in the restriction list, the users are removed from that group. [Refer Scenario 6A,6B,6C,6D]
  • When you have set group rules and set groups via attribute mapping, the groups will be managed only by considering the groups in the group rules. [Refer Scenario 7,8A,8B]
  • When the Group Management Restriction List is empty and you have created a group rule which does not have any groups selected in it, then the users will be removed from all the groups. [Refer Scenario 3] 
  • The Group Management Restriction List will not be considered if the groups are managed through Group Rules. However, the Group Management Restriction List will be considered if the groups are managed through Attribute Mapping.
  • If the groups attribute mapping evaluation does not return any result for a particular user's preview, then this user will not be added to/removed from groups. 
  • When the Report Only checkbox is selected and the Report Mode is selected as normal, debug, or verbose, the integration logs will contain the details of the group names to which the users will be added.
  • The group names containing a hyphen, underscore, or space are accepted.

The following is the behavior of groups in Synchronization Integration version 2:

  • Only when the checkbox Enable Group Management is selected in the configuration of an integration, it is possible to manage the groups in the target application.
  • When the Group Management Restriction List is empty and you have not set any group rules and have not set any attribute mapping for groups, the users will be removed from all the groups. [Refer Scenario 1A,1B]
  • When the Group Management Restriction List is empty and you have set valid group rules, users will be added/retained in the groups specified in the group rules and the user will be removed from all other groups. [Refer Scenario 2]
  • When Group Management Restriction List is not empty and you have not set any group rules, users will be removed from all the groups mentioned in the restriction list.  [Refer Scenario 1C]
  • When Group Management Restriction List is not empty and you have set valid group rules, if the group is specified in the group rules and the group is mentioned in the restriction list, the users are added to that group. However, if the group is not specified in the group rules and the group is mentioned in the restriction list, the users are removed from that group. [Refer Scenario 4A,4B] 
  • When the Group Management Restriction List is empty and you are managing groups via attribute mapping, if a group is part of the attribute mapping evaluation, the users are added/retained in that group. However, if a group is not part of the attribute mapping evaluation, the users are removed from that group. [Refer Scenario 5]
  • When the Group Management Restriction List is not empty and you are managing groups via attribute mapping, then if a group is part of the attribute mapping evaluation and the group is mentioned in the restriction list, the users are added/retained in that group. However, if a group is not part of the attribute mapping evaluation and the group is mentioned in the restriction list, the users are removed from that group. [Refer Scenario 6A,6B,6C,6D]
  • When you have set group rules and set groups via attribute mapping, the groups will be managed only by considering the groups in the group rules. [Refer Scenario 7,8A,8B]
  • When the Group Management Restriction List is empty and you have created a group rule which does not have any groups selected in it, then the users will be removed from all the groups. [Refer Scenario 3]
  • The group names containing a hyphen, underscore, or space are not accepted.
