What is Orchestration

When used in the context of system administration, orchestration is defined as the automated configuration, coordination, and management of computer systems and software. As opposed to the term automation which generally means completing a single task or function without human intervention, orchestration arranges tasks to optimize a workflow. Aquera provides a large set of application connectors that facilitate synchronization between identity management providers and popular applications. By themselves, these connectors can greatly improve IT productivity and reliability. Aquera's orchestrations take this functionality to the next level by enabling workflows between two or more applications. Aquera supports two types, or categories of orchestrations, scheduled and event. Both of these can be invoked manually at any time, but scheduled orchestrations facilitate automatic periodic orchestration runs, and event-based orchestrations are invoked automatically when there is a change in a managed source attribute. Event orchestrations require configuration in the application and are initiated by the source application itself outside of the Aquera platform. Not all applications support asynchronous event processing, but all applications with an Aquera connector can be included in a scheduled orchestration.

How to Create Orchestration

  1. Create the two application entries - ADP Workforce Now and Active Directory
  2. Add Orchestration
  3. Enter a name to the Orchestration - test
  4. Apply SCIMToSCIM - An orchestration that keeps two applications fully synchronized
  5. Select Source application - the source of truth and Target - the application that will be updated to stay in sync with the source
    1. The source is prepopulated
    2. The target is client-specific
  6. Then Save Changes
  7. Run Manually
  8. Show report

Orchestration Expressions

NAME : Provide a Name to the Orchestration

ORCHESTRATION TYPE : Choose the Orchestration Type, for Example, SCIMtoSCIM, Scheduled

SOURCE APPLICATION : References the source entry object

TARGET APPLICATION : References the dest entry object

ENT : The expression used to determine which target record matches a given source record.

REPORT ONLY : When selected, a report is generated which shows what would be changed without actually making the changes.
UPDATE LIMIT : The maximum number of records to update in a single run.
ADMIN EMAIL LIST: Distribution list specifying administrative email recipients.
MATCH EXPRESSION : Expression used to determine which target record matches a given source record.
INCREMENTAL MODE : When selected, lastModified is maintained for the source application, and only source records newer than the timestamp will be processed. This operation should be considered when there the number of source records is very large.
IMPORT LIMIT: The maximum number of source records to import. This can be useful while determining/debugging mapping conditions.

SOURCE SCIM FILTER: Optional SCIM filter applied to source import.

CREATE CONDITION: Optional expression used to determine if a given user should be created.

UPDATE CONDITION: Optional expression used to determine if a given user should be updated.

MAP MANAGER: Select to map manager to target.
