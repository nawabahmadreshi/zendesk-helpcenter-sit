ADP iHCM Users to Active Directory Users with ServiceNow Approvals

Overview

The Aquera platform allows your organization to scale complex data management by automating the synchronization of data across resource types. This guide covers the end-to-end process of setting up a script synchronization to sync the Users resource of ADP iHCM to the Users resource of Active Directory based on approvals from ServiceNow.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure following application connectors in the Aquera Admin portal:
    • ADP iHCM: Refer to the ADP iHCM Configuration Guide for details.
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for details.
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details.

Synchronization Setup

This section details how the synchronization must be configured to allow data push from ADP iHCM to Active Directory based on approvals from ServiceNow.

Architecture Overview

When a user is created in ADP iHCM, Aquera checks if the user is already present in Active Directory. If the user is not present, Aquera checks if there is an existing ticket for the user in ServiceNow. The following actions take place based on the presence of a ServiceNow ticket:

  • If there is an existing ticket for the user in ServiceNow, Aquera checks the ticket's status. The user is synced to Active Directory when the ticket is in Resolved status.
  • If a matching ticket is not found in ServiceNow, Aquera creates a ticket for the user in ServiceNow and assigns it to the approver. The user is synced to Active Directory when the ticket is in Resolved status.

Create Script Synchronization

Perform the following steps to create a new script synchronization:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations from the left side panel.
  3. Click Create Synchronization.
  4. Select ADP iHCM Users to Active Directory Users with ServiceNow Approvals.
  5. Enter a name and a description for the synchronization.
  6. Select your ADP iHCM, Active Directory, and ServiceNow connector instances.
  7. Click Continue. A new page is displayed to configure the synchronization.

Run Script Synchronization

You can set event or schedule triggers to run the script synchronization. For more details, refer to the Executing a Synchronization section.

After the script synchronization is successfully executed, Aquera syncs the users created in ADP iHCM as users in Active Directory based on the approval in ServiceNow.
