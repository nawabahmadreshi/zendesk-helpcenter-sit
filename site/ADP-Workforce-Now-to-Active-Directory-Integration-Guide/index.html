ADP Workforce Now to Active Directory Integration Guide

Overview 

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration keeps Active Directory (AD) user accounts in sync with ADP Workforce Now, covering both employees and candidates (pre-hires). The integration securely connects to AD via LDAP, fetches updated worker and candidate records from ADP, and compares them with existing AD accounts. Using the configured matching attributes, it decides whether to create, update, or delete accounts. It can also generate strong passwords, follow flexible username-naming rules, and update only the fields that have changed. 

With configurable filters, correlation attributes, and naming policies, it gives you full control over automating the end-to-end provisioning, updating, and de-provisioning of AD accounts as workers' and candidates' data in ADP change.

Key Functions

The ADP Workforce Now to Active Directory integration provides the following functions:

  • Collects workforce data from ADP by retrieving current employees and candidates (pre-hires), including incremental updates since the last successful run. This ensures the integration always works with the most accurate hiring, status, and profile information.

  • Reads and compares Active Directory users to gather existing AD accounts and attributes, then matches them to ADP records using configurable correlation attributes.

  • Creates, updates, or deletes AD accounts by building the required payloads from ADP data. It generates unique usernames and email addresses based on naming policies, updates only fields that have changed, and safely removes accounts that are no longer needed.

  • Generates secure passwords and manages account states by producing strong, policy-compliant passwords when new accounts are created and setting the correct userAccountControl flags to enable, disable, or require password changes as specified.

The following are the prerequisites:

    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal:

    • ADP Workforce Now: Add and configure the ADP Workforce Now connector in the Aquera Admin portal with the appropriate API credentials. Refer to the ADP Workforce Now Configuration Guide for details on creating the connector and setting up authentication.

    • Active Directory (LDAP): Add and configure the Microsoft Active Directory connector in the Aquera Admin portal. Refer to the Active Directory Configuration Guide for instructions on creating the connector, setting up authentication, setting up a secure LDAP connection, and providing the required service account credentials.

Integration Setup

This section describes the configuration required to set up the ADP Workforce Now to Active Directory integration in the Aquera Admin Portal.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to Active Directory as the integration template.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and your Active Directory connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User LifeCycle Management: Select this option to automatically create, update, and delete Active Directory user accounts based on ADP Workforce Now data. Selecting this option enables the following settings:
    • Select User Type: Select the user type from the drop-down menu. The available options are Candidates and Workers.
      Note: The Manage Candidates section appears under the Configuration tab when Candidates is selected, the Manage Workers section appears when Workers is selected, and both sections appear when both options are selected.
  • Enable Group Assignment: Select this option to enable group assignment functionality for this integration. When enabled, the system allows automatic or manual assignment of users to groups based on defined rules, mappings, or attributes (such as department, role, or region). When disabled, group membership updates and assignments are skipped during user synchronization or provisioning.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this option to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this option to generate logs that outline data changes.
  • Enable Report Generation: Select this option to generate the integration report.
  • Enable Notifications: Select this option to enable notifications related to integration.
  • Enable Log Insights: Select this option to visualize the integration-related metrics.
  • Enable Debug Mode: Select this option to generate logs outlining data changes that would occur for debugging.

Manage Candidates

Configure the following user lifecycle management settings for candidates (pre-hires):

  • Enable ADP Worker Sync: Select to synchronize ADP employee records with Active Directory.
    For example, enabling this creates an AD account such as jane.doe@company.com automatically when Jane is hired in ADP.

    • ADP Worker Correlation Attribute: Select to choose the ADP field used to match employee records with Active Directory users. For example, set this to associateOID so each ADP worker is matched to an AD user whose employeeID equals that associateOID.

    • AD Correlation Attribute for Workers: Select to choose the Active Directory field that must match the selected ADP worker attribute. For example, choose employeeID so the integration pairs ADP’s associateOID with the employeeID field in AD.

    • Enable ADP Candidate Sync: Select to synchronize ADP candidate records with Active Directory.
      For example, enabling this allows the system to create pre-hire accounts like candidate.john@company.com for interview or onboarding access.

  • Fetch Candidates by Application Status: Select to filter candidate records by chosen ADP application statuses when retrieving job applications. For example, selecting| and for Pre-Hire Not Started and Background Check Cleared to pull only candidates who have accepted offers and passed their background checks.

  • Delete Users by Application Status: Select to remove candidate accounts in Active Directory when their ADP application status matches the chosen de-provisioning status. For example, set this to Offer Declined, so candidate accounts are deleted when a candidate declines an offer.

  • Workforce Now Filter: Select to apply a filter to limit which employee records are retrieved during synchronization. For example, use the filter location eq 'Seattle' to sync only employees who work in the Seattle office.

  • Creation Limit: Enter a numeric value to limit the number of new Active Directory accounts created in a single synchronization run. For example, set the limit to 10 so no more than ten new AD accounts are created during a single sync cycle.

Manage Workers

Configure the following user lifecycle management settings for workers:

Correlation and Filters

  • Workforce Now Filter: Select a filter to limit which employee records are retrieved during synchronization. For example, choose Active Workers from the drop-down menu to sync only employees with an active status in ADP Workforce Now.
  • Select Correlation Attribute from ADP: Select the correlation attribute in ADP Workforce Now that is used to match users with Active Directory records as a correlation during synchronization. For example, associateOID.
  • Select Correlation Attribute from Active Directory: Select the corresponding correlation attribute in Active Directory that is used to match users with ADP Workforce Now records as a correlation during synchronization. For example, employeeID.

Create & Update Workers

  • Map Manager For Worker: Select this option to include manager reference.

Naming Policy

Configure the following naming policy settings:

Generate sAMAccountName

  • Enter Your Prompt to Create sAMAccountName: Enter your policy for generating the sAMAccountName. For example, enter the following prompt and click Analyze and Preview:

    Generate three username outputs using the following rule:

    Take the first name and create three variants by using its first 1, 2, and 3 letter, respectively.

    Combine these first-name letters with the truncated last name to form three usernames, inserting the middle initial in between.

    Make sure that the total length does not exceed 10 characters.

    All characters must be in lowercase and contain no spaces.

    Preview: Based on the prompt and the default input (First Name: Alexander, Last Name: Johnson, Middle Name: Michael), the generated sAMAccountName values are amjohnson, almjohnson, and alemjohnso, as displayed under the Preview section. To view the default name values, turn on the Advanced Preview Options toggle.
    Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name, middle name, and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated sAMAccountName values will appear in the Preview section.

Generate Common Name (CN)

  • Generate Common Name (CN) using: Select the drop-down option for generating a common name for the user. The following are the available options:
    • Use sAMAccountName as CN: Select this option to use the sAMAccountName generated from the Generate sAMAccountName section as the common name.
    • Create Policy to Generate CN: Select this option to define a custom policy for generating the Common Name. Selecting this option enables the following settings:
      • Enter Your Prompt to Create a Common Name: Enter your policy for generating the common name. For example, enter the following prompt and click Analyze and Preview:

        Generate a first name where the first character of the first name is capitalized, followed by a space and a last name where the first character is capitalized, followed by numbers up to 2, but the first one should be without a number.

        Preview: Based on the prompt and the default input (First Name: john, Last Name: smith), the generated common name values are John Smith, John Smith1, and John Smith2, as displayed under the Preview section. To view the default name values, turn on the Advanced Preview Options toggle.
        Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated common name values will appear in the Preview section.

Generate UserPrincipalName (UPN)

  • Generate User Principal Name (UPN) using: Select the drop-down option for generating a user principal name for the user. The following are the available options:
    • Use sAMAccountName as UPN: Select this option to use the sAMAccountName generated from the Generate sAMAccountName section as the user principal name.
    • Create Policy to Generate UPN: Select this option to define a custom policy for generating the user principal name. Selecting this option enables the following settings:
      • Enter Your Prompt to Create a UPN: Enter your policy for generating the User Principal Name (UPN). For example, enter the following prompt and click Analyze and Preview:

        Generate three username outputs using the following rule:

        Take the first name and create three variants by using its first 1, 2, and 3 letter, respectively.

        Combine these first-name letters with the truncated last name to form three usernames, inserting the middle initial in between.

        Make sure that the total length does not exceed 10 characters.

        All characters must be in lowercase and contain no spaces.

      • Preview: Based on the prompt and the default input (First Name: Alexander, Last Name: Johnson, Middle Name: Michael), the generated user principal name values are amjohnson, almjohnson, and alemjohnso, as displayed under the Preview section. To view the default name values, turn on the Advanced Preview Options toggle.

      • Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name, middle name, and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated common name values will appear in the Preview section.

    • Active Directory Domain (DNS Name): Enter the primary DNS domain name used for your Active Directory (AD), as well as for UPN and email suffixes. For example, contoso.com.
      Notes:
      • Provide a fully qualified domain name such as contoso.com (without protocols, paths, or a trailing dot).
      • This value is case-insensitive and will be used to
        • Construct userPrincipalName values (for example, jdoe@contoso.com).
        • Normalize email addresses.
        • Infer DC components if no Base DN is provided (e.g., contoso.com → DC=contoso, DC=com).
      • If multiple domains are accepted in your organization, provide the default or primary domain used by most users.
      • This value is not confidential.

        • Construct userPrincipalName values (for example, jdoe@contoso.com).

Password Policy

Configure the following password policy settings:

  • Generate Password: Select this option to automatically create a secure password for each new Active Directory user based on the selected type and length.
    Note: Enabling this ensures each new user receives a randomly generated, policy-compliant password during provisioning.

    • Password Type: Select the desired password complexity level from the drop-down menu. The following are the available options:

      • Basic: Includes uppercase letters, lowercase letters, and digits.
      • Strong: Includes uppercase letters, lowercase letters, digits, and special characters.
      • Advanced: Includes all of the above, with additional checks and exclusion of ambiguous characters.

    • Password Length: Specify the desired length of the generated password. For example, enter 12 to generate a 12-character password instead of the default length of 8.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.

The following tables are available:

  • Active Directory OU Mapping Table: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options: 

    • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
    • Upload CSV: Click Upload CSV File to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
      Notes:
      • The key name within a table should be unique. The key name can include a combination of letters and numbers.
      • The key name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

    Click Save to save the changes made.
