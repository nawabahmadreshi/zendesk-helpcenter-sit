Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating data synchronization from any selected source application to Jira Service Management.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This hook-based integration is triggered only after the user sync integration is completed. The hook-based integration is configured during the setup of the user sync integration.

Note: The source application selected for this hook-based integration must be the same as the source application configured in the user sync integration. For example, if Active Directory is configured as the source application in the user sync integration, then Active Directory should also be selected as the source application in this hook-based integration to perform the automated ticket generation process for Joiner, Mover, or Leaver events in Jira Service Management.

Key Functions

This integration performs the following functions:

  • Automatically creates Jira tickets for Joiner, Mover, and Leaver events.
  • Supports ticket generation based on any user sync integrations between connected applications.
  • Allows configuration of Jira projects, work types, and change request categories.
  • Supports attribute mapping and data transformation using JavaScript expressions.
  • Generates logs and execution details for monitoring and troubleshooting integration activities.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Applications in the Aquera Admin Portal: 
    • Source Connector: Ensure that you add and configure the source application connector in the Aquera Admin portal. 
    • Jira Service Management Connector: Ensure that you add and configure the Jira Service Management connector in the Aquera Admin portal. Refer to the Jira Service Management Cloud Configuration Guide for details. 

Integration Setup

This section outlines how the integration must be configured to create JML tickets in the JIRA Service Management application when the corresponding events are triggered in the selected source application.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Jira Service Management. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Click Continue. The integration is created.

Once created, the following tabs are displayed:

  • Overview
  • Source Application
  • JIRA
  • Execution
  • Notification
  • Schedule

Overview

The Overview tab provides a summary of the integration capabilities and also allows you to enable the ticket management feature.

  • Enable Ticket Management: Turn on the toggle button to automatically generate JML tickets in the Jira Service Management application when the corresponding events are triggered in the selected source application.

Source Application

This tab allows you to search and select the source application connector to be used for this integration.

  • Select your Source Application Instance: Select the source application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

JIRA

This tab allows you to search and select the target application connector to be used for this integration. It also enables you to configure the settings that determine when JML tickets are created in JIRA Service Management.

  • JIRA Settings
  • Ticket Generation

JIRA Settings

The JIRA Settings section is for configuring the required JIRA application instance. The following fields are displayed in this tab:

  • Select JIRA Application: Select the JIRA Service Management application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Enable Ticket Creation Trigger Mode: Turn on this toggle button to configure how Jira tickets are created and processed for the selected source application using the available execution modes. When enabled, the following field is displayed:
    • Select the Mode: Select the mode from the dropdown menu. Currently, only the Hook-Based Ticket Creation is displayed.

Ticket Generation

The Ticket Generation section allows you to define the Jira project, work type, and event-specific settings required for ticket creation workflows. The following fields are displayed in this section:

If the List View toggle button is disabled, the following parameters are displayed:

  • Select JML Type: Select the employee lifecycle event type from the dropdown menu to which it should trigger Jira ticket creation. If you do not find the required event type in the list, click the refresh icon next to the field to reload the available event types.
    The following event types are available in the dropdown menu:
    • Joiner: Select Joiner to generate tickets only for users who have recently joined the organization.
    • Mover: Select Mover to generate tickets only for users whose role, department, or access has changed.
    • Leaver: Select Leaver to generate tickets only for users who have left the organization.
  • JIRA Space: Select the Jira project space where tickets for the configured JML events will be created and managed. If you do not find the required JIRA space in the list, click the refresh icon next to the field to reload the available JIRA space.
  • JIRA Work Type: Select the JIRA work type that should be used when creating tickets for the configured JML workflows.

  • Select change type for Mover: Select the change types that should trigger Jira change requests for Mover events, such as department updates or employee detail changes.
    Note: This parameter is displayed only if you have selected the Mover option in the Select JML Type parameter.

    Attribute Map

    This section allows you to control how each attribute is synchronized between the Source and JIRA Service Management.
    Source attributes are listed on the left side and the target attributes are listed on the right side of the page. You can modify only the source attributes as needed, to align with your specific use case.
    JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
    Attributes marked with a red asterisk (*) are required for successful provisioning.
    The Search Attributes field allows you to filter the list of available attributes for mapping.
    After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Execution

The Execution tab allows you to configure advanced and operational settings before executing the integration. The following sections are displayed in this tab:

  • Advanced settings
  • Operational Settings

Advanced Settings

The Advanced Settings section allows you to test and validate the integration before running it for all users.

  • Report Only Mode: Turn on this toggle button to generate a report detailing the data changes that would happen, but without applying them.

Operational Settings

The Operational Settings section allows you to configure logging, reporting, and log insights for the integration. These settings help you review integration activity, generate reports, and view dashboard insights for integration runs. It has the following settings:

  • Logging Settings
  • Report Settings
  • Log Insights Settings

Logging Settings

This section enables you to customize the log generation settings. It has the following settings:

  • Select Log Type: Select the log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Log Level: Select the level of detail to include in logs. The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Select this if you want to choose attributes to be included in logs. When selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Select this if you want to choose attributes to be excluded from the logs. When selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.

Notifications

This tab allows you to configure email notifications for integration events, such as Joiner, Mover, or Leaver, and to define the recipients and report details included in the notification emails.

If the List View toggle button is disabled, the following parameters are displayed:

  • Template Name: Enter a name for the notification template.
  • Select Application: Select applications for notification configuration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available application connectors.
  • Select Event for Notification: Select the event that will trigger a notification. For example, select Joiner Ticket Created to get email notifications every time when a user is created in the organization.
    The available options are:
    • Joiner Ticket Created: Select this option if you want to create a joiner ticket in Jira Service Management only for users who have recently joined the selected source application.
    • Mover Ticket Created: Select this option if you want to create a mover ticket in Jira Service Management only for users whose role, department, or access has changed in the selected source application.
    • Leaver Ticket Created: Select this option if you want to create a leaver ticket in Jira Service Management only for users who have recently left the selected source application.
  • Notification Delivery Channel: Select the channel to be used for sending notifications. Currently, only Email is supported. When selected, the following field is displayed:
  • Email: Select this option if you want to send notifications through email. When selected, the following fields are displayed for further configuration:
    • Custom Notification Recipients: Enter the email addresses of the recipients to whom the email notifications will be sent. By default, the notifications are sent to the user's work email. When selected, the following parameters are displayed for further configuration:
      • Custom Email Address: Enter the email IDs of the users to whom you want to send email notifications.
      • Subject: Enter the content that must be used as the email subject.
  • Attach Report: Select this checkbox to attach a CSV report of the integration along with the notification.

To add another configuration, click the + icon adjacent to the List View toggle button.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
