ADP <-> Procore - Users and Timesheets Sync Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the process for setting up an integration to fetch employee timesheet data from Procore and synchronize it with ADP Workforce Now, as well as user data from ADP Workforce Now to Procore.

To identify user records across both systems, the integration uses the employeeID attribute from Procore and matches it to either badgeID, positionID, or workerID in ADP Workforce Now as a correlation attribute for the timesheet synchronization. On the other hand, for the user lifecycle management synchronization, the integration uses either badgeID, positionID, or workerID from ADP Workforce Now and matches it to the employeeID attribute in Procore as the correlation attribute.

Key Functions

The following are the key functions of the integration:

  • Procore to ADP Workforce Now Timesheet Synchronization Integration:
    • Retrieves user's timesheet data from Procore within a specified timeframe.
    • Syncs newly added or updated timesheet records from Procore to ADP Workforce Now.
    • Supports both full and incremental sync modes to ensure only the relevant timesheet data is transferred.
    • Generates a CSV report of the synced timesheets based on the integration configuration, enabling visibility into data changes pushed to ADP.
    • Supports filtering by project name(s), cost code(s), or task code(s), as well as labor charge fields, to target specific timesheet data during synchronization.
    • Filters timesheet data retrieved from Procore based on a defined source filter and its corresponding value to limit the synchronization scope.
  • ADP Workforce Now to Procore User Synchronization Integration:
    • Retrieves worker data from ADP Workforce Now within a specified timeframe.
    • Syncs newly added or updated worker records from ADP Workforce Now to people in Procore resource planning.
    • Supports both full and incremental sync modes to ensure only the relevant worker data is transferred.
    • Generates a CSV report of the synced workers based on the integration configuration, enabling visibility into data changes pushed to Procore.
    • Allows configuration of ADP Workforce Now statuses to be treated as inactive in Procore, ensuring appropriate user deactivation or termination.
    • Filters worker records retrieved from ADP based on a defined source filter and its corresponding value to limit the synchronization scope.
    • Allows selective inclusion of specific PII attributes from ADP in integration reports.

Integration Setup

This section outlines the configuration required to integrate Procore and ADP Workforce Now, enabling the seamless transfer of timesheet data from Procore to ADP Workforce Now and user data from ADP Workforce Now to Procore.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Procore to ADP Timesheets sync.
  5. Enter a name and a description for the integration.
  6. Select your Procore and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors used in the integration. It allows you to search for and update these application connectors.

The applications shown in this section depend on the type of synchronization the integration supports:

Use Case Source Application Target Application
Timesheet Sync Procore ADP Workforce Now
User Lifecycle Management Sync ADP Workforce Now Procore

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenario(s)

Configure the following integration scenario(s):

  • Select the Integration Module(s): Select one or more integration modules from the dropdown list as per your use case. The following are the available options:

    • User Lifecycle Management: Select this checkbox to sync ADP Workforce Now workers with people in Procore Resource Planning.
      Note: The Manage Users section will be displayed only when this checkbox is selected.

    • Timesheet Synchronization: Select this checkbox to enable timesheet sync from Procore to ADP.
      Note: The Manage Timesheets section will be displayed only when this checkbox is selected.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Incremental Mode: Select this checkbox to run the integration in incremental mode.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Report Generation: Select this checkbox to generate the integration report.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values when choosing the Full Details option.

Report Settings

Configure the following report generation settings:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Enable Integration Report Download: Select this checkbox to allow downloading the integration report in CSV format.

Notification Settings

Configure the following settings for email notifications:

  • Recipients' Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
  • Email Subject: Enter the content that must be used as the email subject.
  • Email Body: Enter the content that must be used as the email body for the email notifications.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.

Analytics Settings

Configure the following analytics settings:

  • Select the Analytics Detail Level: Select the desired level of detail from the dropdown menu on the log insights dashboard. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Manage Users

Configure the following User Lifecycle Management Settings:

  • Enable ADP Workers to People in Procore Resource Planning Sync: Select this checkbox to sync ADP Workforce Now workers with people in Procore Resource Planning.
  • Select Source Field for Correlation: Select the source field for correlation in User Lifecycle Management from the dropdown menu. The available options are Position ID in ADP, Associate ID in ADP, and Badge in ADP under Time and Attendance in Employment Profile.
  • Select Target Field for Correlation: Select the target field for correlation in User Lifecycle Management from the dropdown menu. The available option is Employee ID in Procore.
  • Select ADP Statuses to Be Considered as Active False: Select the list of statuses from ADP Workforce Now that are to be considered false for deactivating or terminating the user in Procore. For example, if you select  Terminated, Retired from the dropdown, the users with Terminated and Retired statuses will be regarded as inactive in Procore.
  • Select the Source Filter: Select the ADP worker filter to filter the workers while retrieving from ADP Workforce Now. The available options are Filter by Position ID in ADP and Filter by Associate ID in ADP.
  • Enter the Source Filter Value: Provide the value for the selected source filter. For example, UZY000133.
  • Select the Create Condition: Select the create condition to determine whether a user should be created in the target application. The available options are Create Only Active Users from ADP and Create All Users from ADP.
  • Select the Update Condition: Select the update condition to determine whether a user must be updated in the target application. The available options are Update Only Active Records from ADP, Update Only Terminated Records from ADP, Update All Records from ADP, and Custom Update Condition.
  • Import Limit: Specify the maximum number of user records that can be imported into the target application during a single run of the integration.
  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single run of the integration.
  • Select the PII Attributes from ADP: Select the Personally Identifiable Information (PII) attributes from ADP Workforce Now to include in the report.

Manage Timesheets

Configure the following Timesheet Synchronization Settings:

  • Enable Timesheet Sync from Procore to ADP: Select this checkbox to enable timesheet sync from Procore to ADP.
  • Select the Source Field for Correlation: Select the source field for correlating in timesheet synchronization. The available option is Employee ID in Procore.
  • Select the Target Field for Correlation: Select the target field for correlation in timesheet sync. The available options are Position ID in ADP, Associate ID in ADP, and Badge in ADP under Time and Attendance in Employment Profile.
  • List Labor Charge Fields: Provide the list of labor charge fields in ADP Workforce Now to sync with the ADP Timecards. For Example, Work Class, Job.
  • List Procore Project Name(s): Enter the full name of the Procore projects for the timecard sync. For example, TEST - Productive Project, TEST - Sandbox Project.
  • List Procore Cost Code(s) or Task Code(s): Enter the Procore cost codes or task codes for the timesheet sync. For example, 05-4000-Framing Interior, 09-2500-drywall hanger.
  • Select the Source Filter: Select the source filter to filter the users’ timesheet while retrieving from Procore. The available options are Filter by Employee ID in Procore and Filter by TimeCard Status in Procore.
  • Enter the Source Filter Value: Provide the value for the selected source filter. For example, 000000133.
  • Select the Target Filter: Select the target filter to filter the ADP Workforce Now workers. The available options are Filter by Position ID in ADP and Filter by Associate ID in ADP.
  • Enter the Target Filter Value: Provide the value for the selected target filter. For example, UZY000133. 
  • Select the Date to Filter Timesheets from Procore: Select a filter when you are using the source.EmployeeFilter for a single user, or when running the full sync for the first time. The available options are Start Date and Updated Date.
  • Enter Date/DateTime Value for the Selected Date Filter: Enter the start date in YYYY-MM-DD format when StartDate is the selected filter, and use the YYYY-MM-DD UTC format when UpdatedDate is the selected filter.
  • Select the Create Condition(s): Select the create condition(s) to determine whether a timesheet should be created in the target application. The available options are Create Approved Timecard from Procore, Create Pending Timecard from Procore, Create Completed Timecard from Procore, and Create Reviewed Timecard from Procore.
  • Select the Update Condition(s): Select the update condition(s) to determine whether a timesheet must be updated in the target application. The available options are Update Approved Timecard from Procore, Update Pending Timecard from Procore, Update Completed Timecard from Procore, and Update Reviewed Timecard from Procore.
  • Import Limit: Specify the maximum number of timesheets that can be imported into the target application during a single integration run.
  • Create Limit: Specify the maximum number of timesheets that can be created in the target application during a single integration run.
  • Update Limit: Specify the maximum number of timesheets that can be updated in the target application during a single integration run.

Attribute Map

The Procore to ADP Timesheets sync integration maps the Procore and ADP Workforce Now attributes.

The attributes displayed in this section depend on the type of synchronization the integration supports:

  • TimeCard Mapper: For timesheet synchronization, the integration uses Procore attributes as the source and maps them to the corresponding ADP Workforce Now attributes as the target.
  • People Mapper: For user synchronization, the integration uses ADP Workforce Now attributes as the source and maps them to the appropriate Procore attributes as the target.

Only the source attributes can be modified, as needed, to align with your specific use case.
