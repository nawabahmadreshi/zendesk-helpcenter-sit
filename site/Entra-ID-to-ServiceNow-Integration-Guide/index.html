Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Paylocity to Brivo integration, which synchronizes user data from Entra ID to ServiceNow to support user lifecycle and group management.

To identify user records across both systems, the integration uses the Mail attribute from Entra ID and matches it to the Email in ServiceNow as a correlation attribute.

Key Functions

The Entra ID to ServiceNow integrations provide the following functions:

  • User Lifecycle Management: Maps users from Entra ID to ServiceNow and creates, updates, or deactivates users based on their activity status.
  • Onboarding Tickets: Creates onboarding tickets for new Entra ID users who need to be added to ServiceNow.
  • Offboarding Tickets: Creates offboarding tickets for existing users in both Entra ID and ServiceNow.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for details.
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Paylocity to Brivo.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID to ServiceNow Integration.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and ServiceNow connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User Lifecycle Management: Select to enable user provisioning and perform incremental updates.
  • Enable Onboarding Ticket Management: Select to enable onboarding ticket from Entra ID to ServiceNow.
  • Enable Offboarding Ticket Management: Select to enable offboarding ticket from Entra ID to ServiceNow.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize the integration-related metrics.

User Management

The User Management section enables you to customize the user integration settings according to your specific use case. It has the following settings:

User Provisioning Settings

Configure the following user provisioning settings:

  • Source Correlation Attribute: Select the Entra ID attribute for correlation from the dropdown menu. The available option is Mail.
  • Target Correlation Attribute: Select the ServiceNow attribute for correlation from the dropdown menu. The available option is Email.
  • Enable Source Filter: Select to enable the source filter.
    • Source Filter: Select the Entra ID attribute to filter the users while retrieving from Entra ID. The available options are Mail and User Principal Name.
    • Source Filter Value: Provide the value for the selected source filter.
  • Create User: Select to create a user from Entra ID to ServiceNow.
  • Update User: Select to update a user from Entra ID to ServiceNow
  • Deactivate User: Select to deactivate a user from Entra ID to ServiceNow
  • Enable Incremental Mode: Select to run the integration in incremental mode.
  • Sync User Manager: Select to assign the user's Entra manager as the ServiceNow manager.
  • Microsoft Entra ID application Id for extension properties:

User Provisioning Threshold Settings

Configure the following provisioning threshold settings:

  • Import Limit: Specify the maximum number of user records that can be imported in the target application during a single integration run.
  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single integration run.
  • Deactivate Limit: Specify the maximum number of user records that can be deactivated from the target application during a single integration run.

Create User Settings

Configure the following create user settings:

  • Create Condition: Select a condition from the dropdown menu to determine when users are created.
    • Custom Create Condition: Enter the JavaScript expression to create a new user in the target application. 

Update User Settings

Configure the following update user settings:

  • Update Condition: Select a condition from the dropdown menu to determine when users are updated.
    • Custom Update Condition: Enter the JavaScript expression to update a user in the target application.
