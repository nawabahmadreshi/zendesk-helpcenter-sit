ADP Workforce Now Workers to Microsoft 365 Users

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

The integration retrieves employee data from ADP Workforce Now, matches and processes corresponding user accounts in Microsoft 365. It automatically identifies all employees who are no longer active in ADP Workforce Now, due to termination, retirement, or other status changes and disables their associated user accounts in Microsoft 365.

The process also handles new hires, updates user information when employee details change, and reactivates accounts if employees return to active status, ensuring that Microsoft 365 always reflects the current workforce as managed in ADP Workforce Now.  The integration supports both simulation (report-only) and full execution modes.

This integration performs the following tasks:

  • Scans all workers in ADP Workforce Now and all users in Microsoft 365 to identify differences and required actions.
  • Keeps employee records consistent between ADP Workforce Now and Microsoft 365 by automatically synchronizing data.
  • Detects new hires in ADP and creates corresponding Microsoft 365 accounts for them, including assigning licenses and generating passwords.
  • Updates Microsoft 365 user accounts when employee information (like name, job title, department, manager, contact details) changes in ADP.
  • Suspends (disables) Microsoft 365 accounts when employees are terminated, retired, or otherwise marked inactive in ADP.
  • Reactivates Microsoft 365 accounts if an employee is marked active again in ADP.
  • Removes or reassigns Microsoft 365 licenses when accounts are disabled or reactivated.
  • Sends summary reports and optional welcome emails to administrators, new users, or their managers after each sync.
  • Provides a simulation mode to preview all proposed changes without making any actual updates, supporting safe review before live runs.
  • Handles errors gracefully, offering clear explanations and guidance for issues like permission problems, data conflicts, or external service failures

Integration Setup

This section outlines the configuration and execution process for integrating ADP Workforce Now with Microsoft 365. It describes how to create the integration, define operational parameters, and establish reporting and notification workflows to integrate the ADP Workforce Now and Microsoft 365.

Create Integration

Perform the following steps to create a new script integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to integrations from the left side panel.
  3. Click Create integration.
  4. Select ADP Workforce Now Workers to Microsoft 365 Users.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Microsoft 365 connector instances.
  7. Click Continue. A new page is displayed to configure the integration.

Set Up Integration Parameters

The following are the preferences you can set for the ADP Workforce Now Workers to Microsoft 365 Users integration:

  1. Default Usage Location: Country or region code for assigning Microsoft 365 services to new users (e.g., "US").
  2. Recipients Email Addresses:Email address for receiving integration reports and notifications.
  3. Administrator Name: Name used for personalizing report and notification messages.
  4. Worker Filter Condition: Filter expression for selecting ADP workers included in integration.
  5. Maximum Records to Import: Maximum number of ADP records processed in a single synchronization; 0 for no limit.
  6. Update Microsoft 365 Username on Update: Option for updating the Microsoft 365 username (userPrincipalName) when employee details change in ADP.
  7. Send Welcome Email to User’s Personal Address: Option for sending a welcome email to the user’s personal email address upon account creation.
  8. Sort Report By: Column used for sorting report data, such as Operation, Worker Name, or Status.
  9. Sort Order: Sorting order for report data, either ascending or descending.
