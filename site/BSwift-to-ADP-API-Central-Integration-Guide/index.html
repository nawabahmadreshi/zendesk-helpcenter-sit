Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the Bswift file connector integration for mapping employee benefits deductions to ADP API Central. It processes CSV files containing employee ID, deduction code, rates, goals, and other details via FTP and pushes them to the ADP Workforce Now API Central..

Key Functions

The BSwift to ADP API Central integrations provide the following functions:

  • Reads the benefits CSV file from the Bswift FTP server.

  • Maps employee ID, deduction code, and payroll data to ADP format.

The following are the prerequisites:

Integration Setup

This section outlines how the integration must be configured to push employee data from BSwift to ADP Workforce Now for ADP API Central Customers.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select BSwift to ADP Workforce Now for ADP API Central Customers.
  5. Enter a name and a description for the integration.
  6. Select your BSwift and ADP Workforce Now for ADP API Central Customers connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

  • ADP API Central reads employee data from ADP Workforce Now and writes benefits deductions back to it
  • Write Demographic uploads ADP employee data as CSV to BSwift for benefits enrollment.
  • Read Benefits downloads BSwift CSV to update ADP deductions (start/change/stop).
  • Read Demographic Headers retrieves CSV column names to validate the BSwift demographic format

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenario settings:

  • Enable Election Benefits Sync: Select to enable election benefits (deduction) sync from Bswift to ADP API Central Customers.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Log Insights: Select to visualize the integration-related metrics.
  • Enable Notifications: Select to enable the integration-related notifications.

Benefits Sync Settings

The Benefit Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Available CSV Date Formats: Enter the date format in uppercase letters only. For example, DD-MM-YYYY or D/M/YY.
  • Start Benefits Sync: Select to enable synchronization of new benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, benefit records marked as start events in the BSwift CSV file are created in ADP.
  • Stop Benefits Sync: Select to enable synchronization of ended or stopped benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, benefit records marked as stop events in the BSwift CSV file are ended in ADP.
  • Change Benefits Sync: Select to enable synchronization of changes to existing benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, modified benefit records in the BSwift CSV file are updated in ADP.
  • Import Limit: Specifies the maximum number of benefit election records (CSV rows) that can be processed from the BSwift file in a single integration run. When this limit is reached, any additional benefit records in the file are skipped for that run.
