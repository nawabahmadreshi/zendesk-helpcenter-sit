Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize employees from Namely to Entra ID. It also provides options to configure rules for assigning users to groups and for assigning licenses to users. 

Key Functions

The Entra ID / Namely integration performs the following functions:

  • Retrieves the data of employees from Namely and creates them as users in Entra ID.
    • Generates a unique UPN, email address, and password for each user, depending on the configuration.
    • Assigns users to groups in Entra ID according to the configured rules.
    • Assigns a license to users based on the configured rules.
  •  
    • Full synchronization is always performed for the candidate data. For worker data, you can configure the integration to run either a full or incremental sync.

    • You can set the CSV file location and the CSV file name in the integration settings.

      Note: The SFTP server name is configured in the FTP Generic connector in the Aquera Admin Portal.

  • Writes back the worker email, work phone number, and any custom attributes from a CSV file to ADP Workforce Now, based on the integration settings.
    • You can configure the CSV file location and the CSV file name from which the integration must read the data to be written back to ADP.
  • Generates reports and logs, and sends notification emails based on the integration settings.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for configuration details.
    • FTP Generic: Refer to the FTP Generic Configuration Guide for configuration details.
  • Prepare CSV File with the Appropriate Headers: Make sure that a CSV file with the appropriate column headers is created and placed in the SFTP location, which you will configure in the integration settings. This is necessary for populating the CSV header column names on the attribute mapping pages for candidate and worker sync.

Integration Setup

This section outlines how the integration must be configured to synchronize Workflow iForms and Person iForms from iCIMS to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / SFTP.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and FTP Generic connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the ADP Workforce Now and FTP Generic application connectors that you have selected for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and settings according to your use case. It has the following settings:

Integration Scenarios

Enable the following features based on your requirements:

  • Enable workers sync: Select this checkbox to sync worker data from ADP Workforce Now to the SFTP server. When selected, the Worker Sync Settings section and the Worker to SFTP attribute map setting section are displayed for further configuration. If this checkbox is not selected, the worker data will not be retrieved and processed by the integration.
  • Enable candidate management: Select this checkbox to sync candidate data from ADP Workforce Now to the SFTP server. When selected, the Candidate Management section and the Candidate to SFTP attribute map setting section are displayed for further configuration. If this checkbox is not selected, the candidate data will not be retrieved and processed by the integration.
  • Enable writeback: Select this checkbox to enable the integration to write back the worker email, work phone number, and any custom attributes from a CSV file in the SFTP server to ADP Workforce Now. When selected, the Writeback Settings section and the Writeback Map attribute map setting section are displayed for further configuration.

General Settings

Configure the following general settings:

  • SFTP Write File Path: Specify the folder path in the SFTP server where the integration must create the CSV file containing the worker data and candidate data. For example, Aquera/ADPData/dataexport.csv
    Note: The SFTP Server name is configured on the FTP Generic connector configuration page.
  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When selected, the Log Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When selected, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When selected, the Log Insights section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When selected, the Notifications section is displayed for further configuration.

Synchronization Settings

This section enables you to configure the integration to run in full sync or incremental sync mode.

  • Enable incremental mode: Select this checkbox to run the integration in incremental mode. When selected, the integration retrieves and processes only the worker data that have been added or modified in ADP since the last integration execution. When this checkbox is not selected, the integration performs a full sync and processes all worker records during each run.
    Note: Candidate data is always processed using a full sync, regardless of this setting.
  • Full sync frequency: The frequency by which the integration must run in full sync mode when the incremental mode is enabled. For example, if the integration is scheduled to run every 6 hours and the Full sync frequency is set to daily, the integration will run in incremental mode every 6 hours and perform a full sync once every day.
    Note: This setting is displayed only if you have selected the Enable incremental mode checkbox.

Worker Sync Settings

This section details the following settings for synchronizing the worker data.

  • Source filter: This setting enables you to filter worker record(s) from ADP Workforce Now and sync only the filtered worker records to the SFTP server. You can filter by Associate ID, Position ID, or select Custom to define a JavaScript-based filter expression. Select None to apply no filtering and include all worker records.
  • Source filter value: Enter the Associate ID or the Position ID to filter a worker record, based on your selection in Source Filter. This field is displayed only if you have selected Filter by Associate ID or Filter by Position ID in Source Filter.
  • Custom source filter expression: Provide the JavaScript expression to filter the worker records from ADP. This field is displayed only if you have selected Custom in Source Filter.
  • Enable import limit: Select Yes to limit the number of worker records processed in a single integration run. Select No if you do not want to set a limit.
  • Import Limit: Enter the maximum number of worker records that are allowed to be processed by the integration in a single run. This setting is displayed only if you have configured Yes in Enable import limit.

Candidate Management

This section details the setting for synchronizing the candidate data.

  • Include candidates with application status: Select the application statuses of candidates in ADP Workforce Now that must be included in the sync to the SFTP server.

Writeback Settings

This section details the following settings for writing back data from the SFTP server to ADP Workforce Now:

  • SFTP read file path: Specify the folder path in the SFTP server from where the integration must read the CSV file containing the worker data for writeback to ADP Workforce Now. For example, Aquera/ADPWritebackData/workerdata.csv
    The work email, work phone number, and any custom attributes (configured in the Writeback attribute map) will be written back to ADP. 
    Note: The SFTP Server name is configured on the FTP Generic connector configuration page.
  • Writeback correlation header: Select the CSV column name that contains the unique identifier for each worker record. This column value will be used by the integration to identify the matching worker record in ADP. 
  • Writeback correlation attribute: Select the corresponding worker attribute in ADP Workforce Now that will be used to match against the selected CSV column name to identify the correct worker record. 

Attribute Map

The Attribute Map tab contains settings to configure the following mappings:

  • ADP Workforce Now worker attributes to CSV columns for worker sync
  • ADP Workforce Now candidate attributes to CSV columns for candidate sync
  • CSV columns to  ADP Workforce Now worker attributes for writebacks 

Worker to SFTP

The Worker to SFTP section enables you to map ADP Workforce Now worker attributes to the columns in the CSV file within the SFTP server. This section is displayed only if you have selected Enable workers sync.

Note: For the CSV column header names to be populated in the Target Attribute section of the attribute map, make sure that the CSV file containing header rows is already placed in the SFTP location.

Candidate to SFTP

The Candidate to SFTP section enables you to map ADP Workforce Now candidate attributes to the columns in the CSV file within the SFTP server. This section is displayed only if you have selected Enable candidate management.

Note: For the CSV column header names to be populated in the Target Attribute section of the attribute map, make sure that the CSV file containing header rows is already placed in the SFTP location.

Writeback Map

The Writeback Map section enables you to map CSV columns to the ADP Workforce Now worker attributes. This section is displayed only if you have selected Enable writeback.

Mapping can be set for work email, work phone number, and all custom attributes in ADP Workforce Now. 

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable logging parameter in the Parameters section.

  • Select Log Type: Select a log type. This will decide the level of detail to be included in logs. For example, choosing Employee Number with All Data includes employee number, operations, field names, and all values (PII and non-PII).

  •  Select Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the log.
    • Include List: Select this option to include specific attributes in the log.
      • Custom Inclusion List: Enter the attributes to be included in the log. This parameter is available only if you select Include List in the Select Log Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the log.
      • Custom Exclusion list: Enter the attributes to be excluded from the log. This parameter is available only if you select Exclude List in the Select Log Level parameter.
  • Enter PII fields: Enter the fields that contain personally identifiable information (PII) to be included in logs. 
  • Enter GUID fields: Enter the fields that contain globally unique identifiers to ensure they are handled correctly in logs.
  • Enter EmployeeId fields: Enter the fields that represent employee identification values to be displayed consistently in logs. For example, adding associateId ensures the employee identifier is included in logs.

  • Select Report Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the report.
    • Include List: Select this option to include specific attributes in the report.
      • Custom Inclusion List: Enter the attributes to be included in the report. This parameter is available only if you select Include List in the Select Report Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the report.
      • Custom Exclusion List: Enter the attributes to be excluded from the report. This parameter is available only if you select Exclude List in the Select Report Level parameter.
  • Enter PII fields: Enter the fields that contain personally identifiable information (PII) to be included in reports. 
  • Enter GUID fields: Enter the fields that contain globally unique identifiers to ensure they are handled correctly in reports.
  • Enter EmployeeId fields: Enter the fields that represent employee identification values to be displayed consistently in logs. For example, adding associateId ensures the employee identifier is included in reports.

Notifications

This section enables you to customize the notification settings. It has the following settings:

Note: This section appears only if you have enabled the Enable notifications setting in the Parameters section.

Notification Template Settings

Configure the following settings to customize notifications:

  • Send email notifications for success: Select this checkbox to send an email notification on successful integration runs. When selected, the Success Execution section is displayed for further configuration.
  • Send email notifications for failures: Select this checkbox to send an email notification on failed syncs in an integration run. When enabled, the Failure Execution section is displayed for further configuration.
  • Attach the report to email: Select this checkbox to receive an email notification with an attached report for each integration execution.

Success Execution

Configure the following notification settings for successful integration runs:

  • Send To: Enter the email addresses of the admins to whom the email notifications will be sent upon successful integration runs.
  • Cc: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the subject in email notifications.
  • Email Body: Enter the content that must be used as the email body for notifications sent upon successful sync.

Failure Execution

Configure the following settings for failed syncs:

  • Send To: Enter the email addresses of the admins to whom the email notifications will be sent upon failed syncs.
  • Cc: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the subject in email notifications.
  • Email Body: Enter the content that must be used as the email body for notifications sent upon failed form syncs.

Notification Report Settings

Configure the following email notification report settings:

  • Select Report Type: Select a notification report type. This will decide the level of detail to be included in the report in the email notifications.
    The following are the available options:

    • Minimum: Select this option to include only operation details.
    • Metadata: Select this option to include operation details and attribute names.
      Note: You may ignore the other options in the drop-down list.
  • Select Report Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the notification report.
    • Include List: Select this option to include specific attributes in the notification report.
      • Custom Inclusion List: Enter the attributes to be included in the notification report. This parameter is available only if you select Include List in the Select Report Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the notification report.
      • Custom Exclusion List: Enter the attributes to be excluded from the notification report. This parameter is available only if you select Exclude List in the Select Report Level parameter.

Note: This section appears only if you have enabled the Enable Log Insights setting in the Parameters section.

  • Enter Dashboard Name: Enter the name of the Log Insights dashboard that must be created after the integration run.

  • Select analytics type: Select an analytics type. This will decide the level of detail to be included in the dashboard.
    The following are the available options:

    • Minimum: Select this option to include only operation details.
    • Metadata: Select this option to include operation details and attribute names.
      Note: You may ignore the other options in the drop-down list.
  • Select analytics Level: Select the Log Insights detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the Log Insights data.
    • Include List: Select this option to include specific attributes in the Log Insights.
      • Included Attributes: Enter the attributes to be included in the Log Insights. This parameter is available only if you select Include List in the Select analytics Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the Log Insights.
      • Excluded Attributes: Enter the attributes to be excluded from the Log Insights. This parameter is available only if you select Exclude List in the Select analytics Level parameter.
