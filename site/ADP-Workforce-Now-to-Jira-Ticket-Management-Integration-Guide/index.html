Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the user's lifecycle from ADP Workforce Now to Jira for the selected work type. This integration ensures that all user lifecycle events in ADP Workforce Now are accurately tracked and managed within the Jira system for seamless operational workflows.

Key Functions

The ADP Workforce Now to JIRA Ticket Management Integration provides the following functions:

  • Whenever an employee is onboarded, transferred, or terminated in the ADP Workforce Now application, the integration automatically creates a ticket in the Jira application.
  • If a ticket is already available for an employee in Jira, the integration updates the details of the ticket based on the integration settings.
  • When an employee is rehired, a ticket will be created.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Jira Server: Refer to the Jira Server Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from ADP Workforce Now to Jira Server.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to Jira Ticket Management.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Jira Server connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Please Enable Use Cases

Configure the following integration scenarios:

  • Enable Ticket Management: Select this checkbox to enable ticket management in your Jira instance for the preferred scenarios.

Advance Setting

Global Settings

Configure the following global settings:

  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Dashboard Generation: Select this checkbox to enable operational search and dashboard capabilities for querying onboarding data.
  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.

Ticket Management

The Ticket Management section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Source Application Settings

Configure the following source application settings:

  • Workforce Now Filter*: Select the Workforce Now filter to filter employees while retrieving from Workforce Now. The available options are None, Position ID, Worker Status Terminated, Worker Status Leave, Worker Status Deceased, Worker Status Active, Home Work Location Code, and Worker ID.
    • None: Select this option if you do not want to apply any user filters.
    • Position ID: Select this option to filter employees by their position ID.
      • Source Filter Value For Position ID: Enter the position ID to filter employees by this value.
    • Worker Status Terminated: Select this option to filter employees with the terminated status.
    • Worker Status Leave: Select this option to filter employees with the leave status.
    • Worker Status Deceased: Select this option to filter employees with the deceased status.
    • Worker Status Active: Select this option to filter employees with the active status.
    • Home Work Location Code: Select this option to filter employees by their home work location code.
      • Home Work Location: Enter the home work location code to filter employees by this value.
    • Worker ID: Select this option to filter employees by their worker ID.
      • Source Filter Value: Enter the worker ID to filter employees by this value.
  • Execution Mode*: Select an integration execution mode from the dropdown menu. The available options are Incremental and Full.

Jira Server

Configure the following Jira Server settings:

  • HR Events to Trigger Ticketing*: Select the JML operations that must be supported by the integration for creating or updating a Jira ticket. The available options are Joiner, Mover, and Leaver.
    • New Joiner Ticket Action*: Select the action to perform when a new user is hired. The available options are Create and Update. 
    • Jira Project Key for Onboarding Tickets: Select the project key that should be assigned to the onboarding tickets. The available options are ON and TEST.
    • Jira Issue Type for Onboarding Tickets: Select the issue type that should be assigned to the onboarding tickets. The available options are Sub-task, Epic, Task, Bug, Improvement, and New Feature.
    • Resolved Statuses for Onboarding Tickets: Select the resolved status that should be assigned to the onboarding tickets. The available options are To DO, In Progress, In Review, and Done.
    • Leaver Ticket Action: Select the action to perform when an employee is offboarding. The available options are Create and Update. A leaver ticket will be created if the Create option is selected. If the Update option is selected and a ticket already exists for the employee, the existing leaver ticket will be updated.
    • Jira Project Key for Offboarding Tickets: Select the project key that should be assigned to the off-boarding tickets. The available options are ON and TEST.
    • Jira Issue Type for Offboarding Tickets: Select the issue type that should be assigned to the offboarding tickets.
    • Resolved Statuses for Offboarding Tickets: Select the resolved status that should be assigned to the offboarding tickets. The available options are To DO, In Progress, In Review, and Done.
    • Jira Project Key for Mover Tickets: Select the project key that should be assigned to the mover tickets. The available options are HP, IN, MOV, OF, OFF, ON, ONBOARDEMP, and TEST.
    • Jira Work Type for Mover Tickets: Select the work type that should be assigned to the mover tickets. The available options are Sub-task, Task, Incident, Service Request, Service Request with Approvals, Change, and Problem.
    • Mover Ticket Action: Select the action to perform any changes in the job-related details of an employee. The available options are Create and Update.
    • Resolved Statuses for Mover Tickets: Select the resolved status that should be assigned to the mover tickets. The available options are Open, In Progress, Reopened, Resolved, Closed, To Do, Done, Declined, Waiting for support, Waiting for customer, Pending, Canceled, Escalated, Waiting for approval, Awaiting CAB approval, Planning, Awaiting implementation, Implementing, Peer review / change manager approval, Work in progress, Completed, Under investigation, Under review, Approved, Cancelled, and Rejected.
  • Schedule ticket: Select this checkbox to create a joiner or mover ticket before the actual creation date.
    • Shift by days: Enter the number of days in advance the ticket should be created before the actual creation date. For example, if the actual creation date is the 10th of next month and you want the ticket to be created on the 2nd, enter 8 in this field.
