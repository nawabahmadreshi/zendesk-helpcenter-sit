ADP Lyric HCM to Entra ID Writeback

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration helps write back the Email and Business Phone from Entra ID to the ADP Lyric HCM system.

To identify user records across both systems, the integration uses the employee ID attribute from Entra ID and matches it to the associate ID in the ADP Lyric HCM as a correlation attribute.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for details.
    • ADP Lyric HCM: Refer to the ADP Lyric HCM Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to worker-related data from ADP Lyric HCM.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Lyric HCM to Entra ID Integration.
  5. Enter a name and a description for the integration.
  6. Select your ADP Lyric HCM and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Full Sync: Select this checkbox to enable full sync.
    Note: When enabled, the integration script ensures that employee contact information remains consistent across Microsoft Entra ID and ADP Lyric HCM. It retrieves user data from Entra ID using either a full synchronization or incremental updates with delta tokens, then matches each user to the corresponding employee in ADP Lyric using the employee ID.

    After matching, the script compares the email addresses and business phone numbers in both applications. Based on this comparison, it determines whether to create new contact records, update existing ones, delete outdated information, or skip data that has not changed. The required updates are then applied by calling ADP Lyric APIs to update the employee records.
