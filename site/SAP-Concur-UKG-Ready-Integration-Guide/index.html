Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employee data from UKG Ready to SAP Concur.

Key Functions

This integration automates identity lifecycle management between UKG Ready (an HR/Payroll system) and SAP Concur (an expense management system) with the following primary capabilities:

  • Creates new users in SAP Concur when they are added in UKG Ready.
  • Updates existing SAP Concur users when their information changes in UKG Ready.
  • Deactivates SAP Concur users when employees are terminated in UKG Ready.
  • Creates and deletes users based on a specific date and time.
  • Synchronizes employee expense records from SAP Concur to UKG Ready.
  • Supports incremental synchronization to process only changed records.
  • Sends email notifications for various events, such as user creations, updates, and terminations.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • UKG Ready: Refer to the UKG Ready Configuration Guide for configuration details.
      Note: The following tenant attribute must be configured during connector setup:
      • Company Name
      • API Key
    • SAP Concur: Refer to the SAP Concur Configuration Guide for configuration details.
      Note: The following tenant attributes must be configured during connector setup:
      • Webservices Company UUID
      • Webservices Request Token
      • Auto assign ExpenseApprovers role to the expense approver of the user
      • Auto assign RequestApprovers role to the request approver of the user

Integration Setup

This section details the configuration and mapping process for integrating UKG Ready with SAP Concur. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for SAP Concur / UKG Ready in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Configuration
  • Overview
  • UKG Ready
  • SAP Concur
  • Policy
  • Attribute Map
  • Execution
  • Notification
  • Schedule

Configuration

The following section is displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to select these application connectors.

Overview

On this tab, you can view integration details and the list of features supported by the integration. Scroll down to the Features List section to enable or disable integration features based on your use case.

Feature List

This section lists the features supported by the integration. You can enable or disable each feature based on your requirements.

The following are the supported features:

  • Enable Employee Management: Turn on this toggle button if your use case requires synchronizing employee data from UKG Ready to SAP Concur. When enabled, the integration provisions new user accounts, updates user attributes based on changes, and deprovisions accounts according to employee lifecycle events and data changes in SAP Concur. When enabled, the Manage Employees section is displayed for further configuration.
  • Enable Expense Management: Turn on this toggle button if your use case requires synchronizing expense records from SAP Concur to UKG Ready. When enabled, the Manage Expense section is displayed for further configuration.
  • Enable Monitored Provisioning: Turn on this toggle button if you want to review the changes to be performed by the integration before they are applied to SAP Concur. When enabled, the system evaluates integration executions against the defined monitoring rules and conditions, and then creates tasks for review.
  • Enable Time-Based Onboarding/Offboarding: Turn on this toggle button to allow onboarding and offboarding users based on the configured date, time, and timezone.
  • Enable Notification Management: Turn on this toggle button to allow email notifications to be sent for the integration runs.
  • Enable Report Management: Turn on this toggle button to allow report generation for the integration runs.

UKG Ready

Navigate to the UKG Ready tab to identify the UKG Ready application connector used for the integration and configure the required settings.

This section includes the following configuration sections:

  • Application Settings
  • Time-Based Settings

Application Settings

This section allows you to identify the UKG Ready application connector for the integration and define onboarding and offboarding criteria.

The following settings are displayed on this page for configuration:

  • Select your UKG Ready application instance: Select the UKG Ready application connector that you have configured in the Aquera Admin portal for the integration.
  • Company name: Enter the short name of the company configured in your UKG Ready instance. Refer to the UKG Ready configuration guide for details on obtaining the company name.
  • Filter employees for creation: By default, all active employees in UKG Ready are considered for account creation in SAP Concur. Turn on this toggle button to filter the employees that must be created.
    When enabled, the following fields are available for configuration:
    • Trigger employee creation: Select how employee accounts from UKG Ready must be filtered for creation in SAP Concur. The following is the available option for selection:
      • Based on status: Select this option to filter the employees based on their status in UKG Ready for creation in SAP Concur. When selected, the integration uses the configured statuses to determine the employees who are eligible for creation.
      • Select statuses for employee creation: Select one or more employee statuses that must be considered active. Employees with these statuses in UKG Ready are considered for creation in SAP Concur.
  • Filter employees for termination: By default, all deactivated employees in UKG Ready are considered for termination in SAP Concur. Enable this toggle to filter the employees who must be terminated.
    When enabled, the following fields are available for configuration:
    • Trigger employee termination: Select how employee accounts from UKG Ready must be filtered for termination in SAP Concur. The following is the available option for selection:
      • Based on status: Select this option to filter the employees based on their status in UKG Ready for termination in SAP Concur. When selected, the integration uses the configured statuses to determine the employees who are eligible for termination.
      • Select statuses for employee termination: Select one or more employee statuses that must be considered inactive. Employees with these statuses in UKG Ready are considered for termination in SAP Concur.

Time-Based Settings

This section allows you to define the timezone and schedule for onboarding and offboarding processes.

The following fields are displayed on this page:

  • Method used to identify the applicable time zone: Select a method to identify the applicable timezone. Currently, only the following option is supported:
    • Manual timezone selection: Select this option to create or terminate employees using Coordinated Universal Time (UTC), regardless of the user's local time. When selected, the following fields are displayed for further configuration:
      • Select desired timezone: Select the timezone that must be considered for creating or terminating users in SAP Concur. 
      • Onboarding time: Enter the time that must be considered for onboarding users in SAP Concur. The time has to be provided in the 24-hour format (HH:MM). For example, 03:40
      • Offboarding time:  Enter the time that must be considered for offboarding users in SAP Concur. The time has to be provided in the 24-hour format (HH:MM). For example, 03:40

SAP Concur

The SAP Concur tab enables you to identify the SAP Concur application connector used for the integration and configure the user management settings.

This section includes the following configuration sections:

  • Target Application Settings
  • Manage Employees
  • Manage Expenses

Target Application Settings

The Target Application Settings section allows you to identify the SAP Concur application connector for employee management and define the required settings.

The following fields are displayed on this page:

  • SAP Concur Application: Select the SAP Concur application connector that you have configured in the Aquera Admin portal for the integration.
  • Enable Full Sync: Turn on this toggle button to run the integration in full synchronization mode. When enabled, all employees from UKG Ready will be synced to SAP Concur on every integration run. When disabled, only the employee changes in UKG Ready since the last integration run are synced to SAP Concur.

Manage Employees

The Manage Employees section enables you to customize the integration settings for employee lifecycle management according to your specific use case. Based on the defined settings, the integration synchronizes employee data from UKG Ready to SAP Concur.

Note: This section appears only if the Enable Employee Management setting is enabled in the Feature List section.

The following fields are displayed on this page:

  • Employee Match Settings: Configure the following settings in this section:
    • Customize how users are matched between UKG Ready and SAP Concur: By default, employee records are matched using the employee ID in UKG Ready and the employee number in SAP Concur. Turn on this toggle button to use different source and target attributes for employee matching.
      When enabled, the following field is displayed for further configuration:
      • Select a user matching condition: Select one of the following conditions to determine how users are matched between UKG Ready and SAP Concur.
        • Correlate employees using UKG Ready Employee ID and SAP Concur Employee Number: Select this option to use the employee ID in UKG Ready and the employee number in SAP Concur to find the matching users.
        • Correlate employees using custom attributes: Select this option to match users based on a different set of attributes in UKG Ready and SAP Concur. When selected, the following fields are available for further configuration:
          • UKG Ready attribute for matching users: Select an attribute that uniquely identifies a user record in UKG Ready. For example, select employeeId if you want to use it to match user records.
          • SAP Concur attribute for matching users: Select an attribute in your SAP Concur application that must be used to match the selected UKG Ready attribute value to identify matching users. For example, select employeeNumber if you want to use it to match user records.
  • Safeguard Settings: Configure the following safeguard settings:
    • Enable safeguard limit: Turn on this toggle button to define limits to restrict the number of provisioning operations performed by the integration. When enabled, the following fields are available for further configuration:
      • Safeguard limit type: Select whether the safeguard limits are defined as a number or a percentage. Currently, only a number value is supported.
      • Apply safeguard as: Specify whether a single safeguard limit has to be applied across all operations or separate limits need to be configured for each operation type.
        • Common for all: Select this option to apply the safeguard limit across all operations. When selected, the following field is displayed for the further configuration:
          • Safeguard limit for all operations: Enter a value to define the safeguard limit for all operations. After the limit is reached, no create, update, or terminate operations will be executed.
        • Per operation type: Select this option to define and apply different safeguard limits for each operation type. When selected, the following fields are displayed for further configuration:
          • Enable create limit: Enable this toggle button to apply a safeguard limit for user creations.
            •  Set the creation limit: Enter the maximum number of creations that are allowed to be performed in a single integration run. After the limit is reached, create operations will no longer be executed.
          • Enable update limit: Enable this toggle button to apply a safeguard limit for user updates.
            •  Update limit: Enter the maximum number of updates that are allowed to be performed in a single integration run. After the limit is reached, update operations will no longer be executed.
          • Enable termination limit: Enable this toggle button to apply a safeguard limit for user terminations.
            • Termination limit: Enter the maximum number of terminations that are allowed to be performed in a single integration run. After the limit is reached, termination operations will no longer be executed.
  • Enable employee creation: Turn on this toggle button to customize the creation settings. When enabled, the Employee Creation Settings section is displayed for further configuration.
  • Enable employee update: Turn on this toggle button to customize the update settings. When enabled, the Employee Update Settings section is displayed for further configuration.
  • Enable employee termination: Turn on this toggle button to customize the termination settings. When enabled, the Employee Termination Settings section is displayed for further configuration.

Employee Creation Settings

The Employee Creation Settings section lists the settings specific to user creations in SAP Concur. 
Note: This section is displayed only if you enable the Enable employee creations toggle button. 

  • Create user X days before hire date: Enter the number of days before the hire date when employee records must be created in SAP Concur. For example, if the hire date of an employee is June 25 and you have configured 5 in this field, the employee record will be created in SAP Concur on June 20.
  • Apply time based onboarding: Turn on this toggle button to allow creating employees in SAP Concur based on the defined timezone settings in the Time-Based Settings section.

Employee Update Settings

The Employee Update Settings section lists the settings specific to user updates in SAP Concur. 
Note: This section is displayed only if you enable the Enable employee updates toggle button. 

  • Select changed attributes: Select the attributes that must be updated in SAP Concur.

Employee Termination Settings

The Employee Termination Settings section lists the settings specific to user terminations in SAP Concur. 
Note: This section is displayed only if you enable the Enable employee terminations toggle button. 

  • On termination, select (Employee): Select the action to be performed for an employee in SAP Concur when the employee is terminated in UKG Ready. For example, select Disable account to disable the account in SAP Concur.
  • Apply time based offboarding (Employee): Turn on this toggle button to allow terminating employees in SAP Concur based on the defined timezone settings in the Time-Based Settings section.
  • Disable account after X days from termination (Employee): Enter the number of days after the termination date when the employee account must be disabled in SAP Concur. For example, if the termination date is June 25 and you have configured 5 in this field, the employee account will be disabled in SAP Concur on June 30.

Manage Expenses

The Manage Expense section enables you to configure the integration settings for expense management according to your specific use case.

Note: This section appears only if the Enable Expense Management setting is enabled in the Feature List section.

It has the following settings:

  • Match Expression: This section enables you to choose the correlation attributes in the source and target applications used to identify matching records. It has the following settings:
    • Correlation Attribute in UKG Ready: Select the correlation attribute available in UKG Ready to distinctly find the associated SAP Concur record.
    • Correlation Attribute in SAP Concur: Select the correlation attribute available in SAP Concur to distinctly find the associated UKG Ready record.
  • Expense Type Configuration: This section enables you to define the expense type, expense category, and expense approval statuses for the integration. Based on this configuration, the expense records from SAP Concur will be updated in UKG Ready.
    It has the following settings: 
    • Default Expense Type: Select the expense type to be managed. The following is the available option:
      • Traveller: Select this to manage travel expenses. When selected, the following field is displayed for further configuration:
        • Select the Travel Expense Category: Select one or more travel expense categories to be managed. For example, if you select Airfare and Hotel in this field, only the related expenses will be considered for synchronization.
    • Select the Expense Approval Status: Select the expense approval statuses to be considered for integration. For example, if you select Approved in this field, only expenses with the selected status will be considered for integration.
    • Fetch Expense Reports for Last X Days: Enter the number of days in the past for which expense reports should be fetched during the initial sync. For example, entering 365 will fetch expense reports from the last 365 days.
    • Compensation Type: Select the compensation type in UKG Ready where imported expense reports will be added as additional compensation.

Policy

The Policy tab enables you to define the monitoring rules for reviewing the changes to be performed in SAP Concur.

Monitoring Rules

This section enables you to define the monitoring rules and the email addresses of the administrators who must be monitoring the changes.

Monitoring Settings

  • Supervisor/IT admin email addresses for approval reminders: Enter the email addresses of the administrator or the IT admin that should receive the list of pending changes and decide whether to approve or reject the changes. For details on analyzing and reviewing the changes, refer to the Monitored Provisioning Setup and Reviews.
  • Who should be monitored: Select whether you want to monitor all users or only the users filtered based on a condition.

Rule Detail (Monitoring)

This section enables you to define rules for monitored provisioning. By default, the Assignment Monitoring Rules are available. You can modify or add new rules based on your use cases.

Attribute Map

The Attribute Map tab defines how the data is mapped between the UKG Ready abnd SAP Concur aplications.

The User Lifecycle Managament section defines how employee data is mapped from UKG Ready to users in SAP Concur. The default mapping of attributes is displayed on the page. The UKG Ready attributes are listed on the left side and the SAP Concur attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Execution

The Execution tab allows you to configure various settings required for the integration execution. The following section are displayed in this tab:

  • Advanced Settings
  • Operational Settings

Advanced Settings

It has the following settings:

  • Enable Report-Only Mode: Enable this toggle button to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable debug mode: Turn on this toggle button to generate detailed logs for debugging purposes.
  • Enable source filter: Enable this toggle button to to filter a single employee record from UKG Ready to sync to SAP Concur. When enabled, the following fields are displayed where you need to select the attribute name and the value by which an employee needs to be filtered:
    • Select the filter type: Select the attribute name to filter an employee.
    • Filter value: Enter a value for the selected attribute by which an employee must be filtered.

Logging Settings

This section enables you to customize the log generation settings.

It has the following settings:

Log Insights Settings

  • Select Log Insight Level: Select the log insight level. The following is the available option:

Notification

Navigate to the Notifications tab to configure notification settings for the integration. The following section is available in this tab for configuration:

Configuration

This section allows you to define notification settings for provisioning events.

If the List View toggle button is disabled, the following parameters are displayed:

  • Notification template name: Enter a name for the notification template.
  • Select application: Select application for notification configuration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors.
  • Select event for notification: Select the provisioning event that will trigger a notification. For example, select Account Provisioned to get email notifications every time accounts are provisioned to the target application.
  • Notification delivery channel: Select the channel to be used for sending notification. The available option is:
    • Email: Select this option  if you want to send notification through email. When selected, the following fields are displayed for further configuration:
      • Custom Notification Recipients: By default, notifications are sent to the user’s work email ID. If you want to send notifications to any other recipients, turn on this toggle button.
        When enabled, the following fields are displayed for further configuration:
        • Send notifications to: Select the additional recipients to send email notifications. For example, select Supervisor if you want to send email notifications to the user's supervisor. The following field is displayed when you select Custom in this field:
          • Custom email address: Enter the email IDs of the users to whom you want to send email notifications.
      • Email subject: Enter the content that must be used as the email subject.
  • Notification body template: Enter the content to be used as the notification body. You can use available customization features, such as variables and Improve Description to configure the email body.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
