Tables in Workflows

← Back to index

You can add tables in a Workflow that defines a name-value pair to create custom attributes and use it in Workflows.

Perform the following steps to add a table in a Workflow:

  1. Navigate to the Start Workflow configuration panel > Tables section.
  2. Click + New Table to create a new table.
  3. Provide a table name and description.
    Note: For details on the naming convention for the table names, refer to the General Rules for Names section.
  4. Each row in the table corresponds to a key-name and key-value pair. You can add rows to the table using one of the following options:
    • Add Row: Click Add Row to manually add a row by providing a key-name and a key-value.
    • Import CSV: Click Import CSV to upload rows to the table using a CSV file. The CSV file should have 2 columns with headings NAME and VALUE.
      Notes:
      • The key-name within a table should be unique. The key-name can include a combination of letters and numbers.
      • The key-name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

  5. Click Add at the bottom of the panel to add the table.

  6. Ensure to also click Save Changes at the top-right corner of the Workflow page to save the table to the Workflow.

You can perform the following actions on tables by clicking the vertical ellipsis displayed next to a table entry in the Tables section:

  • Edit: Click Edit to modify a table. After you make changes to the table, click Update at the bottom of the panel to update the table. Ensure to also click Save Changes at the top-right corner of the Workflow page to save the table to the Workflow.
  • Export: Click Export to export a table. This will download the table in a CSV file format.
  • Delete: Click Delete to delete a table.

A Table can be accessed in a Workflow using the following formats:

  • To specify the key name defined in a table, enclose it within double quotes:
    <table_name>["<key_name>"]
    Example: CountryCode["United States"]
  • To specify a variable name or an application's schema attribute name as the key name, do not enclose it in double quotes:
    <table_name>[<key_name>]
    Example: CountryCode[<userObject>.addresses[work].country]
