Greenhouse Recruiting Candidates to Microsoft 365 User Creation

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

This integration automates the creation of Microsoft 365 (MS 365) user accounts based on new hires or updates from Greenhouse Recruiting. It maps candidate data to Microsoft user attributes, securely generates temporary passwords, creates the user via Microsoft Graph API, and provides detailed reporting and notification mechanisms for success and failure cases. Debugging and audit trails are supported via logs, CSV reports, and optional dashboards.

The integration is triggered when a candidate's status changes in Greenhouse Recruiting, such as when a candidate is hired.

This integration performs the following tasks:

  • Listens for new hire events from Greenhouse Recruiting and receives candidate data payloads.
  • Generates a secure temporary password following a configurable password policy.
  • Creates a new Microsoft 365 user account using the Microsoft Graph API with the mapped data and temporary password.
  • Handles success by logging the event, generating CSV reports, sending success notification emails, and writing data to the dashboard, if enabled.
  • Handles failure by logging errors, generating failure CSV reports, sending failure notification emails, and writing failure data to the dashboard, if enabled.
  • Sends detailed notification emails to managers or admins with user credentials and integration status.
  • Produces audit-ready CSV reports capturing all relevant user data, operation results, timestamps, and error details.
  • Enables dashboard analytics to monitor integration outcomes via dashboards.

Note: You must set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to this Aquera integration. Refer to the Appendix for details.

    • Microsoft 365: Refer to the Microsoft 365 Configuration Guide for instructions on creating the Microsoft 365 application connector and setting up the authentication.

Integration Setup

This section details the configuration process for integrating Greenhouse Recruiting candidates with Microsoft 365 user accounts. It covers creating the integration, mapping candidate data fields to users in MS 365, and configuring event-driven triggers to create Microsoft 365 users whenever a new hire event occurs in Greenhouse Recruiting. The process includes secure password generation, user attribute mapping, automated account creation, and comprehensive reporting and notification to ensure a transfer of new hire information from Greenhouse Recruiting to Microsoft 365.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Greenhouse Recruiting Candidates to Microsoft 365 Users.
  5. Enter a name and a description for the integration.
  6. Select your Greenhouse Recruiting Candidates and Microsoft 365 connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Parameters

The Parameters section contains essential parameters that determine how the integration operates. Each setting is crucial in customizing the integration's behavior to meet your organization's needs.

  • Enable Integration Report Download: Allows downloading the integration report as a CSV file for each run.
  • Report Only: Generates a report of the data changes that would occur without actually updating any user records.
  • Debug Mode: Produces detailed logs of integration activity to assist with troubleshooting.
  • Enable Dashboard Generation: Sends integration results to a dashboard for visual analysis and monitoring.
  • Send Integration Report via Email: Sends a CSV report of each integration run to designated recipients.
  • Recipients' Email Addresses: A comma-separated list of email addresses to receive integration notifications and reports.
  • Email Subject: The subject line is used for all integration notification emails.
  • Email Body on Successful Integration Execution: Messages are included in notification emails when user creations or updates succeed.
  • Email Body on Failed Integration: Messages are included in notification emails when user creation or updates fail.
  • Password Policy: Defines password requirements, including minimum length and allowed special characters.
  • Password Length: Minimum number of characters required for user passwords.
  • Allow Special Characters in Password: Special characters permitted in user passwords (e.g., !@#$%^&*).
  • Activate Account: Determines whether new user accounts are active immediately upon creation
  • Assign License to User: Enables user access to specific products and services.
  • Event Type: Event that triggers the integration, such as hire_candidate.
  • Domain Principal Name: Domain used for user principal names (e.g., <>@yourcompany.onmicrosoft.com).
  • Assign License to User: Assign licenses to users to enable access to specific products and services.

Set Up Event Triggers

You need to set up the event triggers to receive candidate hire events from the Greenhouse Recruiting application and run the integration.

  1. In the Trigger section, click Event.
  2. Select the Greenhouse Recruiting connector configured in your Aquera Admin tenant in the Event Source drop-down.
    The URL field displays the endpoint URL to which Greenhouse Recruiting must send the candidate hire events. When configuring the webhooks, this URL must be configured in the Greenhouse Recruiting application. Refer to the Setting Up Webhooks in Greenhouse Recruiting section in the Appendix for details.
  3. Click Create Token to generate a token. Save this token securely.

Run the Integration

The integration is triggered when a candidate in Greenhouse Recruiting is hired. Upon receiving this event, it processes the data and creates a corresponding user account in Microsoft 365.

Once a new employee is successfully created, the integration sends confirmation emails to HR. If any issues arise during the process, such as missing information or system errors, the integration automatically retries and sends urgent alerts to notify the relevant teams.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.

Appendix

Setting Up Webhooks in Greenhouse Recruiting

This section details how to set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to the Aquera integration for processing.

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center on the left navigation pane and click Webhooks.
  4. Enter the following fields to configure your new webhook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for. For example, Aquera Integration
    • When: Select the event, Candidate has been hired, from this drop-down list.
    • Endpoint URL: Copy the URL from the Trigger section on the Greenhouse Recruiting Candidates to the Freshservice integration setup page of the Aquera Admin portal and paste it in this field. Refer to the Set Up Event Triggers section for details.
    • Secret key: Enter the token generated in the Trigger section on the Greenhouse Recruiting Candidates to Freshservice integration setup page of the Aquera Admin portal. Refer to the Set Up Event Triggers section for details.
    • Deactivated?: Make sure that the No option is selected.
  5. Click Advanced Settings and then provide the Basic Authorization Credentials, where the token generated in the Aquera Admin portal must be provided as the password for authentication. The username can be any random string. Refer to the Set Up Event Triggers section for details on obtaining the token.
  6. Click Create webhook to complete the webhook setup.

When you create a new webhook, Greenhouse Recruiting will ping the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the webhook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the webhook.

Note: If the webhook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Webhook section and edit the webhook configuration to fix the error. After you have fixed the configuration issue, make sure that you set the Deactivated? option to No.
