Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration synchronizes parent data from HUBBE SYSTEMS, creates a user account for each parent, assigns them to the appropriate groups, and grants access to the relevant courses in Brivo.

  

NOTE: During the initial integration run, parent records will be retrieved from the HUBBE SYSTEMS and created in the Brivo instance. Each parent record will be assigned a unique external ID by the integration. If some parent records were created in Brivo prior to the integration and do not have an external ID, the integration will not be able to identify them. Since the external ID is used as the correlation attribute to determine whether a record already exists, duplicate parent accounts may be created during the first integration run.

Key Functions

The Brivo Security Suite / SIS Hubbe Integration performs the following functions:

  • Parents from HUBBE SYSTEMS are provisioned into Brivo, where a user account is created for each parent and then assigned to the groups corresponding to their children.
  • The Brivo team will assign the necessary permissions to each parent account and provide a security PIN, enabling access to their children’s academic courses, surveillance, and other related features.
  • This integration supports only full synchronization.
  • Based on the parent status in HUBBE SYSTEMS, user accounts in Brivo are automatically created, updated, or suspended in the corresponding groups. Any parent removed from HUBBE SYSTEMS will be suspended from the matching Brivo group during the next integration run.

Integration Setup

This section details the configurations required for this integration. It covers setting up the applications involved in the integration and configuring the integration parameters for creating and updating user details in Brivo. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Brivo Security Suite / SIS Hubbe.
  5. Enter a name and a description for the integration.
  6. Select your HUBBE SYSTEMS and Brivo connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Note: All parameters marked with a red asterisk (*) are mandatory parameters.

Applications

The Applications section displays the application connectors required for the integration. This section allows you to search and select the HUBBE SYSTEMS application from which the parents data need to be retrieved, and the Brivo application in which the parent's must be assigned to student courses/groups.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following settings for integration scenarios:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Enable User Lifecycle Management*: Select this to retrieve the details of parents and the courses to which the students are enrolled. The Manage Users section will be displayed to configure user management settings only when this checkbox is selected.
  • HUBBE SYSTEMS Agency ID*: Enter the unique agency ID of your HUBBE SYSTEMS from which the parent data must be retrieved.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this to generate logs outlining data changes.
    Note: The Logging section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this to generate the integration report.
    Note: The Reporting section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this to enable the integration-related notifications.
    Note: The Notifications section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this to visualize the integration-related metrics.
    Note: The Log Insights section will be displayed only when this checkbox is selected.
  • Debug Setting: Select to generate logs outlining data changes that would occur for debugging. 
    Note: The Debug Mode section will be displayed only when this checkbox is selected.

Manage Users

Source Filter Settings

Configure the following source filter settings:

  • Source Filter: Select a filter for retrieving parents from HUBBE SYSTEMS. The available options are All Active Parents and All Inactive Parents.
    • Select All Active Parents to filter parents based on their active status as defined in the Parent Active Statuses parameter.
    • Select All Inactive Parents to filter parents based on their inactive status as defined in the Parent Inactive Statuses parameter.
  • Parent Active Statuses: Select the statuses from the dropdown menu that should be considered active parent statuses in HUBBE SYSTEMS. The supported value is Active. This parameter is displayed only when All Active Parents is selected in the Source Filter parameter.
  • Parent Inactive Statuses: Select the statuses from the dropdown menu that should be considered inactive parent statuses in HUBBE SYSTEMS. The supported value is Inactive. This parameter is displayed only when All Inactive Parents is selected in the Source Filter parameter.

Group Assignment

Configure the following group assignment settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Group Assignment to User*: Select the attribute from the dropdown menu, the course ID from Hubbe will be validated against the selected attribute to find the matching group in Brivo. The available options is site_id.

Provisioning Settings

Configure the following provisioning settings:

  • Create User Condition: Select a condition from the dropdown menu that must be applied while creating parents in the Brivo instance. Only the parents that meet the condition configured in this field will be created. The available options are Create All Users, Create Active Users Only, and Custom.
    • If you select Create All Users, all users retrieved from the HUBBE SYSTEMS will be created in Brivo.
    • If you select Create Active Users Only, only active users in HUBBE SYSTEMS will be created in Brivo.
  • Custom Create Condition: Enter an expression that defines the user creation condition in the Brivo instance. This parameter is displayed only if you select Custom in the Create User Condition parameter.
  • Update User Condition: Select a condition from the dropdown menu that must be applied while updating parents in the Brivo instance. Only the parents that meet the condition configured in this field will be updated. The available options are Update All Users, Update Active Users Only, and Custom.
    • If you select Update All Users, all HUBBE SYSTEMS users will be updated in Brivo.
    • If you select Update Active Users Only, only active users in HUBBE SYSTEMS will be updated in Brivo.
  • Custom Update Condition: Enter an expression that defines the parent update condition in the Brivo instance. This parameter is displayed only if you select Custom in the Update User Condition parameter. 
  • Terminate User Condition: Select a condition from the dropdown menu that must be applied while terminating parents in the Brivo instance. Only the parents that meet the condition configured in this field will be terminated. The available options are Terminate Inactive Users Only and Custom. If you select Terminate Inactive Users Only, all inactive users in the HUBBE SYSTEMS will be terminated in Brivo.
  • Custom Terminate Condition: Enter an expression that defines the parent termination condition in the Brivo instance. This parameter is available only if you select Custom in the Terminate User Condition parameter.

Provisioning Threshold

Configure the following provisioning threshold:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Enable Create Limit: Enter this toggle button to set a limit on the number of parent records that can be created during each integration run. When enabled, the following parameter is displayed:
    • Create Limit*: Enter the maximum number of parent records that can be created during each integration run. By default, the value is 1.
  • Enable Update Limit: Enter this toggle button to set a limit on the number of parent records that can be updated during each integration run. When enabled, the following parameter is displayed:
    • Update Limit*: Enter the maximum number of parent records that can be updated during each integration run. By default, the value is 5.
  • Enable Delete Limit: Enter this toggle button to set a limit on the number of parent records that can be deleted during each integration run. When enabled, the following parameter is displayed:
    • Delete Limit*: Enter the maximum number of parent records that can be deleted during each integration run. By default, the value is 5.

Correlation Settings

Configure the following correlation settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Source Correlation Attribute* Select the HUBBE SYSTEMS field that the integration must use to find the matching parent record in Brivo. The available option is contact_id.
  • Target Correlation Attribute*: Select the Brivo field that the integration must use to find the matching parent record in HUBBE SYSTEMS. The available option is externalId.
