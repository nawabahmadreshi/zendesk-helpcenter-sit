User Integration Setup Guide - Ceridian Dayforce to Okta

← Back to index

Overview

This guide covers the end-to-end tasks involved in setting up an orchestration to sync users in Ceridian Dayforce with Okta.

The following tasks are involved in setting up the orchestration from Ceridian Dayforce to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Orchestration Setup
  • Running Orchestration

Initial Preparation

Ceridian Dayforce

  • Service Account with Appropriate roles: A service account must be created with access to the Web Services feature. The username and password of this account are required for authentication.
  • Feature-Level and Field-Level Access Permissions: Appropriate feature-level and web services field-level access permissions must be enabled in Dayforce. 

For details, refer to the Dayforce Configuration Guide.

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • Ceridian Dayforce. Refer to the Ceridian Dayforce Configuration Guide for details.
  • Okta. Refer to the Okta Configuration Guide for details.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from Ceridian Dayforce to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Considerations

  • The criteria for matching a user record from Ceridian Dayforce with that in the Okta.

General Analysis Steps

Before setting up the orchestration, the following analyses need to be done:

  • Fill out the attached mapping template for mapping Ceridian Dayforce user attributes to Okta user attributes.

For detailed analyses of Ceridian Dayforce users and the users in Okta using scripts, refer to the Appendix.

You can also analyze the users data using the Reports option in the Data tab of the Applications page in the Aquera Admin console. Refer to the User Reports Guide for details.

Building Orchestration

This document describes how to set up the orchestration using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set Ceridian Dayforce as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during the orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Running Orchestration

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if the correct data is updated in the Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in Dayforce.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in Ceridian Dayforce and Okta before setting up the orchestration. 

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add Dayforce and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

A few sample scripts are provided here for your reference. The scripts may vary based on your use case and the attributes to be retrieved for analysis.

Analyzing the Users in Dayforce 

Use the below sample script in a scheduled script orchestration for generating a report from Dayforce for analysis of user status, department, usertype, and so on. 

log.info('DAYFORCE ID, DAYFORCE ACTIVE, DAYFORCE GIVENNAME, DAYFORCE FAMILYNAME, DAYFORCE USERNAME, DAYFORCE MIDDLENAME, DAYFORCE TITLE, DAYFORCE DEPARTMENT, DAYFORCE DIVISION, DAYFORCE USERTYPE');
let users = df.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}

Analyzing the Users in Okta

Use the below sample script in a script orchestration for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'

+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
