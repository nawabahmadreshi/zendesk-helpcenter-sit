Integration Setup

This section outlines how the integration must be configured to push user-related data from Dayforce / Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Okta.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Okta connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following feature based on your requirements:

  • Enable Users Sync: Select this checkbox to sync user records from Dayforce to Okta. When enabled, the User Sync and User Life Cycle sections will be displayed for further configuration.
  • Enable Write Back: Select this checkbox to write back the user records from Okta to Dayforce. When enabled, the Write Back section will be displayed for further configuration.

Global Settings

Enable the following settings based on your requirements:

  • Incremental Mode: Select this checkbox to execute the integration in the incremental import mode. If this checkbox is not selected, the integration will be executed in full sync.
    Note: Select this parameter only after the first integration run if you want to execute the integration in incremental import mode.
  • Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Generate Unique Username/Email Using AI Widget: Select this checkbox to generate a unique username and email using the AI widget. When enabled, the Define Policy tab will be displayed for further configuration.
  • Create Custom Attributes in Okta: Select this checkbox to create custom attributes in Okta. When enabled, the Custom Attributes tab will be displayed for further configuration.

Feature Modules

Enable the following settings based on your requirements:

  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.

User Sync

The User Sync section enables you to customize the integration settings according to your specific use case. It has the following settings:

User Correlation Settings

Configure the following settings:

  • Dayforce Correlation Attribute*: Select the Dayforce attribute to distinctly find the associated Okta user record. For example, select Employee Number.
  • Okta Correlation Attribute*: Select the Okta attribute to distinctly find the associated Dayforce user record. For example, select Locality.

User Settings

Configure the following settings:

  • Import Limit: Enter the maximum number of user records to be processed in a single integration run. For example, entering 10 processes only includes the first ten user records.
  • Source Filter*: Select a filter type to filter the employees. The available options are All Employees, All Full Time Employees, Employee Numbers To Fetch, and Employment Statuses to Fetch. Based on the selection, the following parameters are displayed.

Logging

The Logging section enables you to customize the integration settings according to your specific use case. It has the following settings:

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following parameters are displayed once you select Custom:
      • Attribute Filter*: Select the attribute filter to include or exclude attributes in the log. Based on your selection, the following parameters are displayed:
      • Included Attributes: Select the attributes to be included in the log. This parameter is displayed only if the Log Inclusion List option is selected in the Attribute Filter parameter.
      • Excluded Attributes: Select the attributes to be excluded from the log. This parameter is displayed only if the Log Exclusion List option is selected in the Attribute Filter parameter.
    • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values for each event when choosing the Full Details or Custom option.
