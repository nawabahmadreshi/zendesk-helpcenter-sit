Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Pinpoint-to-Dayforce onboarding integration, which automatically creates Dayforce employee records for hired candidates when an Onboarding Process Step is configured. The available configuration options are Background Check, Diversity and Inclusion, Handbook, Policies and Documentation.

The integration identifies eligible onboarding applications in Pinpoint and transfers the required data to Dayforce to generate corresponding employee records.

To identify candidate records across both systems, the integration uses the Social Security Number attribute from Pinpoint. It matches it to the Social Security Number in Dayforce as a correlation attribute.

Key Functions

The Pinpoint to Dayforce integration provides the following functions:

  • Fetches onboarding applications from Pinpoint for the selected onboarding step, Diversity and Inclusion.
  • Collects candidate and job details, then filters candidates who are ready to move forward based on their onboarding stage.
  • Converts Pinpoint candidate data into the format required by Dayforce.
  • Checks whether the employee already exists in Dayforce before creating a new record.
  • Creates new employee records in Dayforce, following the configured limit for the number of records that can be created at once..

Integration Setup

This section outlines how the integration must be configured to push candidate data from Pinpoint to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Pinpoint to Dayforce Integration.
  5. Enter a name and a description for the integration.
  6. Select your Pinpoint and Dayforce connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Debug Mode

The Debug Mode section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Source Filter: Determines how candidate records are filtered from the source system before onboarding processing begins. For example, choosing a specific filter restricts onboarding to only candidates that meet the defined criteria, while choosing no filter allows all candidates to be processed.

    • Onboarding Application ID: Filters candidates based on a specific onboarding application identifier from the source system. For example, entering 55117 ensures that only candidates associated with onboarding application ID 55117 are included in the onboarding flow, and all other candidates are ignored.

      • Source Filter Value: Specifies the exact onboarding application identifier that candidate records must match to be included in the onboarding flow. For example, entering 55117 ensures that only candidates associated with the onboarding application ID 55117 are processed, while all other candidates are ignored.

    • Custom: Allows filtering candidates based on a custom source attribute defined in the integration configuration. For example, you can filter candidates using a custom field such as sourceSystemCode, jobRequisitionId, or any other attribute exposed by the source, so that only candidates matching that custom value are processed.

    • None: Disables source-based filtering so that all incoming candidate records are considered for onboarding. For example, selecting None means every candidate received from the source system will be processed, regardless of application ID or custom attributes.

XRefCode Rules

The XRefCode Rules section defines how cross-reference codes (XRefCodes) are assigned and managed during the Pinpoint to Dayforce integration. These rules ensure that candidate records from Pinpoint are accurately mapped to corresponding values in Dayforce.

XRefCode codes are assigned when new candidate applications are transferred. They also manage field changes, ensuring mapping accuracy and preventing duplicate or invalid XRefCode assignments.

Perform the following steps to create a new XRefCode rule:

  1. Click Add New Rule to create a new XRefCode rule. 
    The following fields are displayed:
    1. Click Choose Attribute to view the list of available Pinpoint candidate attributes.
    2. Use the search bar to locate the attribute you want to use as a condition for the rule.
    3. Select the required attribute and configure its value to define the rule condition.
    4. Only candidates meeting this condition are included under the XRefCode rule.
    5. If multiple attributes and values are specified, the integration applies an AND condition between them. For example, to satisfy the rule criteria, a candidate must belong to both the specified Division (ACME and Hooli) and the Department (Engineering).
    6. Select Under Pay Type, select one or more Dayforce groups to which candidates meeting the rule condition should be assigned. The list displays all available groups from the target Dayforce application.
  2. Click Add Row to add another condition and XRefCode rule for the same Dayforce attribute.
  3. Click to delete a row by using the delete icon at the right end of the row:
  4.  Click Switch to Expression to write a JavaScript expression and set the condition. 

Note: You can configure XRefCode rules in the same way for additional Dayforce entities, including: Pay Class, Pay Group, Department, Contact Information Types, Employment Status Reasons,  Employment Statuses, Org Units, Jobs, Positions, Countries, Location, Roles, and Cultures.
