Workflows Use Case Guide: Employee Onboarding

← Back to index

Overview

Workflows facilitate the automation of highly complex business processes within an organization. Aquera's Workflows streamline processes related to employee onboarding, transitions, and offboarding, enabling IT departments to save time, enhance operational efficiency, cut costs, and boost productivity.

By utilizing Workflows, organizations can accelerate the onboarding and offboarding of employees by managing application access for authorized users and revoking access and licenses for those who are no longer authorized.

This guide elaborates on the utilization of Aquera's Workflows for the onboarding of new employees in an organization.

The following are the tasks involved in setting up this use case:

  1. Retrieve new hires from an Applicant Tracking System (ATS).
  2. Create a user record in a Human Resource Information System (HRIS).
  3. Create a service ticket in an IT Service Management (ITSM) tool to generate an employee number and a company email ID and assign a laptop to the new hire.
  4. Send a welcome email to the new hire.

To perform these tasks, it is essential to configure multiple workflows tailored to the requirements of specific departments such as Human Resources (HR), Information Technology (IT), and others. Consequently, the primary workflow can trigger sub-workflows associated with specific departments to distribute tasks relevant to the corresponding departments.

Notes: To understand how to set up workflows, refer to the Workflows User Guide. For details on configuring elements within workflows, refer to the Workflow Elements Configuration Guide.

Employee Onboarding Workflow

Create and set up an Employee Onboarding workflow that performs all the onboarding tasks for the new hires in an organization: 

  1. Configure the Triggered By option of the Employee Onboarding workflow based on your use case. For example, you can set a weekly schedule as shown here:


  2. Add an ATS application to be used in the Workflow by clicking Add Application in the Components panel. For example, you can use the Greenhouse Recruiting application to retrieve the new hires to be onboarded. Drag and drop the Greenhouse Recruiting element to the Workflow. Refer to the Add an Element section of the Workflow User Guide for further details.

    In the configuration of the Greenhouse Recruiting element, select the operation Import All Candidates and provide the filter condition to filter candidates based on their hired status. For details on configuring an Application element, refer to the Applications section of the Workflow Elements Configuration Guide.


  3. Add the For-each element to iterate over each hired candidate.
  4. Add the Invoke Workflow element to invoke an HR Workflow that performs creating a new user record in an HRIS application for each new hire.

    The input to the HR Workflow is a candidate object that contains the details required to create a user record. The output of the HR Workflow is a boolean value of recordCreated that indicates if the user record was successfully created or not.

    For details on configuring the HR Workflow, refer to the HR Workflow section. 


  5. Click Mapper to open the Attribute Mapper and provide values for the attributes of the input object candidate, which will be passed to the HR Workflow. The right side shows the attributes of the candidate object. In the Value/Expression field, you can provide a value or expression to map to the attributes of the candidate object.

    The following is a sample candidate from the Greenhouse Recruiting application:

    In the candidate schema, if the array attributes such as emailAddresses, phoneNumbers, and addresses have predefined (canonical) values for the type sub-attribute, they can be accessed using the find function as shown in the following example:
    hiredCandidate.emailAddresses.find(emailAddresses => emailAddresses.type == "personal").value
    This will retrieve the personal email address of the candidate.

    However, if array attributes such as socialMediaAddresses do not have sub-attributes with predefined (canonical) values, they can be accessed in the following format:
    <objectVariableName>.<attributeName>[indexValue].value
    For example, hiredCandidate.socialMediaAddresses[0].value will retrieve the value from the 0th index of the social media address.


  6. Add the If-else element to evaluate the return value from the HR workflow.


  7. After a user record is created in an HR application, add the Invoke Workflow element to invoke an IT Workflow that performs creating a ticket in an ITSM tool to issue laptops for the new hires.

    The input to the IT Workflow is an employee object that contains the details required to create a service ticket. The output is a company object that contains the employee data generated such as the employee number, company email ID, userName, and a temporary password for the company portal.

    For details on configuring the IT Workflow, refer to the  IT Workflow section. 


  8. Click Mapper to open the Attribute Mapper and provide values for the attributes of the input object employee, which will be passed to the IT Workflow. In the Value/Expression field, you can provide a value or expression to map to the attributes of the employee object. 


  9. Add and configure the Send Email element to send welcome emails to the new employees.

HR Workflow

Create and set up an HR workflow that creates user records in an HRIS application for the new hires:

  1. Configure the Triggered By option of the HR workflow as shown here since the Employee Onboarding workflow triggers the HR workflow.


  2. You can configure input parameters, a return, and variables for the workflow in the Definition section.

  3. Configure the input to be received by the HR workflow from the Employee Onboarding workflow. For this use case, define the input as an Object.

    You can define your own schema by selecting the option Define New for the input object candidate.


  4. Configure the return to be sent by the HR workflow to the Employee Onboarding workflow. For this use case, define the return as a Boolean to indicate if the user record was created in the HRIS application.


  5. Add an HRIS application to be used in the Workflow by clicking Add Application in the Components panel. For example, you can use the BambooHR application to create a user record for the new hire. Drag and drop the BambooHR element to the Workflow. Refer to the Add an Element section of the Workflow User Guide for further details.
     
    In the configuration of the BambooHR application element, select the operation Create User. 


  6. Click Mapper to open the Attribute Mapper and provide values for the attributes of the BambooHR user object. In the Value/Expression field, you can provide a value or expression to map to the attributes of the BambooHR user object. 


  7. Add the Set Variable element to set a value for the output variable recordCreated to indicate that the user record is created. 

IT Workflow

Create and set up an IT workflow that creates service tickets in an ITSM tool to allocate laptops to the new employees:

  1. Configure the Triggered By option of the IT workflow as shown here since the Employee Onboarding workflow triggers the IT workflow.
    |

  2. You can configure input parameters, a return, and variables for the workflow in the Definition section.

  3. Configure the input to be received by the IT workflow from the Employee Onboarding workflow. For this use case, define the input as an Object.

    You can define your own schema by selecting the option Define New for the input object employee.


  4. Configure the return to be sent by the IT workflow to the Employee Onboarding workflow. For this use case, define the return as an Object.

    You can define your own schema by selecting the option Define New for the return object company.


  5. Add an ITSM tool to be used in the Workflow by clicking Add Application in the Components panel. For example, you can use ServiceNow to create a ticket for onboarding the new hire. Drag and drop the ServiceNow element to the Workflow. Refer to the Add an Element section of the Workflow User Guide for further details.

    In the configuration of the ServiceNow element, select the operation Create Ticket to create a ticket for assigning a laptop to the new hire. 
  6. Click Mapper to open the Attribute Mapper and provide values for the attributes of the Ticket object. In the Value/Expression field, you can provide a value or expression to map to the attributes of the Ticket object.


  7. Once a ticket is open, this workflow generates an employee number, company email ID, userName, and a temporary password for the new hires. 
    You can use the Persistent Data element to store the last used employee number. 
  8. When a new employee joins the organization, increment the value of the Persistent Data and set this as the employee number of the new employee. 


  9. Add the Set Object element to set a value for the return object company. 


  10. Click Mapper to open the Attribute Mapper and provide values for the attributes of the return object company. In the Value/Expression field, you can provide a value or expression to map to the attributes of the company object.  

    Following is a sample script for password generation that you can provide in the Value/Expression field: 
    let chars = "0123456789abcdefghijklmnopqrstuvwxyz@#$%^&*()ABCDEFGHIJKLMNOPQRSTUVWXYZ";
let passwordLength = 12;
let password = "";
for(var i = 0; i <= passwordLength; i++){
     let randomNumber = Math.floor(Math.random() * chars.length);
     password += chars.substring(randomNumber, randomNumber +1); 
}
    password;
