Attribute Mapping in Integrations

← Back to index

Overview

This guide provides details about Attribute Mapping in integrations. Attribute Mapping will synchronize data from the source application to the target application. You can write JavaScript expressions to modify the value of the source attribute before updating the value in the target attribute.

You can set attribute mapping in User Integration, Bulk Integration, and Event-based Integrations. Attribute Mapping simplifies executing create, update, and delete operations, as it automatically reflects the changes made to a source attribute in the mapped target attribute when the integration runs.

Attribute Mapping

Navigate to your integration and click the Attribute Map tab to perform attribute mapping. The default mapping of SCIM attributes from the source to the target application is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the Attribute Map page.


The attributes from the source application are specified as SOURCE.<attributeName>.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

You can perform the following actions as per your requirements:

Provide Source Attribute Values

In the SOURCE ATTRIBUTE/EXPRESSION field, you can include source attributes using SOURCE.<attributeName>. You can also include static values such as strings, booleans, and numbers. Enclose strings in double quotes. For boolean, provide the value true or false without quotes.

Preview User Data

To preview the attribute values of a user from the source application, click the Enter objects for preview search box, enter the user's SCIM ID or username in the source search box, choose Map on Create or Map on Update, and click Search.

When the Map on Create option is selected, you will see the source search box.

When the Map on Update option is selected, you will see both source and target search boxes.

Note: It is mandatory to enter the SCIM user ID or username of the source application while using Map on Update. You can choose to leave the target search box blank based on your use case.

From the search results, enable the Preview toggle.

To preview the attribute values of a user from the target application, click the Enter objects for preview search box, choose Map on Update, enter the user's SCIM ID or username in the source and target search boxes, and click the Search button.

Note: It is mandatory to configure the source search box to preview the target application values.

You can enter expressions with the DEST keyword under the SOURCE ATTRIBUTE/EXPRESSION field to view the target attribute values. For example, you can use the expression DEST.addresses[work].streetAddress to view the work street address from the target application.

You can use %ATTEMPT under Advanced Options to preview the %ATTEMPT values used in the expressions. For more details, refer to the %ATTEMPT section in the Integration Expressions - Keywords, Methods, and Functions guide.


Notes:

  • When the attributes or expressions provided in the SOURCE ATTRIBUTE/EXPRESSION field are invalid or do not return a value, the preview will not appear.
  • When the attributes or expressions provided in the SOURCE ATTRIBUTE/EXPRESSION field contain diacritics (such as teléfono), the preview will not accurately show the diacritic signs.

Filter Attributes by an Attribute Name

To filter the list of attributes by the attribute's name, enter the attribute name in the Search Attributes box.

Filter Attributes by the Attribute Type

To filter the list of attributes by the attribute type, click the Attribute Filter icon at the top-right corner of the page. You can filter Core, Enterprise, or IAN attributes from the attribute list.

For example, to list the Core and IAN attributes, select the Core and IAN checkboxes. It will list all the Core and IAN attributes in the Attribute Map tab.

Select a Mapping Option for an Attribute

Set the appropriate mapping option from the source attribute to the target attribute. Specify if the mapping of an attribute is to be done during both create and update operations, the create operation only, or the update operation only. It is also possible to choose not to map from the source to the target attribute.

Click the dropdown list present between source and target attributes and select any of the following options for an attribute:

  • Map on Create and Update: The target attribute will be updated during the create and update operations
  • Map on Create: The target attribute will be updated only during the create operation.
  • Map on Update: The target attribute will be updated only during the update operation. 
  • Don't Map: The target attribute will never be updated.

For example, if you select the Map on Create option for the attribute SOURCE.addresses[work].country, this attribute value will only be updated in the target application during the create operation.

Note: All the target attributes with mutability as "Immutable" support only the Map on Create option. It does not support Map on Create and Update and Map on Update options.

Filter Attributes by the Mapping Option

You can filter the list of attributes using the Map filter present between the source and target attributes.

For example, if you select the Map on Create and Update option, it will list all the target attributes that will be updated during the create and update operations.

Add a New Row to the Attribute Map

To add a new row to map a source attribute to a target attribute, click Add Row to Map at the end of the attribute mapping list.

Enter the source and target attribute details and select a mapping option. Then, click the check mark to add the new row to the attribute map.

Note: Dynamic attributes are not listed by default. You must add and map dynamic attributes manually.

Write Expressions in Attribute Mapping

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.


On clicking the Expand icon, the Expression Editor window appears, where you can write JavaScript expressions to perform operations on the attribute values. These expressions can contain variables, attribute names, and JavaScript functions. When the integration runs, the expressions are evaluated and the resulting values are updated in the target application.

For details on Aquera's keywords, inbuilt methods, and functions, refer to the Integration Expressions - Keywords, Methods, and Functions guide.

For details on the logical operators such as AND, OR, and NOT to be used in attribute mapping, refer to the article Syntax for Logical Operators in Integrations

Delete an Attribute Mapping

To delete an attribute mapping, navigate to the target attribute field and click the Delete icon.

Notes

  1. The read-only attributes of the target application will not be displayed in the attribute mapping , and the mapping is not required to be set. 
  2. If an attribute is of complex type and its multivalued property is true, then the type sub-attribute will not be displayed under the source or target application for attribute mapping. Also, if the attribute is of a complex type and multivalued is true and there is no type sub-attribute, then the attribute will not be displayed under the source or target application for attribute mapping.
  3. If the source or target application is changed, you can click the refresh icon available next to the target application and click Confirm to fetch new attributes (attributes of the new application).
    This will not affect the existing attribute mapping. The attributes of the old application will not be removed automatically, and you need to manually remove all the attributes that are not required. All new attributes of the new application will be listed. All existing attributes that are available in the new application will be as it is.
  4. Set the mapping option to Map on Create for password attributes.
  5. You can write JavaScript expressions directly in the Expression field/Expression Editor or you can write in an external JavaScript code editor and copy it to the Expression Editor so that the expressions will be valid.
  6. If you are using comments in the Expression field/Expression Editor, ensure to use only Multi-line comments since Single-line comments may not work. Multi-line comments start with /* and end with */ as shown in the following example:
    /* This is a 
        multi-line comment */
  7. It is possible to have two different attribute mapping rows for the same target attribute: one with Map on Create and the other with Map on Update.
  8. When the SOURCE ATTRIBUTE/EXPRESSION field is empty and you are providing a new value or expression for the attribute mapping, ensure to change the default Mapping Option of Don't Map to the appropriate mapping option (such as Map on Create and Update, Map on Create, or Map on Update) in the Map field.
  9. When the SOURCE ATTRIBUTE/EXPRESSION field has a DEST attribute, ensure to select the mapping option as Map on Create and Update or Map on Update, otherwise the value will be Undefined.
  10. The groups target attribute is an array of strings which are the display names of the groups.
