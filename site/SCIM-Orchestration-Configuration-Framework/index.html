Introduction to SCIM Orchestration

The Aquera SCIM Orchestration Configuration Framework provides a common basis for the configuration of many of Aquera's orchestrations. These include SCIMToSCIM, SCIM Batch Import, Message Listener, and more. The framework balances configuration simplicity with maximal flexibility.

Layout

The layout of orchestrations that are members of the SCIM Orchestration family share a common look and feel in Aquera Admin. There is a configuration tab, with the subset fields that are meaningful to the individual orchestration, and Attribute Map, Tables, and Group Rules tabs which are common to all orchestrations.

Expressions

Many fields accept expressions, which evaluate to true, false, or actual value. Aquera has developed JavaScript SCIM extensions designed to ease Orchestration configuration. Each field defined later in this document clearly indicates if it takes an expression. Knowledge of rudimentary JavaScript is highly beneficial to the configuration task, and knowledge of the specific usage of the source and destination SCIM fields is also important. You will see that Aquera will automatically layout a good starting point based on your specific source and destination applications, and then you will complete the configuration and test it before going live.

SCIM Expression Extensions

SCIM Extensions are defined here and discussed more completely later. These extensions are relevant in the context of one source record and one destination record, as is the case for the Configuration tab match expression, Attribute Map, and Group Rules.

SOURCE -> replaced with the SOURCE record object, example: SOURCE.userName
DEST -> replaced with the DEST record object, example: DEST.userName
scimArray[type] -> replaced with the specific array entry where scimArray.type == type, example: SOURCE.emails[work]
%USEC -> replaced with a number string which makes the result unique, useful only in Attribute Maps, example: name.givenName + name.familyName + "%USEQ"
%ATTEMPT -> replaced with empty string on updates, replaced with the creation attempt number, starting at one, on creation, useful only in Atrribute Maps, example: name.givenName + name.middleName.slice(0, %ATTEMPT) + name.familyName
TABLE.tableName[name] -> replaced with the value associated with name as defined in the Tables tab; example: TABLE.titles[SOURCE.title]

Configuration

Configuration section is designed to Add the Basic Details, Creation of E-Mail Details, Deactivation of E-Mail Details and Advanced details of the Orchestration.

Attribute Mapping

Attribute mapping is done between source and target. The mapping simplifies the process of executing the CRUD operations on the attributes, as the changes made to the source attribute will automatically get reflected at the target once the mapping is completed and the changes are saved.

Tables

Tables were designed as a mechanism to simplify the mapping process. It is possible to achieve Table functionality in pure JavaScript, but the resulting expressions can be large, unwieldy, and fragile. Tables are created as a set of name/value pairs, and they make it straightforward to map an attribute based on one of many names. Tables are defined in the Tables tab, and they are referenced from within a mapping in the Attribute Map tab using the TABLE SCIM extension.

Group Rules

The Group Rules tab facilitates the mapping of group membership. Rules can be created using basic conditions, or they can be created using the full capabilities of JavaScript SCIM Extensions. Each entry defines a set of groups that a user will be included in the destination should the expression evaluate to true. Destination SCIM group display names are used in all group rules.

Schedule

Once the orchestration is completed, updates reflecting the changes made at the source in the target can be automated. Users can select a schedule for the data refresh from the time slots given under 'Schedule'. If there are no data changes, the user can opt for Never. Depending on the frequency with which the data changes are made, the automation can be done from once every 20 minutes to once in 2 hours.
