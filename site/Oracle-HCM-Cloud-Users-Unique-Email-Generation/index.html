Oracle HCM Cloud Users - Unique Email Generation

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration generates a unique work email ID for the newly onboarded workers in Oracle HCM Cloud by verifying the emails available across multiple systems such as Okta, Google Workspace, Active Directory and so on. The integration then updates the unique email ID for the workers in the Oracle HCM Cloud application.

Key Functions

The Oracle HCM Cloud Users - Unique Email Generation integration performs the following functions:

  • Retrieves data of workers from your Oracle HCM Cloud instance in an incremental mode.
    • You can specify the types of workers that must be retrieved by the integration. The worker types can be workers, employees, contingent workers, and pending workers
    • You can specify the lookback period (in days) to retrieve the workers whose hire dates fall within that date range.
  • Generates an email ID based on the expression/script provided in the Attribute Map section of the integration.
  • Validates the generated email ID for uniqueness across various systems specified in the integration. For example, the email ID is validated against Active Directory, Okta, and Google Workspace.
    • If the generated email ID already exists for a user in any of those systems, the integration generates another email ID based on the script and checks for the uniqueness again in all those systems. This process continues until a unique email ID is generated.
  • Updates the unique email ID to the worker record in Oracle. 
  • Sends automated, detailed notification emails to administrators on the email updates.
  • Produces CSV reports capturing the details of the workers whose email IDs have been updated in Oracle.
  • Enables log insights to monitor integration outcomes and performance.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal, based on your requirements:
    • Oracle HCM Cloud: Refer to the Oracle Cloud HCM Configuration Guide for instructions on creating the Oracle HCM Cloud application connector and setting up the authentication.
    • Google Workspace: Refer to the Google Workspace Configuration Guide for instructions on creating the Google Workspace application connector and setting up the authentication.
    • Okta: Refer to the Okta Configuration Guide for instructions on creating the Okta application connector and setting up the authentication.
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for instructions on creating the Okta application connector and setting up the authentication.

Integration Setup

This section details the configurations required for this integration. It covers setting up the applications involved in the integration and configuring the integration preferences for updating the unique email IDs for workers in Oracle Cloud HCM. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Oracle Unique Email Writeback.
  5. Enter a name and a description for the integration to identify it.
  6. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to search and select the Oracle HCM Cloud application in which the unique email ID has to be updated for workers, and the applications against which the uniqueness of the email ID must be validated.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the updates that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.

Source Settings

Configure the following settings for Oracle Cloud HCM:

  • Select the Source Application: Select the Oracle Cloud HCM application from which the worker data needs to be retrieved for generating the unique email IDs.
  • Days Before Hire Takes Effect: Enter the lookback period (in number of days) to retrieve the workers whose hire dates fall in that date range. For example, if you specify 2 in this field, all workers who have their hire date in the last 2 days will be retrieved by the integration.
  • Select Worker Type: Select the types of workers that must be retrieved by the integration. The available worker types are All Workers, Employee, Contingent Worker, and Pending Worker.

Target Settings

Configure the following settings:

  • Select the Target Applications: Select the applications against which the uniqueness of the generated email IDs needs to be validated.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: All the integration configurations are captured in the logs.
    • Full Details: All the actions performed by the integration are captured in the logs. The logs include the following details:
      • The total number of workers retrieved from oracle
      • The number of successful email updates
      • The number of unique email generation failures. If any unique email generation failure has encountered, the logs show the details of the employees for whom the error has occurred.

Notification Settings

Configure the following settings for email notifications.

Success Notification Settings

Configure the following email notification settings for successful integration runs:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications must be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications for successful integration runs. For example, Oracle Workers Email ID Generation Report
  • Admin Email Notification Template: Enter the content that must be used as the email body for the email notifications.

Error Notification Settings

Configure the following email notification settings for integration errors:

  • Recipients of Error Notifications: Enter the email addresses of the recipients who must be notified about integration errors.
  • Error Email Subject: Enter the content that must be used as the email subject for sending email notifications for integration errors. For example, Oracle Workers Email ID Generation Error
  • Error Email Template: Enter the content that must be used as the email body for the email notifications.

Report Settings

Configure the following report settings:

  • Send Integration Report via Email: Select this checkbox to send an email notification with a CSV report for integration runs. The CSV report will have the details of the employees for whom the unique email ID has been updated.

Attribute Map

The integration uses the expression provided in the Attribute Map to generate and update unique email IDs for Oracle workers.
