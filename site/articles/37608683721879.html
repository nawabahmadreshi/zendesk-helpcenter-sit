Dayforce to Active Directory Employees Integration - Initial Setup Guide

← Back

Overview

Before you start configuring the Dayforce to Active Directory integration, you must first create the Dayforce and Active Directory connectors. A set of prerequisite setups must be completed to configure these connectors and verify the connectivity.

This guide walks you through those initial setup steps that must be completed in sequence.

Note: This integration synchronizes the employee data from your Dayforce instance to users in your Active Directory instance. 

Dayforce Setup

Make sure that the following setups are completed in the specified order. It is recommended that these steps be carried out by your organization's HR personnel or Dayforce administrator.

  1. Log in to Dayforce.
  2. Import the role with the minimal permissions. This role has the list of permissions, authorizations, and the required features enabled.
  3. Navigate to System Admin > User and then create a service account (non-human account) and assign the role to the account.
  4. Log in to the Aquera Admin portal.
  5. Navigate to Applications and add the Dayforce application connector. Make sure that the following configurations are completed:
    1.  Provide an application name and the Tenant value, which is the hostname available in the URL of your Dayforce instance. The tenant value can be one of the following:
      • Test instance: test.dayforcehcm.com
      • Production instance: www.dayforcehcm.com
      • Config instance: config.dayforcehcm.com
      • Any self-maintained instances: The tenant value differs based on the hostname in your login URL. For example, aquera.dayforcehcm.com
    2. In the Tenant Attributes section, enter the Namespace/Company in which this account is configured. This is the company value you would provide when you log in to your Dayforce web console.
    3. Click Create to add the Dayforce application connector.
  6. In the Credentials tab, provide the username and password of the service account that you have created in Dayforce, and then click Save Changes.
  7. Click Test Connection to verify the credentials are valid and have sufficient permissions to retrieve employee data from Dayforce.

Active Directory Setup

Make sure that the following setups are performed in the same sequence.  It is recommended that these steps be carried out by your organization's IT administrator/ Active Directory administrator. Kindly contact Aquera Support personnel if you need assistance with the setup.

  1. Log in to the Active Directory server.
  2. Create a service account (with local admin privileges) exclusively for Aquera integration and authentication.
  3. Assign permissions to the service account based on your requirements. Refer to the Configuring Service Account Permissions in Active Directory guide for details.
  4. Aquera provides two mechanisms to connect from Aquera Cloud to your on-prem Active Directory: Direct Connection and Agent. Refer to the Direct Connection vs Agent Connection guide for details.
  5. Log in to the Aquera Admin portal.
  6. Navigate to Applications and add the Active Directory application connector. Make sure that the following configurations are completed:
    1.  Provide an application name and the Tenant value. The tenant value must be one of the following:
      • If you select the Direct connection type, enter the IP address or the Fully Qualified Domain Name (FQDN) of the Active Directory server (domain controller) and the port number in the format: {IP-or-FQDN}:{port}. 
        The port number is typically 636 while using encryption (TCP LDAPS) or 389 for clear text. 
      • If you select the Aquera Agent connection type, enter proxy in the Tenant field and provide the details in Agent Group Name, Hostname, and port. Refer to the Aquera Agent Installation and Configuration Guide and the Connection section of the Active Directory Configuration Guide for details.
    2. In the Tenant Attributes section, configure the following fields:
      • User Base Distinguished Names
      • User Object Classes
      • User Filter
      • Group Base Distinguished Names
      • Group Object Classes

        Refer to the Aquera Tenant Attributes section of the Active Directory (AD) Configuration Guide for details.
    3. Click Create to add the Active Directory application connector.
  7. In the Credentials tab, provide the username and password of the service account that you have created in Active Directory, and then click Save Changes.
  8. Click Test Connection to verify the credentials are valid and have sufficient permissions to retrieve the user data in Active Directory.

Once the test verification is successfully completed in both the connectors, you are ready to configure the Dayforce Employees to Active Directory Users integration.
