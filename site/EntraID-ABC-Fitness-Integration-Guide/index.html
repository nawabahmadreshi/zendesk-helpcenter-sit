EntraID / ABC Fitness Integration Guide

Overview

The Aquera platform enables organizations to automate and scale identity data management by orchestrating integrations across enterprise systems. This integration facilitates automated employee provisioning from Microsoft Entra ID to ABC Fitness, ensuring consistent and reliable synchronization of employee data.

The integration retrieves users from Microsoft Entra ID based on configured filters and conditions. It evaluates each user against defined criteria, including source filters, sync conditions, and attribute mappings, to determine eligibility for processing.

Using configured correlation attributes, such as email addresses or external identifiers, the integration checks whether corresponding employee records exist in ABC Fitness. If no matching record is found, a new employee profile is created with the mapped attributes such as first name, last name, email, department, and club association. Department values are dynamically resolved to align with the target system’s structure, ensuring accurate assignment.

If a matching employee already exists, the integration skips the record to prevent duplicate creation. Similarly, users who do not meet the configured conditions or fail validation checks are excluded from processing.

This integration focuses on provisioning employee records while maintaining data consistency and operational efficiency across systems.

Key Functions

The Entra ID to ABC Fitness integration provides the following functions:

  • Create employee records in ABC Fitness using Entra ID data: When a user is retrieved from Microsoft Entra ID and meets the configured criteria, such as sync mode, filters, and create conditions, the integration creates a corresponding employee record in ABC Fitness. The employee record is populated with key attributes, including first name, last name, email, department, club association, and other fields, which are mapped according to the configured attribute mappings.

  • Prevent duplicate employee creation using correlation attributes: Before creating a new employee, the integration evaluates existing records in ABC Fitness using configured source and target correlation attributes, such as email or external identifiers. If a matching record is found, the integration skips creation to prevent duplicate entries and maintain data consistency.

  • Apply filtering and conditions to control user processing: The integration processes users based on configured filters and conditions, including source filters, create conditions, and sync configurations. Only users who meet these conditions are considered for provisioning, while others are skipped.

  • Map and transform Entra ID attributes to the ABC Fitness schema: User attributes from Microsoft Entra ID are mapped to the ABC Fitness employee schema using configurable attribute mappings. These mappings define how source attributes are transformed and populated in the target system.

  • Resolve departments dynamically in ABC Fitness: The integration retrieves department data from ABC Fitness and dynamically maps Entra ID department values to corresponding department identifiers. This ensures that each employee is assigned to the correct department in the target system. If a valid department mapping is not found, the employee record is skipped based on configuration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register. Contact your Aquera account manager to obtain access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/

  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following connectors:

    • Microsoft 365 Entra ID: Add and configure the Microsoft Entra ID connector. This connector reads user data from Entra ID to create employees in ABC Fitness. Refer to the Microsoft 365 Entra ID Configuration Guide for details on app registration, Graph API permissions, and completing tenant consent.

    • ABC Fitness: Add and configure the ABC Fitness connector with the required API credentials. This connector is used to create and manage employee records. Refer to the ABC Fitness Configuration Guide for authentication and setup details.

Integration Setup

This section outlines the configuration required to integrate Entra ID with ABC Fitness in the Aquera Admin Portal.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal
  2. Go to Integrations
  3. Click Create Integration
  4. Search for Entra ID / ABC Fitness in the search box next to the Explore Templates section. 
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and description for the integration.
  7. Select connector instances:
    • Microsoft Entra ID
    • ABC Fitness
  8. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following features based on your requirements:

  • Enable Employee Lifecycle Management: Select to process users from Microsoft Entra ID for employee provisioning in ABC Fitness based on configured conditions. For example, when selected, eligible users are evaluated during execution and created as employee records in ABC Fitness.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them. For example, when enabled, the integration shows which users would be created but does not make any changes.
  • Enable Logging: Select to generate logs outlining data changes. For example, when enabled, you can review logs to see how each user was processed.
  • Enable Notifications: Select to enable the integration-related notifications. For example, you receive alerts when the integration completes or fails.
  • Enable Report Generation: Select to generate the integration report. For example, a CSV report is created showing created, skipped, and failed records.
  • Enable Log Insights: Select to visualize the integration-related metrics. For example, you can view trends such as the number of users processed over time.
  • Run Full Sync Days Interval: Defines how often a full sync is executed. For example, if set to 7, a full sync runs every 7 days, while other runs may be incremental.

Manage Employees

This section defines how users from Entra ID are identified, filtered, and created as employees in ABC Fitness.

Correlation Settings

These settings determine how users are matched between Entra ID and ABC Fitness to prevent duplicates:

  • Source Correlation Attribute: Defines the attribute from Entra ID used to uniquely identify users. For example, onPremisesExtensionAttributes.extensionAttribute12 is used as the unique identifier from Entra ID.
  • Target Correlation Attribute: Defines the corresponding attribute in ABC Fitness used for matching. For example, a barcode is used to match existing employee records in ABC Fitness.

Provisioning Settings

These settings control which users are eligible for employee creation:

  • Create Employee Condition: Specifies which users are eligible for employee creation. For example, when set to Active Only, only active Entra ID users are processed.

Provisioning Threshold

These settings control limits to manage processing and prevent excessive operations:

Create Limit: Select the maximum number of employee records to be created in a single execution. The following options are available in the dropdown:

  • Unlimited: Select this option to allow all eligible employee records to be created without restriction.
  • Value: Select this option to specify a limit. For example, a value of 100 allows up to 100 employee creations in a single execution.
  • Import Limit: Select the maximum number of users to retrieve and process from Microsoft Entra ID in a single execution. The following options are available in the dropdown:
    • Unlimited: Select this option to process all users matching the configured criteria without restriction.
    • Value: Select this option to specify a limit. For example, a value of 500 limits processing to 500 users in a single execution.

Sensitive Fields

This section defines which fields are treated as sensitive and handled securely in logs, reports, and processing.

Configure the following sensitive field settings:

  • PII Fields: Specifies personally identifiable information that should be masked or protected. For example, fields such as email or phoneNumber can be redacted to prevent exposure in logs.
  • GUID Fields: Defines fields that contain unique identifiers. For example, fields like id or objectId can be marked to ensure proper handling and tracking.
  • Employee ID Fields: Identifies fields used as employee identifiers. For example, fields such as employeeId or barcode can be specified to ensure consistent identification across systems.

Log Settings

This section allows you to configure how integration execution data is logged for monitoring and troubleshooting.

Note: The Log Settings section appears when you select the Enable Log Settings checkbox in the General Settings section of the Parameters tab.

Configure the following log settings:

  • Select Log Type: Select the type of data to include in logs. For example, choosing Employee Number with All Data includes employee number, operations, field names, and all values (PII and non-PII).
  • Select Log Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary, and SSN ensures these fields are not captured.

Note: The Report Settings section appears when you select the Enable Report Settings checkbox in the General Settings section of the Parameters tab.

Notifications

This section allows you to configure email notifications for integration execution outcomes, including success, failure, and skipped records.

Note: The Notification section appears when you select the Enable Notifications checkbox in the General Settings section of the Parameters tab.

Configure the following notification settings:

Notification Template Settings

Configure the following notification template settings:

  • Send Email Notification For Success: Enable to send email notifications when the integration completes successfully. For example, when enabled, a success report is sent after execution.
  • Send Email Notification For Failure: Enable to send email notifications when the integration fails. For example, when enabled, a failure alert is sent if errors occur.
  • Attach report to email: Enable to attach the generated report to notification emails. For example, when enabled, the CSV report is included in the email.
  • Send email notification for skipped records: Enable to send notifications when records are skipped. For example, users skipped due to filters or validation are reported via email.

Log Insights Settings

This section allows you to configure analytics and dashboard visibility for integration activity and performance metrics.

Note: The Log Insights section appears when you select the Enable log insights checkbox in the General Settings section of the Parameters tab.

Configure the following log insight settings:

      • Custom Inclusion List: Enter the attributes that should be included in Log Insights.

      If the Custom exclusion list is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be omitted from the Log Insights.

Source Filter

This section allows you to restrict processing to specific users based on defined criteria.

  • Enable Employee Sync Source Filter: Enable to process only users that match a specific filter condition. For example, when enabled, only users matching the defined attribute and value are processed.

  • Employee Sync Source Filter: Select the attribute to filter users. For example, choosing EmployeeID filter filters users by their employee ID.
    The following options are available in the dropdown:

    • EmployeeID: Select this option to filter users by Employee ID.
    • Expression: Select this option to define a custom filter using an expression.

      • Source Filter Value: Enter the value or expression used to filter users from Microsoft Entra ID. For example, enter 12345 to process only users with that Employee ID, or define an expression to filter users based on specific attribute conditions.

Attribute Map

The Entra ID to ABC Fitness integration maps Microsoft Entra ID attributes to ABC Fitness employee fields. You can modify the source attributes as needed to align with your business requirements.

You can use JavaScript expressions to transform or construct values before mapping them to target attributes. Write the expression directly in the source field or use the expand option to define complex logic.

All attributes marked with a red asterisk (*) are required fields for successful employee creation in ABC Fitness.

After completing the source-to-target attribute mapping, click Save Changes at the top-right corner of the page to apply the configuration.

The attribute map displays a single mapping table:

Employee Mapper

This section defines how Entra ID user attributes are mapped to ABC Fitness employee fields. For example, attributes such as firstName, lastName, email, departmentId, and barcode are mapped to create employee records in ABC Fitness.

Tables

This section allows you to define and manage reference data used during integration to enrich or override attribute values in ABC Fitness.

Tables store key–value pairs that the integration references at runtime when required data is not available in Entra ID or when additional mapping logic is needed.

Available Tables

Note: The following tables represent a specific use case within the integration:

  • EOSEmailExclusionTitles: Defines job titles that should be excluded from processing. For example, users with titles listed in this table are skipped during employee creation.

  • locationCode_clubCode: Maps location codes from Entra ID to corresponding club codes in ABC Fitness. For example, a location code like NYC01 can be mapped to a club code such as 101.

Managing Table Data

Click a table row to open the Update Table window. Each row represents a key–value pair, where Name is the lookup key and Value is the mapped result used at execution time.

You can manage table entries using the following options:

  • Add Row: Allows you to manually add a key–value pair. For example, add Manager → Exclude to define a rule.

  • Upload CSV File: Allows bulk upload of entries. The CSV must contain two columns: NAME and VALUE, where each row represents a key–value pair.

  • Search: Allows you to quickly find specific entries within the table.

After making changes, click Save to apply updates.

Note: Each key (Name) must be unique and is used as the lookup identifier during integration. Key names can include letters, numbers, and most special characters, except /, \, and ".

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
