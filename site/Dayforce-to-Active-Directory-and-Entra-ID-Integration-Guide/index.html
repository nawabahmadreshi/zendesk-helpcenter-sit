Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration synchronizes employee and candidate data from Dayforce to Active Directory and Microsoft Entra ID to automate the full employee lifecycle. It provisions new user accounts for onboarding, updates user attributes when employee information changes, and deprovisions or disables users upon termination based on Dayforce status. It enables centralized identity management and supports automated application access and entitlement assignment using predefined rules.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes employee and candidate data from Dayforce to Active Directory and Microsoft Entra ID.
  • Manages group assignments.
  • Generates password for new users in Active Directory and Microsoft Entra ID.
  • Sends notifications about provisioning events.
  • Generates detailed logs and reports.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • User Account in Entra ID: You must have a user account in Entra ID with permission to configure provisioning (for example, Application Administrator, Cloud Application Administrator, Application Owner, or Global Administrator).
    Notes:
    • The Exchange Administrator role is required for Exchange-related features, such as mailbox conversion.
    • The SharePoint Administrator role is required for SharePoint-related features, such as providing OneDrive access to the user's manager.
  • Admin Access to Dayforce: You must have admin access to your Dayforce instance.
  • Dayforce Field-Level Access: The specific field-level permissions must be enabled for the service account to manage user information in Dayforce. For more details, refer to
  • Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Dayforce: Refer to the Dayforce Configuration Guide for configuration details.
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for configuration details.
    • Microsoft 365: Refer to the Microsoft 365 Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating Dayforce with Active Directory and Microsoft Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce to Active Directory / Entra ID in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • Source
  • Directory
  • Policy
  • Execution
  • Notification Center
  • Tables

Features List

This section lists the features supported by the integration. You can enable or disable each feature based on your requirements.

The following are the supported features:

  • Enable Entra ID for Employee Management / Enable Employee Management in Entra ID: Enable this option to synchronize employee data with Microsoft Entra ID. When enabled, the integration provisions new user accounts, updates user attributes based on changes, and deprovisions accounts according to employee lifecycle events and data changes in Dayforce.
  • Enable Entra ID for Candidate Management / Enable Candidate Management in Entra ID: Enable this option to synchronize candidate data with Microsoft Entra ID. When enabled, candidate accounts can be provisioned and managed in Microsoft Entra ID.
  • Enable Active Directory for Employee Management / Enable Employee Management in Active Directory: Enable this option to synchronize employee data with Active Directory. When enabled, the integration creates, updates, and deprovisions user accounts in Active Directory based on employee lifecycle events and data changes in Dayforce.
  • Enable Active Directory for Candidate Management / Enable Candidate Management in Active Directory: Enable this option to synchronize candidate data with Active Directory. When enabled, candidate accounts can be provisioned and managed in Active Directory.

Identity Source Configuration / Source Configuration for Employee Management

In this section, you must configure the source application and the required conditions for employee management.

The following parameters are displayed for configuration:

  • Source Application / Source application: Select the Dayforce application connector that you have configured in the Aquera Admin portal for employee management. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Employee Onboarding/Offboarding Decision Mode / Onboarding/Offboarding Criteria: Select one of the following options from the dropdown menu to define how employee accounts from Dayforce are evaluated for creation and termination in the directory:
    • Based on the Status / Based on status: Select this option to evaluate account creation and termination based on the employee’s status in Dayforce. When selected, the integration uses the configured statuses to determine which users are eligible for account creation or termination in the directory. The following parameters are available for further configuration:
      • Status(es) to be considered for Account Activation / Statuses for account activation: Select the statuses that should be considered active. Employees with these statuses in Dayforce are considered for account creation in the directory.
      • Status(es) To Be Considered for Account De-activation / Statuses for account deactivation: Select the statuses that should be considered inactive. Employees with these statuses in Dayforce are considered for account termination in the directory.
    • Based on the Date / Based on date: Select this option to evaluate account creation and termination using a date attribute in Dayforce. When selected, the integration uses the configured date attribute to determine when employee accounts are considered for creation or termination in the directory. The following parameters are available for further configuration:
      • Date to be Considered for Create Account / Date for account creation: Select the Dayforce date attribute used to determine when an account is considered for creation in the directory.
      • Date to be Considered For Termination / Date for account termination: Select the Dayforce date attribute used to determine when an account is considered for termination in the directory.

Candidate Source Configuration / Source Configuration for Candidate Management

In this section, you must configure the source application and the required conditions for candidate management.

The following parameters are displayed for configuration:

  • Candidate Source Application / Source application: Select the Dayforce application connector that you have configured in the Aquera Admin portal for candidate management. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
    Note: The source application can be the same for employee and candidate management.
  • Candidate Onboarding/Offboarding Decision Mode / Onboarding/Offboarding Criteria: Select one of the following options from the dropdown menu to define how candidate accounts from Dayforce are evaluated for creation and termination in the directory:
    • Based on the Status / Based on status: Select this option to evaluate account creation and termination based on the candidate’s status in Dayforce. When selected, the integration uses the configured statuses to determine which candidates are eligible for account creation or termination in the directory. The following parameters are available for further configuration:
      • Candidate Statuses to be considered As Active / Statuses for account activation: Select the statuses that should be considered active. Candidates with these statuses in Dayforce are considered for account creation in the directory.
      • Candidate Statuses to be considered As Inactive / Statuses for account deactivation: Select the statuses that should be considered inactive. Candidates with these statuses in Dayforce are considered for account termination in the directory.
    • Based on the Date / Based on date: Select this option to evaluate account creation and termination using a date attribute in Dayforce. When selected, the integration uses the configured date attribute to determine when candidate accounts are considered for creation or termination in the directory. The following parameters are available for further configuration:
      • Candidate Date to be Considered for Create Account / Date for account creation: Select the Dayforce date attribute used to determine when an account is considered for creation in the directory.
      • Candidate Date to be Considered For Termination / Date for account termination: Select the Dayforce date attribute used to determine when an account is considered for termination in the directory.
  • Report Name Used To Fetch Candidates / Report name for candidate retrieval: Enter the name of the Dayforce report that contains candidate data. The integration retrieves candidate information from the specified report.
    Note: A report can be configured in the Dayforce instance by navigating to Data & Analytics > Reporting > Reports.
  • Status Field Name For Candidate / Candidate status field name: Enter the field name in the report that contains candidate status information. This field is used by the integration to evaluate candidate status.
  • Candidate Last Modified FieldName / Candidate last modified field name: Enter the field name in the report that contains the last modified timestamp. This field is used by the integration to identify updates to candidate records.

Mailbox Settings

This section allows you to configure and manage settings related to a user’s mailbox. The following fields are available for configuration in this section:

  • Forward emails sent to deactivated user to manager / Forward emails of a deactivated user to their manager: Select this checkbox to automatically forward all incoming emails of a deactivated user to their manager's email address. This ensures that important communications are not missed and can be handled appropriately after the user is deactivated.
  • Disable Exchange ActiveSync (mobile) on deactivation / Disable Exchange ActiveSync on deactivation: Select this checkbox to disable Exchange ActiveSync for users upon deactivation. If not selected, the deactivated user may continue to access emails on previously configured mobile devices. Enabling this option prevents mobile email access after deactivation.
    Note: Exchange ActiveSync enables users to access email, calendar, contacts, and tasks on mobile devices, and allows them to continue using this data even when offline.
  • Remove recurring calendar meetings / Remove recurring meetings from calendar on deactivation: Select this checkbox to automatically remove all recurring meetings organized by the user when the user is deactivated.
  • Convert user mailbox to shared mailbox / Convert Users regular mailbox to shared mailbox on deactivation: Select this checkbox to convert the private (regular) mailbox of a user to a shared mailbox on user deactivation.
  • Remove shared mailbox delegations / Remove shared mailbox delegations on user deactivation: Select this checkbox to remove the shared mailbox delegations on user deactivation. If this checkbox is selected, all delegated access to other users’ shared mailboxes will be removed from the user on deactivation.
  • Add shared mailbox permissions to manager / Add shared mailbox permission to manager on deactivation: Select this checkbox to add the shared mailbox permission to the user's manager when the user is deactivated.
    Note: Deactivating and converting the shared mailbox and assigning permission to the manager changes will be visible in the Exchange Online admin portal.
  • Send auto-response after deactivation / Send auto-reply message after the user is deactivated: Select this checkbox to set an automatic out-of-office reply in a user's mailbox upon deactivation. When enabled, the following fields are displayed for further configuration:
    • Out-of-office / auto-reply message configuration / Auto-reply message configuration: Enter the message that must be sent as an automatic out-of-office reply on behalf of a user when the user is deactivated. - This field must be hidden when the parent field is disabled.
    • Auto-reply end date (in days): Enter the number of days after which you want to disable the automatic out-of-office reply from a deactivated user's mailbox. For example, if you configure 90 in this field, the automatic out-of-office reply will not be sent after 90 days of deactivation of a user account. - This field must be hidden when the parent field is disabled.

Note: The Exchange Administrator role is required for Exchange-related features.

Device Settings

This section allows you to manage users’ device access. The following fields are available for configuration in this section:

  • Sign out user from all devices / Sign out user from all devices on deactivation: Select this checkbox to sign out users from all devices automatically when their account is deactivated.
  • Remove mobile devices on deactivation: Select this checkbox to remove the mobile devices of a user when the user is deactivated. This will remove all connections between the user’s mailbox and their mobile devices.
  • Remove Intune-managed devices on deactivation: Select this checkbox to remove all devices of a user that are managed or pre-enrolled through Microsoft Intune Admin Center when the user is deactivated.
    Note: This feature requires an active Intune license.

Groups Settings

This section allows you to configure users’ group settings. The following fields are available for configuration in this section:

  • Remove user from all groups on deletion / Remove user from groups on deletion: Select this checkbox to remove users from groups when the user is deleted.
  • Remove user from all groups on deactivation / Remove user from groups on deactivation: Select this checkbox to remove users from groups when the user is deactivated.
  • Reassign group ownership to manager if user is sole owner / Transfer group ownership to manager on deactivation or deletion: Select this checkbox to automatically remove a user's group ownership and assign it to their manager when the user is deactivated or deleted. This applies only when the user is the only owner of the group.

WriteBack Settings / Writeback Settings

This section allows you to configure settings for updating data back to the source system. The following fields are available in this section:

  • Enable HR WriteBack / Enable Writeback: Select this checkbox to enable updating attributes in Dayforce (source system) based on changes in the directory.
  • Write Back to / Select attributes for writeback: Select which attributes should be updated in Dayforce when their values change.
  • WriteBack Source Attribute / Source application attribute: Select the correlation attribute available in the directory that uniquely identifies a user record in Dayforce. For example, select employeeId if you want to use it to match user records in Dayforce.
  • WriteBack Target Attribute / Target Application Attribute: Select the correlation attribute available in Dayforce that uniquely identifies a user record in the directory. For example, select EmployeeNumber.
