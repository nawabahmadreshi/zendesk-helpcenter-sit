How Aquera shares User’s OneDrive with their Manager

← Back

Aquera uses a combination of Microsoft Graph APIs and SharePoint Online PowerShell (PnP) to grant a manager access to a user’s OneDrive securely. The process is outlined below:

Step 1: Retrieve the tenant’s root SharePoint site.

Aquera first identifies the customer’s SharePoint tenant by calling the Microsoft Graph API.

Microsoft Graph API:

GET /sites/root

This returns the root SharePoint site for the tenant, which Aquera uses to determine the correct SharePoint environment for subsequent operations.

Step 2: Connect to SharePoint Online.

Aquera establishes a secure connection to SharePoint Online using approved credentials.

PowerShell (PnP) Command:

Connect-PnPOnline -Url <TenantRootSiteUrl> -Credentials <ServiceCredentials>

This authenticated connection allows Aquera to perform SharePoint administrative actions in the customer’s tenant.

Step 3: Retrieve the user’s OneDrive site URL.

Aquera retrieves the OneDrive (MySite) URL associated with the user.

Microsoft Graph API:

GET /users/{userId}?$select=mySite

This returns the SharePoint site URL that backs the user’s OneDrive, which is required to manage permissions.

Step 4: Assign the manager as an owner of the OneDrive site.

Aquera grants the user’s manager ownership permissions on the OneDrive site.

PowerShell (PnP) Command:

Set-PnPTenantSite -Url <UserOneDriveSiteUrl> -Owners <ManagerUPN>

This action adds the manager as an owner of the user’s OneDrive.
