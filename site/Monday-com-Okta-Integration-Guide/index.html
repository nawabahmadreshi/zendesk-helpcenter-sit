Key Functions

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Applications in the Aquera Admin Portal: 
    • Monday Connector: Ensure you add and configure the source application connector in the Aquera Admin portal. Refer to the Monday Configuration Guide for details.
    • Okta Connector: Ensure you add and configure the Okta connector in the Aquera Admin portal. Refer to the Okta Configuration Guide for details. 

Integration Setup

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Monday.com / Okta. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Select the Monday.com connector and Okta connector from the corresponding drop-down lists.
  7. Click Continue. The integration is created.

Once created, the following tabs are displayed:

  • Configuration
  • User provisioning
  • Okta
  • Attribute Map
  • Schedule

Configuration

In this tab, you can view all the parameter settings for the integration to be enabled based on the use cases.

The following field is visible in this section:

  • Parameters
  • Log Settings
  • Report Settings
  • Log Insights
  • Notifications

Parameters

The parameters section is for configuring and enabling the required settings for this integration. The following field is displayed in this section:

  • Enable Report-Only Mode: Enabling this will execute the integration run without making any changes or updates by only generating the report.
  • Enable Log Settings: Select this checkbox to generate logs tracking data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Report Settings: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights Settings: Select this to visualize the integration-related metrics. When enabled, the Insights Settings section is displayed for further configuration. 
  • Enable Notifications: This section allows you to configure email notifications for integration events, such as successful, failed execution, or send pre-transaction emails and to define the recipients and report details included in the notification emails. When enabled, the Notification Template Settings will appear.

Log Settings

The Log Settings section allows you to configure the required settings to be logged. The following field is displayed in this section:

  • Select log type: Select the log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Log Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.
  • Enter PII Fields: Specifies personally identifiable information that should be masked or protected. For example, fields such as email or phoneNumber can be redacted to prevent exposure in logs. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
  • Enter GUID Fields: Defines fields that contain unique identifiers. For example, fields like id or objectId can be marked to ensure proper handling and tracking. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
  • Enter Employee Number Fields: Identifies fields used as employee identifiers. For example, fields such as employeeId or number can be specified to ensure consistent identification across systems. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.

Notification

This section allows you to configure the notification settings for this integration. When enabled, the Notification Template Settings will appear.

Notification Template Settings

Configure the following settings to define when notification emails are sent.

  • Send email notification for success: Enable this to send an email when the integration completes successfully. For example, when the integration finishes without errors, a success notification is sent to the configured recipients. When enabled, Success Execution Settings will be displayed.
  • Send email notification for failure: Select to send an email when the integration fails. For example, if the integration encounters an error, a failure notification is sent to the configured recipients.
  • Attach report to email: Select to include the integration report as an attachment in notification emails. For example, enabling this option attaches the execution summary to the notification email.
  • Send Pre-Transaction email: 

Success Notification Settings

Configure the following settings to define the recipients and content of success notification emails.

  • Success notification recipients: Enter the email addresses that should receive success notification emails. For example, add admin@company.com to receive confirmation when the integration completes successfully.
  • Success notification CC: Enter additional email addresses to receive a copy of success notifications. For example, include teamlead@company.com to keep the team informed.
  • Success email subject: Enter the subject line used for success notification emails. For example, Integration Execution Success.
  • Success email body: Enter the message content sent when the integration completes successfully. For example, include a summary of the run, such as the number of users processed and any key updates.

Failure Notification Settings

Configure the following settings to define the recipients and content of failure notification emails.

  • Failure notification recipients: Enter the email addresses that should receive failure notification emails. For example, add support@company.com to receive alerts when the integration encounters an error.
  • Failure notification CC: Enter additional email addresses to receive a copy of failure notifications. For example, include itadmin@company.com to notify system administrators.
  • Failure email subject: Enter the subject line used for failure notification emails. For example, Integration Execution Failure.
  • Failure email body: Enter the message content sent when the integration fails. For example, include error details and troubleshooting guidance.

User Provisioning

The User Provisioning tab allows you to filter the users in the board group section in the Monday instance. This helps for user provisioning according to your specified use case. Configure the following settings to achieve user provisioning.

  • Select Monday.com application instance: Select the required application instance from the dropdown.
  • Monday.com Board ID: Provide the board ID available in the domain URL, which is available in the search bar. 
  • Group Name: Enter the group name you want to configure.
