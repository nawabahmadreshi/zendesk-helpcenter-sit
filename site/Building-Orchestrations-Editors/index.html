Building Orchestrations - Editors

← Back to index

Overview

In orchestration, input data can be edited by way of adding additional inputs in the form of javascript. By using the Editor feature, users can edit the code to update the inputs. The Editor feature is enabled for ScimToScim and Custom Script orchestrations and this guide helps you understand the step-by-step procedure. It also gives you the detail of Connector Definition Editor.

ScimToScim Orchestration Editor

From your Aquera Admin UI, go to Orchestrations and click Create Orchestration on the top right. 

Provide a Name to the orchestration and click dropdown to Select Orchestration Type

Select ScimToScim from the pop-up
Select Source and Target applications and click Create on the top right

Go to Attribute Map 

As you navigate the cursor Source Attributes/Expressions, an Expand icon appears on the screen. Click the icon next to the attribute that requires an edit

A pop-up of Edit Expression appears on the screen

You can take the help of the prompter that pops-up on the screen to edit expression

After adding/editing/removing expressions click OK 

Click OK and then Save Changes on the top left

Custom Script Orchestration Editor

From your Aquera Admin UI, go to Orchestrations and click Create Orchestration on the top right.

Provide a Name to the orchestration and click dropdown to Select Orchestration Type

Select Custom Script from the pop-up

Choose Application-1 (mandatory) and Application-2 (optional) that you are going to use in your script

Click Create on the top left

Go to Script section and add your javascript. 

The connections can be referenced from within the script as APP[0] or APP[1]. The operations available are get, put, post, patch, delete. In addition logging is available by using utils.log("string")

You can avail the help text that pops-up on the screen

You can also get help in editing the javascript from pop-up windows

Click Save Changes on the top right

To schedule the updates, go to Schedule, select a duration and click Save Changes on the top right

Connector Definition Editor

Display Name: Enter the display name for the application

Description: Enter a description for the application.

Pick your protocol: Select the protocol as SCIM, this will populate the fields in the Schema accordingly

Copy this URL: Copy the URL and paste it at the time of provisioning in the Identity Service Provider.

Tenant: The Tenant detail of each application is given in this section 

Authentication: A detailed description of steps to be followed to authenticate the application is given in this section

Schema: Schema consists of SCIM Attributes and Custom Attributes of Users.

Source Connection Type: This is of two types - Direct Connection and Flat File

If the connector supports Flat File, then:

Provide Display Name and Description, select Flat File, provide email ID of the File Uploader and click Create on the top right. 

Go to Upload

Click Upload on the top right

Click Upload File

Select relevant file

The file gets uploaded

Click done

Go to Credentials tab and click Save Changes on the top right

Use the token during the app configuration in the Identity Server
