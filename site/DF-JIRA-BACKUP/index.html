Overview

The Aquera platform allows your organization to scale complex data management by automating the integration between various applications.

Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. This enables organisations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration enables synchronization between Dayforce and Jira Service Management Server, automating the creation of onboarding and offboarding tickets for employees. It continuously retrieves and reconciles employee lifecycle events from Dayforce, such as hires and terminations, with Jira, ensuring timely and accurate issue creation for IT operation tickets.

Core Integration Tasks

This integration performs the following tasks:

  • Retrieves employee data from Dayforce, supporting incremental updates based on hire and termination date ranges.

  • Accesses and processes detailed employee records, including contact info, department, and job metadata.

  • Detects new hires and leavers, then correlates them with existing Jira tickets using employee numbers or work email addresses.

  • Automatically creates onboarding or offboarding Jira tickets using mapped field values, supporting both Jira Cloud and Jira Server environments.

  • Prevents duplicate tickets by checking existing issues in Jira and skipping already-tracked employees.

Integration Setup

This section outlines the configuration process for integrating Dayforce with Jira Service Management Server. It includes the steps for creating the integration, defining parameters, and enabling automated synchronization of employee lifecycle events from Dayforce into Jira.

This integration synchronizes employee data from Dayforce, including new hires, terminations, and rehires, and creates corresponding Jira tickets for onboarding, offboarding, and other HR-related events. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce User to Jira Server Ticket.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Jira Server connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Global Settings 

Configure the following global settings:

  • Enable Report-Only Mode: Simulate the integration run without creating or updating any Jira tickets. Use this to verify configurations and preview changes safely.

  • Enable Ticket Management: Automatically create, update, or skip Jira tickets based on employee lifecycle events from Dayforce (e.g., onboarding, offboarding).

  • Enable User Lifecycle Management: Manage user-related actions, such as provisioning, updates, or deactivation, in connected IT systems when employee records are updated.

  • Enable Organization Structure and Data Sync: Sync organizational elements, such as departments, locations, or business units, from Dayforce into target systems for alignment and reporting.

  • Enable Email Notifications: Send real-time email alerts to admins for successful executions, failures, or skipped actions, keeping stakeholders informed.

  • Enable Report Generation: Produce structured CSV reports summarizing ticket creation results, employee processing outcomes, and integration status.

  • Enable Logging: Capture detailed logs of integration actions, API responses, and errors to support operational audits and root cause analysis.

  • Enable Analytics: Log full transaction data to analytics dashboards (e.g., OpenSearch) to monitor integration trends, track issues, and measure performance over time.

Notification Settings 

Configure the following notification settings:

  • Admin Email Addresses: Enter one or more email addresses to receive integration notifications, including success summaries, failures, and CSV reports.

  • Admin Email Subject: Define the subject line for notification emails sent to admins. Helps recipients quickly identify integration-related updates.

  • Admin Email Notification Template: Customize the content of the email body sent with each execution. You can include summaries, instructions, or branded messaging.

  • Send Integration Report via Email: Enable this option to attach a CSV report of the integration run (including ticket IDs, employee status, and error details) to the admin notification.

Logging Settings 

Configure the following logging settings:

  • Log Detail Level: Select the level of detail to capture in integration logs.

    • Minimal: Logs only high-level actions and errors.

    • Full: Includes detailed execution flow, mapping outputs, and ticket actions.

    • Debug: Captures low-level system behavior, payloads, and API responses for troubleshooting.

Reporting Settings

Configure the following reporting settings:

  • Reporting Detail Level: Select the level of detail to include in the integration report.

    • Summary: Includes only high-level outcomes (e.g., number of tickets created, skipped, failed).

    • Full: Includes detailed records for each employee processed, including correlation data, ticket status, and error remarks.

Analytics Settings

Configure the following analytic settings:

  • Analytics Detail Level: Select the level of detail to log for analytics and dashboard reporting.

    • Summary: Logs only high-level statistics (e.g., success/failure counts).

    • Full: Logs detailed event-level data, including ticket payloads, correlation attributes, and operation results.

Ticketing Management Settings

Configure the following ticket management settings:

  • HR Events to Trigger Ticketing: Select one or more employee lifecycle events that should initiate ticket creation workflows.

  • New Joiner Ticket Action: Choose the action to perform when a new hire is detected (e.g., create onboarding ticket, update existing records).

  • Mover Ticket Action: Specify the action to take when an employee changes roles, departments, or locations.

  • Leaver Ticket Action: Select the action to trigger when an employee is offboarded (e.g., termination, resignation, retirement).

  • Rehire Ticket Action: Define the behavior for creating tickets when a previously terminated employee is rehired.

  • Jira Project Key for Offboarding Tickets: Enter the project key in Jira where offboarding tickets should be created.

  • Jira Issue Type for Offboarding Tickets: Provide the Jira issue type to be used for offboarding (e.g., Task, Service Request).

  • Correlation Field in Jira for Offboarding: Select the Jira field used to match offboarding employees (e.g., Employee Number, Email).

  • Correlation Pattern (Regex): Provide the regex pattern used to extract the correlation value from Jira fields or the summary. For Example, the pattern EmployeeNo:\s*(\w+) will extract 12345 from a summary such as: “Create laptop request. EmployeeNo: 12345”

  • Jira Project Key for Onboarding Tickets: Enter the project key in Jira where onboarding tickets should be created.
  • Jira Issue Type for Onboarding Tickets: Specify the Jira issue type to be used for onboarding workflows.
  • Resolved Statuses for Onboarding Tickets: Select the statuses that mark onboarding tickets as resolved or complete.
  • Correlation Field in Jira for Onboarding: Choose the Jira field used to link onboarding tickets with Dayforce records.
  • Target ITSM Platform: Select the ITSM platform where tickets will be created (e.g., Jira Service Management Server).
  • HR Source System: Select the HR platform that provides employee data (e.g., Dayforce).

  • Upcoming Hire Date Window (Days): Enter how many days into the future the system should look for new hires.

    Example: If you enter 10, the integration fetches employees whose hire date is within the next 10 days, starting from today.

  • Upcoming Termination Date Window (Days): Enter how many days into the future the system should check for upcoming terminations.

    Example: If you enter 7, the integration fetches employees whose termination date is within the next 7 days, starting from today.

  • Dayforce Field for Onboarding Correlation: Select the Dayforce field to be used for correlating onboarding tickets (e.g., Employee Number).
  • Dayforce Field for Offboarding Correlation: Select the Dayforce field to be used for correlating offboarding tickets.

Tables

The tables section includes the tables that are used to translate values between applications that use different formats or terminologies for the same data. For instance, Dayforce might store a job title as "Senior Software Engineer," while Jira expects SWE-III to trigger the ticketing workflow correctly.

Without this key value pair table, the integration may create a ticket with incomplete data or route it incorrectly. By configuring a mapping, such as Senior Software Engineer to SWE-III, the integration standardizes input, prevents mismatches, and enables precise automation between HR and IT systems. For more details, refer to the Using Tables in Integrations guide.

Attribute Mapping

The Attribute Mapping tab is where you define how Dayforce attribute fields are translated into the corresponding Jira attribute fields required during ticket creation. For example, you may map Work Email from Dayforce to Jira’s Reporter field or Job Title to a custom field. These mappings ensure that the correct information is accurately populated in every Jira ticket. By using the mapping,  you enable the integration to align HR data with IT workflows, reduce data mismatches, and maintain process integrity across onboarding, offboarding, and other lifecycle events.

Run the Integration

The integration can be scheduled to run automatically at customizable intervals. Available frequencies include 15 minutes, 30 minutes, 1 hour, 4 hours, 6 hours, 12 hours, or 24 hours. You can also enable recurring executions to run the integration continuously on your chosen schedule.

On successful execution, the CSV file will be sent to the specified recipients. In the event of integration errors, notifications and reports will also be sent to the specified email addresses.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
