Dayforce to ServiceNow Data Synchronization

← Back to index

The Aquera platform allows your organization to efficiently scale complex data management by automating the integration of diverse resource data across different applications and systems.

The Dayforce to ServiceNow Data Synchronization integration automates the synchronization of employee and organizational data from Dayforce to ServiceNow. The integration ensures that workforce and organizational information in ServiceNow remains aligned with the source data in Dayforce.

The integration synchronizes the Users, Departments, Locations,a nd Job Families resources from Dayforce to ServiceNow.

In addition to data synchronization, the integration automatically creates and manages ServiceNow tickets for employee lifecycle events detected in Dayforce. These events include:

  • Employee onboarding
  • Employee role changes
  • Employee offboarding

During each integration run, the system compares records between Dayforce and ServiceNow and performs the necessary actions to keep the systems aligned. For example:

  • If a record exists in Dayforce but not in ServiceNow, the integration creates it in ServiceNow.
  • If a record exists in both systems but contains different values, the integration updates the ServiceNow record.
  • If a record exists in ServiceNow but no longer exists in Dayforce, the integration deletes the record when deletion is enabled.

The integration runs automatically based on the schedule configured in the Schedule tab.

Key Features

  • Automated Ticket Generation: Automatically creates and updates ServiceNow tickets for employee onboarding, role changes, and offboarding events detected in Dayforce.
  • Employee and Organizational Data Sync: Synchronizes users, departments, locations, and job families from Dayforce to ServiceNow to keep organizational data aligned.
  • Incremental Data Processing: Processes only records that have changed since the last integration run, improving efficiency and reducing processing time.
  • Flexible ServiceNow Request Options: Supports ticket creation using standard tickets, Service Catalog items, or Order Guides.
  • Pre-Event Ticket Creation: Allows onboarding and offboarding tickets to be generated before the event date to prepare access, equipment, and system updates.
  • Configurable Sync Controls: Provides filters to control which records are created, updated, or deleted during synchronization.
  • Record Processing Limits: Allows administrators to set limits on the number of records processed in each integration run.
  • Debug and Monitoring Tools: Includes debugging filters, execution logs, and reports to monitor integration activity and troubleshoot issues.

Ticket Generation Workflow

The integration automatically creates or updates ServiceNow tickets when employee lifecycle events occur in Dayforce.

The system evaluates employee event dates, such as:

  • Hire Date
  • Role Change Effective Date
  • Termination Date

Based on these events, the integration performs the following actions:

  • Creates a new ticket if no ticket currently exists for the event.
  • Updates an existing ticket if the event details change after the ticket has been created.

For example, if an employee’s hire date or termination date is modified in Dayforce, the integration updates the corresponding ServiceNow ticket to reflect the new information.

Tickets can be created using different ServiceNow request mechanisms depending on how the integration is configured:

  • Standard ServiceNow tickets (such as Incident, HR Case, or Change Request)
  • Service Catalog requests
  • Order Guides that request multiple items in a single workflow.

Integration Setup

This section describes how to configure the Dayforce to ServiceNow Data Synchronization in the Aquera Admin portal. It explains how to create the integration, configure parameters, and enable automated synchronization between the two systems.

The integration synchronizes employees, departments, locations, and job families from Dayforce to ServiceNow. It also generates ServiceNow tickets for employee lifecycle events, including onboarding, role changes, and offboarding.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce to ServiceNow Data Synchronization.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and ServiceNow connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Integration Scenarios

  • Enable Ticket Management: Select this checkbox to create and update onboarding and offboarding tickets in ServiceNow based on Dayforce events.
  • Enable Department Management: Select this checkbox to synchronize department records between Dayforce and ServiceNow.
  • Enable Location Management: Select this checkbox to keep location records in ServiceNow aligned with Dayforce organizational data.
  • Enable User Management: Select this checkbox to create, update, or deactivate ServiceNow user accounts from Dayforce employee records.
  • Enable Job Family Management: Select this checkbox to synchronize job family data from Dayforce to ServiceNow. When enabled, the Manage Job Families section is displayed to configure job family management integration settings.

Operational Settings

Configure the following execution related settings:

  • Enable Report-Only Mode: Select this option to generate a report detailing the data changes that would occur, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Email Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize the integration-related metrics.
  • Debug Settings: Select to generate logs outlining data changes that would occur for debugging.

Manage Tickets

This section allows you to configure how the integration creates and manages employee lifecycle tickets in ServiceNow based on events detected in Dayforce.

Correlation Settings

This section defines the attributes used to match employee records between Dayforce and ServiceNow when processing onboarding and offboarding events.

  • Source Correlation Attribute: Select the Dayforce employee attribute used to identify and match employee records from the Dayforce. For example, if EmployeeNumber is selected, the integration will use the employee’s EmployeeNumber value from Dayforce to identify the employee when processing onboarding or offboarding events.
  • Target Correlation Attribute: Select the ServiceNow employee attribute used to match the employee record with the value received from Dayforce. For example, if Employee number is selected, the integration will match the EmployeeNumber received from Dayforce with the Employee number field in ServiceNow to locate the correct employee record.

Onboarding Tickets Generation
This section allows you to configure how onboarding tickets are created in ServiceNow for new employees detected in Dayforce.

  • Enable Onboarding Ticket Generation: Select to enable onboarding ticket creation in ServiceNow for new hires from Dayforce. For example, when an employee’s status changes to Active in Dayforce, an onboarding ticket is automatically created in ServiceNow.
    • Statuses to fetch Dayforce Employees: Select the Dayforce employment statuses to include for onboarding ticket creation. For example, if the status Active is selected, the system will only evaluate employees whose Dayforce status is Active and will ignore employees marked as Terminated, Inactive, or other statuses.
    • Select Onboarding Date Type: Select which employee date from Dayforce is used to decide when the onboarding ticket should be created in ServiceNow. For example, if HireDate is selected and an employee’s hire date is July 1, the onboarding ticket will be created based on that July 1 date.
  • Enable Ticket Pre-Processing: Select to create onboarding tickets a few days before the onboarding date. For example, create tickets 5 days before an employee’s hire date to prepare access or assets in advance. Refer to the Sample Ticket Description Template.
    • Create ticket X-days before provided date: Enter the number of days before the selected onboarding date, the system should generate the ServiceNow ticket. For example, if you choose Hire Date and set the workflow to create a ticket 5 days before the date, the ticket will be created 5 days before the employee’s hire date. Note: The selected Onboarding Date Type determines the provided date.
  • Enable Employee Number Filter: Select to limit onboarding ticket creation to a specific employee number for testing. For example, use this to test ticket creation for employee number 100245 before enabling it for all employees.
    • Set Onboarding Employee Number Value: Enter the starting employee number that will be assigned to newly onboarded employees when the system automatically generates employee IDs during the onboarding process. This value acts as the base number from which the next employee numbers are incremented. For example, if the value is set to 2200, the first new employee created through onboarding will receive the employee number 2200, the next employee will receive 2201, then 2202, and so on as additional employees are onboarded.

  • ServiceNow Trigger: Select the method used to create onboarding requests in ServiceNow. For example, select Service Catalog Item if onboarding requests should be generated through ServiceNow catalog requests.

    The following options are available:

    • Ticket: Creates a standard ServiceNow ticket for the onboarding request.
      • Ticket Type: Select the ServiceNow ticket record type used for onboarding, such as Incident, Change Request, Problem, HR Case, Catalog Request, or Custom Ticket Type. For example, selecting HR Case creates onboarding requests as HR case records in ServiceNow.
    • Service Catalog Item: Creates an onboarding request using a ServiceNow service catalog item.
      • Service Catalog Service Request: Select the catalog item that should be requested. Available items are filtered based on the selected Service Catalog Category. For example, you may select items such as New Employee Laptop, VPN Access, or Email Setup.
    • Order Guide: Creates an onboarding request using a ServiceNow order guide, which groups multiple catalog items into a single request.
      • Order Guide: Select one or more order guides that should be triggered during onboarding. For example, an order guide such as New Employee Setup may include requests for Laptop, VPN Access, and Email Setup.

Offboarding Tickets Generation 
This section allows you to configure how offboarding tickets are created in ServiceNow when employees are terminated in Dayforce.

  • Enable Offboarding Ticket Generation: Select to enable automatic offboarding ticket creation in ServiceNow for employees marked as terminated in Dayforce. For example, when an employee’s termination date is reached, an offboarding ticket is automatically generated in ServiceNow.
  • Statuses to fetch Dayforce employees:* Select the Dayforce employment statuses to include for offboarding ticket creation. For example, choose statuses like Terminated or Inactive to capture employees leaving the organization.
  • Select Offboarding Date Type: Select the employee date field (for example, Termination Date or Last Working Day) used to determine when offboarding tickets should be created.
  • Enable Ticket Pre-Processing: Select to create offboarding tickets a few days before the onboarding date. For example, create tickets 3 days before the termination date to ensure system access removal in time. Refer to the Sample Ticket Description Template.
    • Create ticket X-days before provided date: Enter the offboarding date type that should be used to calculate when the ticket will be created. For example, if you choose Termination Date and set the workflow to create a ticket 3 days before the date, the ticket will be created 3 days before the employee’s termination date.
      Note: The selected Offboarding Date Type determines the provided date.
  • Enable Employee Number Filter: Select to limit offboarding ticket creation to a specific employee number for testing. For example, use this to verify the process for employee number 102198 before running for all terminations.
    • Set Offboarding Employee XRefCode Value: Specifies the minimum employee reference code used to identify employees for offboarding ticket creation. For example, if the value is set to 200562, the system will create offboarding tickets only for employees whose XRefCode is 200562 or higher.

  • ServiceNow Trigger: Select the method used to create offboarding requests in ServiceNow. For example, select Flow Designer Trigger if the offboarding process is handled through automated workflows configured in ServiceNow Flow Designer.

    The following options are available:

    • Ticket: Creates a standard ServiceNow ticket for the offboarding request.
      • Ticket Type: Select the ServiceNow ticket record type used for offboarding (for example, Change Request, HR Case, Problem, Catalog Request, or Custom Ticket Type). For example, select Change Request to create tickets that track employee deprovisioning tasks.
    • Service Catalog Item: Creates an offboarding request using a ServiceNow service catalog item.
      • Service Catalog Service Request: Select the specific ServiceNow catalog item that should be requested. The available items are automatically filtered based on the selected Service Catalog Category.
    • Order Guide: Creates an offboarding request using a ServiceNow order guide.
      • Order Guide: Select one or more ServiceNow order guides to trigger this workflow. An order guide groups multiple catalog items (such as hardware return, system access removal, and account deactivation) into a single request.

Manage Job Families 

This section allows you to configure how job family records from Dayforce are synchronized with ServiceNow, including creation, update, deletion, and filtering rules for job family data.

  • Source Correlation Attribute: Select the Dayforce attribute used to uniquely identify and match job family records between Dayforce and ServiceNow. For example, if XRefCode is selected, the integration will use the XRefCode value from Dayforce to identify and match job family records during synchronization.
  • Target Correlation Attribute: Select the ServiceNow attribute used to match job family records with the corresponding records from Dayforce. For example, if Job Family Name is selected, the integration will match the Job Family Name from Dayforce with the Job Family Name field in ServiceNow to locate the correct job family record.
  • Fetch Jobs without Departments: Allows job family records to be created in ServiceNow even if the associated department information is not available. For example, if a job family such as Data Engineering exists in Dayforce but does not have a department assigned, enabling this option allows the job family to still be created in ServiceNow.
  • Enable Job Family Creation: Allows the integration to create new job family records in ServiceNow based on job family data received from Dayforce. For example, if a job family named Data Science exists in Dayforce but does not exist in ServiceNow, enabling this option will create the Data Science job family in ServiceNow. The following option becomes visible only when Enable Job Family Creation is enabled:
    • Enable Creation Filter Condition: Allows you to define a filter that controls when job family records should be created in ServiceNow. For example, when Enable Job Family Creation is selected, the Enable Creation Filter Condition option appears, allowing you to configure filter rules for creating job families. When this option is enabled, the following fields become visible:
      • ServiceNow Attribute for Filter: Select the ServiceNow attribute used to filter job family records during creation. For example, if Job Family Name is selected, the integration will evaluate the Job Family Name field in ServiceNow when determining whether a job family should be created.
      • Match Condition: Select the rule used to compare the attribute value when applying the filter. For example, if Equals To is selected, the filter will match only job family records whose attribute value exactly matches the specified value.
      • ServiceNow Job Family Value: Enter the value used by the filter to determine whether the job family should be created. For example, if Job Family Name, Equals To, and Developers are configured, the integration will check whether the Developers job family exists in ServiceNow. If it does not exist, the integration will create it.
  • Enable Job Family Update: Allows the integration to update existing job family records in ServiceNow based on changes detected in Dayforce. For example, if the Engineering job family is updated in Dayforce, enabling this option allows the integration to update the Engineering job family record in ServiceNow. The following option becomes visible only when Enable Job Family Update is enabled:
    • Enable Update Filter Condition: Allows you to define a filter that controls which job family records should be updated in ServiceNow. For example, when Enable Job Family Update is selected, the Enable Update Filter Condition option appears, allowing you to configure filter rules for updating job families. When this option is enabled, the following fields become visible:
      • ServiceNow Attribute for Filter: Select the ServiceNow attribute used to filter job family records during updates. For example, if Job Family Name is selected, the integration will evaluate updates only for job family records matching the specified value.
      • Match Condition: Select the rule used to compare the attribute value during update filtering. For example, if Equals To is selected and the value is Engineering, only the Engineering job family will be updated.
      • ServiceNow Job Family Value: Enter the value used to determine which job family records should be updated. For example, entering Engineering ensures that only the Engineering job family is updated during synchronization.
  • Enable Job Family Deletion: Allows the integration to delete job family records in ServiceNow when they no longer exist in Dayforce. For example, if the Support job family is removed from Dayforce, enabling this option allows the integration to delete the Support job family record in ServiceNow. The following option becomes visible only when Enable Job Family Deletion is enabled:
    • Enable Delete Filter Condition: Allows you to define a filter that controls which job family records should be deleted in ServiceNow. For example, when Enable Job Family Deletion is selected, the Enable Delete Filter Condition option appears, allowing you to configure filter rules for deleting job families. When this option is enabled, the following fields become visible:
      • ServiceNow Attribute for Filter: Select the ServiceNow attribute used to filter job family records during deletion. For example, if Job Family Name is selected, the integration will evaluate deletion conditions based on the Job Family Name field in ServiceNow.
      • Match Condition: Select the rule used to compare attribute values when evaluating deletion filters. For example, if Equals To is selected and the value is Contractors, the filter will apply only to the Contractors job family.
      • ServiceNow Job Family Value: Enter the job family value used to determine which job family records should be deleted. For example, if Contractors is specified and the Contractors job family exists in ServiceNow but not in Dayforce, the integration will delete the Contractors job family record from ServiceNow.
  • Enable Sync Record Limits: Allows you to limit the number of job family records processed during each integration run. For example, enabling this option allows administrators to control how many job family records can be created, updated, or deleted in a single synchronization cycle. When this option is enabled, the following fields become visible:
    • Creation Record Limit: Specify the maximum number of job family records that can be created during a single integration run.For example, if 100 is entered, the integration will create no more than 100 job family records during a single run.
    • Update Record Limit: Specify the maximum number of job family records that can be updated during a single integration run. For example, if 100 is entered, only 100 job family records will be updated in that run.
    • Deletion Record Limit: Specify the maximum number of job family records that can be deleted during a single integration run. For example, if 10 is entered, the integration will delete no more than 10 job family records in that run.
  • ServiceNow Table Name: Enter the ServiceNow table where job family records will be synchronized. For example, if u_job_families is entered, the integration will create, update, and delete job family records in the u_job_families table in ServiceNow.

When this option is enabled, the following fields become visible:
    • ServiceNow Attribute for Deletion Filter: Select the ServiceNow attribute used to filter user records before deletion. For example, if Name is selected, the integration evaluates deletion conditions based on the Name field in ServiceNow.
    • Deletion Match Condition: Select the rule used to compare attribute values when applying the deletion filter. For example, if Contains is selected and the value Contractor is specified, the filter applies to users whose Name field contains Contractor.
    • ServiceNow User Value for Deletion Filter: Enter the value used to determine which user records should be deleted. For example, if Contractor is specified, the integration deletes user records containing Contractor if they no longer exist in Dayforce.
  • Enable Data Change Limits: Allows you to limit the number of user records that can be created, updated, or deleted during each integration run. For example, enabling this option allows administrators to control the number of changes processed in a single sync cycle to prevent unintended bulk modifications. The following options become visible only when Enable Data Change Limits is enabled:
    • User Creation Record Limit: Specify the maximum number of user records that can be created during one integration run. For example, if 50 is entered, the integration creates no more than 50 user records during a single run.
    • User Update Record Limit: Specify the maximum number of user records that can be updated during one integration run. For example, if 100 is entered, the integration updates no more than 100 user records during that run.
    • User Deletion Record Limit: Specify the maximum number of user records that can be deleted during one integration run. For example, if 10 is entered, the integration deletes no more than 10 user records during that run.
  • Enable Incremental User Sync: Allows the integration to synchronize only user records that have been created, updated, or deleted since the last successful integration run. For example, if only three employee records were modified in Dayforce, enabling this option ensures that only those three users are processed during synchronization instead of processing all user records.

Dayforce Department Debug Settings

The Dayforce Department Debug Settings has the following settings:

  • Dayforce Department Filter: Select a filter from the dropdown menu to retrieve data of a specific department from Dayforce and run the integration for debugging purposes. The following are the available options:
    • None: Select this to retrieve all department records from Dayforce without applying any filter.
    • XRefCode: Select this to retrieve a specific department using its XRefCode.
