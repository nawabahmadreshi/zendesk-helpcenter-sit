ADP Workforce Now API Central Workers to Okta Users Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the integration of data across resource types. 

This guide covers the end-to-end tasks in setting up an integration to sync the Workers resource of ADP Workforce Now API Central to the Users resource of Okta.

Prerequisites

The following are the prerequisites:

  • Admin Access to Okta: You must have admin access to your Okta instance. You can visit the Okta portal to get access to an Okta instance.
  • Enable XaaS access:  You must open a support case with Okta to enable XaaS in your Okta instance.
  • Source Priority for Username and Email Update: In case of username and email update for Okta profile, ensure that the username and email attributes in Okta are mastered by the correct system. Refer to the Source Priority section for more details.
  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure following application connectors in the Aquera Admin portal to integrate with Okta:
    • ADP Workforce Now API Central: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for details.
    • Okta XaaS: Refer to the Okta Anything-as-a-Source (XaaS) Configuration Guide for details.

For details on the limitations of Okta's Anything-as-Source (XaaS) solution, refer to the Okta documentation.

Source Priority

If your use case requires updating username and email for Okta profile, ensure that these attributes in Okta are mastered by the correct system. Perform the following steps to configure the source priority:

  1. In Okta Admin Console, navigate to Directory > Profile Editor and select the User Profile.
  2. On the Profile Editor page, scroll down to the Attributes section.
  3. Find username attribute and click the info (i) icon.
  4. Set the Source Priority as follows:
    • Select Inherit from Profile Source if you want ADP Workforce Now to be the source of truth to provide and maintain the attribute value.
    • Select Inherit from Okta if the attribute must be managed in Okta and then pushed to ADP Workforce Now. In this case, Okta becomes the source of truth.
  5. Repeat the same steps for the email attribute and set the appropriate source priority based on your requirements.

Limitation

If the Import Safeguard setting is enabled in Okta with a deletion % limit, integration will fail if it tries to delete more users than the configured deletion % limit.

To fix this, you must navigate to Reports > Import Monitoring, and resolve any such error manually.

Integration Setup

This section details how the integration must be configured to push workers' data from ADP Workforce Now API Central to Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now API Central Workers to Okta Users.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now API Central and Okta connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following options are displayed:

  • Preferences: In the Preferences section, specify the parameter values for integration execution as follows:
    • Report Only: Select True to generate a report that shows the data that would be changed, without actually making those changes. This can be selected for the first instance of execution to identify any issues. You can select False after verifying the report and confirming that the result is as expected.
    • Source Filter: Provide an expression that can be used for filtering the users during source import. For example, workers/workAssignments/reportsTo/positionID eq '93V000061'.
      The following table lists the supported filters:
      Filter Supported Operator Example Value
      reportsTo positionID eq workers/workAssignments/reportsTo/positionID eq '93V000061'
      homeOrganizationalUnits eq workers/workAssignments/homeOrganizationalUnits/nameCode/codeValue eq 'XXX' and workers/workAssignments/homeOrganizationalUnits/typeCode/code Value eq 'XX'
      statusCode eq workers/workAssignments/assignmentStatus/statusCode/codeValue eq 'T'
      workerld eq workers/workerID/idValue eq 'XXXXXXX'
      positionID eq workers/workAssignments/positionID eq '93V000061'
      Location Code eq workers/workAssignments/homeWorkLocation eq 'USA'
      Employee Name eq workers/person/legalName/givenName eq 'Name'
  •  Schedule: In the Schedule section, you can specify when the integration should run automatically. Refer to the Integration Schedule Setup Guide for further details.

Create Custom Attributes in Okta

You can create custom fields in Okta to include additional ADP Workforce Now API Central Worker attributes in the User object of Okta. You must configure all the custom attributes individually. After the configuration, Aquera automatically populates the fields that you have created in Okta from the Worker resource of ADP Workforce Now.

Perform the following steps to create custom attributes in Okta:

  1. Sign in to your Okta instance.
  2. Navigate to Directory > Profile Editor and locate the custom identity source integration you've created.
  3. Click Add Attribute to add a new attribute.
  4. Configure the attribute using the same name as ADP Workforce Now in the Variable name field and fill in other fields. 

    Notes:
      • You must remove special characters in the name and replace periods with underscores.
        For example, the equivalent name for the attribute primaryAssignment.baseRemuneration.effectiveDate would be primaryAssignment_baseRemuneration_effectiveDate.
      • You must remove spaces from a custom attribute that is part of personal, employment, and assignment section. For example, if Field Name is a custom attribute under personal section in the ADP Workforce Now, use personal_FieldName as the Variable name.
  5. Click Save to save the changes or Save and Add Another to create another custom attribute. 

Map Custom Attribute to Okta Profile

Perform the following steps to map the custom attributes to the Okta profile:

  1. Navigate to Mappings > Configure User mappings.
  2. Map the custom attribute to an equivalent attribute in the Okta user profile and click Save Mappings.
  3. If the equivalent attribute is not present in the default Okta user profile, open the default profile from Directory > Profile Editor.
  4. Click Add Attribute to add a custom attribute to the Okta default user profile.
  5. Fill in the attribute's display name, description, and set the user permission. Click Save Attribute to save the changes.
  6. Map the custom identity source attribute to the attribute in the Okta default profile and click Save Mappings.

Note: You can repeat these steps to add all employee-related attributes in ADP Workforce Now to Okta.
