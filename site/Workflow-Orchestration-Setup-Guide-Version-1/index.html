Overview

This guide explains how you can create an orchestration of type workflow. This orchestration enables you to set up a workflow of actions to be performed on users data across applications without actually writing the script. The various actions that can be performed on users data are import, create, update, delete, and so on. A javascript is internally created based on the workflow that you have setup. When the orchestration runs, this script is executed.

You can set the orchestration to run based on a schedule or based on the events (user data changes) in an application.

Setting Up a Workflow Orchestration

You can achieve automatic user onboarding process from an application to multiple applications with a single workflow orchestration setup. You can add multiple applications to the workflow, as required.

Prerequisites

Make sure that the application connectors are added and configured correctly in Aquera Admin console before creating the orchestration. Refer to the respective Aquera connector configuration guides for details.

Creating the Orchestration

  1. Log in to the Aquera Admin.
  2. Navigate to Orchestrations and click Create Orchestration. The Add Orchestration pop-up window is displayed.

     

  3. Enter a name for the orchestration in the Orchestration Name field.
  4. Click the Workflow tab and select Workflow.
  5. Click Done. The following page is displayed to setup the workflow. 

     

  6. Click Create to create the orchestration record. A confirmation message is displayed on the page.

The following procedures are involved in the Workflow orchestration setup:

  • Setting a Trigger for the Orchestration
  • Designing the Workflow
    • Actions
    • Flows
    • Functions
    • Tasks
    • Variables
  • Adding Tables

Setting a Trigger for the Orchestration

You can set one of the following triggers for the orchestration.

Event Trigger

The orchestration is triggered when specific events occur in an application. 

  1. In the Add Trigger section, select Event and then select the application from the Select Source drop-down list. The workflow will be executed when a specific event (for example, user creation) occurs in the selected application.
    When you click Create to create the workflow orchestration, a URL is generated for event orchestration. This is the URL to which all events from the selected application will be sent for processing. You have to copy this URL and paste it in the appropriate setup section of the application from which the event data needs to be received.
  2. Click Create Token to generate a token for authentication purpose. This token has to be saved securely and needs to be configured in the Identity Service Provider or in the specific application based on your use case.

 

Schedule

The orchestration runs based on the schedule set for the orchestration. For details on the schedule setup, refer to Scheduling the Orchestration Run.

 

Setting Up the Workflow

You can set up the workflow of tasks that must be executed when the orchestration runs. The workflow contains the series of operations that must be performed on the user objects across the selected applications. The following options are available to setup the workflow:

  • Actions
  • Flows
  • Functions
  • Tasks
  • Variables

Each of these options is explained in the following sections.

Actions

The Actions option enables you to add the required applications to the orchestration. You can also add the various methods (operations) and the Open API methods that need to be performed on users in those applications. Only the methods supported by a specific application connector are available for inclusion in the workflow. 

Adding Applications

The following procedure is used to add the applications in which the various user operations need to be performed.

  1. Click the Actions icon   and click Add Applications. A pop-up window is displayed as follows.

     

  2. In Application Name, search for the application to be added to the workflow and select the required application.
  3. In the Variable field, enter the variable name that must be used to refer to the application in the workflow. The orchestration script will have with this variable to refer to the application.
  4. Click Add to add the application to the Actions pane.

     

    You can add as many applications as required by following the above steps. 

    Note: 

    You can edit the application variable name or delete an application from the orchestration using the Edit and Delete options next to an application.

    You can delete an application only if it is not added to the workflow. If the application is added to the workflow, you must first remove it from the workflow before deleting it.
  5. Click Save Changes at the top right corner of the page to update the application details in the orchestration. A confirmation message is displayed on the page.

Flows

The Flows option lists the IF condition and the For Each loop that can be added to the workflow.

You can drag and drop these items to the workflow editor area. Refer to the Dragging and Dropping Workflow Items section for details on dragging and dropping items from the panel to the workflow editor area.

Each of the flows is explained in detail in the following table.

Flow Description

The For Each statement is used to execute a function or a block of code once for each item in an array.

Tasks

You can add people tasks to the workflow orchestration. People tasks can be created in Aquera Admin console for any critical tasks that need people involvement. For detail information about People Tasks, refer to the People Tasks User Guide.

When you click the Tasks icon, all the people tasks that have bee created so far in the Admin console are listed. You can drag an drop the required pople tasks that must be part of the workflow to the workflow editor area.

The following fields are displayed in all People Tasks:

Field Description
Assigned To

The email ID of the user to which the task must be assigned. When the orchestration runs, a people task instance is created and assigned to this user.
Due Date

The date by which the assigned person must perform an action on this task. 

Note: More fields may be displayed for a people task based on the number of data fields configured in that task definition.

Adding Tables

Tables are used when an attribute is empty or a particular data is not sent from an application, and the user wants to update a value for the attribute during the orchestration.

For example, if you want the hire date of one set of employees to be updated with one value and that of another set of employees with a different value during the user update/create operation, you can add a table containing the hire date of the first set of employees.

  2. Click Add Table. The Add Table window is displayed.

     

  3. Enter the table name and description.

    The table inputs can be entered either manually as rows or by uploading a CSV file.

    • To enter manually, the user can click Add Row.

Scheduling the Orchestration Run

You can set the frequency by which the orchestration must run.

  1. In the Add Trigger section, select Schedule and expand the section.

    You can either select a basic schedule or set an advanced schedule.

    Basic Schedule

    You can select a frequency by which the orchestration must run, from a pre-defined list of frequencies ranging from 15 minutes to 24 hours.

  2. Select Basic and then select the frequency of execution from the drop-down.

 

Note: If you select Never, the orchestration never runs automatically. You can manually run the orchestration when required.

Advanced Schedule

You can use the Advanced schedule option to schedule the orchestration as per your requirements.

  • Select Never if you do not want the orchestration to run automatically.
  • Select Minutes to schedule the orchestration to run every X minutes, where you need to select the value of X from the Repeat Every drop-down list.

 

  • Select Hourly to schedule the orchestration to run every X hours, where you need to select the value of X from the Repeat Every drop-down list.

  • Select Daily to schedule the orchestration to run every X days at a specific time, where you need to select X and the time from the corresponding drop-down lists. You can add up to a maximum of 5 day schedules, if required.

  • Select Weekly to schedule the orchestration to run on specific days of a week at a specific time. The time that you select here is the UTC time.

  1. Click Save Changes to save the orchestration schedule.
