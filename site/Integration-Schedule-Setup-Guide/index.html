Integration Schedule Setup Guide

← Back to index

Overview

This guide provides details about how to schedule integration to run automatically. Integrations can be scheduled to run automatically based on a timeframe of the user's choice.

Select one of the following schedule options depending on the Frequency by which an Integration should be run.
 

  • Basic
  • Advanced 

Note: The configuration and behaviour of the Weekly schedule under the Advanced setup have changed starting from version v5 of the User Integration and Script integration. Refer to the Advanced Schedule section for details.

Basic Schedule

You can set a basic schedule by which the integrations should be automatically run from the following list of time intervals:

  • 15 minutes
  • 30 minutes
  • 1 hour
  • 4 hours
  • 6 hours
  • 12 hours
  • 24 hours

Notes: 

  • The start time of the next integration run is based on the end time of the previous integration run. For example, if you have set the basic schedule as 1 hour and the end time of the last integration run is 1:20 UTC, then the next integration run will start at 2:20 UTC.
  • If you select Never, the integration never runs automatically. You can manually run the integration when required.

Advanced Schedule

You can set an advanced schedule for the integrations to run automatically. The following options are available under the Advanced setup:

Never

Select Never if you do not want the integration to run on a scheduled basis.

 

Continuous

Select Continuous to run the integration continuously. 
Note: There can be a delay of maximum 5 minutes to start the next instance of the integration after the previous instance ends. 

Minutes

Select Minutes to schedule the integration to run every X minutes, where you need to select X minutes from the Repeat Every drop-down list. The available options for the minutes are 10, 15, 20, and 30 minutes.
Note: The start time of the next integration run is based on the end time of the previous integration run. For example, if the end time of the previous integration run is 00:30 UTC and you have set the advanced schedule as 30 minutes, then the next integration run will start at 1:00 UTC.

Hourly

Select Hourly to schedule the integration to run every X hours, where you need to select X hours from the Repeat Every drop-down list. The available options for the hours are 1, 2, 3, 4, 6, 8, and 12 hours. 
Note: The start time of the next integration run is based on the end time of the previous integration run. For example, if the end time of the previous integration run is 2:00 UTC and you have set the advanced schedule as 2 hours, then the next integration run will start at 4:00 UTC.

 

Daily

Select Daily to schedule the integration to run every X days at a specific time, where you need to select X days and the time from the drop-down lists. You can set up a maximum of 8 daily schedules if required. The available options for the days are 1, 2, 3, 4, and 6. The Starting at(Time) that you select here is in UTC. 

Weekly

Select Weekly to schedule the integration to run on specific days of the week within a time period. You have the option to set the frequency of execution in minutes or hours.

You can select the days of the week on which the integration must run on a weekly basis. You can set when the integration must start on a day in the Start Time field.

You can choose whether to set the execution frequency in minutes or hours. The available options for the minutes are 10, 15, 20, and 30 minutes. The available options for the hours are 1, 2, 3, 4, 6, 8, and 12 hours. 

When you set the frequency in minutes or hours, you will see the option to set the end time. You must enable the End Time (Optional) toggle to set the end time. The integration will then run only within the specified start and end times. The Start Time and End Time are in UTC.


 

Weekly Schedule Setup in V5 Version of User Integration and Script Integration

Starting from version 5 of the User integration and the Script integration, the weekly schedule setup has the following two additional configurations. These configuration fields will be displayed only after you have saved the integration.

  • Start Date: Select the date from which the weekly schedule becomes effective. The integration will first run automatically on or after this date, depending on the selected weekdays and the week frequency set in the Repeat every field.
  • Repeat every: Specify how frequently the integration must run in terms of weeks. This option enables you to schedule the integration to run weekly, bi-weekly, once in 3 weeks, or once in 4 weeks. For example, if you set 2 in this field, the integration will run every 2 weeks on the selected weekdays.

Consider the following weekly schedule setup, for instance:

  • The current day is 16-May
  • Start Date is set to 20-May, which falls on Tuesday of the following week.
  • Repeat every is set to 2 weeks.
  • The schedule is configured to execute every Monday, Wednesday, and Friday

With the above schedule setup, the scheduled integration will first run on 21st May, which is the next available scheduled day (Wednesday) within the starting week (the start date 20-May is in this week).

If the Start Date was set to 24-May, the integration would not run in the following week since there are no scheduled days between the Start date and the end of the week. The first integration would instead run on 02-June as the 2-week interval is applied from the start date.

Notes

  • When integration is scheduled, the state column of the integration page displays the day and the UTC time of the integration run.   • The next instance of integration does not start unless the previous instance of the integration has stopped execution. In such scenarios, some of the scheduled integrations may be skipped.

Disable or Enable Scheduled Runs

You can enable or disable scheduled integration runs using the Schedule Enabled toggle at the top of the Schedule tab.

  • When the toggle is ON, the integration runs automatically based on the defined schedule.
  • When the toggle is OFF, scheduled runs are disabled, but the integration can still be executed manually.
    Note: Disabling an integration does not affect an execution that is already in progress.
