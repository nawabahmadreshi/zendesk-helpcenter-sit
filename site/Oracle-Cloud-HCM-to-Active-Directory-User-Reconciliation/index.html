Overview

Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT environments.

The Oracle Cloud HCM to Active Directory User Reconciliation integration automates the synchronization of employee lifecycle status between Oracle Cloud HCM and Active Directory. It ensures that user accounts in Active Directory accurately reflect the current status of employees in Oracle Cloud HCM.

In this integration, Oracle Cloud HCM acts as the source of truth for employee data. Active Directory uses this information to manage user accounts and access to enterprise systems. The integration matches Oracle Cloud HCM employees with Active Directory users using configurable fields and keeps both systems aligned.

The integration continuously evaluates employee status in Oracle Cloud HCM and automatically updates Active Directory. When an employee is no longer active, the corresponding Active Directory account is disabled. When an employee is rehired, the account can be re-enabled to restore access.

If configured, the integration can also identify Active Directory users who are not found in Oracle Cloud HCM and take action on those accounts to help maintain a clean and accurate directory.

Key Features

The Oracle Cloud HCM to Active Directory User Reconciliation integration performs the following key functions:

  • Automated User Deactivation: Automatically disables Active Directory accounts for employees who are no longer active in Oracle Cloud HCM.

  • Rehire Reactivation: Re-enables Active Directory accounts for employees who are rehired, restoring their access.

  • Configurable User Matching: Matches Oracle Cloud HCM employees with Active Directory users using configurable correlation fields.

  • Reverse Reconciliation: Optionally identifies and processes Active Directory users that are not found in Oracle Cloud HCM.

  • Scoped Execution: Supports filtering by specific users, user types, or Active Directory Organizational Units (OU) for controlled runs.

  • Lifecycle-Based Processing: Evaluates employee status based on termination, work relationships, assignments, and lifecycle timing rules.

  • Simulation Mode: Allows running the integration in report-only mode to review changes before applying them.

  • Processing Limits: Supports limits on the number of users fetched or deactivated to prevent large-scale unintended changes.

  • CSV Reporting: Generates detailed reports with reconciliation results for audit and tracking purposes.

  • Email Notifications: Sends notifications with execution summaries and reports to configured recipients.

Integration Setup

This section details the configuration and execution process for integrating Oracle HCM with Active Directory. It covers creating the integration, configuring system preferences, and setting up reporting and notification mechanisms to automate the reconciliation of user accounts between Oracle HCM and Active Directory (AD).

The integration extracts user data from Oracle HCM, identifies and processes corresponding accounts in Active Directory, and produces audit-ready reports. It automatically detects all employees who are no longer active in Oracle Cloud HCM (whether due to termination or the absence of active work relationships) and deactivates their associated user accounts in the Active Directory.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Oracle Cloud HCM to Active Directory User Deactivation.
  5. Enter a name and a description for the integration.
  6. Select your Oracle Cloud HCM and Active Directory connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to search and select the ADP Workforce Now (GEP) application from which the employee details need to be retrieved, and the Microsoft Dynamics 365 F&O application in which the employee details must be synced.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Operational Settings

Configure the following operational settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.

  • Enable Report Generation: Select to generate the integration report.

  • Enable Notifications: Select to enable the integration-related notifications.

  • Enable Logging: Select to generate logs outlining data changes.

  • Enable Log Insights: Select to visualize the integration-related metrics.

Correlation Settings

This section defines how users are matched between Oracle Cloud HCM and Active Directory:

  • Source Field for Correlation: Select the source field from Oracle Cloud HCM used to identify and match users in Active Directory. For example, PersonNumber can be used to uniquely identify employees.

  • Target Field for Correlation: Select the target field from Active Directory used to identify and match users with Oracle Cloud HCM records. For example, employeeID can be used to match employee records.

Oracle HCM Settings

This section controls how users are retrieved and filtered from Oracle Cloud HCM before reconciliation:

  • Log Users Not Found in Oracle HCM: Enable to log users that exist in Active Directory but are not found in Oracle Cloud HCM. For example, if a user exists in Active Directory but no longer has a record in Oracle HCM, the user will be listed in the report for review.

  • Enable Source Filter for Oracle Users: Enable to filter specific Oracle Cloud HCM users based on the selected correlation field. For example, you can limit the integration to process only selected employees instead of all users.

  • Filter on Oracle Correlation Field: Enter one or more values to filter Oracle Cloud HCM users using the selected correlation field. For example, if the correlation field is PersonNumber, entering 7170 will process only that employee, and entering 7170, 7200 will process multiple employees.

Active Directory Settings

This section defines how user accounts are managed and updated in Active Directory during the integration:

  • Enable User Deactivation: Enable to allow deactivation of Active Directory user accounts. For example, when an employee is terminated in Oracle Cloud HCM, the corresponding Active Directory account is disabled.

  • Reactivate Rehired Oracle HCM Users: Enable to automatically reactivate Active Directory accounts for employees who are rehired in Oracle Cloud HCM. For example, if a former employee rejoins the organization, their account is re-enabled.

  • Deactivate Users Without Work Relationships: Enable to deactivate users who do not have active work relationships or assignments in Oracle Cloud HCM. For example, a user with no active assignments is considered inactive, and their account is disabled.

  • Deactivate Users Missing from Oracle HCM: Enable to deactivate Active Directory users that are not found in Oracle Cloud HCM. For example, if a user exists in Active Directory but has no corresponding record in Oracle HCM, the account is disabled.

  • Enable OU-Based Filter for AD Users: Enable to filter Active Directory users based on Organizational Units (OU). For example, you can restrict processing to users within specific departments or locations.

  • Filter AD Users by Organizational Unit (OU): Enter one or more Organizational Units to limit which Active Directory users are processed. For example, entering OU=India,DC=company,DC=com processes only users within that OU.

  • Enable Limited Deactivations/Rehires: Enable to limit the number of users processed for deactivation or reactivation in a single run. For example, you can control changes during testing or phased rollout.

  • Maximum Deactivation/Rehire Count: Enter the maximum number of users that can be deactivated or reactivated in one run. For example, setting 600 ensures that no more than 600 users are updated in a single execution.

Report Settings

This section defines how integration reports are generated and made available after each run.

  • Enable Integration Report Download: Enable to generate and download a CSV report for each integration run. For example, after the integration completes, you can download a report that shows all processed users, actions taken, and results.

Notification Settings

This section defines how email notifications are sent for integration runs and their outcomes:

  • Send Email Notfication: Enable to send email notifications for integration runs. For example, admins receive an email each time the integration completes.

  • Send Integration Report via Email: Enable to send the CSV report as an email attachment after each run. For example, the report is automatically shared with stakeholders after execution.

  • Recipients' Email Addresses: Enter the email addresses of recipients who should receive notifications. For example, admin@company.com, and  itteam@company.com will receive all integration updates.

  • Email Subject on Successful Integration Runs: Enter the subject line for success notifications. For example, Active Directory / Oracle HCM Reconciliation - Success.

  • Email Body on Successful Integration Runs: Enter the email content for successful integration runs. For example, include a summary of processed users and actions taken.

  • Enable Email Notification for Failed Integration to DevOps: Enable to send failure alerts to the DevOps team. For example, if the integration fails, the DevOps team is notified immediately.

  • Email Body on Failed Integration Runs: Enter the email content for failed integration runs. For example, include error details and failure reasons to assist troubleshooting.

Log Settings

This section defines how integration logs are generated and the level of detail captured during execution:

  • Log Level: Select the level of detail to capture in integration logs. For example, choosing Summary displays only operation counts, while Full includes detailed information about each processed user.
    The following are the available options:

    • None: Do not generate logs. For example, no execution details will be recorded.

    • Summary: Capture high-level information such as counts of processed, deactivated, and reactivated users. For example, you can quickly see how many users were updated.

    • Full: Capture detailed logs for each user processed, including actions taken and outcomes. For example, you can trace exactly what happened to each user during the run.

Log Insights

This section defines how integration analytics and metrics are generated and displayed on the dashboard.

  • Enable Dashboard Analytics Generation: Enable to generate analytics and insights for the integration on the Aquera dashboard. For example, you can view metrics such as number of users processed, deactivated, and reactivated after each run.
