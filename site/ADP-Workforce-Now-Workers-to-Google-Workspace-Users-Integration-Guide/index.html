Overview

The Aquera platform allows your organization to scale complex data management by automating data synchronization between various applications.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide details how to set up an integration that synchronizes Workers from ADP Workforce Now to Users in Google Workspace.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Connectors in the Aquera Admin Portal: You need to configure the following connectors in the Aquera Admin portal for the integration:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Google Workspace: Refer to the Google Workspace Configuration Guide for details.

Integration Setup

Aquera integration automatically imports workers from ADP Workforce Now and provisions these users in Google Workspace.

Create an Integration

Perform the following steps to create an integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now Workers to Google Workspace Users.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Google Workspace connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Preferences

The Preferences section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Admin Name: Enter the Admin's name that will be displayed in the email notification. For example, admin1.
  • Admin Email: Enter the admin's email address to which notifications on this integration have to be sent. For example, admin1@example.com.

Trigger

The Trigger section enables you to schedule the execution of the integration.

You can identify one of the following types to trigger the execution:

  • Event: You can identify an event source to trigger the execution.
  • Schedule: You can specify when the integration should run automatically. Refer to the Integration Schedule Setup Guide for further details.

Tables

The Tables section enables you to add tables in the integration. Tables define a key-name and key-value pair to create custom attributes. Refer to the Using Tables in Integrations guide for more details.

Integration Details

In the Integration Details section, you can view the execution history, author name who created the integration, and date when the integration was created.

You can download a report of an integration run in CSV format by clicking the execution history indicators and navigating to the Execution Timeline panel.

Run the Integration

You need to set up an event or a schedule trigger for executing the integration. For further details on how to set up triggers, refer to the Integration Using Scripts guide. 

After the integration is successfully executed, Aquera synchronizes the workers from ADP Workforce Now to users in Google Workspace.

Notes

  • When a new user is created in Google Workspace, a unique ID will be generated automatically.
  • A welcome email will be sent to the new user with their username and a link to set a password for signing into their Google account.
