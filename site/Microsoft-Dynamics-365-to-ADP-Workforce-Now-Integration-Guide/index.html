Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process for setting up an integration to generate a CSV report containing time entries and labor charge fields from Microsoft Dynamics 365 to ADP Workforce Now. The CSV report is then shared with the specified admins via email.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Microsoft Dynamics 365: Refer to the Microsoft Dynamics 365 Configuration Guide for details.
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.

Integration Setup

This section outlines the configuration required to send the generated CSV file to the designated users via email and to display the integration results on the dashboard. It covers creating the integration, configuring system preferences, scheduling the integration to run, and distributing the generated CSV to the designated users. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select <Integration Template Name>.
  5. Enter a name and a description for the integration.
  6. Select your Microsoft Dynamics 365 and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
  • Enable Analytics: Select this checkbox to analyze integration-related metrics in a dashboard.

Source Settings

Configure the following source application settings:

  •  
  • Provide the NetSuite Custom Field Name for the Position ID: Enter the custom field name to store the employee's position ID within NetSuite.
  • Provide the NetSuite Custom Field Name for the Serial Number: Enter the custom field name to capture the serial number associated with the NetSuite TimeBill.
  • Source Time Entries Filter Condition: Select the source filter to filter the employees' time entries while retrieving from NetSuite.
  • Fetch TimeBills From NetSuite Based on X Days: Enter the number of days before the current date to retrieve the TimeBills from NetSuite.
  • Select Time Type from NetSuite: Select the time type for the employees' time entries while retrieving from NetSuite.
  • Select the NetSuite Correlation Attribute: Select the NetSuite attribute to correlate and match employee records between the source and target applications.

Target Settings

Configure the following target application settings:

  • Create Labor Codes in ADP Workforce Now: Select to create Labor Charge Fields (LCFs) in ADP Workforce Now based on the employees' project ID and serial number.
  • Select the ADP Workforce Now Correlation Attribute: Select the ADP Workforce Now attribute to correlate and match employee records across source and target applications.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values when choosing the Full Details option.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.

Report Settings

Configure the following report generation settings:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Enable Integration Report Download: Select this checkbox to allow downloading the integration report in CSV format.

Notification Settings

Configure the following settings for email notifications:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
  • Admin Email Subject on Failed Integration Execution: Enter the content that must be used as the email subject in the notifications sent for integration errors.

Analytics Settings

Configure the following analytics settings:

  • Select the Analytics Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details option.

Attribute Map

The Microsoft Dynamics 365 to ADP Workforce Now integration maps the Microsoft Dynamics 365 and ADP Workforce Now attributes.

The attributes displayed in this section depend on the type of synchronization the integration supports:

  • Dynamics 365 to ADP Workforce Now Time Entries Mapper: For timesheet synchronization, the integration uses Procore attributes as the source and maps them to the corresponding ADP Workforce Now attributes as the target.
  • Dynamics 365 to ADP Workforce Now Labor Charge Fields Mapper: For user synchronization, the integration uses ADP Workforce Now attributes as the source and maps them to the appropriate Procore attributes as the target.

Only the source attributes can be modified, as needed, to align with your specific use case.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.

The following tables are available:

  • Labor Charge Fields Allocation Lookup: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options: <Depending upon the integration use case, you can document accordingly>

    • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
    • Upload CSV: Click Upload CSV to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
      Notes:
      • The key name within a table should be unique. The key name can include a combination of letters and numbers.
      • The key name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

    Click Save to save the changes made.
