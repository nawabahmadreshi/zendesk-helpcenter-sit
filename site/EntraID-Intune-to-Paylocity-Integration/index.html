EntraID (Intune) to Paylocity Integration

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The integration connects Microsoft Entra ID (Intune) and Paylocity to keep device ownership records accurate and up to date. It uses Microsoft 365 App Registration for secure authentication, allowing Aquera to read device and user data. It reads device data such as serial number, userPrincipalName (email), and companyName from Entra ID, and matches them with employee records in Paylocity using the configured custom text fields and company ID. Once matched, the integration updates or removes serial numbers in the employee’s Paylocity record to ensure each asset is correctly linked to its owner in Entra ID.

The integration identifies matching user records by comparing the userPrincipalName (email) and companyName attributes from Entra ID with the corresponding email address and company ID in Paylocity. If no matching employee is found in Paylocity, the device is moved to the free pool, indicating it is not currently assigned to any user.

Key Functions

The Microsoft Entra ID (Intune) to Paylocity Asset Synchronization integration performs the following key functions:

  • Uses the Microsoft 365 App Registration to connect securely with Microsoft services and access data from Intune and Entra ID.
  • Reads device details such as serial number, email (userPrincipalName), company name, and model from Microsoft Intune.
  • Retrieves employee records from Paylocity for the configured company IDs.
  • Matches each device from Intune with the correct employee in Paylocity using email and company ID.
  • Updates or removes the serial number in Paylocity so that each device is linked to the right employee.
  • Moves devices without a matching employee to the free pool, indicating they are unassigned.
  • Creates a summary report of all updates, skips, and errors, and can email the report to administrators.

Integration Setup

This section outlines how the integration must be configured to synchronize device ownership and asset records between Microsoft Entra ID (Intune) and Paylocity.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left-side panel.
  3. Click Create Integration.
  4. Select Microsoft Entra ID (Intune) to Paylocity Asset Synchronization from the list of available integrations.
  5. Enter a name and a description for the integration.
  6. Select your connector instances for Microsoft Intune, Microsoft 365 App Registration, and Paylocity.
  7. Click Continue. The Configuration page is displayed to configure the integration settings. 

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Microsoft Entra ID (Intune), Microsoft 365 App Registration, and Paylocity application connectors used for the integration. You can change or update these connector instances if required.

Parameters

The Parameters section allows you to customize the integration behavior according to your specific requirements. It includes the following settings:

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Notifications: Select this checkbox to enable email notifications related to the integration. The Notifications section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.
  • Enable Report Generation: Select this checkbox to generate the integration report. The Report Settings section will be displayed only when this checkbox is selected.

Advanced Settings 

Configure the following advanced settings:

  • Enable Asset Management: Select to activate the asset management feature for the company. When enabled, the integration manages device ownership in Paylocity based on asset data from Microsoft Entra ID (Intune). 

Asset Management

Source Settings

Configure the following settings in EntraID:

  • Advanced Sync & Filters: Select to filter and synchronize device records during each run.
    Options include:
    • Source Filter: Apply a specific filter to limit the devices synchronized.
    • Incremental Sync: Sync only updated records since the last run.
    • Full Run: Synchronize all available devices without filtering.
  • Device Sync Filter: Select to define the attribute used to identify which devices are synchronized from Microsoft Entra ID (Intune).
    Options include:
    • Microsoft Entra Device ID: Filter devices using their unique Entra device ID.
    • Model: Filter devices based on the model type.
  • Source Filter Value: Enter the value that corresponds to the selected device filter.
    The integration retrieves only those Intune devices that match this filter value.

Target Settings

Configure the following settings in Paylocity:

  • Company IDs: Enter one or more Paylocity company IDs to specify which organizations will be managed in this integration. This defines the scope of employee data for synchronization.
  • Number of Devices to Sync: Enter the maximum number of devices to include in a single integration run. Use this to limit batch size and improve performance when processing large datasets.
  • Custom Label Name for Correlation: Enter the name of the custom text field in Paylocity used to store and match device serial numbers. The integration uses this field to correlate and update device ownership records accurately.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu. 
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: Controls which event attributes are included or excluded.   
        • Included Attributes: Select the attributes to be included in the log.
        • Excluded Attributes: Select the attributes to be excluded from the log.

Report Settings

Configure the following settings to customize the CSV report:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following options only appear when the report detail level is set to Custom.
      • Attribute Filter: Controls which attributes are included or excluded in the report.
        • Included Attributes: Select the attributes to be included in the CSV report.
        • Excluded Attributes: Select the attributes to be excluded from the CSV report.

Notification Settings

Configure the following settings for email notifications:

  • Notify Admin Via Email: Select to send email notifications to all admins.
    • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications must be sent.
    • Admin Email Subject: Enter the content that must be used as the email subject for the email notifications to the admins.
    • Send Integration Report via Email: Select to receive email notification with a report for all integration runs.
    • Select the Notification Report Detail Level: Select the desired level of notification detail from the dropdown menu.
      The following are the available options:
      • Summary only: Attribute details are not populated, and it displays the count of CRUD operations.
      • Full details: All attribute details are populated.
      • Custom: Includes or excludes selected attribute details.
    • Notify Employees via Email: Select to send email notifications to all employees.

Log Insights Settings

Configure the following log insight settings:

  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Custom Analytics Index Name: Enter a custom name for the analytics index to store and organize integration log data.

Attribute Map

The Attribute Mapping tab defines how source system attributes from Microsoft Entra ID (Intune) correspond to target fields in Paylocity. Administrators can select the source system (Intune) for each mapping and specify the source attribute or expression that should populate a specific Paylocity custom field.

The mapping ensures that device and asset information from Microsoft Intune, such as Model, Device ID, Manufacturer, Operating System, and OS Version, are accurately reflected in Paylocity employee records.

Note: The custom field names in Paylocity may vary depending on your company’s configuration. Verify that the selected fields match your organization’s defined custom text fields before running the integration.
