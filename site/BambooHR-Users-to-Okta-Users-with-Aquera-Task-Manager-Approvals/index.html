BambooHR Users to Okta Users with Aquera Task Manager Approvals

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the synchronization of data across resource types. This guide covers the end-to-end process of setting up a script synchronization to sync the Users resource of BambooHR to the Users resource of Okta based on approvals from Aquera Task Manager.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure following application connectors in the Aquera Admin portal:
    • BambooHR: Refer to the BambooHR Configuration Guide for details.
    • Okta: Refer to the Okta Configuration Guide for details.

Synchronization Setup

This section details how the synchronization must be configured to allow data push from BambooHR to Okta based on approvals from Aquera's Tasks module.

Architecture Overview

When an employee is created in BambooHR, Aquera checks if the user is already present in Okta. If the user is not present, Aquera checks if there is an existing user provisioning task in Aquera Task Manager. The following actions take place based on the presence of a task:

  • If there is an existing task for the user in Aquera Tasks, Aquera checks the task's status. The user is synced to Okta when the task is in Approved status.
  • If a matching task is not found in Aquera Tasks, Aquera creates a task for the user in Aquera Tasks and assigns it to the assignee (approver). You can configure the assignee through the integration script. The user is synced to Okta when the task is in Approved status.

Create Script Synchronization

Perform the following steps to create a new script synchronization:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations from the left side panel.
  3. Click Create Synchronization.
  4. Select BambooHR Users to Okta Users with Aquera Task Manager.
  5. Enter a name and a description for the synchronization.
  6. Select your BambooHR and Okta connector instances.
  7. Click Continue. A new page is displayed to configure the synchronization.

Run Script Synchronization

You can set event or schedule triggers to run the script synchronization. For more details, refer to the Executing a Synchronization section.

After the script synchronization is successfully executed, Aquera syncs the employees created in BambooHR as Users in Okta based on the approval in Aquera Tasks.
