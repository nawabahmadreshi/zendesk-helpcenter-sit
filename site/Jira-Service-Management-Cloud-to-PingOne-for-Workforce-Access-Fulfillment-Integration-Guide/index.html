Jira Service Management Cloud to PingOne for Workforce Access Fulfillment Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the setup involved in the integration for synchronizing the user's privileges from Jira Service Management Cloud to the user groups in PingOne for Workforce.

To identify the matching user records across both systems, the integration reads one of the following fields in the tickets from Jira Service Management Cloud and matches it against the email or username attribute in PingOne for Workforce, based on your integration configuration:

  • Email address
  • Corporate email address

Key Functions

The Jira Service Management Cloud to PingOne for Workforce Access Fulfillment integration performs the following functions:

  • Retrieves all the tickets in a specific status that belong to a specific project and work type from Jira Service Management Cloud
  • Reads the email address or the corporate email address of the user from each ticket
  • Identifies a user with the matching email address or username in PingOne for Workforce, as per your configuration
  • Reads the privileges of the user from the Jira ticket and updates the user's group information in the PingOne for Workforce with those privileges.
  • Transition the ticket to a specific status, based on your configuration

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • PingOne for Workforce: Refer to the PingOne for Workforce Configuration Guide for details.
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured for granting access to users in PingOne for Workforce, based on their privileges available in Jira Service Management Cloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Jira Cloud to PingOne for Workforce Access Fulfillment.
  5. Enter a name and a description for the integration.
  6. Select your PingOne for Workforce and Jira Service Management Cloud connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the PingOne for Workforce and Jira Service Management Cloud application connectors used for the integration. You can change the application instances used by the integration, if required.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the integration features via this section.

  • Enable Manage Access: Select this checkbox to manage access fulfillment. This is a mandatory setting. 
    Note: The Manage Access section is displayed only when you select this checkbox.

General Settings

Configure the following general settings, based on your requirements:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Notifications: Select this checkbox to enable email notifications related to the integration. The Notifications section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report. The Report Settings section will be displayed only when this checkbox is selected.

PingOne Environment

Select the specific PingOne environment that must be used in the integration.

Note: Make sure that you click the Refresh icon next to the parameter field to refresh the values populated in the field.  Otherwise, you will see only the values that are available by default in the integration template.

  • Select PingOne Environment ID: Select the PingOne Environment ID from the dropdown, where the user's group information has to be updated. For example, Administrators.
    Note: Whenever you change the environment ID, make sure that the corresponding client ID (as the username) and client secret (as the password) are also updated in the PingOne for Workforce connector to enable authentication and update group information. 

Manage Access

Set the configurations related to the access fulfillment.

Note: Make sure that you click the Refresh icon next to each parameter field in the Manage Access section to refresh the values populated in those fields. Otherwise, you will see only the values that are available by default in the integration template.

Access Requirement Fulfillment

Configure the following access fulfillment settings:

  • Project Name: Select the Jira project from which the tickets need to be retrieved. For example, Integration
  • Work Type: Select the type of tickets that need to be retrieved. For example, select Joiner to retrieve only the joiner tickets.
  • Ticket Pickup Status: Select the status of the tickets that must be retrieved by the integration for processing. For example, select Open to retrieve all the tickets of Open status.
  • Post-Update Status: Select the status to which a ticket must be transitioned after the user's privileges are successfully updated in PingOne. Select None if you do not want the ticket status to be changed.
    Note: The user privileges are not updated and the ticket status will not be changed in the following scenarios:
    • No matching user is found in PingOne
    • Multiple matching user records are found in PingOne

Correlation Field Settings

Configure the following fields to define how users from Jira tickets are matched with user accounts in PingOne:

  • Jira Correlation Field: Select the Jira ticket field that must be used for identifying a user. For example, Corporate email address.
  • PingOne Correlation Field: Select the corresponding target attribute in PingOne for Workforce against which the Jira Correlation Field value will be matched to find the user. For example, email

Notification Settings

Configure the following settings for email notifications:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications on integration runs will be sent.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.

Note: The following counts are displayed in a tabular format in the email body:

  • Total number of Jira tickets retrieved
  • Successful user group updates
  • Failed user group updates
  • Skipped users
  • Users not found in PingOne
  • Duplicate users found in PingOne

Log Settings

Configure the following settings to customize the integration logs:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary: Only the count of operations is displayed in the logs.
    • Full: In addition to the count of operations, the logs also display the transaction status, the ticket details, the groups assigned and removed from users, and the ticket transition status.

Note: The following counts are displayed in the logs:

  • Total number of Jira tickets retrieved
  • Successful user group updates
  • Failed user group updates
  • Skipped users
  • Users not found in PingOne
  • Duplicate users found in PingOne

Report Settings

The CSV report will be sent as an attachment with the email notifications if the notification settings are enabled. The report can also be downloaded by navigating to the Execution Timeline panel and clicking Download Report.

Configure the following settings to customize the CSV report:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary: Only the count of operations is displayed in the report.
    • Full: The report displays the transaction status, ticket details, groups assigned and removed from users, and the ticket transition status.

Attribute Map

The Attribute Map controls what attribute value from Jira must be updated to the group attribute in  PingOne. By default, the integration is configured to update Privileges values from Jira Service Management Cloud to the group attribute in PingOne for Workforce. 

Modify the source attributes, if required, based on your use case.

Note: The update to the Jira issue description will occur only when Ignore Description Update is set to false. If Ignore Description Update is true, the description will be skipped during the update.
