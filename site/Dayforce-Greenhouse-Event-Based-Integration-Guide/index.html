Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing Greenhouse Recruiting hire and unhire webhooks with Dayforce to enable employee creation, updates, rehires, and terminations, including optional signed offer letter upload to the employee record.

Key Functions

The Greenhouse Recruiting to Dayforce event-based integration provides the following functions:

  • Synchronizes hire, rehire, internal move, and unhire events from Greenhouse to Dayforce, creating, updating, reactivating, or terminating employee records as needed.
  • Uploads signed offer letters from Greenhouse to the employee’s document file in Dayforce.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for details.
    • Dayforce: Refer to the Dayforce Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push hire, rehire, internal move, and unhire events from Greenhouse Recruiting to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Greenhouse Recruiting [Event] in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable candidate to employee sync: Select this checkbox to sync hired candidates from Greenhouse Recruiting into Dayforce as employees. When enabled, the Candidate Configuration section is displayed for configuring additional integration settings. 

General Settings

Configure the following general settings:

  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable log insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.

Sensitive Fields

The Sensitive Field Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • PII Fields: Enter the field names that contain personally identifiable information (PII).
  • GUID Fields: Enter the field names that contain globally unique identifiers (GUIDs).
  • Employee ID Fields: Enter the field names that contain employee IDs.

Log Settings

Notification Template Settings

Configure the following notification template settings:

Notification Report Settings

Configure the following settings for the notification reports:

  • Select report type: Select the desired report type from the dropdown menu. For example, select Employee Number with Non-PII Data.
  • Select report detail level: Select the desired report detail level. The following are the available options:
    • Full details: All attribute details are displayed.
    • Summary only: Attribute details are not displayed.
    • Inclusion list: Select this option to report only the attributes specified in the Custom inclusion list.
      • Custom inclusion list: Select the attributes to be included in the report.
    • Exclusion list: Select this option to report only the attributes specified in the Custom exclusion list.
      • Custom exclusion list: Select the attributes to be excluded from the report.

Event

The Event tab is used to configure the webhook details. 

Provide the following details and save the changes:

  • Event Source: Select your Paycor application from which employee-termination events are triggered.
  • URL: Copy this URL and paste it into the appropriate webhook field on the Paycor Webhooks setup page. For details on setting up a webhook in the Paycor application, refer to the Set Up Webhooks in Paycor guide.
  • Refresh Token: Click Refresh Token to generate a new token. Copy the generated token and paste it into the Event Secret field of the Paycor webhook under the Event Subscription tab on the Demo Application page. For details on setting up a webhook in the Paycor application, refer to the Set Up Webhooks in Paycor guide.
