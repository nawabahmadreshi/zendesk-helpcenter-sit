Aquera Contractor Management to Okta Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide describes the setup required for integrating the contractors from Aquera's Contractor Management application with Okta. It is designed to handle HR use cases such as onboarding new contractors, updating existing contractor information, and offboarding contractors who have ended their engagements. 

Key Functions

The Aquera Contractor Management to Okta integration performs the following functions:

  • Retrieves all the contractors from Aquera Contractor Management
  • Creates all the contractors, contractors in Engaged status, or a specific set of contractors in Okta (based on an integration setting)
  • Deletes contractors in Assigned status or a specific set of contractors from Okta (based on an integration setting)

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Aquera Contractor Management: Refer to the Aquera Contractor Management Configuration Guide for details.
    • Okta XaaS: Refer to the Okta XaaS (Anything-as-a-Source) Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured for synchronizing users from Aquera Contractor Management to Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Aquera Contractor Management to Okta.
  5. Enter a name and a description for the integration.
  6. Select your Aquera Contractor Management and Okta XaaS connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Aquera Contractor Management and Okta XaaS application connectors used for the integration. You can change the application instances used by the integration, if required.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the integration features via this section.

  • Enable Contractor Management: Select this checkbox to integrate contractors from Aquera Contractor Management with Okta users. This is a mandatory setting. 
    Note: The Contractor Settings section is displayed only when you select this checkbox.
  • Enable Deletion of User when Deactivated in Okta: Select this checkbox to delete the user record upon deactivation in Okta.
    Note: The Okta Deletions section is displayed only when you select this checkbox.
  • Link Manager in Okta: Select this checkbox to link the manager custom object in Okta.
  • Enable Okta Email Writeback: Select this checkbox to enable updating the email ID of contractors from Okta to the Aquera Contractor Management application.
  • Enable Auto-Creation of Custom Attributes in Okta: Select this checkbox to authorize the creation of new custom attributes in Okta.
    Note: The Target Settings section is displayed only when you select this checkbox.
  • Generate Unique Username/Email: Select this checkbox to automatically generate unique usernames and email IDs in Okta.
    Note: The Unique Identity section is displayed only when you select this checkbox.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. 
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging. Note: The Contractor Management Debug Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report. 
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable email notifications related to the integration. 
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. 
    Note: The Log Insights Settings section will be displayed only when this checkbox is selected.

Contractor Settings

The Contractor Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if you have selected the Enable Contractor Management setting in the Parameters section.

  • Users to be created in Okta*: Enables you to choose which contractors must be created as users in Okta. The following options are available:
    • All Contractors: Select this option to create all the contractors as users in Okta.
    • All Engaged Contractors: Select this option to create only those contractors in Engaged status as users in Okta.
    • Custom: Select this option to define a JavaScript expression that specifies the upsert condition for creating a contractor in Okta. A field, Custom Upsert Condition, is displayed to configure the expression.
  • Delete Condition Type*: Select the delete condition type to specify which contractor records to be deleted from the custom application (identity source) in Okta.
    The following are the available options:
    • Basic: Select to set a delete condition based on the contractor's status.
    • Custom: Enter a JavaScript expression that determines whether a contractor must be deleted in Okta. Make sure that the expression returns a boolean value (true or false). This field is displayed only if you selected Custom from the Users to be deleted in Okta field.
  • Users to be deleted in Okta: Allows you to choose which contractors must be deleted in Okta. The following options are available:
    • All Invited Contractors: Select this option to delete contractors who have an Invited status.
    • All Submitted Contractors: Select this option to delete contractors who have a Submitted status.
    • All Validated Contractors: Select this option to delete contractors who have a Validated status.
    • All Unengaged Contractors: Select this option to delete contractors who have an Unengaged status.
    • All Engaged Contractors: Select this option to delete contractors who have an Engaged status.
    • All Assigned Contractors: Select this option to delete contractors who have an Assigned status.
    • All Deactivated Contractors: Select this option to delete contractors who have a Deactivated status.
  • Organization ID: Enter the organization ID of the contractor.
  • Enable Incremental Mode: Select this checkbox to run the integration in incremental mode.

Target Settings

This section enables you to customize the target settings according to your specific use case.

Note: This section appears only if you have selected the Enable Auto-Creation of Custom Attributes in Okta setting in the Parameters section.

It has the following settings:

  • Custom Attributes to be Created in Okta: Select the Aquera Contractor Management attributes that are not available in Okta by default and must be created as custom attributes in Okta.

    Notes: 

    • When you select an attribute in Aquera Contractor Management, a corresponding custom attribute will be populated in the Attribute Map tab under the target application column (once you refresh the list). This allows you to map Aquera Contractor Management attributes to the custom attributes that will be created in Okta.
    • When you add a new attribute in Aquera Contractor Management, click the refresh icon next to the Custom Attributes to be Created in Okta field. This ensures that the new attribute appears in the attribute list for selection.
    • The selected custom attributes will be automatically created in Okta during the next integration run, and their data will be synchronized in subsequent runs.

Logging Settings

Configure the following settings to customize the logs:

Note: This section appears only if you have selected the Enable Logging setting in the Parameters section.

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the log.
      • Excluded Attributes: Select the attributes to be excluded from the log.

Notification Settings

Configure the following settings for email notifications:

Note: This section appears only if you have enabled the Enable Notifications setting in the Parameters section.

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications on integration runs will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs. If the checkbox is not selected, the email is sent without a CSV report attachment.

Unique Identity Settings

Configure the following settings to customize the unique identity parameters:

Note: This section appears only if you have selected the Generate Unique Username/Email setting in the Parameters section.

  • Correlation for Existing Users in Okta: This section allows you to select correlation attributes in Aquera Contractor Management and Okta to identify the associated Okta users.
    • Okta Correlation Attribute: Select the Okta attribute to be correlated with the Aquera Contractor Management correlation attribute. For example, Employee number.
  • Generate Unique Usernames in Okta: Select this checkbox to generate a unique username for users in Okta. If this checkbox is not selected, the unique usernames will not be generated for the new users. 
    • Unique Username Generation Prompt: Enter the prompt to generate a unique username.
    • Generate Unique Usernames for Existing Users in Okta: Select this checkbox to generate unique usernames for existing users in Okta.
  • Generate Unique Email ID in Okta: Select this checkbox to generate a unique email ID for users in Okta. If this checkbox is not selected, the unique email IDs will not be generated for the new users. 
    • Unique Email ID Generation Prompt: Enter the prompt to generate a unique email ID.
    • Generate Unique Email ID for Existing Users in Okta: Select this checkbox to generate a unique email ID for existing users in Okta.

Okta User Deletion

Configure the following settings to customize the Okta deletions:

Note: This section appears only if you have selected the Enable Deletion of User when Deactivated in Okta settings in the Parameters section.

  • Remove Deprovisioned Users in Last X Days*: Enter the number of days after deprovisioning to automatically delete users in Okta. For example, if you enter 2, users will be deleted 2 days after they are deprovisioned.
