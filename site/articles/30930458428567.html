Overview

The Aquera platform enables your organization to scale complex data management by automating the integration between various applications.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the Active Directory to Dayforce Writeback integration to sync the Users resource of Active Directory (AD) to the Employees resource of Dayforce.

This integration helps write back the Business Email, Business Mobile, SSO Login, and Business Phone to the Dayforce system.
Note: Candidates complete Dayforce onboarding using a personal email, after which Aquera writes back the business identifier (email or employee number) to Dayforce as the Federation ID used for Single Sign-On. For details, refer to the Appendix.

This integration performs a full user sync from AD to Dayforce when you execute the integration for the first time, and then updates the record from AD to Dayforce incrementally.

Refer to the Attribute Map section to know the list of supported attributes that can be updated.

The following are the prerequisites:

Integration Setup

This section details how the integration must be configured to push user's data from Active Directory (AD) to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Active Directory to Dayforce Writeback.
  5. Enter a name and a description for the integration.
  6. Select your Active Directory (AD) (version 8) and Dayforce (version 25 or above) connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Global Settings

Configure the following global settings:

  • Enable Notifications: Select this to enable the integration-related notifications.
  • Enable Report-Only Mode: Select this to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Analytics: Select this to analyze integration-related metrics in a dashboard.
  • Enable Report Generation: Select this to generate the integration report.

Source Settings

Configure the following source application settings:

Note: The parameters marked with a red asterisk (*) are mandatory.

  • Active Directory Filter*: Select the filter to be applied for retrieving users from Active Directory.
    The following are the available filters:
    • None: Select this to consider all AD users without applying any filters.
    • Employee Number: Select this to filter a user by the employee number specified in the Source Employee Number field.
      Note: The Employee Number of a user in Active Directory is mapped to the Employee Number of a user in Dayforce.
    • Employee ID: Select this to filter a user by the employee ID specified in the Source Employee ID field.
  • Source Employee Number*: Enter the employee number to use as a filter when Employee Number is selected as the Active Directory Filter. For example, 100078. This parameter is available only if you select Employee Number in the Active Directory Filter parameter.
  • Source Employee ID*: Enter the employee ID to use as a filter when Employee ID is selected as the Active Directory Filter. For example, 440. This parameter is available only if you select Employee ID in the Active Directory Filter parameter.
  • Active Directory Update Condition*: Select one of the following update conditions:
    • All Users: Select this to update all user records, including both active and inactive users.
    • Only Active Users: Select this to update only users who are currently active in the system.
    • Custom: Select this to apply custom criteria to determine which users should be updated.
  • JavaScript Condition for Active Directory*: Enter an expression to define the update condition that will be applied during the update process. For example, new Date(hireDate1) <= new Date() is used to update the employee record in Dayforce on the hire date of an employee.
    Note: This parameter is available only if you select     Custom in the Active Directory Update Condition parameter.

Target Settings

Configure the following target application settings:

Note: The parameters marked with a red asterisk (*) are mandatory.

  • Active Directory Attributes to Match Dayforce Employee Number*: Select the Active Directory attributes that will be used to match the Dayforce Employee number attribute. The available options are Employee Number and Employee ID.
    When both attributes are selected, they will be prioritized in the specified order. For example, if you select Employee Number and Employee ID, the system will first attempt to use the Employee Number attribute. If it is not available for a user, it will use Employee ID.
    Note: Both attributes, or at least the Employee Number attribute must be available in the user record.
  • Dayforce attributes to be updated when user is hired: Select the Dayforce attributes that need to be updated when a user is hired. The available options are Business Email, Business Mobile, SSO Login, and Business Phone.
    Note: The phoneNumber value in Active Directory must be in the correct format, along with the country code. For example, +1-201-555-0123.
  • Update attributes X days before employee is hired in Dayforce: Enter the number of days before an employee’s hire date to trigger the update of selected attributes. This parameter is available only when an option is selected in the Dayforce attributes to be updated when user is hired parameter.
    Note: The HireDate attribute reflects the most recent hire date of the employee, not the original hire date.
  • Dayforce Update Condition Type*: Select the Dayforce update condition.
    The following are the available update conditions:
    • All Employees: Select to update all Dayforce employees.
    • Custom: Select this to customize which users need to be updated.
    • JavaScript Expression: Select this to use a JavaScript condition.
  • Dayforce Custom Condition: This section allows you to configure the Dayforce custom condition. The following parameters are available only when you select Custom in the Dayforce Update Condition Type parameter:
    • Select Conditions to Apply on Dayforce Employee*: Select one or more Dayforce employee attributes to filter employees for the updated condition. The following parameters are available only if the corresponding attributes are selected in this parameter.
      • EmploymentStatus Codes*: Enter the employment status codes.
      • Department Codes*: Enter the department codes. 
      • Job Codes*: Enter the job codes.
      • Position Codes*: Enter the position codes.
      • Pay Class Codes*: Enter the pay class codes.
      • Pay Group Codes*: Enter the pay group codes.
      • Pay Policy Codes*: Enter the policy codes.
      • Pay Type Codes*: Enter the pay type codes.
      • Payroll Policy Codes*: Enter the payroll policy codes.
        Notes:
        • If you provide a single value in the above code fields, such as 12 in the Department Codes field and 34 in the Employment Status Codes field, only the users with Department Codes of 12 and Employment Status Codes of 34 will be updated.
        • If you enter multiple values in the above code fields, such as 12 and 23 in the Department Codes field, and 34 and 45 in the Employment Status Codes field, only the users with Department Codes of 12 or 23 and Employment Status Codes of 34 or 45 will be updated.
        • To exclude a particular code from this condition, prefix the value with an exclamation mark (!).
  • JavaScript for Dayforce Update Condition*: Provide a JavaScript condition (as a string) to decide whether an employee from the Dayforce API response to be upserted. The expression runs once for each employee object and must return either true or false. This parameter is available only if the JavaScript Expression is selected from the Dayforce Update Condition Type parameter.

Attribute Map

The Attribute Map tab in Active Directory to Dayforce Writeback defines the Active Directory attributes that are mapped to the Dayforce attributes for syncing SSO login, work email, and work phone number attribute records in the Dayforce system. Only the source attributes can be modified, as needed, to align with your specific use case.

Note: The Telephone number attribute in Active Directory is mapped to the Business Phone and Business Mobile attributes in Dayforce. Based on your selection in the Dayforce Attributes to be Updated field in the Parameters section, the corresponding attribute will be updated.

Appendix

Candidate Onboarding and Single Sign-On Setup

When a candidate is added to Dayforce, they receive a first-time access email at their personal email address. This email allows the candidate to begin and complete the Dayforce onboarding process.

Once their onboarding process is completed in Dayforce and the Aquera integration finishes provisioning the employee to the Active Directory, the candidate’s business (work) email address is created. Aquera's Active Directory to Dayforce Writeback integration then writes the business email back to Dayforce. This information is stored in the Dayforce Federation ID, which is used as the Single Sign-On (SSO) login identifier.

Depending on the organization’s configuration, either the business email address or the Employee Number can be used as the Federation ID. The configured identifier is then used by the Identity Provider (IdP) to authenticate the user and enable SSO access.
