Dayforce / Okta Integration Guide

← Back to index

  This guide applies to the latest public version of the integration (version 2.0.0).

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any users created, updated, or terminated in Dayforce are automatically updated in Okta based on the defined schedule. The integration also supports the writeback of business email addresses and federated IDs from Dayforce to Okta.

To identify user records across both systems for user sync from Dayforce to Okta, the integration uses the attribute configured in the Source Field for Correlation parameter in Dayforce and matches it to the attribute configured in the Target Field for Correlation parameter in Okta as a correlation attribute.

Key Functions

The Dayforce / Okta Integration performs the following functions:

  • Synchronizes user records from Dayforce to Okta.
  • Creates new users in Okta when they are added in Dayforce.
  • Updates existing Okta users when their information changes in Dayforce.
  • Deactivates or deletes Okta users when they are no longer active in Dayforce.
  • Generates usernames and email IDs for users in Okta based on configured policies.
  • Updates business email addresses and federated ID's from Okta to Dayforce (writeback).
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Dayforce: Refer to the Dayforce Configuration Guide for configuration details.
    • Okta: Ensure that you configure the Okta (version 11) or Okta XaaS (Anything-as-a-Source) connector in the Aquera Admin portal.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Dayforce to Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Okta.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Okta connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following feature based on your requirements:

  • Enable Users Sync: Select this checkbox to synchronize user records from Dayforce to Okta. When enabled, the User Sync and User Life Cycle sections will be displayed for further configuration.
  • Enable Write Back: Select this checkbox to write back the user's business email address and federated ID from Okta to Dayforce. When enabled, the Write Back section will be displayed for further configuration.

Global Settings

Enable the following settings based on your requirements:

  • Incremental Mode: Select this checkbox to execute the integration in the incremental import mode. If this checkbox is not selected, the integration will be executed in full sync.
    Note: Select this parameter only after the first integration run if you want to execute the integration in incremental import mode.
  • Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Generate Unique Username/Email Using AI Widget: Select this checkbox to generate a unique username and email address using the AI widget. When enabled, the Define Policy tab will be displayed for further configuration.
  • Create Custom Attributes In Okta: Select this checkbox to create Dayforce custom attributes in Okta. When enabled, the Custom Attributes tab will be displayed for further configuration.

Feature Modules

Enable the following settings based on your requirements:

  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.
    Note: Contact the Aquera support team to execute the integration in debug mode.

User Sync

The User Sync section allows you to set up filters for users in Dayforce before integrating them with Okta. It has the following settings:

User Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Source Field for Correlation*: Select the Dayforce attribute to distinctly find the associated Okta user record. For example, select Employee Number.
  • Target Field for Correlation*: Select the Okta attribute to distinctly find the associated Dayforce user record. For example, select Locality.

User Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Import Limit: Enter the maximum number of user records to be imported into Okta during each integration execution. Once this limit is reached, no additional records will be processed. For example, if you enter 10, only the first ten user records will be imported into Okta.
  • Source Filter*: Select a filter type from the dropdown menu. Based on your selection, only employee records that satisfy the selected condition are imported. The following are the available options:
    • All Employees: Select this option to filter and import all the employees into Okta from Dayforce.
    • All Full Time Employees: Select this option to filter and import all full-time employees into Okta from Dayforce.
    • Employee Numbers To Fetch: Select this option to filter employees by their employee numbers. When selected, the following parameter is displayed:
      • Employee Numbers*: Enter the user's employee number that must be imported into Okta from Dayforce.
    • Employment Statuses to Fetch: Select this option to filter employees by their employment statuses. When selected, the following parameter is displayed:
      • Employment Statuses*: Select the user's employment statuses from the dropdown menu. Only employees with these statuses will be imported into Okta from Dayforce.
  • Properties to Fetch for Dayforce Employees: Select one or more properties from the dropdown menu. Only attributes related to the selected properties are included in the integration.
  • Include Manager's Full Details: Select this checkbox to include the user's manager's full details in the integration.
  • Select the Okta user statuses that should be included for lifecycle management: Select one or more Okta user statuses from the dropdown menu. Only users with the selected statuses will be included in the integration.

User Life Cycle

The User Life Cycle section allows you to customize the integration based on your preferred conditions for creating, updating, and deactivating users in Okta. It has the following settings:

Create Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Create Limit: Enter the maximum number of employee records that can be created in Okta from Dayforce in each integration execution. Once this limit is reached, no additional records will be processed. For example, entering 7 allows only 7 employee records to be created in Okta during each integration execution.
  • Create Condition*: Select a condition from the dropdown menu. Only employee records that satisfy the selected condition are created. The following are the available options:
    • Always Create Employees: Select this option to create all the newly created employees in Okta from Dayforce.
    • Only Create Active Employees: Select this option to create all active employees in Okta from Dayforce. When enabled, the following parameter is displayed:
      • Statuses To Be Considered As Active*: Select one or more statuses that must be considered as active. Only employees with these statuses will be created in Okta from Dayforce.
    • Only Create Employees Based On Onboarding Date: Select this option to create employees based on their onboarding date. When enabled, the following parameters are displayed:
      • Attribute To Consider For Onboarding Date*: Select an attribute from the dropdown menu. The value of the selected attribute will be considered as the onboarding date. For example, if you select the Start Date attribute, its value will be considered the user’s onboarding date.
      • Onboarding Date Offset Type*: Select an onboarding date offset type from the dropdown menu. The following are the available options:
        • Create Before Onboarding Date: Select this option to create the user in Okta from Dayforce before the user's actual onboarding date.
        • Create After Onboarding Date: Select this option to create the user in Okta from Dayforce after the user's actual onboarding date.
        • Same Day: Select this option to create the user in Okta from Dayforce on the same day as the user's onboarding date.
      • Onboarding Offset days*: Enter the number of days for offset. This parameter is available only if you select Create Before Onboarding Date or Create After Onboarding Date in the Onboarding Date Offset Type parameter.
    • Custom: Select this option if you want to customize the user creation condition. When selected, the following parameter is displayed:
      • Custom Create Condition: Enter an expression that defines the condition for creating users in Okta from Dayforce. Only users who satisfy this condition will be created in Okta from Dayforce.

Update Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Update Limit: Enter the maximum number of employee records that can be updated in Okta from Dayforce in each integration execution. Once this limit is reached, no additional records will be processed. For example, entering 10 allows only 10 employee records to be updated in each integration execution. 
  • Update Condition*: Select a condition from the dropdown menu. Only employees who satisfy the selected condition will be updated in Okta. The following are the available options:
    • Always Update Employees: Select this option to update all employees into Okta from Dayforce.
    • Only Update Active Employees: Select this option to update all active employees into Okta from Dayforce. When enabled, the following parameter is displayed:
      • Statuses To Be Considered As Active*: Select the statuses that must be considered as active. Only employees with these statuses will be updated in Okta from Dayforce.
    • Custom: Select this option if you want to customize the user updation condition. When selected, the following parameter is displayed:
      • Custom Update Condition: Enter an expression that defines the condition for updating users into Okta from Dayforce. Only users who satisfy this condition will be updated into Okta from Dayforce.

Deactivate Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Deactivate Limit: Enter the maximum number of employee records that can be deactivated in Okta during the each integration execution. Once this limit is reached, no additional records will be processed. For example, entering 7 allows only 7 employee records to be deativated in each integration execution.
  • Deactivate Condition*: Select a condition from the dropdown menu. Only employees who satisfy the selected condition will be deactivated in Okta. The following are the available options:
    • Always Deactivate Employees: Select this option to deactivate all employees in Okta.
    • Only Deactivate Employees with Below Employment Statuses: Select this option to deactive the employees by their employment statuses. When selected, the following parameter is displayed:
      • Only Deactivate Employees with Below Employment Statuses*: Select one or more statuses that must be considered for deactivating an employee in Okta. Only employees with these statuses will be deactivated in Okta.
    • Only Deactivate Employees Based On Offboarding Date: Select this option to deactivate employees based on their offboarding date. When enabled, the following parameters are displayed:
      • Attribute To Consider For Offboarding Date*: Select an attribute from the dropdown menu. The value of the selected attribute will be considered as the offboarding date. For example, if you select the Termination Date attribute, its value will be considered the user’s offboarding date.
      • Offboarding Date Offset Type*: Select an offboarding date offset type from the dropdown menu. The following are the available options:
        • Deactivate Before Offboarding Date: Select this option to deactivate the user in Okta before the user's actual offboarding date.
        • Deactivate After Onboarding Date: Select this option to deactivate the user in Okta after the user's actual offboarding date.
        • Same Day: Select this option to deactivate the user in Okta on the same day as the user's offboarding date.
      • Offboarding Offset days*: Enter the number of days for offset. This parameter is available only if you select Deactivate Before Offboarding Date or Deactivate After Onboarding Date in the Offboarding Date Offset Type parameter.
    • Custom: Select this option if you want to customize the user offboarding condition. When selected, the following parameter is displayed:
      • Custom Deactivate Condition: Enter an expression that defines the condition for deactivating users in Okta. Only users who satisfy this condition will be deactivated in Okta.

Write Back

This section enables you to write back user records from Okta to Dayforce. It has the following settings:

Writeback Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Source Field for Correlation*: Select the Okta attribute to distinctly find the associated Dayforce user record. For example, select roles.
  • Target Field for Correlation*: Select the Dayforce attribute to distinctly find the associated Okta user record. For example, select Login Id.

Writeback Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Import Limit: Enter the maximum number of user records to be imported into Dayforce during each integration execution. Once this limit is reached, no additional records will be processed. For example, if you enter 10, only the first ten user records will be imported into Dayforce.
  • Source Filter*: Select a filter from the dropdown menu by which the users should be filtered in Okta. The following are the available options:
    • All Employees: Select this option to write back all users from Okta to Dayforce.
    • Employee Number To Fetch: Select this option to filter the employees by their employee numbers.
      • Employee Numbers: Enter one or more user employee numbers. Only the users with these employee numbers are considered for write back from Okta to Dayforce.
    • DF Employee Numbers To Fetch: Select this option to filter the employees by their Dayforce employee numbers.
      • Df Employee Numbers to Fetch: Enter one or more user Dayforce employee numbers. Only the users with these employee numbers are considered for write back from Okta to Dayforce.
  • Select the Okta user statuses that should be included for Writeback*: Select one or more user's statuses from the dropdown menu. The employee's only with these statuses will be included for write back from Okta to Dayforce.
  • Properties to Fetch For Dayforce Employees: Select one or more properties from the dropdown menu. Only attributes related to the selected properties are included in the integration.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

User Sync

This section enables you to define how data must be mapped from Dayforce to Okta. Based on your configuration, Dayforce employee records details are mapped to Okta user records.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
