Overview

This guide provides information on the integration logs. The logs are generated when an integration runs as per the schedule set, event trigger, or manual execution.

You can view the integration logs in one of the following ways:

  1. Navigate to Logs > Integration on the Aquera Admin portal. The Integration logs are listed on the page. 
  2. Click on an integration to view the detailed integration logs.

Or,

  1. Navigate to Integrations and click on an integration to view the integration configuration page. 
  2. Click the ellipsis icon at the top-right corner of the page and then click Latest Logs.
    • If you have selected a user integration or the Dayforce to Active Directory integration, the Execution Timeline panel is displayed with the execution history of the selected integration.
      1. Click View Logs next to an execution record. The detailed logs of that execution are displayed at the bottom of the page.
      2. Click Goto Logs to navigate to the Logs > Integration page, which lists the specific log of that particular integration run.
    • If you have selected a script-based integration, the detailed logs of the latest execution are displayed at the bottom of the page.
      • Click the Expand icon at the right corner of the Logs section to navigate to the Logs > Integration page, which lists the logs of that particular integration.

Integration Logs

The following are the columns displayed on the integration logs page:

  • DATE: The date and time when the integration started. This is the user's local system time.
  • EXECUTION TIME: The time the integration process took to run. It is in the format HH:MM:SS. 
  • DISPLAY NAME: The display name of the integration.
  • ID: The ID of the integration.
  • NAME: The type of integration.
  • RESPONSE DATA: The status (progress indicators) of the integration. The following are the valid statuses:
    • running: The integration process is currently running.
      • getting records: The integration is performing a get all users operation from the source or the target application.
      • {ApplicationName} currentRecord of totalRecords: The integration periodically shows the number of user records retrieved out of the total records. For a full import, it shows both the source and the target number of user records retrieved.
        For example, BambooHR 200 of 1000.
        Note: You can also see the value {ApplicationName} currentRecord of 999999. For example, Okta: 3456 of 999999. The number 999999 is the default value set when the application has a very large number of records and the integration is unable to determine the exact number of records in the application.
      • processing: The integration is processing the users based on the Match Expression of the integration configuration.
      • scanning: This is shown for incremental runs and whenever the Import Limit is set. The integration process determines the set of matching target users associated with the source users.
      • updating: The integration is performing an update user operation.
    • success: The integration was completed successfully.
    • failure: The execution of the integration process failed. Examples: Failure to read schemas, failure to retrieve all users.
    • errors: Errors may occur during the execution of the integration process due to the following reasons:
      • User data transaction errors such as create/update user failure.
      • The integration engine is not running. Please contact Aquera Support (support@aquera.com) to get this issue resolved.
    • cancelled: The user has cancelled the integration execution.
    • incomplete: The integration engine was not able to complete the execution of the integration. Please contact Aquera Support (support@aquera.com) to get this issue resolved.

Filtering in Integration Logs

You can search for integration logs by entering the following values in the Filter events search box:

  • DISPLAY NAME
  • ID
  • NAME
  • RESPONSE DATA
  • runId
  • logId
  • tenantId
  • orchId
  • status

Note: For all the scheduled-based integrations, there may be a slight delay for an integration entry to appear in Logs > Integration. There may also be a slight delay in updating the integration status after it has completed its execution.

Filtering based on the Execution Period

You can filter the logs based on the time the integrations were executed. A dropdown at the top of the page allows you to filter logs by time range, including All, 5 minutes, 1 hour, 6 hours, 1 day, or a Custom range. Click Custom to filter the logs within a specific date and time.

Select a start date and an end date from the Calendar shown. Click Apply.
Note: The default values of 00:00:00 and 23:59:59 are automatically set for the start time and end time respectively.

The date and time of each log are displayed in the Date column on the page. Click button to choose if you want to display the log time in your local time or UTC. The date and time will be displayed based on the selection.

Deleting Logs

A user having the tenant Owner, SuperAdmin, or Admin role can delete the integration logs. 

The following pop-up is displayed when the user clicks the delete icon at the top right corner of the page.

  • Select All to delete all the transaction logs from the page.

  • Select All Transactions for last to delete the transactions for a specific number of days. Enter the number of days for which the transactions have to be deleted in the Days field. For example, if you enter 1 in the Days field, the transaction logs in the last 24 hours will be deleted. 

Notes:

  • Transaction logs of integrations in running status cannot be deleted.
  • The completed integration logs having errors status cannot be deleted.
  • The delete icon will remain disabled until the current deletion process is completed.

Retention Period of Logs

The retention period of integration logs is 1 year; the logs will be automatically deleted after a year.

Downloading Log Report

The integration log report contains the details of the execution of an integration.

To download the log report for an integration record, navigate to Logs > Integration, select the desired integration log record, and click the Download Log Report button at the bottom of the detailed log panel.

Integration Detailed Logs

This section provides information on the detailed logs for a user/bulk/data integrations. To view the detailed logs of an integration, navigate to Logs > Integration and click on an integration log record.

The following details are displayed in an integration log:

  • ID: The title of the integration log file, which is the log ID generated for each integration record.
  • Header: The first line is the header, which displays the basic details of the integration. Refer to the Header section for details.
  • Configuration: This section displays the configuration details as specified on the configuration page of the integration. Refer to the Configuration section for details.
  • User records: This section displays the details of the operations performed on users and groups. Refer to the User records section for details.
  • Summary: This section displays the summary of the integration run. Refer to the Summary section for details.

Header

The first line of the integration log is the header, which displays the following information:

  • The display name of the integration.
  • startTime: The start time of the integration in GMT format.
  • runId: The system-generated ID that is unique for each integration execution. This runId will be used by the Aquera Support team for debugging purposes in case of any integration issues.
  • logId: The ID generated for the integration log.
  • tenantId: The ID of your tenant.
  • orchId: The ID of the configured integration.
