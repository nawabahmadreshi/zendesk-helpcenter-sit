Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The ADP Workforce Now to SQL Server integration extracts the workers, payroll, department, and personal contacts resource types and their selected attributes from the ADP Workforce Now - API Central app and creates the tables in the configured SQL Server database. 

Refer to the ADP Workforce Now / Microsoft SQL Server Integration Template Version History to view new features and improvements added in different versions of this integration template.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal for integration:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for details.
    • Microsoft SQL Server: Refer to the Microsoft SQL Server Configuration Guide for details.
  • If Dynamic Data Masking is enabled on the SQL Server, ensure that the SQL Server user configured for the Aquera integration is granted the UNMASK permission using the following query:
    GRANT UNMASK TO <dbUserName>;

Integration Setup

This section outlines the configuration required to integrate ADP Workforce Now API Central with Microsoft SQL Server, enabling seamless transfer of pre-hire (candidate) and worker data from ADP Workforce Now to SQL Server.

Create Integration

Perform the following steps to create the integration:

  1. Perform the following steps to create the integration:
  2. Log in to the Aquera Admin portal.
  3. Navigate to Integrations from the left side panel.
  4. Click Create Integration.
  5. Search for ADP Workforce Now / Microsoft SQL Server in the search box next to the Explore Templates section.
  6. Click the integration template to display the Create New Integration pop-up window.
  7. Enter a name and a description for the integration.
  8. Select both application connector instances from the dropdown menus.
  9. Click Continue. The integration configuration page is displayed to configure the integration.

Examples for Creating a View in Microsoft SQL Server

Our integration currently stores ADP Workforce Now data in a raw, normalized structure. If you require the data in a denormalized or reporting-friendly format, you can create database-level views to transform and consolidate the data as needed. Below are example scripts for creating such views in Microsoft SQL Server.

View: V_Worker

This view retrieves consolidated current worker demographic and contact information by combining data from the workers and Personal_Contacts tables. The following is the script to create the view:

Create View V_Worker as
WITH CTE_Worker AS (
  SELECT 
    a.AssociateOID,
    a.LegalFirstName, 
    a.LegalLastName,
    a.BirthDate,
    a.Gender,
    b.AddressLineOne,
    b.AddressLineTwo,
    b.MobileCellPhoneFormattedNumber,
    b.LandlineAlternatePhoneFormattedNumber 
  FROM dbo.workers a WITH (NOLOCK)
  LEFT JOIN dbo.Personal_Contacts b WITH (NOLOCK) 
    ON a.worker_id = b.worker_id 
    AND b.Is_Current = 1 
  WHERE a.Is_Current = 1  
)
SELECT * FROM CTE_Worker;

View: V_Worker_Assignments

This view provides a consolidated dataset of current workers and their active assignment details by combining worker demographic information with assignment records. The following is the script to create the view:

CREATE VIEW V_Worker_Assignments AS
WITH Current_Workers AS (
  SELECT 
    AssociateOID,
    LegalFirstName, 
    LegalLastName,
    BirthDate,
    Gender
  FROM dbo.workers WITH (NOLOCK)
  WHERE Is_Current = 1
)
SELECT 
  w.AssociateOID,
  w.LegalFirstName,
  w.LegalLastName,
  wa.Assignment_Key,
  wa.Position_ID,
  wa.Job_Title,
  wa.Worker_Type,
  wa.Reports_To_Name,
  wa.Assignment_Status
FROM Current_Workers w
LEFT OUTER JOIN dbo.Worker_Assignment wa WITH (NOLOCK) 
  ON w.worker_id = wa.worker_id 
  AND wa.Is_Current = 1 ;

View: V_Payroll_Summary

This view produces a consolidated payroll run summary by combining current payroll header data with related payment distribution and payment allocation details. The following is the script to create the view:

CREATE VIEW V_Payroll_Summary AS
WITH CTE_Current_Payroll AS (
  SELECT 
    ItemID,
    PayrollGroupShortName,
    PayrollYear,
    PayrollWeekNumber,
    PayrollRunNumber,
    PayrollProcessingJobStatusShortName
  FROM dbo.Payroll WITH (NOLOCK)
  WHERE Is_Current = 1
)
SELECT 
  p.ItemID,
  p.PayrollYear,
  p.PayrollWeekNumber,
  p.PayrollRunNumber,
  p.PayrollGroupShortName,
  -- Distribution Details
  pd.AssociateOID,
  pd.CheckAmount,
  pd.PaymentDocumentID,
  -- Allocation Details
  pa.PaymentAllocationID,
  pa.PaymentAllocationSchemeCode
FROM CTE_Current_Payroll p
LEFT OUTER JOIN dbo.Payment_Distributions pd WITH (NOLOCK) 
  ON p.ItemID = pd.PayrollItemID 
  AND pd.Is_Current = 1
LEFT OUTER JOIN dbo.Payment_Allocations pa WITH (NOLOCK) 
  ON p.ItemID = pa.PayrollItemID 
  AND pa.Is_Current = 1;

View: vw_PayrollInstructionDetails

This view combines current payroll instructions with related general deductions and garnishments into a single, easy-to-use dataset for each worker. The following is the script to create the view:

CREATE VIEW vw_PayrollInstructionDetails AS
SELECT
   -- Payroll_Instructions
   p.AssociateOID,
   p.WorkAssignmentStatusCodeValue,
   p.PayrollGroupCodeValue,
   p.PayrollGroupCodeShortName,
   p.PayrollFileNumber,
   -- General_Deduction_Instructions
   g.DeductionCodeValue AS GD_DeductionCodeValue,
   g.DeductionCodeShortName AS GD_DeductionCodeShortName,
   g.DeductionCodeLongName AS GD_DeductionCodeLongName,
   g.DeductionRateValue AS GD_DeductionRateValue,
   -- Garnishment_Instructions
   gr.DeductionCodeValue AS GR_DeductionCodeValue,
   gr.DeductionCodeShortName AS GR_DeductionCodeShortName,
   gr.GarnishmentAmountValue AS GR_GarnishmentAmountValue
FROM Payroll_Instructions p
LEFT JOIN General_Deduction_Instructions g
   ON p.id = g.PayrollInstructionId
   AND g.Is_Current = 1
LEFT JOIN Garnishment_Instructions gr
   ON p.id = gr.PayrollInstructionId
   AND gr.Is_Current = 1
WHERE p.Is_Current = 1;

Troubleshooting

This section discusses issues that might occur during integration runs and the steps to resolve them.

Issue: The data retrieved by the integration from the SQL Server is masked. 

Resolution: This issue occurs when the database user account used by the integration lacks permission to unmask the masked data retrieved from the SQL Server database. You can grant the unmask permission to the database user using the following SQL query:

GRANT UNMASK TO <dbUserName>;
