Oracle Cloud HCM to Active Directory User Reconciliation

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. 

The Oracle Cloud HCM to Active Directory (AD) User Reconciliation integration automates the deactivation of user accounts in Active Directory when the termination date has passed or no work relationships exist for the users in Oracle Cloud HCM. It also reactivates the Active Directory users if the employee is rehired in Oracle Cloud HCM.

This integration performs the following tasks:

  • Retrieves all the employees whose termination date has passed or matches the current date.
  • Retrieves all the employees who do not have active work relationships in the Oracle Cloud HCM.
  • Tracks how users are matched (by person number, person ID, email) and counts for processed, deactivated, skipped, and fetched users from Oracle HCM and Active Directory.
  • Deactivates the corresponding user accounts in the Active Directory.
  • Reactivates Active Directory accounts for employees rehired in Oracle HCM.
  • Generates a CSV report with the list of users that are processed as well as deactivated based on a preference setting
  • Collects and formats email recipients for notifications and reporting.
  • Logs the current integration configuration, including run settings, deactivation and reporting flags, email recipients, and user filtering criteria.
  • Initializes the CSV report with headers for audit and compliance tracking.
  • Prints a detailed summary at the end of the run, including fetch stats, reconciliation results, matching breakdown, CSV reporting, and email status.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Oracle Cloud HCM: Refer to the Oracle Cloud HCM Configuration Guide for instructions on creating the Oracle Cloud HCM application connector and setting up the authentication.
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for instructions on creating the Active Directory application connector and setting up the authentication.

Integration Setup

This section details the configuration and execution process for integrating Oracle HCM with Active Directory. It covers creating the integration, configuring system preferences, and setting up reporting and notification mechanisms to automate the reconciliation of user accounts between Oracle HCM and Active Directory (AD).

The integration extracts user data from Oracle HCM, identifies and processes corresponding accounts in Active Directory, and produces audit-ready reports. It automatically detects all employees who are no longer active in Oracle Cloud HCM (whether due to termination or the absence of active work relationships) and deactivates their associated user accounts in the Active Directory.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Oracle Cloud HCM to Active Directory User Deactivation.
  5. Enter a name and a description for the integration.
  6. Select your Oracle Cloud HCM and Active Directory connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Integration Parameters

The Parameters section allows you to customize the integration settings according to your use case. It has the following configuration options:

  1. Enable Dashboard Generation: Provides a visual dashboard to review and analyze integration results.
  2. Recipients' Email Addresses: List of email addresses that will receive notifications about integration runs and their outcomes.
  3. Report Only: Runs the integration in simulation mode, reporting potential changes without applying them.
  4. Enable Integration Report Download: Sends a CSV email report summarizing the integration execution details.
  5. Send Email Notifications: Allows users to download the integration report from notification emails in CSV format.
  6. Debug Mode: Generates detailed logs showing data changes during integration to assist with troubleshooting and debugging.
  7. Deactivate Users Without Work Relationships: Automatically disables user accounts without active work relationships.
  8. Deactivate Users Missing from Oracle HCM: Disables user accounts in the Active Directory for employees who are no longer found in the Oracle HCM system.
  9. Log Missing Users from Oracle HCM: Records users found in Active Directory but missing from Oracle HCM to help identify data inconsistencies.
  10. Match Users by Email if Person Number is Missing: Use the email address to identify users when the personnel number is unavailable during integration.
  11. Email Notification for Failed Integration: Email alerts to specified recipients when an integration run encounters errors or fails.
  12. Email Subject on Successful Integration Execution: Specifies the subject of email notifications sent after successful integration runs.
  13. Email Subject on Failed Integration: Specifies the subject of email notifications sent when integration fails or encounters errors.
  14. Send Integration Report via Email: Select to receive email notification with a CSV report for all integration runs.
  15. Filter AD Users by Organizational Unit (OU): Limits integration processing to users within a specified Organizational Unit (OU) in Active Directory, allowing for targeted or scoped runs.
  16. Reactivate Rehired Oracle HCM Users: Automatically reactivates Active Directory accounts for employees rehired in Oracle HCM.
  17. Enable User Deactivation: Allows for the deactivation of user accounts, restricting access while preserving data for compliance and future reactivation. 
  18. Maximum Deactivation Count: Set the maximum number of users allowed to be deactivated at one time. Enter a numeric value to limit batch deactivations.
  19. Retrieve Data from Source Condition: The Condition that determines which data is extracted from the source application during integration.

Set Up Triggers

You need to set up a trigger to execute the integration automatically. This integration supports scheduled triggers.

You can set up the integration based on a basic or advanced schedule. For details on setting up a schedule, refer to the Integration Schedule Setup Guide.

Run the Integration

The integration can be scheduled to run automatically at customizable intervals. Available frequencies include 15 minutes, 30 minutes, 1 hour, 4 hours, 6 hours, 12 hours, or 24 hours. You can also enable recurring executions to run the integration continuously on your chosen schedule. You can also click Execute to run the integration manually.
On successful execution, the CSV file will be sent to the specified recipients. In case of integration errors, notifications and reports will also be sent to the email addresses. 

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel. 
The CSV report is generated only if you have selected the Generate a CSV report option in the Preferences section.
