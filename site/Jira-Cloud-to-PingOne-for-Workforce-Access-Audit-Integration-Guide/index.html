Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the user's group name(s) from PingOne for Workforce to the ticket description in Jira Service Management Cloud for the selected work type.

To identify user records across both systems, the integration uses the emails attribute from Jira Service Management Cloud and matches it to the emails attribute in the PingOne for Workforce as a correlation attribute.

Key Functions

The Jira Cloud to PingOne for Workforce Access Audit Integration provides the following functions:

  • Retrieves the work email address from the Jira Service Management Cloud ticket and identifies a user with the same email address in PingOne for Workforce.
  • If exactly one matching user is found, it retrieves their group name from PingOne for Workforce.
  • Appends the user's group information to the description field of the Jira Service Management Cloud ticket.
  • Changes the status of the ticket based on predefined settings.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • PingOne for Workforce: Refer to the PingOne for Workforce Configuration Guide for details.
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push the group name from PingOne for Workforce to Jira Service Management Cloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Jira Cloud to PingOne for Workforce Access Audit.
  5. Enter a name and a description for the integration.
  6. Select your PingOne for Workforce and Jira Service Management Cloud connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenario(s)

Configure the following integration scenario(s):

  • Select the JML Operation(s): Select one or more JML operations for which the ticket description should be updated with the group name fetched from PingOne for Workforce. The available options from the dropdown are Joiner, Mover, and Leaver.
    Note: The Joiner Settings, Mover Settings, and Leaver Settings sections will be displayed based on your selection.
  • Select PingOne Environment ID: Select the PingOne Environment ID from the dropdown, where the user's group name information will be retrieved and synchronized. For example, Administrators.
    Note: Whenever you change the environment ID, make sure that the corresponding client ID (as the username) and client secret (as the password) are also updated in the PingOne for Workforce connector to enable authentication and retrieve group name information.
  • Execute Full Run: Select this checkbox to execute a full integration run manually at any time.
    Note: By default, a full integration run occurs every two days. To manually trigger a full integration run between these scheduled executions, select this checkbox.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications. The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
  • Enable Report Generation: Select this checkbox to generate the integration report.

Joiner Settings

Configure the following joiner settings:

  • Project Name: Select the project to retrieve project-specific joiner tickets.
  • Worker Type: Select the corresponding worker type to retrieve joiner tickets. For example, select Joiner from the dropdown to retrieve all joiner tickets for the selected project.
  • Ticket Pickup Status: Select the ticket statuses from the dropdown to retrieve the corresponding joiner tickets. For example, select Open and Reopened from the dropdown that retrieves all joiner tickets in the Open or Reopened status into your integration.
  • Post-Update Ticket Status: Select the ticket status to which Joiner tickets should transition after their group descriptions are updated. For example, select In Progress from the dropdown, which automatically updates the status of the joiner tickets that we retrieved using the Ticket Pickup Status field to In Progress.
  • Update Only Ticket Status If Ticket Description Exists: Select this checkbox to update only the ticket status when the ticket description is already up to date.

Correlation Field Settings

Configure the following correlation field settings:

  • Select Correlation Field in Jira: Select the source field in Jira Service Management Cloud that is used to match users with PingOne for Workforce records as a correlation during synchronization. For example, Corporate email address.
  • Select Correlation Field in PingOne: Select the corresponding target field in PingOne for Workforce that is used to match users with Jira Service Management Cloud records as a correlation during synchronization. For example, email.

Mover Settings

Configure the following mover settings:

  • Project Name: Select the project to retrieve project-specific mover tickets.
  • Worker Type: Select the corresponding worker type to retrieve mover tickets. For example, select Mover from the dropdown to retrieve all mover tickets for the selected project.
  • Ticket Pickup Status: Select the ticket statuses from the dropdown to retrieve the corresponding mover tickets. For example, select Open and Reopened from the dropdown that retrieves all mover tickets in the Open or Reopened status into your integration.
  • Post-Update Ticket Status: Select the ticket status to which mover tickets should transition after their group descriptions are updated. For example, select In Progress from the dropdown, which automatically updates the status of the mover tickets that we retrieved using the Ticket Pickup Status field to In Progress.
  • Update Only Ticket Status If Ticket Description Exists: Select this checkbox to update only the ticket status when the ticket description is already up to date.
