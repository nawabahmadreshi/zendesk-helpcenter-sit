Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the end-to-end tasks for setting up an integration that synchronizes location creation, updates, and inactivation from HubSpot to ADP WorkforceNow.

To identify user records across both systems, the integration uses the Clinic ID attribute from HubSpot Contacts and matches it to the Code in the ADP Workforce Now as a correlation attribute.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • HubSpot Contacts: Refer to the HubSpot Contacts Configuration Guide for details.
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for API Central for details.

Integration Setup

This section outlines how the integration must be configured to push Client ID data from HubSpot Contacts to ADP Workforce Now.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select HubSpot Contacts to ADP Workforce Now - API Central.
  5. Enter a name and a description for the integration.
  6. Select your HubSpot Contacts and ADP Workforce Now - API Central connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Integration Features
    Configure the following Integration features settings:
    • Manage Organizational Data: Select this checkbox to manage the organizational data.
    • Manage Company Sync Params: Select this checkbox to enable the company parameters before running the sync.
  • Execution Settings
    Configure the following execution settings:
    • Enable Log Setting [Enable Logging]: Select this checkbox to generate logs outlining data changes.
    • Enable Reporting Setting [Enable Report Generation]: Select this checkbox to generate the integration report
    • Enable Notification Setting [Enable Notifications]: Select this checkbox to enable the integration-related notifications.
    • Enable Insights Setting [Enable Log Insights]: Select this checkbox to visualize the integration-related metrics.
  • Enable Testing Mode: Select this checkbox to generate logs outlining data changes that would occur for testing.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.
  • HubSpot Company Properties: Enter a list of company properties to retrieve from HubSpot for data transformation, mapping, reporting, or synchronization.
  • Full Sync Expiry (Days): Enter the number of days a full sync remains valid before a new full sync is triggered. 
  • HubSpot Object ID Filter: Enter the HubSpot company object ID to filter and limit data retrieval to that specific company record.

Advance Settings

Configure the following advanced settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them
  • Enable Incremental Sync Mode: Select this checkbox to run the integration in incremental mode.
  • Create Limit: Enter a numeric value to restrict the number of new records created in the target system per execution.
  • Send Integration Report as CSV:
  • Send Integration Report via Email: Select to receive email notification with a report for all integration runs
