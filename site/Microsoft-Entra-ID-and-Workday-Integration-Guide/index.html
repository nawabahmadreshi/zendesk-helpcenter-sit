Overview 

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

The Aquera Workday–Microsoft Entra ID integration helps keep employee identity information synchronized between Workday and Microsoft Entra ID. It supports multiple identity management scenarios, including user lifecycle management, employee photo synchronization, time-aware lifecycle management, and email writeback.

When the integration runs, it compares employee records in Workday with user accounts in Microsoft Entra ID. Employees are matched using a shared identifier, such as Employee ID, to ensure updates are applied to the correct record.

For example, employee profile photos can be synchronized so that photos stored in Microsoft Entra ID remain aligned with employee profiles in Workday. Microsoft Entra ID serves as the trusted source for employee photos, while Workday uses them to maintain accurate, consistent employee profiles for workforce operations.

The integration runs in controlled batches, respects system limits, and supports report-only mode, allowing administrators to review potential changes before they are applied.

Key Functions

The integration provides the following key functions to keep employee identity information synchronized between Workday and Microsoft Entra ID:

  • User Lifecycle Management: Automatically creates and updates user accounts in Microsoft Entra ID based on employee data stored in Workday.

  • Photo Synchronization: Syncs employee profile photos from Microsoft Entra ID to the corresponding employee profiles in Workday.

  • Time-Aware Lifecycle Management: Executes lifecycle actions based on effective dates, such as creating accounts on an employee’s start date or disabling them on a termination date.

  • Email Writeback: Updates the employee’s email address in Workday when it changes in Microsoft Entra ID.

  • Report-Only Mode: Allows administrators to preview the actions the integration would perform without applying any updates.

Integration Setup

This section describes the configuration steps required to set up Microsoft Entra ID to Workday User Photo Sync integration in Aquera.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left navigation panel.
  3. Click Create Integration.
  4. Select Microsoft Entra ID / Workday.
  5. Enter a name and description for the integration.
  6. Click Continue. The Configuration page is displayed.

Configuration

The following sections are available under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to select:

  • Microsoft Entra ID (Azure AD) is the source application from which user profile photos are retrieved.
  • Workday is the target application for updating employee profile photos.

Both connectors must be configured with valid credentials and appropriate permissions to support user lookup and photo writeback.

Parameters

The Parameters section allows you to customize the operation of the Microsoft Entra ID to Workday User Photo Sync integration. These settings control the behavior of the integration and help tailor it to your organization’s requirements.

Integration Features 

Configure the following integration features:

  • Enable User LifeCycle Management: Enable automatic creation and updating of user accounts in Microsoft Entra ID based on employee records in Workday. For example, when a new employee Barbara Jensen is added in Workday with Employee ID 10245, the integration can create a corresponding user account for Barbara Jensen in Microsoft Entra ID.
  • Enable Photo Sync: Enable synchronization of employee profile photos from Microsoft Entra ID to Workday. For example, if Barbara Jensen updates her profile photo in Microsoft Entra ID, the updated photo is synced to Barbara Jensen’s employee profile in Workday.
  • Enable Time Aware Lifecycle Management (LCM): Enable lifecycle actions to run based on effective dates and configured time offsets. For example, the user account for Barbara Jensen can be automatically created in Microsoft Entra ID on her official Workday start date.
  • Enable email writeback from Entra to Workday: Enable synchronization of updated email addresses from Microsoft Entra ID back to Workday. For example, if Barbara Jensen’s email address changes in Microsoft Entra ID from barbara.jensen@company.com to barbara.jensen2@company.com, the updated email is written back to Barbara Jensen’s Workday record.

General Settings

Configure the following general settings:

  • Enable incremental mode: Select to process only records that have changed since the last run. For example, if Barbara Jensen’s profile photo was updated after the previous integration run, only Barbara Jensen’s record is processed instead of syncing all employees. 
  • Enable naming policy settings: Select to manage and apply naming policies for user accounts created or updated through the integration. For example, the integration can generate a username such as barbara.jensen@company.com for Barbara Jensen based on the configured naming policy.
  • Enable report-only mode: Select this to generate a report detailing the data changes that would occur, without applying them. For example, the report may show that Barbara Jensen’s email would be updated in Workday, but no actual change is made.
  • Enable logging: Select to generate logs outlining data changes. For example, the logs may record that Barbara Jensen’s profile photo was successfully synchronized to Workday.
  • Enable notifications: Select to send integration-related notifications, including success, warning, and error summaries, after each run. For example, administrators may receive a notification confirming that Barbara Jensen’s record was updated successfully.
  • Enable report generation: Select to generate a detailed report summarizing the results of the integration run. For example, the report may list that Barbara Jensen’s account was created, updated, or skipped.
  • Enable debug mode: Select to generate detailed logs showing data changes that would occur during the integration run for troubleshooting purposes. For example, debug logs may show the exact attributes being updated for Barbara Jensen’s account.
  • Enable log insights: Select to visualize integration-related metrics and trends. For example, administrators can review metrics showing how many records, including Barbara Jensen’s, were processed during the integration run.

Correlation Settings

Configure the following correlation settings:

  • Entra Correlation: Selects the field in Microsoft Entra ID that the integration uses to identify each user. When an Employee ID is selected, the integration reads the employee ID from the Entra user profile and uses it to find the matching employee in Workday.
  • Workday Correlation: Selects the Workday field that the integration uses to identify employee records. When an Employee ID is selected, the integration searches for an employee record in Workday that matches the selected Employee ID.  
    These settings ensure that the integration correctly matches each Entra user to the right employee record in Workday before updating profile photos.

Manage Users

Configure the following settings to control how user accounts are created, updated, and terminated in Microsoft Entra ID based on employee records in Workday.

Create Users

Configure the following settings to create new user accounts in Microsoft Entra ID for eligible employees from Workday:

  • Create condition for users: Select the condition that determines which Workday users are eligible for account creation in Microsoft Entra ID. For example, selecting Create Active Users allows the integration to create an account for Barbara Jensen when her Workday status becomes active.
  • Enable creation limit: Select to limit the number of user accounts created during a single integration run. For example, if the limit is enabled, only a defined number of new users, including Barbara Jensen, are created in that run.
    When Enable creation limit is selected, the following field is displayed:
    • User creation limit: Specify the maximum number of user accounts that can be created during a single integration run. For example, entering 25 allows the integration to create up to 25 new users such as Barbara Jensen during that run.

Update User

Configure the following settings to update existing user accounts in Microsoft Entra ID based on changes to employee records in Workday:

  • Update condition for users: Select the condition that determines which users are eligible for updates in Microsoft Entra ID. For example, selecting Update Active Users Only allows the integration to update Barbara Jensen's profile details only if her Workday status is active.
  • Enable updation limit: Select to limit the number of user updates processed during a single integration run. For example, if the limit is enabled, updates for employees such as Barbara Jensen are processed only up to the configured limit.
    When the Enable updation limit is selected, the following field is displayed:
    • User updation limit: Specify the maximum number of user records that can be updated during a single run. For example, entering 50 allows updates for up to 50 users, including Barbara Jensen, during that run.

Terminate User

Configure the following settings to disable or terminate user accounts in Microsoft Entra ID when the corresponding employee records in Workday become inactive or terminated:

  • Terminate condition for user: Select the condition that determines which users are eligible for termination in Microsoft Entra ID. For example, selecting Terminate Inactive Users Only disables the account for Barbara Jensen when her Workday status becomes inactive.
  • Enable termination limit: Select to limit the number of user terminations processed during a single integration run. For example, if the limit is enabled, only a specified number of accounts such as Barbara Jensen’s are terminated during that run.
    When the Enable termination limit is selected, the following field is displayed:
    • User termination limit: Specify the maximum number of user accounts that can be terminated during a single run. For example, entering 20 allows the integration to terminate up to 20 users, including Barbara Jensen, in that run.

Debug Mode 

This section allows administrators to run the integration for a specific Workday user for testing and troubleshooting purposes. It helps verify configuration settings and review how the integration processes a particular employee record without running the integration for all users.

Debug Settings

Configure the following settings to run the integration for a specific Workday user for testing and troubleshooting:

  • Workday filter: Select the attribute used to filter and retrieve a specific Workday worker from Workday. For example, selecting Employee ID allows the integration to retrieve the record for Barbara Jensen using her Employee ID.

    When Employee ID is selected, the following field is displayed:

    • Employee ID: Enter the Employee ID of the Workday worker to run the integration for that specific user. For example, entering 10245 runs the integration only for Barbara Jensen, allowing administrators to test or troubleshoot the integration for that employee.

Account Provisioning

This section defines how user accounts are created in Microsoft Entra ID when new employees are provisioned from Workday. It allows administrators to configure password generation rules, the default domain, and how the User Principal Name (UPN) and email address are generated.

Password Settings

Configure the following settings to define how passwords are generated for new user accounts created in Microsoft Entra ID:

  • Generate password: Select the password generation method used when creating new user accounts. For example, selecting Configure Password Policy enables the integration to automatically generate a password for Barbara Jensen according to the configured password rules.
  • Password length: Specify the length of the generated password. For example, entering 12 generates a 12-character password for Barbara Jensen’s account during user creation.
  • Allowed characters: Select the character types that can be used when generating passwords. For example, selecting Include Lowercase, Include Uppercase, Include Numbers, and Include Special Characters allows the generated password for Barbara Jensen to include all these character types.
  • Allowed special characters: Specify the special characters that can be included in generated passwords. For example, characters such as ! @ # $ % may be included when generating a password for Barbara Jensen’s account.

Domain Name

Configure the default domain used when generating the User Principal Name (UPN) and email address for new user accounts in Microsoft Entra ID:

  • Default domain name*: Specify the default domain used when generating the User Principal Name (UPN) and email address. For example, entering atest.onmicrosoft.com generates a UPN such as barbara.jensen@atest.onmicrosoft.com for Barbara Jensen.

Generate UserPrincipalName (UPN)

Configure how the User Principal Name (UPN) is generated for new user accounts in Microsoft Entra ID. The UPN is the username the user uses to sign in:

  • Generate user principal name (UPN) using prompt: Specify how the User Principal Name prefix is generated before the domain is appended. For example, the integration may generate an email address such as barbara.jensen@atest.onmicrosoft.com for Barbara Jensen. 
    If duplicate usernames exist, the integration can append a number to ensure uniqueness. For example, if barbara.jensen@atest.onmicrosoft.com already exists, the integration may generate barbara.jensen1@atest.onmicrosoft.com.

Generate Email

Configure how email address is generated for new user accounts in Microsoft Entra ID. The generated email can be used as the user’s primary email address:

  • Generate email using: Select how the email address is generated for new users. For example, selecting Use UserPrincipalName (UPN) As Email sets Barbara Jensen’s email address to barbara.jensen@atest.onmicrosoft.com based on the generated UPN.

Write Back for Email

This section configures how email address changes in Microsoft Entra ID are written back to the corresponding employee record in Workday.

Email Configuration

  • Email field on Workday: Select the Workday email attribute that will be updated when email writeback occurs. For example, selecting Work-Email updates the Workday work email field for Barbara Jensen when her email changes in Microsoft Entra ID.
  • Entra email field: Select the Microsoft Entra ID attribute that will be used as the source email value for writeback to Workday. For example, selecting mail writes the value of barbara.jensen@company.com from Microsoft Entra ID to the Workday Work-Email field for Barbara Jensen.
  • Enable writeback limit: Select to limit the number of email updates written back to Workday during a single integration run. For example, enabling this option ensures that only a specified number of email updates, including Barbara Jensen’s email update, are processed in one run.
    When Enable for writeback limit is selected, the following field is displayed:
    • Import limit for writeback: Specify the maximum number of email updates that can be written back to Workday during a single integration run. For example, entering 25 allows the integration to update the Workday email field for up to 25 employees, including Barbara Jensen, during that run.

Photo Sync

This section configures how employee profile photos are synchronized between Workday and Microsoft Entra ID.

Photo Sync Settings

Configure how employee profile photos are synchronized between Workday and Microsoft Entra ID, including the direction of synchronization and limits on the number of photo updates processed during each integration run.

  • Photo Sync Behavior: Select to sync employee profile photos from Workday to Microsoft Entra ID and from Microsoft Entra ID to Workday, and define how photos are updated, processed, and retained in each direction. For example, if Barbara Jensen’s profile photo is being synced, the integration follows these rules:

    • Workday to Entra ID:
      • Supports incremental sync. For example, if only Barbara Jensen’s photo was updated, only her photo is processed.
      • Supports larger photo sizes when using Office 365 Connector v19. For example, Barbara Jensen’s high-resolution photo in Workday can be synced to Entra ID using this connector.
      • Deleting a photo in Workday removes it from Entra ID during sync.
    • Entra ID to Workday:
      • Only full sync is supported (incremental sync is not available). For example, even if only Barbara Jensen’s photo was updated, all employee photos are processed.
      • Deleting a photo in Entra ID does not remove it from Workday. For example, if Barbara Jensen’s photo is deleted in Entra ID, it will still remain in her Workday profile.
  • Enable limit: Select to limit the number of photo updates processed during a single integration run. For example, if the limit is enabled, only a defined number of employee photos, including Barbara Jensen’s profile photo, are synchronized during that run.
    When the Enable limit is selected, the following field is displayed:
    • Photo sync limit: Specify the maximum number of employee photos that can be synchronized during a single integration run. For example, entering 25 allows the integration to sync photos for up to 25 employees, including Barbara Jensen, during that run.

Time Aware LCM

This section allows you to configure time-based lifecycle management settings. It enables the integration to perform actions, such as user provisioning, based on an employee’s onboarding date, defined date offsets, and scheduled execution time.

Time Aware Settings

Configure the lifecycle scenarios that should use time-aware processing based on onboarding dates and configured offsets:

  • Enable time aware settings: Select the lifecycle scenario that should use time-aware processing. For example, selecting Onboarding Date Offset/Time Aware allows the integration to create the Microsoft Entra ID account for Barbara Jensen based on her onboarding date and configured offset.

Onboarding Schedule Settings

Configure the schedule and offset rules used to perform lifecycle actions based on an employee’s onboarding date:

  • Onboarding action timing: Select when the action should happen based on the employee’s onboarding date. For example, choosing Before Onboarding Date creates the Microsoft Entra ID account for Barbara Jensen before her onboarding date.

  • Days before onboarding: Specify the number of days before the onboarding date when the lifecycle action should occur. For example, entering 15 creates the Microsoft Entra ID account for Barbara Jensen 15 days before her onboarding date.

  • Timezone for onboarding: Select the timezone used when evaluating the onboarding date and time. For example, selecting UTC ensures the onboarding lifecycle action for Barbara Jensen runs according to UTC time.

  • Onboarding time of day: Specify the exact time when the onboarding action should run. For example, entering 14:50 schedules the provisioning of Barbara Jensen’s account to run at 14:50 UTC on the scheduled onboarding date.

Note: The Report Settings appear when you select the Enable Report Generation checkbox in the General Settings section of the Parameters tab.

Report Settings

Configure the level of detail included in the integration report generated after each integration run:

  • Select report type: Select the type of information to include in the integration report.
    The following options are available:
    • Minimum: Includes only the operations performed during the integration run, such as create, update, or terminate.     • Full: Includes detailed report information such as the employee number, the operation performed, the field names, and the updated values, including both PII and non-PII data. For example, the report may show Barbara Jensen/Department updated to Sales. This corresponds to the option Employee Number with All Data – Employee # + operations + field names + all values (PII + non-PII).

Notifications

This section allows you to configure email notifications for integration events, such as successful or failed execution, and to define the recipients and report details included in the notification emails:

Notification Settings

Configure the following settings to define when notification emails are sent.

  • Send email notification for failure: Select to send an email when the integration fails. For example, if the integration encounters an error, a failure notification is sent to the configured recipients.

Attribute Mapper

This section defines how employee attributes from Workday are mapped to the corresponding user attributes in Microsoft Entra ID. These mappings control how employee data is used when creating or updating user accounts in Entra ID during the integration run.

Each mapping specifies how a source attribute or expression from Workday is transformed and assigned to a target attribute in Entra ID. This ensures that employee identity information, such as names, email addresses, phone numbers, and location details, remains consistent between the two systems.

Mappings may use direct attributes from Workday or expressions that combine or transform values before sending them to Entra ID. For example, preferredFirstName and preferredLastName from Workday may be combined to populate the Display Name attribute in Entra ID.

Administrators can review and modify these mappings to align with organizational identity requirements. Any updates made in this section determine how worker data from Workday is provisioned and synchronized to Microsoft Entra ID during user creation and updates.

Update the Integration to the Latest Version

You can upgrade an integration created from a template to the latest version to access new features, improvements, and fixes.

Upgrade an Integration

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module in the Aquera Admin Portal.

  2. Locate the integration you want to upgrade.

  3. Click the vertical ellipsis (⋮) next to the integration and select View Details.

  4. The Details panel displays the following sections:

    Integration details: Displays the integration name, description, creation date, last updated date, and the user who created or last modified the integration.

    Template details: Displays the current integration version under Current Version.
    If a newer version is available, it appears under New Version Available along with a summary of the changes and an Upgrade option.

  5. Click Upgrade. The Upgrade Integration window appears.

  6. Provide the following details for the upgraded integration:

    • New integration name: Enter a name for the upgraded integration.

    • Description: Enter a description for the upgraded integration.

    The window also displays the current version and the new version that will be applied.

  7. Click Upgrade.

A new integration is created using the upgraded template version while retaining the configuration settings from the existing integration.
