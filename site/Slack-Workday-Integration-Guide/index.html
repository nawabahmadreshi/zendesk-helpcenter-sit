Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the Workday to Slack synchronization, which synchronizes user data, such as profile pictures, from Workday to Slack, with Slack serving as the reference system for profile photo updates.

To identify user records across both systems, the integration uses the Email attribute from Workday and matches it to the Employee Number in Slack as a correlation attribute.

Key Functions

The Workday to Slack integration provides the following key functions:

  • Supports profile photo additions, updates, and deletions.
  • Detects and skips users with duplicate correlation values.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Workday: Refer to the Workday Configuration Guide for details.
    • Slack: Refer to the Slack Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Workday to Slack.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Slack / Workday integration.
  5. Enter a name and a description for the integration.
  6. Select both application connector instances from the dropdown menus.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable Photo Sync: Select this checkbox to sync user profile pictures from Workday to Slack.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Incremental Mode: Select this checkbox to run the integration in incremental mode.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the data is saved to Insights.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.

Integration Settings

The Integration Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if Enable Photo Sync is enabled in the Integration Scenarios section.

Workday Settings

Configure the following Workday settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Workday Correlation Attribute*: Select the Workday attribute to uniquely identify the associated Slack user. For example, if you select Email in this field, the Email attribute will be used as a correlation attribute.
  • Include Contingent Workers: Select this checkbox to include contingent workers when retrieving worker data from Workday.
  • Include Inactive Workers: Select this checkbox to include inactive workers when retrieving worker data from Workday.
  • Include Active Workers: Select this checkbox to include active workers when retrieving worker data from Workday.

Slack User Settings

Configure the following Slack settings:

  • Enable Photo Updates: Select this checkbox to update the Slack profile photo when the Workday photo changes. If this checkbox is not selected, profile photo updates are ignored.
    Notes: 
    • Cropping is optional using crop_x, crop_y, and crop_w. If not provided, the full image is used, and non-square images are resized to fit with empty space.
    • The image size must be at least 512x512 pixels and no larger than 1024x1024 pixels.
  • Enable Photo Removal: Select this checkbox to remove the Slack profile photo when the corresponding Workday photo is deleted. If this checkbox is not selected, the Slack profile photo is not removed when the Workday photo is missing.
  • Slack Correlation Attribute*: Select the Slack attribute to uniquely identify the associated Workday user. For example, if you select userName in this field, the userName attribute will be used as a correlation attribute.
  • Slack Update Condition: Select a condition from the dropdown menu to determine when users are updated. The following are the available options:
    • All Users: Select this option to update all Slack users.
    • All Active Users: Select this option to update all active Slack users.

Log Settings

The Log Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if Enable Logging is enabled in the Parameters section.

  • Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Notification Settings

The Notifications section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Notification is enabled in the Parameters section. All parameters marked with a red asterisk (*) are mandatory.

  • Admin's Email Addresses*: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin's Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.

Report Settings

The Report Settings section enables you to customize the integration report settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Report Generation is enabled in the Parameters section.

  • Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Debug Mode

The Debug Mode section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if Enable Debug Mode is enabled in the Parameters section. All parameters marked with a red asterisk (*) are mandatory.

  • Workday Filter*: Select the Workday attribute to filter the users while retrieving from Workday.
  • Workday Filter Value*: Provide the value for the selected source filter.
  • Import Limit: Enter the maximum number of user records that should be imported for debugging during each integration run. By default, the value is 100.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
