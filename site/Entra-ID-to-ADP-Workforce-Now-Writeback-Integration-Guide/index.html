Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to write back business email addresses and business phone numbers of users created in Entra ID to ADP Workforce Now. 

Key Functions

The ADP Workforce Now / Entra ID Writeback integration performs the following functions:

  • Retrieves the business email addresses and business phone numbers of the user accounts that were created by Aquera's ADP Workforce Now / Entra ID integration in Entra ID.
  • Updates the business email addresses and business phone numbers to the corresponding employee records in ADP Workforce Now.
  • Generates reports and logs, and sends notification emails based on the integration settings.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now:  For configuration details, refer to the ADP Workforce Now Configuration Guide.
    • Entra ID: For configuration details, refer to the Microsoft Entra Native Configuration Guide or the Microsoft Entra Native Basic Auth Configuration Guide, depending on the type of authentication to be used for the connector. <Should this be M365?>>

Integration Setup

This section outlines how the integration must be configured to write back business email addresses and business phone numbers from Entra ID to ADP Workforce Now.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / Entra ID Writeback.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Overview page is displayed to start configuring the integration.

Overview

The Overview tab provides a summary of the integration capabilities and also allows you to enable the write-back feature. It has the following section:

Directory

The Directory tab enables you to set the application connector that must be used to retrieve the data of Entra ID users. It also enables you to set the conditional rules for retrieving a specific set of users from Entra ID and the writeback configurations.

Directory Settings

Data Synchronization Settings

  • Enable recurring full sync: Select this checkbox to set the frequency of full synchronization from Entra ID to ADP. During a full synchronization, business email addresses and business phone numbers of all eligible user records in Entra ID are synchronized to the corresponding employee records in ADP.  When the checkbox is enabled, the following parameters will be displayed to set up the full sync schedule.
    If this checkbox is not selected, only incremental synchronization will be performed based on the configured schedule. In this mode, only the business email addresses and business phone numbers that were added in Entra ID since the last synchronization run will be updated in ADP.
    Note: The first execution of the integration always performs a full sync.
  • Full sync schedule: Select the frequency by which the integration must run in full sync mode. For example, selecting Every 6 hours from the drop-down menu will execute the integration every 6 hours in full synchronization mode.
    For example, if the integration schedule is configured as Continuous and the Full sync schedule is configured as Every 6 hours, the integration executes continuously based on the configured schedule settings. However, one execution in every 6 hours will be in full synchronization mode. All other executions between those full synchronizations are in incremental mode, if incremental sync is enabled.
    For instance, if a full synchronization execution happens at 12 AM, the next full synchronization execution happens at 6 AM. All executions between 12 AM and 6 AM are in incremental sync mode.
  • Start schedule from: Specify when the full sync schedule must begin. 
    • Start now: Select this option for the full sync to begin from the next scheduled run based on the configured schedule.
    • From specific date: Specify the date and time from which the full sync schedule should start, in the Start date and Start time fields.
  • Disable incremental sync: Turn on this toggle button to run the integration only in full synchronization mode and skip incremental synchronization runs. When disabled, the integration runs in both full and incremental synchronization modes based on the configured schedule settings.

Conditional Writeback Settings

  • Enable conditions for writeback: Select this checkbox to specify the conditions that determine which user records are eligible for the writeback of business email addresses and business phone numbers. The Custom conditions for writeback field is displayed to define the conditions.  
  • Custom conditions for writeback: Specify the conditions used to determine whether users are eligible for the writeback of their business email addresses and business phone numbers to ADP Workforce Now. This field supports AI-powered prompts using Aquera Intelligence (AI Assistant) to generate the conditional expression automatically. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

Application Setting

  • Entra ID Application: Select the connector that must be used by the integration to retrieve users data from Entra ID.

Writeback Settings

  • Enable write back: Enable this toggle for the integration to support writeback of business email addresses and business phone numbers to ADP Workforce Now. This setting is mandatory.
  • Entra ID User Match Attribute: Select the Entra ID user attribute that must be used by the integration to compare and find a matching employee record in ADP Workforce Now.
  • ADP Employee Match Attribute: Select the corresponding employee attribute in ADP Workforce Now that must be used to find the matching employee record.
  • Select attributes for writeback: Select the employee attributes that must be updated back from Entra ID to ADP. The attributes supported by the integration are Business phone and Business email.
