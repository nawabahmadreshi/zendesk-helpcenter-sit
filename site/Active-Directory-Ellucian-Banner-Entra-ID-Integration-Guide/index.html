Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employees and students data between Ellucian Banner and Entra ID.

To identify records across both systems, the integration uses the Banner ID from Ellucian Banner and matches it to the Employee ID in Entra ID as a correlation attribute.

Key Functions

This integration automates identity lifecycle management between Ellucian Banner and Entra ID (formerly Azure AD) with the following primary capabilities:

  • Synchronizes user records from Ellucian Banner to Microsoft Entra ID.
  • Creates new users in Entra ID when they are added in Ellucian Banner.
  • Updates existing Entra ID users when their information changes in Ellucian Banner.
  • Deactivates or deletes Entra ID users when they are no longer active in Ellucian Banner.
  • Assigns users to groups in Entra ID based on configured rules.
  • Generates passwords, email IDs, and UPNs (User Principal Name) for users in Entra ID based on configured policies.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Tenant account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure application connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Ellucian Banner: Refer to the Ellucian Banner Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating Ellucian Banner with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Active Directory / Ellucian Banner / Entra ID in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select your Ellucian Banner and Microsoft 365 connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Once created, the following tabs are displayed:

  • Overview
  • Ellucian Banner
  • Directory
  • Policy
  • Execution
  • Notification
  • Schedule

Attribute Mapping

This section defines how employee attributes from Ellucian Banner are mapped to the corresponding user attributes in Microsoft Entra ID. These mappings control how employee data is used when creating or updating user accounts in Entra ID during the integration run.

Each mapping specifies how a source attribute or expression from Ellucian Banner is transformed and assigned to a target attribute in Entra ID. This ensures that employee identity information, such as names, email addresses, phone numbers, and location details, remains consistent between the two systems.

Mappings may use direct attributes from Ellucian Banner or expressions that combine or transform values before sending them to Entra ID. For example, preferredFirstName and preferredLastName from Ellucian Banner may be combined to populate the Display Name attribute in Entra ID.

Administrators can review and modify these mappings to align with organizational identity requirements. Any updates made in this section determine how worker data from Ellucian Banner is provisioned and synchronized to Microsoft Entra ID during user creation and updates.

Assignment Rules

This section explains how to edit access rules for assigning rules for employees and students in Entra ID.

Perform the following steps to edit access rules:

  1. Click the vertical ellipsis next to any column, and then click Add Attribute to view the list of available attributes you can use to set the assignment rule.
    If you want to remove the existing column, click Delete Column.
  2. Select the attribute you want to use to define a condition.
  3. In the next row, double-click and select the attribute value to define the rule. If multiple values are selected, OR condition is applied among them.
  4. If required to add another condition and assignment for the same selected attribute, click Add Row.
    Note: You can add any number of rows within a rule.
  5. If required to include another attribute to the assignment rule, click the vertical ellipsis next to a column, and then click Add Attribute.
    If multiple columns are added for a rule, AND condition is applied between different columns.
    You can use multiple attributes and their corresponding values to define an assignment rule.
  6. If needed, click the + icon at the top right corner to add another rule, as explained above.
  7. Click Save Changes at the top-right corner of the page to save the configuration.
  8. To see other action items, such as View Info, Insert Row, Copy Row, Disable Rule, and Delete Row, click the horizontal ellipsis next to the row.

Naming Policy

This section allows you to configure the generation of UserPrincipalName and email:

It has the following settings to configure:

  • Generate UserPrincipalName (UPN): Turn on this toggle if you want to perform the UPN generation.
    • UPN: The User Principal Name (UPN) attribute is the user logon name (ULN) that is used to log in to a domain. By convention, UPN should map to the user's email address.
      A UPN consists of a prefix (the user account name) and a suffix (a DNS domain name), separated by the @ symbol. The UPN format is username@DomainName. For example, barbarajensen@aquera.com.
  • Generate email: Turn on this toggle if you want to perform the email generation.
    • Mail: The Mail attribute is the user's email address, often set to match the UPN. The mail should be unique within the same domain. The mail attribute typically follows standard email formatting. For example, barbarajensen@example.com.
  • Generate email using: Select the required value to generate the emails using custom values. If you do not find the required sources in the list, click the refresh icon available next to the field to reload the available sources.

Navigate to the Password Policies section to define how passwords are generated in the target applications. This section allows you to configure password generation rules, complexity requirements, and length constraints.

  • Maximum password length: Enter the maximum number of characters allowed in a password.

  • Password complexity requirements: Select the password complexity requirements. This defines the character types that must be included in the password. The available options are as follows:
    • Require uppercase letters: Select this if you want passwords to include at least one uppercase character (A–Z).
    • Require lowercase letters: Select this if you want passwords to include at least one lowercase character (a–z).
    • Require numbers: Select this if you want passwords to include at least one numeric character (0–9).
    • Require special characters: Select this if you want passwords to include at least one special character. If you select this option, the following field is displayed for further configuration:
      • Allowed special characters: Select the special characters permitted in passwords. Only the selected characters can be used when special characters are required.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
