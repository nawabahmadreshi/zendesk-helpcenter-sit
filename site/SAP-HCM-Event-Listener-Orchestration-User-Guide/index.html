SAP HCM Event Listener Orchestration User Guide

← Back to index

Overview

This guide explains the event listener orchestration to be set up in the Aquera Admin application to receive employee events from the SAP HCM application and process them for identity management systems. It also explains the basic configurations to be done in the SAP system to send those employee events to Aquera's HR Onboarding gateway.

Employee Data Transmission from SAP HCM

SAP HCM transmits employee data with other systems in the IDOC (Intermediate Document) format through the ALE (Application Link Enabling) interface. Aquera Engine reads the IDOC format and converts it into the SCIM format  before sending it to the identity providers.

Creating Orchestration in Aquera Admin

The following procedures are involved in the orchestration setup:

  • Adding the SAP HCM Application - Refer to the SAP HCM Configuration Guide for details on configuring the SAP HCM application in the Aquera Admin console.
  • Setting up SAP HCM Event Listener Orchestration

Setting up SAP HCM Event Listener Orchestration

  1. On the Aquera Admin Home page, navigate to Orchestrations module, and click on Create Orchestration. 
  2. Enter the following details on the Add Orchestration page:
    • Enter a Name for the orchestration.
    • Select SAP HCM Event Listener from the orchestration type drop-down list. 
  3. Enter the following on the Configuration page:  
    • Select the SAP HCM application, which you have created in the previous procedure, as the Source Application.
    • Select the Target Application to which the events received from the SAP HCM system must be sent.
    • Enter other details on the page, according to your requirement.

4. Click Create.

5. Click the Event Detail tab to see the event URL. This event URL contains the Host and the Path Prefix values that need to be configured while Creating an RFC Connection in the SAP system. 

6. Click Create Token to generate a token.

7. Copy the token and save it securely. This token needs to be entered as the Password in the Logon & Security tab while Creating an RFC Connection in the SAP system.

8. Close the pop-up window.

9. Click Save changes to finish the orchestration setup.

Logging on to SAP

  1. Open the SAP Logon (SAP GUI) application.
  2. Click Connections and double-click on the SAP system to which you want to connect.
  3. Enter the user credentials (for example, SAP* or DDIC) provided by SAP.
  4. After you log on to an SAP system, the SAP Easy Access screen appears. This is the user-specific point of entry into the SAP system.

Creating Users for ALE Transfer

This procedure explains how to create a user and assign proper authorizations to the user. This user account is required to perform the IDOC transfers. The transfer of IDOCs will be done through RFC (Remote Function Call) and this user will be tied to that communication process.

1. Execute the SU01 (User Maintenance) transaction code in the Command field on the SAP Easy Access screen.

The User Maintenance: Initial Screen is displayed.

2. Enter the username for the new user in the User field, and click the Create icon.

3. Add other details of the user, as required, on the Maintain Users screen.

4. In the Logon Data tab, enter the following:

  • Select the User Type as Dialog.
  • Enter the initial password for the user in the New Password and Repeat Password fields. On first login, the user will be prompted to reset their password.
    Note: The password must be of minimum 8 characters in length and must contain at least one capital letter and one number.

5. In the Profiles tab, assign the following profiles to the user for full authorization:

  • SAP_ALL
  • SAP_NEW
  • S_A.CPIC
  • S_A.SYSTEM

These profiles determine the authorizations of the user.

6. Click the Save icon to create the user. 

The user has to be configured for every instance of SAP, for example, test, production.

7. Log out and log in with the new user account that you have created now. 

Defining Logical System

A logical system must be defined for the Aquera endpoint (target system) to send IDOCs from SAP. This logical system must be assigned to a SAP client. This enables the system to recognize the target system as an RFC (Remote Function Call) destination.

1. Execute the SALE transaction code in the Command field on the SAP Easy Access screen.

The Display IMG screen is displayed.

2. Navigate to Basic Settings > Logical Systems and click Define Logical System. The Change View Logical Systems screen is displayed.

3. Click New Entries. 

4. Add a logical system entry.

  • In the Log.System column, enter a name for the logical system that you want to create. The following naming convention is recommended for the logical systems: <XXX>CLNT<NNN> where <XXX> is the system name and <NNN> is the client.
  • In the Name column, enter a description of the logical system.

5. Click the Save icon. A workbench request is generated. 

Note: If the request number is not displayed, you must create the request number by clicking the Create Request icon on this window. Click the Save icon after entering a short description.

5. Click OK to create the transport request.

Setting Up SSL for Secure Transactions

This section explains the following procedures that are required to set up SSL for secured transactions of data from SAP:

  1. Downloading Security Certificates from Amazon Trust Services
  2. Creating SSL Client Identity Entry for PSE
  3. Creating PSE 
  4. Uploading Amazon Trust Certificates

Downloading Security Certificates from Amazon Trust Services

1. Open the following link in a browser: https://aws.amazon.com/blogs/awsforsap/sap-idoc-integration-with-amazon-s3-by-using-amazon-api-gateway/.

2. Scroll down to the Configure RFC connection in SAP section of the page and click the Amazon Trust Services link.

3. Download each of the certificates listed in the table in DER format to your system. The certificates are created in the system with the following names:

  • AmazonRootCA1.cer
  • AmazonRootCA2.cer
  • AmazonRootCA3.cer
  • AmazonRootCA4.cer
  • SFSRootCAG2.cer

Creating SSL Client Identity Entry for PSE

An entry must be created for SSL client PSE (Personal Security Environment).

1. Execute the STRUST transaction code in the Command field on the SAP Easy Access screen. The Trust Manager screen is displayed.

2. Click Environment > SSL Client Identities. A message box is displayed indicating that the table is cross-client.

3. Click OK.  The Change View "SSL Client Identities of System" screen is displayed.

4. Click New Entries. Note: Click the Edit icon to switch to the Edit mode.

5. Indicate Identity code and description for this new record.

6. Click the Save icon.

1. Execute the STRUST transaction code in the Command field on the SAP Easy Access screen. The Trust Manager screen is displayed.

2. Right-click on the new SSL Client Identity and click Create.
Note: If the Trust Manager screen is in View mode, switch to the Change mode first. 

The following screen is displayed.

3. Enter the name, organization, encryption algorithm, and key strength.

4. Click OK.

A new PSE is now available to upload the Amazon trust certificates.

Creating an RFC Connection

An HTTP connection must be established from the SAP system (client) to the HTTP server.

NOTE: Do not edit any of the existing connections configured in the system.

1. Execute the SM59 transaction code in the Command field on the SAP Easy Access screen. The Configuration of RFC Connections screen is displayed.

2. Click HTTP Connections to External Server and then click the Create icon. The RFC Destination screen is displayed.

3. Enter a name for the RFC destination in the Destination field and select the Connection Type as HTTP connection to external server.

4. Click OK and enter the following details.

  • Enter a description for the RFC destination.
  • Click the Technical Settings tab and enter the following details.
      • Host: Enter the host name or the IP address of the target system to which the SAP client needs to be connected. For example, events.aquera.com.  This is available in the event URL in the Setting up SAP HCM Event Listener Orchestration section. 
      • Service No: Enter the port number. The destination host has to be configured in such a way that the specified port understands the corresponding protocol (HTTP or HTTPS). For example, 443.
      • Path Prefix: The sub path that needs to be inserted before ~request_uri. When the connection to this destination is initiated, the system inserts this sub-path before ~request_uri. Refer to the Setting up SAP HCM Event Listener Orchestration section for information on the event URL that contains the sub path to be configured here. The sub path is the one after aquera.events.com in the URL (including the slash).

4. Click the Save icon. The following message box is displayed.

5. Click OK.

6. Click the Logon & Security tab and enter the following details.

  • Select Basic Authentication and enter the username and password. When the user accesses the target system, the user ID and password must be provided for authentication. Refer to the Setting up SAP HCM Event Listener Orchestration section for information on the token number that needs to be entered here as the password.
  • Select Do Not Send Logon Ticket.
  • Select SSL as Active.
  • Select the  SSL client PSE, which contains the certificate that is to be used for the authentication. The target system treats the SAP system as a trusted system.

7. Click the Special Option tab and enter the following details.

  • Timeout: ICM Default Timeout. It indicates the maximum response time for the HTTP request.
  • HTTP Version: HTTP 1.1
  • Compression: Inactive
  • Compressed Response: Yes
  • Accept Cookies: No

8. Click the Save icon and click Connection Test to verify that the connection can be established. The username and password has to be provided while testing the connection.

Configuring IDOC Port

Configure an IDOC port of type XML HTTP using the RFC destination created in the previous procedure.

1. Execute the WE21 transaction code in the Command field on the SAP Easy Access screen. The Ports in IDOC Processing screen is displayed.

2. Click XML HTTP and click the Create icon. The Add XML HTTP Port screen is displayed.

3. Enter the following details.

  • Enter a name for the Port and a Description.
  • Select the RFC destination created in the previous step from the drop-down list.
  • Select the Content Type as Text/XML.

4. Click the Save icon.

Creating Partner Profile for the Logical System

A partner must be created for the Logical System that you have created earlier in the Defining Logical System section.

1. Execute the WE20 transaction code in the Command field on the SAP Easy Access screen. The Partner profiles screen is displayed.

2. Click the Create icon.

3. Enter the name of the logical system created previously in the Partner No field.

4. Enter the Partn. Type as LS, which indicates logical system, and specify the SAP user that was created in the Creating Users for ALE Transfer section in the Agent field.

5. Click the Save icon.

6. Create Outbound parameter for the partner profile as follows.

  • Click the  (Create outbound parameter) icon below the Outbound section on the screen. For outbound parameters, we must provide port and IDOC details because port describes the system to which the IDOC has to flow. 
  • Enter the Message Type as HRMD_A.
  • Enter the IDOC port that was created in the previous procedure in the Receiver port field.
  • Select the Transfer/Pass IDoc Immed option.
  • Enter the IDOC Type as HRMD_09.
  • Click the Save icon.

7. Similarly, create the Inbound parameter. Mention the process code details which determines the inbound function module for processing the incoming data.

8. Click the Save icon.

9. Return to the previous screen and click the Classification tab.

10. Enter the partner status as Active as shown below.

11. Click the Save icon.

Creating the Distribution Model

The distribution model defines the sender, the receiver, and the types of information that are transmitted.

1. Execute the SALE transaction code in the Command field on the SAP Easy Access screen. The Display IMG screen is displayed.

2. Expand Modelling and Implementing Business Processes and  click Maintain Distribution Model and Distribute Views.The Display Distribution Model screen is displayed.

3. Switch to Edit mode, and click Create Model View. 

The Create Model View window is displayed.

4. Enter a description of the model view and a technical name in the Short text and Technical name fields respectively.

5. Click OK to create the model view.

6. Select the model view and click Add message type.

The Add Message Type window is displayed.

7. Enter the Client ID of your SAP instance as the Sender, the logical system defined in the Defining Logical System section as the Receiver, and HRMD_A as the Message Type.

8. Click OK and click the Save icon.

9. Expand the model view that you have created and double-click on No filter set.The Change Filter screen is displayed.

10. Click Data filtering and click Create Filter Group.

11. Double-click on Infotype. The following window is displayed.

12. Click the Insert Row icon to enter the types of information to be sent to the receiver. The list of all available info types are displayed.

13. Select the info types of users that must be communicated through IDOC. The following information types must be added to the data filter. Based on the types of information to be transferred, more info types need to be added.

  • 0000 - Actions, such as, any updates to a user.
  • 0001 - Organizational assignment
  • 0002 - Personal data
  • 0006 - Address
  • 0105 - Communication data such as user's email address

14. Double-click on Type of related object and add the following types of objects that need to be sent through IDOC:

  • K - Cost Center
  • P - Person (employees)
  • S - Position of employees in the organization

15. Double-click on Object type and add the following types of objects: K, P, and S

16. Click OK and click the Save icon.

Activating Change Pointers

You have to activate change pointers in SAP. Only the changed user data needs to be sent from the SAP system. A change pointer can identify the changed user data. 

1. Execute the BD61 transaction code on the SAP Easy Access screen. The Activate Change Pointers Generally screen is displayed.

Or
Navigate to the Activate Change Pointers Generally screen as follows:

  • Execute the SALE transaction code in the Command field on the SAP Easy Access screen. The Display IMG screen is displayed.
  • Expand Modeling and Implementing Business Processes > Master Data Distribution > Replication of Modified Data > Activate Change Pointers - Generally.

2. Verify that the Change pointers activated (in general) checkbox is selected. If not, select it and save the change.

3. Click the Back button on the menu bar and enter the transaction code, BD50, on the SAP Easy Access screen. The Change View "Activate Change pointers for Message Type": Overview screen is displayed.

4. Activate the change pointer for the message type, HRMD_A.

This step ensures that all changes to personnel numbers or the organizational structure are transferred automatically through IDOCs. The change pointers can be activated for each functional module in SAP, as per your requirements.

A request is generated.

Customization of Info Types 0000, 0001, and 0002

This step is for validation purpose.

1. Execute the PA30 transaction code on the SAP Easy Access screen. The Maintain HR master Data screen is displayed.

2. Check the existing personnel actions defined in the system for Hiring, Organizational reassignment /change, Leaving, and Rehiring.

3. Customize the values for SAP system infotype 0000 (Actions) as follows. The actions are configured based on the country where the organization is located. The codes indicated below for personal actions are sample codes.

  • Enter the employee number, select the Actions infotype and click the Edit icon.

  • Select to use the following personnel action types:
    • Code 01 for  Hiring
    • Code 02 for Organizational reassignment/changes
    • Code 10 for Leaving 
    • Code 12 for Reentry into company /Rehiring

  • Set the Employment status as follows.
    • Code 0 for  Withdrawn (used for personnel actions type 10)
    • Code 1  for Inactive (used when an employee is suspended)
    • Code 2 for  Retiree (used for personnel action type 10)
    • Code 3 for Active  (used for personnel action types 01, 02, and 12)

4. Customize the values for SAP system infotype 0001 (Organizational Assignment) as follows. The values mentioned here are sample values.

  • Enter the employee number, select the Organizational Assignment infotype and click the Edit icon.

  • The following fields need to be set based on your requirements:
    • CoCode: Company Code. At least one company code must be configured for each country.
    • Cost Ctr: Cost center. If it is necessary to transfer the configuration of cost centers between systems, it has to be managed with a separate IDOC of message type: COSMAS. We are currently not configuring the cost centers with the Aquera interface.
    • Pers.area and Subarea: Personnel areas and sub areas. You can have multiple personnel area codes and sub areas codes configured for each company code.
    • EE group: Employee groups. You can have multiple employee groups configured for each country to distinguish between various categories of employees.
    • EE subgroup: Employee sub-groups. You can have multiple employee sub groups configured for each country to distinguish between various categories of employees.
    • Payr.area: Payroll area. You can have multiple payroll areas configured for each country to distinguish between various categories of payment algorithms.

NOTE: All HR functions (except Organizational Management) must be in one regional system. You cannot divide them between different systems and then distribute their data.

Distribution of Changes

For each change in the filter table, a distribution of the related objects is necessary. 

This is achieved with an automatic job running after RBDMIDOC with variant HRMD_A.

1. Execute the BD21 transaction code on the SAP Easy Access screen. The Creating IDoc Type from Change Pointers screen is displayed.

Or
Navigate to the Creating IDoc Type from Change Pointers screen as follows:

  • Execute the SALE transaction code in the Command field on the SAP Easy Access screen. The Display IMG screen is displayed.
  • Expand Modeling and Implementing Business Processes > Master Data Distribution > Replication of Modified Data > Creating IDocs from Change Pointers > Define Variants.

2. Create running variant HRMD_A with message type HRMD_A as follows:

  • Click the Save icon. The Variant Attributes screen is displayed.

  • Enter the variant name as HRMD_A and enter a description.
  • Click the Save icon.

3. Define a job to generate an IDOC automatically for changes in employee data, as follows: 

  • Execute the SM36 transaction code on the SAP Easy Access screen. The Define Job screen is displayed.

  • Enter a name for the job and click the Step icon to define the job steps.
  • Enter RBDMIDOC as the the ABAP program name and select the HRMD_A variant, and then click the Save icon.
  • Return to the Define Job screen and click the Start condition icon. The Start Time screen is displayed.

  • Click Date/Time and set the date and time when the job must run for the first time.
  • Click Period Values to set the frequency by which the job must run.

  • Set the frequency as Hourly and click the Save icon. The job is created.You can review the details of the job by executing the transaction code, SM37.

Verification of iDOC Creation

After all the above configurations are completed, verify that an IDOC can be generated.

1. Execute the PFAL transaction code in the Command field on the SAP Easy Access screen. The HR: ALE Distribution of HR Master Data screen is displayed.

2. Enter the following details.

  • Plan Version: 01
  • Object type: P
  • Object ID: The employee number(s) whose records need to be transferred. Leave this field blank to generate an IDOC for the entire list of employees.
  • Select one of the following Transfer modes:
    • Insert: If data is being sent for the first time to the target system
    • Update:  If personnel details are already available in the target system
  • Receiv. Partner No.: The name of the logical system
  • Message type: HRMD_A
3. Execute the program by clicking the Execute icon on the menu bar.
4. Execute the WE05 transaction code in the Command field and verify that the IDOC is generated.

Sending Employee Data Modifications

The change pointer for the message type HRMD_A was switched on to track all changes for HR master data and organizational data automatically.

The RBDMIDOC report creates the outbound IDOCs for the ALE transfer.We have already created a job in the Distribution section to run the report automatically on an hourly basis.

When you create or change user data, change pointers mechanism flags those changes in the change pointers table BDCP2. The program RBDMIDOC reads this table and creates master and communication IDOCs.

Processing of Inbound IDOCs

1. Execute the OYEB transaction code on the SAP Easy Access screen.

2. Click Yes to activate inbound IDOCs.

3. Execute the BD51 transaction code on the SAP Easy Access screen.

4. Enable the IDOC_INPUT_HRMD option to allow debugging.

The basic configurations in the SAP HCM system are now completed.
