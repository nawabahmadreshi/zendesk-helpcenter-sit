Overview

In the SCIM-to-SCIM orchestration, attribute mapping is done between source and target. The source attribute has to be edited in accordance with that of the target. The mapping simplifies the process of executing Create, Update/Edit and Delete attributes, as the changes made to the source attribute will automatically get reflected at the target once the mapping is completed and the changes are saved.

Functions

%ATTEMPT -- This substitution variable will contain the number of the attempt to create the user.  When an orchestration does not match on an existing user in the target system it will attempt to create a new user.  The first attempt at creating this user will have the variable %ATTEMPT set to 1.  If the user that is attempted to be created fails because of a duplicate user (this will return a 409 in the error code) then the map will be executed again with the %ATTEMPT set to 2.  This continues until the account is created correctly or an error other than 409 is returned.

Special Attributes

The $ref attribute is a special attribute in the SCIM User and group objects.  It can be used to directly access the referenced object through the path to that object.  All user attributes are available for use.  For example, SOURCE[ENT].manager.ref[ENT].employeeNumber can be used to access the employeeNumber of the current user objects manager.

Mapping Process 

On the Aquera Admin page, go to 'Orchestration' and Click on 'Attribute Map'

Attributes are seen under 'Source' and 'Target'

If you want to map a specific attribute, you can search it by name in the Search Box

After choosing the attribute that has to be mapped, Click on the dropdown next to the 'Arrow' mark that is located in between the Source and Target attributes

Select the relevant mapping option from the list and edit the source attribute accordingly

Once the attribute mapping is completed, Click on 'Save Changes' on the top right

You can see a message 'Update Orchestration Successful'
