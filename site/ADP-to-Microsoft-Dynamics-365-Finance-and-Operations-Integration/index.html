ADP to Microsoft Dynamics 365 Finance and Operations Integration

Overview 

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

The ADP to D365 Finance and Operations Integration automates the transfer and synchronization of workforce data across multiple enterprise systems. It connects ADP (the HR system), Microsoft Entra ID (Azure AD) (the identity system), Microsoft Dynamics 365 Finance & Operations (the finance and operations platform), and Exflow (the invoice approval system, an add-on for Dynamics 365). The integration ensures that HR, finance, and IT applications share a unified and accurate view of employee and organizational information across the systems.

In this integration, ADP provides the main source of employee and organizational data. Microsoft Dynamics 365 Finance & Operations (D365 F&O) uses this data to manage payroll, finances, and workforce operations. Microsoft Entra ID (Azure AD) manages user accounts and access permissions, while Exflow uses the synchronized user information to handle invoice and expense approvals.

The integration continuously monitors ADP for any changes and automatically updates the connected systems to keep all data synchronized. When a new employee is added in ADP, a matching record is created in Microsoft Dynamics 365 Finance & Operations (D365 F&O). If configured, corresponding user accounts are also generated in Microsoft Entra ID (Azure AD) and Exflow with the correct access permissions.

Any updates to employee details, such as job title, department, or reporting structure are automatically reflected across all systems to maintain accuracy and consistency. When an employee leaves the organization, the integration deactivates or terminates the employee record in D365 F&O and disables the related user accounts in Entra ID and Exflow to prevent unauthorized access.

In addition to employee data, the integration synchronizes job, position, and organizational hierarchy data, ensuring that organizational hierarchies and reporting relationships remain accurate and up to date within D365 F&O.

Key Functions

The ADP to Microsoft Dynamics 365 Finance & Operations (D365 F&O) Integration performs the following key functions:

  • Retrieves employee and organizational data from ADP: Fetches the latest employee records from ADP, including details such as name, job title, department, location, and employment status. 

  • Creates and updates employee records in Microsoft D365 F&O: Creates new employee records in D365 F&O when new hires are added in ADP and updates existing records when employee details change.

  • Synchronizes user accounts with Microsoft Entra ID (Azure AD): Generates or updates user accounts in Entra ID based on ADP data. When employees join, change roles, or leave, their corresponding user accounts are automatically created, updated, or disabled to maintain proper access permissions.

  • Synchronizes approvals and user workflow in Exflow: Updates Exflow with the most recent approver and reviewer information from D365 F&O so that all invoice and expense approvals are routed to the correct users without delay.

  • Updates job and position data in D365 F&O: Transfers job and position information from ADP to D365 F&O to maintain organizational hierarchies and reporting structures. 

  • Deactivates or deletes records for terminated employees: Detects employee terminations in ADP and automatically deactivates or deletes corresponding records in D365 F&O. The integration also disables user accounts in Entra ID and Exflow to prevent unauthorized access.

Integration Setup

This section details the configurations required for this integration. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to Microsoft Dynamics 365.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to search and select the ADP application from which the employee details need to be retrieved, and the Microsoft Dynamics 365 F&O application in which the employee details must be synced.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

This section enables the following features of the integration:

  • Enable Employee Lifecycle Management: Enable the synchronization of employee lifecycle events from ADP Workforce Now to Microsoft Dynamics 365 Finance & Operations employees.

  • Enable User Lifecycle Management: Enable the synchronization of user lifecycle changes from Microsoft Entra ID to Microsoft Dynamics 365 Finance & Operations users for account provisioning and updates.

  • Enable Warehouse Worker Lifecycle Management: Allow synchronization of warehouse worker lifecycle events from ADP Workforce Now into Microsoft Dynamics 365 Finance & Operations warehouse worker records.

  • Enable ExFlow User Lifecycle Management: Enable synchronization and management of Microsoft Dynamics 365 user accounts within ExFlow for invoice and expense approval workflows.

  • Enable Developer Mode: Enable developer-specific features for the integration, such as enhanced logging, detailed payload visibility, debugging information, and test or simulation capabilities to help with troubleshooting, validation, and development activities.

Execution Features 

Configure the following general settings.

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would occur during the integration run, without applying them in Microsoft Dynamics 365 Finance & Operations.

  • Enable Incremental Mode: Select to process only the records updated in ADP Workforce Now since the last successful integration run, rather than retrieving the full dataset on each run.

  • Enable Notifications: Select to send integration-related notifications, including success, warning, and error summaries, after each run.

  • Enable Reporting: Select to generate detailed reporting output for each integration run, including per-record results.

  • Enable Logging: Select to capture logs for all evaluated operations, conditions, decisions, and errors during the integration run.

  • Enable Log Insights: Select to display integration metrics and insights such as record counts, trends, success and failure statistics, and error breakdowns within the Log Insights view.

  • Enable Limits On Sections: Enabling this option provides additional controls on each section to limit the number of records processed during creation, update, and termination operations.

Manage HR Employees

Configure filters to control which employees are included in the synchronization:

  • Enable Employee Sync Filter: Enable this option to apply a source-based filter that restricts which employees are synchronized from ADP Workforce Now.
  • Employee Sync Source Filter: Select the ADP attribute used to filter employees. Available options include Associate ID or Position ID.
  • Source Filter Value: Enter the value corresponding to the selected source filter. Only employees matching this value will be processed by the integration.

Position Sync Settings

Configure how employee position information is synchronized between ADP Workforce Now and Microsoft Dynamics 365 Finance & Operations:

  • Enable Manager Position Sync for Employees: Enable synchronization of manager position relationships for employees in Dynamics 365 Finance & Operations.

  • Enable Employee Position Management (HR Workers): Enable position lifecycle management for employee (HR worker) records during ADP Workforce Now to Dynamics 365 Finance & Operations synchronization.

  • Enable Position WriteBack to ADP Workforce Now System: Enable writing position-related updates from Dynamics 365 Finance & Operations back to ADP Workforce Now.

Provisioning Settings

Configure conditions that control when employee records are created, updated, or terminated in Microsoft Dynamics 365 Finance & Operations:

  • Create Employee Condition: Select when a new employee should be created in Microsoft Dynamics 365 Finance & Operations based on records received from ADP Workforce Now. For example, when ALL is selected, employees are created for every ADP worker processed by the integration, when Active Only is selected, employees are created only for workers whose status is active in ADP, and when None is selected, the integration does not create any new employee records.
  • Update Employee Condition: Select when existing employee records in Microsoft Dynamics 365 Finance & Operations should be updated based on changes from ADP Workforce Now. For example, when ALL is selected, employee records are updated regardless of worker status, when Active Only is selected, updates occur only for active workers, when Inactive Only is selected, updates apply only to inactive workers, and when None is selected, no employee records are updated by the integration.
  • Terminate / Deactivate Condition: Select when employee records should be terminated or deactivated in Microsoft Dynamics 365 Finance & Operations. For example, when Inactive Only is selected, employees are terminated or deactivated when their corresponding ADP worker status becomes inactive, while None disables automatic termination or deactivation through the integration.

Manage System Users

Configure filters to control which system users are included in the synchronization:
Note: Fields marked with an asterisk (*) are mandatory.

  • Enable User Sync Source Filter: Enable this option to apply a source-based filter that restricts which system users are synchronized. When enabled, only users matching the selected filter criteria will be processed.
  • Source Filter Key*: Select the attribute used to identify and filter system users. Available options include Employee ID, User Principal Name, and Email. For example, when Employee ID is selected, the integration evaluates system users based on their associated employee identifier.
  • User Sync Source Filter Value: Enter the value corresponding to the selected source filter key. For example, if the source filter key is Employee ID and the value is G3RAXBJ6758CYNNP, only the system user associated with that employee ID will be synchronized.

Role Assignments

Configure how roles and organizational access are assigned to system users during provisioning:

  • Enable Default Role Assignment for System Users: Enable this option to automatically assign default roles to newly created system users in Microsoft Dynamics 365 Finance & Operations.
  • Enable Organization Assignment for System Users: Enable this option to automatically assign one or more organizations to system users during role assignment. For example, when enabled, organizations such as AOUT and CTRC can be assigned to the user as part of role provisioning.
  • List of Organizations Assign to System Users: Enter a comma-separated list of organization IDs to be assigned to system users during role assignment. For example, entering CTRC,AOUT assigns the user access to both organizations.

Provisioning Settings

Configure conditions that control when system user records are created, updated, or deleted in Microsoft Dynamics 365 Finance & Operations. 
Note: Fields marked with an asterisk (*) are mandatory.

  • Create User Condition*: Select when a new system user should be created. For example, when ALL is selected, system users are created for all applicable records processed by the integration, when Active Only is selected, system users are created only for active users, when Inactive Only is selected, system users are created only for inactive users, and when None is selected, no new system users are created.
  • Update User Condition*: Select when existing system user records should be updated based on changes from the source system. For example, when ALL is selected, system users are updated regardless of status, when Active Only is selected, updates occur only for active users, when Inactive Only is selected, updates occur only for inactive users, and when None is selected, no system user records are updated.
  • Delete Condition*: Select when system users should be deleted or deactivated. For example, when Inactive Only is selected, system users are deleted or deactivated when their corresponding source status becomes inactive, while None disables automatic deletion through the integration.
  • Enable Deletion During Termination / Deactivation of System Users: Enable this option to allow system users to be deleted during termination or deactivation events, rather than only being disabled.

Manage Warehouse Workers

Provisioning Settings

Configure conditions that control when warehouse worker records are created and updated in Microsoft Dynamics 365 Finance & Operations. 

Note: Fields marked with an asterisk (*) are mandatory and must be configured for the integration to run.

  • Create Condition for Warehouse Worker*: Select when a new warehouse worker should be created in Microsoft Dynamics 365 Finance & Operations based on records received from ADP Workforce Now. For example, when ALL is selected, warehouse workers are created for every applicable ADP worker processed by the integration, when Active Only is selected, warehouse workers are created only for workers whose status is active in ADP, when Inactive Only is selected, warehouse workers are created only for inactive workers, and when None is selected, the integration does not create any new warehouse worker records.
  • Update Condition for Warehouse Worker Sync*: Select when existing warehouse worker records in Microsoft Dynamics 365 Finance & Operations should be updated based on changes from ADP Workforce Now. For example, when ALL is selected, warehouse worker records are updated regardless of worker status, when Active Only is selected, updates occur only for active workers, when Inactive Only is selected, updates occur only for inactive workers, and when None is selected, no warehouse worker records are updated during synchronization.

Manage Exflow Users

This section allows you to configure how ExFlow user records are synchronized and managed in Microsoft Dynamics 365 ExFlow. It controls which users are included in the integration and defines the conditions under which ExFlow users are created or updated. 
Note: Fields marked with an asterisk (*) are mandatory and must be configured before the integration can run.

Source Filter Settings

Configure filters to control which ExFlow users are included in the synchronization. 

  • Enable Source Filter: Apply a source-based filter to restrict which ExFlow users the integration processes. When enabled, only users matching the selected filter criteria are synchronized.
  • Source Filter Key*: Select the attribute used to identify and filter ExFlow users. Available options include UserID, User Principal Name, or Email. For example, when UserID is selected, the integration evaluates ExFlow users based on their unique user identifier.
  • Source Filter Value: Enter the value corresponding to the selected source filter key. For example, if the source filter key is UserID and the value is Aquera2, only the ExFlow user with that UserID will be processed by the integration.

Provisioning Settings

Configure conditions that control when ExFlow user records are created and updated in Microsoft Dynamics 365 ExFlow. 
Note: Fields marked with an asterisk (*) are mandatory.

  • Create ExFlow User Condition*: Select when a new ExFlow user should be created based on data received from the integration. For example, when ALL is selected, ExFlow users are created for all applicable records processed by the integration, when Active Only is selected, ExFlow users are created only for users whose status is active, when Inactive Only is selected, ExFlow users are created only for inactive users, and when None is selected, the integration does not create any new ExFlow user records.
  • Update Condition for ExFlow Users*: Select when existing ExFlow user records should be updated based on changes from the source system. For example, when ALL is selected, ExFlow user records are updated regardless of user status, when Active Only is selected, updates occur only for active users, when Inactive Only is selected, updates occur only for inactive users, and when None is selected, no ExFlow user records are updated during synchronization.

Notification Settings

This section allows you to configure how integration notifications and reports are generated and delivered after each integration run. It controls the level of detail included in notification reports, how reports are shared via email, and additional safeguards around data visibility and failure alerts.

Note: Fields marked with an asterisk (*) are mandatory and must be configured before notifications can be sent.

Notification Report Settings

Configure the level of detail and content included in integration notification reports.

  • Select the Notification Report Detail Level*: Select the level of detail to include in notification reports sent after an integration run. For example, when Full is selected, the notification report includes complete execution details for all processed records, while Custom allows you to control exactly which attributes are included in the report.
  • Select Custom Mode*: This field is required when the notification report detail level is set to Custom. It determines how attributes are selected for reporting. For example, when Inclusion List is selected, only the specified attributes are included in the notification report.
    • Attributes Inclusion List: Enter the list of attribute names to be included in the notification report when using custom reporting. For example, entering EmployeeId, Email, Status ensures that only these attributes appear in the notification output.

    • Attributes Exclusion List: Specify the list of attribute names that should be excluded from notification reports when using custom notification detail settings. Any attributes listed here will be omitted from the report even if they are available in the source data. For example, entering PersonalEmail, PhoneNumber ensures that these attributes are not included in the notification email or attachment, helping reduce noise or prevent sensitive data from being shared.

  • Attach Notification Summary Report: Enable to include the integration summary report as an email attachment. For example, when enabled, recipients receive a downloadable report file along with the notification email after each integration run.
  • Allow Attribute Values in Email Reports: Enable to include attribute-level data in notification emails. For example, when enabled, detailed attribute values such as employee identifiers or status fields are captured and included in notification reports.
  • Restrict Notifications to Organization Domain: Enable to restrict notification emails to recipients within the configured organization email domain. For example, when enabled, notifications are delivered only to recipients with email addresses belonging to the same organization domain.
  • Notify Aquera Support on Integration Failures: Enable to automatically notify Aquera Support when an integration run encounters failures. For example, when enabled, Aquera Support is alerted in case of critical errors to assist with faster investigation and resolution.

Notification Template for User Creation

Configure the email template used for user creation notifications:

  • Primary Recipients: Specify one or more email addresses or distribution lists that should receive user creation notifications. For example, entering barbara.jane@aquera.com ensures that the specified recipient receives details whenever a new user is created.

  • CC Recipients: Enter additional email addresses to be copied on the notification. For example, adding a shared mailbox provides broader visibility for user-creation events.

  • Email Subject Line: Define the subject line for the notification email. For example, a subject such as ADP ENTA D365 Multi Sync Integration Completed Successfully clearly communicates the outcome of the integration run.

  • Email Message Body: Enter the content of the notification email body. For example, you can include a summary message that describes the integration status, the users created, and any warnings or errors encountered during the run.

Log Insights Settings

Configure how integration execution data and metrics are captured and displayed in Log Insights. These settings control the level of detail shown for each event, including whether attribute-level information is included or excluded. 
Note: Fields marked with an asterisk (*) are mandatory.

  • Log Insights Detail Level*: Select the level of detail to display in Log Insights. When Summary Only is selected, attribute-level details are not displayed, and only the total count of create, update, and delete operations is shown. When Full Details is selected, all available attribute details for each event are displayed. When Custom is selected, you can control exactly which attributes are included or excluded from the insights.
  • Custom Insight Level*: This option is available only when the Log Insights detail level is set to Custom. It determines whether attributes are managed using an inclusion or exclusion approach. For example, selecting Inclusion List shows only the specified attributes in Log Insights, while selecting Exclusion List shows all attributes except those explicitly excluded.
    • Attributes Inclusion List: Specify the list of attribute names that should be included in Log Insights when using the Inclusion List option. For example, entering EmployeeId, Status, Department ensures that only these attributes appear in the insights view.
    • Exclusion Attributes List: Specify the list of attribute names that should be excluded from Log Insights when using the Exclusion List option. For example, entering PersonalEmail, PhoneNumber ensures that these attributes are omitted from the insights to reduce noise or avoid displaying sensitive information.
  • Show Attribute Values for Each Event: Enable this option to display attribute values for each logged event when using the Full Details or Custom insight level. For example, when enabled, Log Insights will show the actual before-and-after values of attributes for create, update, or delete events. This option requires consent to temporarily store event data for analysis purposes.

Attribute Map

The Attribute Map section shows how Dayforce attributes are mapped to Active Directory attributes. These mappings determine which Dayforce fields are used to create or update the corresponding Active Directory fields.

You can review and modify the source and target attributes as needed to support your business requirements. Any changes you make here will directly affect how user data flows from Dayforce to Active Directory during provisioning and updates.

Tables 

Tables allow you to define and maintain reference data that can be used to update or enrich attributes in a target application during integration runs when the required data is not available in the source application.

By using tables, you can map predefined key–value pairs that the integration can reference at runtime to populate or override attribute values in the target system. This is especially useful when the source system does not provide certain attributes, or when additional business logic is required to derive values.

Available Tables

Each table listed represents a specific use case within the integration.

  • D365 System User Roles: This table maps system user identifiers or role-related keys to their corresponding role values in Microsoft Dynamics 365. For example, the table can be used to determine which D365 roles should be assigned to a system user based on a provided key.
  • ExFlow System User Creation Rule Table: This table defines rules that determine which users should be created or synchronised as ExFlow users. For example, a key such as CEO with a value of YES indicates that users matching this rule should be created in ExFlow, while a value of NO excludes them.

Managing Table Data

Click a table row to open the Update Table window. Each row in a table represents a key value pair, where the Name acts as the lookup key and the Value defines the corresponding attribute or rule applied during synchronization.

You can manage table entries in the following ways:

  • Add Row: Use this option to manually add a new entry by specifying a key name and its corresponding value.
  • Upload CSV File: Use this option to upload multiple rows from a CSV file. The CSV file must contain exactly two columns with the headers NAME and VALUE, and each row represents a key–value pair to be added to the table.
  • Export: Use this option to download the existing table data for review or offline editing.

After making changes, click Save to apply and persist the updates.

Note:  The key name within a table must be unique and is used as the lookup identifier during integration execution. Key names can include a combination of letters and numbers and support most special characters, except forward slash (/), backslash (\), and double quotes (").
