Overview

Orchestration is the automated configuration, management, and coordination of applications and services. In the SCIM-to-SCIM orchestration, the SCIM (System for Cross-domain Identity Management) gateway services manage synchronous connections between identity management solutions and target applications. The orchestration takes place between the SCIM-based source application and the SCIM-based target application. 

Configuration 

Configuration enables users to arrange various elements of orchestration in a particular form. This is the first step of orchestration, in which User has to provide a name and select the orchestration type from the dropdown, besides enabling a good number of features to obtain the maximum advantage of the orchestration process. 

After selecting 'Orchestration' from the Aquera's menu, User has to Click on 'Configuration'. Provide a name to the orchestration and select a type of orchestration from the dropdown.

SCIM-to-SCIM has been selected. 

Select a 'Source Application' and 'Target Application' from the dropdown. 

The 'Update Limit' indicates the maximum number of records to be updated in a single run and the 'Termination Delay' is all about the number of days to wait after a record has been deactivated before deleting associated target record.

When selecting 'Enable Caching', the application results are cached. This can be useful while bringing up a new orchestration. (Once everything is working, this should be disabled). 

The 'Admin Email List' specifies the administrative email recipients. User can add the list of recipients here. 

'Match Expression' is used to determine which target record matches a given source record. Here, the 'Employee Number' is chosen as an expression that matches the employees at the source and the target with the same number. User can chose date of birth or any other unique identification number as an expression. 

When the User selected 'all users present in the target application, but absent in the source application, will be deactivated.

When the User selected 'Incremental Mode', lastModified is maintained for the source application, and only source records newer than the timestamp will be processed. This operation should be considered only when the number of source records is very large.

The 'Incremental Mode Initial Timestamp' overrides the incremental timestamp. It must be a valid javascript date string, for example 2020-06-18T08:30z, or 2020-06-18T01:30-0700.

The 'Import Limit' indicates the maximum number of source records to import. This can be useful while determining/debugging mapping conditions. In the normal circumstances, the User can choose 'unlimited' as the value. 

The 'Source SCIM Filer' is an optional SCIM filter applied to source import.

'SOURCE.active create_condition' is an optional expression used to determine if a given user should be created.

Similarly, 'SOURCE.active Update Condition' is also an optional expression used to determine if a given user should be updated 

When the User checks the 'Send New User's Manager Email', an email to be sent to a new user's manager.

'Creation of Distribution List' is the  list of recipients who will receive email when the orchestration creates a new target. The emails to be mentioned here shall be separated by a comma. 

'Creation Email Subject' is used when sending creation email reports.

'Creation Email Body' is the template used as body for creation emails. Target attributes may be included.

When User checked 'Send Deactivated User's Manager Email', an email to be sent to a deactivated user's manager automatically. 

'Deactivation Distribution List' is the list of recipients who will receive email when the orchestration deactivates a target. The Email IDs should be separated by a comma. 

'Deactivation Email Subject' is the subject used when sending deactivation email reports.

'Deactivation Email Body' is the template used as body for deactivation emails. (Target attributes may be included).

User can check 'Enable Group Management' to enable the group management. 

When the 'Group Management Restriction List' is specified, group management is limited to the set of comma separated groups. When not specified, 'Enable Group Management' applies to all groups.

By selecting 'Map Manager', User can map the manager from source to target. 

For the 'Phone Number Formatting' User can select one from the list of "disabled", "automatic", "E164", "INTERNATIONAL", or "NATIONAL[,country=US], where ",country=US" is optional and "US" can be replaced with other countries.

User can choose the 'Password Length', which is the length of password generated when creating new users.

When the User checked 'Report Only' a report is generated which shows what would be changed without actually making the changes.

After selecting the required and relevant fields and entering adequate values, Click on 'Save Changes' to save the Configuration.
