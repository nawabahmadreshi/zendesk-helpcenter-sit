Greenhouse Recruiting Event Listener Setup Guide

← Back to index

Overview

This guide covers information on how the Greenhouse Recruiting Event Listener orchestration can be set up in the Aquera Admin portal. It also details the webhook setup to be done in the Greenhouse Recruiting system to send events to Aquera.

When an event occurs in Greenhouse Recruiting – for example, when a candidate is hired – and a webhook is configured for this event, data about the event is sent to the Aquera endpoint.

Once the setup is complete, all the events configured in the Greenhouse Recruiting system will be sent to Aquera and processed for user provisioning purposes.

The following procedures are involved in the Webhook setup:

  • Setting Up Integration in Aquera
  • Setting Up Webhooks in Greenhouse Recruiting

Setting Up Integration in Aquera

Prerequisite

Make sure that the Greenhouse Recruiting application connector is configured successfully in the Aquera Admin portal. For details on how to configure the connector, refer to the Greenhouse Recruiting Configuration Guide.

  1. Log in to the Aquera Admin portal.
  2. Navigate to the Integrations module, and click Create Integration.
  3. Search for Greenhouse Recruiting Event Listener and select it from the list of integration types available.
  4. Enter the following details on the Create New Integration pop-up window and click Continue:
    • Provide a name for the integration.
    • Enter a description for the integration.
  5. Enter the following on the Configuration page: 
    • Select the name of the Greenhouse Recruiting application connector from the Source Application drop-down.
    • Select the name of the target application from the Target Application drop-down.
    • Enter the expression to match the user records between source and target applications in the Match Expression box. 
  6. Click Create at the top-right corner of the page.
  7. Set the mapping of attributes from source to target.
  8. Click the Event Detail tab to see the event URL.
  9. Click Create Token to generate a token.

  10. Copy the token for future reference and close the Generate Token pop-up window.

  11. Click Save Changes at the top-right corner of the page.

Setting Up Webhooks in Greenhouse Recruiting

Prerequisite

Make sure that the Greenhouse Recruiting application user has permissions to manage and configure webhooks.

Setting Up Webhooks

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center on the left navigation pane and click Webhooks.
  4. Enter the following fields to configure your new webhook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for.
    • When: Select the events in Greenhouse Recruiting that cause the webhook to activate.
    • Endpoint URL: Copy the URL from the Event Detail tab in the Aquera Admin console and paste it in this field. Refer to the Setting Up Integration in Aquera section for details.
    • Secret key: Enter the token generated in the Event Detail tab on the Aquera Admin Portal.
    • Deactivated?: Make sure that the No option is selected.
  5. Click Advanced settings and then provide the Basic Authorisation Credentials where in the token generated from the Event Detail tab in the Aquera Admin console must be provided as the password for authentication. The username can be any random string. Refer to the Setting Up Integration in Aquera section for details on the token.
  6. Click Create webhook to complete the webhook setup.

When you create a new webhook, Greenhouse Recruiting will send a ping to the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the webhook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the webhook.

Note: If the webhook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Web Hook section and edit the webhook configuration to fix the error. Make sure that you set the Deactivated? option to No after you have fixed the configuration issue.
