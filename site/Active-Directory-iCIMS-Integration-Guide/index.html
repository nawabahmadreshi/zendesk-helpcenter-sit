Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the iCIMS to Active Directory integration to sync the user's data.

To identify user records across both systems, the integration uses the email address attribute of candidates in iCIMS and matches it to the extensionAttribute6 attribute of users in Active Directory as the correlation attribute.

Note: When creating a user in the Active Directory (AD) application, a unique Distinguished Name (DN) is assigned to the user. The DN will be checked against all existing users to ensure it is not already in use. If the DN is unique, the user will be created.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • iCIMS: Refer to the iCIMS Configuration Guide for details.
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for details.

Integration Setup

This section details how the integration must be configured to push user's data from iCIMS to Active Directory (AD).

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select iCIMS Candidates to Active Directory Users.
  5. Enter a name and a description for the integration.
  6. Select your iCIMS and Active Directory connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable Logging: Select this to generate logs outlining data changes. If selected, the Log Settings section is displayed.
  • Enable Notifications: Select this to enable and manage the integration-related notifications. If selected, the Notification Settings section is displayed.
  • Enable Report-Only Mode: Select this to generate a report detailing the data changes that would happen, but without applying them.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: The Full Details option is selected by default to populate all the attribute details.

Notification Settings

Configure the following notification settings:

Email Settings

Configure the following email settings:

  • Send User Creation Email to: Indicate who should be notified when a user account is successfully created. The available option is Custom.
  • Recipient's Email Addresses: Enter the email addresses of users to whom the email notifications will be sent.
  • Email Subject on Successful User Creation: Enter the content that must be used as the email subject for sending notifications to recipient's email addresses upon user creation.
  • Email Body on Successful User Creation: Enter the content that must be used as the email body for sending notifications to recipient's email addresses upon user creation.
  • Email Subject on Successful User Creation in Report Only Mode: Enter the email subject to be used for sending notifications to recipients in report-only mode, without creating actual user accounts.
    Note: This email subject is used only if the Enable Report-Only Mode parameter is enabled in the General Settings section.
  • Email Body on Successful User Creation in Report Only Mode: Enter the email body to be used for sending notifications to recipients in report-only mode, without creating actual user accounts.
    Note: This email body is used only if the Enable Report-Only Mode parameter is enabled in the General Settings section.
  • Email Subject for User Creation Failure: Enter the content that must be used as the email subject for sending notifications to recipient's email addresses upon user creation failure.
  • Email Body for User Creation Failure: Enter the content that must be used as the email body for sending notifications to recipient's email addresses upon user creation failure.

Attribute Map

The Attribute Map tab defines the iCIMS attributes that are mapped to the Active Directory attributes for syncing user creation in Active Directory. Only the source attributes can be modified, as needed, to align with your specific use case.

Tables

In the Tables tab, you can add tables to the integration. Refer to the Using Tables in Integrations guide for more details.

Event

The Event tab is used to configure the webhook details.

Provide the following details and save the changes:

  • Event Source: Select your iCIMS application from which all joiner events are triggered.
  • URL: Copy this URL and paste it in the iCIMS Webhooks setup page. For details, refer to the Set up the Message Listener Webhook in iCIMS section.
  • Create Token: Click this button to generate the token that must be passed on the iCIMS Webhooks setup page. For details, refer to the Set up the Message Listener Webhook in iCIMS section.

Set up the Message Listener Webhook in iCIMS

You need to perform the following procedures to set up the event notification rules in iCIMS that trigger the iCIMS events.

Configure Recruiting Workflow Statuses

Perform the following steps to configure the recruiting workflow statuses:

  1. Select Search from the menu bar, then select Recruiting Workflow to access the Recruiting Workflow search.
  2. Within the Recruiting Workflow Search: Filters section, click Show more to expand the section, if necessary, then click Add Filter.
  3. Type status in the search box on the filter popup and select Status under Filters > General. Then, click AddSelected. This is to filter based on specific statuses. 
  4. Select a status from the list to the right of the added filter. For example, to filter hired candidates, select the status Hired: Hired.
  5. You can select multiple statuses from the list by pressing the Ctrl (or Cmd) key on your keyboard and clicking all the relevant statuses in the list.
  6. Click the save icon next to the Search Template field to save the template.
  7. In the Save Search Template popup, provide a title and description for the template. Select the checkbox Use this template, then select User Groups, and click Save. 

Create Event Notification Rule

Perform the following steps to create a rule for event notifications:

  1. Select Admin from the menu bar and navigate to Global (Platform & Portals) > Event Notifications.
  2. Click Create Rule.
  3. In the Create Rule page, provide the following details:

    • Rule Name: Enter a rule name.
    • Enabled: This checkbox will be disabled when creating a new rule.
    • Platform Event: Select the event name Workflow Status Change.
    • Associated Object: Select Recruiting Workflow. 
    • Search Templates: Select the template name created earlier. 
    • Integration URL: Enter the Aquera event URL from the Event tab of the iCIMS Candidates to Active Directory Users Integration in the Aquera Admin portal.
    • Authentication: Select Basic Authentication.
    • Integration Username: Enter the value AqueraListener.
    • Integration Password: Enter the token generated from the Event tab of the iCIMS Candidates to Active Directory Users Integration in the Aquera Admin portal.
  4. Click Save to save the event notification rule.

Enable Event Notification Rule

New rules cannot be enabled until the search engine is rebuilt. The Enabled checkbox in event notification rules can only be selected after the rebuild.

Perform the following steps to rebuild the search engine:

  1. Select Admin from the menu bar and navigate to Tools > Soft Reset.
  2. Click the Reset button under the section Reset Search Engine.

  3. You will need to wait sometime for the search engine to reset.

Perform the following steps to enable the event notification rule:

  1. Select Admin from the menu bar and navigate to System Configuration > Talent Management.
  2. In the Setup tab, click the expand icon to expand your integration.
  3. Under the Event Notifications section, click Associate Existing Event.
  4. Select the checkbox for your event notification rule name and click Save.
  5. Select your rule under Event Notifications.
  6. In the Edit Rule page, select the checkbox Enabled and click Save.

Once the setup is complete, any change in a candidate's workflow status will trigger an event notification from the iCIMS talent platform to Aquera’s iCIMS Message Listener. The listener processes the event and updates the candidate data in the relevant target application.
