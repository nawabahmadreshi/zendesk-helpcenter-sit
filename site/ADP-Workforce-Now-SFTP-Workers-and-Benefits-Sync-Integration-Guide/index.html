Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the integration between ADP Workforce Now API Central and Bswift for employee synchronization and benefit election processing. The integration supports employee data synchronization from ADP to Bswift and processes benefit election files from Bswift to ADP through secure file transfer and automated data transformation workflows.

Key Functions

The ADP Workforce Now / SFTP Workers integration supports bidirectional data exchange for employee synchronization and benefit election processing.

Employee Synchronization (ADP to Bswift)

The integration supports employee data synchronization from ADP Workforce Now to Bswift by performing the following functions:

  • Retrieves employee data from ADP Workforce Now.
  • Transforms employee data into the required file format.
  • Delivers employee data files to Bswift through the FTP server.

Benefit Election Processing (Bswift to ADP)

The integration supports benefit election processing from Bswift to ADP by performing the following functions:

  • Reads benefit election files from the Bswift FTP server.
  • Transforms and processes benefit election data for ADP payroll and benefits administration updates.

Integration Setup

This section outlines how the integration must be configured to push employee data from BSwift to ADP Workforce Now for ADP API Central Customers.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / SFTP [Benefits] in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

  • ADP API Central reads employee data from ADP Workforce Now and writes benefits deductions back to it. 
  • Write Demographic uploads ADP employee data as CSV to BSwift for benefits enrollment.
  • Read Benefits downloads BSwift CSV to update ADP deductions (start/change/stop).
  • Read Demographic Headers retrieves CSV column names to validate the BSwift demographic format

Workers Sync Settings

The Worker Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Worker Settings

Configure the following worker settings:

  • Enable Worker Filtering: Select to apply filtering options while retrieving a worker from ADP Workforce Now.
    • Source Filter*: Select the ADP Workforce Now filter to filter the workers while retrieving from ADP Workforce Now. The available options are Filter By Associate ID, Filter By Position ID, and Import Limit.
    • Source Filter Value*: Provide the value for the selected source filter.
      Note: This option will appear when you select Filter By Associate ID or Filter By Position ID as the source filter.
    • Worker Import Limit*: Specify the maximum number of worker records that can be imported from ADP Workforce Now during a single run of the integration.
      Note: This option will appear when you select Import Limit as the source filter.
  • Employee Creation Condition in BSwift: Select the create condition for creating a worker in BSwift. The available options are Create All Workers, Create Active Users, and Create Full-Time Workers.
  • Employee Create Limit*: Specify the maximum number of worker records that can be imported into BSwift during a single run of the integration.
  • Use Direct CSV Header: Select to manually define CSV headers instead of reading them from the file path configured in the application's tenant attribute.
    Note: If this checkbox is selected, the CSV Column Header section is enabled.
  • Custom Filter: Enter a custom filter condition to retrieve specific workers from ADP Workforce Now during synchronization. Use supported operators such as eq, ne, gt, lt, gte, and lte combined with AND or OR logics. Field names and values must be enclosed within single quotes. For example, enter 'Worker Status' eq 'Active' to filter only active workers. To apply multiple conditions, enclose each condition within brackets (), such as ('Worker Status' eq 'Active') AND ('Worker Category' eq 'Full Time') to filter only active full-time workers.
    Note: When source filtering is applied, the custom filter validates only the worker data returned from the applied source filter.

CSV Column Header Settings

Configure the following CSV column header settings:

  • CSV Column Header Names*: Enter the column header names to create in the CSV file. The column values are generated based on the mappings configured in the Worker to Employee Sync section.

Benefits Sync Settings

The Benefit Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Date Formats Available in CSV: Enter the date format in uppercase letters only. For example, DD-MM-YYYY or D/M/YY.
  • Enable Start Benefits Sync: Select to enable synchronization of new benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, benefit records marked as start events in the BSwift CSV file are created in ADP.
  • Enable Stop Benefits Sync: Select to enable synchronization of ended or stopped benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, benefit records marked as stop events in the BSwift CSV file are ended in ADP.
  • Enable Change Benefits Sync: Select to enable synchronization of changes to existing benefit elections or deduction records from BSwift to ADP Workforce Now (API Central). When enabled, modified benefit records in the BSwift CSV file are updated in ADP.
  • Import Limit: Specifies the maximum number of benefit election records (CSV rows) that can be processed from the BSwift file in a single integration run. When this limit is reached, any additional benefit records in the file are skipped for that run.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can find the following mapping sections under this tab:

  • Worker to Employee Sync: ADP Workforce Now worker attributes are mapped with BSwift employee attributes using column header names.
  • Election Benefits Map: BSwift employee attributes are mapped with the ADP Workforce Now worker. This section has certain mapping limitations defined by ADP Workforce Now. Refer to the ADP API documentation for details on supported fields and constraints: ADP Developer Resources.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
