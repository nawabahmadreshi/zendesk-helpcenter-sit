Watchdog Setup Guide: Sailpoint Identity Security Cloud

Overview

Aquera Watchdog is a monitoring and validation tool designed to identify data synchronization issues between integrated applications. It operates independently of the Aquera integration and analyzes application events to detect inconsistencies between source and target systems.

After an integration execution is completed, you can run the Watchdog to retrieve the event logs, analyze the activities, and report any anomalies or data discrepancies across the applications. For example, if a user is created in the source application, Watchdog checks for a corresponding user provisioning event in the target application and reports an anomaly if no matching event is found. Aquera Watchdog helps ensure guaranteed delivery of events across integrated systems by continuously monitoring and validating data synchronization between the source and target applications.

Watchdog ensures data consistency across systems, provides visibility into sync failures, and helps proactively identify and resolve integration issues. 

You can even run the Watchdog for a specific application. When you run the application Watchdog, a dashboard is generated with a consolidated view of the activities in the application. The dashboard can be accessed through the Insights module. 

This guide provides details on how to configure and run the Watchdog for the Sailpoint Identity Security Cloud application. 

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure the SailPoint Identity Security Cloud Connector in the Aquera Admin Portal: You must configure the SailPoint Identity Security Cloud in the Aquera Admin Portal to connect to the application for retrieving the event logs. Refer to the SailPoint Identity Security Cloud Configuration Guide for details.

Watchdog Setup

This section describes how to configure the Watchdog that monitors the events in SailPoint Identity Security Cloud.

Create Integration Watchdog

Follow these steps to create the integration watchdog:

  1. Log in to the Aquera Admin Portal.

  2. Navigate to Integrations.

  3. Click Create Integration.

  4. Select SailPoint Identity Security Cloud [Watchdog].

  5. Enter a name and description for the Watchdog.

  6. Choose your SailPoint Identity Security Cloud connector instance.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the SailPoint Identity Security Cloud connector that you have selected while creating the integration watchdog. This section enables you to search and select a different instance of the connector, if required.

Parameters

The Parameters section enables you to customize the Watchdog settings according to your use case. It has the following settings:

  • Initial log lookback period: Specify the time period from which the watchdog retrieves historical logs from SailPoint Identity Security Cloud during its initial execution. After the first run, the watchdog processes only the event logs that have been created since the last run. These logs are used by Watchdog to analyze the events. The following options are available:
    • Last: Select this option to retrieve logs from a specific duration in the past. You can specify the number of days, hours, or minutes for which logs should be retrieved.
    • Since the date: Select this option to retrieve and process logs starting from a specific date. A calendar field is displayed to select the date.
    • From the beginning of available logs: Select this option to retrieve all event logs currently available in SailPoint Identity Security Cloud.

Policy

The Policy tab enables you to set the access policies of users for Applications, Groups and Directory. 

When the Watchdog runs, these policies are evaluated to identify deviations and potential access violations. These policies serve as the basis for auditing user access and detecting compliance issues. Based on the evaluation results, Watchdog generates Audit dashboards and reports that help administrators review and analyze access compliance. 

Access Policy

The Access Policy section allows you to define the SailPoint roles and access profiles that users should or should not be assigned.

Role Policy

Turn on the Enable toggle to configure the policies for role assignments. The following sections are displayed.

Required Roles

Use this section to define the conditions for assigning eligible users to the various roles in SailPoint Identity Security Cloud. Users who meet these conditions are expected to be present in those roles.

  1. Turn on the Enable toggle to configure the policies for required roles.
  2. Click on Choose Attribute to select a user attribute that you want to use in the condition.
  3. After selecting the attribute, specify one or more attribute values. Users who match any of the selected attribute values (OR operation) are considered eligible for the role.
  4. If required, add more attributes and values. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. This means users who satisfy all attribute conditions in that row must be part of the corresponding role.
  5. In the Roles field, select the roles to which users matching the defined conditions should be assigned.
  6. If required, click Add Row to add another condition within the same rule. 

  7. If required, click the + icon at the top-right corner of the rule setting section to add another rule with a set of rows/conditions for required roles. 

    When multiple rules are set, the integration evaluates every rule for each user.

Forbidden Roles

Use this section to define the conditions under which users must not be assigned specific SailPoint roles. Users who satisfy the configured conditions are expected not to have the selected roles.

The policy setup is similar to the Required Roles procedure.

Access Profile Policy

Turn on the Enable toggle to configure the access profile policies. The following sections are displayed.

Required Access Profiles

Use this section to define the conditions under which users must be assigned specific SailPoint access profiles. Users meeting the configured conditions are expected to have the selected access profiles.

To configure required access profile policies:

  1. Turn on the Enable toggle.
  2. Click Choose Attribute to select a user attribute for the policy condition.
  3. Select the corresponding attribute value(s). Users matching any selected value (OR condition) satisfy that attribute condition.
  4. If required, add additional attributes to the same row. Multiple attributes within a row are evaluated using an AND operation.
  5. In Choose Target, select one or more SailPoint access profiles that users meeting the defined conditions must be assigned.
  6. Click Add Row to add additional conditions within the same rule.
  7. Click the + icon to create additional rules with their own conditions.

When multiple rules are configured, Watchdog evaluates every rule independently for each user.

Forbidden Access Profiles

Use this section to define the conditions under which users must not be assigned specific SailPoint access profiles. Users meeting the configured conditions are expected not to have the selected access profiles.

The policy setup is similar to the Required Access Profiles procedure.

Schedule

The integration Watchdog can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details. 

Watchdog Insights

This section details the data covered in the dashboard created by the Watchdog for the SailPoint Identity Security Cloud events. 

  1. Log in to the Aquera Admin portal.
  2. Navigate to Insights.
  3. Click on the integration watchdog name, for example, SailPoint identity Security Cloud Watchdog. The dashboards are displayed.

The dashboard provides a comprehensive view of the identity lifecycle activity, entitlement changes, policy compliance, and access provisioning within SailPoint Identity Security Cloud. It includes the following insights:

 

  • Displays the total number of identity lifecycle events during the selected reporting period.

  • A pie chart showing the percentage distribution of workers across different departments.

  • Detailed logs of events in the Dayforce application and the Entra ID directory.
