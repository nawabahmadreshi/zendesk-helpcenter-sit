Array Mapping in Data Integration

← Back to index

Overview

An application can have attributes of the type array of objects with sub-attributes. You can map target attributes of the type array of objects in the attribute mapping of Data Integration. The array of objects is denoted by the symbol [{}] in the attribute map. To map a source attribute to a target attribute that is an array of objects, click on the Map Array of Objects option corresponding to the target attribute. You can then add the source array of objects from which the sub-attributes need to be mapped to the sub-attributes under the target array of objects.

Based on the array mapping that you configure through the Map Array of Objects option, an expression will be generated and displayed on the Attribute Map page.

Note: You cannot directly enter an expression in the Source Attribute/Expression field against a target array of objects. 

Adding Object Arrays for Mapping

On clicking the Map Array of Objects option, you will see the options to map source object arrays to the selected target object array. Click + Add to see the list of object arrays available in the Source application resource.

You can either select one of the Source object arrays to map its sub-attributes to the target sub-attributes or select Single Object to add a new object array. You can add multiple object arrays if required.

Adding a New Object Array for Mapping

From the list that is displayed on clicking the + Add, select Single Object to create a new object array for mapping.

The sub-attributes of the target application's object array are displayed on the right side under Target Attribute. You can provide a mapping from a source attribute to a target sub-attribute by selecting a source attribute from the dropdown list or writing expressions in the Enter Attribute field. 

To enter a source attribute, you must follow the format SOURCE.<sourceAttributeName>. For example, SOURCE.value, where value is an attribute in the Source application.

If the target object array that you have selected has another object array within it, you will see the Map Array of Objects option to configure the mapping for that object array also, as shown in the above image.

Adding a Source Array for Mapping

From the list that is displayed on clicking the + Add, select a relevant source attribute array to map to the target attribute.

To enter a source attribute within the selected source object array, follow the format <objectVariableName>.<sourceAttributeName>. The default variable name of the object array is item. Adding an Array within a Source Array

You can add an array of source objects within another object array and map the sub-attributes from that object array to the target object array attributes. Perform the following steps to add an object array within a source array of object:

  1. Click the ellipsis icon next to the source object array and click Add New Array.

    The list of all the source object arrays is displayed.
  2. Select the source object array to be added. The selected object array is displayed within the other object array, as follows:

    Similarly, you can add any number of hierarchical levels of source object arrays.

You can set the mapping of attributes from these arrays to the target array attributes.

Providing a Condition and a Filter

You can provide a condition based on which the array values are mapped to the target attribute. You have to click Add Condition and then provide a JavaScript expression in the Condition field that evaluates to a boolean value. In this example, only if the active value returned by the source is true, the object array mapping is effective. Otherwise, an empty object array will be mapped.

You can provide a filter to filter the items within the selected source attribute object array. You can provide JavaScript expressions in the Filter field that should evaluate to a boolean value. 

Based on the mapping provided for the target sub-attributes with the conditions and filter expressions set, an array map expression is formulated and displayed at the top level of the attribute mapping. The following is a sample array map expression:

[...(SOURCE.active?[...[...SOURCE.employeeManagers.map((item)=>({emplRef.accountId:Number(item.ManagerXRefCode),
index:Number(item.ManagerBadgeNumber)}))].filter((item)=>
item.DerivationMethod === 2)]:[])]  

Note: You cannot modify this expression generated by the integration.

Single Array Index Mapping

You can map a source sub-attribute of an array of a specific index to a target attribute in the attribute mapping. Select an attribute of type array of objects in the Source Attribute/Expression field and specify a dot, it will list all the sub-attributes of that attribute array.

Select a relevant sub-attribute to map to the target attribute. Then, update the index n to the required index of the array from which you want to map to the target attribute.

Ensure to click Save Changes to save the mappings.

Known Limitation

Array mapping does not support the use of variable assignments (such as let, const, and var).
Array mapping also does not support if-else statements. To write conditional logic, use a conditional or ternary operator that has the following syntax: {condition} ? {expressionIfTrue} : {expressionIfFalse}
