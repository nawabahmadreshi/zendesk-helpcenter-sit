Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize the cost center data from ADP Workforce Now to ADP iHCM.

To identify cost center records across both systems, the integration uses the Employee number attribute from ServiceNow and matches it to the Employee ID in the Active Directory as a correlation attribute.

Key Functions

The ADP iHCM / ADP Workforce Now Cost Center Sync integration performs the following functions:

  • Creates new cost centers in ADP iHCM when new cost centers are added in ADP Workforce Now.
  • Updated cost center details in ADP iHCM when any changes are made to the existing cost centers in ADP Workforce Now.
  • Deletes the corresponding cost centers in ADP iHCM when cost centers are deleted in ADP Workforce Now.
  • Performs full synchronization of cost centers on each integration run.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for configuration details.
    • ADP iHCM: Refer to the ADP iHCM Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to synchronize cost center data from ADP Workforce Now to ADP iHCM.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP iHCM / ADP Workforce Now Cost Center Sync.
  5. Enter a name and a description for the integration.
  6. Select your ADP iHCM and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenario

Enable the following feature based on your requirements:

  • Enable Worker Onboarding: Select this checkbox to create worker records from SolarWinds Service Desk to ADP Workforce Now. When enabled, the Worker Management section is displayed for further configuration.

General Settings

Configure the following general settings:

  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable reporting generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable log insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable debug mode: Select to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Settings section is displayed for further configuration.
  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.

Worker Management

The Worker Management section allows you to customize integration settings to manage workers based on your specific use cases.

SolarWinds Service Desk Settings

Configure the following SolarWinds Service Desk settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Ticket status for worker creation*: Select a ticket status from the dropdown menu. Only tickets with the selected status from SolarWinds Service Desk will be used to identify workers to be created in ADP Workforce Now. 
  • Ticket status to set after worker creation*: Select a ticket status from the dropdown menu. Once the worker record is successfully created in ADP Workforce Now, the ticket status in SolarWinds Service Desk will be updated to the selected status.
  • Import limit: Enter the maximum number of worker records to be processed in a single integration execution. For example, entering 10 processes only includes the first ten worker records.

ADP Workforce Now Settings

Configure the following ADP Workforce Now settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Onboarding template*: Select an ADP Workforce Now template from the dropdown menu. The worker records will be created in the selected template.

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable logging parameter in the Parameters section.

  • Select Log Type: Select a log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data. 
  •  Select Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the log.
    • Include List: Select this option to include attributes in the log.
      • Custom Inclusion List: Enter the attributes to be included in the log. This parameter is available only if you select Include List in the Select Log Level parameter.
    • Exclude List: Select this option to exclude attributes from the log.
      • Custom Exclusion list: Enter the attributes to be excluded from the log. This parameter is available only if you select Exclude List in the Select Log Level parameter.
