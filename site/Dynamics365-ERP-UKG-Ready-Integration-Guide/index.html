Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any users created, updated, or terminated in Entra ID are automatically updated in Microsoft Dynamics 365 based on the defined schedule.

To identify user records across both systems from Entra ID to Microsoft Dynamics 365, the integration uses the attribute configured in the Entra ID Correlation Attribute parameter in Entra ID and matches it to the attribute configured in the Microsoft Dynamics 365 Finance and Operations Correlation Attribute parameter in Microsoft Dynamics 365 as a correlation attribute.

Key Functions

The Dynamics 365 / Entra ID Integration performs the following functions:

  • Synchronizes user records from Entra ID to Microsoft Dynamics 365.
  • Creates new HR workers and system users in Microsoft Dynamics 365 when they are added in Entra ID.
  • Updates existing HR worker and system user records in Microsoft Dynamics 365 when the corresponding user information changes in Entra ID.
  • Terminates the HR worker and system user records in Microsoft Dynamics 365 when the corresponding user records are deleted from the Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

    • Entra ID: Refer to the Entra ID Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to Microsoft Dynamics 365.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dynamics365-ERP / UKG Ready.
  5. Enter a name and a description for the integration.
  6. Select your Microsoft Dynamics 365 and UKG Ready connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Manage Employees

This section enables you to customize the integration settings for managing employees according to your specific use cases.

Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  •  Source Correlation Attribute*: Select the UKG Ready attribute to distinctly find the associated Microsoft Dynamics 365 employee record. For example, EmployeeId.
  • Target Correlation Attribute*: Select the Microsoft Dynamics 365 attribute to distinctly find the associated UKG Ready employee record. For example, PersonnelNumber.

Source Filter

Configure the following settings to filter employee records in UKG Ready.

  • Source Filter: Select an attribute to filter UKG Ready users for provisioning to the Microsoft Dynamics 365 application. For example, if you select EmployeeId, UKG Ready users will be filtered based on this attribute.
  • Source filter Value: Enter the value by which UKG Ready users must be filtered for the attribute selected in the Source Filter parameter. For example, if you enter EMP001, UKG Ready users with this employee number will be filtered and provisioned to the Microsoft Dynamics 365 application.

Create Settings

Configure the following settings to apply a condition for creating employee records from UKG Ready to Microsoft Dynamics 365.

  • Create Condition Type: Select a condition to specify which UKG Ready employee records must be created in the Microsoft Dynamics 365 application. Only employee records that satisfy this condition will be created in the Microsoft Dynamics 365 application. The available options are None and Basic. Select None if you . Select Basic if you want to filter UKG Ready users by status before creating them in the Microsoft Dynamics 365 application. The following parameter is displayed when the Basic option is selected:
    • Eligible Worker Statuses for Create: Select one or more worker statuses from the dropdown menu. For example, if you select Active, only users with the Active status are filtered and created in the Microsoft Dynamics 365 application.

Provisioning Settings

Configure the following settings to apply a condition for provisioning employee records from Entra ID to Microsoft Dynamics 365.

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Create Condition*: Select a condition to specify which Entra ID employee records must be created in the Microsoft Dynamics 365 application. Only employee records that satisfy this condition will be created in the Microsoft Dynamics 365 application. For example, if you select Active Only, only employees with an active status are provisioned in the Microsoft Dynamics 365 application. 
  • Update Condition*: Select a condition to specify which Entra ID employee records must be updated in the Microsoft Dynamics application. Only employee records that satisfy this condition will be updated in the Microsoft Dynamics 365 application. For example, if you select ALL, all Entra ID employee records will be updated to the Microsoft Dynamics 365 application.

Provisioning Threshold

Configure the following settings to limit the employee records provisioned from Entra ID to Microsoft Dynamics 365.

  • Create Limit: Enter the maximum number of employee records that can be created from Entra ID to Microsoft Dynamics 365 in a single integration run. For example, entering 7 allows only 7 employee records to be created in a single integration run.
  • Update Limit: Enter the maximum number of employee records that can be updated from Entra ID to Microsoft Dynamics 365 in a single integration run. For example, entering 5 updates only 5 employee records in a single integration run.

Attribute Map

The Attribute Mapper tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
