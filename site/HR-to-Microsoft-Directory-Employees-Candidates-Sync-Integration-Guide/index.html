Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating employee and candidate synchronization from HR applications to Active Directory or Entra ID.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The HR to Microsoft Directory integration facilitates the synchronization of employee and candidate records from supported Human Resource (HR) systems to Microsoft Active Directory (AD) or Microsoft Entra ID, based on the configured integration setup. Whenever an employee record is created or updated in the source HR system, the corresponding user account information is automatically provisioned or updated in the target Microsoft directory.

This integration currently supports employee synchronization from any of the following HR systems: 

  • Dayforce
  • ADP Workforce Now

This guide covers details on how to set up the HR to Microsoft Directory integration to enable automated synchronization of employee and candidate data from the supported HR systems to Active Directory or Entra ID. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
     
  • Configure Applications in the Aquera Admin Portal:  
     
    • HR Connector: Ensure you add and configure the HR connector in the Aquera Admin portal. For details, refer to the corresponding connector configuration guides.
    • Microsoft Entra Native: If the integration is intended to provision employee accounts to Microsoft Entra ID, ensure that you add and configure the Entra ID connector in the Aquera Admin portal. Refer to the Microsoft Entra Native Configuration Guide for details.
    • Active Directory Connector: If the integration is intended to provision employee accounts to Active Directory, ensure you add and configure the Active Directory connector version 8 in the Aquera Admin portal. Refer to the Active Directory Version 8 Configuration Guide for details.

Note:

When you configure the Active Directory connector, specify the hostname or IP address of a dedicated Active Directory domain controller as the tenant value. You must not use the hostname of a load balancer that may be deployed in front of the domain controllers for high availability purposes.

Integration Setup

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select HR to Microsoft Directory [Employee/Candidate Sync]. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration tab of the integration setup page is displayed. The configurations are categorized under various sections within the Configuration tab.
  7. In the Connections section, select the source and target applications.
    • From the Source Application drop-down list, select the HR system from which employees must be retrieved. The drop-down lists only the HR application connectors supported by the integration.
    • Select the Microsoft directory (Active Directory or Entra ID)  to which users must be provisioned from the Target Application drop-down list. The drop-down lists only the Active Directory and Entra ID connectors configured in the tenant.
  8. Click Save Changes to save the integration.
  9. Set the configurations as per your requirements. Refer to the Integration Configurations Reference Guide for details.

Configuration

The Configuration tab includes a list of configurations required to set up the integration.

Connections

The Connections section enables you to configure the application connectors that you want to use for the integration.
Note: The Source Application drop-down list shows only the HR system supported by the integration and the Target Application drop-down list shows all the Active Directory and Entra ID connector instances configured in your tenant.

Source Resource Type: Select the type of record you want to sync from the drop-down list.

The following resource type available:

  • EmployeeF: Select this option to synchronize employee records between the HR system and the directory.
  • CandidateF: Select this option to synchronize candidate records between the HR system and the directory.

Employee Sync

To sync the employee record, you must specify the Source Employee Attribute and the Target Employee Attribute that must be used to match employee records between the HR system and the directory in the Source Match Expression section. 

When a candidate is converted into an employee, enabling the Enable Candidate Management toggle converts the existing candidate record into an employee record. If the toggle is disabled, the system creates a new employee record while retaining the candidate record, which can result in duplicate entries.

Note: Click Switch to Expression Mode to configure a JavaScript expression that must be used to identify the matching records. For example, SOURCE.EmployeeNumber == DEST.EmployeeNumber

For detailed information on all the configuration fields in the Configuration tab, refer to the Integration Configurations Reference Guide.

Candidate Sync

To sync the candidate record, you must specify the Source Candidate Attribute and the Target Candidate Attribute that must be used to match candidate records between the HR system and the directory in the Source Match Expression section. 

Note: Click Switch to Expression Mode to configure a JavaScript expression that must be used to identify the matching records.

For detailed information on all the configuration fields in the Configuration tab, refer to the Integration Configurations Reference Guide.
