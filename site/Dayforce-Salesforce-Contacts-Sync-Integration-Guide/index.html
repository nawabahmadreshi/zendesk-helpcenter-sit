Integration Setup

This section outlines how the integration must be configured to push user-related data from Jira Service Management to Active Directory.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce to Salesforce.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Salesforce connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable Manage Contact Lifecycle Management: Select this checkbox to synchronize user's contact data from Dayforce to Salesforce.
  • Dayforce server timezone offset (from GMT)*: Enter the timezone 
  • Enable Incremental Mode: Select to run the integration in incremental mode.

Integration Execution Features Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

Contact Life Cycle Management

The Contact Management section enables you to customize the integration settings for contractor management according to your specific use cases.

Salesforce Settings Manage contact lifecycle management

Configure the following settings:

  • Create Contacts: Select to create a contact in the Salesforce application only if the contact details are not available.
  • Update Contacts: Select to update the user’s contact details in the Salesforce application if the contact information is not up to date.
  • Verify User Before Creating Contact: Select to create a contact only after confirming that the corresponding Salesforce user exists and is active. If the user is unavailable or inactive, contact creation will be ignored, and the same will be updated in the integration report.
  • Map Manager to Contact: Select to map the user's direct manager's contact details 
  • Create Contacts Only for New Hires: Select to create contacts in the Salesforce application only for new hires.

Eligible Statuses for Contact Creation And Update Statuses Settings

Configure the following settings:

  • Eligible Statuses for Contact Creation: Select one or more user statuses. Contacts will be created in the Salesforce application only for users with the selected statuses.
  • Eligible Statuses for Contact Update: Select one or more user statuses. Contacts will be updated in the Salesforce application only for users with the selected statuses.

Correlation Settings

Configure the following correlation settings:

  • Dayforce Correlation Attribute: Select a Dayforce attribute to distinctly find the associated Salesforce contact record. For example, if you select EmployeeId in this field, the EmployeeId attribute will be used as a correlation attribute.
  • Salesforce Correlation Attribute: Select a Salesforce attribute to distinctly find the associated Dayforce contact record. For example, if you select Department in this field, the Department attribute will be used as a correlation attribute.

User Lookup Configuration

Configure the following settings:

  • Salesforce User Field Attribute: Select a Salesforce user attribute to distinctly find the associated Salesforce contact record. For example, if you select Username attribute in this field, 
  • Salesforce Contact Field: Select a Salesforce contact field to distinctly find the associated Salesforce user record.

Debug Settings

This tab enables you to set debug options to help troubleshoot and fix integration issues.

Note: The Debug Settings tab appears when you select the Enable debug mode checkbox in the Operational Settings section of the Parameters tab.

  • Enable single-user testing: Select to debug the integration for a single user.
  • Source Dayforce EmployeeNumber: Enter the employee number of the Dayforce user to be used for debugging purposes.
  • Import Limit: Enter the maximum number of user contacts to be imported to Salesforce for debugging in a single integration run.

Notifications

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

Notification Settings

Configure the following notification settings:

  • Send the Notification Summary Report As An Attachment Send Integration Report via Email: Select this checkbox to receive an email notification with a report for each integration execution. When enabled, the Email Report Settings section is displayed for further configuration.
  • Consent to Send Email Report with Attribute Values Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values for each event when choosing the Full Details or Custom option.
  • Select the Notification Report Detail Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    •  Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the  Attribute Filter* drop-down menu with the following options:
      • Included Attributes: Enter the attributes to be included in the email report. This parameter is available only if you select Included Attributes in the Attribute Filter section.
      • Excluded Attributes: Enter the attributes to be excluded from the email report. This parameter is available only if you select Excluded Attributes in the Attribute Filter section.
  • Admin's Email Address: Enter the email addresses of the admins to whom the email notifications will be sent upon successful integration.
  • Domain: Enter a comma-separated list of domains that should receive integration-related notifications. For example, if you enter aquera.com, all email addresses with this domain will receive the notifications.
  • Email Subject: Enter the content that must be used as the email subject for sending email notifications.
  • Email Body: Enter the content that must be used as the email body for sending email notifications.
