Set Up Webhooks in Pinpoint

Overview

When a specific set of events occurs in an application or system, Aquera integrations can process those events and update those changes in a different system or application. The events can be specific data changes in the application such as creation of a new user, creation of a job, scheduling of an interview, acceptance of an offer, and so on. For the application to send those events, you must configure webhooks in the application and specify the set of events for which the notifications have to be sent. When you configure webhooks in that application, you must also specify Aquera's event integration URL to which you want to send those event notifications. 

This guide explains the webhooks setup to be done in the Pinpoint application to enable the application to send events to Aquera. After the webhooks setup is completed in Pinpoint, when a specific event occurs, Pinpoint sends a JSON payload containing the event details to the Aquera event integration URL configured in the webhooks. Depending on the type of event, the payload data vary.

Aquera integration then processes these events and updates the corresponding data in a target application.

Prerequisite

You must set up an integration in the Aquera Admin portal to process the Pinpoint events. For details, refer to the Integration Using Scripts guide.

Set Up Webhooks in Pinpoint

This section details the procedure to be followed in Pinpoint to set up Webhooks.

  1. Login to Pinpoint using your admin credentials.
  2. Navigate to Company > Profile > Webhooks.
  3. Click New webhook.
  4. Specify the event URL that was generated in the Aquera Admin portal when you set up the integration. For details, refer to the Integration Using Scripts guide.
    Note: Along with the event URL, an authentication token is also displayed in the Aquera Admin portal. However, this token must not be configured in PinPoint as PinPoint webhooks do not support authentication.
  5. Enable the list of events for which the payload has to be sent to Aquera. The following table lists all the events that can be enabled in Webhooks:

    New application

    		 User created on company Onboarding stage completed (V2)
    Application stage changed Job created Talent Pipeline candidate created
    Offer accepted Job updated Requisition created
    Applicant hired Job deleted Requisition approved
    Document request accepted Onboarding workflow completed Interview scheduled
    Application moved Onboarding completed
  6. Save the webhooks configuration.

You can send test events to Aquera to verify that the configuration is successful.

Note: The timeout period for a request is set as 5 seconds. If there is no response to a request in 5 seconds, the request times out and will be considered as unsuccessful. Any unsuccessful requests will be retried based on the retry policy.
