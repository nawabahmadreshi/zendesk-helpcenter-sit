Overview

This integration retrieves employee data from a CSV file stored on a Paycom SFTP server and loads it into a Jenzabar Microsoft SQL Server (MSSQL) database table.

The integration runs on a configured schedule, reads the CSV file from the Paycom SFTP location, optionally filters records, maps CSV columns to Jenzabar database fields, and writes the data into the target database table.

The integration supports both Insert and Upsert operations and provides logging, reporting, analytics, notification, and recovery capabilities to support reliable data synchronization.

Key Functions

  • Retrieves employee data from a CSV file stored on the Paycom SFTP server.
  • Synchronizes CSV data into a Jenzabar MSSQL database table.
  • Supports Insert and Upsert database write operations.
  • Maps Paycom CSV columns to Jenzabar database fields.
  • Filters records before processing using configurable row-filter rules.
  • Supports cache recovery for failed database write operations.
  • Provides execution logging, reporting, and analytics.
  • Supports masking of sensitive data in logs, reports, and notifications.
  • Sends success, failure, and skipped-record email notifications.
  • Supports scheduled execution using basic or advanced schedules.

Prerequisites

Before configuring the integration, ensure that:

A Paycom SFTP application is configured in Aquera.
A Jenzabar MSSQL application is configured in Aquera.
The Paycom SFTP server contains the CSV file to be processed.
The Jenzabar database, schema, and target table are accessible.
Required database permissions are available for inserting or updating records.
Required CSV columns are available for attribute mapping.
