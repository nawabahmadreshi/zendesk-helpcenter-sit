Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the process for setting up an integration to retrieve employee demographic and compensation data from ADP API Central, convert that data into CSV format, and deliver the resulting file to Bswift via FTP.

Key Functions

The ADP API Central to BSwift integrations provides the following functions:

  • Retrieves employee demographic and compensation data from ADP Workforce Now.
  • Transforms the retrieved data into CSV format.
  • Delivers the CSV file to Bswift via FTP server.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now for ADP API Central Customers: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for details.
    • BSwift: Refer to the FTP Generic Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push worker data from ADP Workforce Now for ADP API Central Customers to BSwift via FTP.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Central to BSwift Users Sync.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now for ADP API Central Customers and BSwift connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenario

Configure the following integration scenario:

  • Enable Workers Sync: Select this checkbox to enable workers sync from ADP to Bswift. If the checkbox is not selected, the worker records will not be synced.
    Note: The Workers Sync Settings section will be displayed only when this checkbox is selected.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
    Note: The Log Insights Settings section will be displayed only when this checkbox is selected.

Workers Sync Settings

Configure the following settings in the Workers Settings section:

  • Enable Worker Filtering: Select this checkbox to apply filtering options while retrieving a specific worker from ADP Workforce Now. When this checkbox is selected, the following settings will be displayed:
    Note: Leave this checkbox unchecked when running a full sync integration.
    • Source Filter: Select the source filter to filter the workers while retrieving from ADP Workforce Now. The available options are Filter By Associate ID, Filter By Position ID, and Import Limit.
    • Source Filter Value: Provide the value for the selected source filter. This option will appear when you select Filter By Associate ID or Filter By Position ID as the source filter.
    • Worker Import Limit: Specify the maximum number of worker records that can be imported from ADP Workforce Now during a single run of the integration.
      Note: This option will appear when you select Import Limit as the source filter.
    • Employee Creation Condition in BSwift: Select the create condition for creating a worker in BSwift. The available options are Create All Workers, Create Active Users, and Create Full-Time Workers.
      Note: After you select the source filter and supply its value, any worker retrieved must satisfy this creation condition for the integration to produce the desired result.
    • Employee Create Limit: Specify the maximum number of worker records that can be imported into BSwift during a single run of the integration.

If Enable Worker Filtering is not checked, the following options become visible for the full integration sync:

  • Employee Creation Condition in BSwift: Select the create condition for creating workers in BSwift. The available options are Create All Workers, Create Active Users, and Create Full-Time Workers.
  • Employee Create Limit: Specify the maximum number of worker records that can be imported into BSwift during a single run of the integration.

Configure the following settings in the CSV Column Header Settings section:

  • CSV Column Header Names: Enter the worker attributes to serve as CSV column headers in the report.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.
  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details or Custom option.

Report Settings

Configure the following report settings:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.
  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details or Custom option.

Log Insights Settings

Configure the following log insight settings:

  • Select the Log Insights Detail Level: Select the desired log insights detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.
  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details or Custom option.
  • Dashboard Name for Log Insights: Enter the name you want to use for the Log Insights dashboard.
    Note: Modifying the value will create a new dashboard. Logs will no longer be sent to the old one (if it exists).

Notification Settings

Configure the following settings in the Email Report Section:

  • Send Integration Report via Email: Select this checkbox to receive an email notification with a report for all integration runs. When this checkbox is selected, the following settings will be displayed:
    • Select the Notification Report Detail Level: Select the desired notification report level from the dropdown menu. The following are the available options:
      • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
      • Full Details: All attribute details are displayed.
      • Custom: Includes or excludes selected attribute details. Selecting this option will display the Attribute Filter drop-down menu with the following options:
        • Included Attributes: Select the attributes to be included in the report.
        • Excluded Attributes: Select the attributes to be excluded from the report.
  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details or Custom option.
  • Notify Aquera Support on Integration Failure: Select this checkbox to send an alert to Aquera Support if the integration fails.

Configure the following settings in the Admin Email Section:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Only to This Domain: Restrict sending emails only to addresses at the specified domain in this field. For example, aquera.com. The sender is only allowed to send emails to email addresses whose domain matches the aquera.com domain.
