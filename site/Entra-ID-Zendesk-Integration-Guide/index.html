Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any users created, updated, or terminated in Entra ID are automatically updated in Zendesk based on the defined schedule.

To identify user records across both systems for user sync from Entra ID to Zendesk, the integration uses the attribute configured in the Select Source Attribute for Matching the User parameter in Entra ID and matches it to the attribute configured in the Select Target Attribute for Matching the User parameter in Zendesk as a correlation attribute.

Key Functions

The Entra ID / Zendesk integration supports the following primary capabilities:

  • Synchronizes user records from Entra ID to Zendesk.
  • Creates new user records in Zendesk when they are added in Entra ID.
  • Updates existing Zendesk user records when their information changes in Entra ID.
  • Suspend existing Zendesk user records when they are terminated in Entra ID.
  • Users will be assigned to the corresponding organizations in Zendesk based on their departments in Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Zendesk:
    • Entra ID: Ensure that you configure the Entra ID connector in the Aquera Admin portal. Refer to the Entra ID Configuration Guide for configuration details.
    • Zendesk: Ensure that you configure the Zendesk connector in the Aquera Admin portal. Refer to the Zendesk Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to Zendesk.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Entra ID / Zendesk in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration page is displayed to configure the integration.

User Configuration

This section allows you to configure user synchronization settings that determine which user records are created, updated, or suspended in Zendesk based on Entra ID settings.

Select Source Application

This section allows you to identify the source application used for the integration.

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Select Entra ID Application Instance*: Select the Entra ID application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

User Configuration

Configure the following settings to define which user records must be syncronized into Zendesk from Entra ID:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Create Condition*: Select a condition from the dropdown menu. Based on your selection, only user records that satisfy the selected condition are created in Zendesk. The following are the available options:
    • All Users: Select this option if you want to create all user records in Zendesk from Entra ID.
    • All Active Users: Select this option if you want to create only active user records in Zendesk from Entra ID.
    • Custom: Select this option if you want to define a condition to identify user records to be created in Zendesk. When selected, the following parameter is displayed:
      • Custom Create Condition*: Enter a JavaScript expression that defines the condition for creating user records from Entra ID to Zendesk.
  • Update Condition*: Select a condition from the dropdown menu. Based on your selection, only user records that satisfy the selected condition are updated in Zendesk. The following are the available options:
    • All Users: Select this option if you want to update all user records in Zendesk from Entra ID.
    • All Active Users: Select this option if you want to update only active user records in Zendesk from Entra ID.
    • Custom: Select this option if you want to define a condition to identify user records to be updated in Zendesk. When selected, the following parameter is displayed:
      • Custom Update Condition*: Enter a JavaScript expression that defines the condition for updating user records from Entra ID to Zendesk.
  • Suspend Condition*: Select a condition from the dropdown menu. Based on your selection, only user records that satisfy the selected condition are suspended in Zendesk. The following are the available options:
    • All Deactivated Users: Select this option if you want to deactivate all user records in Zendesk from Entra ID.
    • Custom: Select this option if you want to define a condition to identify user records to be suspended in Zendesk. When selected, the following parameter is displayed:
      • Custom Suspend Condition*: Enter a JavaScript expression that defines the condition for suspending user records from Entra ID to Zendesk.
  • Delete Users from Zendesk on Termination: Enable this toggle button to delete user records from Zendesk when the corresponding user records are deactivated in Entra ID.
  • Unassign Organization Membership For User On Termination: Enable this toggle button to automatically unassign the user's organization memberships in Zendesk when the corresponding users are terminated in Entra ID.

User Management

This section enables you to define the conditions that determine which users from Entra ID are synchronized to Zendesk.

Match Users Between Entra ID and Zendesk

This section enables you to define the user correlation settings between Entra ID and Zendesk.

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Customize How Users Are Matched Across Systems: Enable this toggle button if you want to define which user record in Entra ID should be correlated with a user record in Zendesk. When enabled, the following parameters are displayed:
    • Select How Users Are Matched Between Entra ID and Zendesk*: Select a match type from the dropdown menu that specifies the match condition. Currently, only Correlate Users Using Custom Attributes in Source and Target Applications is supported.
    • Select Source Attribute for Matching the User*: Select the Entra ID attribute to distinctly find the associated Zendesk user record. For example, select employeeId.
    • Select Target Attribute for Matching the User*: Select the Zendesk attribute to distinctly find the associated Entra ID user record. For example, select email.

Safeguard Limits

Configure the following parameters to define the safeguard limits for the integration: 

  • Enable Safeguard Limit: Enable this toggle button to define limits to restrict the number of provisioning operations performed by the integration. When enabled, the following fields are available for further configuration:
    • Safeguard Limit Type: Select whether the safeguard limits are defined as a number or a percentage. Currently, only a number value is supported.
    • Apply Safeguard Scope: Specify whether a single safeguard limit has to be applied across all operations or separate limits need to be configured for each operation type.
      • Common For All: Select this option to apply the safeguard limit across all operations. When selected, the following field is displayed for the further configuration:
        • Safeguard limit for all operations: Enter a value to define the safeguard limit for all operations. After the limit is reached, no create, update, or terminate operations will be executed.
      • Per operation type: Select this option to define and apply different safeguard limits for each operation type. When selected, the following fields are displayed for further configuration:
        • Enable create operation limit: Enable this toggle button to apply a safeguard limit for user creations.
          • Set create limit: Enter the maximum number of creations that are allowed to be performed in a single integration run. After the limit is reached, create operations will no longer be executed.
        • Enable update operation limit: Enable this toggle button to apply a safeguard limit for user updates.
          •  Set Update limit: Enter the maximum number of updates that are allowed to be performed in a single integration run. After the limit is reached, update operations will no longer be executed.
        • Enable termination operation limit: Enable this toggle button to apply a safeguard limit for user terminations.
          • Set Terminate limit: Enter the maximum number of terminations that are allowed to be performed in a single integration run. After the limit is reached, termination operations will no longer be executed.

Execution

This section enables you to customize the execution settings for the integration. It has the following sub-section:

Advance Settings

Configure the following advance settings:

  • Report Only Mode: Enable this toggle button to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Debug Mode: Enable this toggle button to generate logs outlining data changes that would occur for debugging.
  • Enable Source Filter: Enable this toggle button to filter the source records. When enabled, the Source Filter section is displayed.
  • Enable Logging: Enable this toggle button to generate logs outlining data changes. When enabled, the Logging Settings section is displayed for further configuration in the Operational Settings section under the Monitoring tab.
  • Enable Email Notifications: Enable this toggle button to turn on integration-related notifications. When enabled, the Notification section is displayed for further configuration under the Monitoring tab.
  • Enable Full Sync Mode: Enable this toggle button to execute the integration in the full sync mode. If this toggle button is not enabled, the integration will be executed in the incremental import mode after the first execution. By default, the first integration execution always runs in full sync mode.
  • Enable Log Insights: Enable this toggle button to visualize the integration-related metrics. When enabled, the Insights Settings section is displayed for further configuration in the Operational Settings section under the Monitoring tab.
  • Enable Reporting: Enable this toggle button to generate the integration report. When enabled, the Report Settings section is displayed for further configuration in the Operational Settings section under the Monitoring tab.
Source Filter

This section allows you to configure filters to limit which records are processed by the integration. Only records that match the defined filter criteria are included.

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Select The Filter Type: Select an Entra ID attribute from the dropdown menu by which the user records must be filtered. The available options are User Principal Name, Employee ID, and Custom.
  • Filter Value: Enter the value to filter Entra ID user records based on the attribute selected in the above parameter.
  • Custom Source Filter Expression*: Enter a JavaScript expression the define which user records must be filtered in Entra ID. This parameter is displayed only when Custom option is selected in the Select The Filter Type parameter.
