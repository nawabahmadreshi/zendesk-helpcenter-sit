Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing complex IT environments and business applications.

This integration reads employee records from Microsoft Entra ID and uses that data to create and manage employee profiles in ABC Fitness. It retrieves users from Entra ID based on configured criteria. 

The integration checks whether a corresponding employee already exists in ABC Fitness using configured correlation attributes, such as email address or external identifiers. If no matching employee is found, the integration creates a new employee record in ABC Fitness. It maps and populates key fields, including first name, last name, email, department, club association, and any additional mapped attributes. If an employee already exists, the integration avoids duplicate creation by validating against cached records and correlation logic. The department values are dynamically resolved by mapping Entra ID attributes to corresponding department identifiers in ABC Fitness, ensuring accurate alignment with the target system’s structure.

When users no longer meet the configured criteria, such as exclusion rules or filtering conditions, the integration skips processing for those users.

Key Functions

The Entra ID to ABC Fitness integration provides the following functions:

  • Create employee records in ABC Fitness using Entra ID data: When a user is retrieved from Entra ID and meets the configured criteria (such as sync mode, filters, and exclusion rules), the integration automatically creates a corresponding employee record in ABC Fitness. The new employee is populated with key details, including name, email, department, club assignment, and other mapped attributes, based on the configured transformation logic.

  • Prevent duplicate employee creation using correlation attributes: Before creating a new employee, the integration checks existing ABC Fitness records against configured correlation attributes, such as email addresses or unique identifiers. If a match is found, the integration skips creation to avoid duplicates, ensuring data integrity across systems.

  • Apply filtering and exclusion rules to control user processing: The integration evaluates users against configured filters and exclusion rules, such as job titles or attribute-based conditions. Only users who meet these criteria are processed, allowing organizations to control which records are created in ABC Fitness.

  • Map and transform Entra ID attributes to the ABC Fitness schema: User attributes from Entra ID are mapped to the ABC Fitness employee schema using configurable mappings. 

  • Resolve departments dynamically in ABC Fitness: Entra ID department values are dynamically mapped to department identifiers in ABC Fitness. The integration retrieves department data, matches department names and assigns the correct department ID to each employee record.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register. Contact your Aquera account manager to obtain access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/

  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following connectors:

    • Microsoft 365 Entra ID: Add and configure the Microsoft Entra ID connector. This connector reads user data from Entra ID to create employees in ABC Fitness. Refer to the Microsoft 365 Entra ID Configuration Guide for details on app registration, Graph API permissions, and completing tenant consent.

    • ABC Fitness: Add and configure the ABC Fitness connector with the required API credentials. This connector is used to create and manage employee records. Refer to the ABC Fitness Configuration Guide for authentication and setup details.

Integration Setup

This section outlines the configuration required to integrate Entra ID with ABC Fitness in the Aquera Admin Portal.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal
  2. Go to Integrations
  3. Click Create Integration
  4. Search for Entra ID / ABC Fitness in the search box next to the Explore Templates section. 
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and description for the integration.
  7. Select connector instances:
    • Microsoft Entra ID
    • ABC Fitness
  8. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following features based on your requirements:

  • Enable Employee Lifecycle Management: Enables lifecycle-based processing of users from Entra ID, such as handling new hires and updates. For example, when enabled, new users from Entra ID are automatically processed for employee creation in ABC Fitness.

General Settings

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them. For example, when enabled, the integration shows which users would be created but does not make any changes.

  • Enable Logging: Select to generate logs outlining data changes. For example, when enabled, you can review logs to see how each user was processed.

  • Enable Notifications: Select to enable the integration-related notifications. For example, you receive alerts when the integration completes or fails.

  • Enable Report Generation: Select to generate the integration report. For example, a CSV report is created showing created, skipped, and failed records.

  • Enable Log Insights: Select to visualize the integration-related metrics. For example, you can view trends such as the number of users processed over time.

  • Run Full Sync Days Interval*: Defines how often a full sync is executed. For example, if set to 7, a full sync runs every 7 days, while other runs may be incremental.

Manage Employees

This section defines how users from Entra ID are identified, filtered, and created as employees in ABC Fitness.

Correlation Settings

These settings determine how users are matched between Entra ID and ABC Fitness to prevent duplicates:

  • Source Correlation Attribute: Defines the attribute from Entra ID used to uniquely identify users. For example, onPremisesExtensionAttributes.extensionAttribute12 is used as the unique identifier from Entra ID.

  • Target Correlation Attribute: Defines the corresponding attribute in ABC Fitness used for matching. For example, barcode is used to match existing employee records in ABC Fitness.

Provisioning Settings

These settings control which users are eligible for employee creation:

  • Create Employee Condition: Specifies which users are eligible for employee creation. For example, when set to Active Only, only active Entra ID users are processed.

Provisioning Threshold

These settings control limits to manage processing and prevent excessive operations:

  • Create Limit: Defines the maximum number of employees that can be created in a single run. For example, 0 disables creation, while 100 allows up to 100 employee creations.

  • Import Limit: Defines the maximum number of users to process from Entra ID. For example, a limit of 500 restricts processing to 500 users per run.

All Club Numbers

This setting defines where employees will be created in ABC Fitness:

  • Club Numbers: Specifies the list of club numbers where employees will be created. For example, entering 101, 102, and 103 assigns employees to those clubs.

Data Protection Settings

This section defines which fields are treated as sensitive and handled securely in logs, reports, and processing.

  • PII Fields: Specifies personally identifiable information that should be masked or protected. For example, fields such as email or phoneNumber can be redacted to prevent exposure in logs.

  • GUID Fields: Defines fields that contain unique identifiers. For example, fields like id or objectId can be marked to ensure proper handling and tracking.

  • Employee ID Fields: Identifies fields used as employee identifiers. For example, fields such as employeeId or barcode can be specified to ensure consistent identification across systems.
