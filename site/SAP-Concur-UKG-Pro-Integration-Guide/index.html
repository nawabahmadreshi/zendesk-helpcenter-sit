SAP Concur / UKG Pro Integration Guide

Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employee data between UKG Pro and SAP Concur.

To identify records across both systems, the integration uses either the employee ID or email ID (based on selection) from UKG Pro, and matches it to the employee number or username (based on selection) in the SAP Concur as a correlation attribute.

Key Functions

This integration automates identity lifecycle management between UKG Pro (an HR/Payroll system) and SAP Concur (an expense management system) with the following primary capabilities:

  • Creates new users in SAP Concur when they are added in UKG Pro.
  • Updates existing SAP Concur users when their information changes in UKG Pro.
  • Deactivates SAP Concur users when employees are terminated in UKG Pro.
  • Creates and deletes users based on a specific date and time.
  • Assigns appropriate roles in SAP Concur based on job titles from UKG Pro.
  • Updates employee expense records from SAP Concur to UKG Pro (writeback).
  • Supports testing mode for validation.
  • Supports incremental synchronization to process only changed records.
  • Sends email notifications for various events, such as user creations, updates, and terminations.

The following are the prerequisites:

    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • UKG Pro: Refer to the UKG Pro Configuration Guide for configuration details.

      • Client ID
      • Client Secret
      • Service Account Username
      • Service Account Password
      • Service Account Customer API Key
    • SAP Concur: Refer to the SAP Concur Configuration Guide for configuration details.
      Note: Only the following tenant attributes must be configured during connector setup:
      • Webservices Company UUID
      • Webservices Request Token
      • Auto assign ExpenseApprovers role to the expense approver of the user
      • Auto assign RequestApprovers role to the request approver of the user

Integration Setup

This section details the configuration and mapping process for integrating UKG Pro with SAP Concur. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select SAP Concur / UKG Pro.
  5. Enter a name and a description for the integration.
  6. Select your UKG Pro and SAP Concur connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Features

Enable the following feature based on your requirements:

  • Enable Employee Lifecycle Management: Select this checkbox to synchronize employees' records from UKG Pro to SAP Concur. When enabled, the Manage Employees section is displayed to configure further integration settings.
  • Enable Expense Management: Select this checkbox to synchronize the expense records from SAP Concur to UKG Pro. When enabled, the Manage Expense section is displayed to configure further integration settings.
  • Enable Entitlement Management: Select this checkbox to synchronize the entitlement assignments (roles) from UKG Pro to SAP Concur. When enabled, the Entitlements tab is displayed for further configuration.

Execution Settings

Configure the following settings:

  • Enable Debug Mode: Select this checkbox to execute the integration with a test user for debugging purposes. When enabled, the Debug Settings fields are displayed within the Operational Settings section for further configuration.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings fields are displayed within the Operational Settings section for further configuration.
  • Enable Reporting: Select this checkbox to generate the integration report. When enabled, the Report Settings fields are displayed within the Operational Settings section for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings fields are displayed within the Operational Settings section for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Center section is displayed for further configuration.

Manage Employees

The Manage Employees section enables you to customize the integration settings for employee lifecycle management according to your specific use case. It synchronizes employee data from UKG Pro to SAP Concur.
Note: This section appears only if the Enable Employee Lifecycle Management setting is enabled in the Parameters section.

It has the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Match Expression: This section enables you to choose the correlation attributes in the source and target applications used to identify matching records. It has the following settings:
    • Source Correlation Attribute: Select the correlation attribute available in UKG Pro to distinctly find the associated SAP Concur record. The available options are EmployeeID and Email.
    • Target Correlation Attribute: Select the correlation attribute available in SAP Concur to distinctly find the associated UKG Pro record. The available options are EmployeeNumber and UserName.
  • Filter Employees: This section enables you to filter employees by selecting the statuses of employees that would be considered active or inactive. It has the following settings:
    • Statuses to be considered Active*: Select one or more UKG Pro Employee statuses that must be considered active.
    • Statuses to be considered Inactive*: Select one or more UKG Pro Employee statuses that must be considered inactive.
  • Create Condition: This section enables you to define the create condition for the integration. Based on this configuration, the employees from UKG Pro will be created in SAP Concur. It has the following settings:
    • Employees to be Created in SAP Concur: Select the UKG Pro employees whose records must be created in SAP Concur. It has the following options:
      • All Active Employees: Select this if you want all active employees to be created in SAP Concur.
      • All Active Full-Time Employees: Select this if you want only active full-time employees to be created in SAP Concur.
      • All Active Part-Time Employees: Select this if you want only active part-time employees to be created in SAP Concur.
      • Custom: Select this to configure an expression to identify employees to be created in SAP Concur. When selected, the following field is displayed:
        • Custom Expression: Provide an expression to identify employees to be created in SAP Concur.
    • Map Manager: Select this checkbox to include employee's manager information during creation in SAP Concur. If this checkbox is not selected, the manager data will not be synchronized from UKG Pro to SAP Concur during creation.
      Note: Ensure that the Auto assign ExpenseApprovers role to the expense approver of the user tenant attribute is enabled in your SAP Concur application connector.
    • Enable Time-Based User Creation: Select this checkbox to create users in SAP Concur based on their timezone. When enabled, user accounts are created according to configured time rules instead of being created immediately.
      When you select this checkbox, the following section is displayed for further configuration:
      • Time-Based Onboarding: This section enables you to configure the settings for time-based onboarding. It has the following parameters:
        • UKG Pro Attribute for Onboarding Date: Select an attribute to determine the onboarding date. For example, when you select Original Hire Date in this parameter, the Original Hire Date will be used to calculate the onboarding date.

        • Onboarding Date Offset: Select the Onboarding date offset. The following options are available:

          • Offset Before Onboarding Date: Select this to set the onboarding date earlier by a specified number of days.
          • Offset After Onboarding Date: Select this to delay the onboarding date by a specified number of days.
          • No Offset: Select this if you do not want to apply any offset to the onboarding date.

          Note: When you select either Offset Before Onboarding Date or Offset After Onboarding Date, the following parameters are displayed for configuration:

          • Days of Offset before Onboarding: Enter the number of days for offset before onboarding. For example, if you enter 5 in this field, the user will be onboarded 5 days before the specified date.
            Note: This field is available only when you select Offset Before Onboarding Date in the Onboarding Date Offset field.
          • Days of Offset after Onboarding: Enter the number of days for offset after onboarding. For example, if you enter 5 in this field, the user will be onboarded 5 days after the specified date.
            Note: This field is available only when you select Offset After Onboarding Date in the Onboarding Date Offset field.
          • Timezone for Onboarding: Select the timezone to onboard employees. The following are the available options:
            • UTC: Select this to onboard employees using Coordinated Universal Time (UTC), regardless of the employee’s local time.
            • Based on Source Timezone Attribute: Select this to onboard employees based on their work location’s timezone. When selected, the following field is displayed:
              • Source Attribute for Timezone: Select an attribute to determine the employee's timezone. For example, Work Address.
          • Onboarding Time: Enter the time for the onboarding in 24-hour format (HH:MM). For example, 03:40
  • Update Condition: This section enables you to define the update condition for the integration. Based on this configuration, the employees from UKG Pro will be updated in SAP Concur. It has the following settings:
    • Employees to be Updated in SAP Concur: Select the UKG Pro employees whose records must be updated in SAP Concur. It has the following options:
      • All Active Employees: Select this if you want all active employees to be updated in SAP Concur.
      • All Active Full-Time Employees: Select this if you want only active full-time employees to be updated in SAP Concur.
      • All Active Part-Time Employees: Select this if you want only active part-time employees to be updated in SAP Concur.
      • Custom: Select this to configure an expression to identify employees to be updated in SAP Concur. When selected, the following field is displayed:
        • Custom Expression: Provide an expression to identify employees to be updated in SAP Concur.
    • Attributes to be Updated: Select the UKG Pro attributes that must be considered for updates in SAP Concur. For example, if Name is selected in this field, and an employee’s name has been updated in UKG Pro, the updated value will be reflected in SAP Concur after the integration execution. However, if Name is not selected in this field, the updated value will not be updated in SAP Concur.
  • Delete Condition: This section enables you to define the delete condition for the integration. Based on this configuration, the UKG Pro employees will be deleted from SAP Concur. It has the following settings:
    • Employees to be Deleted in SAP Concur: Select the UKG Pro employees whose records must be deleted from SAP Concur. It has the following options:
      • All Inactive Employees: Select this checkbox if you want all inactive employees in UKG Pro to be deleted from SAP Concur.
      • Custom: Select this to configure an expression to identify employees to be deleted from SAP Concur. When selected, the following field is displayed:
        • Custom Expression: Provide an expression to identify employees to be deleted from SAP Concur.
    • Enable Time-Based User Deletion: Select this checkbox to delete users from SAP Concur based on their timezone. When enabled, user accounts are deleted according to configured time rules instead of being deleted immediately.
      When you select this checkbox, the following section is displayed for further configuration:
      • Time-Based Offboarding: This section enables you to configure the settings for time-based offboarding. It has the following parameters:
        • UKG Pro Attribute for Offboarding Date: Select the attribute to determine the offboarding date. For example, when you select Termination Date in this parameter, the value of the Termination Date attribute will be used to calculate the offboarding date.

        • Offboarding Date Offset: Select the Offboarding date offset. The following are the available options:

          • No Offset: Select this if you do not want to apply any offset to the offboarding date.
          • Offset Before Offboarding Date: Select this to set the offboarding date earlier by a specified number of days.
          • Offset After Offboarding Date: Select this to delay the offboarding date by a specified number of days.

          Note: When you select either Offset Before Offboarding Date or Offset After Offboarding Date, the following parameters are displayed for configuration:

          • Days of Offset before Offboarding: Enter the number of days for offset before offboarding. For example, if you enter 5 in this field, the user will be deleted 5 days before the specified date.
            Note: This field is available only when you select Offset Before Offboarding Date in the Offboarding Date Offset field.
          • Days of Offset after Offboarding: Enter the number of days for offset after offboarding. For example, if you enter 5 in this field, the user will be deleted 5 days after the specified date.
            Note: This field is available only when you select Offset After Offboarding Date in the Offboarding Date Offset field.
          • Timezone for Offboarding: Select the timezone to delete employees. The following are the available options:
            • UTC: Select this to delete employees using Coordinated Universal Time (UTC), regardless of the employee’s local time.
            • Based on Source Timezone Attribute: Select this to delete employees based on their work location’s timezone. When selected, the following field is displayed:
              • Source Attribute for Timezone: Select the attribute to determine the employee's timezone.
          • Offboarding Time: Enter the time for the offboarding in 24-hour format (HH:MM). For example, 03:40.

Entitlements

The Entitlements tab is displayed only when it is enabled in the Parameters section.

It allows you to configure how entitlement assignments (roles) are managed through this integration.

In this tab, you can define conditions that control how entitlements are assigned to employees.

Perform the following procedure to create role assignment rules:

  1. Click Add New Rule to create a new assignment rule. The following fields are displayed:
  2. Click Choose Attribute to view the list of UKG Pro employee attributes.
  3. Search and select the attribute using which you want to set a condition. For example, Job Title.
  4. Select the attribute value to define the rule. Only those users who meet this condition will satisfy this assignment rule. If multiple values are provided, an OR condition is applied among those values. For example, in the following sample condition, an employee's job title must be either Benefits Administrator or Benefits Manager for them to satisfy this assignment rule.
  5. In the Entitlement Assignment column, select the roles to which the employees who satisfy the specified condition must be assigned. The list shows all the roles from the target SAP Concur application.
  6. Click Add Row to add another condition and assignment for the same UKG Pro attribute.

    Notes:
    • You can add any number of rows within a rule.
    • You can delete a row by using the delete icon at the right end of the row.
  7. [Optional] If you want to write a JavaScript expression to set the condition, click Switch to Expression.
    Note: When you switch to the Expression mode or to the Basic Condition mode, all the configured conditions of the rule will be cleared.
    1. Select one of the following based on your requirement:
      • For Conditions Only: Select this if you want to write a JavaScript expression only for the conditions.
         
      • For Conditions and Assignments: Select this if you want to write expression for both, conditions and assignments.
    2. Click Continue to configure the expression.

      Note: Expressions can use all attributes in the source and destination applications. The configured expressions must always be evaluated to true or false.
      SOURCE must be used in an expression to represent an attribute in the source application and DEST to represent an attribute in the destination application.
      For example, SOURCE.EmployeeNumber, SOURCE.HomeOrganization.ShortName, and DEST.EmployeeNumber.
  8. Click Add New Rule to add another assignment rule, as explained above.
    Note: You can add any number of assignment rules. However, you can see only 5 rules on a single page. If you add more than 5 rules, navigation options will be available at the bottom of the page to navigate to other pages for configuring or viewing the remaining defined rules.

Note: The integrations assigns roles to the user but doesn't remove the existing assigned roles. You must unassign the required roles manualy in SAP Concur.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
