Active Directory (AD) to HaloITSM

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the integration of data across resource types.

Aquera's integrations facilitate the automatic integration, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide details how to set up AD to HaloITSM integration for the following use case:

When a new user is created or updated in Active Directory (AD), this integration will generate a service ticket in HaloITSM for logging purposes. The service ticket will include the Active Directory login details of the user such as Display Name, UPN, and phone number. If there are any issues with user creation or generating the service ticket, this integration will create an incident ticket in HaloITSM to help track and resolve the issue.

This AD to HaloITSM integration will be used in the integration hook configurations of the user integration from a source application (for example, ADP Workforce Now) to Active Directory. This AD to HaloITSM integration will be executed when a new user is created or an existing user is updated in Active Directory as part of the user integration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Connectors in the Aquera Admin Portal: You need to configure the following connectors in the Aquera Admin portal for the integration:
    • Active Directory: Refer to the Active Directory (AD) Configuration Guide for details.
    • HaloITSM: Refer to the HaloITSM Configuration Guide or details.

Integration Setup

The AD to HaloITSM integration creates a service ticket in HaloITSM to log Active Directory user details for every new user creation or update. If there is an issue with user creation or service ticket creation, this integration will create an incident ticket in HaloITSM.

Create an Integration

Perform the following steps to create an integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select AD to HaloITSM.
  5. Enter a name and a description for the integration.
  6. Select your AD and HaloITSM connector instances.
  7. Click Continue. A new page is displayed to configure the integration.
  8. Click Create to create the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

The Configuration section lets you update application instances, preferences, triggers, and tables for the integration.

Preferences

Preferences are variables used as inputs for the integration script. You can update the preference values based on your use case.
The following are the preferences for the AD to HaloITSM integration:

  • Service request ticket type name for user onboarding: Enter the name of the HaloITSM ticket type that must be used to log and manage service requests for new user creations in Active Directory. For example, Useronboard
  • Incident ticket type name: Enter the name of the HaloITSM ticket type that must be used to log and manage incidents in HaloITSM. For example, Incident
  • Client ID: Enter the unique HaloITSM client ID assigned to your organization.
  • Site ID: Enter the unique ID of the site with which the tickets created by the integration will be associated.
  • Send Email Notifications?: Select this checkbox to send email notifications for integration runs. The email addresses to which the notifications have to be sent must be set in the Send email notifications to field. If this checkbox is not selected, email notifications will not be sent on integration runs.
  • Incident impact level: Enter the incident impact level based on the severity of the incident (1- High, 2- Medium, 3-Low). For example, 1
  • Incident urgency level: Enter the urgency level that determines how quickly the incident needs to be addressed (1- High, 2- Medium, 3-Low). For example, 2
  • Enable debug?: Select this checkbox to create detailed logs for debugging purposes. If this checkbox is not selected, detailed logs will not be created by the integration.
  • Send email notifications to: Enter the comma-separated list of email addresses to which notifications must be sent on integration runs. This field will be effective only if you have selected the Send Email Notifications? checkbox.
  • Service request ticket type name for user updates: Enter the name of the HaloITSM ticket type that must be used to log and manage service requests for user updates in Active Directory. For example, Userupdate
  • Not Apply Template in the API?: -----------------
  • Service request ticket type name for user offboarding: Enter the name of the HaloITSM ticket type that must be used to log and manage service requests for user deactivations in Active Directory. For example, Useroffboard
  • AD Admin ID: Enter the ID of the Active Directory admin to be used to create incidents in HaloITSM in case the user creations/updates/deactivations fail in Active Directory.
  • HaloITSM Admin ID: Enter the ID of the HaloITSM admin to be used to create incidents in HaloITSM in case the ticket creations fail in HaloITSM.
  • Generate CSV report: Select this checkbox to generate a CSV report of the integration runs. The report will have data ------------
  • Enable Dashboard Logging: Select this checkbox to create create integration activity logs for detail analysis on the Dashboard. You can access the detail activity logs for analyses from the Search Dashboards module. If this checkbox is not selected, detail activity logs cannot be accessed for this integration from the Search Dashboards module.
  • Report-only mode: Select this checkbox to only generate a report that shows the HaloITSM tickets that would be created, without actually making those changes in HaloITSM. Only when the checkbox is not selected, the actual changes are done by the integration in HaloITSM.

Run Integration

When the integration runs, a service ticket is created in HaloITSM for each new user creation or user update to log Active Directory user details. An incident ticket is created in HaloITSM when there is an issue with user creation or service ticket creation. If the sendMail preference is true, an email notification will be sent to the user's manager for service ticket creation, incident creation, or failures.
