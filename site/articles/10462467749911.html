Overview

The Greenhouse Recruiting application provides the best-in-class hiring software solution to find and hire candidates who will deliver exactly what you need for each role.

This guide helps you to create the Greenhouse Recruiting connector on the Aquera platform for provisioning candidates and recruiters. The Greenhouse Recruiting connector leverages the Greenhouse Harvest API to retrieve and modify users in the Greenhouse Recruiting application. 

For further details on the Greenhouse Recruiting application, visit the Greenhouse Recruiting website. 

Supported Features

The Greenhouse Recruiting connector supports the following features:

Resource Import Provisioning
Users
  • Full import
  • Filter users
  • Create a user
  • Update a user
  • Activate or deactivate a user with userType as recruiter
Candidates
  • Full import
  • Incremental import
  • Filter candidates

Available on demand
Employees
  • Full import
  • Incremental import
  • Filter employees
  • Create an employee
  • Update an employee
Offers
  • Full import
  • Incremental import
  • Filter Offers
 Available on demand

Note: The Users resource contains information of the candidates and recruiters in the Greenhouse Recruiting application. The Employees resource contains information of the recruiters in the Greenhouse Recruiting application.

The Greenhouse Recruiting connector also supports various features as per the OpenAPI specification. Refer to the OpenAPI Specification section for further details.

Prerequisites

The following are the prerequisites:

  • Administrator access to the Greenhouse Recruiting instance: You must have the admin credentials for your Greenhouse Recruiting instance.
  • Greenhouse API Key: You must have an API Key for authentication. Refer to the Authentication section for further details.

Connector Configuration

Perform the following steps to configure the Greenhouse Recruiting connector:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Applications and click Add Application.
  3. Search for Greenhouse Recruiting and click the Greenhouse Recruiting icon.
  4. Enter the values for the following fields under the Configuration tab:
    • Basic Details:
      • Display Name: Enter a display name for your application.
      • Description: Enter a description for your application.
      • Source Owner: Enter valid email IDs of the source owners.
      • Subscribe to error notification: Enable the toggle button to send notification emails to source owners for errors in the user data transactions. Click Edit Preference to select the error types for which you want to receive notifications. After you configure your preference, you will receive error notifications only for the selected error types. By default, all error types are selected.
      • Pick your protocol: Select the protocol from the dropdown list based on your requirement. SCIM is the default value.
      • Copy this URL: This field displays the base URL of the connector API endpoint. The URL is populated based on your selected protocol. Copy and paste this URL while provisioning to the identity service provider or performing OpenAPI operations, if supported.
    • Tenant Attributes: Refer to the Aquera Tenant Attributes section for details.
  5. Click Create to create the Greenhouse Recruiting connector.

Refer to the Getting Started with Connectors guide to learn more about connector setup including different tabs of the connector configuration page.

Aquera Tenant Attributes

The following are the Aquera tenant attributes:

Note: All tenant attributes marked with a red asterisk (*) are mandatory.

  • Admin user's UserId*: Enter the user ID of the admin user. Refer to the Obtaining the admin user's user ID section for more details. If you do not provide a value or the provided value is invalid, then a 500 error will be returned by the connector.
  • List of inactive application statuses: Enter a comma-separated list of application statuses that must be considered inactive. The possible values are rejected, hired, converted, and active. The candidate will be deactivated when the status of all their applications matches with any of the specified statuses.
  • PII attributes that need to be retrieved: Enter a comma-separated list of Personal Identifiable Information (PII) attributes that must be retrieved in the response.

      6. Click Save.
  • Exclude attributes: Enter a comma-separated list of attributes that must not be retrieved in the user or candidate response. This tenant attribute is applicable only for Users (only of userType candidate) and Candidates resources.
    Notes:
    • The following attributes cannot be excluded in the Users and Candidates responses:
      • Users: active, userName, userType, name.givenName, name.familyName, name.formatted, candidateId, and recruiterId.
      • Candidates: firstName, lastName, active, and status.
    • You must use [] brackets with or without quotes for sub-attributes. For example: 
      • urn:ietf:params:scim:schemas:ian:2.0:User[applications][answers___question__answer]
      • urn:ietf:params:scim:schemas:ian:2.0:User[“applications”][“rejection_reason__type__name”]
      • urn:ietf:params:scim:schemas:ian:2.0:User[last_activity], urn:ietf:params:scim:schemas:extension:enterprise:2.0:User[organization], photos

      • applications[jobDetails][salary_range]

    For example, if you provide the value recruiter in this field, then the Users resource will contain only candidates and the connector can perform operations on the Candidates resource, but not the Employees resource.
    Similarly, if you provide the value candidate in this field, then the Users resource will contain only recruiters and the connector can perform operations on the Employees resource, but not the Candidates resource.

  • Login Password: Enter the password of the Greenhouse Recruiting login instance. If SSO is enabled, provide the password of the Identity Provider account.
  • Greenhouse Recruiting Login Domain*: Provide the domain name of your Greenhouse Recruiting login instance. For example, if your instance is app.greenhouse.io, the domain name will be app.
  • Update Recruiter Job Title: Select this checkbox to update the job title of users with the recruiter userType. For example, if this checkbox is selected, the user's job title will appear in the response and can be updated. If this checkbox is not selected, the title attribute will not appear in the response.
    Note: You must configure the Username or Email and the Login Password tenant attributes to use this feature.

Obtaining the Admin User ID

Perform the following procedure to obtain the admin user ID:

  1. Log in to the Greenhouse Recruiting instance with your administrator account.
  2. Click the Configure icon at the top right of the page.
  3. Click Users on the left navigation pane.
  4. Click Export to Excel to download and view the user ID of the admin.
  5. The user ID of the admin will be available next to the admin's name in the User ID column.

Authentication

The Greenhouse Recruiting connector supports Bearer Token Authentication. Provide the API Key generated in your Greenhouse Recruiting administrator account as the bearer token for authentication.

Generating the API Key

To create the API key, a user must be granted permission Can manage ALL organization’s API Credentials in the Developer permission section.

Perform the following steps to generate an API key:

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon at the top-right corner of the page.
  3. From the left navigation pane, navigate to Dev Center > API Credentials.
  4. Click Create new API key. The Create new credential pop-up window is displayed.
  5. Provide the following details in the pop-up window:
    • Select Harvest from the API type drop-down list.
    • Select Custom from the Partner drop-down list.
    • Enter a short description for the integration in the Description field.
  6. Click Manage Permissions. The API Key is displayed. This API key must be saved securely. It must be used as the bearer token for authentication.
  7. Click Close.
  8. Under the Manage Permissions section on the next page, expand the Harvest V1 option, and select the following options to perform the required operations using this API key:









  9. Click Save.

Adding Custom Fields

Perform the following procedure to add the custom fields in the Greenhouse Recruiting application:

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon at the top-right corner of the page.
  3. Click Custom Options on the left navigation page. The custom field types supported for your instance are available under the Company custom fields section, and these custom field types will vary for each instance.
  4. Select the required Company custom field type under the Company custom fields section. 
    Note: The connector supports only Candidates, Jobs, and Offers custom field types.
  5. Click Add field to add a custom field.
  6. Provide a field name and select a field type for the custom field, and click Save to create the custom field.


    The custom field name created is displayed under the Immutable field key.
    Notes:
    • Any special characters provided in the Field name is replaced with an underscore.
    • The custom field name shown under the Immutable field key is immutable and this custom field name cannot be modified.

The custom fields for each type will have the following format in the schemas and the resources:

  • Candidates custom field type:
    custom_<Immutable field key>
    Example: custom_university_name
  • Jobs custom field type:
    job_customField_<Immutable field key>
    Example: job_customField_employment_type
  • Offers custom field type:
    offer_customField_<Immutable field key>
    Example: offer_customField_offer_expire_date
     

In the format, the Immutable field key will be replaced with the actual custom field name.

OpenAPI Specification

OpenAPI Specification is an API description format to define the structure and syntax of REST APIs. You can view the OpenAPI specification document in an API tool which defines the elements of an API. The OpenAPI operations can be used in Aquera's workflows and integrations using scripts.

Perform the following steps to view the OpenAPI specification in an API tool:

  1. Copy the OpenAPI URL from Copy this URL field on the connector configuration page of the Aquera Admin portal.
  2. Add /document at the end of the copied URL.
  3. Run the URL in an API tool, such as Postman using the GET method with the correct authentication credentials. The tool will display the OpenAPI specification document with all the supported OpenAPI operations. You can view the parameters and request body in the specification document.

You can also view the OpenAPI specification document by clicking the View OpenAPI Document button.

Performing OpenAPI Operation

Using an API Tool

You can perform OpenAPI operations using an OpenAPI tool as follows:

  1. Copy the OpenAPI URL from Copy this URL field on the connector configuration page. 
  2. Add /invoke at the end of the copied URL.
  3. Paste the URL in an API tool.
  4. Provide the required credentials for authentication.
  5. Pass the operation ID, parameters, and properties in the request body.
    Note: The parameters/properties vary depending on the type of operation.
  6. Run the URL using the POST method.

The following is the format to be passed in the request body:

{
"operationId": <<Operation ID>>,
"parameters": {<<parameter name>>: <<parameter value>>},
"requestBody": {<<property name>>: <<property value>>}
}

Using Workflows

You can perform OpenAPI operations using Workflows. Contact the Aquera support team for configuring Workflows.

Using Scripts

You can perform OpenAPI operations using Scripts. Refer to the Integration Using Scripts guide for details on how to set up an integration using scripts.

OpenAPI operations can be invoked using the following format:

<AppVariableName>.<OpenAPI_MethodName>({parameterName: parameterValue})

Here, <AppVariableName> is the name of the variable that you have configured in the script to refer to the provisioned connector, and the <OpenAPI_MethodName> is the name of the OpenAPI method to invoke the corresponding OpenAPI operation. 

Provide a comma-separated list of parameter names with parameter values enclosed in curly braces.

For further details on how to invoke OpenAPI methods using scripts, refer to the Working With Integration Scripts guide.

Updating the Primary Email ID of the Recruiter User Type

The following is the procedure to update the primary email ID of the user with the Recruiter user type:

  1. Add the email ID of the user that you want to make as the primary email ID using a PUT request. The user will receive a verification email from Greenhouse to confirm the new email address.
  2. The user must click the verification link in the email and enter the password associated with their current primary email ID used to log into Greenhouse Recruiting.
  3. On the connector configuration page, enter values for the Username or Email, Login Password, and Greenhouse Recruiting Login Domain tenant attributes. These are required to update the primary email.
  4. Send the PUT request to set the new email ID as primary. You can also update the Primary email through the Greenhouse Recruiting UI.

Permissions for Managing a Job

A user must have one of the following permissions to create, update, or delete a job:

  • Job Admin: This user can view and manage assigned jobs. To set this permission for a user, perform the following steps:

    2. Click Configure icon at the top-right corner of the page.
    3. Click Users on the left menu.
    4. Select a user to grant the Job Admin permission.
    5. Under the Permissions section, click Reassign permission.
    6. Click Assign for Job Admin.
    7. Under User-Specific Permissions, select the checkbox Create new jobs. The default role Job Admin: Private will be set.
    8. Click Save permissions.

       
  • Site Admin: This user can manage all jobs and user permissions. To set this permission for a user, perform the following steps:
    1. Log in to the Greenhouse Recruiting application.
    2. Click Configure icon at the top-right corner of the page.
    3. Click Users on the left menu.
    4. Select a user to grant the Site Admin permission.
    5. Under the Permissions section, click Reassign permission.
    6. Click Assign for Site Admin.
    7. Under User-Specific Permissions, select the following checkboxes:
    8. Click Save permissions.

Permissions for Creating a Job

Template Jobs

To create a job using a template job ID, you must set the following permission for the user:

  1. Log in to the Greenhouse Recruiting application.
  2. Click Configure icon at the top-right corner of the page.
  3. Click Users on the left menu.
  4. Select a user to grant the permission.
  5. Under the Permissions section, click Reassign permission.
  6. Click Assign for Job Admin.
  7. Under Job-Based Permissions, click Add. A new window opens.
  8. Click the pencil icon for the template job the user needs access.
  9. Select the permission Job Admin: Private or Job Admin: Standard.
  10. Click Save.

Non-Template Jobs

To create a job without using a template job ID, you must set the following permission for the user:

  1. Log in to the Greenhouse Recruiting application.
  2. Click Configure icon at the top-right corner of the page.
  3. Click Users on the left menu.
  4. Select a user to grant the permission.
  5. Under the Permissions section, click Reassign permission.
  6. Click Assign for Job Admin.
  7. Under Job-Based Permissions, click Add. A new window opens.
  8. Click the pencil icon for All Future Jobs.
  9. Select the permission Job Admin: Private or Job Admin: Standard.
  10. Click Save.
