Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The ADP Workforce Now to Microsoft Entra ID User Provisioning integration automates the synchronization of worker data from ADP Workforce Now to Microsoft Entra ID. This integration ensures that user accounts in Entra ID are created, updated, and deactivated automatically based on authoritative HR data from ADP.

The integration reads worker records from ADP Workforce Now, applies configured eligibility rules and matches each worker to an existing Entra ID user using a defined correlation attribute. If no matching user is found, a new account is created. If a user already exists, the integration updates the account when changes are detected, such as job title, department, manager, or contact information.

When a worker becomes inactive or no longer meets the configured conditions, the integration updates the user’s account status according to the defined lifecycle rules. This helps organizations maintain accurate identity data, enforce access policies, and reduce manual administrative effort.

Key Features 

  • Creates new Microsoft Entra ID user accounts for eligible workers from ADP Workforce Now, including automatic temporary password generation.
  • Updates existing Entra ID users by detecting and applying only ADP attribute changes.
  • Deactivates Entra ID accounts based on configurable lifecycle rules, with deactivation prioritized over updates when both conditions apply.
  • Supports flexible user correlation using configurable attributes such as Associate ID, Position ID, or Work Email.
  • Applies configurable eligibility rules for create, update, and deactivate actions, including custom filter logic and date-based conditions.
  • Supports date offsets for onboarding and offboarding to defer provisioning or deactivation until the configured effective date.
  • Manages Microsoft Entra ID group memberships based on defined assignment rules.
  • Processes only records that have changed since the last successful run using stored last-modified timestamps.

Integration Setup

This section outlines the configuration required to establish integration between ADP Workforce Now and Microsoft Entra ID using the Microsoft 365 connector in the Aquera Admin Portal.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. In the left navigation panel, select Integrations.
  3. Click Create Integration.
  4. Select ADP Workforce Now Workers to Entra ID Users as the integration template.
  5. Enter a name and description for the integration.
  6. Select the connector instances that the integration will use:
    • ADP Workforce Now
    • Microsoft 365 (Entra ID)
  7. Click Continue.

The Configuration page is displayed to complete the integration setup.

Configuration

The Configuration tab is used to review and customize the behavior of the ADP Workforce Now to Microsoft Entra ID integration.

The following sections are available under the Configuration tab:

Applications

The Applications section displays the configured source and target connectors for the integration.

This section allows you to review, search, and update the selected connector instances.

Parameters

The Parameters section allows you to configure how ADP Workforce Now worker records are processed and synchronized to Microsoft Entra ID.

This section includes settings that control worker eligibility rules, creation and deactivation behavior, incremental synchronization, optional onboarding and offboarding date offsets, and Entra ID group assignment logic.

These settings determine which workers are provisioned, updated, or deactivated in Entra ID and how group memberships are managed based on ADP data.

Integration Features

  • Enable Worker Management: Enables synchronization of ADP worker records to Microsoft Entra ID. For example, when a new employee is added in ADP, the integration creates the corresponding user in Entra ID.
  • Enable Date Offset: Enables processing of workers based on configured hire or termination date offsets. For example, if a hire date offset of -2 days is configured, the user account can be created two days before the official hire date.
  • Enable Entra Group Management: Enables automatic assignment and removal of users from Microsoft Entra ID groups based on configured rules. For example, if a worker belongs to the Sales department in ADP, the integration can automatically add the user to the Sales-Entra group.

General Settings

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.

  • Enable Logging: Select to generate logs outlining data changes.

  • Enable Notifications: Select to enable the integration-related notifications.

  • Enable Report Generation: Select to generate the integration report.

  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

  • Enable Log Insights: Select to visualize the integration-related metrics.

Workforce Now Settings

  • Enable Incremental Sync: Select to process only workers who have been updated since the last successful integration run. For example, if only three employees were updated in ADP, only those three records are synchronized instead of all workers.
  • Workers Eligible for Provisioning in Entra: Select which ADP workers should be created or updated in Microsoft Entra ID. For example, selecting All Active Workers ensures only currently employed workers are provisioned.
    The following are the available options: 
    • All Workers: Select to provision and update all workers from ADP Workforce Now, regardless of employment status. For example, both active and inactive workers will be synchronized to Entra ID.
    • All Active Workers: Select to provision and update only workers whose employment status is active. For example, terminated or retired workers will not be created or updated.
    • All Active Full-Time Workers: Select to provision and update only workers who are both active and classified as full-time in ADP. For example, part-time or contract workers will be excluded.
    • Custom: Select to define a custom rule to determine which workers should be provisioned. For example, you can configure the integration to provision only workers in a specific department or only those with a valid work email address.
      • Custom Upsert Condition: Enter a JavaScript rule to decide which workers should be created or updated in Entra ID. For example, you can set it to create or update users only if they have a valid work email address.
    • Delete Condition Type: Select how the integration should decide which workers to delete from Entra ID. For example, choose Basic to use standard termination rules, or choose Custom to define your own deletion logic.
      The following are the available options: 
      • Basic: Uses predefined worker status rules to determine who should be deleted. For example, workers marked as terminated or retired in ADP can be automatically removed from Entra ID.
      • Custom: Allows you to define your own rule to decide which workers should be deleted. For example, you can delete users only if their status is terminated and their termination date is in the past.
        • Custom Delete Condition: Enter a JavaScript rule to decide which workers should be deleted in Entra ID. For example, you can configure the integration to delete users only when their employment status is Terminated.
  • Workers Eligible for Deletion in Entra: Select which ADP worker statuses trigger account deletion in Microsoft Entra ID. For example, selecting All Terminated Workers ensures users are removed when their employment ends.

Correlation Settings

The Correlation Settings section defines how worker records from ADP Workforce Now are matched with existing user accounts in Microsoft Entra ID. Proper correlation ensures that users are updated instead of duplicated.

  • Workforce Now Correlation Attribute: Select the ADP Workforce Now field that will be used as the matching value when searching for an existing user in Entra ID. For example, if Work Mail is selected and the worker’s email in ADP is barbara.jensen@example.com, that value is used during the lookup in Entra ID.
    The following are the available options: 
    • Associate ID: The worker’s unique Associate ID from ADP is used as the matching value. For example, if the Associate ID in ADP is A10234, the integration compares it with the selected Entra correlation attribute to find a matching user.
    • Position ID: The worker’s Position ID from ADP is used as the matching value. For example, if the Position ID is POS-7788, that value is compared with the selected Entra attribute during user lookup.
    • Work Mail: The worker’s primary work email address in ADP is used as the matching value. For example, barbara.jensen@example.com is used to search for a user in Entra ID with the same value in the selected attribute.
  • Entra Correlation Attribute: Select the Microsoft Entra ID attribute that will be compared against the selected ADP field. The integration searches Entra ID for a user whose selected attribute matches the ADP value.
    The following are the available options: 
    • Employee ID: The Employee ID attribute in Entra ID is used for comparison. For example, if ADP provides Associate ID A10234 and Employee ID is selected in Entra, the integration looks for a user whose Employee ID is A10234.
    • UPN: The User Principal Name (UPN) in Entra ID is used for comparison. For example, if ADP provides barbara.jensen@example.com, the integration searches for a user whose UPN matches that value.
    • Mail: The primary email address in Entra ID is used for comparison. For example, if ADP provides barbara.jensen@example.com, the integration searches for a user whose Mail attribute matches that value.

Onboarding and Offboarding Settings

Configure when user accounts are created and deactivated in Entra ID based on employee start and termination dates.

Onboarding Settings

  • Account Provisioning Offset: Select when a user account should be created in Entra ID relative to the worker’s onboarding (hire) date. For example, if a worker’s start date is July 10 and you select Create 3 Days Before Start Date, the account will be created on July 7.
    • Create On Start Date (No Offset): The user account is created on the worker’s official start date. For example, if the start date is July 10, the account is created on July 10.
    • Create Before Start Date: The user account is created a specified number of days before the start date. For example, if set to 5 days before, a July 10 start date results in account creation on July 5.
    • Create After Start Date: The user account is created a specified number of days after the start date. For example, if set to 2 days after, a July 10 start date results in account creation on July 12.

Offboarding Settings

  • Account Deprovisioning Offset: Select when a user account should be deactivated in Entra ID relative to the worker’s termination date. For example, if a worker’s termination date is August 31 and you select Deactivate 1 Day After Termination, the account will be deactivated on September 1.
    • Deactivate On Termination Date (No Offset): The account is deactivated on the official termination date. For example, if the termination date is August 31, access is removed on August 31.
    • Deactivate After Termination Date: The account is deactivated a specified number of days after the termination date. For example, if set to 3 days after, a termination date of August 31 results in deactivation on September 3.
