Entra ID / FCM Travel Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any users created, updated, or terminated in Entra ID is automatically updated in FCM Travel based on the defined schedule.

To identify user records across both systems, the integration uses the userPrincipalName attribute from Entra ID and matches it to the EmailAddress in FCM Travel as a correlation attribute.

Key Functions

The Entra ID to FCM Travel Integration performs the following functions:

  • User Lifecycle Management: Handles user creation, update, and deletion from Entra ID to FCM Travel.

  • Traveller GUID Writeback: A unique traveller GUID will be created for each new user integrated from Entra ID into FCM Travel, and this traveller GUID will be written back to the user’s Entra ID profile.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for configuration details.
    • FCM Travel: Refer to the FCM Travel Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to FCM Travel.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID / FCM Travel.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and FCM Travel connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable User Lifecycle Management: Select this checkbox to enable user provisioning and perform incremental updates. The Manage Users section will be displayed to configure user management settings only when this checkbox is selected.

    Note: It is mandatory to enable this checkbox to customize the user's lifecycle management.

  • Enable Traveller GUID Writeback: Select this checkbox to enable the traveller GUID writeback. When a new user is created in Entra ID and integrated with FCM Travel, a unique traveller GUID is generated for the user. When this checkbox is selected, the same GUID will be integrated back into the user’s Entra ID profile.

General Settings

Configure the following general settings:

  • Enable Full Sync: Select this checkbox to execute the integration in full sync.
  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur, for debugging purposes. When enabled, the Debug Mode section is displayed for further configuration.

Manage Users

The Manage User section enables you to customize the integration settings for user management according to your specific use case.

Entra ID Settings

Configure the following Entra ID settings:

  • Entra ID Correlation Attribute*: Select the Entra ID attribute to distinctly find the associated FCM Travel user. For example, userPrincipalName.
  • User Provisioning Configuration*: Select one or more operations from the dropdown menu to provision users into FCM Travel from Entra ID. The available options are Create, Update, and Delete.
  • Map Manager: Select this checkbox to create the employee's manager during user creation in FCM Travel if the user's manager does not exist in the FCM Travel. If this checkbox is not selected, the user's manager will not be created in the FCM Travel.

Create User Settings

The Create User Settings section enables you to customize the integration settings for user creation according to your specific use case. It synchronizes user data from Entra ID to FCM Travel.
Note: This section appears only if you select the Create in the User Provisioning Configuration parameter under the Entra ID Settings section.

  • Create User Limit: Specify the maximum number of user records that must be created in the FCM Travel application during a single integration run.
  • Create Condition: Select a condition from the dropdown to specify which Entra ID user records should be created in FCM Travel. It has the following options:
    • All Users: Select this if you want all users to be created in FCM Travel.
    • All Active Users: Select this if you want all active users to be created in FCM Travel.
    • UserType (Member): Select this if you want all users with the user type as Member to be created in FCM Travel.
    • UserType (Guest): Select this if you want all users with the user type as Guest to be created in FCM Travel.
    • Custom: Select this if you want to create a user based on a custom condition.
      • Custom Create Condition: Enter an expression that defines the user creation condition in the FCM Travel.

Debug Mode

Debug Settings

This section enables you to customize the debug mode settings. It has the following settings:

  • Entra ID Filter Attribute: Select the filter to retrieve data of a specific user from Entra and run the integration for debugging purposes. The following are the available options:

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Paycor application, and you want to update a value for that attribute in AD. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integration guide for details.
