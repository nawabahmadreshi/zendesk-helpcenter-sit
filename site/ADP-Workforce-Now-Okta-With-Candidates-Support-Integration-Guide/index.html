Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the process for setting up an integration to fetch pre-hire (candidates) and worker data from ADP Workforce Now and create or update them in Okta XaaS.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Okta:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Okta XaaS: Refer to the Okta Anything-as-a-Source (XaaS) Configuration Guide for details.

Integration Setup

This section outlines the configuration required to integrate ADP Workforce Now and Okta XaaS, enabling the seamless transfer of pre-hire (candidates) and worker data from ADP Workforce Now to Okta XaaS.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / Okta (With Candidates' Support).
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now connector instance.
  7. Select your CSV Template connector instance.
  8. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following tabs are displayed on the Configuration page:

Applications

The Applications tab displays the Applications section, which displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following sections:

  • Report Only: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Source Filter: Enter the field name to filter the workers when retrieving from ADP Workforce Now.
  • Upsert Condition: Enter the expression to determine whether a new user should be created or an existing user needs to be updated in Okta XaaS.
  • Delete Condition: Enter the expression to validate when a user is deleted in Okta XaaS.
  • Update Manager in Okta by Email in ADP Workforce Now: Select to update the manager ID field in Okta using the manager's email ID from ADP Workforce Now.

  • ADP Workforce Now Workers to be inserted or updated in Okta: Select the worker type that must be created or updated in Okta XaaS.

  • Expression to specify the ADP Workers to be inserted or updated in Okta: Enter the expression to validate the workers that should be created or updated in Okta XaaS.
    Note: This field is only displayed when you select Custom in the ADP Workforce Now Workers to be inserted or updated in Okta in the Identity Source field.

  • ADP Workforce Now Workers to be deleted in Okta: Select the worker status to determine the workers that must be deleted in Okta XaaS. 

  • Standard or Advanced Delete Rules: Select the standard or advance rule to delete the workers in Okta XaaS. For most integrations, the standard rules for deactivation will suffice. If you need a custom expression, you can select the Advance option.

  • Advance Delete Expression: Enter the expression to validate the workers that should be deleted in Okta XaaS.
    Note: This field is only displayed when you select Advance in the Standard or Advanced Delete Rules field.

  • Enable Candidate Support: Select to enable candidate management settings.

Candidate Management

Configure the following settings:

  • Create Users With Application Status: Select the application status to filter candidates when retrieving them from ADP Workforce Now.

  • Okta Login Username Format: Select the username format for the Okta login.

  • Okta Login Username Custom Format: Enter an expression to define the username format for the Okta login.

  • Okta Login Username Domain: Enter the domain name associated with the Okta login username.

  • Okta Primary Email ID Format: Select the Okta primary email ID format.

  • Okta Primary Email Custom Format Expression: Enter an expression to define the Okta primary email ID format.

  • Okta Primary Email Domain: Enter the email domain name associated with the Okta primary email.

  • Delete Users With Application Status: Select the application status to delete candidates from Okta XaaS.

Schedule

Integrations can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details.

Run Integration

You can set up an event or schedule triggers to execute the integration. For further details, refer to the Integration Using Scripts guide.

Once the integration is successfully executed, Aquera sends the CSV report via email to the specified administrators.
