Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the integration to sync Candidates  data from ADP Workforce Now to the ticketing resource in Zendesk.

Key Functions

The ADP Workforce Now to Zendesk integration automates the onboarding tickets with the following primary capabilities:

  • Creates a Zendesk onboarding ticket for each newly created candidate in ADP Workforce Now.

Prerequisites

The following are the prerequisites:

  • Admin Access to Zendesk: You must have admin access to your Zendesk instance. You can visit the Zendesk portal to get access to an Zendesk instance.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Zendesk:
    • ADP Workforce Now: Ensure that you configure the ADP Workforce Now connector in the Aquera Admin portal. Refer to the ADP Workforce Now Configuration Guide for configuration details.
    • Zendesk: Ensure that you configure the Zendesk connector in the Aquera Admin portal.

Integration Setup

This section details the configuration and mapping process for integrating ADP Workforce Now with Zendesk. It outlines the steps to create the integration, set up integration preferences, and define how worker data is mapped from ADP to Zendesk.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / Zendesk.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Zendesk connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Features

Enable the following features based on your requirements:

  • Enable Candidate Management: Select this checkbox to sync candidate records from ADP into the Zendesk onboarding ticket. When enabled, the Candidate Management section is displayed to configure integration settings for candidate management.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Email Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Email Notification Settings section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.

Candidate Management

The Candidate Management section enables you to customize the integration settings for candidate management according to your specific use case.
Note: This section appears only if you have selected the Enable Candidate Management setting in the Parameters section.

It has the following settings:

  • Select ADP Workforce Now Application Instance: Select the instance in which the candidates data is available that triggers Zendesk to create tickets and workflows.
  • Enable Onboarding Ticket Creation: Enabling onboarding ticket creation will create a ticket in Zendesk for every matching candidate from ADP Workforce Now.

Candidate Filter Types

Select the required filter to fetch the candidates data from ADP Workforce Now based on the use case. The following types are listed below:

  • Application Status: Select the type of application status to filter out the candidates from ADP Workforce Management. When selecting application status, Include Candidates With Application Status section will appear.
    • Include Candidates With Application Status: This field allows you to select the required application status you want to filter. The following are the available options:
      • Pre-Hire Not Started: Allows only the applications with Pre-Hire not started status.
      • Accepted Offer: Allows only the applications with Accepted Offer status.
      • Hire-Pending: Allows only the applications with Hire-Pending status
      • Hired for Different Position: Allows only the applications with Hired for Different Position status.
      • Background Check: Allows only the applications with Background Check status.
  • Custom Status: Select any custom value. When the Custom Status option is selected, the Candidate Lookback Window section will appear.
    • Custom Application Statuses: Enter the custom application status value to get specific candiates applications.
  • Lookback window: Select a custom number of days from which the candidates data will be collected. When the lookback window option is selected, the Candidate Lookback Window section will appear.
    • Candidate Lookback Window: Set the number of days to retrieve candidates whose records have been created or updated within that period.
  • Email: Select if you require filtering the candidates by their email. When the Email option is selected, the Filter Candidates by Email section will appear.
    • Filter Candidates by Email: This field is optional. Enter a full or partial email address to filter candidates by email. Leave blank to retrieve all candidates.

Zendesk

The Zendesk section enables you to customize the integration settings for Zendesk configuration according to your specific use case.

It has the following settings:

  • Select target application: Select the Zendesk application instance designated to receive tickets from ADP Workforce Now.
  • Match Employees Between ADP Workforce Now and Zendesk: Select the method by which employees from ADP Workforce Now are identified against existing Zendesk tickets to facilitate updates, reuse, or duplicate prevention.
    • Enable Customize How Employees Are Matched Across Systems: Enabling this helps implement custom logic to match employees from ADP Workforce Now with existing Zendesk records.
    • Select How Employees Are Matched Between ADP Workforce Now and Zendesk: By default, this field selects the method by which employee records are correlated between the source and target applications.
      • Select the source attribute for matching the employee: Selects the ADP candidate attribute whose value will be used to identify in the Zendesk record.
      • Select Target Attribute for Matching the Employee: Selects the Zendesk ticket attribute to be matched against the ADP candidate correlation value.

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable Logging setting in the Parameters section.

  • Select Log Type: Select the type of data to include in logs. For example, choosing (Minimum - Operations only; no GUID or fields) includes only the operations.
  • Select Log Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.
  • Enter PII Fields: Enter the fields that contain personally identifiable information (PII) to be included in logs. For example, adding dateOfBirth or SSN includes these fields based on the selected log type.
  • Enter GUID Fields: Enter the fields that contain globally unique identifiers to ensure they are handled correctly in logs. For example, adding workerId ensures the value is treated as a system identifier.
  • Enter EmployeeId Fields: Enter the fields that represent employee identification values to be logged consistently. For example, adding associateId ensures the employee identifier is included in logs.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
