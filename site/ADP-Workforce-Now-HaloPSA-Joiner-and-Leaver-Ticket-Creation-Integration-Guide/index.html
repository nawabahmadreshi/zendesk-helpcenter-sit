Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks involved in setting up the ADP Workforce Now / HaloPSA integration, which creates onboarding and offboarding tickets in HaloPSA based on employee onboarding and offboarding events in ADP Workforce Now.

To use this integration, you must first create a User Integration in the Aquera Admin portal that retrieves newly onboarded / offboarded employees from ADP Workforce Now and then creates/deletes the corresponding user in a target system (for example, Active Directory). After creating the user integration,  you need to configure the ADP Workforce Now / HaloPSA integration by following the steps in this guide. After this integration is setup, you must select this integration in both the New User Integration Hook and the Deprovision User Integration Hook configuration fields within the user integration. 

When the user integration runs, the ADP Workforce Now / HaloPSA integration is triggered for every successful creation or deletion of a user in the target system defined in the user integration.

Key Functions

The ADP Workforce Now / HaloPSA integration performs the following functions:

  • Reads the details of the onboarded / offboarded employees from the events sent by the user integration.
  • Creates an onboarding ticket in HaloPSA for a new employee or an offboarding ticket for a terminated employee. The ticket type depends on the configuration in the integration.
  • Assigns the ticket to an Agent in HaloPSA based on the integration configuration.
  • Creates integration reports based on the integration configuration 
  • Sends email notifications on integration runs along with the integration report based on the integration  configuration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • ADP Workforce Now: Ensure that you configure the ADP Workforce Now connector in the Aquera Admin portal. Refer to the ADP Workforce Now Configuration Guide for configuration details.
    • HaloPSA: Ensure that you configure the HaloPSA connector in the Aquera Admin portal. Refer to the HaloPSA Configuration Guide for configuration details. 

Integration Setup

This section details the configurations for creating onboarding and offboarding tickets in HaloPSA. It outlines the steps to create the integration and set up the integration preferences.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / HaloPSA.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and HaloPSA connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the ADP Workforce Now and HaloPSA connector instances used by the integration. This section allows you to search for a different instance of the application connectors and select them.

Parameters

The Parameters section enables you to customize the integration features and global settings according to your specific use case.

Global Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification section is displayed for further configuration.
  • Enable Reporting: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Dashboard Generation: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.

Integration Features

Enable the following features based on your requirements:

  • Enable Onboarding Ticket Generation: Select this checkbox for the integration to handle the creation of onboarding tickets in HaloPSA. When this checkbox is selected, the Onboarding Ticket section is displayed.
  • Enable Offboarding Ticket Generation: Select this checkbox for the integration to handle the creation of offboarding tickets in HaloPSA. When this checkbox is selected, the Offboarding Ticket section is displayed.

Onboarding Ticket

This section details the following settings that are used to configure the creation of onboarding tickets in HaloPSA:

  • Assign Agent: Select this checkbox if you want the integration to add an agent name to the onboarding ticket. When this checkbox is selected, the Agent drop-down list is displayed. When this checkbox is not selected, the ticket will not be assigned to an agent.
  • Allow Selection of End User: Select this checkbox if you want to add the end user details in the ticket. When this checkbox is selected, the End User drop-down list is displayed. When this checkbox is not selected, the end user details will not be added to the ticket.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Post-Setup Configuration

After the ADP Workforce Now / HaloPSA integration is setup, you must select this integration in both the New User Integration Hook and the Deprovision User Integration Hook configuration fields within the User Integration. 

Note: You must have already setup a User Integration to retrieve onboarding and offboarding employees from ADP Worforce Now and to create or delete that user in a target system.

Integration Execution

The integration is triggered automatically based on the new user hook and deprovision user hook configured in the user integration.
