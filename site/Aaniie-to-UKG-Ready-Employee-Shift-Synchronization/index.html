Aaniie to UKG Ready Employee Shift Synchronization

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the synchronization of data across resource types.

This integration automates the synchronization of employee shift schedules between the Aaniie system and UKG Ready. It ensures that scheduled shifts from Aaniie are accurately reflected in UKG Ready for workforce management, reporting, and compliance purposes. The integration is configurable with built-in reporting and notification capabilities.

This integration performs the following tasks:

  • Fetches scheduled shift data from Aaniie within a specified date range.
  • Retrieves the relevant schedule, existing shifts, and employee records from UKG Ready.
  • Maps each Aaniie shift to a UKG employee using unique identifiers.
  • Converts shift times to UKG Ready’s required ISO 8601 format with timezone offset.
  • Filters out duplicate shifts already present in UKG Ready.
  • Validates shifts to ensure they do not extend beyond the allowed schedule period, and notifies if any are invalid.
  • Enriches each shift with the employee’s default cost center and skill information from UKG Ready.
  • Prepares and bulk-creates new, validated shifts in UKG Ready.
  • Generates a CSV report summarizing the synchronization process.
  • Sends notification emails with the report and details to the configured recipients.

Integration Setup

This section outlines the configuration and execution process for integrating scheduled shifts from Aaniie with UKG Ready. It describes how to create the integration, define operational parameters, and establish reporting and notification workflows.

This integration reads scheduled shift data from the Aaniie application and creates corresponding shift records in the UKG Ready application, ensuring workforce schedules are synchronized.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Aaniie to UKG Ready Employee Shift Synchronization.
  5. Enter a name and a description for the integration.
  6. Select your Aaniie and UKG Ready connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Lookback Period (Days):  Defines how many days in the past the integration will fetch shifts from Aaniie. For example, entering 30 will include all shifts starting from 30 days ago up to today.
  • Forward Lookup Period (Days):  Specifies how many days into the future the integration will fetch shifts from Aaniie. For example, entering 15 will include all shifts from today up to 15 days in the future.
  • Schedule Name: Identifies the specific schedule in UKG Ready where the shifts will be created.
  • Company ID:  Enter the Company ID for your UKG Ready environment.
  • Report Only: Select to run the integration in report-only mode without creating shifts.
  • Send Integration Report via Email:  If enabled, the integration will email a summary report and details of the run to the specified recipients.
  • Enable Integration Report Download: When enabled, the integration will produce a downloadable CSV file summarizing the results of each run.
  • Recipients' Email Addresses: The email address where reports and notifications about the integration run will be sent.

Attribute Mapping

The Aaniie Shifts to UKG Ready integration maps the most commonly used Aaniie shift attributes to the corresponding UKG Ready shift fields. Additionally, mapping for cost center and skill is performed using UKG Ready data.

Begin Date Shift Start Date
Begin Time Shift Start Time
End Date Shift End Date
End Time Shift End Time
(N/A) Schedule Name
(N/A) Company ID
(N/A) Cost Centers
(N/A) Skill

Run the Integration

The integration can be scheduled to run automatically at customizable intervals. Available frequencies include 15 minutes, 30 minutes, 1 hour, 4 hours, 6 hours, 12 hours, or 24 hours. You can also enable recurring executions to run the integration continuously on your chosen schedule.

On successful execution, the CSV file will be sent to the specified recipients. In case of integration errors, notifications and reports will also be sent to the email addresses.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
