ADP Workforce Now API Central / Entra ID Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the configuration for the ADP Workforce Now API Central/Entra ID integration. This integration automates provisioning across ADP Workforce Now API Central and Microsoft Entra ID. It synchronizes users and candidate data, manages group assignments, and handles onboarding, updates, and terminations. It also supports automated access assignments and provides detailed notifications, logs, and reports.

To identify user records across both systems, the integration uses candidate attributes from the ADP Workforce Now API Central and matches them to user attributes in Entra ID, using the matching as a correlation attribute.

Key Functions

The ADP Workforce Now API Central / Entra ID integration provides the following functions:

  • Creation of users
  • Update the users
  • Deactivation of users
  • Deletion of users
  • Group Assignment
  • Group Revocation

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.

  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • ADP Workforce Now API Central: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for details.
    • Entra ID: Refer to the Entra ID Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push candidates' data from ADP Workforce Now API Central to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now API Central / Entra ID in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The following tabs are displayed for you to configure the integration:

Overview

This tab displays comprehensive details about the integration and its related information.

Source

This tab displays the different sections used to identify the ADP Workforce Now API Central application connector and configure the candidate-related settings.

This tab has the following configuration sections:

Application Setting

This section enables you to identify the ADP Workforce Now API Central application connector used for the integration and configure the candidate-related import settings. The following fields are displayed in this section:

  • Candidate Source Application: Select the ADP Workforce Now API Central application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Retrieve the complete manager profile for candidates: Select to retrieve the manager's profile of all candidates.
  • Retrieve Candidates with Job Applications Submitted 'X' Days Prior: Enter the number of days 
  • Candidate Status(es) to be considered for Account Activation: Select the candidate statuses for which the account will be activated.
  • Candidate Status(es) to be considered for Account De-Activation: Select the candidate statuses for which the account will be deactivated.

Directory

This section enables you to identify the Entra ID application connector used for the integration and configure the group-related settings, account correlation, and other account settings.

This tab has the following configuration sections:

Directory Settings

This section enables you to configure application provisioning. In this section, you can define how records from Microsoft Entra ID are provisioned to the target application and how attributes are mapped between the systems.

This section enables you to configure the target Entra ID application connector involved in the integration. It also allows you to define how passwords are generated for new users in the directory.

Groups Settings

This section allows you to configure group settings for users. 

Candidates Group Configuration

The following fields are available for configuration in this section:

  • Remove extra groups assigned to the user during the update: Select this to remove users from groups where they already exist but are not included in the group assignments after the group rules are evaluated.
  • Remove user from groups on deactivation: Select this to automatically remove the user from all assigned groups when the user account is deactivated (disabled).

Manage Candidate - Entra ID

This section enables you to configure account correlation and account activation, deactivation, and update settings, as well as define attribute mappings.

Account Correlation Settings

The following fields are available for configuration in this section:

  • Correlation Attribute from Source Application: Select the correlation attribute from ADP Workforce Now API Central. The available options are Email, Person OID, and Associate OID.
  • Correlation Attributes from Target Application: Select the correlation attribute from ADP Workforce Now API Central. The available options are User principal name, Email, Other emails, and On-premises Extension attribute 1.

Account Activation Settings

The following fields are available for configuration in this section:

  • Account Assignment Policy: Select a policy or list of policies from the dropdown menu to determine how candidate accounts are assigned during provisioning. If you do not find your assignment policy in the list, click the refresh icon available next to the field to reload the available policies.
  • Force password change on first login: Click to force candidates to change their password on their first sign-in. 
  • Enable Account Activation Limit: Click to limit the number of candidate accounts that can be activated during a single integration run. When enabled, the following field is displayed:
    • Set the Creation Limit: Specify the maximum number of candidate records that can be created in the Entra ID during a single integration run.

Account Update Settings

The following fields are available for configuration in this section:

  • Account Assignment Policy: Select a policy or list of policies from the dropdown menu to determine how candidate accounts are updated during provisioning. If you do not find your assignment policy in the list, click the refresh icon available next to the field to reload the available policies.
  • Enable Account Updation Limit: Click to limit the number of candidate accounts that can be updated during a single integration run. When enabled, the following field is displayed: 
    • Set the Update Limit: Specify the maximum number of candidate records that can be updated in the Entra ID during a single integration run.

Account Deactivation Settings

The following fields are available for configuration in this section:

  • Enable Account Deactivation Limit: Select to limit the number of candidate accounts that can be deactivated during a single integration run. When enabled, the following field is displayed:  
  • Set the Deactivation Limit: Specify the maximum number of candidate records that can be deactivated in Entra ID during a single integration run.
  • On Deactivation, Perform Actions on Candidate: Select an action or a list of actions from the dropdown menu to determine what action should be taken when a candidate is deactivated. The available options are Delete Account, Disable Account, and Reset Password.

Attribute Mapping

This section allows you to control how each attribute is synchronized between ADP Workforce Now and Entra ID.
Source attributes are listed on the left, and target attributes on the right. You can modify only the source attributes as needed to align with your specific use case.
JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
Attributes marked with a red asterisk (*) are required for successful provisioning.
The Search Attributes field allows you to filter the list of available attributes for mapping.
After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.

Policy

The following sections are displayed under the Policy tab:

Access Rules

This section allows you to define rules for assigning users to business groups defined in the Business Groups section.

Candidate Group Assignment Rule

Perform the following steps to define group assignment rules for the candidates:

  1. Click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the assignment rule.
  2. Search and select the attribute you want to use to define a condition. For example, select Manager Department.
  3. In the row below, click to view the attribute values. If multiple values are selected, OR condition is applied among them.
  4. If required to add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  5. If required to include another attribute to the group assignment rule, click the vertical ellipsis in the first column, and then click Add Attribute.
    If multiple columns are added for a rule, AND condition is applied between different columns.
  6. Under the Groups field, click to view the available groups. Select the group to which candidates who satisfy the specified condition will be assigned.
  7. If needed, click the + icon in the top-right corner to add another rule, as explained above.
  8. Click Save Changes at the top-right corner of the page to save the configuration.

Tables

In the Tables tab, you can add tables to the integration. Tables are used to define custom source attributes and synchronize them with the target application. If the required data is not available in the source application, you can define it in a table and use it to update the target application. Once defined, the table can be referenced in attribute mapping expressions to set or update values automatically.

For example, to set the timezone attribute of users in the target application to Europe/Paris, create a table during integration setup with timezone as an attribute name and Europe/Paris as a value. Once defined, reference the table in the source attribute of the attribute mapper using the following format:
TABLE.<table_name>.<key_name> or TABLE.<table_name>[<key_name>]

For example: TABLE.test.timezone

Note: You can configure expressions with nested tables in the attributes mapper, if required.
For example: TABLE.UserHireRecord[TABLE.UserRecord["empDate"]]

Refer to the Using Tables in Integrations guide for more details.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
