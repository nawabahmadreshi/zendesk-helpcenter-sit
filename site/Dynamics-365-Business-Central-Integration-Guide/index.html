Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the end-to-end tasks for setting up an integration that assigns MEM and PLM licenses using Dynamics 365 Business Central and Microsoft Entra ID.

This integration automates the license assignment process by identifying the group code in Dynamics 365 Business Central and assigning MEM, PLM, or both licenses accordingly.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connector in the Aquera Admin Portal: You need to configure the Dynamics 365 Business Central application connector in the Aquera Admin portal.
    Note: While configuring the connector, all Aquera tenant attributes are mandatory for this integration.
  • License: You must have a valid license assigned to the user before security entities can be applied.
  • Web services: You must enable the required web services. Refer to the Setting Up Web Services section for more details.

Setting Up Web Services

Perform the following steps to add (publish) web services in Dynamics 365 Business Central:

  1. Log in to Dynamics 365 Business Central Client.
  2. Use the search bar to find Web Services, then click Web Services from the search results.
  3. On the Web Services page, check if a web service with the required Object Type and Object ID already exists. If found, you can reuse the existing service.

    The following are the object IDs (Page IDs) for different Web Services:
    • Users: 9800
    • SecurityGroupMembers: 9869
    • MEM Licensed Users: 70210976
    • PLM Licensed Users: 70701596
    • Approval User Setup: 663
    • User Setup: 119
    • Sales People Purchasers: 5116
  4. If no matching web service is found, then click + New to create a new one. Fill in the following details:
    • Object Type: Select Page from the dropdown.
    • Object ID: Enter the relevant Page ID.
    • Service Name: Enter a unique name for the service. This will appear in the endpoint URL.
    • Published: Check the box to publish the service.
  5. Once the web service is published, Dynamics 365 Business Central will generate an OData V4 URL automatically.
  6. Copy the OData V4 URL and use it in your integration setup.

Integration Setup

The integration must be configured to assign MEM and PLM licenses using Dynamics 365 Business Central.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dynamics 365 Business Central License Management.
  5. Enter a name and a description for the integration.
  6. Select your Dynamics 365 Business Central connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the application connectors for the integration. This section allows you to search and update application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Maximum Limit: Enter the maximum number of users to process in the integration run.

Users Settings

Configure the following users' settings:

  • OData V4 API Endpoint for Users*: Enter the OData V4 API URL for the users.

Group Members Settings

Configure the following group members' settings:

  • OData V4 API Endpoint for Group Members*: Enter the OData V4 API URL for the group members.

MEM License

The MEM License section enables you to customize the MEM License settings according to your specific use case.

MEM License Settings

Configure the following MEM License Settings:

  • Security Group Code of MEM Licensed Users: Enter the security group code of MEM licensed users.
  • OData V4 API Endpoint for MEM Licensed Users: Enter the OData V4 API URL for MEM licensed users.

MEM Location Settings

Configure the following MEM Location Settings:

  • MEM Location Attribute Name in the Microsoft Entra ID (API Name): Enter the API name of the MEM location attribute as defined in Microsoft Entra ID.
    Notes:
    • Refer to the Microsoft Entra ID to obtain the API names.
    • If the location attribute contains nested objects, specify it using dot notation. For example, for the attribute:
      {
  "onPremisesExtensionAttributes": {
      "extensionAttribute10": "entity1-entity2"
   }
}
      Enter the location attribute as:
      onPremisesExtensionAttributes.extensionAttribute10

PLM License

The PLM License Settings section enables you to customize the PLM License settings according to your specific use case.

PLM License Settings

Configure the following PLM License Settings:

  • Security Group Code of PLM Licensed Users: Enter the security group code of PLM licensed users.
  • OData V4 API for PLM Licensed Users: Enter the OData V4 API URL for PLM licensed users.

Approval User Setup

The Approval User Setup Settings section enables you to customize the Approval User Setup settings according to your specific use case.

Approval User Setup Settings

Configure the following Approval User Setup Settings:

  • OData V4 API for Approval User Setup: Enter the OData V4 API URL to retrieve approval user setup details.
  • OData V4 API for Sales People and Purchasers: Enter the OData V4 API URL to retrieve data for sales people and purchasers.
  • Approval User Setup Group Codes: Select the group to manage approval user configurations.

User Setup

The User Setup Settings section enables you to customize the User Setup settings according to your specific use case.

User Setup Settings

Configure the following User Setup Settings:

  • OData V4 API for User Setup: Enter the OData V4 API URL to retrieve user setup details.
  • User Setup Group Codes: Select the group to manage user configurations.

Tables

In the Tables tab, you can add tables to the integration. Refer to the Using Tables in Integrations guide for more details.

Schedule

Integrations can be scheduled to run automatically based on your choice of timeframe. Refer to the Integration Schedule Setup Guide for further details.

Execute Integration

You can execute an integration manually or schedule it to run automatically. For further details, refer to the User Action on Integrations guide.
