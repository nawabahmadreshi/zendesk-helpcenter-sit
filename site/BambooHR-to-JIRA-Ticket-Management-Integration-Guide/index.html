BambooHR to JIRA Ticket Management Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the user's lifecycle from BambooHR to Jira Service Management Cloud for the selected work type. This integration ensures that all user lifecycle events in BambooHR are accurately tracked and managed within the Jira system for seamless operational workflows.

Key Functions

The BambooHR to JIRA Ticket Management Integration provides the following functions:

  • Whenever an employee is onboarded, transferred, or terminated in the BambooHR application, the integration automatically creates a ticket in the Jira Service Management Cloud application.
  • If a ticket is already available for an employee in Jira, the integration updates the details of the ticket based on the integration settings.
  • When an employee is rehired, a ticket will be created.
  • To identify the existing Jira tickets corresponding to BambooHR user records, the integration matches the employeeNumber or work email attribute from BambooHR to a JIRA Service Management Cloud custom attribute defined in the integration.

Limitation

When a user’s previous job record is updated in the BambooHR application, the system incorrectly treats it as the latest job record. This is a known limitation with the BambooHR system. As a result, the integration creates or updates a ticket in the Jira Service Management Cloud for that record.

To avoid this issue, you must always update the latest job record for an employee in BambooHR.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • BambooHR: Refer to the BambooHR Configuration Guide for details.
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.
  • Setup Webhooks in BambooHR: Your BambooHR admin must set up Webhooks in the BambooHR application to trigger events for employee onboarding, transfers, and terminations. For details on the webhooks setup, refer to the Setting up Webhooks in BambooHR section.
    Note: When configuring webhooks for sending events to Aquera, make sure the field names exactly match those provided in the Aquera portal. If the field names in BambooHR webhooks differ from those in the Aquera integration setup page, the events will not be processed by the integration. In case you want the Aquera integration setup page to show different field names (to align with your BambooHR webhook configuration), please contact the Aquera support personnel.

Integration Setup

This section outlines how the integration must be configured to push the group name from BambooHR to Jira Service Management Cloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select BambooHR to Jira Ticket Management.
  5. Enter a name and a description for the integration.
  6. Select your BambooHR and Jira Service Management Cloud connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable Logging: Select to generate logs outlining data changes. The Logging Settings section is displayed.

JML Settings

Configure the following JML settings:

  • JML Operations: Select the JML operations that must be supported by the integration for creating or updating a Jira ticket. The available options are Joiner, Mover, and Leaver.
    Note: The Joiner Ticket Settings, Mover Ticket Settings, and Leaver Ticket Settings sections will be displayed based on your selection.
  • Webhook Employment Status Fields: Select the webhook fields that must be used to identify any change in the employment status in BambooHR.
    Note: Ensure that you have configured the exact field names as provided in this drop-down list when you have configured webhooks in BambooHR to send the employment status changes. For example, you must configure the field name as Effective Date in BambooHR webhooks instead of the default field name, Employment Status-Effective Date. If you want a different set of names to be shown in this drop-down (to align with the field names in your webhooks setup), contact Aquera support personnel. 

  • Wait Time to Handle Duplicate Events (in Minutes): This field is applicable for all the update events from BambooHR. Enter the delay time (in minutes) after which an update event has to be processed. When a user is created in BambooHR, both the create and update events are triggered. To ensure the update event is processed after a delay, specify a wait time (in minutes) in this field. For example, entering "5" will delay the update event by 5 minutes after the processing of create ticket event. 

Employee Filter Settings

Configure the following settings to filter employees from BambooHR:'

  • Employee Attribute: Select the attribute that must be used for filtering the employees from BambooHR.
  • Operator: Select the operator that must be applied for the filter.
  • Filter Value: Select the attribute value that must be used for filtering the employees.
    For example, if you want to filter an employee with the employee number 1234, provide the following values in the filter settings:
    Employee Attribute: Employee Number
    Operator: Equals
    Filter Value: 1234

Logging Settings

Select the Log Detail Level: Select Full to create logs with all the details.

Joiner Ticket Settings

Configure the settings related to the joiner tickets.

Ticket Management

Configure the following joiner ticket management settings:

  • Project Name: Select the project under which the integration must create the joiner tickets in Jira.
  • Work Type: Select the corresponding work type to create joiner tickets. For example, select Joiner from the dropdown to create tickets of type Joiner under the selected project.

Ticket Correlation Settings

Configure the following joiner ticket update settings if you want an existing ticket to be updated for an employee. If you want a new ticket to be created for every joiner event, leave this section without configuring these settings.

  • BambooHR Correlation Attribute: Select the BambooHR employee attribute that will be used by the integration to find a joiner ticket, if it already exists for a BambooHR employee. The available employee attributes are Work Email and Employee Number. 
  • Jira Correlation Field: Select the joiner ticket field that will be used by the integration to find a ticket that might have already been created for an employee from BambooHR. For example, Email Address.
    Note: Make sure that matching attributes are configured correctly to avoid creating duplicate tickets. For example, if you have selected Work Email as the BambooHR Matching Attribute and Email Address as the Jira Matching Field, the integration uses the values of these two attributes to find whether a joiner ticket already exists for an employee.
  • Ticket Statuses for Updating an Existing Ticket: Select one or more statuses of the joiner tickets that must be updated with the details from BambooHR. If the integration finds an existing joiner ticket for an employee, the integration updates that ticket's details if the status of that ticket matches one of the statuses configured in this field. This field is mandatory if you have configured the correlation attributes.
    Note: The ticket details are updated based on the attributes mapping configured for the joiner tickets in the integration.

Rehire Settings

Configure the settings to identify the rehire records:

  • Employee Inactive Statuses: Select the statuses from the drop-down that must be considered as inactive statuses for an employee. The integration considers an employee as a rehire only if they move from one of these statuses to a status configured in the Employee Active Statuses field.
  • Employee Active Statuses: Select the statuses from the drop-down menu that must be considered as active statuses for an employee. The integration considers an employee as a rehire only if they move to one of these statuses from a status configured in the Employee Inactive Statuses field. For example, if you have selected Terminated in the Employee Inactive Statuses field and Active in the Employee Active Statuses field, an employee from BambooHR will be considered a rehire if their status changes from Terminated to Active.

Attribute Map

The Bamboo HR to JIRA Ticket Management Integration enables you to map the BambooHR employee attributes to the Jira ticket attributes based on your requirements. You can access three attribute maps in this integration - one each for Joiner, Mover, and Leaver ticket mappings. 

Within each attribute map, you can set the source attributes/expressions corresponding to each ticket attribute based on your use case.

Event

The Event tab is used to configure the webhook details.

Provide the following details and save the changes:

  • Event Source: Select your BambooHR application from which all joiner, mover, and leaver events are triggered.
  • URL: Copy this URL and paste it in the Post to URL field on the BambooHR Webhooks setup page. For details on setting up a webhook in the BambooHR application, refer to the Appendix section.
  • Create Token: Click this button to show the Custom Token field. Copy the Private Signature Key from the BambooHR webhooks setup page and paste it in this field.
    For details on setting up a webhook in the BambooHR application, refer to the Setting up Webhooks in BambooHR section.

Appendix

This section outlines the procedure for configuring webhooks in the BambooHR application.

Setting up Webhooks in BambooHR

Webhooks must be set up in BambooHR to monitor any changes made to the fields that you specify and to post that information to Aquera. Aquera's BambooHR webhook will automatically process those changes and send them to the target application.

  1. Log in to your BambooHR account as an administrator. The Home page is displayed.

  2. Click the Settings icon at the top-right corner of the page. The Settings page is displayed.
  3. Click Webhooks. The Webhooks section is displayed on the page.

  4. Click Add Webhook. The Add Webhook page is displayed.

  5. Enter a name for the webhook in the Webhook Name field.
  6. Select the fields to be monitored by the BambooHR system for any changes, from the What fields do you want to monitor? drop-down list. The fields have to be selected one by one. All the selected fields are listed on the page.

    You can remove any selected fields by clicking the close icon next to the field.

  7. Select the same set of fields in the What fields do you want to post? section.
    • Search for the required field by typing the field name  and selecting it,

      or

    • Browse to the individual categories from the Browse Fields list and then add the required field to the Selected Fields list.

  8. Select JSON in the What format should the data be in? section.

  9. In the Post to URL field, paste the URL that has been copied from the Event tab of the BambooHR to Jira Ticket Management integration in the Aquera Admin portal.

  10. Click Generate Key. Copy the Private Signature Key and use it in the Custom Token field in the Event tab on the Aquera Admin portal.
  11. Click Save. The newly created webhook is displayed in the Webhooks section on the Settings page.

    Now, when the specified data of users in the People section of BambooHR is updated, it will be sent to Aquera for processing and sending to the target application.
