Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks involved in setting up the ADP Recruiting Management / ManageEngine ServiceDesk integration, which creates onboarding tickets in ManageEngine ServiceDesk based on employee onboarding events in ADP Recruiting Management.

To use this integration, you must first create a User Integration in the Aquera Admin portal that retrieves newly onboarded employees from ADP Recruiting Management and then creates the corresponding user in a target system (for example, Active Directory). After creating the user integration,  you need to configure the ADP Recruiting Management / ManageEngine ServiceDesk integration by following the steps in this guide. After this integration is set up, you must select this integration in both the New User Integration Hook and the Deprovision User Integration Hook configuration fields within the user integration.

When the user integration runs, the ADP Recruiting Management / ManageEngine ServiceDesk integration is triggered for each successful user creation in the target system specified in the user integration.

Note: Before running this integration for the first time, you must manually create a request in ManageEngine ServiceDesk. This ensures that the required request structure and configurations are initialized. If this step is not completed, the integration may fail during execution.

Key Functions

The ADP Recruiting Management to ManageEngine ServiceDesk integrations provide the following functions:

  • Reads the required data from the user integration during execution.
  • Creates service requests in ManageEngine ServiceDesk based on the configured operation.
  • Processes request creation automatically during integration runs.
  • Creates integration reports based on the integration configuration.
  • Sends email notifications for integration runs along with the integration report, based on the configuration.

Integration Setup

This section outlines how the integration must be configured to push worker data from ADP Recruiting Management to ManageEngine ServiceDesk.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Recruiting Management / ManageEngine ServiceDesk integration.
  5. Enter a name and a description for the integration.
  6. Select both application connector instances from the dropdown menus.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Configure the following integration feature:

  • Enable Onboarding Ticket Generation: Select this checkbox to create onboarding tickets in ManageEngine ServiceDesk for users provisioned from ADP Recruiting Management. When enabled, the Onboarding Ticket section is displayed for configuring additional integration settings.

General Settings

Configure the following general settings:

  • Report only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.

Onboarding Ticket Settings

The Onboarding Ticket Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Onboarding Ticket Generation parameter is enabled in the Integration Features section.

  • New Joiner Ticket Action: Select an action from the dropdown menu that should trigger the creation of a joiner ticket in ManageEngine ServiceDesk. For example, if you select Create, a joiner ticket will be automatically created in ManageEngine ServiceDesk when a candidate is joined in ADP Recruiting Management.
  • Template: Select a template from the dropdown menu to automatically populate ticket details when a candidate joins in ADP Recruiting Management.
  • Request Type: Select the request type from the dropdown menu that should be used for the ticket created when a candidate joins in ADP Recruiting Management.

Sensitive Fields Settings

The Sensitive Fields Settings section enables you to customize the integration settings according to your specific use case. It has the following setting:

  • PII fields: Enter a comma-separated list of field names that contain personally identifiable information (PII). The configured PII fields will be considered as sensitive data and will be excluded from the logs and reports.

    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.

    • Full: All attribute details are displayed.

    • Exclude List: Select this option to log only the attributes specified in the Custom exclusion list.
      • Custom Exclusion list: Select the attributes to be excluded from the log.

Attribute Map

The Attribute Map tab defines how data is mapped from the ADP Recruiting Management to the ManageEngine ServiceDesk application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
