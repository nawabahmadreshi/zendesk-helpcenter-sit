OpenSearch Functions Supported in Integration Scripts

← Back to index

Overview

Aquera's integration scripts support OpenSearch functions that facilitate real-time data ingestion for logging, monitoring, data visualization, and application troubleshooting. These OpenSearch functions enable you to create indexes with records, filter records, update records, and delete records within the dashboard for analytical purposes. The indexed documents are displayed in the Log Insights module, enabling you to analyze and visualize large amounts of data generated by integrations in real-time.

This guide provides a comprehensive overview of the OpenSearch functions supported in Integration Scripts.

Note: This guide is currently for internal use only and must not be circulated to customers.

OpenSearch Functions

The OpenSearch functions can be invoked in integration scripts using the following syntax:

opsSearch.<FunctionName>

The following are the supported OpenSearch functions:

Write

This function allows you to create an index in OpenSearch if it does not exist and add a document (record) to the index. 

Format

write(opsIndexName: string, logObj: object, indexType: string)
Parameters
  • opsIndexName: The index name to create.
  • logObj: The JSON data object for a single document or record.
    Note: When you send the JSON object, you can include an ID. If you choose not to provide one, OpenSearch will automatically generate a unique ID. This ID will serve as the document ID for that particular document (record).
  • indexType: The unique identifier for the type of the Log Insights dashboard template. This is an optional parameter and is used for categorizing the dashboard templates. For example, this parameter can have the value Dayforce-to-AD, which can be a type of dashboard template.
Returns This function does not return anything.
Example

The following sample script creates the OnboardNewHires index and adds three user records to the index:

const user1 = {
"id":2001,
"name": "John Doe",
"age": 30,
"email": "johndoe@example.com",
"isActive": true,
"department":"Sales",
"address": {
"street": "123 Main St",
"city": "Boston",
"state": "MA",
"zip": "02108"
},
"hobbies": ["reading", "hiking", "photography"]
};

const user2 = {
"id":2002,
"name": "Jane Smith",
"age": 28,
"email": "janesmith@example.com",
"isActive": true,
"department":"Development",
"address": {
"street": "456 Oak Ave",
"city": "Chicago",
"state": "IL",
"zip": "60601"
},
"hobbies": ["painting", "cooking", "traveling"]
};

const user3 = {
"id":2003,
"name": "Mike Johnson",
"age": 35,
"email": "mikej@example.com",
"isActive": false,
"department":"Marketing",
"address": {
"street": "789 Pine Rd",
"city": "Seattle",
"state": "WA",
"zip": "98101"
},
"hobbies": ["gaming", "music", "swimming"]
};

const usersArray = [user1, user2, user3];
for (let i = 0; i < usersArray.length; i++) {
const user = usersArray[i];
await opsSearch.write("OnboardNewHires",user);
}

Filter

This function performs a search query for documents (records) in an index.

Format

filter(opsIndexName: string, query: object)
Parameters
  • opsIndexName: The index name to search.
  • query: The query to perform the full-text search in a field.

    query format:
    {
    "query": {
    "match": {
    "field_name": "search_term"
    }
    }
    }
Returns

Returns a JSON object containing search results and metadata.
Example

The following sample script searches for records where isActive is true in the OnboardNewHires index:

const opsIndexName = "OnboardNewHires";
const searchQuery = {
"query": {
"match": {
"isActive": "true"
}
}
}

const searchResult = await opsSearch.filter(opsIndexName, searchQuery);
log.info(`searchResult: ${JSON.stringify(searchResult)}`);

Update

This function allows you to partially or fully update an existing document (record) in an index.

Format

update(opsIndexName: string, docId: string, updateBody: object) 
Parameters
  • opsIndexName: The name of the index where the document resides.
  • docId: The ID of the document to update.
  • updateBody: The fields to update and their new values in the request body.

    updateBody format:
    {
    "field_name":"new_value"
    };
Returns

Returns a boolean value indicating if the update operation was successful or not.
Example

The following sample script updates document (record) ID 2002 in the OnboardNewHires index:

const opsIndexName = "OnboardNewHires";
const docId = "2002";
const updateBody = {
"department":"Engineering"
};

const updateResult = await opsSearch.update(opsIndexName, docId, updateBody);
log.info(`updateResult: ${updateResult}`);

Delete

This function removes a document (record) from an index.

Format

delete(opsIndexName: string, docId: string)
Parameters
  • opsIndexName: The name of the index where the document resides.
  • docId: The ID of the document to delete.
Returns

Returns a boolean value indicating if the delete operation was successful or not.
Example

The following sample script deletes document (record) ID 2002 from the OnboardNewHires index:

const opsIndexName = "OnboardNewHires";
const docId = "2002";
const deleteResult = await opsSearch.delete(opsIndexName, docId);
log.info(`deleteResult: ${deleteResult}`);

Truncate

This function removes all records from an index.

Format

truncate(opsIndexName: string)
Parameters

opsIndexName: The name of the index where the records are present.
Returns

Returns a boolean value indicating if the truncate operation was successful or not.
Example

The following sample script deletes all documents (records) from the OnboardNewHires index:

const opsIndexName = "OnboardNewHires";
const truncateResult = await opsSearch.truncate(opsIndexName);
log.info(`truncateResult: ${truncateResult}`);
