ADP Workforce Now Workers to Microsoft Entra ID Users Integration

← Back to index

Overview

Aquera provides an innovative platform that enables your organization to scale complex data management by automating the integration of workers from the ADP Workforce Now application with users in the enterprise application in Microsoft Entra ID.

An enterprise application in Microsoft Entra ID refers to an application that enables integration with Microsoft Entra ID.

You must have an enterprise application if you want to provision users from a source application to the Microsoft Entra ID using API-driven provisioning to Microsoft Entra ID, or users from a source application to on-premises AD using the API-driven provisioning to on-premises Active Directory.

This guide covers the end-to-end tasks for setting up synchronization between the Workers resource of ADP Workforce Now and the Users resource of Entra ID.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Microsoft Entra ID: Refer to the Microsoft Entra ID Configuration Guide for details.
  • Enterprise application in Entra ID: You must have an enterprise application if you want to provision users from a source application to the Microsoft Entra ID using API-driven provisioning to Microsoft Entra ID, or users from a source application to on-premises AD using the API-driven provisioning to on-premises Active Directory.

Creating an Enterprise Application in Entra ID

If your use case requires provisioning to Microsoft Entra ID, refer to the Creating Enterprise Application for API-driven Provisioning to Microsoft Entra ID section of this guide to create an enterprise application.

If your use case requires provisioning to an on-premises Active Directory, refer to the Creating Enterprise Application for API-driven Provisioning to On-Premises Active Directory section of this guide to create an enterprise application.

Once you have logged into the Microsoft Entra admin center, created the enterprise application, and finished the mappings, you must perform the following additional steps for the ADP Workforce Now to Entra ID integration:

  1. Select the Show advanced options checkbox on the Attribute Mapping page to display the Edit Attribute List page.
  2. Select the Required? checkbox for the id and userName attributes.
  3. Select the Primary Key and Required checkboxes for the externalId attribute.
  4. Select the Type as Reference and Referenced Object Attribute as

Integration Setup

This section outlines the configuration required to integrate ADP Workforce Now with Entra ID, enabling the transfer of workers' data.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now Workers to Entra ID Users.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The Preferences section under Configuration enables you to customize the integration settings according to your specific use case. It has the following configuration parameter:

Report Only: Select this checkbox to generate a report that shows the data that would be changed, without actually making those changes. This can be selected for the first integration instance to identify any issues. You can clear this selection after verifying the report and confirming the expected result.
