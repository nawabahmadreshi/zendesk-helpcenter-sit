Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Dayforce to Homecare Homebase integration

To identify user records across both systems, the integration uses the Employee Number attribute from Dayforce and matches it to the Payroll Number in Homecare Homebase as a correlation attribute.

Integration Setup

This section outlines how the integration must be configured to push user data from Dayforce to Homecare Homebase.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce to Homecare Homebase Integration.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Homecare Homebase connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.
  • Enable Incremental Mode: Select to run the integration in incremental mode.

User Management

The User Management section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Correlation Settings

Configure the following correlation settings:

  • Source Correlation Attribute: Select the source attribute for correlation from the dropdown menu.
  • Target Correlation Attribute: Select the target attribute for correlation from the dropdown menu.
  • Import Limit: Specify the maximum number of user records that can be imported in the target application during a single integration run.

User Provisioning Threshold Settings

Configure the following provisioning threshold settings:

  • Enable Create Limit: Select to restrict the number of new users created during sync.
  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Enable Update Limit: Select to restrict the number of new users updated during sync.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single integration run.
  • Enable Delete Limit: Select to restrict the number of new users deleted during sync.
  • Terminate Limit: Specify the maximum number of user records that can be terminated from the target application during a single integration run.

 

Debug Settings

The Debug Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Source Filter: Select the Paylocity attribute to filter the users while retrieving from Paylocity. The available option is Employee ID.
  • Source Filter Value: Provide the value for the selected source filter.
  • Fetch Limit: Specify the maximum number of user records that can be fetched in the target application during a single integration run.
  • Generate Debug Logs: Select to generate debug logs.
  • Debug Log Detail Level: Select the desired debug log detail level from the dropdown menu. The available options are Full, Minimal, Summary, and Deep.

Log Settings

The Log Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the log.
      • Excluded Attributes: Select the attributes to be excluded from the log.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

  • Recipients' Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent
  • Email Subject on Successful Integration Execution: Enter the content that must be used as the email subject in the notifications sent for successful integration execution
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution

Notification Report Settings

Configure the following settings for the notification reports:

When this checkbox is selected, the Notification Report Settings section is displayed, allowing you to customize the notification report. If the checkbox is not selected, the email is sent without a CSV report attachment.
  • Notification Report Detail Level: Select the desired notification report detail level. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the notification report.
      • Excluded Attributes: Select the attributes to be excluded from the notification report.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.
  • Save Attribute Values: Select to save all the user attribute values from Paylocity during sync.
