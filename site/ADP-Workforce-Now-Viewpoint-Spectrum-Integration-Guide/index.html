ADP Workforce Now / Viewpoint Spectrum Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize data of workers from ADP Workforce Now to users in Viewpoint Spectrum.

Key Functions

The ADP Workforce Now / Viewpoint Spectrum integration performs the following functions:

  • Retrieves the data of workers from ADP Workforce Now and synchronizes the data to a specific company code in Viewpoint Spectrum.
    • Supports filtering the data of a specific ADP worker by their Position ID or Associate ID
    • Supports filtering workers' data based on their statuses, for creations and updates in Viewpoint Spectrum
  • Creates and updates users in Viewpoint Spectrum.
    • Creates new users in Viewpoint Spectrum when new workers are added in ADP Workforce Now
    • Updates user details in Viewpoint Spectrum when any changes are made to the existing workers in ADP Workforce Now
    • Supports configuring conditions for creating and updating users
  • Retrieves workers, and creates and updates users based on the limits set in the integration.
  • Performs full synchronization or incremental synchronization of workers based on the configuration.
  • Generates logs, reports, and analytics dashboards based on the configurations.
  • Sends notification emails with reports on each integration run based on the configuration.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now - API Central: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for configuration details.
    • Viewpoint Spectrum: Refer to the Viewpoint Spectrum Configuration Guide for configuration details.
  • CSV File Containing the Data of Viewpoint Spectrum Users: Make sure that a CSV file containing the data of users is downloaded from your Viewpoint Spectrum instance. This file is required for the integration to retrieve and match the user records with the ADP worker records.

Integration Setup

This section outlines how the integration must be configured to synchronize worker data from ADP Workforce Now to user data in Viewpoint Spectrum.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now / Viewpoint Spectrum.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Viewpoint Spectrum connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the ADP Workforce Now API Central and Viewpoint Spectrum application connectors used for the integration. This section allows you to search and update these application connectors, if required.

Parameters

The Parameters section enables you to customize the integration features and settings according to your use case. It has the following settings:

Integration Features

This section provides options to enable the user lifecycle management.

  • Enable User Lifecycle Management: Turn on this toggle to synchronize the worker data from ADP Workforce Now to users in Viewpoint Spectrum. This is a mandatory setting for this integration. When turned on, the Worker Sync Settings section is displayed for further configuration.

Execution Settings

This section provides the settings to execute the integration in Report-only mode and incremental mode.

  • Report-only mode: Turn on this toggle to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Incremental mode: Turn on this toggle to execute the integration in incremental mode, where the worker data that was created or modified since the last incremental run will be processed by the integration.
  • Enter timestamp for incremental mode: Enter the timestamp that must be considered for the first incremental run in UTC ISO 8601 format (YYYY-MM-DDThh:mm:ss.sssZ). The worker data that will be created or modified in ADP since this timestamp will be processed by the integration. For subsequent incremental runs, the timestamp of the previous incremental run will be considered.

 

General Settings

Configure the following general settings:

  • Debugging: Turn on this toggle to add the details of the API calls in the integration execution logs, for debugging purposes. The details include the API request, the parameters passed in the request, and the response code.
  • Advanced debugging: Turn on this toggle to add extensive details to the integration execution logs for debugging purposes. In addition to the API request, the parameters passed in the request, and the response code, the API response body will also be available in the logs.
  • Debug mode: Turn on this toggle to filter a single worker record, for testing purposes. When the toggle is on, the Debug Settings section is displayed for further configuration.
  • Logging: Turn on this toggle to generate logs outlining data changes. When the toggle is on, the Log Settings section is displayed for further configuration.
  • Email notifications: Turn on this toggle to enable integration-related notifications. When the toggle is on, the Notifications section is displayed for further configuration.
  • Report generation: Turn on this toggle to generate the integration report. When the toggle is on, the Report Settings section is displayed for further configuration.
  • Insights: Turn on this toggle to visualize the integration-related metrics. When the toggle is on, the Log Insights section is displayed for further configuration.

Application Configurations

This section enables you to set the configurations specific to the Viewpoint Spectrum application.

  • Company Code: Enter the company code in your Viewpoint Spectrum instance to which the worker data from ADP must be synced.
  • Viewpoint Spectrum File Upload Format: Select the format of the Viewpoint Spectrum worker data file that must be uploaded. This file is used to identify the matching user records for ADP worker records and to determine which users must be created in Viewpoint Spectrum. 
  • Upload Viewpoint Spectrum employees file: Click Browse File and select the CSV or JSON file containing the data of workers from Viewpoint Spectrum. This file is used to identify the matching user records for ADP worker records and to determine which users must be created in Viewpoint Spectrum. 
    Note: If any user data is modified directly in the Viewpoint Spectrum application through the UI, download a new CSV/JSON file containing the latest user data and upload the updated file. 

User Sync Settings

The User Sync Settings section allows you to customize the settings to manage users (employees) sync.

Note: This section appears only if you have turned on the Enable user lifecycle management toggle in the Parameters section.

User Match Settings

  • Select the match condition: Select the condition to match records between ADP and Viewpoint Spectrum.
    • Select ADP Worker Position ID > Viewpoint Spectrum Employee Code to match an ADP worker with a Viewpoint Spectrum user by comparing the ADP worker’s Position ID with the Viewpoint Spectrum employee code.
    • Select Custom match expression to define a custom condition to match the records.
  • Custom match expression: Specify the condition that must be used to identify the matching records between ADP Workforce Now and Viewpoint Spectrum.

Source Settings

This section enables you to configure the worker data filter settings and to set the worker details that must be retrieved.

  • Select the source filter: Select whether you want to use Position ID or Associate ID to filter a single worker record from ADP.
  • Source filter value: Enter the Associate ID or the Position ID to filter a worker record from ADP Workforce Now. This field is displayed only if you have selected an option from the Select the source filter drop-down list.
  • Select ADP Worker Statuses to Be Considered as Active: Select the statuses of the workers in ADP Workforce Now that must be considered as active. Only the workers with the selected statuses will be created or updated in Viewpoint Spectrum.
  • Select ADP Worker Statuses to Be Considered as Terminated: Select the statuses of the workers in ADP Workforce Now that must be considered as terminated. Only the workers with the selected statuses will be terminated in Viewpoint Spectrum.
  • Fetch Additional Data for Workers: Select the additional worker data fields that must be retrieved for active workers from ADP Workforce Now, along with the core worker data. For example, select FederalTax and StateTax to retrieve the Federal tax and State tax details of each active worker.
  • Fetch Tax Data for Terminated Workers in ADP: Turn on this toggle to retrieve the tax details of terminated workers from ADP Workforce Now and sync the details to Viewpoint Spectrum. 
  • Fetch Tax Data for Future Hire and Rehire Workers: Turn on this toggle to retrieve the tax details of future hires and rehires from ADP Workforce Now and sync the details to Viewpoint Spectrum. 

Target Settings

  • Select Viewpoint Spectrum Statuses to Be Considered as Terminated: Select the statuses of the users in Viewpoint Spectrum that must be considered as terminated. 

Safeguard Limits

This section provides options to limit the number of worker records that will be retrieved from ADP Workforce Now, and that will be created, updated, or terminated in Viewpoint Spectrum.

  • Enable import limit: Turn on this toggle to set a limit to the number of worker records that can be retrieved from ADP Workforce Now.
  • Import limit: Enter the maximum number of worker records that must be retrieved from ADP Workforce Now in a single integration run. This field is displayed only if the Enable import limit toggle is On.
  • Apply Provisioning Limit: Turn on this toggle to set a limit to the number of user records that can be created, updated, or terminated in Viewpoint Spectrum.
  • Limit scope: Select whether you want to set a single limit across creation, update, and termination operations (Common limit) or to define separate limits for each operation (Individual limits).
  • Common limit: Enter the maximum number of user records that are allowed to be created, updated, or terminated in Viewpoint Spectrum a single integration run. This field is displayed only if you have selected Common limit in the Limit scope field.
  • Create limit: Enter the maximum number of user records that are allowed to be created in Viewpoint Spectrum in a single integration run. This field is displayed only if you have selected Individual limit in the Limit scope field.
  • Update limit: Enter the maximum number of user records that are allowed to be updated in Viewpoint Spectrum in a single integration run. This field is displayed only if you have selected Individual limit in the Limit scope field.
  • Terminate limit: Enter the maximum number of user records that are allowed to be terminated in Viewpoint Spectrum in a single integration run. This field is displayed only if you have selected Individual limit in the Limit scope field.

Provisioning Settings

This section provides options to enable the provisioning operations that must be supported by the integration.

  • Enable creation: Turn on this toggle to support the creation of users in Viewpoint Spectrum. When enabled, the Create Settings section is displayed for detailed configurations.
  • Enable update: Turn on this toggle to support the update of users in Viewpoint Spectrum. When enabled, the Update Settings section is displayed for detailed configurations.
  • Enable termination: Turn on this toggle to support the termination of users in Viewpoint Spectrum. When enabled, the Terminate Settings section is displayed for detailed configurations.

Create Settings

This section provides the settings to set the conditions under which ADP workers must be created in Viewpoint Spectrum.

  • Create condition: Specify whether users should be created in Viewpoint Spectrum based on their worker status in ADP or according to a custom condition that you define.
  • Eligible worker statuses for creation: Select the ADP worker statuses that determine a user's eligibility for creation in Viewpoint Spectrum. Users must have one of the selected statuses to be created. This field is displayed only when Based on worker status is selected in the Create condition field.
  • Custom create condition: Enter a JavaScript expression that evaluates whether a user should be created in Viewpoint Spectrum. This field is displayed only when Custom is selected in the Create condition field.

Update Settings

This section provides the settings to set the conditions under which ADP workers must be updated in Viewpoint Spectrum.

  • Update condition: Specify whether users should be updated in Viewpoint Spectrum based on their worker status in ADP or according to a custom condition that you define.
  • Eligible worker statuses for update: Select the ADP worker statuses that determine a user's eligibility for update in Viewpoint Spectrum. Users must have one of the selected statuses to be updated. This field is displayed only when Based on worker status is selected in the Update condition field.
  • Custom update condition: Enter a JavaScript expression that evaluates whether a user should be updated in Viewpoint Spectrum. This field is displayed only when Custom is selected in the Update condition field.

Terminate Settings

This section provides the settings to set the conditions under which ADP workers must be terminated in Viewpoint Spectrum.

  • Terminate condition: Specify whether users should be terminated in Viewpoint Spectrum based on their worker status in ADP or according to a custom condition that you define.
  • Eligible worker statuses for termination: Select the ADP worker statuses that determine a user's eligibility for termination in Viewpoint Spectrum. Users must have one of the selected statuses to be terminated. This field is displayed only when Based on worker status is selected in the Terminate condition field.
  • Custom terminate condition: Enter a JavaScript expression that evaluates whether a user should be terminated in Viewpoint Spectrum. This field is displayed only when Custom is selected in the Terminate condition field.

Debug Settings

This section enables you to customize the settings for debugging the integration behaviour using a single worker record.

Note: This section appears only if you have turned on the Debug Mode toggle in the Parameters section. 

Source Filter

  • Select the source filter: Select whether you want to use Position ID or Associate ID to filter a single worker record from ADP.
  • Enter the source filter value: Enter the Associate ID or the Position ID to filter a worker record from ADP Workforce Now. This field is displayed only if you have selected an option from the Select the source filter drop-down list.

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Logging parameter in the Parameters section.

  •  Log Level: Select the level of details to be logged from the dropdown menu. The following are the available options:
    • Summary: Logs only the count of various operations
    • Full Details: Logs the attribute values for each create and update operation.

Report Settings

This section enables you to customize the CSV report generation settings. It has the following settings:

Note: This section appears only if you have enabled the Report generation parameter in the Parameters section.

  • Report detail level: Select the level of details to be captured in the CSV report (available in the Execution Timeline panel). The following are the available options:
    • Summary: Shows only the count of various operations
    • Full Details: Shows the attribute values for each create and update operation.
  • Enable Integration Report Download: Select this option to allow you to download a copy of the integration report from the integration execution timeline panel.

Notification Settings

This section enables you to customize the notification settings. It has the following settings:

Note: This section appears only if you have enabled the Email notifications setting in the Parameters section.

Notification Report Settings

  • Report detail level: Select the level of details to be captured in the CSV report (attached in the notification email). The following are the available options:
    • Summary: Shows only the count of various operations
    • Full Details: Shows the attribute values for each create and update operation.
  • Send the notification report as an attachment: Select this checkbox to receive an email notification with an attached CSV report for each integration execution.

Notification Template Settings

Configure the following settings to customize notifications:

  • Enable Integration Success Notifications: Turn on this toggle to support sending notification emails on successful integration runs. The Success Notifications Settings section is displayed for further configurations.
  • Enable Integration Failure Notifications: Turn on this toggle to support sending notification emails on failed integration runs. The Failure Notifications Settings section is displayed for further configurations.
  • Enable User Sync Transaction Failure Notifications: Turn on this toggle to support sending notification emails on failed user syncs during integration runs. The User Sync Transaction Failure Notifications Settings section is displayed for further configurations.

Success Notification Settings

Configure the following settings to customize success notifications:

  • Admins' email addresses: Enter the email addresses of the admins to whom the email notifications must be sent upon successful integration runs.
  • Additional recipients (Cc): Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email subject: Enter the content that must be used as the subject in email notifications.
  • Email body: Enter the content that must be used as the email body for notifications sent upon successful runs.

Failure Notification Settings

  • Additional recipients (Cc): Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email subject: Enter the content that must be used as the subject in email notifications.
  • Email body: Enter the content that must be used as the email body for notifications sent upon failed runs.

User Sync Transaction Failure Notification Settings

  • Additional recipients (Cc): Enter the email addresses of additional recipients who should receive a copy of the email notification.

You can enter JavaScript expressions on the source side of the attribute mapping to evaluate and assign a value to the corresponding target attributes.
