EntraID to Dayforce (Write-Back)

← Back to index

Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration automates the write-back of employee contact and SSO information from Microsoft Entra ID to Dayforce. It continuously tracks user profile changes in Entra ID, including updates to Business Email, Business Phone, Business Mobile, and SSO Accounts, and applies those updates to matching employee records in Dayforce. '

To identify matching user records across both systems, the integration reads one of the selected identity fields from Dayforce (such as Work Email, Person Number, or User ID). It matches it against the corresponding UserPrincipalName or Email attribute in Entra ID, as specified in the integration configuration.

Key Functions

This Entra ID to Dayforce integration keeps employee information consistent between both systems through the following main functions:

  • Monitors changes in Entra ID: Regularly checks for updates in user profiles, including Business Email, Business Phone, Business Mobile, and SSO Accounts.
  • Matches records accurately: Uses the Employee ID in Dayforce and the Employee ID (extension attribute) in Entra ID to ensure updates go to the correct employee.
  • Updates employee data in Dayforce: Automatically writes updated Entra ID details back to the corresponding Dayforce employee record.
  • Handles pre-hire employees: Saves data for future hires and syncs it automatically when the employee becomes active in Dayforce.
  • Generates detailed reports: Creates a CSV report summarizing all updated, skipped, and failed records and emails it to administrators for review.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You need an active Aquera tenant with administrator privileges to manage the integration. Register for a tenant at https://admin.aquera.io. 
    Note: If your organization operates in the EU region, use https://admin.eu.aquera.io. Contact your Aquera account manager to obtain tenant access if you do not already have it.
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal for the integration:
    • Microsoft 365: Ensure you add and configure the Microsoft 365 (Client Credentials) connector in the Aquera Admin portal. Refer to the Microsoft 365 Configuration Guide for instructions on creating the Microsoft 365 application connector and setting up the authentication.
    • Dayforce: Ensure you add and configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for instructions on creating the Dayforce application connector and setting up the authentication.

Integration Setup

The Entra ID to Dayforce integration maps the Microsoft Entra ID  and Dayforce attributes.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Entra ID to Dayforce Employee Data Sync.
    The Create New Integration pop-up window is displayed.
  5. Enter a name and description for the integration.
  6. Click Continue. The Configuration tab of the integration setup page is displayed.
  7. Click Create to add the Entra ID to the Dayforce Employee Data Sync integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

General Settings 

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize the integration-related metrics.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.

Logging Settings

Configure the following logging settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. 
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Notification Settings

Configure the following settings for email notifications:

  • Notify Admin Via Email: Select to send email notifications to all admins.
    • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications must be sent.
    • Admin Email Subject: Enter the content that must be used as the email subject for the email notifications to the admins.
    • Admin Email Notification Template: Enter the content that must be used as the email body for the admin email notifications.

Configure the following settings for error notifications:

  • Trigger Emails for Selected Errors: Choose specific error messages that should trigger email notifications. For example, selecting Login Name already exists for another user ensures that admins are alerted whenever duplicate login names are detected during the write-back process.
  • Recipients of Error Notifications: Enter the email addresses of individuals or groups who should receive error alerts. Multiple recipients can be added.
  • Error Email Subject: Define the subject line for the error notification email.
  • Error Email Template: Customize the body of the error email. This template is sent whenever a configured error is triggered.

Report Settings

Configure the following settings to customize the CSV report:

  • Enable Integration Report Download: Select to allow downloading the integration report.
  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Source Settings

Configure the following settings for the source application:

  • Entra ID Filter: Select the filter condition to limit which Entra ID users are included in the synchronization.
    • None: Processes all Entra ID users without restriction.
      Note: This field can also be customized to target specific user groups, departments, or organizational units if needed.
  • Entra ID Update Condition: Choose which Entra ID users should be evaluated for updates based on their account status.
    • All Users: Includes all Entra ID users in the update process.
    • Only Active Users: Limits updates to users who are currently active in Entra ID.
    • Custom: Allows administrators to define specific update rules or conditions.
  • Entra ID Attributes to Match Dayforce Employee Number: Select the Entra ID attribute used to locate and match the corresponding employee record in Dayforce. The integration uses Employee ID as the correlation attribute.

Notes: 

  • When multiple attributes are selected, they are prioritized in order of selection. This ensures that each Entra ID user is accurately matched to their Dayforce employee record before updates are applied.
  • Click the Refresh button to reload and display the latest list of available Entra ID attributes that can be selected for matching with the Dayforce employee number.

Target Settings

Configure the following settings for the target application:

  • Update Attributes X Days Before Employee is Hired in Dayforce: Specify the number of days before the employee’s hire date that the selected attributes should be updated in Dayforce.
    For example, if set to 10, the selected attributes will be written to Dayforce only when the set date condition is met, ensuring that updates occur exactly within the defined pre-hire window.

    Notes: 

    • Future Hire-Date Handling: If the hire date is a future date and the number of days specified in Update attributes X days before the employee is hired exceeds the current difference, and the user is updated in Entra ID, the selected attributes in the preferences should not be updated in Dayforce. Instead, those attribute values should be fetched and stored temporarily, while unselected attributes continue to update in Dayforce.

    • During the next incremental run, when the current date reaches the threshold defined in the Update attributes X days before employee is hired setting, the integration will update the selected attributes in Dayforce and then remove them from the temporary variable.

  • Dayforce Attributes to Be Updated When User Is Hired: Select the Dayforce attributes that should be updated when a user is marked as hired. For example, Business Mobile, SSO Login, Business Phone, and Business Email can be updated once the hire date is reached.
    Notes:
    •  If a user in Entra ID is updated with all four preferred attributes (Business Email, Business Phone, Business Mobile, and SSO Login) and the hire date matches the defined update threshold, those attributes will be written to Dayforce during that run.

    • The Business Phone and Business Mobile numbers must include the country code and be in a valid format; otherwise, an Invalid phone number or Invalid format error will be generated.

    • The integration also supports removal or updates of existing Business Phone and Business Mobile attributes. If these numbers are updated or removed in Entra ID, the same changes are reflected in Dayforce.

      • If the numbers are not removed immediately in Dayforce, the Effective Date is updated, and the numbers are deleted automatically once the effective period ends.

  • Dayforce Update Condition Type: Define which employee records in Dayforce should be included in the update process.
    • All Employees: Updates all eligible employee records.
    • Custom:  Select one or more conditions from Dayforce employee attributes to determine which Entra ID users should be processed. For example, you can define filters based on EmploymentStatus Codes, Department Codes, Job Codes, Pay Class Codes, Pay Group Codes, or Pay Type Codes. 

      Once Custom is selected, the following Dayforce Custom Condition fields (evaluated together using AND logic) are enabled. You can choose one or multiple conditions to apply to the Dayforce employee, and based on your selections, the corresponding options will be displayed as follows:

      • EmploymentStatus Codes: Specify the employment status codes to filter Dayforce employees. For example, Active includes only active employees, while !Prestart excludes prestart employees.

      • Department Codes: Specify the department codes to use as a filter. For example, Consulting includes employees from the Consulting department, while !HR excludes the HR department.

      • Job Codes: Specify the job codes to use as a filter. For example, Executive Assistant (CAN) includes employees with that job code, while !Intern excludes interns.

      • Pay Class Codes: Specify the pay class codes to use as a filter. For example, FT includes only full-time employees, while !PT excludes part-time employees.

      • Pay Group Codes: Specify the pay group codes to use as a filter. For example, CanadaBiWeekly includes the Canadian biweekly pay group, while !HourlyUS excludes U.S. hourly pay groups.

      • Pay Type Codes: Specify the pay type codes to use as a filter. For example, Salaried includes salaried employees, while !Overtime excludes those paid for overtime.

        Note: 

        • Each condition accepts one or more values, and specific entries can be excluded by prefixing them with an exclamation mark (!). For example, to include only active employees, enter Active. To exclude terminated employees, enter !Terminated.

        • Within each condition type (for example, multiple EmploymentStatus Codes or multiple Department Codes, and so on), the values are evaluated using OR logic. If any of the overall Custom conditions fail, the integration stops for that record and does not continue processing until all specified conditions are met.

        • The configured values for each condition must exactly match the corresponding XReference Codes defined in Dayforce.

    • JavaScript Expression: Enables advanced conditional logic using JavaScript to determine eligible updates. For example, user.department =="Integration", and user.accountEnabled ==true.

Attribute Map

The Entra ID to Dayforce integration maps the Entra ID user attributes to the corresponding Dayforce employee attributes. Each mapping specifies a source attribute from Entra ID and a corresponding target attribute in Dayforce. The integration retrieves user data from Entra ID and updates the linked employee record in Dayforce based on these mappings.

The Refresh icon next to the mapping section can be used to reload the latest list of attributes from both applications, ensuring that any newly added or renamed attributes become available for selection.

When the integration runs, it compares the current Entra ID values for each mapped attribute against the existing data in Dayforce. If a difference is detected, the corresponding field in Dayforce is updated.
