User Integration Setup Guide - Dayforce to Active Directory

← Back to index

Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating the synchronization of users from Dayforce to the users in Active Directory.

This guide covers the end-to-end tasks in setting up synchronization to synchronize Dayforce users to Active Directory users. 

Use Case

Aquera provides a real-time, non-persistent integration service between Dayforce and Active Directory. It is important to note that the Aquera integration running inside of the Aquera Platform uses a service account whose permissions are 100% managed separately by the Dayforce Admin for permissions to Dayforce, and the Active Directory Admin for permissions to AD. The setup of what fields can be read and written in Dayforce by the Service Account used by the integrations in the Aquera Platform is done solely by the Dayforce Admin. The same is the case on the AD side.

User Roles

  • The Dayforce Admin can be any HR administrator who controls the API access permissions of each employee attribute in Dayforce and determines what attribute can be read and/or written.
  • The Aquera Admin configures the integration and may not have access to AD or Dayforce environment.
  • The AD Admin can be any Active Directory administrator who controls the LDAP access permissions on specific OUs for the service account that is used to manage users in AD.

Prerequisites

Aquera Tenant Account

Tenant account in the Aquera Admin portal: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.

Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/

Dayforce

The following are the prerequisites for Dayforce:

  • Service Account with Appropriate Roles: A service account must be created with access to the Web Services feature. The username and password of this account are required for authentication.
  • Feature-Level and Field-Level Access Permissions: Appropriate feature-level and web services field-level access permissions must be enabled in Dayforce. 

For details, refer to the Dayforce Configuration Guide.

Active Directory

Determine the architecture to be followed for setting up communication between Aquera and the Windows Active Directory Domain Controller. Refer to this guide for detailed information on the different architectural approaches.

Based on the selected approach, the following procedures may have to be performed:

  • A Windows Server must be ready to deploy Aquera Agent onto the server where the Domain Controller is present. For instructions on installing Aquera Agent, refer to the Installation and Configuration Guide.
  • A firewall must be set up to allow communication with Aquera.
  • The static IP address of Aquera needs to be whitelisted. Contact the Aquera Support team.

To allow Aquera to update the Active Directory (for creating, updating, and deleting users), LDAPS communication to the Active Directory needs to be set up, which requires a TLS connection. Active Directory Certificate Services must be enabled for this. To install Active Directory Certificate Services, visit the Microsoft website.

Configure Applications in the Aquera Admin Portal

Aquera provides SCIM Gateway connectors to support user lifecycle management. Aquera's connectors use the SCIM protocol to perform the read and write operations on the applications. 

You must add and configure the following connectors in the Aquera Admin portal:

  • Dayforce: Refer to the Dayforce Configuration Guide for details.
  • Active Directory: Refer to the Active Directory (AD) Configuration Guide for details.

User Synchronization Setup

Aquera's user synchronization automatically imports users from Dayforce and provisions these users in Active Directory.

Synchronization Setup

Perform the following steps to create a user synchronization:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations on the left panel.
  3. Click Create Synchronization.
  4. Select User Synchronization.
  5. Enter a name and a description for the synchronization.
  6. Click Continue. A new page with the fields and tabs to configure the user synchronization is displayed.
  7. Select a Dayforce connector in the Source Application dropdown field.
  8. Select an Active Directory connector in the Target Application dropdown field.
  9. You can update the configurations as required. For further details, refer to the User Synchronization Setup Guide.
  10. Click Create to create the user synchronization.

Synchronization Configuration

You can set the following configurations in the synchronization:

  • Enter the matching expression to match a user record from Ceridian Dayforce with that in the Active Directory.
  • Configure the fields in the Admin Email Options to receive email notifications for success or failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location and department values from the source.
  • Enable Group Management if Aquera has to manage the groups and group assignment rules based on the group analysis. 

Refer to the User Synchronization Setup Guide for further details on the configurations.

Note: Any custom fields added in Active Directory are not displayed automatically on the Attribute Map page. You must manually add those custom attributes using the Add Row to Map option on the Attribute Map page and then complete the mapping.

Considerations

Consider defining the following when setting up the synchronization:

  • Define a naming convention for the User Principal Name (UPN) in the Active Directory.
  • Define a naming convention for the SAM Account Name in the Active Directory.
  • Define the logical OU mapping rules for user containers.

General Analysis

You can use JavaScript expressions in the mapping to create UPNs and SAM Account Names without duplicates or errors. In the expressions, you can use Aquera's inbuilt methods such as toNormalizedDiacritics() and toSAMAccountName() for this purpose. You can also use keywords such as %ATTEMPT or %UTTEMPT to avoid duplicates. For details, refer to the Synchronization Expressions - Keywords, Methods, and Functions guide.

For a detailed analysis of Ceridian Dayforce users and the users/groups in Active Directory using scripts, refer to the Appendix.

You can also analyze the user's data using the Reports option in the Data tab of the Applications module in the Aquera Admin portal. Refer to the Applications Data Guide for details.

Synchronization Scenarios

The following are the common synchronization scenarios:

Prompting for Password Change at Login

To make it mandatory for the new users in Active Directory to change their password on their first login, map the Active Directory attribute pwdLastSet to 0 with the mapping option as Map on create. If this mapping is not done and users sign into their account for the first time, they will not be prompted for password change.

To prompt the existing users also for a password reset, set the mapping option as Map on Create and Update.

Requirements to Set the User Password

Active Directory requires a secure LDAPS connection (TCP port 636) to set or change user passwords. This is an Active Directory’s security requirement. Random generation of passwords when creating new users does not work without the use of LDAPS. Also, setting the password through attribute mapping requires a secure LDAPS connection.

Note: Active Directory imposes complexity requirements on the password. Ensure the user's password follows the password guidelines as mentioned in the Microsoft Documentation.

Requirements to Set the User Password to Never Expire

To set the user's password to never expire, you must set the IAN.userAccountControl attribute to the value 66048 for an active user and 66050 for an inactive user.

Running the User Synchronization

Click the Execute button at the top right of the user synchronization to manually run the user synchronization. You can also schedule the user synchronization to run automatically at specified intervals by configuring a frequency in the Schedule tab.

You can navigate to Logs > Synchronizations in the Aquera Admin portal to view the logs of the synchronization execution.

Perform the following steps to ensure that the users are updated correctly:

  • Run the synchronization in Report Only mode and determine if the correct data is updated in Active Directory. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data, for example, employeeNumber, in Active Directory.
  • Increment the Update Limit for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Based on your analysis and considerations,
    • Translate the logical maps into synchronization attribute mapping
    • Translate the logical OU/container locations into synchronization tables and expressions
    • Translate the logical group assignment rules into synchronization group rules
  • Modify the attribute mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. Modify the synchronization configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in Dayforce.

After the user synchronization is successfully executed, new users will be created and existing users will be updated in Active Directory.

Appendix

This section provides sample scripts to analyze the existing users in Dayforce and Active Directory before setting up the synchronization. It also provides the script to analyze the existing groups of users in Active Directory. The scripts can be modified based on your use case and the attributes to be retrieved for analysis.

Execute these scripts in a script synchronization. In the script synchronization, add Dayforce and Active Directory applications and set the corresponding variable names. The variable names must be used in the script to refer to the applications. For details on configuring the script synchronization, refer to the Synchronization Setup Using Scripts guide.

Analyzing the Users in Dayforce 

Use the following script in a script synchronization for generating a report from Dayforce for the analysis of users:

log.info('DAYFORCE ID, DAYFORCE ACTIVE, DAYFORCE GIVENNAME, DAYFORCE FAMILYNAME, DAYFORCE USERNAME, DAYFORCE MIDDLENAME, DAYFORCE TITLE, DAYFORCE DEPARTMENT, DAYFORCE DIVISION, DAYFORCE USERTYPE');
let users = df.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}

Analyzing the Users in Active Directory

Use the following script in a script synchronization for generating a report from Active Directory for the analysis of users:

log.info('AD ID,AD ACTIVE,AD EMPNO,AD GIVENNAME,AD FAMILYNAME,AD USERNAME,AD SAMACCOUNT,AD DN');
let users = AD.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user[IAN].sAMAccountName + '","'
+ user[IAN].distinguishedName.split(',').slice(1).join(',') + '"');
}

Analyzing the Groups in Active Directory

Use the following script in a script synchronization to analyze the groups of existing users in Active Directory:

let ad_users = AD.importAllUsers();
let ad_users_length = ad_users.length;
let i = 0;
let total = ad_users_length;
log.info('AD ID,GROUP NAME');
for (i = 0; i < total; i++) {
let groups = ad_users[i].groups;
if (groups) {
for (let group of groups) {
log.info(ad_users[i].userName + ',' + group.display);
}
}
