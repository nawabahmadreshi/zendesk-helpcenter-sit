Integration Logging Modes

← Back to index

Overview

When an integration runs, the logs are saved and can be viewed through the Aquera Admin portal by navigating to Logs > Integration. The log varies based on the logging mode set for the integration.

This guide provides details about the logging modes supported in integrations.

Following is the procedure to view the logs of the recent execution of an integration:

  1. Navigate to Integrations on the left panel of the Aquera Admin portal.
  2. Click the integration record for which you want to view the recent logs.
  3. Click the ellipsis at the top right corner of the integration page.

  4. Click Latest Logs. A Log pane is displayed on the same integration page, which contains the log details of the most recent execution of the current integration. You can click Goto Logs at the top right corner of the Log pane to navigate to the Logs module.

Report Mode

The Report Mode setting is available in the Configuration tab of the integration setup page to set the logging mode of the integration.
Report Mode allows you to set the level of logging to be captured in the integration logs. The following are the valid values listed in the order starting with the least to the most detailed level of logging: 

  • minimal
  • normal
  • debug
  • verbose

Use minimal or normal mode to get the basic integration details. Use debug or verbose mode to get a detailed analysis of the integration.

Logging when Report Only is enabled

The Report Only setting is available in the Configuration tab of the integration setup page. Select this checkbox to generate a report that shows the data that would be changed, without actually making those changes.

Report Only can be enabled for the first instance of the integration run to identify any issues. You can clear this selection after verifying the report and confirming that the result is as expected.

The following table shows the details captured in the logs for each of the logging modes when the Report Only mode is enabled:

Report Mode Log Details
minimal Displays the user names in the source and the target applications.  
normal Displays the list of user attributes that are configured in the Attribute Map of the integration and the data values of those attributes that are passed from the source to the target application.
debug

Displays the response string that is received from the source.

From User Integration V4 onwards, when the Map Manager checkbox is selected in the integration configuration, the manager's ID and userName attribute values are included in the logs.
verbose Displays the list of user attributes that are configured in the Attribute Map of the integration and the data values of those attributes that are passed from the source to the target application. For user update operations, the logs will show the difference in data between the source and the target applications.

Logging during Incremental Imports

The Incremental Mode setting is available in the Configuration tab of the integration setup page. Select this checkbox to run the integration in incremental import.


When Incremental Mode is enabled, the start timestamp of the last run integration is always captured, and the source records that have been added or modified after this timestamp will only be considered for the next integration.

The following table shows the details captured in the logs for each of the logging modes for incremental run:

Report Mode Log Details
minimal Displays the attribute names of the users that were created during the integration. It does not show the data.
normal Displays all the attribute names and the corresponding data of the users that were created during the integration.
debug

Displays all the attribute names and the corresponding data of the users that were created during the integration.

From User Integration V4 onwards, when the Map Manager checkbox is selected in the integration configuration, the manager's ID and userName attribute values are included in the logs.
verbose

Displays all the attribute names and the corresponding data of the users that were created during the integration.

Logging during Full Imports

When Incremental Mode is disabled and Report Only is also disabled in the Configuration tab of the integration setup page, integration runs in Full Import mode.

The following table shows the details captured in the logs for each of the logging modes when a full import is run and the Report Only mode is disabled:

Report Mode Log Details
minimal Displays the username of the users that were created during the integration.
normal Displays the username, first name, and last name of the users that were created during the integration.
debug

Displays the Details of users that were not changed and the entire response string of the users that were created/updated during the integration.

From User Integration V4 onwards, the logs also display the following:

  • For expression failures, the logs mention the failure reason.
  • When the expressions include the keywords %ATTEMPT, %UATTEMPT, or %USEQ, the logs mention the target attribute to which the expression is mapped.
  • The logs mention the attempt value after evaluating the attempt logic.
  • When the Map Manager checkbox is selected in the integration configuration, the manager's ID and userName attribute values are included in the logs.
verbose

Displays the following details:

  • Details of users that were not changed. 
  • The username, first name, and last name of the users that were created during the integration.
  • The request and response strings of the users that were updated during the integration.
