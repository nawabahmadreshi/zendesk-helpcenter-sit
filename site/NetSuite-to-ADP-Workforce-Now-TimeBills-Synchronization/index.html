Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process for setting up an integration to sync time bills from NetSuite with timecards from ADP Workforce Now. The approved TimeBills from NetSuite are transformed and sent to ADP Workforce Now

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Netsuite: Refer to the Netsuite Configuration Guide for details.
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.

Integration Setup

This section outlines the configuration required to synchronize the approved TimeBills from NetSuite with ADP Workforce Now. It covers creating the integration, configuring parameters, and scheduling it to run. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select NetSuite to ADP Workforce Now TimeBills Synchronization.
  5. Enter a name and a description for the integration.
  6. Select your NetSuite and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following tabs are displayed on the Configuration page:

Applications

The Applications tab displays the Applications section, which displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following sections:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Notifications: Select this checkbox to enable notifications related to the integration.
  • Enable Analytics: Select this checkbox to analyze integration-related metrics in a dashboard.

Logging Settings

Configure the following logging settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes.
      Note: The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: controls which event attributes are included or excluded.   
        • Included Attributes: Include only selected attributes in the log.
        • Excluded Attributes: Include all attributes except the selected ones in the log.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details option.
  • Enable Debug Mode: Select this checkbox to generate logs that outline data changes, enabling debugging.

Report Settings

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:

    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

    • Full Details: All attribute details are displayed.

    • Custom: Includes or excludes selected attributes. The following options only appear when the report detail level is set to Custom.

      • Attribute Filter: Controls which attributes are included or excluded in the report.

        • Included Attributes: Select the attributes to include in the CSV report.

        • Excluded Attributes: Select the attributes to exclude from the CSV report.

  •  
  • Enable Integration Report Download: Select this checkbox to allow downloading the integration report in CSV format.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details option.

Notification Settings

Configure the following settings for email notifications:

  • Recipients' Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
  • Email Subject: Enter the content that must be used as the email subject.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
  • Report Employees Without Projects: Generates a report of employees who have no projects assigned.

Log Insights Settings

Configure the following log insight settings:

  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard.
    The following are the available options:

    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

    • Full Details: All attribute details are displayed.

    • Custom: Includes or excludes selected attributes. The following option only appears when the log insights level is set to Custom.

      • Attribute Filter: Controls which attribute details are included or excluded in the analysis.

        • Included Attributes: Select the attributes to include in the analysis.

        • Excluded Attributes: Select the attributes to exclude from the analysis.

Source Settings

Configure the following source application settings:

  • Fetch TimeBills From NetSuite Based on X Days: Enter the number of days before the current date to retrieve the TimeBills from NetSuite.

  • Sync TimeBills for Employee ID: Enter a NetSuite employee ID to sync TimeBills only for that employee.

  • Source Time Entries Filter Condition: Select the source filter to filter the employees' time entries while retrieving from NetSuite.

  • Select the NetSuite Correlation Attribute: Select the NetSuite attribute to correlate and match employee records between the source and target applications.

  • NetSuite Custom Field for Position ID: Enter the NetSuite custom field name that stores the employee’s position ID.

  • Select Time Type from NetSuite: Select the time type for the employees' time entries while retrieving from NetSuite.

Target Settings

Configure the following target application settings:

  • Create Labor Codes in ADP Workforce Now: Select to create Labor Charge Fields (LCFs) in ADP Workforce Now based on the employees' project ID and serial number.
  • Select the ADP Workforce Now Correlation Attribute: Select the ADP Workforce Now attribute to correlate and match employee records across source and target applications.

Attribute Map

The Attribute Map tab in NetSuite to ADP Workforce Now TimeBills Synchronization defines the NetSuite attributes that are mapped to the ADP Workforce Now attributes for syncing TimeBills from NetSuite with time cards in ADP Workforce Now. Only the source attributes can be modified, as needed, to align with your specific use case.

Any entries with missing employee ID, date, or hours are skipped and logged. The project and serial codes (LCFs) are checked in ADP Workforce Now, and if they are missing, they are automatically created. An email is sent upon success, failure, or simulation, as specified in the configuration. The email will include the CSV report and summary, as configured.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide empty or specific data.

The following table is available:

Time Entry Constant Values: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options:

  • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
  • Upload CSV: Click Upload CSV to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
    Notes:
    • The key name within a table should be unique. The key name can include a combination of letters and numbers.
    • The key name supports all the special characters except the following:
      • / (forward slash)
      • \ (backslash)
      • " " (double quotes)

Click Save to save the changes made.
