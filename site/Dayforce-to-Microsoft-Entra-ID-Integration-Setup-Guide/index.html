Dayforce to Microsoft Entra ID Integration Setup Guide

← Back to index

Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating the synchronization of employees' data from Dayforce to Microsoft Entra ID.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, every employee newly created or modified in Dayforce is updated in Entra ID according to the schedule.

This guide covers details on how to set up the HR to Entra ID integration to synchronize employee data from Dayforce to Entra ID. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
     
  • Configure Applications in the Aquera Admin Portal: 
     
    • Dayforce Connector: Ensure you add and configure the Dayforce connector version 26 or a higher version in the Aquera Admin portal. For details, refer to the Dayforce Configuration Guide and Dayforce - Attribute's Screen Name and API Name guide.
       
    • Microsoft Entra Native Connector: Ensure you add and configure the Entra ID connector in the Aquera Admin portal. Refer to the Microsoft Entra Native Configuration Guide for details.

Integration Setup

Aquera's Dayforce to Microsoft Entra ID integration automatically synchronizes data from Dayforce to Entra ID.

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Dayforce to Microsoft Entra ID. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration tab of the integration setup page is displayed. The configurations are categorized under various sections within the Configuration tab.
    • Connection - Select Dayforce connector version 28 as the Source Application and Microsoft Entra Native connector as the Target Application.
  7. Click Create to add the Dayforce to Entra ID integration.
  8. Set the configurations as per your requirements. Refer to the Integration Configurations Reference Guide for details.

Configuration

The Configuration tab includes a list of configurations required to set up the integration. For detailed information on individual configuration fields, refer to the Integration Configurations Reference Guide.

Mapping and Assignments

The Mapping and Assignments tab consolidates the configuration options for setting the user naming attributes and mapping attributes between Dayforce and Entra ID. 

User Naming Attributes

In Entra ID, the user naming attributes are User Principal Name (UPN) and Work Mail. These attributes are used to identify user objects in Entra ID.

  • UPN: The User Principal Name (UPN) attribute is the logon name of a user for logging on to a domain. A UPN consists of a prefix (the user account name) and a suffix (the domain name). The format of UPN is username@DomainName. For example, barbarajensen@example.com
     
  • Mail: The Mail attribute is the user's email address, often set to match the UPN. The mail should be unique within the same domain. The mail attribute typically follows standard email formatting. For example, barbarajensen@example.com

Aquera's Dayforce to Microsoft Entra ID integration enables you to generate unique names for these attributes while creating a user in Entra ID. The following are the available configurations:

  • Suffix: Configure the domain name for UPN and Work Mail.
    • UPN Suffix: Configure the domain name that must be used as the suffix for UPN. You can configure a static domain name as the UPN suffix ( for example, aquera.com) or define conditions to assign a specific UPN suffix.

      To define conditions, click Conditional UPN Suffix and set conditional logic to evaluate attribute values and assign domain names to users accordingly. You can click Add Condition to add more conditions.

      Advanced Expression mode

      If you prefer to write detailed conditional expressions for assigning UPN suffix, click Switch to Advanced Expression to switch to the Advanced mode.


      You need to write conditional expressions in the script editor area and click Add. When you write expressions, the API attribute names have to be used instead of the user-friendly names (Dayforce UI names) that you select in the Basic mode.
      Note: A return value has to be included when you write expressions in the Advanced mode, to assign the appropriate UPN suffix

    • Work Mail Suffix: Configure the domain name that must be used as the suffix in the Work Mail. For example, aquera1.com. If you want the same UPN suffix to be used as the Work Mail suffix, select Same as UPN Suffix.
      To assign domain names based on some conditions, you can click Conditional Work Mail Suffix to define rules based on attribute values and expressions. The procedure is similar to configuring conditions for the UPN suffix.
  • User Naming Formats: Define the formats to be used to generate values for the UPN and Work Mail attributes of a newly created user in Entra ID.
    • Basic Mode: The Basic mode is enabled by default. In this mode, you must select the naming format that must be used for generating the UPN and Work Mail values for a new user in Entra ID. The selected format will be used to generate unique usernames for both user attributes.
      In the basic mode, the UPN and Work Mail attributes are mutually dependent and are evaluated together. This means the integration will verify whether the values generated using the selected format are available for both attributes before assigning them to the new user. If a value is already in use for an attribute (that means, it is already assigned to a user in Entra ID), then the integration will append a number (starting from 1) to the attribute values and check for the availability. This process continues for up to 20 attempts or until unique values are generated for both attributes. Once unique values are found for the attributes, those values will be assigned to the new user.

      Note: The values of the expressions in the basic mode dropdown list are converted into normalized characters. This means that any characters with diacritics are substituted with the nearest equivalent non-diacritic format of those characters.
       

    • Advanced Mode: Enable the Advanced toggle button to switch to the Advanced mode. In this mode, you can set different formats for the UPN and Work Mail attributes using expressions. 


      When you switch to the Advanced mode, you can select whether the Work Mail should be mutually dependent with the UPN.

      You can write JavaScript expressions for UPN and Work Mail attributes to generate unique names for Entra ID users. To access Dayforce attributes in the expressions editor field, you must specify SOURCE.<Attribute Name>. You can click Add Format to add multiple formats. You can include a maximum of 20 formats. Additionally, the Advanced mode allows you to configure separate naming formats for user creation and update operations.

      • If you set UPN and Work Mail as mutually dependent attributes, the integration will check for a unique value in the first format for both attributes. If a unique value is unavailable (already assigned to another user), it will proceed to the next format in sequence for both attributes. The process continues until it finds a format that evaluates to unique user naming values for both attributes. If the integration cannot find unique values across the configured formats, the integration will throw a failure.

      • If UPN and Work Mail are not set as mutually dependent attributes, the integration evaluates each attribute separately, that is, if the integration finds a unique value for even one of the attributes as defined by a format, that value will be assigned to the user. The integration proceeds through the list of formats configured for the other attribute until it finds a unique value for that attribute as well.

        The following is a scenario explaining how the unique names are generated:

        Consider that you have selected UPN and Work Mail as mutually dependent.

        Suppose you have given the following JavaScript expressions for UPN and Work Mail. When a new user with the name Barbara Jane Jensen is to be created in Entra ID, the integration evaluates the JavaScript expressions to assign values for the UPN and Mail attributes of the user.

        UPN:

        Format 1: SOURCE.FirstName + "." + SOURCE.LastName
        Format 2: SOURCE.FirstName + "." + SOURCE.LastName + 1
        Format 3: SOURCE.FirstName + "." + SOURCE.LastName + 2

        Work Mail:
        Format 1: SOURCE.FirstName + "." + SOURCE.LastName
        Format 2: SOURCE.FirstName + "." + SOURCE.LastName + 1
        Format 3: SOURCE.FirstName + "." + SOURCE.LastName + 2
         

        The following are the values generated from the defined expressions for the new user, Barbara Jane Jensen:

        UPN:
        Format 1: Barbara.Jensen
        Format 2: Barbara.Jensen1
        Format 3: Barbara.Jensen2

        Work Mail:

        Format 1: Barbara.Jensen
        Format 2: Barbara.Jensen1
        Format 3: Barbara.Jensen2

         

        Case 1: For format 1, if the UPN and Work Mail values are not assigned to any user, then this format will be used for the new user. In this case, the UPN value of Barbara.Jensen and the Work Mail value of Barbara.Jensen will be assigned to the new user Barbara Jane Jensen.

        Case 2: For format 1, if the Work Mail value is already assigned to an existing user, the integration checks for the next format (which is format 2) for both attributes. This means both UPN and Work Mail values for a defined format sequence must be available to be assigned for a new user creation. For format 2, if the UPN and Work Mail values are not already assigned to any user, then this format will be used for the new user creation. In this case, the UPN value of Barbara.Jensen1 and the Work Mail value of Barbara.Jensen1 will be assigned to the new user Barbara Jane Jensen.

         

Preview Values of User Naming Attributes

You can preview the values of the user naming attributes that will be generated for a new Entra ID user based on the user naming formats and expressions set on the page.

To preview the attribute values, perform the following steps:

  1. Click the Enter objects for preview field at the top of the User Naming Attribute page. A pop-up window is displayed.
  2. Enter the Dayforce employee number in the search box, choose Map on Create or Map on Update, and then click Search.
    • When you select the Map on Create option, you will be able to preview the user naming attribute values of the user that will be created in Entra ID.
    • When you select the Map on Update option, you will see the target search box also. You can enter the matching employee number in the target system if you want to preview the target (Entra ID) user details.

      Note: While using Map on Update, you can choose to leave the target search box blank based on your use case.
  3. [Optional] If you have configured multiple formats for UPN and Work Mail, you can preview the value generated by a particular format by providing the sequence number of the format in the %Attempt field under Advanced Options. For example, if you want to preview the value of the naming format 3 for a user, enter 3 in this field.
  4. Enable the toggle option next to the preview setup field. A preview of the user naming values is displayed based on the first format configured for the user naming attributes.

    Note: You can disable the toggle option to hide the preview.

Attribute Map

The Attribute Map page displays the default mapping from Dayforce Employee attributes to Entra ID user attributes. For details on the Dayforce attributes, refer to the Appendix.

Note: The values of the default mapping are converted into normalized characters. This means that any characters with diacritics are substituted with the nearest equivalent non-diacritic format of those characters.

Perform the following steps to add a new mapping from a Dayforce attribute to an Entra ID attribute:

  1. Click Add Row to Map at the end of the attribute mapping list.
  2. Click Enter attribute / expression on the source side to include a Dayforce attribute.
  3. Type SOURCE. to view all Dayforce attributes and select an attribute from the dropdown list, then click the check mark to add the source attribute. The attributes in the source application must be specified as SOURCE.<attributeName>. For example, SOURCE.active
    You can also map a sub-attribute of a Dayforce array to an Entra ID user attribute in the attribute mapping. Select a Dayforce attribute of the type array of objects in the Enter attribute / expression field, specify the index [n], and type a dot, it will list all the sub-attributes of that attribute array. For example, SOURCE.WorkAssignments[n].Position.XRefCode
    Replace n with the actual index.
  4. In the Attribute field on the target side, select the Entra ID attribute name from the drop-down list.
  5. Select the data type of the Entra ID attribute from the dropdown list.
  6. Select one of the following mapping options:
    • Map on Create and Update: The Entra ID attribute will be updated during the create and update operations.
    • Map on Create: The Entra ID attribute will be updated only during the create operation.
    • Map on Update: The Entra ID attribute will be updated only during the update operation.
    • Don't Map: The Entra ID attribute will never be updated and the mapping will be ignored during the integration runs.
  7. Finally, click the check mark to add the new mapping to the attribute map.

Preview Employee Data

You can preview the attribute values of a Dayforce employee that will be created or updated in Entra ID based on your mappings.

To preview the attribute values of an employee, perform the following steps:

  1. Click the Enter objects for preview field at the top of the Attribute Map page. A pop-up window is displayed.
  2. Enter the employee number (ID of the employee) in the source search box, choose Map on Create or Map on Update, and then click Search.
    • When you select the Map on Create option, you will be able to preview the data of the Dayforce employee attributes that will be created in Entra ID.
    • When you select the Map on Update option, you will see the target search box also. You can enter the matching employee ID in the target system if you want to preview the target (Entra ID) user details. Refer to the Preview Target (Entra ID) User Attributes section for details.

      Note: It is mandatory to enter the employee ID of the Dayforce application while using Map on Update. However, you can choose to leave the target search box blank based on your use case.
  3. Enable the toggle option next to the preview setup field. The employee data are displayed corresponding to each mapping provided on the Attribute Mapper.

    Note: You can disable the toggle option to hide the preview.

Preview Target (Entra ID) User Attributes

You can enter expressions with the DEST keyword in the Enter attribute/Expression field to view the target attribute values. 

  1. To preview the attribute values of a user from the target application, click the Enter objects for preview search box, and choose Map on Update.
  2. Enter the Dayforce employee ID in the source search box and the corresponding target user ID in the target search box, and click the Search button. The preview will be displayed for the target attributes.

Note: It is mandatory to configure the source search box even if you want to preview the target system values.

Notes:

  • When the attributes or expressions provided in the SOURCE ATTRIBUTE/EXPRESSION field are invalid or do not return a value, the preview will not appear.
  • When the attributes or expressions provided in the SOURCE ATTRIBUTE/EXPRESSION field contain diacritics (such as teléfono), the preview will not accurately show the diacritic signs.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or particular data is not sent from the Dayforce application, and you want to update a value for that attribute in Entra ID. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integration guide for details.

Group Management

The Group Management tab provides options to configure how group memberships must be handled through this integration. You can determine the group assignment, retention, and removal of users based on the update or deactivation of users in Dayforce.

Note: You can also set the manage group memberships via Attribute Map (under Mapping and Assignments) where you can write JavaScript expressions in the mapping, which will return an array of groups.

Select Enable Group Management to set the options for managing user groups. If this checkbox is not selected, the integration does not manage group assignments to users in Entra ID.

When you select Enable Group Management, the Group Rules section will be displayed on this tab. Refer to the following section for details.

Set Up Group Rules

You can set rules for assigning users to the Entra ID groups through either the Group Rules section of the Group Management tab or the Attribute Map section of the Mapping and Assignments tab. 
Note: When you use the Attribute Map functionality to set the group rules, you have to write JavaScript conditional expressions to assign the groups. 

In the Group Rules section, you can create conditions that control how users are added to groups. Each rule defines the conditions for assigning users to a set of groups.

Perform the following procedure to create group assignment rules:

  1. Click Add New Rule to create a new group assignment rule. The following fields are displayed.
    • Choose Attribute: Click on this label to view the list of Dayforce employee attributes.
      • Search for the attribute using which you want to set a condition. 
      • Select the required attribute from the list and configure the attribute value to form the rule condition. Only those users who meet this condition will satisfy this group rule. If multiple values are provided, an OR condition is applied among those values. For example, in the following sample condition, a user's Pay Policy XRefCode must be either USA or India for them to satisfy this group assignment rule.
    • Assign To: Select the group(s) to which the users who satisfy the specified condition must be assigned. The list shows all the groups from the target Entra ID application.
  2. Click Add Row to add another condition and group assignment for the same Dayforce attribute. 

    Note 1: At a time, only two rows will be displayed on the page. If you add multiple rows of conditions, use the navigation buttons to navigate to other rows.

    Note 2: You can delete a row by using the delete icon at the right end of the row.
  3. [Optional] If you want to write a JavaScript expression to set the condition, click Switch to Expression.
    Note: When you switch to the Expression mode or to the Basic Condition mode, all the configured conditions of the rule will be cleared.
    • Click Continue to configure the expression.
    • Expressions can use all the employee and user attributes in the source and destination applications. The configured expressions must always be evaluated to true or false.
      SOURCE must be used in an expression to represent an attribute in the source application and DEST to represent an attribute in the destination application. For example, SOURCE.EmployeeNumber, SOURCE.HomeOrganization.ShortName, DEST.employeeType, and so on.  
  4. Click Add New Rule to add another group assignment rule, as explained above.

Set Up Group Retention Policy

The Group Retention section allows you to define how a user's group memberships must be handled when the Dayforce user is updated or deactivated.

On Deactivate

This drop-down list is used to specify what should happen to a user's group memberships when the Dayforce user is deactivated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the deactivated user satisfies some of the group assignment rules and they are not already part of those groups, they will be added to those groups; if they are already part of those groups, they will be retained in those groups.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained, even if the user does not satisfy the rules configured for the group assignments in the Group Rules section.
  • Retain in selected groups: This option has to be used when you want to retain the deactivated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained. Deactivated users will be removed from all other groups.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of users from the deactivated list of users. The group membership will be retained only for the users matching the filter condition. For the remaining set of deactivated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

On Update

This drop-down list is used to specify what should happen to a user's group memberships when the user is updated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the user satisfies the conditions defined in the Group Rules section, they may be added to new groups based on the configured group rules.
    Note: If the Update Deactivated Users configuration is enabled, a deleted or deactivated user will be added to a new group based on the group rules. However, they will not be added back to an existing group from which they have been removed during this integration run.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained. Even if the user does not satisfy a rule configured for the group assignments in the Group Rules section, they will be retained in those groups.
  • Retain in selected groups: This option has to be used when you want to retain the updated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained on user update.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of user records that were updated. The group membership will be retained only for the users matching the filter condition. For the remaining set of updated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

Limitations

  • You can save the integration even without mandatory fields configured, but the integration runs will fail.
  • It is required to download a report of all employees in the Dayforce connector (through the Data tab) to view all the Dayforce attributes in this integration.
  • When the integration creates guest users, the UPN value is automatically generated by the Entra ID in the format, emailaddress#EXT#@tenant.onmicrosoft.com. The UPN naming format configured in the integration will be ignored for guest users.
  • The Incremental Tuning configuration is designed to function only when the match expression only option is selected, as this integration does not support incremental tuning with userName.
  • Debug logging for a full run with a large number of source records can exceed back-end limits, resulting in reports that are too big to save. This will be fixed in a future release. 

  • In the User Naming Attributes tab, the Advanced mode expression does not support if-else statements. To write conditional logic in these expressions, use a conditional or ternary operator with the following syntax:

    {condition}? {expressionIfTrue} : {expressionIfFalse}
