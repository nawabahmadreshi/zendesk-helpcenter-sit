Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

Key Functions

The Entra ID to Microsoft Dynamics 365 Finance and Operations Integration performs the following functions:

  • Synchronizes user records from Entra ID to Microsoft Dynamics 365 Finance and Operations.
  • Creates new HR workers and system users in Microsoft Dynamics 365 Finance and Operations when they are added in Entra ID.
  • Updates existing HR worker and system user records in Microsoft Dynamics 365 Finance and Operations when the corresponding user information changes in Entra ID.
  • Terminates the HR worker and system user records in Microsoft Dynamics 365 Finance and Operations when the corresponding user records are deleted from the Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Entra ID: Refer to the Entra ID Configuration Guide for configuration details.
    • Microsoft Dynamics 365 Finance and Operations: Refer to the Microsoft Dynamics 365 Finance and Operations Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Entra ID to Microsoft Dynamics 365 Finance and Operations.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID / Microsoft Dynamics 365 Finance and Operations.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and Microsoft Dynamics 365 Finance and Operations connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable HR Workers Lifecycle Management: Select this checkbox to synchronize employees (HR workers) data from Entra ID to Microsoft Dynamics 365 Finance and Operations. When enabled, the Manage HR Employees section is displayed to configure further integration settings.
  • Enable System Users Lifecycle Management: Select this checkbox to synchronize the user's (System Users) data from Entra ID to Microsoft Dynamics 365 Finance and Operations. When enabled, the Manage System Users section is displayed to configure further integration settings.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Report Generation: Select to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Notifications: Select to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Log Insights: Select to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Synchronization Limit: Select to limit the synchronization data from Entra ID to Microsoft Dynamics 365 Finance and Operations. When enabled, the Provisioning Threshold section is displayed for further configuration.
  • Enable Incremental Mode: Select to run the integration in incremental mode.

HR Workers Management

The Employee Management section enables you to customize the integration settings for managing the employees according to your specific use cases.

Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Entra ID Correlation Attribute*: Select the Entra ID attribute to distinctly find the associated Microsoft Dynamics 365 Finance and Operations employee record. For example, Employee ID.
  • Microsoft Dynamics 365 Finance and Operations Correlation Attribute*: Select the Microsoft Dynamics 365 Finance and Operations attribute to distinctly find the associated Entra ID employee record. For example, PersonnelNumber.

Entra ID Filter Settings

Configure the following settings:

  • Enable User Filter: Select this checkbox to filter a specific user record from Entra ID. When enabled, the following parameters are displayed:
    • Filter Attribute: Select an attribute to filter Entra ID users for provisioning to the Microsoft Dynamics 365 Finance and Operations application. For example, if you select Employee ID, Entra ID users will be filtered based on this attribute.
    • Attribute Value: Enter the value by which Entra ID users must be filtered for the attribute selected in the Filter Attribute parameter. For example, if you select EMP001, Entra ID users with this employee number will be filtered and provisioned to the Microsoft Dynamics 365 Finance and Operations application.

Provisioning Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  • Create Condition*: Select a condition to specify which Entra ID employee records must be created in the Microsoft Dynamics 365 Finance and Operations application. Only employee records that satisfy this condition will be created in the Microsoft Dynamics 365 Finance and Operations application. For example, if you select Active Only, only employees with an active status are provisioned in the Microsoft Dynamics 365 Finance and Operations application. 
  • Update Condition*: Select a condition to specify which Entra ID employee records must be updated in the Microsoft Dynamics 365 Finance and Operations application. Only employee records that satisfy this condition will be updated in the Microsoft Dynamics 365 Finance and Operations application. For example, if you select ALL, all Entra ID employee records will be updated to the Microsoft Dynamics 365 Finance and Operations application.
  • Termination Condition*: Select a condition to specify which Entra ID employee records must be terminated or deleted in the Microsoft Dynamics 365 Finance and Operations application. Only employee records that satisfy this condition will be terminated or deleted in the Microsoft Dynamics 365 Finance and Operations application. For example, if you select Inactive Only, only employees with an inactive status are terminated or deleted from the Microsoft Dynamics 365 Finance and Operations application.

Provisioning Threshold

Configure the following settings:

  • Import Limit: Enter the maximum number of employee records that can be imported from Entra ID to Microsoft Dynamics 365 Finance and Operations in a single integration run. For example, entering 10 processes only includes the first ten employee records.
    Note: 
  • Create Limit: Enter the maximum number of employee records that can be created from Entra ID to Microsoft Dynamics 365 Finance and Operations in a single integration run. For example, entering 7 allows only 7 employee records to be created in a single integration run.
  • Update Limit: Enter the maximum number of employee records that can be updated from Entra ID to Microsoft Dynamics 365 Finance and Operations in a single integration run. For example, entering 5 updates only 5 employee records in a single integration run.
  • Termination Limit: Enter the maximum number of employee records that can be deleted from Entra ID to Microsoft Dynamics 365 Finance and Operations in a single integration run. For example, entering 10 allows only 10 employee records to be deleted in a single integration run.

Notifications Settings

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

Notification Report Settings

Configure the following settings:

  • Select the Notification Report Detail Level*: Select the report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the  Attribute Filter* drop-down menu with the following options:
      • Included Attributes: Enter the attributes to be included in the email report. This parameter is available only if you select Included Attributes in the Attribute Filter section.
      • Excluded Attributes: Enter the attributes to be excluded from the email report. This parameter is available only if you select Excluded Attributes in the Attribute Filter section.
    • Send the Notification Summary Report As An Attachment Send Integration Report via Email: Select this checkbox to receive an email notification with a report for each integration execution. When enabled, the Email Report Settings section is displayed for further configuration.
    • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values for each event when choosing the Full Details or Custom option.
    • Alert Aquera Support On Integration Failures: Select this checkbox to send a notification alert to the Aquera support on Integration failures.

Notification Template For User Creation Email Notification Settings

Configure the following settings:

  • Email Recipients' Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
  • CC: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the email subject for sending email notifications.
  • Email Body: Enter the content that must be used as the email body for sending email notifications.
  • Restrict to this Domain:
  • CC: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the email subject for email notifications.
  • Email body: Enter the content that must be used as the email body for notifications sent upon successful user creation.
  • Ignore domain when sending notifications: Enter a comma-separated list of domains to ignore for integration-related notifications. For example, if you enter abc.com, email addresses with this domain won’t receive notifications.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integrationsection in the Using Tables in Integrations guide for details.
