Overview

The Aquera platform enables your organization to automate identity lifecycle management by synchronizing employee data across HR, directory, and IT service management systems.

This integration automates identity lifecycle processes between Dayforce, Active Directory, and Freshservice. Dayforce acts as the source of truth for employee data, Active Directory stores and manages user accounts, and Freshservice manages approval-driven journeys for sensitive provisioning actions.

Dayforce changes flow downstream automatically. When an action requires approval, Freshservice journeys are used to gate the Active Directory operation before the account is created, updated, or deactivated.

This integration supports user onboarding, employee updates, offboarding, name changes, group assignments, manager assignments, termination handling, logging, reporting, and report-only execution.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Retrieves employee records from Dayforce.
  • Creates new Active Directory accounts for eligible Dayforce workers.
  • Updates existing Active Directory users when Dayforce attributes change.
  • Routes onboarding, employee update, offboarding, and name change requests through Freshservice journeys when approval is required.
  • Processes approved Freshservice journeys and applies the approved changes to Active Directory.
  • Deactivates or terminates Active Directory users when workers become inactive in Dayforce.
  • Assigns and removes Active Directory groups.
  • Assigns and removes managers in Active Directory.
  • Handles duplicate users across Active Directory domains.
  • Supports report-only mode to simulate changes without applying them.
  • Generates logs, reports, and execution summaries.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal.
  • Dayforce Application Connector in the Aquera Admin Portal: You must configure the Dayforce application connector that will be used to retrieve employee records.
  • Active Directory Application Connector in the Aquera Admin Portal: You must configure the Active Directory connector that will be used to create, update, disable, and manage users.
  • Freshservice Application Connector in the Aquera Admin Portal: You must configure the Freshservice connector that will be used to retrieve journey forms, journey requests, activities, and approval statuses.
  • Required Active Directory Permissions: The configured Active Directory connector must have permissions to create users, update users, disable users, manage groups, assign managers, remove managers, and reset passwords, depending on the enabled use cases.

Integration Setup

This section details the configuration process for integrating Dayforce, Active Directory, and Freshservice. It outlines the steps to create the integration, configure applications, define run parameters, set source filters, configure Freshservice journey approval gates, define Active Directory provisioning behavior, and configure termination handling.

Create Integration

Perform the following steps to create the integration:

  • Log in to the Aquera Admin portal.
  • Navigate to Integrations from the left side panel.
  • Click Create Integration.
  • Search for Active Directory / Dayforce / Freshservice in the search box next to the Explore Templates section, and then select it to create a new integration.
  • Enter a name and description for the integration.
  • Click Continue.

This guide covers the following tabs:

  • Overview
  • Configuration
  • Dayforce

Overview

This tab displays an introduction to the integration and explains how the integration is organized.

The About Overview section explains that the integration automates identity lifecycle management between Dayforce, Active Directory, and Freshservice.

Dayforce is used as the employee source of truth. Active Directory is used to manage user accounts. Freshservice is used to manage IT-driven approval workflows before sensitive Active Directory writes are performed.

The integration performs the following actions during each run:

  • Reconciles a Dayforce employee snapshot against Active Directory and Freshservice.
  • Creates Active Directory accounts for new hires or routes them through an onboarding journey.
  • Updates existing Active Directory users when Dayforce attributes change.
  • Routes employee updates, name changes, and offboarding through Freshservice journeys when configured.
  • Deactivates inactive Dayforce workers in Active Directory immediately or on the configured effective date.
  • Keeps manager assignments and group memberships synchronized.

Configuration

Navigate to the Configuration tab to configure application connections and top-level run parameters.
The following sections are displayed on the page:

  • Applications
  • Parameters

Applications

This section enables you to select the application connectors used by the integration.

The following fields are displayed in this section:

  • Dayforce: Select the Dayforce application connector that will be used to retrieve employee records.
  • AD: Select the Active Directory application connector that will be used to create, update, disable, and manage users.
  • Freshservice V4: Select the Freshservice connector that will be used to create and process journey requests.

After selecting the required applications, click Save Changes at the top-right corner of the page.

Parameters

This section enables you to configure the top-level operating behavior of the integration.

The following settings are displayed in this section:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen without applying them. When enabled, create, update, termination, group, manager, and Freshservice journey actions are simulated.
  • Enable Logging: Select this checkbox to generate logs outlining data changes and execution activity.
  • Enable Notifications: Select this checkbox to enable integration-related notifications.
  • Enable Report Generation: Select this checkbox to generate an integration report.
  • Execute Full Sync (Dayforce to AD): Select this checkbox to execute a full sync from Dayforce to Active Directory. If this option is not selected, only recently updated records from Dayforce are retrieved.
  • Enable Provisioning via Freshservice: Select this checkbox to enable monitored provisioning through Freshservice. If this option is selected, configured Freshservice journeys are used to gate sensitive Active Directory operations. If this option is not selected, direct synchronization from Dayforce to Active Directory is performed.
  • Enable Freshservice Journeys Full Sync: Select this checkbox to fetch all Freshservice journeys. If this option is not selected, only recently created or updated journeys are fetched.
  • Provide lookback period to fetch journeys in incremental mode (in Days): Enter the number of days to look back while fetching Freshservice journeys in incremental mode. The value must be 1 or greater.
  • Dayforce Correlation Field: Select the Dayforce attribute used to match employees with Active Directory users. For example, select EmployeeNumber.
  • AD Correlation Field: Select the Active Directory attribute used to match users with Dayforce employees. For example, select employeeID.

Note: The Dayforce and Active Directory correlation fields are required. The integration uses these fields to determine whether a Dayforce employee already exists in Active Directory.

After configuring the parameters, click Save Changes.

Dayforce

Navigate to the Dayforce tab to define which Dayforce workers are considered for processing. It includes source filters, active status filters, inactive status filters, and an optional custom JavaScript expression.

The following sections are displayed:

  • Source Settings
  • Account Status Configuration

Source Settings

This section controls which Dayforce workers are considered after employees are retrieved from Dayforce.

The following field is displayed:

  • Filter for User Processing: Select the filter that must be applied after Dayforce worker retrieval. The available options include:
    • All Employees: Select this option to consider all retrieved Dayforce employees.
    • Custom: Select this option to define a custom JavaScript expression for filtering workers.

If Custom is selected, the following field is displayed:

  • Custom Source Filter: Enter a JavaScript expression to filter employees for processing in Active Directory. Use worker as the object in the expression.

Example:

worker.department === "Finance"

Account Status Configuration

This section identifies which Dayforce employment statuses should be considered active or inactive.

The following fields are displayed:

  • Active Statuses in Dayforce: Select the employment statuses that must be considered active for the source filter. For example, select ACTIVE.
  • Inactive Statuses in Dayforce: Select the employment statuses that must be considered inactive for the source filter. For example, select TERMINATED.

If you do not find the required status values in the list, click the refresh icon available next to the field to reload the available values.

Freshservice

Navigate to the Freshservice tab to configure approval-driven journey workflows.

The following sections are displayed on the page:

  • Onboarding
  • Employee Updates
  • Offboarding
  • Name Change

Freshservice journeys are used when provisioning through Freshservice is enabled in the Configuration tab. The integration creates or processes journey requests and waits for the configured approval activity before writing approved changes to Active Directory.

Onboarding

The Onboarding section configures the Freshservice journey used when a new active worker exists in Dayforce but does not have an Active Directory account.

The following settings are displayed:

  • Enable Employee Onboarding: Turn on this toggle button to enable employee onboarding through Freshservice.
  • Enable Dayforce user sync to AD on Onboarding journey approval: Select this checkbox to synchronize the approved onboarding journey attributes to Active Directory. If this option is not selected, the onboarding journey attributes will not be written to Active Directory.
  • Select Onboarding Journey Form: Select the Freshservice journey form used to request onboarding for new hires. Click the refresh icon next to the field to reload the available journey forms.
  • Previous journey form names to lookup for onboarding journey requests, if form names updated: Enter previous onboarding journey form names if the form name has changed. This allows the integration to find journey requests that were created with older form names.
  • Onboarding Correlation Key: Select the field in the Freshservice journey request that stores the Dayforce correlation value. This is typically the employee number and is used to match approved journeys back to workers.
  • Select status to fetch Onboarding Journey Requests (Post-Approval): Select the Freshservice journey request statuses that should be fetched for onboarding processing. For example, select IN_PROGRESS.
  • Select journey activity to consider for Onboarding Approval: Select the journey activity used to determine whether onboarding has been approved.
  • Provide activity name for Onboarding Journeys to check approval status: Enter the activity name that must be checked for approval. The integration checks whether the journey activity contains the configured text.
  • Approval: Select the activity attribute used for approval evaluation.
  • Select activity approval which is considered approved for onboarding: Select the approval status value that should be considered approved. For example, select Approved.
  • Enable User Filter: Select this checkbox to filter users for onboarding journey processing.
  • Provide JS Expression to filter Users: Enter a JavaScript expression to filter onboarding journey users. Use workerin the expression.
  • Enable Limit: Select this checkbox to limit the number of onboarding journeys processed in a single run.
  • Provide count to process limited Journeys: Enter the maximum number of onboarding journeys to process.

Note: The onboarding correlation key is required when onboarding is enabled. Without the correlation key, the integration cannot match approved Freshservice journeys back to Dayforce workers.

Active Directory

Navigate to the Active Directory tab to configure Active Directory provisioning behavior.

This tab controls create and update behavior, manager assignment, group assignment, duplicate account handling, and processing limits.

The following sections are displayed:

  • User Sync Features
  • Manage Limit Configuration

User Sync Features

This section allows you to enable or disable Active Directory user synchronization features.

The following settings are displayed:

  • Enable Group Assignments: Turn on this toggle button to assign groups to users during synchronization.
  • Enable Group Removal: Turn on this toggle button to remove extra or redundant group assignments that are no longer needed.
  • Enable Manager Assignments: Turn on this toggle button to assign managers to users during synchronization.
  • Enable Manager Removal: Turn on this toggle button to remove assigned managers from users when required.
  • Deactivate Duplicate Users in Other Domains: Turn on this toggle button to deactivate user accounts in other Active Directory domains when a duplicate user with the same correlation key is found. If disabled, only the primary domain account is affected.
  • Delete Duplicate Users from Other Domains: Turn on this toggle button to permanently delete duplicate user accounts in other Active Directory domains. This action is irreversible. If disabled, duplicate users are only deactivated.
  • Sync Title Updates Directly To AD on Worker’s Reorganization: Turn on this toggle button to update the title directly in Active Directory when a worker is reorganized, instead of routing the update through a Freshservice employee update journey.
  • Sync Department Updates Directly To AD on Worker’s Reorganization: Turn on this toggle button to update the department directly in Active Directory when a worker is reorganized, instead of routing the update through a Freshservice employee update journey.

Manage Limit Configuration

This section allows you to limit the number of Active Directory create, update, and termination operations performed in a single run.

The following settings are displayed:

  • Limit Creates: Turn on this toggle button to cap the number of Active Directory user accounts created during a run.
  • Limit Updates: Turn on this toggle button to cap the number of Active Directory user accounts updated during a run.
  • Limit Terminations: Turn on this toggle button to cap the number of Active Directory user accounts deactivated during a run.
  • Creation Limit: Enter the maximum number of users that can be created in a single run. Once this limit is reached, additional create operations are skipped.
  • Update Limit: Enter the maximum number of users that can be updated in a single run. Once this limit is reached, remaining workers are skipped and re-evaluated during the next run.
  • Termination Limit: Enter the maximum number of Active Directory users that can be deactivated in a single run. Once this limit is reached, remaining termination candidates are skipped and re-evaluated during the next run.

Note: Use limit settings to prevent accidental bulk creation, update, or termination operations.

Termination Config

Navigate to the Termination Config tab to configure how terminated employees are handled in Active Directory.

The following section is displayed on the page:

  • Termination Settings

Termination Settings

This section controls whether terminated Dayforce workers are deactivated in Active Directory and which cleanup actions are performed during termination.

The following settings are displayed:

  • Allow Terminations: Turn on this toggle button to automatically terminate, disable, or deprovision users in Active Directory.
  • Enable Managers Removal on User Termination: Turn on this toggle button to remove the manager assigned to the user during termination.
  • Enable setting Random Password for User on Termination: Turn on this toggle button to set a random password during user termination in Active Directory. The generated password uses 12 characters and includes uppercase letters, lowercase letters, numbers, and special characters.
  • Enable Groups Removal on User Termination: Turn on this toggle button to remove group memberships during user termination in Active Directory.

During execution, the integration evaluates Dayforce termination information and determines whether the user should be deactivated immediately or kept pending for a future run. If the termination condition has not yet been met, the worker remains pending and is re-evaluated later.

Execute Integration

You can execute the integration manually after completing the required configuration.

Perform the following steps to execute the integration:

  1. Configure the required tabs.
  2. Click Save Changes at the top-right corner of the page.
  3. Click Execute.

During execution, the integration performs the following actions:

  1. Logs the execution properties and configured preferences.
  2. Determines whether the run is a full sync or incremental sync.
  3. Retrieves employee records from Dayforce.
  4. Applies the configured Dayforce source filters.
  5. Connects to the configured Active Directory domain.
  6. Prefetches Active Directory users during full runs when no source filter is configured.
  7. Processes pending termination records, if applicable.
  8. Processes approved Freshservice journeys for onboarding, employee updates, name changes, and offboarding.
  9. Evaluates each Dayforce worker against Active Directory.
  10. Creates, updates, skips, or terminates Active Directory users based on configuration.
  11. Creates or processes Freshservice journeys when monitored provisioning is enabled.
  12. Applies manager assignments, manager removals, group assignments, and group removals when enabled.
  13. Applies duplicate user handling based on the configured Active Directory settings.
  14. Generates execution logs, summaries, and reports.

Report-Only Mode Behavior

If Enable Report-Only Mode is selected, the integration does not apply changes to Active Directory.

In report-only mode, the integration simulates actions such as:

  • Creating users
  • Updating users
  • Terminating users
  • Creating Freshservice journeys
  • Assigning managers
  • Removing managers
  • Assigning groups
  • Removing groups
  • Handling duplicate users

The simulated results are included in logs and reports so that administrators can review the expected changes before running the integration in live mode.
