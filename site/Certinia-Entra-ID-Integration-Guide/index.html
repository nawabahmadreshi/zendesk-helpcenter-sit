Certinia / Entra ID Integration Guide

← Back to index

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the integration to sync Users data from Entra ID to the Users and Contacts in Certinia.

To identify records across both systems, Select the correlation attribute that the integration uses to identify and match user records between Entra ID and Certinia.

Key Functions

The Entra ID to Certinia integration automates identity lifecycle management with the following primary capabilities:

  • Syncs Users and Contacts records from Entra ID into the Certinia.
  • Creates new contact attributes in Certinia.
  • Deactivates the corresponding user records in Certinia when the users are deleted in Entra ID.

Prerequisites

The following are the prerequisites:

  • Admin Access to Entra ID: You must have admin access to your Entra ID instance. You can visit the Entra ID portal to get access to an Entra ID instance.
  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Entra ID:
    • Certinia: Ensure that you configure the Certinia connector in the Aquera Admin portal. Refer to the Certinia Configuration Guide for configuration details.
    • Microsoft 365: Ensure that you configure the Microsoft connector in the Aquera Admin portal. Refer to the Microsoft 365 Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating Entra ID with Certinia. It outlines the steps to create the integration, set up integration preferences, and define how worker data is mapped from Entra ID to Certinia.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID / Certinia.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID to Certinia connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the Entra ID to Certinia application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Features

Enable the following features based on your requirements:

  • User Lifecycle Management: Enabling this checkbox configures user lifecycle management. When enabled, it activates user lifecycle management and displays the User Life Cycle section, where you can configure the related integration settings.
  • Contact Lifecycle Management: Enabling this checkbox configures contact lifecycle management. When enabled, it activates contact lifecycle management and displays the Contact Life Cycle section, where you can configure the related integration settings.
  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs tracking data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable Reporting: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration. 
  • Enable Incremental Mode: Enable this checkbox for incremental mode, ensuring that only the most recently changed records are processed during each integration run.

User Lifecycle Management

The User Lifecycle section enables you to customize the integration settings for user lifecycle management according to your specific use case.
Note: This section appears only if you have selected the User Lifecycle Management setting in the Parameters section.

It has the following settings:

  • Enable creations: Enable this checkbox for the creation of user records during the integration process.
  • Enable updates: Enabling this updates the existing user records, ensuring that any changes are reflected during the integration run.
  • Enable terminations: Enable this checkbox to perform termination of user records, allowing the integration to process and apply termination events accordingly.
  • Map manager: Enable this checkbox to perform mapping the manager attribute to the corresponding user records during the integration process.

Match Expression

These settings determine how users are matched between Entra ID and Certinia:

  • Source Correlation attribute: Defines the attributes from Entra ID used to uniquely identify users. For example, User Principal Name is used as the unique identifier from Entra ID.
  • Destination correlation attribute: Defines the corresponding attribute in Certinia used for matching. For example, a Username is used to match existing employee records in Certinia.

Manage Create Conditions

  • Switch to expression mode: Select this checkbox to use custom expressions to create a user.
  • Create condition: Enter the condition used to determine whether a given user record should be created during the integration process. By default, this is set to Create All Users, which ensures all user records are processed for creation.
  • Custom Create Condition: Enter the JavaScript expression used to define the condition for creating a user record.
    Note: This parameter is available only if Switch to Expression Mode is selected.

Manage Update Conditions

  • Switch to Expression Mode: Select this checkbox to use custom expressions to update a user.
  • Update Condition: Enter the condition used to determine whether a given user record should be updated during the integration process. By default, this is set to Update All Users, which ensures all user records are processed for creation.
  • Custom Update Condition: Enter the JavaScript expression used to define the condition for updating a user record.
    Note: This parameter is available only if Switch to Expression Mode is selected.

Manage Termination conditions

  • Termination Condition: Specifies the condition that determines when user records should be terminated during the integration process.

Contact Life cycle

The Contact Life Cycle section enables you to customize the integration settings for contact life cycle management according to your specific use case.
Note: This section appears only if you have selected the Contact Lifecycle Management setting in the Parameters section.

It has the following settings:

  • Create Contacts: Enable this checkbox for the creation of contact records in Certinia during the integration process, ensuring that new contacts are added accordingly.
  • Update Contacts: Enable this checkbox to make updates to existing contact records, ensuring that any changes are reflected during the integration run.
  • Create Contact Only If Certinia User Exists: Enable this checkbox to restrict contact creation to only those records where a matching Certinia user record is found. If not enabled, the contact will be created to all users.
  • Sync Certinia User Data to Certinia Contact: Enable this checkbox to sync fields from the matching Certinia user record into the corresponding contact record. If no matching user is found, the contact record is still created or updated. If not enabled, the sync will not occur.
  • Map Manager to Contact: Enable this checkbox to map each employee's direct manager from Certinia to the corresponding contact record in Entra ID.

Debug Mode

This section enables you to customize the debug mode settings for testing use cases using a single user. It has the following settings:

Note: This section appears only if you have selected the Enable Debug Mode setting in the Parameters section.

  • Enable Single User Testing: Enable this checkbox to configure single user testing mode, allowing the integration to be tested against a specific user record without processing all users. 
    • Source Username: Enter the username of the specific user to test the integration.
  • Import Limit: Enter the maximum number of worker records that should be imported for debugging during each integration run. By default, the value is 100.

  • Consent to Send Email Report with Attribute Values: Enable this checkbox to capture and include all attribute values in the email report.
  • Report Detail Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following parameters are displayed once you select Custom:
      • Attribute Filter: Select the attribute filter to include or exclude attributes in the report.
      • Attributes to be Included: Select the attributes to be included in the report. This parameter is displayed only if the Included Attributes option is selected in the Attribute Filter parameter.
      • Attributes to be Excluded: Select the attributes to be excluded in the report. This parameter is displayed only if the Excluded Attributes option is selected in the Attribute Filter parameter.
  • Email Subject: Enter the content that must be used as the email subject.
  • Email Body: Enter the content that must be used as the email body for all email notifications.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

In this Integration these are the following mapping sections:

  • Entra to Certinia User: Configure the attribute mapping settings for syncing user records from Entra ID to Certinia.
  • Entra to Certinia Contact: Configure the attribute mapping settings for syncing contact records from Entra ID to Certinia.
  • Certinia User to Contact: Configure the attribute mapping settings for syncing user records from Certinia to the corresponding contact records in Certinia. This configuration appears only if you enable Sync Certinia User data to Certinia Contact in Contact Life Cycle.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
