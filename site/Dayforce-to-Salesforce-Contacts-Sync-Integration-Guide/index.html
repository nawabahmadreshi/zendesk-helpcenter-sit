Dayforce to Salesforce Contacts Sync Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any contacts created or updated in Dayforce are automatically updated in Salesforce based on the defined schedule.

To identify user records across both systems, the integration uses the attribute configured in the Jira Service Management Correlation Attribute parameter from Jira Service Management Cloud and matches it to the userPrincipalName in Active Directory as a correlation attribute.

Key Functions

The Dayforce to Salesforce Integration performs the following functions:

  • Synchronizes contact records from Dayforce to Salesforce.
  • Creates new Contacts in Salesforce whenever new employees are added in Dayforce.
  • Updates existing Salesforce Contacts when their information changes in Dayforce.
  • Supports both full synchronization and incremental synchronization to process only changed records.
  • Synchronizes each employee’s manager mapping in Salesforce.
  • Supports filtering employees by status for creating and updating contacts in Salesforce.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Jira Service Management to Active Directory.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce to Salesforce.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Salesforce connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable Contact Lifecycle Management: Select this checkbox to synchronize employee's contact data from Dayforce to Salesforce.
  • Dayforce Time Zone Offset (from GMT)*: Enter the time offset from GMT to be used when synchronizing user contact data from Dayforce to Salesforce. Both positive and negative offset values are supported. For example, if you enter -05:00, the Salesforce server time is considered 5 hours behind GMT, and the data synchronization will be performed based on this offset. This is applicable only during incremental sync.
  • Enable Incremental Mode: Select to execute the integration in incremental mode.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

Manage Contacts

The Contact Management section enables you to customize the integration settings for contractor management according to your specific use cases.

Manage Contact Lifecycle Management

Configure the following settings:

  • Create Contacts: Select this checkbox if your use case requires creating contacts in Salesforce from Dayforce.
  • Update Contacts: Select to update the user’s contact details in Salesforce from Dayforce.
  • Verify user before creating contact: Select to create a contact only after confirming that the corresponding Salesforce user exists and is active. If the employee is unavailable or inactive, contact creation will be ignored.
  • Include Manager Contact Details: Select this checkbox to include the employee’s manager’s contact details in the employee contact report from Dayforce to Salesforce. If this checkbox is not selected, the manager’s contact details will not be included.
  • Create Contacts Only for New Hires: If you select this checkbox, only new hires (employees with start date of today or future) will be created as new Contacts. If this checkbox is not selected, all employee's will be considered for contact creation in Salesforce.

Eligible Statuses for Contact Creation And Update Statuses Settings

Configure the following settings:

  • Eligible Statuses for Contact Creation: Select one or more employee statuses. Contacts will be created in the Salesforce application only for employees with the selected statuses.
  • Eligible Statuses for Contact Update: Select one or more employee statuses. Contacts will be updated in the Salesforce application only for employees with the selected statuses.

Correlation Settings

Configure the following correlation settings:

  • Dayforce Correlation Attribute: Select a Dayforce attribute to distinctly find the associated Salesforce contact record. For example, if you select EmployeeId in this field, the EmployeeId attribute will be used as a correlation attribute.
  • Salesforce Correlation Attribute: Select a Salesforce contact attribute to distinctly find the associated Dayforce user record. For example, if you select Department in this field, the Department attribute will be used as a correlation attribute.

User Lookup Configuration

Configure the following settings:

  • Salesforce User Field: Select a Salesforce user attribute to distinctly find the associated Salesforce contact record. For example, if you select Username attribute in this field, 
  • Salesforce Contact Field: Select a Salesforce contact field to distinctly find the associated Salesforce employee record.

Debug Settings

This section enables you to set debug options to help troubleshoot and fix integration issues.

Note: The Debug Settings tab appears when you select the Enable debug mode checkbox in the Integration Settings section of the Parameters tab.

  • Enable for Single-User: Select to debug the integration for a single employee.
  • Dayforce User EmployeeNumber: Enter the employee number of the Dayforce employee to be used for debugging purposes.
  • Enable for Multiple-User: Select to debug the integration for multiple employees.
  • Import Limit: Enter the maximum number of employee contacts to be imported to Salesforce for debugging in a single integration run.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
