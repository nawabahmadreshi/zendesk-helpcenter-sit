Entra ID Group Management Integration Guide

← Back to index

Overview

The Aquera platform enables your organization to automate group management processes by evaluating user attributes and creating Dynamic Distribution Lists automatically.

This integration manages Dynamic Distribution Lists by evaluating Microsoft Entra ID user attributes such as state, office location, department, and title. Based on the configured rules, the integration generates the appropriate recipient filters and creates Dynamic Distribution Lists in the target environment.

By default, the integration retrieves users from the selected source application and evaluates each user against the configured Dynamic Distribution List rules. If a matching Dynamic Distribution List does not already exist, the integration creates it automatically. If the list already exists, it is skipped and recorded in the logs and reports.

Key Functions

This integration automates Dynamic Distribution List management with the following primary capabilities:

  • Retrieves user records from the selected Entra ID source application.
  • Creates Dynamic Distribution Lists based on user attributes such as State, OfficeLocation, Department, and Title.
  • Supports include and exclude criteria for controlling which attribute values are used to create Dynamic Distribution Lists.
  • Automatically generates recipient filter rules for Dynamic Distribution Lists.
  • Maps generated values to Dynamic Distribution List attributes such as name, displayName, alias, and recipientFilter.
  • Prevents duplicate Dynamic Distribution List creation by checking existing Dynamic Distribution List names before creating new ones.
  • Supports Report Only mode to preview the Dynamic Distribution Lists that would be created without applying changes.
  • Generates logs, CSV reports, dashboard analytics, and email notifications for integration runs.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal.
  • User Account in Entra ID: You must have a user account in Entra ID with the required permissions to retrieve users and manage Dynamic Distribution Lists.
  • Source Application Connector in the Aquera Admin Portal: You must configure the Entra ID or Microsoft 365 source application connector in the Aquera Admin portal.
  • User Attribute Data in Entra ID: The user attributes used for Dynamic Distribution List creation, such as state, officeLocation, department, and jobTitle, must be populated in Entra ID.
  • Email Configuration: If you want to send execution notifications or integration reports by email, email notification settings must be configured.

Integration Setup

This section details the configuration process for the Entra ID Group Management integration. It outlines the steps to create the integration, configure Dynamic Distribution List rules, map attributes, define operational settings, configure notifications, and execute the integration.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for EntraID Group Management in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • Group Management
  • Attribute Mapper
  • Operation Settings
  • Notification
  • Tables
  • Schedule

Overview

This tab displays comprehensive details about the integration and its related operational controls.

The following sections are displayed on the page:

  • Integration Overview
  • DDL Feature Controls
  • Operational Features

Integration Overview

This section displays the purpose of the integration.

The integration manages Dynamic Distribution Lists by evaluating user attributes such as state, office location, department, and title to create the appropriate group filters automatically.

DDL Feature Controls

This section allows you to enable or disable Dynamic Distribution List creation.

The following field is displayed in this section:

  • Enable DDL Creation: Select this checkbox to enable Dynamic Distribution List creation for the integration. If this option is disabled, the integration will not create Dynamic Distribution Lists during execution.

Operational Features

This section allows you to enable or disable operational features used during integration execution.

The following fields are displayed in this section:

  • Enable Report Only Mode: Select this checkbox to generate a report detailing the changes that would happen, without applying them. Use this option to validate the configuration before creating Dynamic Distribution Lists.
  • Enable Logging: Select this checkbox to generate logs outlining data changes and integration activity.
  • Enable Notifications: Select this checkbox to enable integration-related notifications.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Dashboard Generation: Select this checkbox to analyze integration-related metrics in a dashboard.
  • Enable Debug Mode: Select this checkbox to generate additional logs for debugging purposes.

After configuring the required settings, click Save Changes at the top-right corner of the page.

Group Management

Navigate to the Group Management tab to configure the source application and Dynamic Distribution List management rules.

The following sections are displayed on the page:

  • Source
  • DDL Management

Source

This section enables you to identify the source application used to retrieve users for Dynamic Distribution List creation.

The following field is displayed in this section:

  • Source Application: Select the source application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

DDL Management

This section allows you to define Dynamic Distribution List creation rules. Each configuration defines the user attribute to evaluate, whether to include or exclude values, the values to include or exclude, the expression used to calculate attribute values, and the generated Dynamic Distribution List rule.

If the List View toggle button is disabled, the following parameters are displayed:

  • Dynamic Group Definition: Select the user attribute that will be used to create Dynamic Distribution Lists. The available options are:
    • State
    • OfficeLocation
    • Department
    • Title
  • Include/Exclude Criteria: Select whether the configured values should be included or excluded while creating Dynamic Distribution Lists. The available options are:
    • Includes
    • Excludes
  • Includes: Enter the list of attribute values that are allowed to create Dynamic Distribution Lists. This field is displayed when Includes is selected in the Include/Exclude Criteria field.
  • Excludes: Enter the list of attribute values that must be excluded from Dynamic Distribution List creation. This field is displayed when Excludes is selected in the Include/Exclude Criteria field.

  • Attribute Value calculation for rule: This field defines the expression used to calculate the selected attribute value for each user. The calculated value is used to create the Dynamic Distribution List payload.

    For example, if State is selected as the Dynamic Group Definition, the expression returns the user's state value in the expected output format. If the state value is missing or empty, the expression returns an empty object instead of including a null value.

    Click View Preview to review the generated code and sample output.

  • Dynamic Distribution List Rule: This field displays the recipient filter rule used to create the Dynamic Distribution List. The rule defines the criteria and logic used to dynamically select users based on the configured attribute and value.

The integration evaluates each user against the configured rule. If the calculated value satisfies the include or exclude criteria, the integration prepares a Dynamic Distribution List creation payload for that value.

To add another Dynamic Distribution List configuration, click the + icon.

To navigate to multiple configurations, use the arrow icons.

To delete a configuration, click the delete icon.

To view the Dynamic Distribution List configurations in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

After configuring Dynamic Distribution List rules, click Save Changes at the top-right corner of the page.

Attribute Mapper

Navigate to the Attribute Mapper tab to define how calculated values are mapped to Dynamic Distribution List attributes.

The following section is displayed on the page:

  • DDL Creation

DDL Creation

This section contains the Dynamic Distribution List Mapper, which maps source expressions to target Dynamic Distribution List attributes.

Source attributes or expressions are displayed on the left side and target attributes are displayed on the right side.

The mapper includes the following target attributes:

  • name: Defines the name of the Dynamic Distribution List.
  • displayName: Defines the display name of the Dynamic Distribution List.
  • alias: Defines the alias of the Dynamic Distribution List.
  • recipientFilter: Defines the recipient filter rule used by the Dynamic Distribution List.

The integration uses the selected Dynamic Group Definition and calculated rule value to generate the Dynamic Distribution List payload. For example, a calculated rule value can be used to generate names such as DL_<value>.

JavaScript expressions can be used to transform source attribute values and construct the target values.

The Search Attributes field allows you to filter the list of available attributes for mapping.

Use the Enter objects for preview option to preview how the mapping expressions generate target values.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page.

Execute Integration

You can execute the integration manually or schedule it to run automatically.

Perform the following steps to execute the integration manually:

  1. Configure all required tabs.
  2. Click Save Changes.
  3. Click Execute at the top-right corner of the page.

During execution, the integration performs the following actions:

  1. Displays the configured integration preferences in the execution logs.
  2. Retrieves users from the selected source application.
  3. Applies the configured source filter, if any.
  4. Evaluates each user against the configured Dynamic Distribution List rules.
  5. Applies include or exclude criteria.
  6. Generates unique Dynamic Distribution List payloads.
  7. Retrieves existing Dynamic Distribution Lists.
  8. Skips Dynamic Distribution Lists that already exist.
  9. Creates missing Dynamic Distribution Lists, unless Report Only mode is enabled.
  10. Generates logs, reports, dashboard analytics, and notifications based on the configured operational settings.

Report Only Mode Behavior

If Enable Report Only Mode is selected, the integration does not create Dynamic Distribution Lists.

Instead, it identifies the Dynamic Distribution Lists that would be created and records them in logs, reports, and notifications. Use this option to validate your configuration before running the integration in live mode.
