Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration synchronizes worker data from UKG Ready to Okta XaaS. It supports a complete sync (imports all workers from a saved UKG report) and an incremental sync (imports only records changed in the last 31 days using a “changed since” timestamp). After retrieval, it applies configurable rules to determine which workers should be created, updated, or removed in Okta. It converts each record into an Okta-ready profile using a defined attribute map. The integration removes any stale Okta sessions, creates a new one, and sends worker records in batches. After all batches are submitted, it starts the import and monitors it until completion. 

Key Functions

This integration provides the following functions:

  • Retrieves worker records from UKG Ready by either running a complete sync that reads all workers from a saved UKG report, or an incremental sync that uses a “changed since” timestamp (limited to the last 31 days) to pull only updated records.

  • Applies action rules to determine provisioning by evaluating each worker against configurable conditions (including custom field/operator/value checks) to decide whether the user should be created or updated in Okta, or removed when appropriate.

  • Maps UKG fields to Okta user profiles by transforming each worker record through a defined attribute map.

  • Submits batched operations to Okta by sending users in bulk for create/update or delete.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Enable XaaS access:  You must open a support case with Okta to enable XaaS in your Okta instance.
  • Source Priority for Username and Email Update: In case of username and email update for Okta profile, ensure that the username and email attributes in Okta are mastered by the correct system. Refer to the Source Priority section for more details.
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal:
    • UKG Ready: Ensure you add and configure the UKG Ready connector in the Aquera Admin portal. Refer to the UKG Ready Configuration Guide for details.
    • Okta XaaS: Ensure you add and configure the Okta XaaS connector in the Aquera Admin portal. Refer to the Okta XaaS (Anything-as-a-Source) Configuration Guide for instructions on creating the Okta XaaS application connector and setting up the authentication.
  • Prepare a saved worker report (the integration expects a report ID) with all fields needed for mapping.

Note: For details on the limitations of Okta's Anything-as-Source (XaaS) solution, refer to the Okta documentation.

Source Priority

If your use case requires updating username and email for Okta profile, ensure that these attributes in Okta are mastered by the correct system. Perform the following steps to configure the source priority:

  1. In Okta Admin Console, navigate to Directory > Profile Editor and select the User Profile.
  2. On the Profile Editor page, scroll down to the Attributes section.
  3. Find username attribute and click the info (i) icon.
  4. Set the Source Priority as follows:
    • Select Inherit from Profile Source if you want UKG Ready to be the source of truth to provide and maintain the attribute value.
    • Select Inherit from Okta if the attribute must be managed in Okta and then pushed to UKG Ready. In this case, Okta becomes the source of truth.
  5. Repeat the same steps for the email attribute and set the appropriate source priority based on your requirements.

Integration Setup

This section describes how to configure the integration that synchronizes worker data from UKG Ready to Okta XaaS.

Create Integration

Follow these steps to create the integration:

  1. Log in to the Aquera Admin Portal.

  2. In the left panel, go to Integrations.

  3. Click Create Integration.

  4. Select Okta / UKG Ready as the integration template.

  5. Enter a name and description for the integration.

  6. Choose your UKG Ready connector instance as the Source and your Okta XaaS connector instance as the Target.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Feature

The following is the integration feature:

  • Enable User Management: Select this checkbox to create or update users that meet your upsert rules, remove users that meet your delete rules, start the Okta import, and track results. When enabled, the User Sync section is displayed to configure further integration settings.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this option to generate logs that outline data changes. When enabled, the Logging section is displayed to configure further settings.
  • Enable Notifications: Select this option to enable notifications related to integration. When enabled, the Notification Settings section is displayed to configure further settings.
  • Enable Report Generation: Select to generate the integration report. When enabled, the Reporting Settings section is displayed to configure further settings.
  • Enable Dashboard Generation: Select to visualize the integration-related metrics. When enabled, the Insights Settings section is displayed to configure further settings.

User Sync

Configure the following settings for syncing users:

UKG Ready

The following settings are available in this section:

  • Company Name: Enter the exact UKG Ready company short name (case-sensitive). For example, acme.
  • Report ID: Enter the Report ID of the UKG report. For example,: 84703619
  • Full Sync: Select the checkbox to sync all workers with every run. Uncheck to run an incremental sync of only records changed since the last successful run. For example, checked during the first launch, unchecked for nightly incremental runs.

Delete Conditions

The following settings are available in this section:

  • Delete Conditions List: Select the rules that determine when to delete users from Okta. The available options are:
    • Employee Status: Select this if you want to set a delete condition based on the employee's status. When selected, the following field is displayed for configuration:
      • Employee Status: Select the status that must be considered to delete the record. For example, Terminated.
    • Account ID: Select this if you want to delete only the worker with this specific Account ID. When selected, the following field is displayed for configuration:
      • Account ID: Enter the account ID that you want to be deleted. For example, 87452078
    • Custom Condition: Select to build a rule using any UKG Ready field to determine which workers to delete. For example, remove users whose Department equals Temp.
      • Delete Custom Condition Field Name: Enter which UKG Ready field to check. For example, Department.
      • Delete Custom Operator: Select the operator to compare the field value. For example, ==
      • Delete Custom Condition Field Name: Enter the value to be used for the delete condition. For example, Temp,
  • Limit Deletions: Enter the maximum number of users to delete in a single integration run. Leave it blank for no limit. For example, 50.

Upsert Conditions

The following settings are available in this section:

  • Upsert Conditions: Select rules that determine which workers are created or updated in Okta. For example, select Employee Status and Date Hired to sync only active employees hired in the last 30 days. The following are the available options:
    • Account ID: Enter a specific Account ID to sync only that worker. Leave blank to apply other upsert conditions to all eligible workers. For example, 87452078
    • Employee Status: Select the UKG Ready status for workers who should be synced to Okta. Example: Active
    • Employee ID: Enter a specific Employee ID to sync only that worker. Leave blank to apply other conditions. For example, E12345
    • Date Hired: Select to sync workers hired on or after this date. For example, 2025-01-01
    • Date Terminated: Select to sync workers terminated on or after this date. For example, 2025-09-01
    • Date Re-Hired: Select to sync workers who were re-hired on or after this date. For example, 2025-08-15
    • Custom Condition: Select to build a rule using any UKG Ready field to determine which workers to upsert. For example, Sync users whose Location equals HQ.
      • Upsert Custom Condition Field Name: Enter the exact field name from your UKG Ready report to use for comparison (case-sensitive). For example, Location
      • Upsert Custom Operator: Select the operator to compare the field value. For example, ==
      • Upsert Custom Condition Field Value: Enter the value to compare against. For example, HQ
  • Limit Upserts: Enter the maximum number of users to create or update in a single integration run. Leave blank for no limit. For example,100Schedule

Configure the following settings for email notifications:

  • Attach Report to email: Include the CSV report as an attachment to the email.

Attribute Map

The Attribute Map defines how employee data flows from UKG Ready to Okta user profiles during synchronization. This mapping ensures that essential employee information, including Account ID, names, usernames, and contact details, is accurately transferred and maintained across both systems. The integration automatically transforms UKG Ready field data into corresponding Okta attributes. 

To configure additional mappings, click any empty Enter Expression row and specify the UKG Ready field using the exact format, as it appears in your report data. Apply optional transformations using the middle dropdown, then select the target Okta profile attribute from the right dropdown.

Create Custom Attributes in Okta

You can add custom attributes to the App-specific user profile in Okta to include additional Dayforce Employee attributes. You must configure each custom attribute individually. After the configuration, Aquera automatically populates the fields that you have created in Okta from the Employee resource of Dayforce.

Perform the following steps to create custom attributes in Okta:

  1. Sign in to your Okta instance.
  2. Navigate to Directory > Profile Editor and select the App-specific user profile that you've created.
  3. Click Add Attribute to add a new attribute.
  4. Configure the attribute using the same name as Dayforce in the Variable name field and fill in other fields.

    Note: Refer to the Variable Name for Custom Attributes in Okta section for details on the variable name.
  5. Click Save to save the changes or Save and Add Another to create another custom attribute.

Map Custom Attribute to Okta Profile

Perform the following steps to map the custom attributes to the Okta profile:

  1. Navigate to Mappings > Configure User mappings.
  2. Map the custom attribute to an equivalent attribute in the Okta user profile and click Save Mappings.
  3. If the equivalent attribute is not present in the default Okta user profile, open the default profile from Directory > Profile Editor.

     
  4. Click Add Attribute to add a custom attribute to the Okta default user profile.
  5. Fill in the attribute's display name, description, and set the user permission. Click Save Attribute to save the changes.
  6. Map the custom identity source attribute to the attribute in the Okta default profile and click Save Mappings.

Note: You can repeat these steps to add all employee-related attributes in UKG Ready to Okta.
