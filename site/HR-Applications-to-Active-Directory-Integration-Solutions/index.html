Overview

Aquera offers various solutions to help integrate users from any HR system to Active Directory. 

This guide outlines the different solutions provided by Aquera, helping you choose the best approach for your integration needs. You can choose the most suitable solution after analyzing the benefits and drawbacks of each solution.

You can use the Entra ID API-Driven Provisioning by setting up an enterprise application in Entra ID, or you can go with the Any HR Application to Active Directory integration.

Integration Solutions

The following are the two integration solutions provided by Aquera:

  • Integration using Microsoft Entra ID API-Driven Provisioning
  • Any Application to Active Directory Integration

Both integration solutions have two variants: Basic and Advanced. The Basic integration serves as a plug-and-play solution that necessitates no/minimal technical support for deployment. The Advanced integration solution provides greater flexibility by allowing you to set or modify the user attributes mapping in the Aquera Admin portal according to your requirements. You may need Aquera technical support team to set up the Advanced integration. 

Integration using Microsoft Entra ID API-Driven Provisioning

In this integration, you need to create an enterprise application in your Entra ID tenant and then set the mapping of attributes from the application user profile to the AD user profile. The main advantage of this integration is it can support bulk imports of a large volume of users from the source application.

With this integration, you can have Entra ID to send onboarding emails and can manage the entire user life cycle within Entra ID. 

Any Application to Active Directory Integration

In this integration, you can completely set the mappings and control the integration on the Aquera Admin portal. 

This integration is the recommended approach if you want to manage the user naming attributes (User Principal Name, SAM Account Name, CN and Email) and assign users to different OUs based on specific conditions. Refer to this guide for the setup required for this integration.

Parameters

Approximate Deployment Time

This is the approximate time taken to deploy the integration solution and make it work from the time of purchase of the solution.

Required Level of Technical Knowledge

This parameter indicates the level of technical knowledge that you need to setup the integration. If the required level of technical knowledge is Medium or Expert and you do not have a technical personnel to assist you, you can request Aquera Support team for the deployment.  

Require Aquera Technical Support

This parameter indicates if the solution can be deployed with or without help from the Aquera Support team. Aquera Support team can help in setting up the mapping for the user attributes from the source application to Entra ID.

Customization Options

This parameter indicates if customization options are available to you for the solution. For a plug-and-play solution, no customization is available. For the solutions that have customizations available, you have the option to set or modify the mapping of attributes from the source system to Entra ID. 

Bulk Imports of Users

This parameter indicates if the integration supports bulk user creation and bulk user updates.

Support for Organizational Unit (OU) Assignment

This parameter indicates if OU can be assigned to users based on some pre-defined rules. The Any Application to AD integration provides options to set rules for assigning users to different OUs.
