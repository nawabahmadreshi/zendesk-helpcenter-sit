Using Tables in Integrations

← Back to index

Overview

This guide provides details about how to add tables in integrations. Tables define a key-name and key-value pair to define custom attributes in attribute mappings. Once the tables are defined, tables can be used in the expressions of attribute mapping or group rules of User Integration, Bulk Integration, and Event Integrations.

Tables can also be used in the integration scripts. For details on creating tables to be used in scripts, refer to the Integration Using Scripts Guide.

Tables are added in integrations to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or particular data is not sent from an application, and you want to update a value for that attribute in another application. Based on your requirement, you can add any number of tables in an integration.

Adding Tables in Integration

Perform the following steps to add a table in an integration:

  1. Navigate to your integration and click the Tables tab.

  2. Click Add Table which will populate an Add Table window.

  3. Enter a Table Name and Description for the new Table.
    Note:     The table name should be unique. It can include a combination of letters and numbers. It should always start with a letter. It should not have only numbers. It should not contain any spaces or special characters and is case-sensitive. 
  4. Add rows to tables in one of the following ways:  
    • Add Row: To enter rows manually, click Add Row at the bottom of the Add Table window and enter a key name and a key value. Each row in the table corresponds to a key-name and key-value pair.
    • Upload CSV File: To upload rows to the table using a CSV file, click Upload CSV File. Aquera's Integration supports only the .CSV file format and the file should have only two columns. The column headings in the CSV file must be Name and Value.
      A sample CSV file is attached for your reference. You may edit this file to add the required details and upload it. The CSV file can have data of any number of rows.
      Notes      :
        • The key name should be unique. The key name can include a combination of letters and numbers.
        • The key name and key value support all the special characters except the following:
          • / (forward slash) 
          • \ (backslash)
          • " " (double quotes)
        • When the key value contains a comma, the data next to it is considered the next value.
        • If the key value contains comma-separated values, ensure there are no spaces between the values.
        • Do not enclose the key values in double quotes.
  5. Click the Add button at the bottom of the Add Table window to add the table.
  6. Close the table window and click Save Changes at the top right of the integration page to save the table to the integration.
    Note: When a new table is added or a table is modified, then save the changes and do a page refresh to access the table in the integration.

Example Scenarios

Scenario 1: If the hire date of a certain group of employees is not available in the source application and you want the same hire date for all those employees to be posted to the target application, then you can add a table containing the hire date value.

Enter the table name as HireDate and add a description. Add a table row by clicking Add Row and entering the key name as empHireDate and the key value as 2020-09-09.

Scenario 2: If you want a specific hire date value for each employee, then you can add a table with rows for each employee number and their hire date value.

Enter the table name as hireDate and add a description. To add a hire date for an employee with the employee number 12345, add a table row by clicking Add Row and entering the key name as 12345 and the key value as 2018-01-05.

Actions on Tables

You can perform the following actions on tables:

  • You can delete a table row by clicking the delete icon that is displayed when you move the mouse cursor over a table row.
  • After adding a table to the integration, you can view the new table listed on the Tables tab:
  • You can delete a table by clicking the delete icon that is displayed when you move the mouse cursor over a table entry.
  • You can edit a table by clicking a table row. This will populate the Update Table window that allows you to make changes to the existing table.
  • You can export a table by clicking the Export button shown in the Update Table window. This will download the table in a CSV file format.

Using Tables to Include Custom Attributes in Attribute Mapping

If tables are configured in User Integrations and Bulk Integrations, you can access the tables in attribute mapping.

Use the following format in the source attribute of the Attribute Map to use the values from a defined table:
TABLE.<table_name>[<key_name>]

Note: The key name that is defined in a table, must be enclosed within double quotes. For example, TABLE.HireDate["empHireDate"]

However, if the key name is the same as the user attribute name in the application, do not enclose it in double quotes. For example, TABLE.hireDate[SOURCE[ENT].employeeNumber]

Scenario 1: If you want the hire date of all employees in the target application to be updated with the same date value from the table, use the following expression:

TABLE.HireDate["empHireDate"]


Scenario 2:  If you want the hire date of each employee to be updated with the hire date value for that employee number from the table, then use the following expressions:

TABLE.hireDate[SOURCE[ENT].employeeNumber]

Table Lookup

When performing a table lookup in the preview mode of attribute mapping, if the value is not found, the preview will be blank. However, when performing a table lookup during an integration run, if the value is not found, the return value will be shown as undefined.

Using Nested Tables

You can configure expressions with nested tables in the attributes mapping of user integrations, if required. 

Note:

Nested tables are not supported in User Integration version V4. In the V5 version, this feature is enabled only for a specific customer.

Example:

Suppose you have the following two tables:

  • SiteNameToCountryName table containing the Site names and the corresponding Country names as the Name-Value pairs
  • CountryNameToCode table containing the Country names and the corresponding Country codes as the Name-Value pairs

You can use the following expression in the attribute mapping to retrieve the country name and the country code from tables for a user's office location from the source application, and then assign the country code to a target attribute:

(
SOURCE[IAN].officeLoc != undefined ? 
TABLE.CountryNameToCode[TABLE.SiteNameToCountryName[SOURCE[IAN].officeLoc]] 
: ""
)
