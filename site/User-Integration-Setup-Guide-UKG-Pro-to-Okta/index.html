User Integration Setup Guide - UKG Pro to Okta

← Back to index

Overview

This guide covers the end-to-end tasks in setting up synchronization to sync users in UKG Pro with Okta.

The following tasks are involved in setting up the synchronization from UKG Pro to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running Synchronization

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • UKG Pro: Refer to the UKG Pro Configuration Guide for details.
  • Okta: Refer to the Okta Configuration Guide for details.

Synchronization Setup

This section details how the User Synchronization must be configured to allow data push from UKG Pro to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the user synchronization.

Considerations

  • Work email can be retrieved from the UKG Pro system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from UKG Pro with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping UKG Pro user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in UKG Pro as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script synchronization. For scripts, refer to the Appendix.

Building Synchronization

This document describes how to set up the synchronization using the User Synchronization method.

  • User Synchronization - In this synchronization, set UKG Pro as the Source Application and Okta as the Target Application. Refer to the User Synchronization Guide for details on the configurations.

You can set the following configurations in the User Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the User Synchronization Guide for details on User Synchronization configurations.

Running Synchronization

  • Enter the matching expression to match the source records with the destination records.
  • Run the synchronization in Report Only mode and determine if the correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. Modify the synchronization configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in UKG Pro.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow synchronization to update users, you need to disable the setting that sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting that sets Active Directory as profile master.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in UKG Pro and Okta before setting up the synchronization. 

This document explains how analyses can be done using a script synchronization. In this synchronization, add UKG Pro and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script synchronization, refer to the Custom Script Synchronization Guide.

Few sample scripts are provided here for your reference. The scripts may vary based on your use case  and the attributes to be retrieved for analysis.

Analyzing the Users in UKG Pro

Use the below sample script in a script synchronization for generating a report from UKG Pro for analysis of username, user title, work location, and department. The work location and department are mapped to IAN attributes.

let ukg_users = ukg_user.getUsers();

      let i = 0;

      log.info('UKG ID,UKG STATUS,UKG EMPNO,UKG GIVENNAME,UKG FAMILYNAME,
      UKG WORK EMAIL,UKG NICKNAME,UKG USERNAME,UKG MANAGER');

      for (i = 0; i < ukg_users.length; i++)   {

      let ukg_id = ukg_users[i].id;

      let ukg_status = ukg_users[i][IAN].status;

      let ukg_emp = ukg_users[i][ENT].employeeNumber;

      let ukg_given = ukg_users[i].name.givenName;

      let ukg_family = ukg_users[i].name.familyName;

      let ukg_work = ukg_users[i].emails && ukg_users[i].emails.find(x => 
      x.type == 'work') && ukg_users[i].emails.find(x => x.type == 
      'work').value || '';

      let ukg_username = ukg_users[i].userName;

      let ukg_manager = ukg_users[i].manager;

      log.info('"' 

      + ukg_id + '","' 

      + ukg_status + '","'

       + ukg_emp + '","'

       + ukg_given + '","' 

       + ukg_family + '","'

       + ukg_work + '","'

      + ukg_username + '","'

       + ukg_manager + '"');

      }

Analyzing the Users in Okta

OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA 
TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');

for (let user of users) {
log.info('"'
