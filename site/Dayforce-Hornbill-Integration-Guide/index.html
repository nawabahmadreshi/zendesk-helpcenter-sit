Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks involved in setting up the Dayforce / Hornbill integration, which creates joiner, mover, and leaver tickets in Hornbill based on employee creations, updates, and deletions in Dayforce.

 

Key Functions

The Dayforce / Hornbill integration performs the following functions:

  • Reads all the records of employees that were created, updated, and terminated in Dayforce.
    • The integration supports parameters that enable it to look for any future hires, any employment changes with a future effective date, and terminations in the future. Those employees can also be retrieved from Dayforce.
  • Creates Joiner, Mover, and Leaver service requests under specific services and catalogs in Hornbill
  • Assigns specific customer IDs and customer types for Joiner, Mover, and Leaver service requests based on the integration configuration.
  • Populates data in the service request fields based on the attribute maps configured for Joiner, Mover, and Leaver tickets
  • Creates integration reports based on the integration configuration 
  • Sends email notifications on integration runs along with the integration report based on the integration configuration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Dayforce: Ensure that you configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for configuration details.
    • Hornbill: Ensure that you configure the Hornbill connector in the Aquera Admin portal. Refer to the Hornbill Configuration Guide for configuration details. 

Integration Setup

This section details the configurations for retrieving joiner, mover, and leaver data from Dayforce and creating tickets in Hornbill. It outlines the steps to create the integration and set up the integration preferences.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Hornbill.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Hornbill connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the Dayforce and Hornbill connector instances used by the integration. This section allows you to search for a different instance of the application connectors and select them.

Parameters

The Parameters section enables you to customize the integration features and global settings according to your specific use case.

Integration Features

Enable the following features based on your requirements:

  • Enable Onboarding Ticket Generation: Select this checkbox for the integration to handle the creation of onboarding tickets in HaloPSA. When this checkbox is selected, the Onboarding Ticket section is displayed.
  • Enable Offboarding Ticket Generation: Select this checkbox for the integration to handle the creation of offboarding tickets in HaloPSA. When this checkbox is selected, the Offboarding Ticket section is displayed.

Global Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification section is displayed for further configuration.
  • Enable Reporting: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Dashboard Generation: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.

 

Onboarding Ticket

This section details the following settings that are used to configure the creation of onboarding tickets in HaloPSA:

  • Assign Agent: Select this checkbox if you want the integration to add an agent name to the onboarding ticket. When this checkbox is selected, the Agent drop-down list is displayed. When this checkbox is not selected, the ticket will not be assigned to an agent; the agent name in the ticket will be shown as Unassigned. 
  • Allow Selection of End User: Select this checkbox if you want to add the end user details in the ticket. When this checkbox is selected, the End User drop-down list is displayed. When this checkbox is not selected, the end user details will not be added to the ticket.
  • Allow Creation of Tickets for Inactive Users: Select this checkbox to create onboarding tickets for  new employees in inactive status. This setting will be effective only if the Allow Creation of Inactive Users has been enabled in the User Integration that syncs employee data from ADP Workforce Now to the target system for example, Active Directory).
  • New Joiner Ticket Action: Select the action to be performed in HaloPSA when the integration receives the details of a newly onboarded user from ADP Workforce Now. 
    • Select Create to create an onboarding ticket in HaloPSA.
  • Onboarding Ticket Type: Select the HaloPSA ticket type that must be used for creating onboarding tickets. For example, New Hire
    Note: All ticket types from your HaloPSA instance are populated in the drop-down list for your selection.
  • Agent: Select the Agent to whom the onboarding ticket has to be assigned for further action. This field is displayed only if you have selected the Assign Agent checkbox.
    Note: All agent names from your HaloPSA instance are populated in the drop-down list for your selection.
  • End User: Select the end user for whom the onboarding ticket has been raised. This field is displayed only if you have selected the Allow Selection of End User checkbox.
    Note: All end user names from your HaloPSA instance are populated in the drop-down list for your selection.

Logging

This section enables you to customize the log generation settings. It has the following settings:

Note: This section is displayed only if you have selected the Enable Logging checkbox in the Parameters section.

  • Log Detail Level: Select the level of details that must be logged for the integration runs from the dropdown list. The following are the available options:
    • Summary Only: Shows the values set in the ticket fields and a summary of the tickets generated by the integration.
    • Full Details: Shows the onboarding or offboarding payload received by the integration. It also displays the following details:
      • Values that were set in the fields of the onboarding or the offboarding ticket by the integration.
      • Summary of the tickets generated by the integration
    • Custom: Enables you to include or exclude selected attributes in the logs. The following parameters are displayed once you select Custom:
      • Custom Log Level: Select Inclusion List or Exclusion List based on whether you want to specify the attributes to be included or excluded in the logs. Based on your selection, one of the following parameters is displayed:
        • Inclusion Log List: Specify the attributes to be included in the log. The name of the attributes must be the same as the Target Attribute names in the Attribute Mapper.
          Note: This parameter is displayed only if the Inclusion List option is selected in the Custom Log Level parameter.
        • Exclusion Log List: Specify the attributes to be excluded from the log. The name of the attributes must be the same as the Target Attribute names in the Attribute Mapper.
          Note: This parameter is displayed only if the Exclusion List option is selected in the Custom Log Level parameter.
    • Allow to Store Data & Populate Values in Logs: Select this checkbox to enable Aquera to store the employee data for populating them in the logs. If this setting is disabled, the attribute values will not be populated in the logs.

Attribute Map

The Attribute Map tab defines how ADP Workforce Now emplopyee data is mapped to the ticket fields in HaloPSA. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes (ticket fields) are listed on the right side of the page.

Separate attribute maps are available in the integration template for onboarding and offboarding tickets. You can modify only the source attributes in the attribute maps as needed.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the ticket fields. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

After the attribute mapping is completed, click Save Changes at the top-right corner of the page to save the mapping changes.

Post-Setup Configuration

After the ADP Workforce Now / HaloPSA integration is setup, you must select this integration in both the New User Integration Hook and the Deprovision User Integration Hook configuration fields within the User Integration. 

Note: You must have already setup a User Integration to retrieve onboarding and offboarding employees from ADP Workforce Now and to create or delete that user in a target system.

Integration Execution

The integration is triggered automatically based on the new user creations and user terminations processed by the user integration.
