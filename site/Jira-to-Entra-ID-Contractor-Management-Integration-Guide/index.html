Jira to Entra ID Contractor Management Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the contractor's lifecycle from Entra ID to Jira Service Management Cloud for the selected work type. This integration ensures that all contractor lifecycle events in Entra ID are accurately tracked and managed within the Jira system for seamless operational workflows.

Key Functions

The Jira to Entra ID Contractor Management integration provides the following functions:

  • Contractor Creation: The contractor onboarding tickets are retrieved from Jira Service Management and can be filtered by project, issue type, and approval status to identify new user creation requests. Then the corresponding users are created in Entra ID.
  • Contractor Extension Ticket Creation: The contractor's tenure is extended by creating an extension ticket in Jira Service Management, which specifies how many days before the contract end date this ticket must be created. 
  • Contractor Extension: The contractor's tenure is extended based on the end-date calculated as per the extension ticket.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Jira: Refer to the Jira Service Management Cloud Configuration Guide for details.
    • Entra ID: Refer to the Entra ID Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push the JIRA ticket from Jira Service Management Cloud to Entra ID. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to "Integrations" from the left-side panel.
  3. Click Create Integration.
  4. Select Jira to Entra ID Contractor.
  5. Enter a name and a description for the integration.
  6. Select your Jira and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Select Enable Contractor's Lifecycle Management to display the Select Operations dropdown menu. Select one of the following operations (use case) from the dropdown menu:

  • Contractor Creation
  • Contractor Extension Ticket Creation
  • Contractor Extension

General Settings

Configure the following global settings:

  • Jira Timezone: Select the Jira timezone from which the contractor data is being retrieved.
  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize integration-related metrics.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Incremental Mode: Select to run the integration in incremental mode.

Contractor Creation

Configure the following contractor creation settings:

  • Jira Ticket Key: Enter the Jira ticket key to retrieve the onboarding ticket.
  • Jira Project Name: Select the Jira project name from the dropdown menu to retrieve the onboarding tickets.
  • Jira Work Type: Select the Jira work type from the dropdown menu to retrieve the onboarding tickets.
  • Request Type to Fetch Onboarding Tickets: Select the request type from the dropdown menu to retrieve the onboarding tickets.
    Note: This checkbox is displayed based on the chosen Jira project and work type.
  • Jira Ticket Status: Select the Jira status from the dropdown menu to retrieve the onboarding tickets.
  • Auto Close After Contractor Creation: Select to automatically close the Jira ticket once the contractor has been created in Entra ID.
  • Jira Ticket Closure Status: Select the Jira ticket closure status that will be applied when closing the Jira ticket.

Password Policy Settings

Configure the following password policy settings:

  • Password Policy Type: Select the password policy type from the dropdown menu that will be used to create the contractor's initial password.
  • Permitted Character Types: Select the permitted character types for the password creation from the dropdown menu.
  • Generate Contractor's Login Password: Select to generate a random password for the contractor's login.
    Note: This checkbox is displayed when you select Generate Random Password from the Password Policy Type dropdown menu.
  • Password Length: Enter the maximum permissible password length.
  • Permitted Special Characters: Enter the special characters without spaces that are permitted for usage in the password.
  • Send Initial Password to Administrator: Select to send the initial login password for the contractor to the administrator.

Extension Settings

This section is displayed and required, whether you select Contractor Extension Ticket Creation or Contractor Extension from the Select Operations dropdown menu under the Integration Scenarios section. Configure the following extension settings:

  • Jira Project Name: Select the Jira project name from the dropdown menu to retrieve the contractor tickets.
  • Jira Work Type: Select the Jira work type from the dropdown menu to retrieve the contractor tickets.
  • Select the Jira Field for Correlation: Select the Jira field from the dropdown menu to map the Jira attribute with the Entra ID attribute for user identification.
  • Select the Entra ID Field for Correlation: Select the Jira field from the dropdown menu to map the Entra ID attribute with the Jira attribute for user identification.

Extension Ticket Creation Settings

Configure the following contractor extension ticket creation settings:

  • Employee ID in Entra ID: Enter the employee ID in Entra ID to retrieve the contractors.
  • Number of Days Before Contractor's Expiry: Enter the number of days before the contractor's expiration date to retrieve the contractors.

Contractor Extension

Configure the following contractor extension settings:

  • Jira Ticket Key: Enter the Jira ticket key to retrieve the contractor ticket.

  • Select the Jira Field for Approving Contractor Extension: Select the Jira field from the dropdown menu to check the approval status of the contractor extension ticket.

  • Select the Jira Field Type: Select the Jira field type from the dropdown menu for the Jira field you selected above.

  • Jira Field Value: Select the value for the Jira field you selected above, which will be regarded as 'Approved' for the contractor ticket.
    Note: This field is displayed when you select Jira Dropdown from the Select the Jira Field Type dropdown menu.
  • Jira Field Text: Enter the value for the Jira field you have selected above, which will be regarded as 'Approved' for the contractor ticket.
    Note: This field is displayed when you select Jira Text Field from the Select the Jira Field Type dropdown menu.

  • Auto Close After Contractor Extension: Select to automatically close the Jira ticket once the contractor's tenure has been extended in Entra ID.

  • Jira Ticket Closure Status: Select the Jira ticket closure status that will be applied when closing the Jira ticket.

Advanced Settings

Configure the following advanced setting:

Entra ID Employee Import Limit: Enter the maximum number of employees to retrieve for the Jira ticket creation in a single sync.

Report Settings

Configure the following report generation setting:

Enable Integration Report Download: Select this checkbox to allow downloading the integration report in CSV format.

Notification Settings

Configure the following settings for email notifications:

    • Admin's Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
    • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
    • Send Reminder Emails for Contractor Expiration: Select to send the reminder emails before the contractor's expiration date.
    • Select the Reminder Email Schedule: Select the number of days from the dropdown menu to send reminder emails before the contractor's expiration date.

Analytics Settings

Configure the following analytics setting:

Generate Log Insights: Select to visualize integration-related metrics in a dashboard.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details
      If you select Custom, the Attribute Filter dropdown menu is displayed. Select Included Attributes or Excluded Attributes from the dropdown menu to display the corresponding field.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values when choosing the Full Details option.

Attribute Map

The Jira to Entra ID Contractor Management integration maps the attributes from Jira and Entra ID. The attributes displayed in this section depend on the type of synchronization the integration supports:

  • Extension Ticket: For creating a ticket to extend the contractor's tenure, the integration uses Entra ID attributes as the source and maps them to the corresponding Jira Service Management attributes as the target.
  • Contractor Creation: To create a contractor in Entra ID, the integration utilizes Jira Service Management attributes as the source and maps them to the corresponding Entra ID attributes as the target.
  • End Date Update: To extend a contractor's tenure in Entra ID, the integration utilizes Jira Service Management attributes as the source and maps them to the corresponding Entra ID attributes as the target.

Only the source attributes can be modified, as needed, to align with your specific use case.
