Overview

Aquera provides the Microsoft Entra Inbound Provisioning synchronization that allows your organization to scale complex data management by automating the synchronization of users from a source application to the Microsoft Entra ID application. The Microsoft Entra Inbound Provisioning synchronization imports all the users from a source application and performs a bulk synchronization to provision users in the Microsoft Entra ID application. This synchronization uses the bulkUpload API of Microsoft to create or update users as a batch operation using a single bulk request to the Microsoft Entra ID application.

To support user lifecycle management, Aquera provides SCIM Gateway connectors that use the SCIM protocol to carry out the read/write operations on the applications. The SCIM protocol is an IETF standard for automating the exchange of user identity information between applications or IT systems.

This user guide explains how you can set up a Microsoft Entra Inbound Provisioning synchronization in the Aquera Admin portal.

Setting Up a Synchronization

This procedure describes how to set up the Microsoft Entra Inbound Provisioning synchronization.

As a prerequisite, you need to configure SCIM Gateway connectors in the Aquera Admin portal. To explain the procedure, we consider Oracle Cloud HCM as the Source Application and Microsoft Entra ID as the Target Application.

The following Aquera Flow Map shows the high-level visual representation of data flow from the Oracle Cloud HCM application to the Microsoft Entra ID application through Aquera's Microsoft Entra Inbound Provisioning synchronization:

Ensure that the source (Oracle Cloud HCM) and target (Microsoft Entra ID) SCIM Gateway connectors are added and configured correctly in the Aquera Admin portal, before creating the synchronization.

To configure a source connector based on your requirement, refer to the respective connector configuration guides for further details.

Creating a Synchronization

Perform the following steps to create a synchronization:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations on the left panel which displays the list of synchronizations created by the user. 
    Note: The number of synchronization records displayed on this page is based on the display size of the user's screen.
  3. Click Create Synchronization.
  4. Select Microsoft Entra Inbound Provisioning.
  5. Enter a name and a description for the synchronization.
  6. Click Continue. A new page is displayed with the fields and tabs to configure the synchronization.
  7. Select Oracle Cloud HCM as the Source Application and select Microsoft Entra ID as the Target Application on the Configuration tab.
  8. Click Create to create the synchronization.

Basic Details

This section explains the fields to configure the basic details. 

Field Description

Source Application

The application from where the data of users must be retrieved. The created and modified users are retrieved from this application as part of the synchronization.

Target Application

The Microsoft Entra ID application to which the bulk (batch) data of users are created or updated as part of the synchronization. 

Report Only

 Select this checkbox to generate a report that shows the data that would be changed, without actually making those changes. This can be selected for the first instance of synchronization to identify any issues. You can clear this selection after verifying the report and confirming that the result is as expected.

Incremental Mode

 Select this checkbox to run the synchronization in incremental mode. In this mode, the start timestamp of the last run synchronization is always captured, and the source records that have been added or modified after this timestamp will only be considered for the next synchronization. 

Incremental Mode Initial Timestamp

The timestamp that must be considered when you run the synchronization in the Incremental Mode for the first time. The source records that were added or modified after this timestamp will only be considered for the first incremental run of synchronization.

When the Incremental Mode checkbox is selected, ensure to remove the double quotes from this field and provide a valid date string. For example, 2023-04-19T14:15:16.148Z, 2023-04-19T14:15Z, 2023-04-19T14:15-08:00, 2023-04-19T14:15+01:00, 3/15/2024 09:00:00 GMT, 3/15/2024 09:00:00 PDT, 3-15-2024 09:00:00 PDT.

Import Limit

The maximum number of user records that must be retrieved from the source application during a single run of the synchronization. Set this field to unlimited if you do not want to limit the number of imported records. The default value is unlimited which pulls all the users from the source application.

The limit can be set accordingly to verify and debug the source-to-target mapping condition. You can set the import limit to 1 to verify that the user is imported correctly in the Microsoft Entra ID application.

Note: The Import Limit must be set to unlimited when the synchronization runs in the Incremental Mode. Otherwise, a few user updates may be skipped during the incremental runs and the synchronization may take a long time to complete the execution.

Update Limit

The maximum number of user records that are allowed to be updated in the Microsoft Entra ID application during a single run of the synchronization. Set this field to unlimited if you do not want to limit the number of records. When this field is set to 1, it updates a single user in the Microsoft Entra ID application. 

Note: The Update Limit must be set to unlimited when the synchronization runs in the Incremental Mode. Otherwise, a few user updates may be skipped during the incremental runs and the synchronization may take a long time to complete the execution.

Advanced Details

This section explains the fields to configure the advanced details. 

Field Description

Phone Number Formatting

The standard format to which the phone numbers of users must be converted during the synchronization process. Enter one of the following values:

  • disabled: By default, the conversion of phone numbers to a specific format is disabled.
  • automatic: 
  • E164: The E.164 international telephone numbering format is + <country code> <subscriber number including area code>.
  • INTERNATIONAL: International format
  • NATIONAL,country=US: The format that is being followed in a nation. You must replace US with the appropriate country code.

Note: To set the international phone number format of +1-xxx-xxxxx during the synchronization, set the field value as INTERNATIONAL,template=+.-...-.......

Report Mode

The logging mode of synchronization details. This setting controls the level of details to be captured in the synchronization logs.

The following are the valid values listed in the order starting with the least to the most detailed level of logging: 

  • minimal
  • normal
  • debug
  • verbose

The default value is normal. Enter minimal or normal mode to get the basic synchronization details. Enter debug or verbose mode to get a detailed analysis of the synchronization.

For further information on the log-level details for each report mode, refer to the Synchronization Logging Modes guide.

Click Save Changes to save the synchronization after making the required configuration changes.

Mapping of Attributes

You can map the source SCIM attributes to the target SCIM attributes by navigating to the Attribute Map tab. This is required to synchronize the user data from the source application to the Microsoft Entra ID application. You can also write expressions to modify the value of the source attribute before updating the value in the target attribute.

Notes:

  • In this synchronization, the externalId attribute of the Microsoft Entra ID connector is not displayed by default in the Attribute Mapping. You must add it manually using the Add Row To Map. Map the externalId to a unique identifier in the source application such as employee ID, person ID, student ID, or employee number attribute. 
  • Ensure that the userName and displayName attributes of the Microsoft Entra ID connector have an attribute mapping in the SOURCE ATTRIBUTE/EXPRESSION field.

For details, refer to the Attribute Mapping in Synchronizations guide.

Adding and Using Tables

Tables define a name-value pair to create custom attributes. Tables are added in synchronization to create custom source attributes, and POST (sync) them to the Microsoft Entra ID application. Tables are used when the source application attribute is empty or particular data is not sent from the source application, and you want to update a value for that attribute in the Microsoft Entra ID application.

For details on how to add tables and use them in synchronizations, refer to the Using Tables in Synchronizations guide.

Troubleshooting

This section explains the issues that you may encounter when the synchronization runs and the corresponding solutions to resolve those issues.

Description

Resolution

synchronization failure: list is not iterable

Ensure that you have provided the Provisioning API Endpoint of your enterprise application in the Bulk SCIM URL tenant attribute. Refer to the Aquera Tenant Attributes of the Microsoft Entra ID Configuration Guide for further details.

Error 502: Application has refused the connection, retry after sometime

Ensure that you have provided a valid value for the user's externalId, userName, and displayName.

Also, ensure that you have provided the Provisioning API Endpoint of your enterprise application in the Bulk SCIM URL tenant attribute. Refer to the Aquera Tenant Attributes of the Microsoft Entra ID Configuration Guide for further details.

EntrySynchronizationError: Duplicate entries exist in the target

This error will be shown in the Microsoft Entra ID application. To resolve this error, ensure that you have provided a unique externalId for a new user creation.
