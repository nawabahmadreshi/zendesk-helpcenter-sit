Overview

When you set up a Watchdog in Aquera Admin Portal, you will have to setup the Access Policy. Access Policies allow you to specify the groups, applications, OUs (in case of Active Directory), and directory access that users are expected to have or are prohibited from having.

When the Watchdog runs, these policies are evaluated to identify deviations and potential access violations. These policies serve as the basis for auditing user access and detecting compliance issues. Based on the evaluation results, Watchdog generates audit dashboards and reports that provide visibility into access compliance, policy violations, and remediation status. These dashboards help administrators monitor user access, investigate anomalies, and ensure adherence to organizational access policies.

This guide explains the Audit dashboards that will be generated by the Watchdog.

Access the Audit Dashboard

  1. Log in to the Aquera Admin portal.
  2. Click Insights on the left navigation panel. The Insights page is displayed.
  3. Navigate to the Dashboard tab.
  4. Click on the specific dashboard generated by the Watchdog. The details are displayed in the dashboard for auditing purposes. Each of these details is explained below.

Audit Summary

A high-level overview of the audit results is displayed at the top of the page before users drill down into details.

The following details are available in the summary:

  • Total Users Audited: The total number of users evaluated against the configured policies.
  • Total Policy Violations: The number of policy violations detected by the Watchdog. The following are some sample reasons of violations: User missing a required group, user present in a forbidden OU, User assigned to a forbidden application, and so on.
  • Users with Forbidden App Exposure: The number of users having access to the forbidden apps. 
  • Compliance Rate: The policy compliance score in percentage, which indicates the percentage of users who comply with the access policies configured in the Watchdog setup. The widget also shows the target compliance rate and indicates whether the current compliance rate meets the defined target.
  • Critical Aging Violations Open > 90 Days: The number of violations that remain unresolved for more than 90 days.
  • Persistent Violators: The number of users who have repeatedly violated the configured policies. A user is considered a persistent violator when a previously identified violation is resolved, but the same user is found to be in violation again during a subsequent Watchdog run.
  • Mean Time to Remediation (Days): The average number of days taken to resolve violations.
  • Net New Violations (Last 24h): The number of violations that were detected in the last 24 hours.
  • Orphaned Accounts: The number of accounts that exist without a corresponding identity. For example, a user exists in a directory without a corresponding record in the HR system.
