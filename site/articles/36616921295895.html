Uploading and Downloading Files in a File-Based Connector Using a Script

← Back

Overview

For any file-based connector configured in the Aquera Admin portal, you can upload and download files, such as user data, through the Upload tab. The connector retrieves the data from the file for provisioning purposes. Aquera also provides a script-based option to upload and download files.

This guide explains the procedure to upload and download files using the provided script.

Uploading File

You can upload a file using any of the following methods:

  • Using JavaScript
  • Using PowerShell

Notes:

  • The file to be uploaded must be in CSV, XLS/XLSX, txt, DAT, or ZIP format.
  • Users with Owner, Superadmin, or Admin roles can upload files.
  • To upload a file using JavaScript, you must have Node.js installed on your system.
  • To upload a file using PowerShell, you must have the PowerShell module installed on your system.

Using JavaScript

Perform the following steps to upload a file:

  1. Download the attached uploadFile.js file. This file contains the script used to upload a file.

  2. Open the JS file in a text editor and modify the following values:

    • Replace the token value with the API token that you have generated in your Aquera tenant. Refer to the Aquera Admin User Guide to learn how to generate a token.
    • Replace the uri value with your connector API endpoint URL.
    • Replace the filePath value available towards the end of the script with the correct folder path (including file name) containing the file to upload.
  3. Save the JS file.
  4. Open a terminal.
  5. Navigate to the location where the JS file is downloaded.
  6. Run the script from the terminal using the following command:
    node uploadFile.js
  7. After the script execution is complete successfully, you can navigate to the connector configured in the Aquera Admin portal to verify the uploaded file.

Using PowerShell

Perform the following steps to upload a file:

  1. Download the attached uploadFile.ps1 file. This file contains the script used to upload a file.

  2. Open the PS file in a text editor and modify the following values:

    • Replace the filePath value with the correct folder path (including file name) containing the file to upload.
    • Replace the uri value with your connector API endpoint URL.
    • Replace the token value with the API token that you have generated in your Aquera tenant. Refer to the Aquera Admin User Guide to learn how to generate a token.

  3. Save the PS file.
  4. Open a terminal.
  5. Navigate to the location where the PS file is downloaded.
  6. Run the script from the terminal using the following command:
    ./uploadFile.ps1
  7. After the script execution is complete successfully, you can navigate to the connector configured in the Aquera Admin portal to verify the uploaded file.

Downloading File

You can download an uploaded file using any of the following methods:

  • Using JavaScript
  • Using PowerShell

Notes:

  • There must be a file uploaded to the Aquera storage or SFTP storage.
  • Only the latest file that was uploaded successfully will be downloaded.
  • Users with Owner, Superadmin, or Admin roles can upload files.
  • To download a file using JavaScript, you must have Node.js installed on your system.
  • To download a file using PowerShell, you must have the PowerShell module installed on your system.

Using JavaScript

Perform the following steps to download a file:

  1. Download the attached downloadFile.js file. This file contains the script used to download the uploaded file.

  2. Open the JS file in a text editor and modify the following values located towards the end of the script:

    • Replace the tenantId value with your Aquera tenant ID.
    • Replace the applicationId value with the ID of the file connector (application).
    • Replace the token value with the API token that you have generated in your Aquera tenant. Refer to the Aquera Admin User Guide to learn how to generate a token.
  3. Save the JS file.
  4. Open a terminal.
  5. Navigate to the location where the JS file is downloaded.
  6. Run the script from the terminal using the following command:
    node downloadFile.js
  7. After the script execution is complete, the latest active file associated with the connector is downloaded to the location where the JS file is available.

  2. Open the PS file in a text editor and modify the following values located towards the end of the script:

    • Replace the applicationId value with the ID of the file connector (application).

  3. Save the PS file.
  4. Open a terminal.
  5. Navigate to the location where the PS file is downloaded.
  6. Run the script from the terminal using the following command:
