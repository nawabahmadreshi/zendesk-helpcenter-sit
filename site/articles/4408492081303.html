Overview

isolved HCM is a platform to perform HR tasks, such as processing payroll, managing employees' time and attendance, onboarding new employees, and tracking and recruiting applicants to new positions.

This guide helps users create an isolved HCM connector on the Aquera platform.

For further details on the isolved HCM application, visit the isolved HCM website.

Supported Features

SCIM

The isolved HCM connector supports the following features as per the System for Cross-Domain Identity Management (SCIM) protocol:

Resource Import Provisioning
Users
  • Full import
  • Filter users
  • Create a user
  • Update a user
Employees
  • Full import
  • Filter employees
  • Create an employee
  • Update an employee

Prerequisites

The following are the prerequisites:

  • Administrator access to the isolved HCM instance: You must have the admin credentials of your isolved HCM instance.
  • Client ID and Client Secret: You must have a client ID and client secret, provided by the Aquera Support team for authentication. Refer to the Authentication section for further details.

Connector Configuration

Perform the following steps to configure the isolved HCM connector:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Applications and click Add Application.
  3. Search for isolved HCM and click the icon.
  4. Enter the values for the following fields under the Configuration tab:
    • Basic Details:
      • Display Name: Enter a display name for your application.
      • Description: Enter a description for your application.
      • Tenant: Enter the hostname available in the URL for your isolved HCM instance. For example, if the URL is https://devpartner.isolvedhcm.com/, enter devpartner.isolvedhcm.com.
      • Source Connection Type: Select Direct Connection.
        Note: This field is available only if you have access to Aquera's File Collection module.
      • Source Owner: Enter valid email IDs of the source owners.
      • Subscribe to error notification: Enable the toggle button to send notification emails to source owners for errors in the user data transactions. Click Edit Preference to select the error types for which you want to receive notifications. After you configure your preference, you will receive error notifications only for the selected error types. By default, all error types are selected.
      • Pick your protocol: Select the protocol from the dropdown list based on your requirement. SCIM is the default value.
      • Copy this URL: This field displays the base URL of the connector API endpoint. The URL is populated based on your selected protocol. Copy and paste this URL while provisioning to the identity service provider or performing OpenAPI operations, if supported.
    • Tenant Attributes: Refer to the Aquera Tenant Attribute section for details.
  5. Click Create to create the isolved HCM connector.

Refer to the Getting Started with Connectors guide to learn more about connector setup, including different tabs of the connector configuration page.

Aquera Tenant Attribute

Client Name: Enter the isolved client name.

Authentication

The isolved HCM connector supports Basic Authentication. Provide the client ID as the username and the client secret as the password for authentication. The client ID and client secret will be provided by the Aquera Support team.
