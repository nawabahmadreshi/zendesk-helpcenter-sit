Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize employees from Namely to Entra ID. It also provides options to configure rules for assigning users to groups and for assigning licenses to users. 

Key Functions

The Entra ID / Namely integration performs the following functions:

  • Retrieves the data of employees from Namely and provisions them as users in Entra ID.
    • For new users, the integration generates a unique UPN, email address, and password for each user, depending on the configuration.
  • Updates or deactivates Entra ID users based on the conditions configured in the integration.
  • Assigns users to Entra ID groups or removes them based on the configured rules.
  • Assigns a license to users or removes it based on the configured rules.
  • Supports onboarding and offboarding of users in Entra ID based on the date offset configured in the integration.
  • Generates reports and logs, and sends notification emails based on the integration settings.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Namely: For configuration details, refer to the Namely Configuration Guide.
    • Entra ID: For configuration details, refer to the Microsoft 365 Configuration Guide or the Microsoft 365 App Registration Configuration Guide, depending on the type of authentication to be used for the connector.

Integration Setup

This section outlines how the integration must be configured to synchronize employees from Namely to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID / Namely.
  5. Enter a name and a description for the integration.
  6. Select your Namely and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Namely and Entra ID application connectors that you have selected for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and settings according to your use case. It has the following settings:

Integration Features

Enable the following features based on your requirements:

  • Enable user management: Select this checkbox to sync user data from Namely to Entra ID. When selected, the User Management section and the Target User Map attribute map setting section are displayed for further configuration. If this checkbox is not selected, the user data will not be retrieved and processed by the integration.
    Note: This checkbox must be selected for the user sync.
  • Enable User Date Offset: Select this checkbox to enable date offset for user onboarding and offboarding operations in Entra ID. When selected, the User Date Offset section is displayed to set the onboarding and offboarding offset days and time. If this checkbox is not selected, the users will be onboarded or offboarded immediately during the next integration run, after the corresponding event occurs in Namely.
  • Enable Entra Group Management: Select this checkbox to enable assigning and removing users from Entra ID groups. When selected, the Group Rules tab is displayed to set up rules for group assignments. If this checkbox is not selected, user assignments to groups will not be supported.
  • Enable Entra License Management: Select this checkbox to enable assigning and removing licenses for Entra ID users. When selected, the License Rules tab is displayed to set up rules for license assignments. If this checkbox is not selected, license assignments to users will not be supported.

General Settings

Configure the following general settings:

  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When selected, the Log Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When selected, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When selected, the Log Insights section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When selected, the Notifications section is displayed for further configuration.

Synchronization Settings

This section enables you to configure the integration to run in full sync or incremental sync mode.

  • Full sync frequency: Select the frequency by which the integration must run in full sync mode, where all user profiles will be synced from Namely to Entra ID during integration runs. Select None to run the integration in incremental mode, where only the user profile changes in Namely since the last integration run will be processed by the integration.

User Management

This section details the following settings for synchronizing the user data.

Source Filter

  • Source filter: This setting enables you to filter user record(s) from Namely and sync only the filtered worker records to Entra ID. You can filter by email ID, or select Custom to define a JavaScript-based filter expression. Select None to apply no filtering and include all user records.
  • Email: Enter the email ID to filter a user record. This field is displayed only if you have selected Email in Source Filter.
  • Custom source filter expression: Provide the JavaScript expression to filter the user records from Namely. This field is displayed only if you have selected Custom in Source Filter.

Safeguard Limits

  • Enable import limit: Select Yes to limit the number of user records processed in a single integration run. Select No if you do not want to set a limit.
  • Import limit: Enter the maximum number of user records that are allowed to be processed by the integration in a single run. This setting is displayed only if you have configured Yes in Enable import limit.
  • Limit type: Indicate whether the Import limit that you have set is a count or a percentage of user records.

Creation Settings

  • Create condition type: Select the type of condition that you want to configure for creating users in Entra ID:
    • None: Disables user creations.
    • Basic: Allows you to select specific user statuses in Namely that need to be considered for creating users in Entra ID.
    • Custom: Enables you to set the conditions for creating users using a JavaScript-based expression.
  • Eligible user statuses for creation: Select the user statuses that need to be considered for creating the users in Entra ID. For example, if you select Active, only the Active users will be created in Entra ID.
    Note: This field is displayed only if you have selected Basic as the Create condition type.
  • Custom create condition: Define a JavaScript-based expression to specify the conditions for creating users in Entra ID.
    Note: This field is displayed only if you have selected Custom as the Create condition type.

Update Settings

  • Update condition type: Select the type of condition that you want to configure for updating users in Entra ID:
    • None: Disables user updates.
    • Basic: Allows you to select specific user statuses in Namely that need to be considered for updating users in Entra ID.
    • Custom: Enables you to set the conditions for updating users using a JavaScript-based expression.
  • Eligible user statuses for update: Select the user statuses that need to be considered for updating the users in Entra ID. For example, if you select Active, only the Active users will be updated in Entra ID.
    Note: This field is displayed only if you have selected Basic as the Update condition type.
  • Custom update condition: Define a JavaScript-based expression to specify the conditions for updating users in Entra ID.
    Note: This field is displayed only if you have selected Custom as the Update condition type.

Deactivation Settings

  • Deactivate condition type: Select the type of condition that you want to configure for deactivating users in Entra ID:
    • None: Disables user deactivations.
    • Basic: Allows you to select specific user statuses in Namely that need to be considered for deactivating users in Entra ID.
    • Custom: Enables you to set the conditions for deactivating users using a JavaScript-based expression.
  • Eligible user statuses for deactivation: Select the user statuses that need to be considered for deactivating users in Entra ID. For example, if you select Inactive, only the inactive users will be deactivated in Entra ID.
    Note: This field is displayed only if you have selected Basic as the Deactivate condition type.
  • Custom deactivate condition: Define a JavaScript-based expression to specify the conditions for deactivating users in Entra ID.
    Note: This field is displayed only if you have selected Custom as the Deactivate condition type.

User Correlation Settings

This section details the correlation settings for matching users between Namely and Entra ID.

  • Source correlation attribute: Select the user attribute in Namely that must be used for finding a matching user record in Entra ID.
  • Target correlation attribute: Select the corresponding user attribute in Entra ID that must be used for finding the matching user record.

Policy

This section details the settings for generating unique UPN, email, and password while creating users in Entra ID:

Unique UPN Generation

  • Enable unique UserPrincipalName generation: Select this checkbox to enable the integration to generate unique User Principal Names (UPN) for the newly created users in Entra ID. 
  • Unique UserPrincipalName pattern builder: Enter the prompt for generating a unique UPN for Entra ID users. This is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate UPNs during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
  • Use source work email if present: Select this checkbox to assign the work email address available for the user in Namely as the UPN in Entra ID. If the work email address is not available for the user, the Unique UserPrincipalName pattern builder will be used to generate the UPN. 

Unique Email Generation

  • Enable unique email generation: Select this checkbox to enable the integration to generate unique email IDs for the newly created users in Entra ID. 
  • Email generation mode: Select whether the email address has to be the same as the UPN or different from UPN. If you select Different Email, the Unique Email Pattern Builder field is displayed to provide the pattern for the email address.
  • Unique email pattern builder: Enter the prompt for generating a unique email for Entra ID users. This is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate email IDs during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
    Note: This field is displayed only if you have selected Different Email as the Email Generation Mode.
  • Use source work email if present: Select this checkbox to assign the work email address available for the user in Namely as the email address in Entra ID. If the work email address is not available for the user, the Unique UserPrincipalName Pattern Builder will be used to generate the UPN. 

Unique Password Generation

  • Enable password generation: Select this checkbox to enable the integration to generate unique passwords for the newly created users in Entra ID. 
  • Password pattern builder: Enter the prompt for generating unique passwords for newly created Entra ID users. This is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate passwords during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

User Date Offset

This section provides date offset settings that control when users must be onboarded or offboarded in Entra ID relative to their onboarding or offboarding dates in Namely.

Onboarding User Date Offset

This section provides settings to control the onboarding date and time of users to Entra ID.

  • Onboarding date offset: Select whether you want to onboard users to Entra ID before or after their onboarding date in Namely. If you do not want to set a date offset, select No Offset from the list. In this case, the user will be provisioned to Entra ID when their onboarding date is reached and the integration runs.
  • Days of offset before onboarding: Enter the number of days before the onboarding date when new users have to be provisioned to Entra ID. For example, if you configure 5 in this field and the onboarding date of a user in Namely is 27-April, the user will be provisioned to Entra ID on 22-April, which is 5 days before the onboarding date.
    Note: This field is displayed only if you have selected Offset before onboarding date from the Onboarding date offset drop-down list.
  • Days of offset after onboarding: Enter the number of days after the onboarding date when new users have to be provisioned to Entra ID. For example, if you configure 5 in this field and the onboarding date of a user in Namely is 27-April, the user will be provisioned to Entra ID on 02-May, which is 5 days after the onboarding date.
    Note: This field is displayed only if you have selected Offset after onboarding date from the Onboarding date offset drop-down list.
  • Onboarding offset priority: Shows UTC as the standard timezone. UTC ensures all onboarding time calculations are standardized regardless of the user location.
  • Onboarding offset time: Select the exact time of the day after which users will become eligible for onboarding to Entra ID. Users who meet the onboarding date offset criteria will be provisioned only when the integration runs as per its defined schedule, and at or after the configured time.

Offboarding User Date Offset

This section provides settings to control the offboarding date and time of users in Entra ID.

  • Offboarding date offset: Select whether you want to offboard users in Entra ID after their offboarding date in Namely or to disable the offboard offset date. If you do not want to set a date offset, select No Offset from the list. In this case, the user will be deprovisioned in Entra ID when their offboarding date is reached and the integration runs.
  • Days of offset after offboarding: Enter the number of days after the offboarding date when terminated users have to be deprovisioned in Entra ID. For example, if you configure 2 in this field and the offboarding date of a user in Namely is 27-April, the user will be deprovisioned in Entra ID on 29-April, which is 2 days after their offboarding date.
    Note: This field is displayed only if you have selected Offset after offboarding date from the Offboarding date offset drop-down list.
  • Offboarding offset priority: Shows UTC as the standard timezone. UTC ensures all offboarding time calculations are standardized regardless of the user location.
  • Offboarding offset time: Select the exact time of the day after which users will become eligible for offboarding in Entra ID. Users who meet the offboarding date offset criteria will be deprovisioned only when the integration runs as per its defined schedule, and at or after the configured time.

Attribute Map

Group Rules

The Group Rules tab provides options to configure how group memberships must be handled through this integration. 

Group Assignment Rules

You can set up rules for assigning users to the EntraID groups. In each rule, you can set conditions that control how users are assigned to groups.

Perform the following procedure to create group assignment rules:

  1. Define the conditions for assignment to groups using one of the following ways:
    • Using Condition Builder
    • Using Expressions
  2. If required, click Add Row to add another condition within the same rule. When multiple rows are added within a rule, an OR condition is applied to the rows. That means, for each user, the integration evaluates the conditions in each row sequentially. If the conditions defined in a row are satisfied for a user, the user is assigned to that group and the remaining rows in the rule are not evaluated for that user.
  3. If required, click Add New Rule to add another rule with a set of rows/conditions for group assignments. When multiple rules are set, the integration evaluates every rule for each user.
  4. [Optional] Select Remove Groups on Deactivation to remove a user from any Entra ID groups when they are deactivated in Namely.

 

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable logging parameter in the Parameters section.

  • Select Log Type: Select a log type. This will decide the level of detail to be included in logs. For example, choosing Employee Number with All Data includes employee number, operations, field names, and all values (PII and non-PII).

  •  Select Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the log.
    • Include List: Select this option to include specific attributes in the log.
      • Custom Inclusion List: Enter the attributes to be included in the log. This parameter is available only if you select Include List in the Select Log Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the log.
      • Custom Exclusion list: Enter the attributes to be excluded from the log. This parameter is available only if you select Exclude List in the Select Log Level parameter.
  • Enter PII fields: Enter the fields that contain personally identifiable information (PII) to be included in logs. 
  • Enter GUID fields: Enter the fields that contain globally unique identifiers to ensure they are handled correctly in logs.
  • Enter EmployeeId fields: Enter the fields that represent employee identification values to be displayed consistently in logs. For example, adding associateId ensures the employee identifier is included in logs.

Notifications

This section enables you to customize the notification settings. It has the following settings:

Note: This section appears only if you have enabled the Enable notifications setting in the Parameters section.

Notification Template Settings

Configure the following settings to customize notifications:

  • Send email notifications for success: Select this checkbox to send an email notification on successful integration runs. When selected, the Success Execution section is displayed for further configuration.
  • Send email notifications for failures: Select this checkbox to send an email notification on failed syncs in an integration run. When enabled, the Failure Execution section is displayed for further configuration.

Success Execution

Configure the following notification settings for successful integration runs:

  • Email Subject: Enter the content that must be used as the subject in email notifications.
  • Email Body: Enter the content that must be used as the email body for notifications sent upon failed form syncs.
