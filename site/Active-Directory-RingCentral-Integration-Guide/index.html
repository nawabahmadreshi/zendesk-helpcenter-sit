Active Directory / RingCentral Integration Guide

← Back to index

Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating data synchronization from Active Directory users to RingCentral directory.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, every user in Active Directory will be created, updated, or deprovisioned in RingCentral with an extension phone number according to the schedule.

This guide explains how to set up the Active Directory / RingCentral integration to synchronize user data from Active Directory and assign extensions to users in the RingCentral directory.

Key Features

The Active Directory / RingCentral integration performs the following functions:

  • Synchronizes user data from Active Directory to RingCentral.
  • All the newly created users will be assigned with a phone number in RingCentral.
  • Limits the users create, update, and termination operations.
  • Sends notifications upon success, failure, or skipped record events via email.
  • Generates detailed logs and reports.
  • Supports full synchronization. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Applications in the Aquera Admin Portal:
    • Active Directory Connector: Ensure you add and configure Active Directory in the Aquera Admin portal. For details, refer to the Active Directory Configuration Guide.
    • RingCentral Connector: Ensure you add and configure the RingCentral connector in the Aquera Admin portal. Refer to the RingCentral Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Active Directory to RingCentral.

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Active Directory / RingCentral. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Select the Active Directory and RingCentral connectors from the corresponding drop-down lists.
  7. Click Continue. The integration is created.

Once created, the following tabs are displayed:

  • Overview
  • Configuration
  • Active Directory
  • RingCentral
  • Attribute Map
  • Notification
  • Execution
  • Schedule

Overview

In this tab, you can view the integration details and a list of integration settings to be enabled based on the use cases.

Integration Settings

The following are the supported integration settings:

  • Enable User Creation: Turn on this toggle button to create users in RingCentral from Active Directory.
  • Enable User Updation: Turn on this toggle button to update the existing user's data in RingCentral from Active Directory. 
  • Enable user attributes writeback to Active Directory from RingCentral: Turn on this toggle button to writeback all the newly created and updated users data from RingCentral to Active Directory.

Configuration

This section allows you to configure the source and target applications that must be used for this integration.

Applications

Configure the following parameters to define the source and target applications that must be used in this integration:

  • RingCentral: Select the RingCentral connector from the dropdown menu that must be used as the target application for this integration.
  • Active Directory: Select the Active Directory connector from the dropdown menu that must be used as the source application for this integration.

Active Directory

This tab allows you to define filters for Active Directory users to determine which users should be included in the synchronization.

Active Directory Settings

Configure the following settings to define the filters on the Active Directory users:

  • Enable Active Directory OU filter to fetch Users: Turn on the toggle button to select specific OU's in the Active Directory from which the users must be filtered for syncronization. When enabled, the following parameter is displayed for further configuration.
    • fetch Users only from specific OUs: Select one or more OU's from the dropdown list from which the users must be filtered for syncronization in the Active Directory. Only users belonging to the specified Organizational Units (OUs) are fetched from Active Directory for synchronization, instead of retrieving all users across the directory.
  • Enable user filter for complex conditions: Turn on the toggle button to set filter conditions upon the selected OU's in the fetch Users only from specific OUs parameter. When enabled, the following fields are displayed.
    • Select filter type: Select a filter type from the drop-down menu that should be applied to Active Directory users.
      • AI Prompt: Select this option to define a filter condition using Aquera Intelligence (AI Assistant). When selected, the following field is displayed for further configuration.
        • Provide prompt to filter AD users: Enter the prompt to generate JS expressions using AI Assistant.
      • JS Expression: Select this option to define a custom JavaScript expression to filter users dynamically. When selected, the following field is displayed for further configuration.
        • JavaScript logical expression filter for Users in AD: Enter a JavaScript expression that defines the filter condition.
      • Attribute Filter: Select this option to filter users records by providing the required attribute and value. When selected, the following fields are displayed:
        • Select attributes to filter: Select an attribute from the dropdown menu that must be used for filtering. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
        • Provide values to filter based on the above provided attribute key: Enter the value for the attribute selected in the Select attributes to filter parameter. Users with this attribute value will be filtered and considered for synchronization.

      • Attribute Filter: Select this option to filter users records by providing the required attribute and value. When enabled, the following fields are displayed:
        • Select attributes to filter: Select an attribute from the dropdown menu that must be used for filtering. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
        • Provide values to filter based on the above provided attribute key: Enter the value for the attribute selected in the Select attributes to filter parameter.

Correlation Settings

This section allows you to correlate attributes that matches records in Active Directory and RingCentral. The following are the configuration settings:

  • Select correlation key in Active Directory: Select the key to uniquely identify a user in Active Directory inorder to match with RingCentral user. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
  • Select correlation key in RingCentral: Select the key to uniquely identify a user in RingCentral inorder to match with Active directory user. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.

RingCentral Termination Settings

The following are the supported integration settings:

  • Enable User Deactivation: Turn on this toggle button to deactivate the users assigned extension number without removing the users data in RingCentral. When enabled, the following parameter is displayed:
    • Delete Users after Deactivation: Turn on this toggle button to delete the users record from RingCentral directory once the user is deactivated.
  • Enable termination in future: Turn on this toggle button to terminate RingCentral user accounts after a specified number of days from the date the user is marked as terminated in Active Directory (AD). When enabled, the following parameters are displayed for further configuration:
    • Select attribute which contains termination date in AD: Select an attribute from the dropdown menu that contains the termination date in the Active Directory.
    • Provide a number to terminate the user in RingCentral, X days after termination in AD: Enter the number of days to delay user termination in RingCentral after termination in Active Directory. For example, enter 5 to terminate the user in RingCentral 5 days after the user is terminated in Active Directory.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Notification

The Notification tab allows you to configure the notification settings for this integration:

  • Enable Notifications: Turn on this toggle to configure email notifications for integration events, such as successful, failed execution, or skipped records and to define the recipients and report details included in the notification emails. When enabled, the Notification Template Settings section is displayed for further configuration.

Notification Template Settings

Configure the following settings to define when notification emails must be sent.

  • Send Email Notification For Success: Turn on this toggle button to send an email notification when the integration completes successfully. For example, when the integration finishes without errors, a success notification is sent to the configured recipients. When enabled, Success Execution section is displayed for further configuration.
  • Send Email Notification For Failure: Turn on this toggle button to send an email notification when the integration fails. For example, if the integration encounters an error, a failure notification is sent to the configured recipients. When enabled, Failure Execution section is displayed for further configuration.
  • Send email notification for skipped records: Turn on this toggle button to send an email notification when the integration skips executing users records. For example, if the integration encounters an error, a failure notification is sent to the configured recipients. When enabled, Skipping Execution section is displayed for further configuration.
  • Attach report to email: Turn on this toggle button to include the integration report as an attachment in notification emails. For example, enabling this option attaches the execution summary to the notification email. When enabled, Notification Report Settings section is displayed for further configuration.

Success Execution

Configure the following settings to define the recipients and content of success notification emails.

  • Send To: Enter the email addresses that should receive success notification emails. For example, add admin@company.com to receive confirmation when the integration completes successfully.
  • CC: Enter additional email addresses to receive a copy of success notifications. For example, include teamlead@company.com to keep the team informed.
  • Email subject: Enter the subject line used for success notification emails. For example, Integration - Success Report.
  • Integration Execution Success Email Body: Enter the message content sent when the integration completes successfully. For example, include a summary of the run, such as the number of users processed and any key updates.

Failure Execution

Configure the following settings to define the recipients and content of failure notification emails.

  • Send To: Enter the email addresses that should receive failure notification emails. For example, add support@company.com to receive alerts when the integration encounters an error.
  • CC: Enter additional email addresses to receive a copy of failure notifications. For example, include itadmin@company.com to notify system administrators.
  • Email Subject: Enter the subject line used for failure notification emails. For example, Failed Report [Action Required].
  • Email Body: Enter the message content sent when the integration fails. For example, include error details and troubleshooting guidance.

Skipping Execution

Configure the following settings to define the recipients and content of skipped records notification emails.

  • Send To: Enter the email addresses that should receive skipped records notification emails. For example, add support@company.com to receive alerts when the integration encounters a user record skip.
  • CC: Enter additional email addresses to receive a copy of Skipped Records notifications. For example, include itadmin@company.com to notify system administrators.
  • Email Subject: Enter the subject line used for skipped records notification emails. For example, Skip Records.
  • Email Body: Enter the message content sent when the integration skips users records while executing. For example, include error details and troubleshooting guidance.

Notification Report Settings

Configure the report information included in notification emails.

  • Select Report Type: Select the type of report data included in the notification email. The following are the available options:
    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Report Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.

Advanced Settings

This section enables you to customize the advanced settings for the integration. It has the following sub-sections for configuration:

Rollout Configuration

The Rollout Configuration section allows you to enable the report and debud settings for this integration. It has the following settings for further configuration:

  • Enable Report-Only Mode: Turn on this toggle button to execute the integration without making changes by only generating the report.
  • Enable Debug Mode: Turn on this toggle button to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Settings section is displayed for further configuration.
Debug Settings

Configure the following settings for debugging the integration:

  • Enable User Filter: Turn on this toggle button to apply a filter for debugging. When enabled, the following fields are displayed.
    • Select key to filter users in AD: Select an attribute from the dropdown menu by which you want to filter users from Active Directory.
    • Add values to filter: Enter a value based on the above selected attribute key in which you want to filter users from Active Directory.

Limit Settings

This section allows you to configure operation limits for Create, Update, and Termination actions in the RingCentral integration. It has the following sub-sections for configuration:

  • Enable Create Limit: Enable this toggle button to define a limit on the create operation. The RingCentral Create operations will stop automatically once the configured limit is reached. The Update and Termination operations will continue without interruption. When enabled, the following field is displayed for further configuration:
    • Provide create limit: Provide the maximum number of Create operations allowed during the integration run.
  • Enable Update Limit: Enable this toggle button to define a limit on the update operation. The RingCentral Update operations will stop automatically once the configured limit is reached. The Create and Termination operations will continue without interruption. When enabled, Provide update limit field will be displayed.
    • Provide update limit: Provide the maximum number of Update operations allowed during the integration run.
  • Enable Termination Limit: The RingCentral Termination operations will stop automatically once the configured limit is reached. Create and Update operations will continue without interruption. When enabled, Provide termination limit field will be displayed.
    • Provide termination limit: Provide the maximum number of Termination operations allowed during the integration run.

PII Settings

This section allows you to configure how information is detected, handled, and protected. It has the following settings for further configuration:

  • PII Fields: Specifies personally identifiable information that should be masked or protected. For example, fields such as email or phoneNumber can be redacted to prevent exposure in logs. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
  • GUID Fields: Defines fields that contain unique identifiers. For example, fields like id or objectId can be marked to ensure proper handling and tracking. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
  • Employee ID Fields: Identifies fields used as employee identifiers. For example, fields such as employeeId or barcode can be specified to ensure consistent identification across systems. Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
