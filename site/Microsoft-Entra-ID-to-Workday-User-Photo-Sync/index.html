Microsoft Entra ID to Workday User Photo Sync

Overview 

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

The Microsoft Entra ID to Workday User Photo Sync keeps employee profile photos in Workday aligned with the photos stored in Microsoft Entra ID. Microsoft Entra ID is used as the trusted source for employee photos. Workday uses these photos to maintain accurate and consistent employee profiles for everyday workforce operations.

When the integration runs, it reviews users in Microsoft Entra ID and checks whether each user’s profile photo has changed since the previous run. Employees are matched between the two systems using a shared identifier, such as Employee ID. It ensures updates are applied to the correct Workday record.

  • If a photo is updated in Microsoft Entra ID, the updated photo is copied to the employee’s Workday profile. If no change is detected, the employee is skipped, and no update is made. 

  • If a photo is removed in Microsoft Entra ID, the integration records the change for tracking and reporting purposes but leaves the existing photo in Workday unchanged.

The sync runs in controlled batches, respects system limits, and supports a report-only mode, allowing you to review potential changes without updating any data. This ensures that employee photos are up to date.

Key Functions

The Microsoft Entra ID to Workday User Photo Sync integration performs the following key functions:

  • Retrieves user photo information from Microsoft Entra ID: The integration reads users from Microsoft Entra ID and checks the current state of each user’s profile photo, including whether a photo exists and whether it has changed since the last run.

  • Matches Entra users with Workday employees: Each Entra user is matched to the correct Workday employee using a shared identifier, such as an employee ID, so photo updates are applied to the correct worker record.

  • Updates employee profile photos in Workday: When a photo change is detected in Microsoft Entra ID, the updated photo is copied into the employee’s Workday profile to keep records current.

  • Tracks photo removals from Entra ID: If a photo is removed in Microsoft Entra ID, the integration does not remove the existing photo from Workday.

  • Skips unchanged photos to reduce processing: If a user’s photo has not changed, the integration skips that user and makes no update in Workday.

Integration Setup

This section describes the configuration steps required to set up Microsoft Entra ID to Workday User Photo Sync integration in Aquera.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left navigation panel.
  3. Click Create Integration.
  4. Select Microsoft Entra ID to Workday User Photo Sync.
  5. Enter a name and description for the integration.
  6. Click Continue. The Configuration page is displayed.

Configuration

The following sections are available under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to select:

  • Microsoft Entra ID (Azure AD) is the source application from which user profile photos are retrieved.
  • Workday is the target application for updating employee profile photos.

Both connectors must be configured with valid credentials and appropriate permissions to support user lookup and photo writeback.

Parameters

The Parameters section allows you to customize the operation of the Microsoft Entra ID to Workday User Photo Sync integration. These settings control the behavior of the integration and help tailor it to your organization’s requirements.

Integration Features 

Configure the following integration feature:

  • Enable Photo Sync: Allows the integration to update employee profile photos in Workday when photo changes are detected in Microsoft Entra ID. This setting updates photos only and does not remove existing photos from Workday.

General Settings

Configure the following general settings:

  • Enable Notifications: Select to send integration-related notifications, including success, warning, and error summaries, after each run.
  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Log Insights: Select to visualize the integration-related metrics.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.

Correlation Settings

Configure the following correlation settings:

  • Entra Correlation: Selects the field in Microsoft Entra ID that the integration uses to identify each user. When an Employee ID is selected, the integration reads the employee ID from the Entra user profile and uses it to find the matching employee in Workday.

  • Workday Correlation: Selects the Workday field that the integration uses to identify employee records. When an Employee ID is selected, the integration looks for an employee record in Workday with the same Employee ID.

    These settings ensure that the integration correctly matches each Entra user to the right employee record in Workday before updating profile photos.

Using the same correlation attribute in both systems provides a consistent and reliable way to match users and prevents photos from being updated on the wrong employee records.

Entra User Settings

Configure the following Entra user settings:

  • Source Update Condition: Select which Microsoft Entra ID users are included when the integration checks for photo updates.

    The following options are available:

    • All Users: Includes every user in Microsoft Entra ID, regardless of their account status.
    • All Active Users: Includes only users with active Entra ID accounts, skipping disabled users.

Attribute Map

The Attribute Map defines which photo-related attributes in Workday are updated when a change is detected in Microsoft Entra ID. In this integration, the primary attribute, employee profile photo, is managed. When a photo is added, updated, or removed in Entra ID, the mapped attribute ensures that the corresponding change is applied to the correct employee record in Workday.
