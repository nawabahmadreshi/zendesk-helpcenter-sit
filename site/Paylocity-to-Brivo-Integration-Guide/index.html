Paylocity to Brivo Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Paylocity to Brivo integration, which synchronizes user data from Paylocity to Brivo to support user lifecycle and group management.

To identify user records across both systems, the integration uses the Employee ID attribute from Paylocity and matches it to the External ID in Brivo as a correlation attribute.

Key Functions

The Paylocity to Brivo integrations provide the following functions:

  • User Lifecycle Management: Handles user creation, updates, and deactivation (suspend/delete) from Paylocity to Brivo.

  • Group Rules and Assignment: Assigns users to specific groups in Brivo based on work location attributes like location, country, state, and city. Group assignments are ignored if the conditions are not met or if work location details are missing.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Paylocity: Refer to the Paylocity Configuration Guide for details.
    • Brivo: Refer to the Brivo Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Paylocity to Brivo.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Brivo Security Suite / Paylocity in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following integration scenarios based on your requirements:

  • Enable User Lifecycle Management: Select this checkbox to synchronize users' data from Paylocity to Brivo. When enabled, the Manage Users section is displayed to configure further integration settings.
  • Enable Group Assignment: Select this checkbox to enable group assignments in Brivo. When enabled, the Manage Group section is displayed to configure further integration settings.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insight Settings section is displayed for further configuration.

Manage Users

The Manage Users section enables you to customize the user integration settings according to your specific use case. It synchronizes user data from Paylocity to Brivo. It has the following settings:

Note: This section appears only if the Enable User Lifecycle Management setting is enabled in the Parameters section.

Source Filter Settings

Configure the following source filter settings:

  • Filter by Company ID: Enter one or more company IDs to limit which users are synced. Only users associated with the specified company IDs will be included in the integration. For example, specifying AQUERADM1 and AQUERADM2 ensures that only users from those companies are synced.
  • Statuses to be Considered as Active: Select the user statuses that should be considered as active during the sync. Users whose current status matches any of the selected values will be handled as active records in the integration. For example, selecting Active and Leave of Absence ensures users in those statuses are processed as active records.
  • Statuses to be Considered as Deactivated: Select the user statuses that should be considered deactivated during the sync. Users whose current status matches any of the selected values will be handled as deactivated records in the integration. For example, selecting Terminated, Retired, or Deceased ensures users in those statuses are processed as deactivated records.
  • Rehire Employee: Select this option to automatically reactivate a user’s Brivo account when they become active again in Paylocity.
  • Enable Duplicate Employee Handling: Select this option to detect duplicate employee records and apply the selected primary status rule to determine which record is treated as the primary employee. When enabled, the following field will appear for further configuration.
    • Duplicate Employee Primary Status: Select the employee status that will be used as the primary record when duplicate employees are detected. The following are the available options:
      • ACTIVE_EMPLOYEE: Select this option to consider the active employee record as the primary record.
      • INACTIVE_EMPLOYEE: Select this option to consider the inactive employee record as the primary record.

Note: The selections in the Statuses to be Considered as Active and Statuses to be Considered as Deactivated fields must be mutually exclusive. If any values overlap, the integration will fail validation.

User Credentials

Configure the following user credentials settings:

  • Attach mobile pass to user: Select this option to enable users to use the Mobile Pass for access. When enabled, the following fields will appear for further configuration.
    • Mobile Pass Email Field: Select the appropriate email field from the dropdown menu to be used for the mobile pass registration. The following are the available options:
      • Work Address Email: Select this option to use the user’s work email address for mobile pass registration.
      • Home Address Email: Select this option to use the user’s home email address for mobile pass registration.
    • Credentials Attach Condition: Select the condition that determines when credentials are attached to a user. The following are the available options:
      • Create and Update: Select this option to attach credentials when a user record is created and whenever an existing user record is updated.
      • Create Only: Select this option to attach credentials only when a new user record is created.
      • Update Only: Select this option to attach credentials only when an existing user record is updated.

Correlation Settings

Configure the following correlation settings:

  • Source Correlation Attribute: Select the unique identifier attribute from Paylocity that is used for correlation and user record matching with Brivo. For example, select employeeId as the source correlation attribute.
  • Target Correlation Attribute: Select the corresponding unique identifier attribute from Brivo that is used for correlation and user record matching with Paylocity. The available options are externalId and Custom Attributes. For example, select externalId as the target correlation attribute. If you select Custom Attributes, the following field will appear:
    • Custom Attributes: Choose the custom attribute in Brivo that matches the unique identifier used in Paylocity so the system can correctly link and match user records.

Provisioning Settings

Configure the following user provisioning settings:

  • Create Conditions: Select a create condition to define which new user record from Paylocity must be created in Brivo. The following are the available options:
    • None: Select this option if you do not want any new users from Paylocity to be automatically created in Brivo.
    • All Users: Select this option to create all new users from Paylocity in Brivo.
    • All Active Users: Select this option to create only active users from Paylocity in Brivo.
    • Custom: Select this option to define a custom condition for creating users in Brivo. When selected, the following field is displayed:
      • Custom Create Condition: Enter a JavaScript expression to specify which new user records from Paylocity must be created in Brivo. For example, enter User.status.statusType === "A". This example creates users in Brivo only if their status in Paylocity is Active.
  • Update Condition: Select the update condition to define which user record from Paylocity must be updated in Brivo. The following are the available options:
    • None: Select this option if you do not want any users from Paylocity to be automatically updated in Brivo.
    • All Users: Select this option to update all modified user records from Paylocity in Brivo.
    • All Active Users: Select this option to update only the modified records of active users from Paylocity in Brivo.
    • Custom: Select this option to define a custom condition for updating users in Brivo. When selected, the following field is displayed:
      • Custom Update Condition: Enter a JavaScript expression to specify which user records from Paylocity must be updated in Brivo.
  • Delete Condition: Select the delete condition to define which user record from Paylocity must be deleted in Brivo. The following are the available options:
    •  None: Select this option if you do not want any users from Paylocity to be automatically deleted in Brivo.
    • All Deactivated Users: Select this option to delete all inactive user records in Brivo.
    • Custom: Select this option to define a custom condition for deleting users in Brivo. When selected, the following field is displayed:
      • Custom Delete Condition: Enter a JavaScript expression to specify which user records from Paylocity must be deleted in Brivo.
  • Termination Action: Select the termination action from the dropdown to determine how user accounts are handled when the delete condition is met. The following are the available options:
    • suspend: Temporarily suspend the user account in Brivo.
    • delete: Permanently delete the user account from Brivo.
  • Delete Credentials from User: Select this option to remove all access credentials from Brivo users when their accounts are suspended.

User Provision Threshold Settings

Configure the following user provisioning threshold settings:

  • Enable Create Limit: Select this option to restrict the number of new users created in Brivo during a single integration run. When enabled, the following field is displayed:
    • Create Limit: Enter the maximum number of user records that can be created in Brivo during a single integration run.
  • Enable Update Limit: Select this option to restrict the number of new users updated in Brivo during a single integration run. When enabled, the following field is displayed:
    • Update Limit: Enter the maximum number of user records that can be updated in Brivo during a single integration run.
  • Enable Delete Limit: Select this option to restrict the number of user records that can be deleted in Brivo during a single integration run. When enabled, the following field is displayed:
    • Terminate Limit: Enter the maximum number of user records that can be terminated in Brivo during a single integration run.

Group Rules

The Group Rules tab defines how Paylocity users are assigned to Brivo security groups during user creation, updates, and termination. Each rule evaluates user attributes such as City, Country, Division, Location, and State.

When a user meets a rule’s conditions, they are automatically added to or removed from the specified Brivo groups. This ensures Paylocity user records are accurately mapped to the appropriate security groups in Brivo.

Perform the following steps to create a new grouprule:

  1. Click Add New Rule to create a new group rule. 
    The following fields are displayed:
  2. Click Choose Attribute to view the list of available Paylocity location attributes.
  3. Use the search bar to locate the attribute you want to use as a condition for the rule.
  4. Select the required attribute and configure its value to define the rule condition.
  5. Only users who meet this condition are included in the group rule.
  6. Click to delete a rule by using the delete icon.
  7. Click Switch to Expression to write a JavaScript expression and set the condition. 

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
