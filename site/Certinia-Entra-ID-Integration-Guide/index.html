Certinia / Entra ID Integration Guide

← Back to index

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the integration to sync Users data from Entra ID to the Users and Contacts in Entra ID.

To identify records across both systems, the integration uses an internal unique ID of Users from Certinia, and matches it to the External ID of Users in Entra ID as a correlation attribute.

Key Functions

The Certinia to Entra ID integration automates identity lifecycle management with the following primary capabilities:

  • Syncs Users and Contacts records from Certinia into the Entra ID User.
  • Creates new contact attributes in Certinia.
  • Deletes the user records upon deactivation in Entra ID.
  • Automatically generates unique usernames and email IDs in Entra ID.
  • Manages group assignments in Entra ID.

Prerequisites

The following are the prerequisites:

  • Admin Access to Entra ID: You must have admin access to your Entra ID instance. You can visit the Entra ID portal to get access to an Entra ID instance.
  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Entra ID:
    • Certinia: Ensure that you configure the Certinia connector in the Aquera Admin portal. Refer to the Certinia Configuration Guide for configuration details.
    • Entra ID: Ensure that you configure the Okta connector in the Aquera Admin portal.

Integration Setup

This section details the configuration and mapping process for integrating Certinia with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how worker data is mapped from Certinia to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Certinia / Entra ID.
  5. Enter a name and a description for the integration.
  6. Select your Certinia and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the Certinia and Entra ID application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Features

Enable the following features based on your requirements:

  • Enable Manage User Lifecycle Management: Enabling this checkbox configures user lifecycle management as a destination. When enabled, the Manage User Lifecycle Management section is displayed to configure integration settings for worker management.
  • Enable Manage Contact Lifecycle Management: Enabling this checkbox configures contact lifecycle management as a destination. When enabled, the Manage Contact Lifecycle Management section is displayed to configure integration settings for worker management.
  • Enable Report-Only Mode: Enable this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes. When enabled, the Manage User Lifecycle Management section is displayed to configure integration settings for worker management.
  • Enable Logging: Enable this checkbox to generate logs outlining the data changes made during the integration process. When enabled, the Log settings section is displayed to configure integration settings for worker management.
  • Enable Notifications: Enable this checkbox for integration-related notifications, keeping users informed of integration activity and updates. When enabled, the Manage User Lifecycle Management section is displayed to configure integration settings for worker management.
  • Enable Reporting: Enable this checkbox to activate reporting for the integration, providing insights into the data processed during execution. 
  • Enable Log Insights: Enable this checkbox to visualize the integration-related metrics, offering a detailed view of log data and performance indicators. 
  • Enable Incremental Mode: Enable this checkbox for incremental mode, ensuring that only the most recently changed records are processed during each integration run.

User Life Cycle

The User Life Cycle section enables you to customize the integration settings for user life cycle management according to your specific use case.
Note: This section appears only if you have selected the Enable Manage User Lifecycle Management setting in the Parameters section.

It has the following settings:

  • Enable Creations: Enable this checkbox for the creation of user records during the integration process.
  • Enable Updates: Enable this checkbox to enable updates to existing user records, ensuring that any changes are reflected during the integration run.
  • Enable Terminations: Enable this checkbox to perform termination of user records, allowing the integration to process and apply termination events accordingly.
  • Map Manager: Enable this checkbox to perform the mapping of the manager attribute to the corresponding user records during the integration process.

Match Expression

These settings determine how users are matched between Entra ID and Certinia:

  • Source Correlation attribute: Defines the attribute from Entra ID used to uniquely identify users. For example, User Principle Name is used as the unique identifier from Entra ID.
  • Destination correlation attribute: Defines the corresponding attribute in Certinia used for matching. For example, a Username is used to match existing employee records in Certinia.

Manage Create Conditions

  • Switch to Expression Mode: Enable this checkbox to switch the create condition to expression mode, allowing the use of custom expressions to define creation criteria. Provide a JavaScript expression to identify the Users to be created or updated.
    Note: This parameter is available only if you select Switch to Expression Mode in the Manage Create conditions to be Created in Certinia.
  • Create Condition: Enter the condition used to determine whether a given user record should be created during the integration process. By default, this is set to Create All Users, which ensures all user records are processed for creation.

Manage Update Conditions

  • Switch to Expression Mode: Enable this checkbox to switch the update condition to expression mode, allowing the use of custom expressions to define update criteria. Provide a JavaScript expression to identify the Users to be updated.
    Note: This parameter is available only if Switch to Expression Mode is selected in the Manage Update Conditions section.
  • Update Condition: Enter the condition used to determine whether a given user record should be updated during the integration process. By default, this is set to Update All Users, which ensures all user records are processed for creation.

Manage Termination conditions

  • Termination Condition: Enter the condition used to determine whether a given user record should be terminated during the integration process.

Contact Life cycle

The Contact Life Cycle section enables you to customize the integration settings for contact life cycle management according to your specific use case.
Note: This section appears only if you have selected the Enable Manage Contact Lifecycle Management setting in the Parameters section.

It has the following settings:

  • Create Contacts: Enable this checkbox for the creation of contact records in Certinia during the integration process, ensuring that new contacts are added accordingly.
  • Update Contacts: Enable this checkbox to make updates to existing contact records, ensuring that any changes are reflected during the integration run.
  • Create Contact Only If Certinia User Exists: Enable this checkbox to restrict contact creation to only those records where a matching Certinia user record is found.
  • Sync Certinia User Data to Certinia Contact: Enable this checkbox to sync fields from the matching Certinia user record into the corresponding contact record. If no matching user is found, the contact record is still created or updated.
  • Map Manager to Contact: Enable this checkbox to resolve each employee's direct manager from Certinia, locate the corresponding contact record in Entra ID, and set the field on the contact record accordingly.

Debug Mode

This section enables you to customize the debug mode settings for testing use cases using a single user. It has the following settings:

Note: This section appears only if you have selected the Enable Debug Mode setting in the Parameters section.

  • Enable Single User Testing: Enable this checkbox to configure single user testing mode, allowing the integration to be tested against a specific user record without processing all users.
  • Import Limit: Enter the number of user records that should be fetched during each integration run. Set the value to 0 or Unlimited to fetch all available user records.

Notification Settings

This section enables you to customize the notification settings. It has the following settings:

Note: This section appears only if you have enabled the Enable Notifications setting in the Parameters section.

  • Send the Notification Summary Report As An Attachment: Enable this checkbox to receive the integration summary report as an attachment in the notification email.
  • Consent to Send Email Report with Attribute Values: Enable this checkbox to capture and include all attribute values in the email report.
  • Select the Notification Report Detail Level: Select the level of detail to be included in the notification report from the dropdown menu. The following are the available options:
    • Summary: Captures a brief summary of the events processed during the integration run.
      • Full: Captures complete details for all events during the integration run. 
      • Custom: Allows customization of the attributes to be included or excluded in the notification report. The following parameters are displayed once you select Custom.
      • Attribute Filter: Select the mode to include or exclude attributes in the notification report. The following are the available options:
        • Inclusion List: Includes only the selected attributes in the notification report. 
        • Exclusion List: Excludes the selected attributes from the notification report.
      • Included Attributes: Enter the custom attributes to be included in the notification report. This parameter is displayed only if the Inclusion List option is selected in the Attribute Filter parameter.
      • Excluded Attributes: Enter the custom attributes to be excluded from the notification report. This parameter is displayed only if the Exclusion List option is selected in the Attribute Filter parameter.
  • Email Subject: Enter the content that must be used as the email subject.
  • Email Body: Enter the content that must be used as the email body for all email notifications.

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable Logging setting in the Parameters section.

  • Show Attribute Values for Each Event: Select this checkbox to populate and display the attribute values for each event when the Custom option is selected in the log detail level.
  • Select the Log Detail Level: Select the level of detail to be captured in the logs from the dropdown menu. The following are the available options:
    • Summary: Captures a brief summary of the events processed during the integration run.
      • Full: Captures complete details for all events during the integration run. 
      • Custom: Allows customization of the attributes to be included or excluded in the logs. The following parameters are displayed once you select Custom.
    • Attribute Filter: Select the mode to include or exclude attributes in the logs. The following are the available options:
      • Inclusion List: Includes only the selected attributes in the logs. 
      • Exclusion List: Excludes the selected attributes from the logs.
    • Included Attributes: Enter the custom attributes to be included in the logs. This parameter is displayed only if the Inclusion List option is selected in the Attribute Filter parameter.
    • Excluded Attributes: Enter the custom attributes to be excluded from the logs. This parameter is displayed only if the Exclusion List option is selected in the Attribute Filter parameter.
