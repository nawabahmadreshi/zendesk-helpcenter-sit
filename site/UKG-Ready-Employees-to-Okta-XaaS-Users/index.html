UKG Ready Employees to Okta (XaaS) Users

Overview

The Aquera platform allows your organization to scale complex data management by automating data synchronization across resource types. This guide covers the end-to-end tasks involved in setting up an integration to synchronize the Employees resource of UKG Ready with the Users resource of Okta.

The integration performs a full sync on the initial run, processing all employee records in UKG Ready. Subsequent runs execute incremental syncs, processing only changed records. The full sync frequency is configurable in the Parameters section.

This integration performs the following tasks:

  • Retrieves all employees created in UKG Ready.
  • Processes all the records and then upserts (updates or inserts) or deletes user profiles in Okta in bulk. The user records are inserted, updated, and deleted in Okta based on the upsert and delete conditions set in the Parameters section.
  • Performs a full sync or an incremental sync based on the full sync frequency set in the Parameters section.
  • Generates a CSV report of all the processed records.
  • Triggers an email notification.

Prerequisites

The following are the prerequisites:

  • Admin Access to Okta: You must have admin access to your Okta instance. You can visit the Okta portal to get access to an Okta instance.
  • Enable XaaS access:  You must open a support case with Okta to enable XaaS in your Okta instance.
  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Okta:
    • UKG Ready: Refer to the UKG Ready Configuration Guide for details.
    • Okta XaaS: Refer to the Okta Anything-as-a-Source (XaaS) Configuration Guide for details.

For details on the limitations of Okta's Anything-as-Source (XaaS) solution, refer to the Okta documentation.

Integration Setup

This section details the configuration process for integrating UKG Ready with Okta. It covers creating and setting up the integration, setting up parameters, and establishing automated synchronization to ensure employee data flow from UKG Ready into Okta. This integration automates user lifecycle management by provisioning new hires, updating existing user profiles, and handling offboarding events. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select UKG Ready Employees to Okta Users.
  5. Enter a name and a description for the integration.
  6. Select your UKG Ready and Okta connector instances.
  7. Click Continue. A new page is displayed to configure the integration.

Set Up Parameters

The Parameters section allows you to customize the integration settings according to your use case. It has the following configuration parameters:

  • Report only: Select this checkbox if the integration must only generate a report of the changes made in Okta without applying them. If this checkbox is not selected, the integration will make changes (add, update, and delete users) in Okta.
  • Upsert condition: Specify the condition that determines the users that need to be created or updated in Okta(for example, worker.terminationDate == undefined).
  • Delete condition: Specify the condition that determines the users that need to be deleted in Okta(for example: worker.terminationDate != undefined).
  • Full Sync Frequency: Specify the time interval, in number of days, after which a full employee sync has to be performed instead of an incremental sync from UKG Ready to Okta.
  • Debug Mode: Enables detailed logging and monitoring to assist in troubleshooting by providing comprehensive information about each step of the integration process.
  • Enable Dashboard Generation: Monitors integration performance and analyzes trends through customizable graphical dashboards.
  • Enable Integration Report Download: Allows generation of integration results in CSV format.

Set Up Schedule

You can configure the integration to run automatically at specified intervals.

  1. Navigate to the Trigger tab.
  2. Select Schedule as the Type.
  3. Under Schedule Type, select Basic to configure a basic schedule, or click Advanced to configure advanced schedule settings.

For details, refer to the Integration Schedule Setup Guide.

Create Custom Attributes in Okta

You can create custom fields in Okta to include additional UKG Ready Employee attributes in Okta's User object. You must configure all the custom attributes individually. After the configuration, Aquera automatically populates the fields you created in Okta from the Employee resource of UKG Ready. 

Perform the following steps to create custom attributes in Okta:

  1. Sign in to your Okta instance.
  2. Navigate to Directory > Profile Editor and locate the custom identity source integration you've created.
  3. Click Add Attribute to add a new attribute.
  4. Configure the attribute using the same name as UKG Ready in the Variable name field and fill in the other fields. 

    Note: You must remove special characters in the name and replace periods with underscores.
    For example, the equivalent name for the attribute links.badges would be links_badges.
  5. Click Save to save the changes or Save and Add Another to create another custom attribute. 

Map Custom Attribute to Okta Profile

Perform the following steps to map the custom attributes to the Okta profile:

  1. Navigate to Mappings > Configure User mappings.
  2. Map the custom attribute to an equivalent attribute in the Okta user profile and click Save Mappings.
  3. If the equivalent attribute is not present in the default Okta user profile, open the default profile from Directory > Profile Editor.
  4. Click Add Attribute to add a custom attribute to the Okta default user profile.
  5. Fill in the attribute's display name, description, and set the user permission. Click Save Attribute to save the changes.
  6. Map the custom identity source attribute to the attribute in the Okta default profile and click Save Mappings.

Note: You can repeat these steps to add all employee-related attributes in UKG Ready to Okta.

Run Integration

You can set event or schedule triggers to run the script synchronization. For more details, refer to the Executing a Synchronization section.

After the script synchronization is successfully executed, Aquera syncs the Employees created in UKG Ready as users in Okta. The imported users are staged and need admin approval to get provisioned to Okta.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
