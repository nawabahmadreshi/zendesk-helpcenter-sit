Provisioning of Users in Active Directory Groups Based on ServiceNow Catalog Requests

← Back to index

Overview 

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

The ServiceNow / Active Directory - Group Management integration retrieves all the ServiceNow Catalogs that have been created for adding or removing users in Active Directory groups. It then adds or removes those users in the Active Directory groups and changes the status of the ServiceNow catalog to Closed. 

Key Functions

The ServiceNow / Active Directory - Group Management integration performs the following key functions:

  • Retrieves all ServiceNow Catalogs that have been approved
  • Looks for specific fields in each catalog to identify the users that must be added or removed in Active Directory groups
  • Identifies the type of operation (Add or Remove) to be performed for those users in the Active Directory group
  • Identifies the group in which the operation needs to be performed
  • Performs the operation (Add or Remove users) in the specified groups
  • Changes the status of the ServiceNow Catalog to Closed, if the operation is successful for all the users. If the operation has failed for any user, the integration does not modify the status of the Catalog.
  • Generates audit logs, reports, and analytics.
  • Sends an email notification to admins for each integration run.

Integration Setup

This section describes the configuration steps required to set up ServiceNow / Active Directory - Group Management integration in Aquera.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left navigation panel.
  3. Click Create Integration.
  4. Select ServiceNow / AD - Group Management.
  5. Enter a name and description for the integration.
  6. Click Continue. The Configuration page is displayed.

Configuration

The following sections are available under the Configuration tab:

Applications

The Applications section allows you to select the following connectors that are required for this integration:

  • Active Directory: Select the Active Directory (Version 8) application from the drop-down list.
  • ServiceNow: Select the ServiceNow application from the drop-down list.

Parameters

The Parameters section allows you to customize the integration preferences. These settings control the behavior of the integration and help tailor it to your organization’s requirements.

Integration Scenarios 

Enable the following integration feature:

  • Enable Group Management: Select this checkbox to enable provisioning or deprovisioning users in specific groups in Active Directory based on the details provided in the ServiceNow Catalog requests. You must enable this option for the integration. The Group Management section is displayed when you enable this option.

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this option to generate logs outlining the changes performed by the integration. The Logging section is displayed when you enable this option.
  • Enable Report Generation: Select this option to generate integration reports. The Reporting section is displayed when you enable this option.
  • Enable Notification: Select this option to send integration-related notification emails to admins, after each run. The Notification section is displayed when you enable this option.
  • Enable Log Insights: Select this option to visualize the integration-related metrics. The Log Insights section is displayed when you enable this option.

Group Management

This section includes the following options to configure the key functionality of the integration.

Source Configuration

The following are the parameters under the Source Configuration section:

  • Select Catalog: Select the ServiceNow catalog from which the requests must be picked by the integration. For example, Add/Remove Users in a Group.  Only the requests that are in the Approved status will be processed by the integration.
  • Select Source Attribute: Select the source attribute that must be used by the integration to uniquely identify a user in ServiceNow. This attribute value will be matched against a specific Active Directory attribute to identify the matching user records between ServiceNow and Active Directory. For example, Email
  • Select Operation Field: Select the field in the ServiceNow catalog that the integration must use to identify the type of operation (Add or remove user) to be performed in AD groups.
  • Select Option for Add User: Select the value that the operation type field of the catalog must contain for the integration to perform an Add operation. For example, Add. The catalog field that you have selected in the Select Operation Field must contain this value for the integration to add a user to an AD group.
  • Select Option for Remove User: Select the value that the operation type field of the catalog must contain for the integration to perform a Remove operation. For example, Remove. The catalog field that you have selected in the Select Operation Field must contain this value for the integration to remove a user from an AD group.
  • Select AD Group Field: Select the field in the ServiceNow catalog that the integration must use to identify the AD group in which the action has to be performed. For example, Group Name
  • Select User Field: Select the section in the ServiceNow catalog where the integration must find the list of users for which the action has to be performed. For example, Users

Target Configuration

The following parameter is available under the Target Configuration section:

  • Select Target Attribute: Select the target attribute that must be used by the integration to uniquely identify a user in Active Directory. The integration compares the value of the source attribute (selected in Select Source Attribute) against this Active Directory attribute value to identify the matching user records between ServiceNow and Active Directory. For example, Mail

Integration Reports

This section details the reports that are generated by the integration.

Integration Sync Summary

This report is available in the body of the email that is sent by the integration on each run. It includes the following details:

  • The number of ServiceNow catalog requests analyzed 
  • The number of requests successfully processed by the integration (successfully added or removed users in Active Directory groups)
  • The number of failed operations
  • The number of Active Directory groups skipped by the integration (the groups where no users were added or removed)

Email Report

A CSV report is attached to the email if the email notification feature is enabled in the integration. The CSV report has the following details:

  • The request approval status
  • The group in which the operation was performed
  • The type of operation (Add or Remove)
  • The Distinguished Name of the Active Directory user 
  • The operation status (Success or Failure)
  • A message that indicates success or the reason of failure
