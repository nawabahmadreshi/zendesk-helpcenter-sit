Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize employee data, including their contract details and compensation details to the UKG Ready application.

Key Functions

The Bob / UKG Ready integration performs the following functions:

  • Retrieves the data of employees from Bob and synchronizes the data to a specific company code in UKG Ready.
  • Retrieves the employment contract details of employees from Bob and adds those details to the user records in UKG Ready, based on the integration settings.
  • Retrieves the compensation details of employees from Bob and adds those details to the user records in UKG Ready, based on the integration settings.
  • Updates the attributes of employees, contract information, and compensation details in UKG Ready based on the attribute mappings configured in the integration.
  • Retrieves Bob employees, and creates, updates and terminates UKG Ready users based on the limits set in the integration.
  • Generates logs, reports, and analytics dashboards based on the configurations.
  • Sends notification emails with reports on each integration run based on the configuration.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Bob: Refer to the Bob Configuration Guide for configuration details.
    • UKG Ready: Refer to the UKG Ready Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to synchronize worker data from ADP Workforce Now to user data in Viewpoint Spectrum.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Bob / UKG Ready.
  5. Enter a name and a description for the integration.
  6. Select your Bob and UKG Ready connector instances.
  7. Click Continue. The Overview page is displayed to start configuring the integration.

Overview

This section provides an overview of the integration and provides options to enable or disable the key integration features.

User and Access Management Features

  • Enable employee management: Enable this toggle for the integration to sync employee data from Bob to UKG Ready.
  • Enable contracts management: Enable this toggle to sync the employment contract details from Bob to UKG Ready along with the employee data.
  • Enable compensation management: Enable this toggle to sync the payroll details from Bob to UKG Ready along with the employee data.

The Applications section displays the Bob and UKG Ready application connectors used for the integration. This section allows you to search and update these application connectors, if required.

Parameters

The Parameters section enables you to customize the operational settings according to your requirements. It has the following settings:

Operational Settings

Configure the following operational settings:

  • Report-only mode: Turn on this toggle to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Report generation: Turn on this toggle to generate the integration report. When the toggle is on, the Report Settings section is displayed for further configuration.
  • Logging: Turn on this toggle to generate logs outlining data changes. When the toggle is on, the Log Settings section is displayed for further configuration.
  • Email notifications: Turn on this toggle to enable integration-related notifications. When the toggle is on, the Notifications section is displayed for further configuration.
  • Insights: Turn on this toggle to visualize the integration-related metrics. When the toggle is on, the Insights section is displayed for further configuration.

Employee Match Settings

  • Bob employee attribute: Select an employee attribute in Bob that is used to find a matching user in UKG Ready.
  • UKG Ready employee attribute: Select the corresponding UKG Ready employee attribute that is used to find the matching employee records.

Sensitive Fields

The Sensitive Field Settings section enables you to define the employee fields that should be treated as sensitive data and unique identifiers. These settings are used by the integration when generating reports, logs, and Insights data, ensuring that employee information is displayed and handled according to your configured preferences. 

 This section has the following settings:

  • PII Fields: Enter the field names that contain personally identifiable information (PII).
  • GUID Fields: Enter the field names that contain globally unique identifiers (GUIDs).
  • Employee ID Fields: Enter the field names that contain employee IDs.

Note: This section appears only if you have enabled Report generation in the Operational Settings section.

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Report Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Full Details: All attribute details are displayed.
    • Include List: Select this option to report only the attributes specified in the Custom Inclusion List.
      • Custom Inclusion List: Select the attributes to be included in the report.
    • Exclude List: Select this option to report only the attributes specified in the Custom Exclusion List.
      • Custom Exclusion List: Select the attributes to be excluded from the report.

Bob Settings

This tab includes the settings related to the retrieval of employee details from the Bob application.

Eligible Employees Filter

This section enables you to set the configurations for filtering the employees that must be considered for synchronization to UKG Ready.

  • Site filter: Select the sites from which the employees need to be retrieved for synchronizing to UKG Ready. Only employees associated with the chosen site(s) will be retrieved and processed.
  • Manager site filter: Select the sites from which the managers need to be considered for mapping to users in UKG Ready. Only managers belonging to the selected site(s) will be mapped to the users in UKG Ready during synchronization. For the users whose managers are not in the selected sites, the manager mapping will not be performed in UKG Ready. If left blank, all managers are considered for mapping to UKG Ready users.
  • Include inactive Bob employees: Enable this toggle to include the Bob employees whose status is inactive, but the lifecycle status is active. For example, when an employee is on long leave, their status is inactive, however, their lifecycle status is still active (because of active employment). When the toggle is disabled, only the Bob employees whose status is active will be considered for synchronization.

UKG Ready Settings

This tab includes the settings related to the provisioning of employees to the UKG Ready application.

Company Settings

  • Company Name: Specify the company name in your UKG Ready application in which the employees retrieved from Bob must be created.

Employee Management

Worker Settings

  • Employee account creation: Enable this toggle to enable the integration to create employee accounts in UKG Ready. When enabled, the Creation section is displayed for further configurations. When this toggle is disabled, the integration will not create employee accounts in UKG Ready.
  • Employee account update: Enable this toggle to enable the integration to update employee accounts in UKG Ready based on the changes to the employee data in Bob. When this toggle is disabled, the integration will not modify employee accounts in UKG Ready.
  • Employee account termination: Enable this toggle to enable the integration to terminate employee accounts in UKG Ready. When enabled, the Termination section is displayed for further configurations. When this toggle is disabled, the integration will not terminate employee accounts in UKG Ready.
  • Employee account rehire: Enable this toggle to enable the integration to reactivate employee accounts in UKG Ready based on their rehire status in Bob. When this toggle is disabled, the integration will not reactivate employee accounts in UKG Ready.
  • Map manager: Enable this toggle to enable the integration to map users' managers in UKG Ready based on the user data retrieved from Bob. When this toggle is disabled, manager relationships will not be synchronized to UKG Ready
  • Safeguard settings: Enable this toggle to apply limits to the number of operations the integration can perform in UKG Ready in a single run. When enabled, the Safeguard Settings section is displayed for further configurations. When this toggle is disabled, no safeguard limits will be applied for the operations in UKG Ready.

Creation Settings

Note: This section is displayed only if the Employee account creation toggle is enabled.

  • Create future-dated hires: Enable this toggle to allow the integration to create employee accounts in UKG Ready for users whose hire date in Bob is set to a future date.
  • Make the account locked in UKG Ready: Enable this toggle to allow the integration to create employee accounts in a locked state in UKG Ready.

Termination Settings

Note: This section is displayed only if the Employee account termination toggle is enabled.

  • Terminating processing time: Select when the termination event has to be processed by the integration. 
    • Immediately: Select this option to deactivate the employee accounts in UKG Ready in the next integration run after they are terminated in Bob.
    • Custom: Select this option to specify the number of days before the termination date when the account must be deactivated in UKG Ready. 
  • Disable employees before X days of termination: Specify the number of days before their termination date in Bob when the account must be deactivated in UKG Ready. This field is displayed only if you have selected Custom in the Terminating processing time field.
  • Termination status: Enter the termination status that must be updated for the employees in UKG Ready when they are terminated in Bob, and no termination status is available from Bob.

Safeguard Settings

Note: This section is displayed only if the Safeguard settings toggle is enabled.

  • Import limit: Enter the maximum number of employee records that are allowed to be retrieved from Bob in a single integration run.
  • Create limit: Enter the maximum number of employee records that are allowed to be created in UKG Ready in a single integration run.
  • Update limit: Enter the maximum number of employee records that are allowed to be updated in UKG Ready in a single integration run.
  • Terminate limit: Enter the maximum number of employee records that are allowed to be terminated in UKG Ready in a single integration run.
  • Rehire limit: Enter the maximum number of employee records that are allowed to be reactivated in UKG Ready in a single integration run.

Contract Management

Contract Settings

  • Contracts creation: Enable this toggle to enable the integration to add employment contract details of users in UKG Ready.
  • Contracts update: Enable this toggle to enable the integration to update contract records of users in UKG Ready based on the changes to the employment contract data in Bob. 
  • Contracts termination: Enable this toggle to enable the integration to update employment contract records in UKG Ready when they are terminated in Bob.
  • Contracts history mode: Specify whether the integration should synchronize the employee's complete contract history to UKG Ready or only the details of the employee's current contract. 
    • Select Full history to sync the full employment contract history.
    • Select Current only to sync only the current employment contract details.
  • Default contract type: Select the default employment type to be assigned to the contract records if the employment type is not available in the employee contract records retrieved from Bob. 
  • Contracts safeguard: Enable this toggle to set a limit to the number of contract records that the integration can create in UKG Ready in a single run. When enabled, the Contracts limit field is displayed.
  • Contracts limit: Enter the maximum number of contract records that are allowed to be created in UKG Ready in a single integration run. This field is displayed only if you have enabled the Contracts safeguard toggle.

Compensation Management

Compensation Settings

  • Compensation creation: Enable this toggle to enable the integration to add compensation details of users in UKG Ready. 
  • Compensation update: Enable this toggle to enable the integration to update compensation records of users in UKG Ready based on the changes to the compensation data in Bob.
  • Default currency: Select the default currency to be assigned to the employee compensation records if the currency is not available in the compensation records retrieved from Bob. 
  • Compensation safeguard: Enable this toggle to set a limit to the number of compensation records that the integration can create in UKG Ready in a single run. When enabled, the Compensation limit field is displayed.
  • Compensation limit: Enter the maximum number of compensation records that are allowed to be created in UKG Ready in a single integration run. This field is displayed only if you have enabled the Compensation safeguard toggle.
