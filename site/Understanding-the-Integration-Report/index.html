Understanding the Integration Report

← Back to index

Overview

You will receive an integration report via email every time you execute the integration. This report provides a detailed record of each user processed during the ADP Workforce Now and Microsoft 365 integration. It includes information about the action performed, the result, and the values before and after synchronization.

Integration Report Column Headers

The following table lists the names of different columns and their corresponding definitions in the integration report:

Column Name Definition
Worker Name Name of the worker as retrieved from ADP Workforce Now
Worker ID Unique identifier of the worker in ADP
Operation The integration action attempted (e.g., CREATE, UPDATE, DISABLE, IGNORE)
Result Outcome of the operation (e.g., SUCCESS, FAILURE, IGNORE)
Result Details Explanation for the result, especially in the case of IGNORE or FAILURE
Status (Before/After) Employment status before and after the integration
User Principal Name (Before/After) User login (email) before and after the sync
First/Last Name (Before/After) Users' names in Microsoft 365, before and after the integration
Job Title, Department (Before/After) Role-related details, before and after the integration
Address, City, State, ZIP (Before/After) Location info synced from ADP to Microsoft 365
Company Name (Before/After) Employer/company name, before and after the integration
Mail Nick Name (Before/After) User name in Microsoft 365, before and after the integration

Type of User Operations Performed

The following table lists the names of different user operations and their corresponding definitions that were performed in the integration report:

Operation Meaning
CREATE New user accounts were created in Microsoft 365
UPDATE The existing user accounts were updated in Microsoft 365
DISABLE The user accounts were disabled in Microsoft 365 (for inactive users in ADP)
REACTIVATE Previously disabled user accounts in Microsoft 365 were re-enabled
IGNORE No action taken (inactive user account or corresponding user account is not found in ADP)

How to Use the Integration Report

Open the integration report and perform the following to understand the integration report:

  • Validate Integration Success: Check rows with CREATE/UPDATE and SUCCESS values to confirm provisioning.

  • Identify Issues: Use the Result and Result Details columns to diagnose issues or skipped records.

  • Audit Changes: Compare Before/After values to see which user accounts were modified.

  • Filter by Operation: Check new creations, updates, or failures by sorting the Operation column.
