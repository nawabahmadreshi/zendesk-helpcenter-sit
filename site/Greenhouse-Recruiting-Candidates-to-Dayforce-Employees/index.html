Greenhouse Recruiting Candidates to Dayforce Employees

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

This integration synchronizes the candidate data from Greenhouse Recruiting to Dayforce. It extracts the candidate data from Greenhouse Recruiting candidates, creates a new employee record in Dayforce employees, and generates a structured CSV export.

The integration is triggered when a candidate's status changes in Greenhouse Recruiting, such as when a candidate is hired or completes onboarding.

This integration performs the following tasks:

  • Maps and transforms the Greenhouse candidate data into the employee schema required by Dayforce. Converts date to a standard format, standardizes employment types and gender, and sets pay groups based on the office location.
  • Sends the employee data to Dayforce and automatically retries if the system limits the request rate.

  • Logs integration settings and preferences to help monitor performance and troubleshoot issues.

  • Formats and sends notification emails about the integration status, including success or failure details.
  • Handles errors with clear logging for troubleshooting API communication or data transformation issues.

Note: You must set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to this Aquera integration. Refer to the Appendix for details.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for instructions on creating the Greenhouse Recruiting application connector and setting up the authentication.
    • Dayforce: Refer to the Dayforce Configuration Guide for instructions on creating the Dayforce application connector and setting up the authentication.

Integration Setup

This section details the configuration process for integrating Greenhouse Recruiting candidates with Dayforce Employees. It covers creating the integration, configuring system preferences, and setting up event-driven triggers to automate the real-time transfer of new hire data from Greenhouse Recruiting candidates to Dayforce Employees.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Greenhouse Recruiting Candidates to Dayforce Employees.
  5. Enter a name and a description for the integration.
  6. Select your Greenhouse Recruiting Candidates and Dayforce Employees connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Preferences

  • Enable Dashboard Generation: Controls whether a dashboard is generated to visualize and analyze integration results, providing insights into data flow, errors, and performance metrics.
  • Recipients Email Addresses: Specifies the list of email addresses, separated by commas, that will receive notifications related to integration events, such as success reports or error alerts.
  • Role for New Employees: Defines the valid system role assigned automatically to newly created employee records, determining their permissions and access rights.
  • Report Only: When selected, the integration runs in a simulation mode, generating a detailed report of potential data changes without applying any updates to the systems.
  • Enable Integration Report Download: Allows users to download the integration report as a CSV file, facilitating offline review and record-keeping of integration activities and outcomes.
  • Send Email Notification: Enables the automatic sending of email notifications to designated recipients regarding the status and results of the integration process.
  • Department Location Code: A unique identifier representing the department location. 
  • Debug Mode: Enables detailed logging and monitoring to help troubleshoot and diagnose issues during integration.

Set Up Event Triggers

You need to set up the event triggers to receive candidate hire events from the Greenhouse Recruiting application and run the integration.

  1. In the Trigger section, click Event.
  2. Select the Greenhouse Recruiting connector configured in your Aquera Admin tenant in the Event Source drop-down.
    The URL field displays the endpoint URL to which Greenhouse Recruiting will send the candidate hire events. This URL must be configured in the Greenhouse Recruiting application when configuring webhooks. For more details on configuring webhooks, see the setting up Webhooks in the Greenhouse Recruiting application.
  3. Click Create Token to generate a token. Save this token securely.

Run the Integration

The integration runs when a candidate in the Greenhouse Recruiting application is hired. It processes the event received from Greenhouse Recruiting and creates a new employee record in the Dayforce application.

The system automatically sends confirmation emails to the HR teams when a new employee is successfully onboarded. Suppose any issues occur during the setup (like missing information or system errors). 

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.

Appendix

Setting Up Webhooks in Greenhouse Recruiting

This section details how to set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to the Aquera integration for processing.

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center on the left navigation pane and click Webhooks.
  4. Enter the following fields to configure your new webhook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for. For example, Aquera Integration
    • When: Select the event, Candidate has been hired, from this drop-down list. 
    • Endpoint URL: Copy the URL from the Trigger section on the Greenhouse Recruiting Candidates to the Freshservice integration setup page of the Aquera Admin portal and paste it into this field. Refer to the Set Up Event Triggers section for details.
    • Secret key: Enter the token generated in the Trigger section on the Greenhouse Recruiting Candidates to Freshservice integration setup page of the Aquera Admin portal. Refer to the Set Up Event Triggers section for details.
    • Deactivated?: Make sure that the No option is selected.
  5. Click Advanced Settings and then provide the Basic Authorization Credentials, where the token generated in the Aquera Admin portal must be provided as the password for authentication. The username can be any random string. Refer to the Set Up Event Triggers section for details on obtaining the token.
  6. Click Create webhook to complete the webhook setup.

When you create a new webhook, Greenhouse Recruiting will ping the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the webhook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the webhook.

Note: If the webhook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Webhook section and edit the configuration to fix the error. After you have fixed the configuration issue, make sure that you set the Deactivated. Option to No.
