Synchronization Orchestration from BambooHR to Okta

← Back to index

Overview

This guide explains you the step-by-step procedure on how to setup synchronization orchestration from BambooHR to Okta by using Aquera as a connector. BambooHR provides human resources software as a service, while Okta offers identity and access management services. An orchestration between the BambooHR and the Okta will enable the users access authenticated data and avoid duplication of entries. Aquera uses SCIM (System for Cross Domain Identity Management) as the gateway for identity provisioning and offers identity integration as a service. 

To perform the orchestration, the user needs to configure the BambooHR application connector (source application) and the Okta application connector (target application) on the Aquera Admin console. 

Prerequisites

  • The user must have access to the Aquera Admin console.
  • The Orchestrations option is enabled for the user in the Aquera Admin console.
  • The user must have the required access to BambooHR and Okta applications.

Setup Procedures

The following procedures are involved in this orchestration setup:

  • Configuring BambooHR Application Connector
  • Configuring Okta Application Connector
  • Integration Setup in Okta 
  • Verifying Connector Credentials
  • Setting up Synchronization Orchestration

Configuring BambooHR Application Connector

  1. Log in to the Aquera Admin Console and navigate to the Applications page.
  2. Search and select the BambooHR application from the application catalog. 
  3. Configure the BambooHR application connector and click Create.
    For configuration details, refer to the BambooHR Configuration Guide

Configuring Okta Application Connector

  1. Log in to the Aquera Admin Console and navigate to the Applications page.
  2. Search and select the Okta application from the application catalog. 
  3. Configure the Okta application connector and click Create.
    For configuration details, refer to the Okta Configuration Guide

Note: The Tenant value must be from the URL of your Okta instance. 

Importing Users to Okta 

Before executing the orchestration, the user needs to import users data from BambooHR to Okta. The following steps must be performed for importing users.

1. Login to Okta by entering your credentials.

2. Click Admin at the top-right corner of the page, if you have logged in to Oktapreview org.
If you have logged into Okta org, skip to the next step.

3. Expand Applications in the navigate pane and then click Applications. The Applications page is displayed.

4. Click Browse App Catalog and navigate to the Private Apps category in the list.

You can see the Aquera private apps set up by the Okta support team on this page. Refer to the Okta Setup Guide for details on how to get the Aquera private apps enabled in Okta.

5. Click Aquera private app, as BambooHR supports Bearer Token authentication.

6. Click Add to configure your BambooHR application in Okta.

7. Provide an Application Label and click Next.

8. Select the required sign-on method and click Done. Your BambooHR application is now added in Okta.

9. Click Provisioning. 

10. Click Configure API Integration and select Enable API Integration.

11. Enter the following details:

  • Base URL: The interface URL to the Aquera BambooHR connector must be copied from the Aquera Admin console and pasted here.

  • API Token: Enter the API key generated in your BambooHR application. To obtain the API token:
    • Logon to the BambooHR web application. 
    • On the top right corner of the home page, click the user profile icon. 

    • Click API Keys and then click click Add New Key.

    • Enter a unique name for the API key and click Generate Key. A unique API key is generated.

    • Copy this key and click Done.
    • Paste it in the API Token field in Okta. Save this key securely for future use.

12. Click Test API Credentials in Okta. A verification success message is displayed.

13. Click Save. The Provisioning to App section is displayed.

14. Click Edit and enable the following options as per your requirement:

  • Create Users
  • Update User Attributes
  • Deactivate Users
  • Sync Password

15. Click Save.

16. Navigate to Import.

17. Click Import Now. Aquera connector imports the users from BambooHR to Okta. Okta shows the number of users that are imported.

18. Click OK.

19. Verify all the users imported to Okta and click Confirm Assignments. 

Note: If any user assignments could not be performed, a pop-up message similar to the following is displayed. The assignment can fail if any mandatory user attributes for Okta are left blank. In that case, you need to enter those user attributes in BambooHR and import them again. 



20. Click Confirm. The users from BambooHR are created in Okta.

Verifying Connector Credentials

BambooHR

Before performing the orchestration, User needs to check the credentials for the BambooHR and the Okta connectors. This verification ensures the smooth flow of orchestration. 

1. Go to the Bamboo HR Application connector on the Aquera Admin console and click the Credentials tab.

2. Enter the API key copied from the BambooHR application and click Save Changes.

3. Click the Data tab to verify the users data from BambooHR are imported.

Okta

To create authentication token for Okta:

1. Go to Okta Admin page and click Security > API. 

2. Click Create Token. 

3. Provide a name for the token and click Create Token.

4. Copy the token and save it securely.

5. Go to the Okta Application connector on the Aquera Admin console and click the Credentials tab. 

6. Enter the token copied from the Okta application and click Save Changes.

7. Click the Data tab to verify the users data created in Okta are imported.

Setting up Synchronization Orchestration

1. Go to Aquera Admin console and select Orchestrations > Create Orchestration.

2. Enter a name for the orchestration and select the Synchronization orchestration type.

3. Select BambooHR as the Source Application and Okta as the Target Application. 

You can either retain or change the other default settings as per your requirement. It is advisable to select the Report Only mode while performing orchestration for the first time. 

4. Click the Attribute Map tab to check the mapping of attributes from Source to Target. Change the mappings as per your requirement. 

5. Set the orchestration schedule as required.

6. Click Create.

For details on the individual configuration fields in the orchestration setup page, refer to the Synchronization Orchestration Setup Guide.

The orchestration executes as per the schedule set in the orchestration. You can also execute the orchestration manually. The changes made to the users data in BambooHR will be reflected in Okta.
