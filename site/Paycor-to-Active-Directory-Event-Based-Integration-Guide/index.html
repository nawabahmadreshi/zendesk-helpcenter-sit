Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing user creation, user updates, and user termination events from Paycor using Webhooks to Active Directory. This integration runs whenever a user is created, updated, or terminated in Paycor. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Paycor: Refer to the Paycor Configuration Guide for details.
    • Active Directory: Refer to the Active Directory Configuration Guide for details.
  • Set up Webhooks in Paycor: Your Paycor admin must configure webhooks in the Paycor application to trigger events when an employee is terminated. For details on the webhooks setup, refer to the Set Up Webhooks in Paycor guide.
    Note: Ensure that the Employee Information Updated event is enabled when setting up webhooks for employee termination.

Integration Setup

This section outlines how the integration must be configured to push termination events from Paycor to Microsoft Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Active Directory / Paycor [Event] in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors

Parameters

The Parameters section enables you to customize the integration settings and general settings according to your specific use case. It has the following settings:

Integration Settings

  • Enable User Management: Select this checkbox to enable user lifecycle actions in Active Directory. When enabled, the User Management section is displayed for configuring additional integration settings.
  • Enable Report Only-Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.

General Settings

  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Debugging: Select this checkbox to capture additional troubleshooting details during the integration run. When enabled, the Debug Mode section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.

User Management

Correlation Settings

  • Source Correlation Attribute: Select the Paycor attribute used to match a source user with an Active Directory user.
  • Target Correlation Attribute: Select the Active Directory attribute used to find the matching user.

User Management

  • Enable User Creation: Select this checkbox to create users in Active Directory when a matching user is not found.
  • Enable User Update: Select this checkbox to update existing Active Directory users when Paycor user data changes.
  • Enable User Termination: Select this checkbox to deactivate users in Active Directory when they are marked as terminated in Paycor. When enabled, the following fields are displayed:
    • Search OUs for Terminated Users: Select the OUs from the dropdown menu where the integration should look for users marked as terminated. Leave the field empty to search in all OUs.
    • Move Terminated Users To OU: Select the Active Directory OU from the dropdown menu to which users should be moved after their AD accounts are disabled during termination.
    • Leave Statuses Excluded from Group Membership and OU Changes: Select the leave statuses from the dropdown menu where the integration should keep the user’s AD group memberships and current OU unchanged.

Sensitive Fields

The Sensitive Field Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • PII fields: Select the field names that contain personally identifiable information (PII) and should be masked or protected.
  • GUID fields: Select the field names that contain globally unique identifiers (GUIDs) and should be masked or protected.
  • Employee ID fields: Select the field names that contain employee IDs and should be masked or protected.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Password Policy

Application Configuration

  • Use Custom Password Policy: Enable this toggle button to apply a custom password policy instead of the default Active Directory policy. When enabled, the Password Policy Configuration section is displayed for further configuration.

Password Policy Configuration

Configure the following fields to choose the password policy configuration:

  • Maximum Password Length: Set the maximum number of characters allowed in a password.
  • Minimum Password Length: Set the minimum number of characters required for a password.
  • Password Complexity Requirements: Select rules from the dropdown menu to specify the character composition rules that passwords must satisfy. It has the following options:
    • Requires UpperCase letters: Ensures that the password contains at least one uppercase letter (A–Z).
    • Requires LowerCase letters: Ensures that the password contains at least one lowercase letter (a–z).
    • Requires numbers: Ensures that the password contains at least one numeric digit (0–9).
    • Requires Special Characters: Ensures that the password contains at least one special character from the allowed special character set.
  • Allowed Special Characters: Select special characters from the dropdown menu that can be included in the password. 

Event

The Event tab is used to configure the webhook details. 

Provide the following details and save the changes:

  • Event Source: Select your Paycor application from which employee-termination events are triggered.
  • URL: Copy this URL and paste it into the appropriate webhook field on the Paycor Webhooks setup page. For details on setting up a webhook in the Paycor application, refer to the Set Up Webhooks in Paycor guide.
  • Refresh Token: Click Refresh Token to generate a new token. Copy the generated token and paste it into the Event Secret field of the Paycor webhook under the Event Subscription tab on the Demo Application page. For details on setting up a webhook in the Paycor application, refer to the Set Up Webhooks in Paycor guide.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
