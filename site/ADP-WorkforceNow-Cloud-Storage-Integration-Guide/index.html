ADP WorkforceNow / Cloud Storage Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The ADP Workforce Now to Cloud Storage integration extracts Workers and related resource types such as personal contacts, government IDs, and communication details with their selected attributes from the ADP Workforce Now API Central app, storing the data in the configured Cloud Storage environment. This data can be transferred to AWS S3 or Azure Data Lake Storage based on the chosen landing configuration. For each entity, a dedicated folder is created within the specified storage location, containing Parquet or JSON files with the data, along with Delta log files that track metadata about these files.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now - API Central: Refer to the ADP Workforce Now Configuration Guide for ADP API Central Customers for details.
    • Azure Data Lake Storage Gen2: Refer to the Azure Data Lake Storage Configuration Guide for details if you select Azure Data Lake Storage as the landing storage in Databricks settings.

Integration Setup

This section outlines how the integration must be configured to push worker's data from ADP Workforce Now to Cloud Storage.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / Cloud Storage in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Cloud Storage

This tab allows you to identify the storage where the data should be stored.

The following parameters are displayed on this tab:

  • Select the landing storage: Select the landing storage where the generated datasets will be written. The available options are:
    • AWS S3: Select this option if you want the data to be stored in AWS S3. When selected, the following parameters are displayed for configuration:
      • S3 Application: Select the configured AWS S3 application from the dropdown menu.
      • S3 Bucket Name: Enter the name of the S3 bucket where the generated datasets will be stored. 
      • S3 path prefix: Enter the folder path (prefix) within the bucket where the generated datasets will be written. In this path, each selected entity is created as a separate folder containing Parquet or JSON files, as per the selected file type, and Delta log files.
      • File type: Select the format used to store the generated datasets. Supported file types are Parquet and JSON.
    • Azure Data Lake Storage: Select this option if you want the data to be stored in Azure Data Lake Storage. When selected, the following parameters are displayed for configuration:
      • Select Azure Data Lake Application: Select the configured Azure Data Lake Storage application from the dropdown menu. 
      • ADLS File System Name: Enter the name of the Azure Data Lake file system (container) where the generated datasets will be stored. 
      • ADLS Prefix Path: Enter the folder path within the file system where the generated datasets will be written. In this path, each selected entity is created as a separate folder containing Parquet or JSON files, as per the selected file type, and Delta log files.
      • File type: Select the format used to store the generated datasets. Supported file types are Parquet and JSON.

Operation

The following sections are displayed under the Operation tab:

Execution

The Execution section allows you to test and validate the integration before running it for all users, configure synchronization settings, and handle exceptions.

Rollout Configuration

Configure the following option to control the execution behavior:

  • Report only mode: Turn on this toggle button to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Test record limit:  Enter the limit on the number of worker records processed in this run. Set a small number (for example: 10–50) to validate field mappings and data quality before running on the full population. Set to 0 to process all records.

Control how data sync runs from source

Configure the following synchronization settings:

  • Run full synchronization: Turn on this toggle button to execute the integration in the full sync mode. If this toggle button is not enabled, the integration will be executed in the incremental import mode after the first execution. By default, the first integration execution always runs in full sync mode. When this toggle button is turned on, the following parameters are displayed for further configuration:
    • Full sync frequency: Select the frequency by which the integration must run in full sync mode. For example, selecting Every 6 hours from the drop-down menu will execute the integration every 6 hours in full synchronization mode.
      For example, if the integration schedule is configured as Continuous and the Full sync schedule is configured as Every 6 hours, the integration executes continuously based on the configured schedule settings. However, one execution in every 6 hours will be in full synchronization mode. All other executions between those full synchronizations are in incremental mode, if incremental sync is enabled.
      For instance, if a full synchronization execution happens at 12 AM, the next full synchronization execution happens at 6 AM. All executions between 12 AM and 6 AM are in incremental sync mode.
    • Full sync start window: Specify when the full sync schedule must begin. The following are the available options:
      • Start now: Select this option for the full sync to begin from the next scheduled run based on the configured schedule.
      • From specific date: Specify the date and time from which the full sync schedule should start, in the Full sync start date and Full sync start time fields.
    • Full sync start date: Select a date from the calendar on which the first scheduled full synchronization must be executed. 
    • Full sync start time: Select a UTC time from the drop-down menu at which the full synchronization must be executed on the configured start date.
    • Disable incremental synchronization: Select this checkbox to disable incremental executions. When selected, the integration is executed only in full synchronization mode and skips all incremental synchronization executions.
  • Reset last full sync timestamp: Enable this toggle button to clear the last full sync timestamp. The next run will be treated as a full sync. When enabled, the following parameters are displayed:
    • Reset to date: Select the date to reset the full sync timestamp.
    • Reset to time: Select the time to reset the full sync timestamp.

Exceptions handling

Configure the following exception handling settings:

  • Select application: Select the connected source or target application for which you want to configure exception handling.
  • Select exceptions: Select error codes from the dropdown menu to suppress or flag during sync execution.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
