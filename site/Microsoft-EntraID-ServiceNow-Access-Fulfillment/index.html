Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration between Microsoft Entra ID and ServiceNow keeps access packages synchronized across both systems. It retrieves the latest access packages from Entra ID, which include collections of permissions, groups, and applications, and compares them with the catalog choices in ServiceNow. Missing packages are added automatically while existing ones are left unchanged, ensuring the ServiceNow catalog reflects the most current access packages. The integration also retrieves group information from Entra ID and synchronizes it with ServiceNow, keeping group-related data up to date.

In addition, the integration automates access fulfillment. Approved ServiceNow tickets are processed to extract the details of the users and the requested access packages and groups. If the request is for an access package, the integration assigns the user to that package in Entra ID. If the request is for a group, the user is added to the appropriate group in Entra ID. This end-to-end automation ensures accurate provisioning and improves consistency across systems.

Key Functions

The Microsoft EntraID to ServiceNow Access Fulfillment integration provides the following functions:

  • Retrieves the latest access packages from Entra ID by gathering all access packages, which are collections of permissions, groups, and applications that users can request to gain the right level of access. 

  • Synchronizes ServiceNow with Entra ID packages by automatically adding any access packages that are missing in ServiceNow. This keeps ServiceNow aligned with Entra ID without requiring manual catalog maintenance.

  • Retrieves the latest groups from Entra ID by reading all group information and synchronizing it with ServiceNow, so administrators and requesters can always work with the most current group data when making or approving requests.

  • Assigns users to access packages in Entra ID by processing approved ServiceNow tickets. Each ticket contains details of the user and the requested package, and the integration uses this information to automatically assign the user to the correct access package in Entra ID.

  • Assigns users to groups in Entra ID by processing approved ServiceNow tickets. When a ticket specifies a group assignment, the integration reads the user and group details. It adds the user to the correct group in Entra ID, ensuring timely fulfillment of access requests.

Integration Setup

This section describes the configuration required for the integration to transfer access package data from Microsoft Entra ID to ServiceNow.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. From the left-side panel, navigate to Integrations.
  3. Click Create Integration.
  4. Select Entra ID to ServiceNow Access Fulfillment as the integration template.
  5. Enter a name and description for the integration.
  6. Select your Microsoft Entra ID connector instance as the source and your ServiceNow connector instance as the target.
  7. Click Continue. The Configuration page will appear, where you can set up the integration details.

Integration Scenarios

The following are the integration scenarios:

  • Enable Resource Management: Select to enable the different resource management scenarios.
  • Sync Microsoft Entra ID Access Packages to ServiceNow: Select to sync the Entra ID access packages to ServiceNow for access fulfillment.
  • Enable Microsoft Entra ID Access Fulfillment: Select to enable assigning users to access packages.
  • Sync Microsoft Entra ID Groups to ServiceNow: Select to sync the Entra ID groups to ServiceNow for group assignment.
  • Enable Microsoft Entra ID Group Assignment: Select to enable assigning users to Entra ID groups.

Access Package Sync Settings

The following fields are displayed if you have selected Service Catalog as the ServiceNow Component:

  • ServiceNow Service Catalog Variable: Select the ServiceNow service catalog variable under which the access packages must be created or updated.

Access Package Assignment Settings

Configure the following settings for assigning access packages to users:

  • ServiceNow Component: Select the ServiceNow functional component from which the approved access package assignment requests must be retrieved.
    • Select the Ticket or Service Catalogue based on your requirements.

The following fields are displayed if you have selected Ticket as the ServiceNow Component:

  • ServiceNow Ticket Type: Select the ServiceNow ticket type under which the access package assignment tickets are created.
  • ServiceNow Catalog Request Item: Select the ServiceNow catalog request item that has been used for creating access fulfillment tickets.

The following fields are displayed if you have selected Service Catalog as the ServiceNow Component:

  • ServiceNow Catalog Access Package Variable: Select the ServiceNow catalog variable that requests the access package details.
  • ServiceNow Catalog User Email Variable: Select the ServiceNow questionnaire variable that contains the email address of the user to whom the access package will be assigned.
  • ServiceNow Catalog Manager Approval Variable: Select the ServiceNow catalog variable that requests the manager’s approval details.
  • Manager Approval Status Value: Select the manager approval status value that should be regarded as approved.
