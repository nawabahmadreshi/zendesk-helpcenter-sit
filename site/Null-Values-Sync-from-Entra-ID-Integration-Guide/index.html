Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

Microsoft Entra ID currently cannot provision null attributes. If an attribute is set to empty for a user object within the Entra ID application, it will omit that attribute during outbound provisioning to the target applications. For details on this Entra ID limitation, refer to the Microsoft documentation.

To address this limitation, Aquera offers an integration that enables synchronization of null attribute values from Entra ID to a target application. This guide provides details on how to configure that integration and execute it.

Key Functions

The Null Values Sync from Entra ID integration performs the following functions:

  • Detects null values in specific attributes of users within the Entra ID application based on the integration configuration.
  • Synchronizes those null values to the mapped attributes in the configured target application.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Microsoft 365 Connector in the Aquera Admin Portal: Refer to the Microsoft 365 Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to synchronize null attribute values from Entra ID to a target system.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Null Values Sync from Entra ID.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Overview page is displayed with a short description of the integration capabilities.

The following tabs are displayed to configure the integration:

  • Source
  • Target

Source

The Source tab includes sections to configure the source application, the source attributes to be monitored, and the user group to be excluded.

Source Application Settings

This section includes the following settings:

  • Choose the Entra ID application instance: Select the Entra ID application connector to be used for this integration.
  • Select the enterprise application in Entra ID to sync user attributes: Select the Enterprise application in your Entra ID instance from which the user attributes containing null values must be synced to a target system.

Manage Attributes for Null Value Sync and Group Exclusions

This section includes the following settings:

  • Select the user attributes to be considered for null value sync: Select the user attributes that the integration must check for null values. Only the attributes configured in this field will be synchronized when null values are detected.
  • Enter the group name to be excluded from sync: Enter the name of the user group in Entra ID that must be excluded from the integration. The null values sync will not be performed for the users in the provided group.

Target

The Target tab includes sections to configure the target application endpoint URL and the user matching condition.

Target System Connection

This section includes the following setting:

  • Target partner base URL: Enter the API endpoint URL of the partner system to which the user attributes containing null data must be sent for synchronization.

Match Users Between Source and Target

This section includes the following settings:

  • Select how users should be matched in the target system: Select one of the following options to indicate how users must be matched between the source and target systems:

    • Correlate users using the Entra ID user email address or UPN: Select this option to use either the email address of the User Principal Name (UPN) of the users to find the matching users in the source and target systems.

    • Correlate users using a custom extension attribute: Select this option if you want to use an Entra ID extension attribute to match users.

  • Enter the extension attribute to match users: Provide the extension attribute name that must be used to match the users between the source and target systems. This field is displayed only if you have selected the Correlate users using a custom extension attribute option from the Select how users should be matched in the target system drop-down list.
