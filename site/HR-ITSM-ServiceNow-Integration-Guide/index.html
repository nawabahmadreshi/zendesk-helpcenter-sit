Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration retrieves workforce data from the selected HR application, detects lifecycle events based on the configured look-back and look-ahead settings, evaluates any configured eligibility rules, and then creates, updates, closes, or sends notifications for service desk tickets based on the selected lifecycle event configuration

By using the HR application as the system of record, the integration enables organizations to automate service desk fulfillment processes throughout the workforce lifecycle, including candidate onboarding, employee onboarding, job changes, leave of absence, termination, and rehire events. The integration supports configurable lifecycle workflows, event-based request generation, automated ticket management, and multi-channel notifications, helping IT and business teams respond efficiently to workforce changes.

Key Functions

This integration automates service desk processes for lifecycle events with the following capabilities:

  • Identifies workforce lifecycle events from the HR application, including hire, change, leave, termination, and rehire events.
  • Creates service requests or tickets in the configured service desk application based on selected lifecycle events.
  • Supports configurable event timing to create tickets when an event is detected, before the event date, on the event date, or after the event date.
  • Allows custom eligibility rules to control which employee records are included in lifecycle event processing.
  • Automatically updates or closes existing service desk tickets based on configured ticket statuses.
  • Sends event notifications through email and Microsoft Teams.
  • Generates detailed logs, operational reports, and execution summaries.

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.

Integration Setup

This section outlines how the integration must be configured to synchronize workforce lifecycle events from an HR application to a service desk application and automate the creation and management of service requests and tickets.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for HR / ITSM - ServiceNow in the search box next to the Explore Templates section and click it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • Features
  • Source
  • Service Desk
  • Operations
  • Tables
  • Schedule

On this tab, you can view general information about the integration, including its purpose, functionality, and supported capabilities. This section provides a high-level overview of how the integration operates between connected applications.

Features

The Features tab provides an overview of the functional components supported by the integration. These features define the capabilities available within the integration and determine the configuration tabs and settings that are available.

Note: The features displayed on this tab are mandatory for this integration and are enabled by default. These features cannot be modified or disabled because they are required for the integration to function correctly.

HCM Bridge

The HCM Bridge section controls workforce lifecycle processing within the integration.

  • Enable HCM bridge: Enable this option to process workforce lifecycle events from the source application, including onboarding tickets, organizational changes, termination events, and rehire events. When enabled, the Source section is displayed for further configuration.

Service Desk Bridge

The Service Desk Bridge section controls service requests and ticket automation within the integration.

  • Enable service desk bridge: Enable this option to create and manage service requests or tickets based on workforce lifecycle events. When enabled, the Service Desk section is displayed for further configuration. 
    The following option is available:
    • JML lifecycle event: Select this checkbox to create and manage service desk tickets for Joiner, Mover, and Leaver (JML) lifecycle events. When enabled, the Lifecycle Event section is displayed for further configuration.

Operations

The Operations section controls the operational execution capabilities of the integration.

  • Enable operations: Enable this option to allow the integration to run scheduled synchronization jobs, operational workflows, monitoring activities, and execution processes.

Source

The Source tab enables you to configure the HR application used by the integration and define how workforce data is retrieved and processed. It also provides advanced settings to control change detection and future-dated workforce event processing.

Application Settings

The Application Settings section enables you to configure the HR application used by the integration. It allows you to select the application instance from which to retrieve workforce data.

Source Application

The Source Application section allows you to select the HR application connector instance from which workforce data will be retrieved:

  • Source application instance: Select the HR application connector that this integration will connect to for retrieving data. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

Source Configuration

The Source Configuration section allows you to configure how the integration detects workforce lifecycle events and processes future-dated workforce records:

  • Detect changes since (days): Enter the number of days to look back for detecting new or updated records. Only records created or updated within the specified period will be considered.
  • Future hire look-ahead (days): Enter the number of days in advance to retrieve future-dated hires. The integration retrieves workforce records with a hire date that falls within the specified look-ahead period.

Lifecycle Event

The Lifecycle Events section enables you to configure how the integration detects workforce lifecycle events and manages service requests or tickets associated with those events:

  • Create tickets for user lifecycle events: Enable this option to create tickets when configured user lifecycle events occur. When enabled, the Event Configuration section is displayed for further configuration.

Event Configuration

The Event Configuration section allows you to define how the integration processes specific workforce lifecycle events and manages the associated service requests or tickets.

If the List View toggle button is disabled, the following parameters are displayed:

  • Select event type: Select the user lifecycle event for which service desk tickets should be created. If you do not find your event type in the list, click the refresh icon available next to the field to reload the available events. The following are the available options:

    • Hire: Select this event to create tickets for new employee onboarding.
    • Change: Select this event to create tickets when employee information changes.
    • Leave: Select this event to create tickets for employee leave events.
    • Terminate: Select this event to create tickets for employee termination events.

    Notes:

    • Each lifecycle event type is evaluated independently. Failure to create or update a ticket for one lifecycle event does not prevent the integration from processing other configured lifecycle events for the same worker.
    • If a worker has multiple lifecycle events within the same period (for example, Hire, Terminate, and Rehire), the integration processes the most recent lifecycle event based on the effective event date. For example, if Hire, Terminate, and Rehire events are detected for the same worker, the Rehire event will be processed.
  • Event timing: Select when the service desk ticket should be created for the selected lifecycle event. The following are the available options:
    Note: For Hire and Rehire events, the When Detected option supports only present and future effective dates. For Change, Leave, and Terminate events, the When Detected option supports past, present, and future effective dates.
    • When Detected: Select this option to create the ticket immediately when the event is detected.
    • Before Start Date: Select this option to create the ticket before the start date.
      • Days before/after joining date: Enter the number of days before the effective date of the selected lifecycle event when the service desk ticket should be created.
    • On Start Date: Select this option to create the ticket on the start date.
    • After Start Date: Select this option to create the ticket after the start date.
      • Days before/after joining date: Enter the number of days after the effective date of the selected lifecycle event when the service desk ticket should be created.
  • Enable custom eligibility rule: Enable this to process only employees who satisfy the eligibility conditions defined below. Employees who do not meet the conditions will be excluded from event processing. When enabled, additional configuration options are displayed:
    Notes: 
    • If Enable custom eligibility rule is enabled and no rule is specified, the integration returns a validation error.
    • The configured eligibility rule is evaluated for ticket creation, ticket updates, and ticket closure operations.
      • Custom eligibility rule builder: Define the eligibility conditions employees must meet to be included in event processing. Only employees who satisfy these conditions will be processed for the selected event type and any associated actions, such as ticket creation or updates. This field uses Aquera Intelligence (AI Assistant). For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
        Note: For Hire event ticket creation conditions, the Expression Builder may display attributes from multiple lifecycle event contexts. Currently, there is no option to refresh, filter, or customize the attribute list to display only Hire-specific attributes.
  • Enable ticket creation: Enable this option to automatically create a service desk ticket for configured lifecycle events. When enabled, additional configuration options are displayed:
    • Ticket type: Select the type of service desk ticket or request to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available tickets. The following are the available options:
      • Service Catalog: Select this option to create and process requests through the service desk Service Catalog.
        • Ticket category: Select the service desk ticket category to assign to tickets created for the configured lifecycle event. If you do not find your ticket category in the list, click the refresh icon available next to the field to reload the available tickets. For example, select Application and Account Access.
      • Ticket: Select this option to create and process standard service desk tickets.
    • Ticket type name: Select the name of the service desk ticket type to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available ticket types. For example, select Change Password. 
    • Customize how to match the employee from the source with existing tickets: Enable this option to customize how employees from the HR system are matched with existing service desk tickets. When enabled, additional configuration options are displayed:
      Notes: 
      • When a custom correlation attribute is configured with a dynamically generated value and the corresponding attribute is not mapped in the Ticket field mapper, the integration treats each execution as a new request and may create multiple tickets for the same worker. 
      • The same custom correlation attribute, such as Person Number, cannot be reused across multiple lifecycle event types. Reusing the same correlation attribute may cause ticket creation to be skipped.
        • Select the source attribute for matching the employee: Select the HR system attribute to uniquely identify and correlate users between both systems.
        • Select the target attribute for matching the employee: Select the service desk attribute to uniquely identify and correlate users between both systems.
    • Enable ticket auto-update: Enable this option to automatically update service desk JML tickets only when their current status matches one of the selected ticket statuses. When enabled, additional configuration options are displayed:
      • Ticket statuses for auto-update evaluation: Select the service desk ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic updates. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Open.
      • Resolved statuses for auto-updated tickets: Select the service desk ticket status to assign to the ticket after it is automatically updated. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. 
    • Enable ticket auto-close: Enable this option to automatically close the service desk JML ticket after all actions associated with the user lifecycle event have been completed successfully. When enabled, additional configuration options are displayed:
      Note: For incident-based ticket integrations, the Close Code and Close Notes attributes must be mapped in the Ticket field mapper to successfully update ticket status to Resolved or Closed. 
      • Ticket statuses for auto-close evaluation: Select the service desk ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic closure. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
      • Resolved statuses for auto-closed tickets: Select the service desk ticket status to assign to the ticket after it is automatically closed. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
    • Ticket field mapper: This section allows you to map employee data fields from the HR application to the corresponding service desk ticket fields. Source attributes are listed on the left, and target ticket fields are listed on the right. You can modify the source attributes or expressions as needed to align with your use case.
      JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
      Attributes marked with a red asterisk (*) are required for successful provisioning.
      The Search Attributes field allows you to filter the list of available attributes for mapping.
      After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.
      Note: To populate assignment group information in created tickets, map either the assignment group System ID or the assignment group Name in the Ticket field mapper.

    • Enable ticket creation through email notification: Enable this option to create tickets through email notifications when the configured lifecycle event is detected. When enabled, additional configuration options are displayed:

      • Ticket creation email subject: Enter the content that must be used as the email subject in the notifications sent for ticket creation.
      • Ticket creation email recipients: Select one or more recipient groups to whom the email notifications will be sent for ticket creation.
      • Custom recipient email addresses: Enter the email addresses that should be included as custom recipients for notifications sent for ticket creation.
      • Ticket creation email template: Enter the content that must be used as the email body in the notifications sent for ticket creation.

      Notes: 

      • When a configured lifecycle event is triggered, the integration sends a notification to the configured recipients. The recipient can then create the service desk ticket based on the information provided in the notification.
      • When Enable ticket creation through email notification is enabled, the integration sends a notification for every integration run that meets the configured lifecycle event criteria. Each notification can result in the creation of a new service desk ticket. The integration does not perform duplicate ticket detection or duplicate ticket prevention for tickets created through notifications.
    • Enable ticket creation through teams notification: Enable this option to send Microsoft Teams notifications when the configured lifecycle event is detected. When enabled, additional Teams notification settings are displayed: 
      Note: When a configured lifecycle event is triggered, the integration sends a notification to the configured Microsoft Teams channel. The notification can be used to inform stakeholders about the event and provide details required for service request or ticket processing.
      • Select team: Select the Microsoft Teams team where ticket creation notifications will be posted. If you do not find your team in the list, click the refresh icon available next to the field to reload the available teams.
      • Select channel: Select the Microsoft Teams channel within the selected team where ticket creation notifications will be posted. If you do not find your channel in the list, click the refresh icon available next to the field to reload the available channels.
      • Teams notification template: Enter the content that must be used as the notification message sent for ticket creation.

Tables

In the Tables tab, you can add tables to the integration. Tables are used to define custom source attributes and synchronize them with the target application. If the required data is not available in the source application, you can define it in a table and use it to update the target application. Once defined, the table can be referenced in attribute mapping expressions to set or update values automatically.

For example, to set the timezone attribute of users in the target application to Europe/Paris, create a table during integration setup with timezone as the attribute name and Europe/Paris as the value. Once defined, reference the table in the source attribute of the Ticket field mapper using the following format:
TABLE.<table_name>.<key_name> or TABLE.<table_name>[<key_name>]

For example: TABLE.test.timezone

Note: You can configure expressions with nested tables in the ticket field mapper, if required.
For example: TABLE.UserHireRecord[TABLE.UserRecord["empDate"]]

Refer to the Using Tables in Integrations guide for more details.
