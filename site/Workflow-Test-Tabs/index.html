Overview

Aquera provides a solution to set up a Workflow of tasks that can be executed on the data across applications without actually writing a script. The data can be users, groups, tickets, and so on based on the resource types supported by an application. The tasks that can be performed on the data are reading, creating, updating, deleting, and general tasks such as sending emails, creating service requests, and so on.

For example, you can achieve an automatic IT onboarding process for a user from an HR system to multiple applications by setting up a Workflow.

This guide explains how to create, set up, and execute a Workflow. JavaScript is internally created based on the Workflow that you have set up. When the Workflow runs, this script is executed. You can set the Workflow to run based on an event or a schedule set in the Workflow.

Prerequisites

Configure the required connectors before creating a Workflow in the Aquera Admin portal. Refer to the respective connector configuration guides for further details.

Create a Workflow

Perform the following steps to create a Workflow:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Workflows and click Create Workflow at the top right corner of the page.
  3. Enter a name for the Workflow in the Create Workflow window.
  4. Click Create to create the Workflow. Now, you can set up a sequence of tasks in the Workflow.

This section provides an introduction to the sections in the Workflow UI. The following are the sections in the Workflow UI:

Locate Steps 

The Locate Steps option displays all the steps in the Workflow for easy navigation. You can navigate to a step in the Workflow by clicking on the step name. You can also perform a search of the step name in the Search bar and click on the step name to jump to the respective step in the Workflow.

The following are the sections in the Components:

Workflow Editing Canvas

Workflow Editing Canvas is the area where you create the Workflow to set up a sequence of tasks. It has the following sections:

Cards

Cards are the elements in the Workflow. The Event Trigger / Schedule Trigger is the default card in the Workflow.

Configuration Panel

The Configuration Panel is displayed on the right of the Workflow page when you click a card. This panel enables you to configure a card. Refer to the Card Configurations section for further details.

Set a Trigger

You can set the Workflow to run based on a schedule or an event (for example, user data changes) in an application. Click the Event Trigger / Schedule Trigger card that will pop up the Trigger Configuration panel with the options to configure a trigger for the Workflow execution. 

Schedule Trigger

You can set a schedule to trigger the Workflow to run automatically at specific intervals. Select a Basic schedule or an Advanced schedule option depending on the frequency by which the Workflow should be run.

Basic Schedule

You can set a basic schedule by which the Workflow should be automatically run from the following list of time intervals:

  • 15 minutes
  • 30 minutes
  • 1 hour
  • 4 hours
  • 6 hours
  • 12 hours
  • 24 hours

Advanced Schedule

You can set an advanced schedule by which the Workflow should be automatically run from the following list of available options:

  • Select Continuous to run the Workflow continuously.
    Note: There can be a delay of maximum 5 minutes to start the next instance of the Workflow after the previous instance ends.

  • Select Minutes to schedule the Workflow to run every X minutes, where you need to select X minutes from the Repeat Every dropdown list. The available options are from 15 to 60 minutes.

  • Select Hourly to schedule the Workflow to run every X hours, where you need to select X hours from the Repeat Every dropdown list. The available options are from 1 to 24 hours.

  • Select Daily to schedule the Workflow to run every X days at a specific time, where you need to select the day frequency from the Repeat Every dropdown list and select the time from the Starting at(Time) dropdown list. You can set up a maximum of 8 daily schedules if required. The available options for the days are from 1 to 6. The Starting at(Time) that you select here is the UTC time.

  • Select Weekly to schedule the Workflow to run on specific days of the week at a specific time, where you need to select the days in the Repeat Every field and select the time from the Starting at dropdown list. The Starting at time that you select here is the UTC time.

Note: If you select Never, the Workflow never runs automatically. You can manually run the Workflow when required by selecting the Execute button at the top right of the Workflow page.

Event Trigger

The Workflow will be triggered when specific events occur in an application. An event notification occurs when there is a data change in an application such as a new user creation, existing user modification, user deactivation, and so on.

Event Source: The application from which the event notifications have to be captured. You can either type the name of the connector or select the connector from the dropdown list.

Event(s): Select the type of event from the dropdown list. The following is the list of available event types:

  • User Created by Sync Orchestration: When a user is created in the target application during the execution of a Synchronization orchestration, an event will be passed from the Synchronization orchestration to the Workflow. The Workflow will be triggered for each target user creation and the event request will contain the user object of the created user. To receive the event notification, configure the New User Orchestration Hook by referring to the Synchronization orchestration guide.
  • User Updated by Sync Orchestration: When a user is updated in the target application during the execution of a Synchronization orchestration, an event will be passed from the Synchronization orchestration to the Workflow. The Workflow will be triggered for each target user update and the event request will contain the user object of the updated user. To receive the event notification, configure the Update User Orchestration Hook by referring to the Synchronization orchestration guide.

  • User Deactivated by Sync Orchestration: When a user is deactivated in the target application during the execution of a Synchronization orchestration, an event will be passed from the Synchronization orchestration to the Workflow. The Workflow will be triggered for each target user deactivation and the event request will contain the user object of the deactivated user. To receive the event notification, configure the Deprovision User Orchestration Hook by referring to the Synchronization orchestration guide.
  • Users Modified Since: This option will be shown only if the connector mentioned in the Event Source supports incremental import of users. (The connector should support filtering users by the meta.lastModified.) The Workflow will be triggered for each user modified based on the meta.lastModified date and the event request will contain the user object of the modified user. 
  • Application events: A list of application events will be shown if the connector mentioned in the Event Source supports the events resource type. The Workflow will be triggered when there is a data change in the application and the event request will contain the attributes of the event schema. 
    Note: The events for which the Workflow needs to be triggered may need to be configured in the respective application UI. Refer to the respective Connector Configuration Guides for further details on events.

    For details on accessing the event request, refer to the Accessing Event Request section. 

Schedule Event Execution: Set a schedule for the execution of the event trigger. The following is the list of available options:

  • Select Never so the Workflow never runs.

  • Select Continuous to run the Workflow continuously.
    Note: There can be a delay of maximum 5 minutes to start the next instance of the Workflow after the previous instance ends.

  • Select Minutes to schedule the Workflow to run every X minutes, where you need to select X minutes from the Repeat Every dropdown list. The available options are from 15 to 60 minutes. Set the start date and the end date for the schedule to run by selecting the date using the calendar option in the Schedule Start Date and Schedule End Date fields respectively.

  • Select Hourly to schedule the Workflow to run every X hours, where you need to select X hours from the Repeat Every dropdown list. The available options are from 1 to 24 hours. Set the start date and the end date for the schedule to run by selecting the date using the calendar option in the Schedule Start Date and Schedule End Date fields respectively.

  • Select Daily to schedule the Workflow to run every X day at a specific time, where you need to select the day frequency from the Repeat Every dropdown list and select the time from the Starting at(Time) dropdown list. You can set up a maximum of 8 daily schedules if required. The available options for the days are from 1 to 6. The Starting at(Time) that you select here is the UTC time. Set the start date and the end date for the schedule to run by selecting the date using the calendar option in the Schedule Start Date and Schedule End Date fields respectively.

  • Select Weekly to schedule the Workflow to run on specific days of the week at a specific time, where you need to select the days in the Repeat Every field and select the time from the Starting at dropdown list. The Starting at time that you select here is the UTC time. Set the start date and the end date for the schedule to run by selecting the date using the calendar option in the Schedule Start Date and Schedule End Date fields respectively.

Setup the Workflow

This section explains how to set up a Workflow containing a sequence of tasks that must be executed when the Workflow runs. You can perform the following actions to set up a Workflow:

Drag and Drop a Card

To set up a sequence of tasks in the Workflow, drag the required item from the Components section and drop the item on the + icon of the Workflow.

For example, as shown in the following image, drag the Log item from the Components section and hover on the + icon and drop the Log item when a blue line appears over the + icon.

Configure a Card

To configure a card in the Workflow, click on a card that will open a configuration panel on the right of the Workflows page where you can provide the required configurations. Click the collapse icon to close the configuration panel. Refer to the Card Configurations section for further details on configuring the fields in the respective card configuration panel.

The fields marked with a red asterisk (*) are mandatory fields. If you do not configure the mandatory fields for a card, an alert icon will be shown on the card and the message Incomplete Configuration will be displayed when you hover over the alert icon. 

Delete a Card

To delete a card, hover over a card on the Workflow and click the delete icon. A Delete Element confirmation message will pop up asking you to confirm deleting the card.

If you are performing the deletion of a card of which the application attributes are used in the elements below it, then all the elements below it will be deleted. In this case, a Delete Element confirmation message will pop up asking you to confirm deleting all the elements below it.

Reset the Workflow

You can zoom in and zoom out the Workflow on the Workflow editing canvas. Click the + or - on the Reset button on the Workflow editing canvas to perform the zoom-in and zoom-out. Click the Reset button to bring the Workflow back to its actual position and normal size. You can also drag the Workflow on the Workflow editing canvas to view the elements accordingly.

Import All Operation

The Import All operation retrieves all the data records from an application. The Import All operations can be Import All Users, Import All Groups, Import All Roles, and so on based on the resource type supported by the application.

Input

You can provide the following filter conditions if required, to retrieve specific data records from the application:

Field Description

Operator

Select an operator from the dropdown list. For details on the supported operators, refer to the Filtering Data section.
Value

Enter a value for the attribute.

Note: You can include attribute values of the previous elements of the Workflow hierarchy in this field using the $ symbol. 
+ Add more

Click the + Add more button to add a new filter condition.

Note: If multiple filters are added, then the records satisfying all the filter conditions are retrieved.

Click the delete icon to delete a filter condition.

Refer to the respective Connector Configuration Guide for further details on the attributes.

Output

The output is a system-defined variable name that stores the array of the imported data records. 

By default, the variable name is created in the format, <operationName><sequence_number>. The sequence number will be incremented for the same operation for every application added to the Workflow. For example, if the operation is importAllUsers, the first output variable name will be importAllUsers, the next incremental output variable name will be importAllUsers1, and so on. 

Note: Based on your requirements, you may need to add a loop after an Import All operation since this operation will return an array of data objects that you need to iterate over to retrieve a single data object.

For example, if the Workflow includes an application with Import All Users, then you need to add a loop after this task to iterate over the variable importAllUsers and each user object will be stored in the output variable singleImportAllUser.

User Actions on the Workflow

The following are the UI elements to perform user actions on Workflows:

  • Execute: Click Execute on the top right corner of the Workflow page to manually run the created Workflow.
  • Bell Notification Icon: Click the  icon that displays a list of configuration panel notifications for which you have not configured the mandatory configuration fields and click an item from the list to display the respective configuration panel. 

  • Vertical Ellipsis: Click the vertical ellipsis at the top-right corner of the Workflow page to view the following options:
    • Settings: Click Settings to display the Settings Configuration panel to set the log level and execution mode of the Workflow. 

      The following are the supported log levels: 

      Log Level Description
      Default Does not print anything in the logs.
      Info Prints each step of the Workflow execution in the logs.
      Verbose Prints each step of the Workflow execution along with the input and output values in the logs.

      The following are the supported execution modes: 

      Execution Mode Description
      Normal If any step in the Workflow execution fails, then it prints the error messages in the logs and the Workflow continues execution.
      Exception If any step in the Workflow execution fails, then it prints the error messages in the logs, and the Workflow stops execution.          
    • Latest Logs: Click Latest Logs to view the logs of the most recent execution of the Workflow on the log pane. Click Goto Logs on the log pane to navigate to the Logs module to view the current Workflow logs.
      Note: The NAME column in the Logs module for the Workflow logs will be mentioned as FlowBuilder.

    • Delete: Click Delete to launch the Delete Workflow pop-up window and click Confirm to delete the Workflow.

Length Property

When you perform an import all operation on an application, it will return an array of objects. The objects can be users, groups, roles, tickets, or any resource type supported by the application. The total count of the objects returned by the application is specified by the length property. You can select the length property from the dropdown list in any card configuration.

For example, if you perform an Import All Users operation, the importAllUsers object stores the array of users returned by the application, then importAllUsers.length returns the total count of the users.
