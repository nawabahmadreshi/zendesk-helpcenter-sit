Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the integration to sync Candidate data from ADP Workforce Now Recruiting Management to the ticketing resource in Zendesk.

Key Functions

The ADP Workforce Now / Zendesk integration automates the onboarding tickets with the following primary capabilities:

  • Retrieves all the newly created candidates data from ADP Workforce Now.
  • Creates a Zendesk onboarding ticket for each newly created candidate.
  • Generates logs, reports, and analytics dashboards based on the configurations.
  • Sends notification emails with reports on each integration run based on the configuration.

Prerequisites

The following are the prerequisites:

  • Admin Access to Zendesk: You must have admin access to your Zendesk instance. You can visit the Zendesk portal to get access to Zendesk instance.
  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal to integrate with Zendesk:
    • ADP Workforce Now: Ensure that you configure the ADP Workforce Now connector in the Aquera Admin portal. Refer to the ADP Workforce Now Configuration Guide for configuration details.
    • Zendesk: Ensure that you configure the Zendesk connector in the Aquera Admin portal. Refer to the Zendesk Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating ADP Workforce Now / Zendesk. It outlines the steps to create the integration, set up integration preferences, and define how candidate data is mapped from ADP Workforce Now / Zendesk.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / Zendesk in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select ADP Workforce Now / Zendesk.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Scenarios 

  • Enable Candidate Management: Select this checkbox to allow creating Zendesk tickets for newly created candidates in ADP Workforce Now. When enabled, the Candidate Management tab is displayed to configure integration settings for candidate management.

General Settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.
  • Enable Email Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Email Notification Settings section is displayed for further configuration.

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable Logging setting in the Parameters section.

  • Select log type: Select the log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Log Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.
  • Enter PII Fields: Enter the fields that contain personally identifiable information (PII) to be included in logs. For example, adding dateOfBirth or SSN includes these fields based on the selected log type.
  • Enter GUID Fields: Enter the fields that contain globally unique identifiers to ensure they are handled correctly in logs. For example, adding workerId ensures the value is treated as a system identifier.
  • Enter EmployeeId Fields: Enter the fields that represent employee identification values to be logged consistently. For example, adding associateId ensures the employee identifier is included in logs.

Candidate Management

The Candidate Management tab enables you to customize the integration settings for candidate management according to your specific use case.
Note: This section appears only if you have selected the Enable Candidate Management setting in the Parameters section.

It has the following settings:

  • Select ADP Workforce Now Application Instance: Select the ADP Workforce Now application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Candidate Filter Types: Select the required filter to fetch the candidates data from ADP Workforce Now based on the use case.
  • Enable Onboarding Ticket Creation: Enabling onboarding ticket creation will create a ticket in Zendesk for every matching candidate from ADP Workforce Now.
  • Application Status: Select to filter out the candidates from ADP Workforce Now. When selected, the following field will appear.
    • Include Candidates With Application Status: This field allows you to select the required application statuses of the candidates that you want to retrieve.
  • Custom Status: Select this option if you want to provide a custom status to retrieve candidates. When selected, the following field will appear.
    • Custom Application Statuses: Enter the custom application status value to retrieve specific candidates.
  • Lookback window: Select this option to provide the number of days for which the candidates data should be retrieved. When this option is selected, the Candidate Lookback Window field will appear.
    • Candidate Lookback Window: Set the number of days to retrieve candidates whose records have been created or updated within that period.
  • Email: Select this option if you want to filter a specific candidate by their email. When the Email option is selected, the Filter Candidates by Email section will appear.
    • Filter Candidates by Email: Enter a full or partial email address to filter candidates by email. Leave blank to retrieve all candidates.

Zendesk

The Zendesk tab enables you to customize the integration settings for Zendesk application according to your specific use case.

It has the following settings:

  • Select target application: Select the Zendesk application instance in which it will create a ticket. Select the ADP Workforce Now application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Match Employees Between ADP Workforce Now and Zendesk: Select the method by which employees from ADP Workforce Now are identified against Zendesk to make updates, reuse, or duplicate prevention.
    • Enable Customize How Employees Are Matched Across Systems: Enabling this allows to implement custom logic and to match employees from ADP Workforce Now with existing Zendesk records. When enabled, the following fields are displayed.
      • Select How Employees Are Matched Between ADP Workforce Now and Zendesk: This field identifies the correlation attributes to match records in ADP Workforce Now and Zendesk.
        • Select the source attribute for matching the employee: Selects the ADP candidate attribute whose value will be used to identify in the Zendesk record.
        • Select Target Attribute for Matching the Employee: Selects the Zendesk ticket attribute to be matched against the ADP candidate correlation value.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
