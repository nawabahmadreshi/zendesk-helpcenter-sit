Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration provides a SCIM-based provisioning system that synchronizes user data between Microsoft Entra ID (formerly Azure AD) and target applications.

By default, the integration uses the User Principal Name (UPN) from Entra ID and matches it with the Username in the target application to identify user records, unless custom correlation attributes are configured in the Provisioning section.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes user data from Entra ID to target applications.
  • Manages group memberships.
  • Sends notifications about provisioning events via email and Microsoft Teams.
  • Generates detailed logs and reports.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • User Account in Entra ID: You must have a user account in Entra ID with permission to configure provisioning (for example, Application Administrator, Cloud Application Administrator, Application Owner, or Global Administrator).
  • Enterprise Application in Entra ID: You must have an enterprise application in Entra ID for each target application. Refer to the Creating an Enterprise Application in Entra ID section for further details.
  • Microsoft 365 Application Connector in the Aquera Admin Portal: You need to configure the Microsoft 365 application connector in the Aquera Admin Portal. Refer to the Microsoft 365 Configuration Guide for configuration details.
  • Target Application Connector in the Aquera Admin Portal: You need to configure a connector in the Aquera Admin Portal for your target application. For example, if Box is your target application, then you must have a Box application connector in the Aquera Admin Portal. Refer to the configuration guide for your application connector for configuration details.
    Note: If your use case requires multiple target applications, then you must have a connector for each target application.

Creating an Enterprise Application in Entra ID

You must have an enterprise application in Entra ID for each target application included in the integration. This enterprise application will serve as a source for user records and will be used to provision users to the target applications.

Perform the following steps to create an enterprise application:

  1. Log in to your Microsoft Azure portal.
  2. Search and select Enterprise Applications.
  3. Click New application.
  4. Search for the application for which you want to create an enterprise application, and select it from the search results. For example:
  5. Enter a name for the application, and click Create.

    The enterprise application is created and available on the Enterprise applications page.

Integration Setup

This section details the configuration and mapping process for integrating Entra ID with target applications. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Entra ID / Multi Application in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • Applications
  • Policy
  • Execution
  • Notifications
  • Tables
  • Schedule

Overview

On this tab, you can view details about the integration, including its configuration settings.

Applications

Navigate to the Applications tab to configure the provisioning settings for the integration. This tab allows you to select the applications involved in the integration and set up their connection.

The following sections are displayed on the page:

  • Source Configuration
  • PII Attributes
  • Provisioning

Source Configuration

This section enables you to identify the Microsoft Entra ID application connector used for the integration. The following field is displayed in this section:

  • Entra ID Application: Select the Entra ID application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

PII Attributes

This section enables you to define the PII attributes for the integration. The following field is displayed for configuration:

  • Select PII Attributes: Select the Entra ID attributes that you want to be considered as personally identifiable information.

Provisioning

This section enables you to configure application provisioning. In this section, you can define how records from Microsoft Entra ID are provisioned to the target application and how attributes are mapped between the systems.

If the List View toggle button is disabled, the following parameters are displayed:

  • Entra ID Enterprise Application Name: Select the enterprise application that you have created in your Azure portal for this integration. Click the refresh icon next to the field to retrieve all enterprise applications available in your Azure portal.

    Notes:
    • This field identifies the enterprise application created in the Azure portal to drive provisioning for this integration. This is the Entra ID application to which users or groups are assigned.
    • The enterprise applications available for selection depend on the Entra ID application connector selected in the Source Configuration section.
  • Select Target Application: Select the target application connector where records will be provisioned. For example, if your use case is to provision user records from Entra ID to the Box application, select the Box connector that you have created in the Aquera Admin portal for this integration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors.
  • Custom Lookup for User Correlation: By default, user records are matched using the User Principal Name (UPN) from Entra ID and the Username in the target application. Select this checkbox to use a different source and target attribute for correlation. When enabled, the following fields are displayed for further configuration:
    • Entra ID Attribute Used For Lookup: Select the correlation attribute in Entra ID that uniquely identifies a user record in your target application. For example, select employeeId if you want to use it to match user records in the target application.
    • Target Application Attribute Used For Lookup: Select the correlation attribute available in your target application that uniquely identifies a user record in Entra ID. For example, select ENT.employeeNumber if you want to use it to match user records in Entra ID.
  • Generate Password: Select this checkbox if your use case requires generating a password for new users in the target application. When enabled, the following fields are displayed for further configuration:
    • Password Generation Method: Select the method to generate passwords. This defines how passwords should be generated. The available options are as follows:
      • Generate Random Password: Select this if you want passwords to be generated automatically by the system.
      • Generate Password Using Policy: Select this if you want passwords to be generated based on a defined password policy. When selected, the following field is displayed for further configuration:
        • Select Password Policy: Select the password policy to be used for this integration. All password policies created in the Policy tab will be available for selection.
  • Attribute Mapper: This section allows you to control how each attribute is synchronized between Entra ID and the target application.
    Source attributes are listed on the left side and the target attributes are listed on the right side of the page. You can modify only the source attributes as needed, to align with your specific use case.
    JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
    Attributes marked with a red asterisk (*) are required for successful provisioning.
    The Search Attributes field allows you to filter the list of available attributes for mapping.
    After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.

To add another target application in this integration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the Application Provisioning Configuration section in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Policy

Navigate to the Policy tab to define policies for the integration. The following sections are displayed on the page:

  • Configuration
  • Application Groups
  • Business Groups
  • Password Policy
  • Notification Rule

Configuration

This section allows you to identify the method used to define group assignment policies.

In the Group Assignment Policy field, select one of the following methods to define how users are assigned to application groups.

  • Direct Group Assignment: Select this option if you want to assign users directly to groups in the target application based on user attributes, for example, department or job title. When selected, the Application Groups section is displayed for further configuration.
  • Business Group Assignment: Select this option if you want to introduce business facing groups used in your organization. In this assignment, users are first assigned to business groups based on attributes, and then business groups are mapped to the target application groups. When selected, the Business Groups section is displayed for further configuration.

Application Groups

This section allows you to define group assignment rules that directly map users to target application groups. When a user’s attribute value matches a rule, the user is automatically added to the defined target application group.

Note: This section appears only if Direct Group Assignment is selected in the Configuration section.

Perform the following steps to create rules for group assignments:

  1. In the Define Rule section, click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the assignment rule.
  2. Search and select the attribute you want to use to define a condition. For example, select Department.
  3. In the next row, double-click and select the attribute value to define the rule. If multiple values are selected, OR condition is applied among them. For example, in the following sample condition, a user's department must be either QA or Development to satisfy this assignment rule.
  4. If required to add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  5. If required to include another attribute to the group assignment rule, click the vertical ellipsis in the first column, and then click Add Attribute. If multiple columns are added for a rule, AND condition is applied between different columns. For example, in the following sample condition, a user's department must be either QA or Development, and the country must be United States to satisfy this assignment rule. You can use multiple attributes and their corresponding values to define the group assignment rule.
  6. In the Choose Target column, select the target application, and then select the target group names to which users who satisfy the specified condition will be assigned.
  7. If needed, click the + icon at the top right corner to add another rule, as explained above.
  8. Click Save Changes at the top-right corner of the page to save the configuration.

Note: If required to delete a row, click the horizontal ellipsis next to the row and select Delete Row.

Business Groups

This section allows you to manage group assignments using a business-level grouping layer that maps to the target application groups. With this method, users are first assigned to business groups, which are then mapped to target application groups.

Note: This section appears only if Business Group Assignment is selected in the Configuration section.

Perform the following steps to create rules for group assignments:

  1. In the Business Groups field, enter the group names to define business groups used in your organization. Users are assigned to these business groups, which can then be mapped to one or more application groups.
  2. In the User-to-Business Group Rules section, perform the following steps to create rules that assigns  users to one or more business groups based on user attributes:
    1. Click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the assignment rule.
    2. Search and select the attribute you want to use to define a condition. For example, select Department.
    3. In the next row, double-click and select the attribute value to define the rule. If multiple values are selected, OR condition is applied among them. For example, if you select QA and Development in this field, a user's department must be either QA or Development to satisfy this assignment rule.
    4. In the Business Groups column, select the business group to which users who meet the specified condition will be mapped.
    5. If required to add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
    6. If required to include another attribute to the group assignment rule, click the vertical ellipsis in the first column, and then click Add Attribute. If multiple columns are added for a rule, AND condition is applied between different columns. For example, if you select QA for the Department attribute and United States for the Location attribute, the user must belong to QA and be located in the United States to meet this assignment rule.
    7. If needed, click the + icon at the top right corner to add another rule, as explained above.
  3. In the Business Group-to-Application Group Mapping section, perform the following steps to create rules that maps each business group to corresponding groups in the target application:
    1. In the first column, select the business group for which you want to define rule.
    2. In the Choose Target column, select the target application, and then select the target group names to which users who satisfy the specified condition will be assigned.
    3. If needed, click the + icon at the top right corner to add another rule, as explained above.
  4. Click Save Changes at the top-right corner of the page to save the configuration.

Note: If required to delete a row, click the horizontal ellipsis next to the row and select Delete Row.

Password Policy

Navigate to the Password Policy section to define how passwords are generated in the target applications. This section allows you to configure password generation rules, complexity requirements, and length constraints.

If the List View toggle button is disabled, the following parameters are displayed:

  • Policy Name: Enter a unique name to identify the password policy.
  • Maximum Password Length: Enter the maximum number of characters allowed in a password.
  • Minimum Password Length: Enter the minimum number of characters required for a password.
  • Password Complexity Requirements: Select the password complexity requirements. This defines the character types that must be included in the password. The available options are as follows:
    • Require Uppercase Letters: Select this if you want passwords to include at least one uppercase character (A–Z).
    • Require Lowercase Letters: Select this if you want passwords to include at least one lowercase character (a–z).
    • Require Numbers: Select this if you want passwords to include at least one numeric character (0–9).
    • Require Special Characters: Select this if you want passwords to include at least one special character. If you select this option, the following field is displayed for further configuration:
      • Allowed Special Characters: Select the special characters that are allowed to be included in passwords. Only the selected characters can be used when special characters are required.

To add another password policy, click the + icon.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the configurations in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Notification Rule

Navigate to the Notification Rule section to define the policy that controls when and how notifications are triggered.

Perform the following steps to define a notification rule:

  1. In the Define Rule section, click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the rule.
  2. Search and select the attribute you want to use to define a condition. For example, select Department.
  3. In the next row, double-click and select the attribute value to define the rule. If multiple values are selected, OR condition is applied among them. For example, in the following sample condition, a user's department must be either QA or Development to satisfy this rule.
  4. If required to add another condition for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  5. If required to include another attribute to the group assignment rule, click the vertical ellipsis in the first column, and then click Add Attribute. If multiple columns are added for a rule, AND condition is applied between different columns. For example, in the following sample condition, a user's department must be either QA or Development, and the country must be United States to satisfy this rule. You can use multiple attributes and their corresponding values to define a rule.
  6. In the Choose Target column, select Template and then select all template names that you want to include in the rule.
  7. If needed, click the + icon at the top right corner to add another rule, as explained above.
  8. Click Save Changes at the top-right corner of the page to save the configuration.

Note: If required to delete a row, click the horizontal ellipsis next to the row and select Delete Row.

Notifications

Navigate to the Notifications tab to configure notification settings for the integration. The following sections are displayed on the page:

  • Channels
  • Configuration

Channels

This section enables you to customize the notification channels to be used for the integration.

It has the following settings:

  • Enable Email: Select this checkbox to enable integration-related email notifications.
  • Enable Chat: Select this checkbox to enable chat for the integration. When enabled, the following field is displayed:
    • Select Application for Chat Notification: Select the Entra ID application for which you want to enable chat.

Configuration

This section allows you to define notification settings for provisioning events.

If the List View toggle button is disabled, the following parameters are displayed:

  • Template Name: Enter a name for the notification template.
  • Select Applications: Select a target application for which you want to configure the notification settings. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors.
  • Select Event for Notification: Select the provisioning event that will trigger a notification. For example, select Account Provisioned to get email notifications every time accounts are provisioned to the target application.
  • Custom Condition to Send Notification: Enable the toggle button if you want to set a custom condition for notifications. When enabled, the following field is displayed for further configuration:
    • Custom Condition: Select a value to define the custom condition.
  • Notification Delivery Channel: Select the channel to be used for sending notification. The available options are:
    • Email: Select this option  if you want to send notification through email. When selected, the following fields are displayed for further configuration:
      • Custom Notification Recipients: By default, notifications are sent to the user’s work email ID. If you want to send notifications to any other recipients, enable this checkbox.
        When enabled, the following fields are displayed for further configuration:
        • Send To: Select the additional recipients to send email notifications. For example, select Supervisor if you want to send email notifications to the user's supervisor. The following field is displayed when you select Custom in this field:
          • Custom Email Address: Enter the email IDs of the users to whom you want to send email notifications.
      • Subject: Enter the content that must be used as the email subject.
    • Chat: Select this option if you want to send notification through chat. When selected, the following fields are displayed for further configuration:
      • Select Team: Select the team created in Microsoft Teams that will be used to send chat notifications.
      • Select Channel: Select the channel in the selected team where chat notifications will be sent.
  • Template: Enter the content that must be used as the email or chat body for notifications. You can use available customization features, such as variables and Improve Description to configure the email body.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
