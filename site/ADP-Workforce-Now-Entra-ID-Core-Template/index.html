Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration connects ADP Workforce Now with Microsoft Entra ID to automate end-to-end user lifecycle management.

This guide explains how to set up the ADP Workforce Now / Entra ID integration.

Key Functions

The ADP Workforce Now / Entra ID integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes employee records from ADP Workforce Now to Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.
  • Creates new users in Entra ID when they are added in ADP Workforce Now.
  • Updates existing Entra ID users when their information changes in ADP Workforce Now.
  • Deactivates Entra ID users when they are no longer active in ADP Workforce Now.
  • Manages user group assignments in Entra ID.
  • Supports integration hooks for user creation, updates, and deprovisioning events.
  • Sends email notifications for user creation and deactivation events when enabled.
  • Generates and sends reports via email.
  • Supports a report-only mode for validation without changing Entra ID.
  • Supports monitored provisioning.

Integration Setup

This section details the configuration and mapping process for integrating ADP Workforce Now with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped from ADP Workforce Now to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / Entra ID in the search box next to the Explore Templates section, and select it. The Create New Integration pop-up window appears.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The Configuration tab includes a list of configurations required to set up the integration.

Under the Source Match Expression section, you must specify the source attribute and the target attribute that must be used to match employee records from ADP Workforce Now with the user records in Entra ID.

Refer to the Integration Configurations Reference Guide for detailed information on each configuration field.

Mapping and Assignments

The Mapping and Assignments tab provides configuration options for defining user naming attributes and mapping attributes between ADP Workforce Now and Entra ID.

User Naming Attributes

In Entra ID, the user naming attributes are User Principal Name (UPN) and Work Mail. These attributes are used to identify user objects in Entra ID.

  • UPN: The User Principal Name (UPN) attribute is the user logon name (ULN) that is used to log in to a domain. By convention, UPN should map to the user's email address.
    A UPN consists of a prefix (the user account name) and a suffix (a DNS domain name), separated by the @ symbol. The UPN format is username@DomainName. For example, barbarajensen@aquera.com.
  • Mail: The Mail attribute is the user's email address, often set to match the UPN. The mail should be unique within the same domain. The mail attribute typically follows standard email formatting. For example, barbarajensen@example.com.

This integration enables you to generate unique names for these attributes while creating a user in Entra ID. The following configurations are available on this page:

  • Suffix
  • User Naming Configuration

Suffix

This section enables you to configure the domain name for UPN and work mail as follows:

  • UPN/ULN suffix: Enter the domain name to be used as the suffix in the UPN/ULN. You can configure a static domain name (for example, aquera.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional UPN/ULN Suffix checkbox and set conditional logic in the Conditional UPN/ULN suffix field to assign domain names to users accordingly. 

    The Conditional UPN/ ULN suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate domain names during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

  • Work Mail suffix: Enter the domain name to be used as the suffix for the work email address. You can configure a static domain name (for example, aquera1.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional Work Mail Suffix checkbox and set the conditional logic in the Conditional Work Mail suffix field.


    The Conditional Work Mail suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate email domain names during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    If the work email suffix has to match the UPN suffix, select the Work Mail Suffix same as UPN/ULN Suffix checkbox.

User Naming Configuration

This section allows you to define the formats to be used to generate values for the UPN and Mail attributes of newly created users in Entra ID.

The following configuration modes are available:

  • Basic User Naming Configuration: This configuration is enabled by default. In this mode, select a naming format from the ULN/ Mail Format dropdown list that must be used for generating the UPN and Work Mail values for a new user in Entra ID. The selected format will be used to generate unique names for both attributes.

    In this mode, the UPN and Work Mail attributes are mutually dependent and are evaluated together. This means the integration will verify whether the values generated using the selected format are available for both attributes before assigning them to the new user. If a value is already in use for an attribute (that means, it is already assigned to a user in Entra ID), then the integration will append a number (starting from 1) to the attribute values and check for the availability. This process continues for up to 20 attempts or until unique values are generated for both attributes. Once unique values are found for the attributes, those values will be assigned to the new user.

  • Advanced User Naming Configuration: Select the Advanced User Naming Configuration checkbox to switch to the advanced setup mode. In this mode, you can set different formats for each of the user naming attributes.

    When you switch to the advanced mode, you can select whether the work mail should be mutually dependent with the UPN/ULN in the Mutually Dependant Attributes with UPN/ ULN field.

    If you set UPN and Work Mail as mutually dependent attributes, the integration will check for a unique value in the first format for both attributes. Both attributes are evaluated together. If a unique value is unavailable (already assigned to another user), it will proceed to the next format in sequence for both attributes. The process continues until it finds available values for both attributes to assign to a user. If the integration cannot find unique values, the integration will return an error.
    If UPN and Work Mail are not set as mutually dependent attributes, the integration evaluates each attribute separately. If the integration finds a unique value for even one of the attributes as defined by a format, that value will be assigned to the user.

    Define the user naming formats in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields.
    These fields are AI Prompt fields that use Aquera Intelligence (AI Assistant) to generate UPN and work mail values during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    When you add a format in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields, you must also specify when the specified format should be applied. You can choose to apply it On Create, On Update, or On Create & Update.

Attribute Map

Navigate to the Attribute Map section to perform attribute mapping. The Attribute Map section displays the default mapping from Dayforce Employee attributes to Entra ID user attributes.

Note: For details on the Dayforce attributes, refer to the Appendix.

Perform the following steps to add a new mapping from a Dayforce attribute to an Entra ID attribute:

  1. Click +Add Row at the end of the attribute mapping list.
  2. Click on the empty field on the source attribute side to include a Dayforce attribute.
  3. Select an attribute from the dropdown list to add it as the source attribute.
    Alternatively, you can write a JavaScript expression in the source field. Click the Enter Expression option to write the expression. The expression is evaluated at runtime and the resulting value is assigned to the corresponding target attribute.
    Note: Click the icon to expand the Expression editor area to write longer expressions.
  4. In the Attribute field on the target side, select the Entra ID attribute name from the drop-down list.
  5. Select the data type of the Entra ID attribute from the dropdown list.
  6. Select one of the following mapping options:
    • Map on Create and Update: The Entra ID attribute will be updated during the create and update operations.
    • Map on Create: The Entra ID attribute will be updated only during the create operation.
    • Map on Update: The Entra ID attribute will be updated only during the update operation.
    • Don't Map: The Entra ID attribute will never be updated and the mapping will be ignored during the integration runs.
  7. Once done, click the check mark to add the new mapping to the attribute map.

Attribute Map Analysis

The Attribute Map Analysis evaluates and compares user attributes across the source and target applications to identify suitable attribute mappings for the integration. It highlights Matched and Unmatched users to guide accurate data correlation.

  1. Click Run Analysis. The following pop-up window is displayed:
  2. Select the source attribute and the target attribute that must be used for correlation. The values in these attributes must be unique for a user and must be the same across the source and target applications for identifying the matching users.
  3. Click Run analysis to run the Attribute Map analysis. The attribute map analysis will analyze all existing attributes between both applications for the eligible users and will generate a report.
  4. After the analysis has been completed, click View Report to view the Attribute Mapper Report.
    You can change the map options and the source attributes, if required.
  5. Click Export Mapped at the top right corner of the page to export and use this Attribute Map in the integration. Only the attributes that are mapped in the report are exported to the attribute map.

For details on the Attribute Map Report and the actions that can be performed, refer to the Integration Analysis Guide.

Preview Employee Data

You can preview the attribute values of a Dayforce employee that will be created or updated in Entra ID based on your mappings.

To preview the attribute values of a Dayforce employee, perform the following steps:

  1. Click the Enter objects for preview field in the top-right corner of the Attribute Map page. A pop-up window is displayed.
  2. Click the Searchable Attribute dropdown menu and select id.
  3. Enter the employee number (ID of the employee) in the Enter value field, and then click Preview.

    The employee data will be displayed corresponding to each mapping provided on the attribute mapper.

    Notes:
    • The toggle button next to the preview setup field will be enabled by default. You can disable the toggle option to hide the preview.
    • When the attributes or expressions provided in the source attributes column are invalid or do not return a value, the preview will not appear.
