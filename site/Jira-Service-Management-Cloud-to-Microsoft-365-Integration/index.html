Jira Service Management Cloud to Microsoft 365 Integration

Overview

The Aquera platform allows your organization to scale complex data management by automating data synchronization between various applications.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The Jira Service Management Cloud to Microsoft Office 365 Users integration automates the process of onboarding users from the Jira Service Management Cloud application to the Microsoft Office 365 application. This integration is triggered when a user in your Jira Service Management Cloud application is created, updated or deleted.

This integration performs the following tasks:

  • When a user is created or updated in Jira Service Management Cloud, the Onboarding Preference ticket is generated. During integration execution, user details are retrieved and sent to Microsoft 365 for user account creation or updation.
  • "When a user is deleted in Jira Service Management Cloud, an Off-boarding Preference ticket is generated. During the integration execution, the user's details are retrieved and sent to Microsoft 365 to initiate the deletion of the corresponding user account.

This guide details how to set up an integration that synchronizes users from Jira Service Management Cloud to users in Microsoft Entra ID.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Connectors in the Aquera Admin Portal: You need to configure the following connectors in the Aquera Admin portal for the integration:
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.
    • Microsoft Office 365: Refer to the Microsoft 365 Configuration Guide for details.

Integration Setup

Aquera integration automatically imports workers from Jira Service Management Cloud and provisions these users in Microsoft Entra ID.

Create an Integration

Perform the following steps to create an integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Jira Service Management Cloud To Microsoft Entra ID.
  5. Enter a name and a description for the integration.
  6. Select your Jira Service Management Cloud and Microsoft Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Preferences

The Preferences section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Report Only: Select to create a report outlining the data changes that would occur without implementing them.
  • License: Enter the license number.
  • Admin Name: Enter the administrator name that must be included in the report.
  • Admin Email Address: The designated email address that receives all system notifications, alerts, and reports related to the integration.
  • Onboarding Preference: Enter the Onboarding preference from which the deleted user must be retrieved.
  • Full Sync Max Time Limit: Enter the maximum time limit in which the full sync should happen.

  • Off-boarding Preference: Enter the Off-boarding preference.
  • User Location: Enter the user's location.

Run Integration

You can set event or schedule triggers to run the script synchronization. For more details, refer to the Executing a Synchronization section.

After the script synchronization is successfully executed, Aquera syncs the users created in Jira Service Management Cloud as internal contacts in Microsoft Entra ID. The imported users are staged and need admin approval to get provisioned to Microsoft Entra ID.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
