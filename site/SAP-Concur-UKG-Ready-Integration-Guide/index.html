Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employee data from UKG Ready to SAP Concur.

Key Functions

This integration automates identity lifecycle management between UKG Ready (an HR/Payroll system) and SAP Concur (an expense management system) with the following primary capabilities:

  • Creates new users in SAP Concur when they are added in UKG Ready.
  • Updates existing SAP Concur users when their information changes in UKG Ready.
  • Deactivates SAP Concur users when employees are terminated in UKG Ready.
  • Creates and deletes users based on a specific date and time.
  • Supports incremental synchronization to process only changed records.
  • Sends email notifications for various events, such as user creations, updates, and terminations.

Integration Setup

This section details the configuration and mapping process for integrating UKG Ready with SAP Concur. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for SAP Concur / UKG Ready in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • UKG Ready
  • SAP Concur
  • Policy
  • Attribute Map
  • Execution
  • Notification
  • Schedule

Overview

On this tab, you can view integration details and the list of features supported by the integration. Scroll down to the Features List section to enable or disable integration features based on your use case.

Feature List

This section lists the features supported by the integration. You can enable or disable each feature based on your requirements.

The following are the supported features:

  • Enable SAP Concur for Employee Management / Enable employee management: Turn on this toggle button if your use case requires synchronizing employee data from UKG Ready to SAP Concur. When enabled, the integration provisions new user accounts, updates user attributes based on changes, and deprovisions accounts according to employee lifecycle events and data changes, in SAP Concur.
  • Enable Monitored Provisioning / Enable monitoring: Turn on this toggle button if you want to review the changes to be performed by the integration before they are applied to SAP Concur. When enabled, the system evaluates integration executions against the defined monitoring rules and conditions, and then creates tasks for review.
  • Enable Time Based Onboarding/Offboarding / Enable time-based onboarding/offboarding: Turn on this toggle button to allow onboarding and offboarding users based on configured date, time, and timezone.

  • Enable Notification Management / Enable notification management: Turn on this toggle button to allow email notifications to be sent for the integration runs.

Application Settings

This section allows you to identify the UKG Ready application connector for the integration, and define onboarding and offboarding criteria.

The following settings are displayed on this page for configuration:

  • Select your UKG Ready Application Instance / Select the UKG Ready Application: Select the UKG Ready application connector that you have configured in the Aquera Admin portal for the integration.
  • Company Name: Enter the short name of the company configured in your UKG Ready instance. Refer to the UKG Ready configuration guide for details on obtaining the company name.
  • Override Default Employee Filter / Filter employees for creation: By default, all active employees in UKG Ready are considered for account creation in SAP Concur. Enable this toggle to filter the employees that must be created.
    When enabled, the following fields are available for configuration:
    • Trigger Employee Creation / Filter by: Select how employee accounts from UKG Ready must be filtered for creation in SAP Concur. The following is the available option for selection:
      • Based on the Status / Status: Select this option to filter the employees based on their status in UKG Ready for creation in SAP Concur. When selected, the integration uses the configured statuses to determine the employees that are eligible for creation.
    • Select Statuses for Employee Creation / Select statuses: Select one or more employee statuses that must be considered active. Employees with these statuses in UKG Ready are considered for creation in SAP Concur.
  • Override Default Termination Handling / Filter employees for termination: By default, all deactivated employees in UKG Ready are considered for termination in SAP Concur. Enable this toggle to filter the employees that must be terminated.
    When enabled, the following fields are available for configuration:
    • Trigger Employee Termination / Filter by: Select how employee accounts from UKG Ready must be filtered for termination in SAP Concur. The following is the available option for selection:
      • Based on the Status / Status: Select this option to filter the employees based on their status in UKG Ready for termination in SAP Concur. When selected, the integration uses the configured statuses to determine the employees that are eligible for termination.
    • Select Statuses for Employee Termination / Select statuses: Select one or more employee statuses that must be considered inactive. Employees with these statuses in UKG Ready are considered for termination in SAP Concur.

Time-Based Settings

This section allows you to define the timezone and schedule for onboarding and offboarding processes.

The following fields are displayed on this page:

  • Method Used to Identify The Applicable Time Zone / Select a method to identify the applicable timezone: Select a method to identify the applicable timezone. Currently, only the following option is supported:
    • Manual timezone selection: Select this option to create or terminate employees using Coordinated Universal Time (UTC), regardless of the user's local time. When selected, the following fields are displayed for further configuration:
      • Select desired timezone: Select the timezone that must be considered for creating or terminating users in SAP Concur. 
      • Onboarding Time Of Day / Onboarding time: Enter the time that must be considered for onboarding users in SAP Concur. The time has to be provided in the 24-hour format (HH:MM). For example, 03:40
      • Offboarding Time Of Day / Offboarding time:  Enter the time that must be considered for offboarding users in SAP Concur. The time has to be provided in the 24-hour format (HH:MM). For example, 03:40

This section includes the following configuration sections:

  • Target Application Settings

Target Application Settings

The Target Application Settings section allows you to identify the SAP Concur application connector for employee management and define the required settings.

The following fields are displayed on this page:

  • SAP Concur Application / Select the SAP Concur application: Select the SAP Concur application connector that you have configured in the Aquera Admin portal for the integration.
  • Enable Full Sync: Turn on this toggle button to run the integration in full synchronization mode. When enabled, all employees from UKG Ready will be synced to SAP Concur on every integration run. When disabled, only the employee changes in UKG Ready since the last integration run are synced to SAP Concur.

Manage Employees

The Manage Employees section enables you to customize the integration settings for employee lifecycle management according to your specific use case. Based on the defined settings, the integration synchronizes employee data from UKG Ready to SAP Concur.

The following fields are displayed on this page:

Employee Match Settings

  • Customize Lookup for User Correlation / Customize how users are matched between UKG Ready and SAP Concur: By default, employee records are matched using the employee ID in UKG Ready and the employee number in SAP Concur. Turn on this toggle button to use different source and target attributes for employee matching.
    When enabled, the following field is displayed for further configuration:
    • Correlation Between Source And Target Application / Select a user matching condition: 
      Select one of the following conditions to determine how users are matched between UKG Ready and SAP Concur.
      • Match employees using UKG Ready EmployeeId and SAP Concur Employee Number: Select this option to use the employee ID in UKG Ready and the employee number in SAP Concur to find the matching users.
      • Match employees using custom attributes: Select this option to match users based on a different set of attributes in UKG Ready and SAP Concur. When selected, the following fields are available for further configuration:
        • Correlation Attribute from Source Application / UKG Ready attribute for matching users: Select an attribute that uniquely identifies a user record in UKG Ready. For example, select employeeId if you want to use it to match user records.
        • Correlation Attributes from Target Application / SAP Concur attribute for matching users: Select an attribute in your SAP Concur application that must be used to match the selected UKG Ready attribute value to identify matching users. For example, select employeeNumber if you want to use it to match user records.

Safeguard Settings

  • Enable Safeguard Limit / Enable safeguard limits: Turn on this toggle button to define limits to restrict the number of provisioning operations performed by the integration. When enabled, the following fields are available for further configuration:
    • Safeguard Limit Type / Safeguard limit type: Select whether the safeguard limits are defined as a number or a percentage. Currently, only a number value is supported.
    • Apply Safeguard As / Apply safeguard limits to: Specify whether a single safeguard limit has to be applied across all operations or separate limits need to be configured for each operation type.
      • Common for All / All operations: Select this option to apply the safeguard limit across all operations. When selected, the following field is displayed for the further configuration:
        • Safeguard Limit for All Operations / Safeguard limit for all operations: Enter a value to define the safeguard limit for all operations. After the limit is reached, no create, update, or terminate operations will be executed.
      • Per Operation Type / Each operation type: Select this option to define and apply different safeguard limits for each operation type. When selected, the following fields are displayed for further configuration:
        • Enable Create Limit / Enable creation limit: Enable this toggle button to apply a safeguard limit for user creations.
          •  Set the Creation Limit / Create limit: Enter the maximum number of creations that are allowed to be performed in a single integration run. After the limit is reached, create operations will no longer be executed.
        • Enable Update Limit / Enable update limit: Enable this toggle button to apply a safeguard limit for user updates.
          •  Update Limit / Update limit: Enter the maximum number of updates that are allowed to be performed in a single integration run. After the limit is reached, update operations will no longer be executed.
        • Enable Termination Limit / Enable termination limit: Enable this toggle button to apply a safeguard limit for user terminations.
          • Termination Limit / Termination limit: Enter the maximum number of terminations that are allowed to be performed in a single integration run. After the limit is reached, termination operations will no longer be executed.

Employee Termination Settings

The Employee Termination Settings section details the settings specific to user terminations in SAP Concur.

  • Who should be monitored: Select whether you want to monitor all the users or only the users filtered based on a condition.

Rule Detail (Monitoring)

This section enables you to define rules for monitored provisioning. By default, the Assignment Monitoring Rules are available. (Can the following be deleted?>)

Perform the following steps to define the rules:

  1. Click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the assignment rule.
  2. Search and select the attribute you want to use to define a condition. For example, select Primary Work Assignment Departments.
  3. In the below row, double-click and enter the attribute value to define the rule. If multiple values are selected, OR condition is applied among them. For example, in the following sample condition, a user's primary work assignment departments must be either Administrator or Facility Administrator to satisfy this assignment rule.
  4. If required to add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  5. If required to include another attribute to the monitoring rule, click the vertical ellipsis in the first column, and then click Add Attribute.
    If multiple columns are added for a rule, AND condition is applied between different columns. For example, in the following sample condition, a user's primary work assignment departments must be Administrator or FacilityAdministrator, and the active employment status must be active to satisfy this assignment rule. You can use multiple attributes and their corresponding values to define a group assignment rule.
  6. Click the vertical ellipsis next to Choose Target, and then click Select Target Attribute to view the list of targets you can use to set the monitoring rule. For example, select Directory.
  7. In the below row, double-click and select the target applications to define the rule. If multiple values are selected, OR condition is applied among them.
  8. If needed, click the + icon at the top right corner to add another rule, as explained above.
  9. Click Save Changes at the top-right corner of the page to save the configuration.

Note: If required to delete a row, click the horizontal ellipsis next to the row and select Delete Row.

 

Attribute Map

The Attribute Map tab defines how the data is mapped between the UKG Ready abnd SAP Concur aplications.

The User Lifecycle Managament section defines how employee data is mapped from UKG Ready to users in SAP Concur. The default mapping of attributes is displayed on the page. The UKG Ready attributes are listed on the left side and the SAP Concur attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
