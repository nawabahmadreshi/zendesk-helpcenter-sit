Dynamic365-ERP / UKG Ready Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any employees or system users created, updated, or terminated in UKG Ready are automatically updated in Microsoft Dynamics 365 Finance and Operations based on the defined schedule.

To identify user records across both systems from UKG Ready to Microsoft Dynamics 365, the integration uses the attribute configured in the UKG Ready Correlation Attribute parameter in UKG Ready and matches it to the attribute configured in the Microsoft Dynamics 365 Finance and Operations Correlation Attribute parameter in Microsoft Dynamics 365 as a correlation attribute.

Key Functions

The Dynamics 365 / UKG Ready Integration performs the following functions:

  • Synchronizes employees or system user records from UKG Ready to Microsoft Dynamics 365 Finance and Operations.
  • Creates new employees and system users in Microsoft Dynamics 365 Finance and Operations when they are added in UKG Ready.
  • Updates existing employees and system user records in Microsoft Dynamics 365 Finance and Operations when the corresponding user information changes in UKG Ready.
  • Terminates the employees and system user records in Microsoft Dynamics 365 Finance and Operations when the corresponding user records are deleted from the UKG Ready.
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • UKG Ready: Refer to the UKG Ready Configuration Guide for configuration details.
    • Microsoft Dynamics 365 Finance and Operations: Refer to the Microsoft Dynamics 365 Finance and Operations Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from UKG Ready to Microsoft Dynamics 365 Finance and Operations.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dynamic365-ERP / UKG Ready in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select your Microsoft Dynamics 365 Finance and Operations and UKG Ready connector instances.
  8. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Manage Employees: Enable this toggle button to synchronize employee records from UKG Ready to Microsoft Dynamics 365 Finance and Operations. When enabled, the Manage Employees section is displayed for further configuration.
  • Manage System Users: Enable this toggle button to synchronize the system use records from UKG Ready to Microsoft Dynamics 365 Finance and Operations. When enabled, the Manage System Users section is displayed for further configuration.

Execution Features

Configure the following settings:

  • Enable Report-Only Mode: Enable this toggle button to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Incremental Mode: Enable this toggle button to run the integration in incremental mode. If this toggle is disabled, the integration will run in full sync mode. By default, the first integration execution always runs in full sync mode.
  • Enable Logging: Enable this toggle button to generate logs outlining data changes. When enabled, the Logging Settings section is displayed for further configuration.
  • Enable Notifications: Enable this toggle button to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Enable this toggle button to generate the integration report. When enabled, the Reporting Settings section is displayed for further configuration.
  • Enable Log Insights: Enable this toggle button to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Debugging: Enable this toggle button to generate logs outlining data changes that would occur for debugging.
  • Enable Advanced Debugging: Enable this toggle button to generate detailed execution logs, including request and response payloads, mappings, and internal processing steps for advanced troubleshooting.

Manage Employees

This section enables you to customize the integration settings to manage employee records for your specific use cases.

Correlation Settings

Configure the following settings:

Note: All parameters marked with a red asterisk (*) are mandatory.

  •  Source Correlation Attribute*: Select the UKG Ready attribute to distinctly find the associated Microsoft Dynamics 365 Finance and Operations employee record. For example, EmployeeId.
  • Target Correlation Attribute*: Select the Microsoft Dynamics 365 Finance and Operations attribute to distinctly find the associated UKG Ready employee record. For example, PersonnelNumber.

Safeguard Limits

Configure the following settings to limit the employee records provisioned from UKG Ready to Microsoft Dynamics 365 Finance and Operations:

  • Limit type: Select whether the safeguard limits are defined as a number or a percentage. Currently, only a number value is supported.
  • Enable Import Limit: Enable this toggle button to set a limit on the number of employee records that can be imported from UKG Ready into Microsoft Dynamics 365 Finance and Operations. If disabled, all the employee records will be imported. When enabled, the following parameter is displayed:
    • Import Limit: Enter the maximum number of employee records that can be imported from UKG Ready to Microsoft Dynamics 365 Finance and Operations in a single integration run. For example, entering 7 allows only 7 employee records to be imported in a single integration run.
  • Limit Scope: Select a scope from the dropdown menu to specify whether a single safeguard limit has to be applied across all operations or separate limits need to be configured for each operation type. The available operations are All operations and Per operation.
    • All operations: Select this option to apply the safeguard limit across all operations. When selected, the following field is displayed for further configuration:
      • Limit value (all operations): Enter a value to define the safeguard limit for all operations. After the limit is reached, no create, update, or deactivation operations will be executed.
    • Per operation: Select this option to define and apply different safeguard limits for each operation type. When selected, the following parameters are displayed for further configuration:
      • Create limit: Enter the maximum number of creations that are allowed to be performed in a single integration run. After the limit is reached, create operations will no longer be executed.
      • Update limit: Enter the maximum number of updates that are allowed to be performed in a single integration run. After the limit is reached, update operations will no longer be executed.
      • Deactivate limit: Enter the maximum number of deactivations that are allowed to be performed in a single integration run. After the limit is reached, deactivation operations will no longer be executed.

      • Custom Create Condition: Enter a custom expression to define which employee records should be created in Microsoft Dynamics 365 Finance and Operations.

Log Settings

The Log Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if you have enabled the Enable logging parameter in the Execution Features section.

  • Select Log Type: Select the type of data that should appear in the logs. The following are the available options:
    • Minimum: Logs only the operations performed during the integration run. Operations are recorded without GUIDs or field details. For example, the log may display User Created, User Updated, or User Deleted without listing any attributes or identifiers.
    • Metadata: Logs the operations performed along with the names of the fields that were changed, but does not include the GUID. For example, the log may show Updated fields: Department and Title without displaying the associated user identifier.
    • Non-PII Data: Logs the operations performed, field names, and selected non-PII values. The GUID is not included. For example, the log may display Department updated to Sales while masking sensitive information such as date of birth.
    • GUID with Non-PII Data: Logs the GUID together with the operations performed, field names, and selected non-PII values. This option provides detailed traceability while still preventing exposure of sensitive information.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data. 
  • Select Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full details: All attribute details are displayed.
    • Inclusion list: Select this option to log only the attributes specified in the Custom inclusion list.
      • Custom inclusion list: Select the attributes to be included in the log.
    • Exclusion list: Select this option to log only the attributes specified in the Custom exclusion list.
      • Custom exclusion list: Select the attributes to be excluded from the log.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
