User Integration Setup Guide - UKG Ready to Okta

← Back to index

Overview

This guide covers the end to end tasks involved in setting up an orchestration to sync users in UKG Ready with Okta.

The following tasks are involved in setting up the orchestration from UKG Ready to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Create Orchestration
  • Orchestration Setup
  • Running Orchestration

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following application must be added and configured in the Aquera Admin console:

  • UKG Ready. Refer to the UKG Ready Configuration Guide for details.

Create Orchestration

Perform the following steps to create a orchestration from UKG Ready to the Okta application:

  1. Login into the Aquera Admin console.
  2. Click Add Orchestration. The Add Orchestration pop-up menu is displayed.
  3. Enter a name for Orchestration, select Orchestration Type (Synchronization V2), and click Done. 
  4. Select UKG Ready as the Source Application and Okta as the Target Application.
  5. Click Create.

Attribute Mapping

Fill out the mapping template attached with this guide for mapping UKG ready user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from UKG Ready to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Consideration

  • The criteria for matching a user record from UKG Ready with that in the Okta.

For detail analyses of UKG Ready users and the users/groups in Okta using scripts, refer to the Appendix.

Building Orchestration

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in restriction list for scoping of the groups which would be managed by Aquera.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Running Orchestration

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if correct data is updated in the Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data, for example, employeeNumber, in the Okta.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in UKG Ready.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow orchestration to update users, you need to disable the setting which sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting which sets Active Directory as profile master.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in UKG Ready and Okta before setting up the orchestration.

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add UKG Ready and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

Few sample scripts are provided here for your reference. The scripts may vary based on your use case  and the attributes to be retrieved for analysis.

Analyzing the Users in UKG Ready

Use the below sample script in a scheduled script orchestration for validation of UKG Ready User Data Analysis.

UKG ID,UKG USERNAME,UKG FAMILYNAME,UKG GIVENNAME,UKG EMAILS_WORK_VALUE,UKG EMAILS_WORK_PRIMARY,UKG PHONENUMBER_WORK_VALUE,UKG PHONENUMBER_MOBILE_VALUE,UKG ACTIVE,UKG EMPLOYEE_NUMBER, UKG DIVISION, UKG DEPARTMENT,UKG ENT_MANAGER_VALUE,UKG ENT_MANAGER_$REF,UKG ENT_MANAGER_DISPLAYNAME');

 

 let users = ukg.importAllUsers();

 

 let i = 0;

 

 for (i = 0; i < users.length; i++) {

 

    let UKG_Ready_id = users[i].id;

 

    let UKG_Ready_username = users[i].userName;

 

    let UKG_Ready_familyname = users[i].name.familyName;

 

    let UKG_Ready_givenname = users[i].name.givenName;

 

    let UKG_Ready_emailwork = (users[i].emails?users[i].emails.value:'');

 

    let UKG_Ready_emailprimary = (users[i].emails?users[i].emails.primary:'');

    

    let UKG_Ready_phonenumberwork = (users[i].phoneNumbers?users[i].phoneNumbers.value:'')

    

    let UKG_Ready_phonenumbermobile = (users[i].phoneNumbers?users[i].phoneNumbers.type:'');

    

    let UKG_Ready_status = users[i].active;

    

    let UKG_Ready_employeenumber = (users[i][ENT] && users[i][ENT].employeeNumber || '')

     

    let UKG_Ready_division = users[i][ENT].division;

    

    let UKG_Ready_department = users[i][ENT].department;

    

    let UKG_Ready_manager = (users[i].manager?users[i].manager.value:'');

      

    let UKG_Ready_managerref = (users[i].manager?users[i].manager.$ref:'');

    

    let UKG_Ready_managerdisplayname = (users[i].manager?users[i].manager.displayName:'');

  

    log.info('"' 

 

    + UKG_Ready_id + '","' 

 

    + UKG_Ready_username + '","'

 

    + UKG_Ready_familyname + '","' 

 

    + UKG_Ready_givenname + '","'

 

    + UKG_Ready_emailwork + '","'

 

    + UKG_Ready_emailprimary + '","'

 

    + UKG_Ready_phonenumberwork + '","'

 

    + UKG_Ready_phonenumbermobile + '","' 

    

    + UKG_Ready_status + '","'

 

    + UKG_Ready_employeenumber + '","'

 

    + UKG_Ready_division + '","'

 

    + UKG_Ready_department + '","'

 

    + UKG_Ready_manager + '","'

 

    + UKG_Ready_managerref + '","'

 

    + UKG_Ready_managerdisplayname + '"');

 

}

Analyzing the Users in Okta

Use the below sample script in a scheduled script orchestration for validation of Okta User Data Analysis.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');


for (let user of Okta.importAllUsers()) {

    log.info('"'

    + user.id + '","'

    + user.active + '","'

    + (user[ENT] && user[ENT].employeeNumber || '') + '","'

    + user.name.givenName + '","'

    + user.name.familyName + '","'

    + user.userName + '","'

    + user.name.middleName + '","' 

    + user.name.nickName + '","'

    + user.title + '","'

    + user.department + '","'

    + user.division + '","'

    + user.userType +  '"');

}
