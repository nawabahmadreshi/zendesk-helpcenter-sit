Active Directory / RingCentral Integration Guide

← Back to index

Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating data synchronization from Active Directory users to RingCentral directory.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, every user in Active Directory will be created, updated, or deprovisioned with an extension phone number in RingCentral according to the schedule.

This guide covers details on how to set up the Active Directory/RingCentral integration to transfer user data from Active Directory to assign an extension in the RingCentral directory.

Key Features

This integration automates phone number assignments with the following primary capabilities:

  • Synchronizes user data from Active Directory to RingCentral.
  • All the newly created users will be assigned a phone number in RingCentral.
  • Limits the users create, update, and termination operations.
  • Sends notifications upon success, failure, or skipped record events via email.
  • Generates detailed logs and reports.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Applications in the Aquera Admin Portal: 
    • Active Directory Connector: Ensure you add and configure Active Directory in the Aquera Admin portal. For details, refer to the Active Directory Configuration Guide.
    • RingCentral Connector: Ensure you add and configure the RingCentral connector in the Aquera Admin portal. Refer to the RingCentral Configuration Guide for details.

Integration Setup

Aquera's Active Directory / RingCentral integration automatically synchronizes users data from Active Directory to RingCentral based on a configured schedule.

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Active Directory / RingCentral. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Select Active Directory connector and RingCentral connector from the corresponding drop-down lists.
  7. Click Continue. The integration is created.

Once created, the following tabs are displayed:

  • Overview
  • Configuration
  • Active Directory
  • RingCentral
  • Attribute Map
  • Notification
  • Execution
  • Schedule

Overview

In this tab, you can view the integration details and a list of integration settings to be enabled based on the use cases.

The following are the supported integration settings:

  • Enable User Creation: Select to create a user in RingCentral directory from Active Directory.
  • Enable User Updation: Select to update the existing users data in RingCentral from Active Directory. 
  • Enable user attributes writeback to Active Directory from RingCentral: Select to writeback all the newly created and updated users data from RingCentral to Active Directory.

Configuration

The following sections are displayed under the Configuration tab:

The Applications section displays the source and target application connectors for the integration. This section allows you to select the application connectors.

Active Directory

Navigate to the Active Directory tab to configure the Active Directory application connector used for the integration.

  • Enable Active Directory OU filter to fetch Users: When enabled, the fetch Users only from specific OU's field will appear for configuration.
    • fetch Users only from specific OUs: Select the required OU filter from the dropdown list to get the designated data from Active Directory. It only allows the users belonging to the specified Organizational Units (OUs) to be fetched from Active Directory for syncing rather than pulling all users across the entire directory.
    • Enable user filter for complex conditions: When enabled, the following field appears.
      • Select the required filter type from the dropdown. This field includes the following configuration settings:
        • AI Prompt: Use this if you want to define filter conditions using Aquera Intelligence (AI Assistant). When enabled, Provide prompt to filter AD users field will appear where you can provide prompts to generate JS expressions using AI Assistant.
        • JS Expression: Use this if you want to define a custom JavaScript expression to filter users dynamically. When enabled, JavaScript logical expression filter for Users in AD field appears, in which you can provide the expression.
        • Attribute Filter: Use this if you want to filter users records by providing the required attribute and value. When enabled, the following fields will appear:
          • Select attributes to filter: Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
          • Provide values to filter based on the above provided attribute key: Enter the corresponding value of the attribute you want to filter.

RingCentral Settings: Select the type of valid number list in which the user data will be provisioned.

  • Enable user filter for complex conditions: When enabled, the following field appears. Select the required filter type from the dropdown. This field includes the following configuration settings:

    • JS Expression: Use this if you want to define a custom JavaScript expression to filter users dynamically. When enabled, JavaScript logical expression filter for Users in AD field appears, in which you can provide the expression.
    • Attribute Filter: Use this if you want to filter users records by providing the required attribute and value. When enabled, the following fields will appear:
      • Select attributes to filter: Select the required attribute you want to filter from the dropdown. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
      • Provide values to filter based on the above provided attribute key: Enter the corresponding value of the attribute you want to filter.

Correlation Settings

  • This field identifies the correlation attributes to match records in Active Directory and RingCentral. The following are the configuration settings:
    • Select correlation key in Active Directory: Select the key to uniquely identify a user in Active Directory inorder to match with RingCentral user. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.
    • Select correlation key in RingCentral: Select the key to uniquely identify a user in RingCentral inorder to match with Active directory user. If you do not find the required filter in the list, click the refresh icon available next to the field to reload the available attributes.

RingCentral Termination Settings

The following are the supported integration settings:

  • Enable User Deactivation: Enabling this will deactivate the users assigned extension number without removing the users data in RingCentral. When enabled, the following toggle field will appear.
    • Delete Users after Deactivation: Enabling this will delete the users record from RingCentral directory once the user is deactivated.
  • Enable termination in future: This feature automatically terminates RingCentral user accounts after a specified number of days from the date the user is marked as terminated in Active Directory (AD). When enabled, additional fields will appear to select the required attribute and to set the termination period.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Notification

Enable Notifications: This tab allows you to configure email notifications for integration events, such as successful, failed execution, or skipped records and to define the recipients and report details included in the notification emails. When enabled, the Notification Template Settings will appear.

Notification Template Settings

Configure the following settings to define when notification emails are sent.

  • Send email notification for success: Enable this to send an email when the integration completes successfully. For example, when the integration finishes without errors, a success notification is sent to the configured recipients. When enabled, Success Execution Settings will be displayed.
  • Send email notification for failure: Select to send an email when the integration fails. For example, if the integration encounters an error, a failure notification is sent to the configured recipients.
  • Send email notification for skipped records: Select to send an email when the integration skips executing users records. For example, if the integration encounters an error, a failure notification is sent to the configured recipients.
  • Attach report to email: Select to include the integration report as an attachment in notification emails. For example, enabling this option attaches the execution summary to the notification email.

Success Execution Settings

Configure the following settings to define the recipients and content of success notification emails.

  • Send to: Enter the email addresses that should receive success notification emails. For example, add admin@company.com to receive confirmation when the integration completes successfully.
  • Success notification CC: Enter additional email addresses to receive a copy of success notifications. For example, include teamlead@company.com to keep the team informed.
  • Success Email subject: Enter the subject line used for success notification emails. For example, Integration - Success Report.
  • Success email body: Enter the message content sent when the integration completes successfully. For example, include a summary of the run, such as the number of users processed and any key updates.

Failure Notification Settings

Configure the following settings to define the recipients and content of failure notification emails.

  • Send to: Enter the email addresses that should receive failure notification emails. For example, add support@company.com to receive alerts when the integration encounters an error.
  • Failure notification CC: Enter additional email addresses to receive a copy of failure notifications. For example, include itadmin@company.com to notify system administrators.
  • Failure email subject: Enter the subject line used for failure notification emails. For example, Failed Report [Action Required].
  • Failure email body: Enter the message content sent when the integration fails. For example, include error details and troubleshooting guidance.

 Skipped Records Notification Settings

Configure the following settings to define the recipients and content of skipped records notification emails.

  • Send to: Enter the email addresses that should receive skipped records notification emails. For example, add support@company.com to receive alerts when the integration encounters a user record skip.
  • Skipped Records notification CC: Enter additional email addresses to receive a copy of Skipped Records notifications. For example, include itadmin@company.com to notify system administrators.
  • Skipped Records email subject: Enter the subject line used for skipped records notification emails. For example, Skip Records.
  • Skipped Records email body: Enter the message content sent when the integration skips users records while executing. For example, include error details and troubleshooting guidance.

Notification Report Settings

Configure the report information included in notification emails.

  • Select report type: Select the type of report data included in the notification email. The following are the available options:
    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Report Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.

Advanced Settings

This section enables you to customize the advanced settings for the integration. It has the following sub-sections for configuration:

  • Rollout Configuration
  • Limit Settings

Rollout Configuration

Enable Report-Only Mode: Enabling this will execute the integration without making changes by only generating the report.

Enable Debug Mode: Enabling this will generate logs outlining data changes that would occur for debugging. When enabled, the Debug Settings section will appear for further configurations.

  • Enable User Filter: Enabling this allows you to configure more with user filters in debug mode. When enabled, the following fields are displayed.
    • Select key to filter users in AD: Select one attribute from the list of attributes in which you want to filter the users from Active Directory (AD).
    • Add values to filter: Provide the value based on the above selected attribute key in which you want to filter users from Active Directory.

Limit Settings

This section allows you to configure operation limits for Create, Update, and Termination actions in the RingCentral integration. It has the following sub-sections for configuration:

  • Enable Create Limit: The RingCentral Create operations will stop automatically once the configured limit is reached. The Update and Termination operations will continue without interruption. When enabled, Provide create limit field will be displayed.
    • Provide create limit: Provide the maximum number of Create operations allowed during the integration run.
  • Enable Update Limit: The RingCentral Update operations will stop automatically once the configured limit is reached. The Create and Termination operations will continue without interruption. When enabled, Provide update limit field will be displayed.
    • Provide update limit: Provide the maximum number of Update operations allowed during the integration run.
  • Enable Termination Limit: The RingCentral Termination operations will stop automatically once the configured limit is reached. Create and Update operations will continue without interruption. When enabled, Provide termination limit field will be displayed.
    • Provide termination limit: Provide the maximum number of Termination operations allowed during the integration run.
