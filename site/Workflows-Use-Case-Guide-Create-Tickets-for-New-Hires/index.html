Workflows Use Case Guide: Create Tickets for New Hires

← Back to index

Overview

You can set up a Workflow to automatically create tickets in ITSM (IT service management) applications such as Zendesk, ServiceNow, or Freshservice when users are created in HRIS (Human Resources Information System) applications such as BambooHR or Okta. 

Use Case: For every new user creation in an HRIS application, create an IT ticket in an ITSM application, and send an email notification to the IT admin to assign a laptop for the new user.

This use case focuses on creating a User Synchronization from BambooHR to Okta for syncing user data and creating a Workflow to create Zendesk tickets for new users of Okta.

The following tasks are involved in setting up this use case: 

  • Configuring BambooHR, Okta, and Zendesk connectors
  • Creating a User Synchronization from BambooHR to Okta
  • Setting up a Workflow
  • Configuring the Synchronization hook in User Synchronization
  • Running the Synchronization 

For more details on all the capabilities of workflows, refer to the Workflows User Guide.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
  • Configure connectors in the Aquera Admin portal: You must configure BambooHR, Okta, and Zendesk connectors in the Aquera Admin portal.

Note: The connectors to be configured will vary based on your use case. Refer to the configuration guides in the Aquera Help Center for details on configuring the required connectors. 

Creating a User Synchronization

Create a User Synchronization to sync user data from a source to a target application. For this use case, consider BambooHR as the Source Application and Okta as the Target Application and create the User Synchronization. For further details, refer to the User Synchronization Guide.

Setting up a Workflow

Set up a Workflow to create a Zendesk ticket whenever a new user gets created in the target application (Okta) configured in the User Synchronization.

Creating a Workflow

Perform the following steps to create a Workflow:

  1. Log in to the Aquera Admin portal and navigate to Workflow. Click Create Workflow at the top-right corner of the screen.
  2. Click Blank to create a Workflow.
  3. Enter a name and description for the Workflow. Click + Add Application, then select Zendesk connector from the APPLICATION field and enter a variable name for the Zendesk connector in the NAME field. Click Continue. 
  4. Click Create at the top right corner of the page to create the Workflow.

Setting an Event Trigger in the Workflow

Set an event trigger in the Workflow that triggers the Workflow when the User Synchronization creates a new user in Okta.

Perform the following steps to set an event trigger:

  1. Click the Start Workflow element.
  2. Configure the following fields in the Trigger section:

    • Select Triggered By as Event. 
    • Select Source Type as Synchronization.
    • Select Source Name as the name of the Synchronization to sync users from BambooHR to Okta.
    • Select Event Name as User Created by Synchronization so the Workflow receives the event notification from the User Synchronization when a new user is created in Okta. 
    • Select Schedule Event Execution as Continuous to run the Workflow whenever the event triggers. 
  3. Click Save Changes at the top right corner of the page to save the configurations.

Configuring Elements in the Workflow

Perform the following steps to set up the Workflow to create an IT ticket in Zendesk for a new user created in Okta and send an email notification to the IT admin to assign a laptop to the new user:

  1. Drag the Zendesk connector from the Applications section and drop it on the + icon of the Workflow.
  2. Click the Zendesk connector element and select Create Ticket as the operation. 
  3. Click Add More Attributes to customize the contents of the ticket. For example, you can add the priority, status, subject, submitter_id, and comment description so that the ticket is created with the essential information.

    Note: You can add variables or attributes in the fields by typing the $ symbol and selecting an option from the dropdown list.
  4. To send an email notification to the IT admin whenever a ticket gets created, drag the Email from the Helpers section and drop it on the + icon of the Workflow.
  5. Click the Email Configuration element to configure the To, Cc, Bcc, Subject, and Body of the Email. For further details, refer to this section in the Element Configurations guide.
  6. Click Save Changes at the top right corner of the page to save the Workflow.

Configuring the Synchronization Hook in User Synchronization

Perform the following steps to configure the synchronization hook in the User Synchronization to send the event notification to the Workflow for a new user creation in the target application (Okta):

  1. Navigate to the User Synchronization created earlier to sync from BambooHR to Okta.  
  2. Navigate to the Configuration tab > Basic Details section.
  3. Select the Workflow created earlier from the dropdown list in the New User Synchronization Hook configuration.
  4. Click Save Changes at the top right corner of the page to save the configuration. 

Running the Synchronization

You can click the Execute button at the top right of the Synchronization page to manually run the Synchronization. You can also schedule the Synchronization to run automatically at specified time intervals by configuring a time-frequency in the Schedule tab. You can navigate to Logs > Synchronization in the Aquera Admin portal to view the execution logs.

For further details, refer to the Synchronizations Schedule Setup Guide.

Conclusion

After following the aforementioned tasks, when the Synchronization successfully creates new users in Okta, the Workflow automatically gets executed for each new user. The Workflow creates Zendesk tickets for new users and sends notification emails to the IT admin to assign a laptop for the new hires.
