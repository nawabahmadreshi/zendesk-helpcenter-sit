Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, when a candidate accepts an offer in Pinpoint, their information is automatically used to create an onboarding applicant record in ADP Lyric.

The integration collects important details from the candidate’s application in Pinpoint, such as personal information, job details, and offer information. These details are then organized and sent to ADP Lyric so the onboarding process can begin.

The integration can also check for specific onboarding fields that are required for creating the applicant record. If any of the required information is missing, the onboarding process for that candidate is skipped to avoid creating incomplete records.

Once all the required information is available, the candidate’s details are sent to ADP Lyric and the onboarding applicant is created automatically, helping HR teams start the onboarding process without manual data entry.

Key Functions

The Pinpoint to ADP Lyric Onboarding Integration performs the following functions:

  • Retrieves candidate application details from Pinpoint when an offer is accepted.
  • Collects important information from the candidate’s application, such as personal details, job information, and offer details.
  • Prepares the candidate’s information so it can be used to create an onboarding applicant in ADP Lyric.
  • Checks whether the required onboarding information is available before creating the onboarding record.
  • Automatically creates a new onboarding applicant record in ADP Lyric using the candidate’s information from Pinpoint.
  • Skips the onboarding process if the required information is missing, preventing incomplete records from being created.
  • Supports a report-only mode that allows administrators to review the onboarding data without creating records in ADP Lyric.

Integration Setup

This section outlines how the integration must be configured to send candidate onboarding information from Pinpoint to ADP Lyric.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.

  2. Navigate to Integrations from the left side panel.

  3. Click Create Integration.

  4. Search for Pinpoint / ADP Lyric in the search box next to the Explore Templates section and click it to create a new integration.

  5. Enter a name and a description for the integration.

  6. Select your Pinpoint and ADP Lyric connector instances.

  7. Click Continue. The Configuration page is displayed to configure the integration.

Onboarding 

Select the onboarding fields that must be available in the Pinpoint application before creating an onboarding applicant in ADP Lyric.

Note: Skip the onboarding process for a candidate if any selected field is missing in the candidate’s application to avoid creating incomplete records.

Onboarding field validation

Select one or more fields that must be available in the candidate’s Pinpoint application before creating the onboarding record in ADP Lyric.

  • Hire date: Specify the date when the candidate is expected to start the job.
  • SSN: Enter the candidate’s Social Security Number.
  • Date of birth: Enter the candidate’s date of birth.
  • Reports to: Specify the manager or supervisor the candidate will report to.
  • Address1: Enter the primary address of the candidate.
  • Address2: Enter additional address details such as apartment or unit number.
  • Job codes: Specify the job code associated with the candidate’s position.
  • HR departments: Specify the HR department responsible for the candidate.
  • Payroll departments: Specify the payroll department associated with the candidate.

Note: Enable or disable fields based on onboarding requirements. Process onboarding only for candidates who have all selected fields available in their Pinpoint application.

Modify attribute mappings

Update the source attribute expressions as needed to match your data structure or business requirements. The target attributes represent the ADP Lyric onboarding fields and typically do not require modification. You can modify the value of a source attribute by writing a JavaScript expression. The expression can transform, combine, or derive values from the source data before sending them to the target field.

Enter the expression directly in the source attribute field, or click the expand icon next to the field to open a larger editor for writing longer expressions.

Tables

Tables allow you to define and maintain reference data that the integration can use to populate or translate attribute values during execution.

You can use tables when the required values are not directly available in the source application or when the integration needs to map source values to predefined values expected by the target system.

Each table contains predefined key-value pairs that the integration references at runtime to determine the correct value to populate in the target application.

Available Tables

The following tables are available in this integration.

  • Disability status: This table defines mappings for disability status values used during onboarding.
  • Emergency contact relationship: This table defines the relationship types used for emergency contacts, such as spouse, parent, or sibling.
  • Gender for insurance compliance: This table defines gender values required for insurance or compliance-related fields.
  • Legal entity code: This table defines legal entity identifiers used to associate employees with the correct organization entity.
  • Location code: This table defines location identifiers used to assign employees to specific work locations.
  • Marital status: This table defines marital status values used in employee personal information.
  • Pay groups: This table defines pay group values used to assign employees to payroll groups.
  • Race/ethnicity code: This table defines race and ethnicity classification codes used for compliance and reporting.
  • Veteran status: This table defines veteran status values used for regulatory and reporting purposes.

Managing Table Data

Click a table row to open the Update Table window. Each row in a table represents a key–value pair, where the Name acts as the lookup key and the Value defines the corresponding attribute or rule applied during synchronization.

You can manage table entries in the following ways:

  • Add Row: Use this option to manually add a new entry by specifying a key name and its corresponding value.
  • Upload CSV File: Use this option to upload multiple rows from a CSV file. The CSV file must contain exactly two columns with the headers NAME and VALUE, and each row represents a key–value pair to be added to the table.
  • Export: Use this option to download the existing table data for review or offline editing.

After making changes, click Save to apply and persist the updates.

Note: The key name within a table must be unique and is used as the lookup identifier during integration execution. Key names can include letters and numbers and support most special characters, except forward slash (/), backslash (), and double quotes (").

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements.

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • Integration details: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • Template details: Displays the current version number and its features under the Current version section. If a newer version is available, the version number and the changes are shown under New Version available with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration. The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
