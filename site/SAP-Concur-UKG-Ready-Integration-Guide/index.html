Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employee data from UKG Ready to SAP Concur.

Key Functions

This integration automates identity lifecycle management between UKG Ready (an HR/Payroll system) and SAP Concur (an expense management system) with the following primary capabilities:

  • Creates new users in SAP Concur when they are added in UKG Ready.
  • Updates existing SAP Concur users when their information changes in UKG Ready.
  • Deactivates SAP Concur users when employees are terminated in UKG Ready.
  • Creates and deletes users based on a specific date and time.

 

  • Assigns appropriate roles in SAP Concur based on job titles from UKG Ready.
  • Synchronizes employee expense records from SAP Concur to UKG Ready.
  • Supports testing mode for validation.
  • Supports incremental synchronization to process only changed records.
  • Sends email notifications for various events, such as user creations, updates, and terminations.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for SAP Concur / UKG Ready in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Business Groups

A business group is a logical group created within the integration system to make group assignments easier. It works as an intermediate layer between the source system and the target application groups.

In this approach, users are first assigned to business groups based on specific attributes configured in the Access Rules section. These business groups are then mapped to the appropriate groups in the target applications.

The Business Groups section allows you to define business groups and map them to the target application groups.

In this section, if the List View toggle button is disabled, the following parameters are displayed:

  • Name: Provide a name for the group.
  • Description: Provide a short description for the group.
  • Owner: Enter the email addresses of the group owners.
  • Is Sub group: Enable this parameter to include other business groups as a subgroup to this group. Users assigned to the selected subgroups will also be part of this group. When enabled, the following parameter is displayed for further configuration:
    • Sub Business Groups: Select one or more business groups you want to include in this group.
  • Business Groups with Application Groups: Select this checkbox to map the business group to the target application groups.
    When enabled, the following fields are displayed for further configuration:
    • Select Application: Select the target applications for which you want to manage group assignments through this business group.
    • Entitlement Type: Select an entitlement type available in your target applications to which users will be assigned through this business group. For example, select Licenses.
    • Entitlements: Select an entitlement from the dropdown menu to which users will be assigned through this business group. For example, select Licenses.

Access Rules

This section allows you to define rules for assigning users to business groups defined in the Business Groups section.

Perform the following steps to define access rules for the employees:

  1. Click the vertical ellipsis next to Choose Attribute, and then click Select Attribute to view the list of available attributes you can use to set the assignment rule.
  2. Search and select the attribute you want to use to define a condition. For example, select Employment Status Reason.
  3. In the below row, double-click and select the attribute value to define the rule. If multiple values are selected, OR condition is applied among them. For example, in the following sample condition, a user's employment status reason must be either New Hire or Rehire to satisfy this assignment rule.
  4. If required to add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  5. If required to include another attribute to the group assignment rule, click the vertical ellipsis in the first column, and then click Add Attribute.
    If multiple columns are added for a rule, AND condition is applied between different columns. For example, in the following sample condition, employee's Employment Status Reason must be either New Hire or Rehire, and Primary Work Assignment Department must be Engineering to satisfy this assignment rule. You can use multiple attributes and their corresponding values to define a group assignment rule.
  6. Under the Organisational Unit (OU) field, double-click and select the OU to which users who satisfy the specified condition will be assigned.
  7. If needed, click the + icon at the top right corner to add another rule, as explained above.
  8. Click Save Changes at the top-right corner of the page to save the configuration.

Note: If required to delete a row, click the horizontal ellipsis next to the row and select Delete Row.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

Within the Attribute Map tab, switch between the User Lifecycle Management and Expense sync sections and configure attribute mappings for each section as needed.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Log Insights Settings

This section enables you to customize the log insight settings.

Note: This section appears only if you have enabled the Enable Log Insights setting in the Parameters section.

It has the following settings:

  • Log Insight Type: Select the log insight type. This will decide the level of details to be included for log insight.
    The following are the available options:
    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
      The following fields are displayed for further configuration if Metadata, Non-PII Data, GUID with Non-PII Data, or Employee Number with Non-PII Data are selected:
      • Log Insight Level: Select the log insight level. The following are the available options:
        • All: Select this to include all attribute details for analysis.
        • Include List: Select this if you want to choose attributes to be included for analysis. When selected, the following field is displayed:
          •  Attributes to be Included: Select the attributes to be included. For example, PersonNumber.
        • Exclude List: Select this if you want to choose attributes to be excluded from the analysis. When selected, the following field is displayed:
          • Attributes to be Excluded: Select the attributes to be excluded. For example, FirstName.

Refer to the Log Insights guide for more details.

Entitlements

The Entitlements tab is displayed only when it is enabled in the Parameters section.

It allows you to configure how entitlement assignments (roles) are managed through this integration.

In this tab, you can define conditions that control how entitlements are assigned to employees.

Perform the following procedure to create role assignment rules:

  1. Click Add New Rule to create a new assignment rule. The following fields are displayed:
  2. Click Choose Attribute to view the list of UKG Ready employee attributes.
  3. Search and select the attribute using which you want to set a condition. For example, Job Title.
  4. Select the attribute value to define the rule. Only those users who meet this condition will satisfy this assignment rule. If multiple values are provided, an OR condition is applied among those values. For example, in the following sample condition, an employee's job title must be either Benefits Administrator or Benefits Manager for them to satisfy this assignment rule.
  5. In the Entitlement Assignment column, select the roles to which the employees who satisfy the specified condition must be assigned. The list shows all the roles from the target SAP Concur application.
  6. Click Add Row to add another condition and assignment for the same UKG Ready attribute.

    Notes:
    • You can add any number of rows within a rule.
    • You can delete a row by using the delete icon at the right end of the row.
  7. [Optional] If you want to write a JavaScript expression to set the condition, click Switch to Expression.
    Note: When you switch to the Expression mode or to the Basic Condition mode, all the configured conditions of the rule will be cleared.
    1. Select one of the following based on your requirement:
      • For Conditions Only: Select this if you want to write a JavaScript expression only for the conditions.
         
      • For Conditions and Assignments: Select this if you want to write expression for both, conditions and assignments.
    2. Click Continue to configure the expression.

      Note: Expressions can use all attributes in the source and destination applications. The configured expressions must always be evaluated to true or false.
      SOURCE must be used in an expression to represent an attribute in the source application and DEST to represent an attribute in the destination application.
      For example, SOURCE.EmployeeNumber, SOURCE.HomeOrganization.ShortName, and DEST.EmployeeNumber.
  8. Click Add New Rule to add another assignment rule, as explained above.
    Note: You can add any number of assignment rules. However, you can see only 5 rules on a single page. If you add more than 5 rules, navigation options will be available at the bottom of the page to navigate to other pages for configuring or viewing the remaining defined rules.

Note: The integrations assigns roles to the user but doesn't remove the existing assigned roles. You must unassign the required roles manualy in SAP Concur.
