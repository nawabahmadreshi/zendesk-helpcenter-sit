Watchdog Setup Guide: Dayforce to Entra ID Integration

← Back to index

Overview

Aquera Watchdog is a monitoring and validation tool designed to identify data synchronization issues between integrated applications. It operates independently of the Aquera integration and analyzes application events to detect inconsistencies between source and target systems.

After an integration execution is completed, you can run the Watchdog to retrieve the event logs, analyze the activities, and report any anomalies or data discrepancies across the applications. For example, if a user is created in the source application, Watchdog checks for a corresponding user provisioning event in the target application and reports an anomaly if no matching event is found. Aquera Watchdog helps ensure guaranteed delivery of events across integrated systems by continuously monitoring and validating data synchronization between the source and target applications.

When you run the integration Watchdog, a dashboard is generated with a consolidated view of the activities across the systems, including any detected anomalies. The dashboard can be accessed through the Insights module. Refer to the Watchdog Insights section for the details captured in the dashboard.

Watchdog ensures data consistency across systems, provides visibility into sync failures, and helps proactively identify and resolve integration issues. 

This guide provides details on how to configure and run the Watchdog for the integration that synchronizes employee data from Dayforce to user accounts in Entra ID. It also covers the Watchdog settings used to track and display tickets created in ServiceNow in response to employee lifecycle events (new employee onboarding, employee changes and transfers, and employee terminations) in Dayforce.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal to connect to those applications for retrieving the event logs:
    • Dayforce: Ensure you add and configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for details.
    • Microsoft 365: Ensure you add and configure the Microsoft 365 connector in the Aquera Admin portal. Refer to the Microsoft 365 Configuration Guide for details.
    • ServiceNow: Ensure you add and configure the Microsoft 365 connector in the Aquera Admin portal. Refer to the ServiceNow Configuration Guide for details.

Watchdog Setup

This section describes how to configure the Watchdog that monitors the user sync from Dayforce to Entra ID and the corresponding JML ticket creations in ServiceNow.

Create Integration Watchdog

Follow these steps to create the integration watchdog:

  1. Log in to the Aquera Admin Portal.

  2. In the left panel, go to Integrations.

  3. Click Create Integration.

  4. Select Dayforce / Entra ID / ServiceNow [Watchdog].

  5. Enter a name and description for the integration.

  6. Choose your Dayforce, Microsoft 365, and ServiceNow connector instances.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Dayforce and Microsoft 365 connectors that you have selected while creating the integration watchdog. This section enables you to search and select a different instance of the application connectors, if required.

Parameters

The Parameters section enables you to customize the Watchdog settings according to your use case. It has the following settings:

  • Dayforce employee match attribute: Select the Dayforce employee attribute that must be used to uniquely identify an employee record and match with an Entra ID user record.
  • Entra ID user match attribute: Select the corresponding Entra ID user attribute that is used to match against the selected Dayforce attribute to identify the correct Entra ID user record.
  • Initial log lookback period: Specify the time period from which the watchdog retrieves historical logs from Dayforce and Entra ID during its initial execution. After the first run, the watchdog processes only the event logs that have been created since the last run. These logs are used by Watchdog to compare and analyze the events across both systems. The following options are available:
    • Last: Select this option to retrieve logs from a specific duration in the past. You can specify the number of days, hours, or minutes for which logs should be retrieved.
    • Since the date: Select this option to retrieve and process logs starting from a specific date. A calendar field is displayed to select the date.
    • From the beginning of available logs: Select this option to retrieve all logs currently available in Dayforce and Entra ID.

Policy

The Policy tab enables you to set the access policies of users for Applications, Groups and Directory. 

When the Watchdog runs, these policies are evaluated to identify deviations and potential access violations. These policies serve as the basis for auditing user access and detecting compliance issues. Based on the evaluation results, Watchdog generates Audit dashboards and reports that help administrators review and analyze access compliance across the connected systems. Refer to the Watchdog: Audit Dashboard Guide for details on Audit dashboard.

Access Policy

The Access Policy section allows you to define policies that determine where users should be placed within specific Enterprise applications, groups, and the directory.

Application Policy

Turn on the Enable toggle to configure the access policies for applications. The following sections are displayed.

Required Applications

Use this section to define the conditions under which users must be part of specific Enterprise applications in Entra ID. Users who meet these conditions are expected to be present in those applications.

  1. Turn on the Enable toggle to configure the access policies for required applications.
  2. Click on Choose Attribute to select a user attribute from the Dayforce application that you want to use in the condition.
  3. After selecting the attribute, specify one or more attribute values. Users who match any of the selected attribute values (OR operation) are considered eligible for the application.
  4. If required, add more attributes and values. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. This means users who satisfy all attribute conditions in that row must be part of the corresponding application.
  5. In the Applications field, select the applications where users matching the defined conditions should be present.
  6. If required, click Add Row to add another condition within the same rule. 

  7. If required, click the + icon at the top-right corner of the rule setting section to add another rule with a set of rows/conditions for required applications. 

    When multiple rules are set, the integration evaluates every rule for each user.

Forbidden Applications

Use this section to define the conditions under which users must not be part of specific OUs. Users who meet these conditions are not expected to be present in those OUs.

The policy setup is similar to the Required Applications procedure.

Group Policy

Turn on the Enable toggle to configure the access policies for groups. The following sections are displayed.

Required Groups

Use this section to define the conditions under which users must be members of specific groups. Users who meet these conditions are expected to be present in those groups.

The policy setup is similar to the Required Applications procedure.

Forbidden Groups

Use this section to define the conditions under which users must not be part of specific groups. Users who meet these conditions are not expected to be present in those groups.

The policy setup is similar to the Required Applications procedure.

Directory Policy

Turn on the Enable toggle to configure the directory access policies. 

You can set the conditions that define which users should and should not have access to the directory.

When you configure the policy, you must select Yes or No in the Can Access? field that indicates whether the user satisfying the condition must get access to the directory or not.

Attribute Policy

The Attribute Policy section enables you to select the attributes for which you want to see the values in the Insights dashboard. Only the user attributes that are selected will be displayed in Insights logs.

ITSM

The ITSM tab enables you to set the configurations for monitoring the ServiceNow ticket creations for the employee lifecycle events such as new employee onboarding, employee changes and transfers, and employee terminations. You can also specify the correlation attributes used to match Dayforce employee records with their corresponding ITSM tickets, enabling Watchdog to validate ticket creation and identify any discrepancies.

Application Configurations

This section enables you to set the ITSM configurations for the Watchdog.

  • ITSM Enabled: Turn on the toggle to retrieve the logs from ServiceNow to monitor the JML ticketing events. The configuration fields are displayed.
  • Enable common catalog for user lifecycle events: Turn on this toggle if the same catalog is used in ServiceNow for tracking Joiner, Mover (profile updates, department changes, and so on), and Leaver events. If different catalogs are used for Joiner, Mover, and Leaver events, then turn off the toggle. The remaining fields in this section vary based on this selection.
    • When Toggle is disabled:
      • Joiner catalog type: Select the ServiceNow catalog used to create Joiner requests. Watchdog uses this catalog to identify and retrieve relevant Joiner tickets. 
      • Mover catalog type: Select the ServiceNow catalog used to create employee profile change requests. Watchdog uses this catalog to identify and retrieve relevant Mover tickets. 
      • Leaver catalog type: Select the ServiceNow catalog used to create employee termination requests. Watchdog uses this catalog to identify and retrieve relevant termination tickets. 
    • When Toggle is enabled:
      • Catalog type: Select the common ServiceNow catalog that is used to create Joiner, Mover, and Leaver requests. Watchdog uses this catalog to identify and retrieve relevant Joiner, Mover, and Leaver tickets. 
      • Lifecycle event classification attribute: Select the attribute in the ITSM ticket that indicates the type of employee lifecycle event (Joiner, Mover, or Leaver). Watchdog reads this field to determine how each ticket should be classified.
      • Joiner: Enter the value of the Lifecycle event classification attribute that represents a new employee onboarding event.
      • Mover: Enter the value of the Lifecycle event classification attribute that represents an employee transfer or change event.
      • Leaver: Enter the value of the Lifecycle event classification attribute that represents an employee termination event.
  • Dayforce employee match attribute: Select the attribute that can be used to uniquely identify an employee in Dayforce.
  • ServiceNow attribute to find the matching ticket: Select the corresponding field in the ServiceNow catalog that stores the unique employee identifier. Watchdog matches this field against the Dayforce employee match attribute to identify the tickets generated for the employee events.

Watchdog Insights

This section details the data covered in the dashboard created by the Watchdog for the ADP Workforce Now to Entra ID integration. 

  1. Log in to the Aquera Admin portal.
  2. Navigate to Insights.
  3. Click on the integration watchdog name, for example, Dayforce to Entra ID Watchdog. The dashboards are displayed.

The dashboard provides a comprehensive view of workforce events and their synchronization across Dayforce and Entra ID. It includes the following insights:

  • Displays the count of Joiner, Mover, and Leaver events in Dayforce, along with corresponding user creations, updates, and deactivations in Entra ID. This helps identify any discrepancies in user data between the two systems.
  • A checklist that verifies whether a corresponding Entra ID user record has been created for each employee added in Dayforce.
  • A timeline view illustrating a worker’s journey, from hiring in Dayforce to account creation in Entra ID, and from termination to account deactivation.
  • A dashboard showing the upcoming hires and terminations, categorized by departments
  • A trend graph that provides visibility into synchronization patterns or gaps between HR and directory events.
  • A geographical map showing the number of Entra ID user logins across various locations.
  • A list of Entra ID applications with the highest number of user logins.
  • The list of users who have been inactive for a long time.
  • A trend graph showing the most frequently accessed applications over a period of time.
  • A pie chart showing the percentage distribution of workers across different departments.
  • A horizontal bar chart showing the distribution of workers across various managers.
  • Insights into workforce distribution by geographic location, and by department and country.
  • A line chart illustrating hiring trends compared to employee terminations over time.
  • Detailed logs of events in the Dayforce application and the Entra ID directory.
