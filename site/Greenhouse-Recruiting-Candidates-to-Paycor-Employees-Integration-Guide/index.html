Overview

The Aquera platform allows your organization to scale complex data management by automating the integration of data across resource types. This guide covers the end-to-end tasks in setting up a script integration to sync the Candidates resource of Greenhouse Recruiting to the Employees resource of Paycor.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for details.
    • Paycor: Refer to the Paycor Configuration Guide for details.

Integration Setup

This section details how the integration must be configured to allow data push from Greenhouse Recruiting to Paycor.

Create Integration

Perform the following steps to create a new integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Greenhouse Recruiting Candidates to Paycor Employees.
  5. Enter a name and a description for the integration.
  6. Select your Greenhouse Recruiting and Paycor connector instances.
  7. Click Continue. A new page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Preferences

The Preferences section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Report Only: Select to create a report outlining the data changes that would occur without implementing them.
