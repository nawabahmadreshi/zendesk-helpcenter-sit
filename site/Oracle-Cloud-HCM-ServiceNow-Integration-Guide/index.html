Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration synchronizes employee lifecycle events from Oracle HCM to ServiceNow and automatically creates or updates ServiceNow catalog requests or tickets based on workforce changes.

By default, Oracle HCM serves as the primary source of employee data, enabling the integration to automatically trigger ServiceNow fulfillment processes for onboarding, job changes, leave of absence, termination, and rehire events.

Key Functions

This integration automates employee lifecycle management with the following capabilities:

  • Automates employee onboarding, job change, leave of absence, termination, and rehire processes.
  • Creates ServiceNow Service Catalog requests based on employee lifecycle events
  • Creates and manages ServiceNow table-based tickets.
  • Generates requests dynamically based on employee attributes and event data.
  • Supports dynamic mapping of ServiceNow catalog variables.
  • Automatically updates existing requests when employee data changes.
  • Generates detailed logs and reports.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Oracle Cloud HCM: Refer to the Oracle Cloud Configuration Guide for details.
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Entra ID to ServiceNow.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Oracle Cloud HCM / ServiceNow in the search box next to the Explore Templates section and click it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select your Oracle Cloud HCM and ServiceNow connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Once created, the following tabs are displayed:

  • Overview
  • Source
  • Service Desk
  • Operations
  • Features

Overview

On this tab, you can view integration details and the list of additional features supported by the integration. Scroll down to the Integration Features section to enable or disable these features based on your use case.

Lifecycle Event

Configure the following ticket creation settings for user lifecycle events:

  • Create Tickets for User Lifecycle Events: Enable this option to create tickets when configured user lifecycle events occur automatically. When enabled, the Event Configuration section is displayed for further configuration.

Event Configuration

Configure the following event settings:

If the List View toggle button is disabled, the following parameters are displayed:

  • Select Event Type: Select the user lifecycle event for which service desk tickets should be created. If you do not find your event type in the list, click the refresh icon available next to the field to reload the available events. The following are the available options:
    • Hire: Select this event to create tickets for new employee onboarding.
    • Change: Select this event to create tickets when employee information changes.
    • Leave: Select this event to create tickets for employee leave events.
    • Terminate: Select this event to create tickets for employee termination events.
  • Event Timing: Select when the service desk ticket should be created for the selected lifecycle event. The following are the available options:
    • When Detected: Select this option to create the ticket immediately when the event is detected.
    • Before Start Date: Select this option to create the ticket before the start date.
      • Days Before Joining Date: Enter the number of days before the employee's joining date when the service desk ticket should be created.
    • On Start Date: Select this option to create the ticket on the start date.
    • After Start Date: Select this option to create the ticket after the start date.
      • Days After Joining Date: Enter the number of days after the employee's joining date when the service desk ticket should be created.
  • Enable Conditional Event Detection: Enable this option to define a custom condition for detecting lifecycle events. When enabled, additional configuration options are displayed:
    • Custom Ticket Creation Rule: Define a custom rule to control when tickets are created for the selected lifecycle event. Use filters or expressions to specify the required conditions.
      This field uses Aquera Intelligence (AI Assistant). For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
  • Enable Ticket Creation: Enable this option to automatically create a ServiceNow ticket for configured lifecycle events. When enabled, additional configuration options are displayed:
    • Ticket Type: Select the type of ServiceNow ticket or request to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available tickets. The following are the available options:
      • Service Catalog: Select this option to create and process requests through the ServiceNow Service Catalog.
      • Ticket: Select this option to create and process standard ServiceNow tickets.
    • Ticket Category: Select the ServiceNow ticket category to assign to tickets created for the configured lifecycle event. If you do not find your ticket category in the list, click the refresh icon available next to the field to reload the available tickets. For example, select Application and Account Access.
    • Ticket Type Name: Select the name of the ServiceNow ticket type to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available ticket types. For example, select Change Password. 
    • Customize User-to-Ticket Matching: Enable this option to define custom rules for matching Oracle HCM employees with existing ServiceNow tickets. When enabled, additional configuration options are displayed:
      • Source Correlation Attribute: Select the Oracle Cloud HCM attribute to uniquely identify and correlate users between both systems.
      • Target Correlation Attribute: Select the ServiceNow attribute to uniquely identify and correlate users between both systems.
    • Enable Ticket Auto-Update: Enable this option to automatically update ServiceNow JML tickets only when their current status matches one of the selected ticket statuses. When enabled, additional configuration options are displayed:
      • Ticket Statuses for Auto-Update Evaluation: Select the ServiceNow ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic updates. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Open.
      • Resolved Statuses for Onboarding Tickets: Select the ServiceNow ticket statuses that indicate an onboarding ticket has been completed or resolved. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Open.
    • Enable Ticket Auto-Close: Enable this option to automatically close the ServiceNow JML ticket after all actions associated with the user lifecycle event have been completed successfully. When enabled, additional configuration options are displayed:
      • Ticket Statuses for Auto-Close Evaluation: Select the ServiceNow ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic closure. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
      • Ticket Statuses for Auto-Close: Select the ServiceNow status that will be assigned to the ticket when it is automatically closed. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
    • Attribute Mapper: This section allows you to control how each attribute is synchronized between Entra ID and the target application.
      Source attributes are listed on the left side and the target attributes are listed on the right side of the page. You can modify only the source attributes as needed, to align with your specific use case.
      JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
      Attributes marked with a red asterisk (*) are required for successful provisioning.
      The Search Attributes field allows you to filter the list of available attributes for mapping.
      After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.

To add another lifecycle event in this integration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the Lifecycle Event section in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
