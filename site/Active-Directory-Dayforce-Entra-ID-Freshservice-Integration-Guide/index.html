Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Active Directory / Dayforce / Entra ID / Freshservice as the integration template.
  5. Enter a name and a description for the integration.
  6. Select Dayforce, Active Directory, Entra ID, and Freshservice connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • AD Application List*: Select your Active Directory application from the dropdown menu. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

General Settings

Configure the following general settings:

  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable log insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable full sync: Select this checkbox to execute the integration in full sync. If this checkbox is not selected, the integration will be executed in the incremental import mode, syncing only new and updated records from Dayforce to Active Directory.
  • Enable provisioning via Freshservice:
  • Enable Freshservice journey full sync: 
    If this checkbox is checked, all Journeys will be fetched from Freshservice, else only recently created or updated ones will be fetched.

Manage sObjects

Configure the following sObjects management settings:

  • Source Field for Correlation: Select the Salesforce attribute for correlation from the dropdown menu.

  • Target Field for Correlation: Select the Active Directory attribute for correlation from the dropdown menu.

  • Create Salesforce Objects in Active Directory: Select to automatically create corresponding Active Directory objects whenever new records are added in Salesforce. If disabled, new Salesforce records will not create objects in AD.
  • Update Salesforce Mapped Objects in Active Directory: Select to automatically update existing Active Directory objects when mapped Salesforce records are modified. If disabled, changes in Salesforce will not be updated in AD.

User Update

Configure the following user update settings:

  • Enable Critical AD Attribute Updates: Select to modify critical Active Directory attributes, such as Common Name, Email ID, User Principal Name, and SAM Account Name. If disabled, accidental changes could not affect the user's identity or access.
    • Update CN (rename DN): Select to rename the user's Common Name (CN) in Active Directory and update the Distinguished Name (DN).
    • Update UPN (User Principal Name): Select to update the user's logon name (UPN), which is created by combining the user's first and last name with the UPN suffix, for example, jane.doe@example.com. 
    • Update Primary Email: Select to update the user's primary email address in Active Directory, for example, jane.doe@example.com. 
    • Update sAMAccountName: Select to update the legacy logon (sAMAccountName). It must be unique within the domain and is limited to 20 characters.

Password Policy

Configure the following password policy settings:

  • Generate Password: Select to generate a random password.
    Note: It is mandatory to enable this checkbox to customize the password settings.

    • Password Type*: Select the desired password complexity level from the drop-down menu. The following are the available options:

      • Basic: Includes uppercase letters, lowercase letters, and digits.
      • Advanced: Includes uppercase letters, lowercase letters, digits, and special characters.
        • Allowed Special Characters: Enter the set of special characters permitted in a user's password.

    • Password Length: Enter the maximum permissible password length.

Insights Settings

Configure the following insights settings:

  • Enable Log Insights: Select to visualize the integration-related metrics.

Naming Policy

Configure the following naming policy settings:

Generate sAMAccountName

  • Enter Your Prompt to Create sAMAccountName: Enter your policy to generate the sAMAccountName. For example, enter the following prompt and click Analyze and Preview:

    Generate 4–5 samAccountName options using the following rules: 

    Start with the first letter of the first name followed by the full last name to form the base samAccountName.

    If that samAccountName is already taken, append a numeric suffix (such as 1, 2, or 3) to generate additional options.

    Convert all characters to lowercase, remove spaces and special characters from both first and last names, and limit the final samAccountName to a maximum of 20 characters.

Preview: Based on the prompt and the default input (First Name: Alexander, Last Name: Johnson), the generated samAccountName values are ajohnson, ajohnson1, ajohnson2, ajohnson3, and ajohnson4, as displayed under the Preview section. To view the default name values, turn on the Advanced Preview Options toggle.

Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated sAMAccountName values will appear in the Preview section.

Generate Common Name (CN)

  • Generate Common Name: Select an option to generate the user’s common name from the dropdown. The following are the available options:
    • Use sAMAccountName as CN: Select to use the sAMAccountName generated from the Generate sAMAccountName section as the common name.
    • Use Policy to Generate CN: Select to define a custom policy for generating the common name. When selected, this option enables the following settings:

      • Enter Your Prompt to Create a Common Name: Enter your policy to generate the common name. For example, enter the following prompt and click Analyze and Preview:

        Generate up to six CN values using the following rules:

        Build the base CN as "Firstname Lastname" or "Firstname Middlename Lastname" when a middle name is provided.

        For each name part, capitalize the first character and make all remaining characters lowercase (e.g., "James Edward Smith").

        Escape or remove special characters to ensure validity.

        If the base CN exists, generate additional variants by appending numbers 1 through 5 directly at the end (e.g., James Edward Smith1, James Edward Smith2).

        Preview: Based on the prompt and the default input (First Name: James, Middle Name: Edward, Last Name: Smith), the generated common name values are James Edward Smith, James Edward Smith1, and James Edward Smith2, as displayed under the Preview section. To view the default name values, turn on the Advanced Preview Options toggle.
        Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name, middle name, and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated common name values will appear in the Preview section.

Generate Email

  • Generate Email: Select an option to generate the user's email ID from the dropdown. The following are the available options:
    • Use sAMAccountName as Email: Select to use the sAMAccountName generated from the Generate sAMAccountName section to generate an email.
    • Use UserPrincipalName as Email: Select to use the user principal name generated from the Generate UserPrincipalName (UPN) section to generate an email.
    • Use Different Policy to Generate Email: Select to define a custom policy for generating the email. When selected, this option enables the following settings:

      • Enter Your Prompt to Create an Email: Enter your policy to generate an email, excluding the domain. For example, enter the following prompt and click Analyze and Preview: 

        Generate at least 5 unique email prefixes (no domain) using firstname, middlename (if available), and lastname:

        First attempt: first letter of firstname + full lastname (e.g., ajohnson).

        Second attempt: If middlename exists, first letter of firstname + first letter of middlename + full lastname (e.g., amjohnson).

        Subsequent retries: Append numbers 1-5 to the primary attempt (e.g., ajohnson1 through ajohnson5 if no middlename, or amjohnson1-5 otherwise).

        Always convert the output to lowercase, remove all spaces and special characters, and use the shortest valid prefixes first.
        Example input: firstname = “Alexander”, middlename = “Michael”, lastname = “Johnson”.

        Preview: Based on the prompt and the default input (First Name: Alexander, Middle Name: Michael, Last Name: Johnson), the generated email prefixes are ajohnson, amjohnson, ajohnson1, ajohnson2, and ajohnson3, as displayed under the Preview section. To view and adjust the default name values used for generating these prefixes, turn on the Advanced Preview Options toggle.
        Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name, middle name, and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated email prefix values will appear in the Preview section.

    • Use Common Name as Email: Select to use the common name generated from the Generate Common Name (CN) section to generate an email.

Generate UserPrincipalName (UPN)

  • Generate User Principal Name: Select an option to generate the user's UPN from the dropdown. The following are the available options:
    • Use sAMAccountName as UPN: Select to use the sAMAccountName generated from the Generate sAMAccountName section as the user principal name.
    • Use Different Policy to Generate UPN: Select to define a custom policy for generating the user principal name. When selected, this option enables the following settings:

      • Enter Your Prompt to Create a UPN: Enter your policy to generate the User Principal Name (UPN). For example, enter the following prompt and click Analyze and Preview:

        Generate 5 UPN prefix outputs using the following rules:

        First attempt: Combine the first letter of the first name with the full last name.

        Second attempt: If a middle name exists, use the first letter of the first name + first letter of the middle name + full last name.

        If no middle name exists, append numbers 1 through 4 to the first attempt (e.g., ajohnson1, ajohnson2).

        Convert all characters to lowercase, remove spaces and special characters.

        Limit total length as needed and return only the prefix (no domain).

Preview: Based on the prompt and the default input (First Name: Alexander, Middle Name: Michael, Last Name: Johnson), the generated user principal name prefixes are ajohnson, amjohnson, amjohnson1, amjohnson2, and amjohnson3, as displayed under the Preview section. To view and adjust the default name values used for generating these prefixes, turn on the Advanced Preview Options toggle.

Advanced Preview Options: If you want to test with custom values, turn on the Advanced Preview Options toggle to display the Add Custom Values panel. There, you can enter the first name, middle name, and last name, and click Refresh Preview. Based on these values and the defined prompt, the generated user principal name values will appear in the Preview section.

Attribute Map

The Attribute Map tab defines how data is mapped from the Source Application to the Target Application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
