Oracle E-Business Suite to Okta Orchestration Setup Guide

← Back to index

Overview

This guide covers the end to end tasks involved in setting up an orchestration to sync users in the Oracle E-Business Suite (Oracle EBS) with Okta.

The following tasks are involved in setting up the orchestration from Oracle EBS to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Orchestration Setup
  • Running Orchestration

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in Aquera Admin console:

  • Oracle EBS. Refer to the Oracle E-Business Suite Configuration Guide for details.
  • Okta. Refer to the Okta Configuration Guide for details.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from Oracle EBS to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Considerations

  • Work email can be retrieved from the Oracle EBS system or be generated by Aquera orchestration and updated back to the system.
  • The criteria for matching a user record from Oracle EBS with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping the Oracle EBS user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in Oracle EBS as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script orchestration. For scripts, refer to the Appendix.

Building Orchestration

This document describes how to set up the orchestration using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set Oracle E-Business Suite as the Source Application and Okta as the Target Application.

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in restriction list for scoping of the groups which would be managed by Aquera.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Running Orchestration

Perform the following steps to run the orchestration:

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in the Oracle E-Business Suite.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in Oracle EBS and Okta before setting up the orchestration. 

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add Oracle EBS and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

Few sample scripts are provided here for your reference. The scripts may vary based on your use case  and the attributes to be retrieved for analysis.

Analyzing the Users in Oracle E-Business Suite

Use the below sample script in a scheduled script orchestration for generating a report from Oracle EBS for analysis of user status, location, department, and so on.

let adp_users = ADP.importAllUsers();

let i = 0;

log.info('ADP ID,ADP STATUS,ADP EMPNO,ADP GIVENNAME,ADP FAMILYNAME,ADP WORK EMAIL,ADP LOCATION CODE,ADP JOBCODE,ADP DEPT');

for (i = 0; i < adp_users.length; i++) {

    let adp_id = adp_users[i].id;

    let adp_status = adp_users[i][IAN].status;

    let adp_emp = adp_users[i][ENT].employeeNumber;

    let adp_locationcode = adp_users[i][IAN].LocationCode || '';

    let adp_job = adp_users[i][IAN].jobCode || '';

    let adp_dept = adp_users[i][IAN].departmentCode || '';

    let adp_given = adp_users[i].name.givenName;

    let adp_family = adp_users[i].name.familyName;

    let adp_work = adp_users[i].emails && adp_users[i].emails.find(x => x.type == 'work') && adp_users[i].emails.find(x => x.type == 'work').value || '';

    log.info('"' 

    + adp_id + '","' 

    + adp_status + '","'

    + adp_emp + '","'

    + adp_given + '","' 

    + adp_family + '","'

    + adp_work + '","'

    + adp_locationcode + '","'

    + adp_job + '","'

    + adp_dept + '"');

}

Analyzing the Users in Okta

Use the below sample script in a script orchestration for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.importAllUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}
