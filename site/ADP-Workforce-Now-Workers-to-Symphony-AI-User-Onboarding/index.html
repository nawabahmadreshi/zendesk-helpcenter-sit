Overview 

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

This Integration guide helps you retrieve employee details from ADP Workforce Now, format and sanitize the worker data, and create a user onboarding ticket in Symphony AI. The integration triggers when a candidate’s status changes to Hired in ADP Workforce Now using an event webhook.

This integration performs the following tasks:

  • Retrieves employee details from ADP, including name, email, phone, department, job title, start date, and manager.
  • Formats ADP data by combining fields (e.g., first, last name) and sanitizes the values (removing line breaks, escaping quotes).
  • Generates a SymphonyAI ticket payload for onboarding with mapped ADP fields.
  • Creates a SymphonyAI service request using an API and includes sync summaries and error details.
  • Logs failures in OpenSearch with timestamps, error codes, and payloads for auditing.
  • Sends email/SMS alerts to admins for unresolved errors.
  • Generates a CSV compliance report listing synced employees, errors, and timestamps.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for instructions on creating the ADP Workforce Now application connector and setting up the authentication.
    • Symphony AI: Refer to the SymphonyAI Configuration Guide instructions on creating the application connector and setting up the authentication.
  • Permissions to configure webhooks: Ensure the ADP Workforce Now worker has enough permissions to configure webhooks.

Integration Setup

This section describes the integration process for integrating ADP Workforce Now with Symphony AI. It covers the details of creating the integration, configuring system preferences, and using event-driven triggers to enable real-time synchronization of new hire information from ADP Workforce Now to Symphony AI.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now Workers to Symphony AI User Onboarding.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Symphony AI connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Preferences

The Preferences section allows you to customize the User Onboarding service request automation. It includes the following parameters:

  • Onboarding Service Request Type: Enter the exact name of the ticket type used to create onboarding requests. This is a required field. Providing an invalid name will prevent ticket creation and trigger error notifications to the configured email recipients.
  • Recipient Email Addresses: Provide one or more valid email addresses, separated by commas, to receive automated ticket submission notifications. Ensure these emails have permission to create service requests; incorrect formatting or unauthorized addresses may cause submission failures.
  • Send Email Notifications: Enable email alerts for ticket updates, approvals, and errors. When disabled, no notifications will be sent. Alerts are delivered to the addresses specified in the Recipient Email Addresses.
  • Debug Mode: Enable this setting to generate detailed logs for troubleshooting integration issues. When active, logs capture step-by-step execution data, essential for diagnosing errors and failures.
  • Default Email Recipient: Specify a fallback email address to receive notifications if no manager is assigned during onboarding. Use a valid email to ensure the delivery of essential alerts.
  • Approver Email ID: Specify the email address responsible for approving onboarding requests. Invalid or unreachable addresses may delay the approval process.
  • Request Instance: Assign the ticket to a valid department or instance to ensure correct routing and processing. Using an invalid name may cause misrouting or delays.
  • Service Request Description: Provide a description for all auto-generated tickets to help teams quickly identify the request’s origin and purpose.
  • Enable Integration Report Download: Select this option to allow exporting the onboarding data as a CSV file for auditing or reporting. 
  • Enable Dashboard Generation: Enable operational search and dashboard capabilities for querying onboarding data. 
  • Report Only: Enable this mode to restrict the system to read-only operations, preventing accidental changes to onboarding workflows. It is ideal for auditing or review phases.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the Execution Status indicators in the History column and navigating to the Execution Timeline panel.

Run Integration

You can set up an event or schedule triggers to execute the integration. For further details, refer to the Integration Using Scripts guide.

Once the integration is successfully executed, Aquera syncs the workers created in ADP Workforce Now and creates a ticket in Symphony AI.
