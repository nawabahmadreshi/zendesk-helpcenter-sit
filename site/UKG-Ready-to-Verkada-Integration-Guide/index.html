UKG Ready to Verkada Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines how employee data is synchronized between UKG Ready and Verkada. It also controls HR changes, including new hires, updates, terminations, and group assignments.

To identify user records across both systems, the integration uses the accountId and employeeId attributes from UKG Ready and matches them to the external ID in Verkada as a correlation attribute.

Key Functions

The UKG Ready to Verkada integrations provide the following functions:

  • Creates Verkada users from UKG Ready hires, using mapped details such as name and email.

  • Updates the Verkada user records when their UKG Ready information changes.

  • Deactivates Verkada users when the UKG Ready termination status changes.

  • Assigns Verkada users to groups using UKG Ready cost centre information.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • UKG Ready: Refer to the UKG Ready Configuration Guide for details.
    • Verkada: Refer to the Verkada Configuration Guide for details.

Integration Setup

The integration must be configured to push employee-related data from UKG Ready to the user-related data in Verkada.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select UKG Ready Employees to Verkada Users.
  5. Enter a name and a description for the integration.
  6. Select your UKG Ready and Verkada connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User Lifecycle Management: Select to enable user provisioning and perform incremental updates.
  • Enable Group Management: Select to assign users to groups based on their cost centre.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Incremental Mode: Select to run the integration in incremental mode.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Log Insights: Select to visualize the integration-related metrics.

Manage Users

The Manage Users section enables you to customize the integration settings according to your specific use case. It has the following settings:

User Provisioning Settings

Configure the following source settings:

  • Create User: Select to allow the integration to create a new user in the target application.
  • Update User: Select to allow the integration to update an existing user in the target application.
  • Deactivate User: Select to allow the integration to deactivate a user from the target application.
  • Report ID For Sync: Enter the report ID that includes employee details to be used for syncing.
  • Source Field for Correlation: Select the source attribute for correlation from the dropdown menu. The available options are Account ID and Employee ID.
  • Users to be Excluded from Sync: Enter a comma-separated list of user email addresses to exclude them from sync.
  • Create Condition: Select a condition from the dropdown menu to determine when users are created.
  • Update Condition: Select a condition from the dropdown menu to determine when users are updated.
  • Deactivate Condition: Select a condition from the dropdown menu to determine when users are deactivated.

User Provisioning Threshold Settings

  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single integration run.
  • Delete Limit: Specify the maximum number of user records that can be deleted from the target application during a single integration run.

Debug Settings

The Debug Settings section enables you to customize the user integration settings according to your specific use case. It has the following settings:

  • Source Filter: Select the UKG Ready attribute to filter users while retrieving from UKG Ready. The available options are Account ID and Employee ID.
  • Source Filter Value: Enter the value of the selected source filter.
  • Import Limit: Specify the maximum number of user records that can be imported in the target application during a single integration run.

Log Settings

The Log Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter* drop-down menu with the following options:
      • Included Attributes*: Select the attributes to be included in the log.
      • Excluded Attributes*: Select the attributes to be excluded from the log.

Report Settings

The Report Settings section enables you to customize the email notification settings according to your specific use case. It has the following settings:

  • Select the Report Detail Level*: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter* drop-down menu with the following options:
      • Included Attributes*: Select the attributes to be included in the report.
      • Excluded Attributes*: Select the attributes to be excluded from the report.

Notification Settings

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

Notification Settings

Configure the following settings for the notification:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Email Subject on Successful Integration Execution: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • Email Subject on Failed Integration Execution: Enter the content that must be used as the email subject in the notifications sent for integration errors.
  • Send Failure Email to SRE: Select to automatically send an email to the SRE team in the event of an integration failure.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors

Log Insights

The Log Insights section enables you to customize the integration settings according to your specific use case. It has the following settings:

Log Insights Settings

Configure the following settings to customize the log insights:

    • Full Details: All attribute details are displayed.
  • Save CSV Report: Select to save the CSV sync report to your Aquera dashboard.
