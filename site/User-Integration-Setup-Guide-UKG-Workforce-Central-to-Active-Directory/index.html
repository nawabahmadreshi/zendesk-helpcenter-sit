User Integration Setup Guide - UKG Workforce Central to Active Directory

Overview

This guide covers the end-to-end tasks involved in setting up synchronization to sync users in UKG Workforce Central with Active Directory.

The following tasks are involved in setting up the user synchronization from UKG Workforce Central to Active Directory:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running Synchronization

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Active Directory

Determine the architecture to be followed for setting up communication between Aquera and the Windows Active Directory Domain Controller. Refer to this Aquera guide for detailed information on the different architectural approaches.

Based on the selected approach, the following procedures may have to be performed:

  • A Windows Server must be ready to deploy Aquera Agent onto the server where Domain Controller is present. For instructions on installing Aquera Agent, refer to the Installation and Configuration Guide.
  • A firewall must be set up to allow communication with Aquera.
  • The static IP address of Aquera needs to be whitelisted. Contact the Aquera Support team.

To allow Aquera to make updates to Active Directory (for creating, updating, and deleting users), LDAPS communication to Active Directory needs to be set up, which requires a TLS connection. Active Directory Certificate Services must be enabled for this. To install Active Directory Certificate Services, see docs.microsoft.com.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • UKG Workforce Central. Refer to the UKG Workforce Central Configuration Guide for details.
  • Active Directory. Refer to the Active Directory Configuration Guide for details.

Synchronization Setup

This section details how to configure the User Synchronization to allow data push from UKG Workforce Central to Active Directory. 

It also describes the various analyses and the important points to be considered before setting up the user synchronization.

Considerations

  • The naming convention for the User Principal Name (UPN) in the Active Directory
  • The naming convention for the SAM Account Name in the Active Directory
  • Fields currently populated in the Active Directory
  • Work email can be retrieved from the UKG Workforce Central system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from UKG Workforce Central with that in the Active Directory.

General Analysis Steps

Before setting up the user synchronization, the following analyses need to be done:

  • Fill out the mapping template attached with this guide for mapping UKG Workforce Central user attributes to Active Directory user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.
  • Expression language can be used in the mapping to create UPNs and SAM Account Names without duplicates or errors. In the expressions, you can use inbuilt methods such as toNormalizedDiacritics() and toSAMAccountName() for this purpose. You can also use the keywords such as %ATTEMPT, %USEQ, SEQUENCENUMBER to avoid duplicates. For details, refer to the Synchronization- Expressions Guide.
  • Define the logical OU mapping rules for user containers.

For detailed analyses of UKG Workforce Central users and the users/groups in Active Directory using scripts, refer to the Appendix.

Building Synchronization

This document describes how to set up the synchronization using the User Synchronization method.

  • User Synchronization:  In this synchronization, set UKG Workforce Central as the Source Application and Active Directory as the Target Application. Refer to the User Synchronization Setup Guide for details on the configurations.

You can set the following configurations in the User Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables to create lookup tables based on location and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list to scope the groups Aquera would manage.

Refer to the User Synchronization Setup Guide for details on User Synchronization configurations.

Synchronization Scenarios

This section details the Synchronization scenarios.

Creating a user of type NORMAL_ACCOUNT in the Active Directory

To create a user of type NORMAL_ACCOUNT in Active Directory, set the attributes mapping as follows:

Enter "512" as the source attribute and map it to the [IAN].userAccountControl attribute of AD. Set the mapping criterion as Map on Create.

Note: 512 is the decimal value corresponding to the NORMAL_ACCOUNT property flag in Active Directory. For all property flags and the corresponding decimal values, refer to the Microsoft documentation.

Escaping Special Characters in OUs and DNs for Active Directory

To escape any special characters in OUs and DNs in the expressions, use a double-backslash before the special character. For example, if the OU name is #Terminated, mention it in the expression as \\#Terminated. 

DEST[IAN].distinguishedName.split(",")[0] + ",OU=\\#Terminated, DC=local"

Prompting for Password Change at Login

To make it mandatory for the new users in Active Directory to change their password on their first login, map the Active Directory attribute pwdLastSet to "0" with the mapping option of Map on create only. If this mapping is not done and users sign into their account for the first time, it will not prompt them to change their password.

To prompt the existing users also for a password reset, set the mapping option as Map on Create and Update.

Requirements to set user password

Active Directory requires a secure LDAPS (LDAP over SSL) connection to set or change user passwords. This is an Active Directory’s security requirement. Random generation of passwords when creating new users does not work without using LDAPS. Also, setting the password through attribute mapping requires a secure LDAPS connection.

Note: If the connection is not secured (non-SSL), Active Directory does not allow you to update the user password. 

Active Directory imposes strength requirements on the password, and the requirements vary from one instance to another.

Running Synchronization

  • Enter the matching expression to match the source records with the destination records.
  • Run the synchronization in Report Only mode and determine if the correct data is updated in the Active Directory. If the data is not correctly updated, change the matching expression to match users correctly and/or update the data, for example, employeeNumber, in the Active Directory.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Based on your analysis and considerations,
    • Translate the logical maps to the Aquera attribute maps for the fields
    • Translate the logical OU/container locations into tables and expressions
    • Translate the logical group assignment rules into Aquera rules
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. If errors are encountered, modify the synchronization configurations to fix them.
  • Once all users are synced, user creations and updates are based on the incremental changes in UKG Workforce Central.

Analyzing the Users in Active Directory

Use the following sample script in a script synchronization to generate a report from Active Directory to analyse user domain, status, etc.

log.info('AD ID,AD ACTIVE,AD EMPNO,AD GIVENNAME,AD FAMILYNAME,AD USERNAME,AD SAMACCOUNT,AD DN');
for (let user of AD.getUsers()) {
    log.info(
        '"' + user.id + '","' +
        user.active + '","' +
        (user[ENT] && user[ENT].employeeNumber || '') + '","' +
        user.name.givenName + '","' +
        user.name.familyName + '","' +
        user.userName + '","' +
        user[IAN].sAMAccountName + '","' +
        user[IAN].distinguishedName.split(',').slice(1).join(',') + '"'
    );
}
