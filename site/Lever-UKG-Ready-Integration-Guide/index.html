Lever / UKG Ready Integration Guide

← Back to index

This guide applies to the latest public version of the integration (version 1.3.0).

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration automates the synchronization of newly hired candidate information, including basic details, base compensation, and pay information, from Lever to UKG Ready. The integration ensures that candidate data captured in Lever is converted into new‑hire or employee records in UKG Ready.

This guide covers the end-to-end tasks in setting up the integration to sync the newly hired candidate information from Lever to UKG Ready.

Key Functions

The Lever to UKG Ready integrations provides the following functions:

  • Synchronizes newly hired candidate information (basic details, base compensation, and pay data) from Lever to UKG Ready.
  • Converts Lever candidate records into new‑hire or employee records in UKG Ready.
  • Whenever a new candidate is hired in Lever, an event triggers the automatic transfer of their data to UKG Ready.
  • Handles successful events by logging details, generating CSV reports, sending notification emails, and updating log insights data, if enabled.
  • Handles failure events by logging errors, creating failure-specific CSV reports, sending failure notifications, and updating dashboard analytics, if enabled.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Lever: Refer to the Lever Configuration Guide for details.
    • UKG Ready: Refer to the UKG Ready Configuration Guide for details.
  • Set up Webhooks in Lever: Your Lever admin must configure webhooks in the Lever application to trigger events when a new candidate is added. For details on the webhooks setup, refer to the Set Up Webhooks in Lever guide.
  • Configure Webhook Signature Token in the Lever Connector: Configure the signature token generated in your Lever application after configuring the webhook in the CANDIDATE HIRED category. For details on the webhook signature token in the Lever application, refer to the Set Up Webhooks in Lever. For configuring the Webhook Signature Token For Candidate hired Event tenant attribute in the connector, refer to the Lever Configuration Guide.

Integration Setup

This section outlines how the integration must be configured to push newly hired candidate data from Lever to UKG Ready.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Lever / UKG Ready in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select your Lever and UKG Ready connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Create Candidate Event: Select the event that will trigger the synchronization of new candidate data from Lever to UKG Ready. For example, select Candidate Hired from the dropdown menu to synchronize newly hired candidate data.
  • UKG Ready Company Name: Enter the short name of the company configured in your UKG Ready instance. Refer to the UKG Ready Configuration Guide for more details.
    Note: This is a required field.
  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Report Generation: Select this checkbox to generate the integration report.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications. When enabled, the Notification Settings section will be displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Analytics Settings section will be displayed for further configuration.
  • Create Base Compensation: Select this checkbox to include base compensation details during employee creation in UKG Ready.
  • Create Employee Pay Information: Select this checkbox to include employee pay information during employee creation in UKG Ready.
  • Support Auto-Generated Employee ID Sequence: Select this checkbox to automatically assign a sequential employee ID when creating an employee in UKG Ready. When selected, the system automatically generates a unique, sequential employee ID for each new employee. If this checkbox is not selected, the employee ID is determined based on the attribute mapping configured in the Attribute Map tab.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Log Custom Mode drop-down menu with the following options:
      • Inclusion: Select the attributes to be included in the report. Choosing this option will display the Log Inclusion Attributes menu, where you can add the desired attributes.
      • Exclusion: Select the attributes to be excluded from the report. Choosing this option will display the Log Exclusion Attributes menu, where you can specify the attributes to exclude.
  • Show attribute values for each event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.

Report Settings

Configure the following report generation settings:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Report Custom Mode drop-down menu with the following options:
      • Inclusion: Select the attributes to be included in the report. Choosing this option will display the Report Inclusion Attributes menu, where you can add the desired attributes.
      • Exclusion: Select the attributes to be excluded from the report. Choosing this option will display the Report Exclusion Attributes menu, where you can specify the attributes to exclude.
  • Enable Integration Report Download: Select this checkbox to allow downloading the integration report in CSV format.
  • Show attribute values for each event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

Notification Settings

Configure the following settings for email notifications:

  • Recipients' Email Addresses: Enter the comma-separated email addresses of the recipients to whom the email notifications will be sent.
  • Success Email Subject: Enter the content that must be used as the email subject for successful integration execution.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications for integration failure.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
  • Failure Email Subject: Enter the content that must be used as the email subject for integration failure.

Analytics Settings

Configure the following analytics settings:

  • Select the Analytics Detail Level: Select the desired level of detail from the dropdown menu on the log insights dashboard. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting this option will display the Analytics Custom Mode drop-down menu with the following options:
      • Inclusion: Select the attributes to be included in the report. Choosing this option will display the Analytics Inclusion Attributes menu, where you can add the desired attributes.
      • Exclusion: Select the attributes to be excluded from the report. Choosing this option will display the Analytics Exclusion Attributes menu, where you can specify the attributes to exclude.
  • Show attribute values for each event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Event

The Event tab is used to configure the webhook details. 

Provide the following details and save the changes:

  • Event Source: Select your Lever application from which candidate-hiring events are triggered.
  • URL: Copy this URL and paste it into the appropriate webhook field on the Lever Webhooks setup page. For details on setting up a webhook in the Lever application, refer to the Set Up Webhooks in Lever guide.
  • Create Token: Do not generate a token, as it is not required or used in the configuration.
