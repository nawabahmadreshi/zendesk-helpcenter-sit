UKG Ready to Okta Reconciliation

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

The integration between UKG Ready Employees and Okta Users automates the management of user accounts in Okta based on the employee lifecycle events, such as hiring, re-hiring, and termination, recorded in the UKG Ready. It continuously synchronizes employment status changes, ensuring that Okta accounts are activated or deactivated as needed.

It does not create new Okta accounts, but it exclusively manages the activation and deactivation of existing users in response to the lifecycle events.

This integration performs the following tasks:

  • Retrieves employee data from UKG Ready using a specified report ID and company information.
  • Fetches all user accounts from Okta, including both active and de-provisioned users.
  • Matches Okta users to UKG employees using the employee ID as a unique identifier.
  • For each matched user:
  • Determines if the user should be activated (hired or re-hired) or deactivated (terminated) in Okta, based on employment dates and time zone.
  • Schedules activation at the start of the hire/rehire date and deactivation at the end of the termination date, respecting the employee's local time zone.
  • Identifies Okta users who are not found in UKG Ready and flags them for deactivation (configurable via parameters).
  • Supports exemptions for specific employee IDs, limiting or preventing deactivation for those users.
  • Optionally limits the number of deactivations per run, based on configuration settings.
  • Generates a detailed CSV report summarizing all actions taken (activated, deactivated, skipped, not found, etc.), including updated Okta statuses, if configured.
  • Sends the integration report via email to designated recipients if email settings are provided.

Integration Setup

This section explains how to configure the integration between UKG Ready and Okta for automated user lifecycle management. It covers creating the integration, setting up parameters, and the process for enabling scheduled synchronization of employee status changes from UKG Ready to Okta.

The integration activates existing Okta users at the beginning of their hire or rehire date and deactivates them at the end of their termination date, using employee data from UKG Ready. The integration determines activation and deactivation timing for each user according to their employment dates in UKG Ready, ensuring these actions occur at the correct local time for the employee’s assigned time zone.

Create Integration

Follow these steps to configure the UKG Ready to Okta integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to the Integrations section in the left panel.
  3. Click Create Integration.
  4. Select UKG Ready to Okta Reconciliation.
  5. Enter a name and description for the integration.
  6. Select your UKG Ready and Okta connector instances.
  7. Click Continue to proceed to the configuration.

Set Up Parameters

The Parameters section contains essential parameters that determine how the integration operates. Each setting is crucial in customizing the integration's behavior to meet your organization's needs.

  • Company Name or Company ID: Refers to the name or unique identifier of the company in UKG Ready, which is used for retrieving employee and time zone data.
  • Employee IDs Excluded from Deactivation: Specifies the list of employee IDs that will not be deactivated in Okta, even if they are marked as terminated in UKG Ready.
  • Deactivation Limits: Provides a restriction on the number of user deactivations allowed in a single integration run as a safety measure.
  • Maximum Allowable Deactivations: Sets the maximum number of user deactivations permitted per integration run when Deactivation Limits are enabled.
  • Enable Integration Report Download: Allows downloading the integration report in CSV format.
  • Report Only:  Generates a report outlining the data changes that would occur without implementing them.
  • Report ID: The unique identifier of the UKG Ready report used to fetch employee data.
  • Recipients' Email Addresses: The email addresses of the recipients to whom the email notifications will be sent.
  • Deactivate Missing Users: Deactivates the Okta users not found in the latest UKG Ready report.
  • Email Subject: The subject line used for all integration report emails.
  • Email Body: The message content used as the body for all integration report emails.

Attribute Mapping

The integration retrieves employee data from UKG Ready by specifying a report ID and company information. It also collects all user accounts from Okta, including those that are active or de-provisioned, ensuring a proper view of user and employee statuses.

The integration uses the employee ID as the unique identifier to match Okta users with UKG employees. This comparison ensures accurate association between user records. Using this mapping, user statuses in Okta are synchronized with the latest employment data from UKG Ready.

  • For each matched user, the integration evaluates key employment events recorded in UKG Ready, such as Hire and Rehire.
  • Activation in Okta is scheduled at the start of the hire or rehire date, based on the employee’s local time zone.
  • Deactivation in Okta is scheduled at the end of the termination date, also according to the employee’s local time zone.
  • The Time Zone is considered to ensure that granting or revoking access occurs at the appropriate local time for each user.
  • Users present in Okta but not found in the UKG Ready are identified and may be flagged for deactivation, depending on configuration settings. The integration supports exemptions for specific employee IDs, allowing administrators to limit or prevent deactivation for designated users. 

Run the Integration

The integration can be scheduled to run automatically at customizable intervals. Available frequencies include 15 minutes, 30 minutes, 1 hour, 4 hours, 6 hours, 12 hours, or 24 hours. You can also enable recurring executions to run the integration continuously on your chosen schedule.

On successful execution, the CSV file will be sent to the specified recipients. In case of integration errors, notifications and reports will also be sent to the email addresses.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
