User Integration Setup Guide - Oracle HCM Cloud to Okta

← Back to index

Overview

This guide covers the end-to-end tasks involved in setting up integration to sync users in Oracle HCM Cloud with Okta.

The following tasks are involved in setting up the integration from Oracle HCM Cloud to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Integration Setup
  • Running Integration

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • Oracle HCM Cloud. Refer to the Oracle HCM Cloud Configuration Guide for details.
  • Okta. Refer to the Okta Configuration Guide for details.

Integration Setup

This section details how the integration must be configured to allow data push from Oracle HCM Cloud to Okta. 

Considerations

  • Fields currently populated in the Okta
  • The criteria for matching a user record from Oracle HCM Cloud with that in the Okta.
  • Work email can be retrieved from the Oracle HCM Cloud system or be generated by Aquera integration and updated back to the system.
  • The criteria for matching a user record from Oracle HCM Cloud with that in the Okta.

General Analysis Steps

Before setting up the integration, the following analyses need to be done:

  • Fill out the attached mapping template for mapping the Oracle HCM Cloud user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.
  • You can analyze the users in Oracle HCM Cloud as well as in Okta using one of the following ways:
    • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
    • Using Script integration. For scripts, refer to the Appendix.

Building Integration

This document describes how to set up the integration using the User Integration method.

  • User Integration - In this integration, set Oracle HCM Cloud as the Source Application and Okta as the Target Application. Refer to the User Integration Setup Guide for details on the configurations.

You can set the following configurations in the User Integration:

  • Set the Admin Email Options to receive email notifications for success/failures during the integration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the User Integration Setup Guide for details on User Integration configurations.

Running Integration

  • Enter the matching expression to match the source records with the destination records.
  • Run the integration in Report Only mode and determine if the correct data is updated in the Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the integration logs thoroughly to ensure that no errors are encountered. Modify the integration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in Oracle HCM Cloud.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow integration to update users, you need to disable the setting that sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting which sets Active Directory as profile master.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in Oracle HCM Cloud and Okta before setting up the integration. 

This document explains how analyses can be done using a script integration. In this integration, add Oracle HCM Cloud and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script integration, refer to the Integration Using Scripts Guide.

A few sample scripts are provided here for your reference. The scripts may vary based on your use case and the attributes to be retrieved for analysis.

 

Analyzing the Users in Oracle HCM Cloud

Use the below sample script in a script integration for generating a report from Oracle HCM Cloud for analysis of user status, department, usertype, and so on. 

log.info('OHCM ID, OHCM ACTIVE, OHCM GIVENNAME, OHCM FAMILYNAME, OHCM USERNAME, OHCM MIDDLENAME, OHCM TITLE, OHCM DEPARTMENT, OHCM DIVISION, OHCM USERTYPE');
let users = oracle.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}

 

Analyzing the Users in Okta

Use the below sample script in a script integration for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}
