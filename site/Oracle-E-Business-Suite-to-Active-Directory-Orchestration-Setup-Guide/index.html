Oracle E-Business Suite to Active Directory Orchestration Setup Guide

← Back to index

Overview

This guide covers the end to end tasks involved in setting up an orchestration to sync users in Oracle E-Business Suite with Active Directory.

The following tasks are involved in setting up the orchestration from Oracle E-Business Suite to Active Directory:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Orchestration Setup
  • Running Orchestration

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Active Directory

Determine the architecture to be followed for setting up communication between Aquera and the Windows Active Directory Domain Controller. Refer to this Aquera guide for detail information on the different architecture approaches.

Based on the selected approach, the following procedures may have to be performed:

  • A Windows Server must be ready to deploy Aquera Agent on to the server where Domain Controller is present. For instructions on installing Aquera Agent, refer to the Installation and Configuration Guide.
  • Firewall must be set up to allow communication with Aquera.
  • The static IP address of Aquera needs to be whitelisted. Contact Aquera Support team.

To allow Aquera to make updates to Active Directory (for creating, updating, and deleting users), LDAPS communication to Active Directory needs to be set up, which requires a TLS connection. Active Directory Certificate Services must be enabled for this. To install Active Directory Certificate Services, see docs.microsoft.com.

Configuring Applications in Aquera Admin

The following applications must be added and configured in Aquera Admin console:

  • Oracle E-Business Suite. Refer to the Oracle E-Business Suite Configuration Guide for details.
  • Active Directory. Refer to the Active Directory Configuration Guide for details.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from Oracle E-Business Suite to Active Directory. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Considerations

  • Naming convention for the User Principal Name (UPN) in the Active Directory
  • Naming convention for the SAM Account Name in the Active Directory
  • Fields currently populated in the Active Directory
  • The criteria for matching a user record from Oracle E-Business Suite with that in the Active Directory.

General Analysis Steps

Before setting up the orchestration, the following analyses need to be done:

  • Fill out the attached mapping template for mapping the Oracle E-Business Suite user attributes to Active Directory user attributes.
  • Expression language can be used in the mapping to create UPNs and SAM Account Names without duplicates or errors. In the expressions, you can use the inbuilt methods such as toNormalizedDiacritics() and toSAMAccountName() for this purpose. You can also use the keywords such as %ATTEMPT, %USEQ, SEQUENCENUMBER to avoid duplicates. 

    For details, refer to the Orchestration - Expressions Guide.

  • Define the logical OU mapping rules for user containers.

For detail analyses of the Oracle E-Business Suite users and the users/groups in Active Directory using scripts, refer to the Appendix.

You can also analyze the users data using the Reports option in the Data tab of the Applications page in Aquera Admin console. Refer to the User Reports Guide for details.

Building Orchestration

This document describes how to set up the orchestration using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set Oracle E-Business Suite as the Source Application and Active Directory as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Note: 

Any custom fields added by you in the Active Directory are not displayed automatically on the Attribute Map tab. You must manually add those custom attributes using the Add Row to Map option on the Attribute Map page and then complete the mapping.

Escaping Special Characters in OUs and DNs for Active Directory

To escape any special characters in OUs and DNs in the expressions, use double-backslash before the special character. For example, if the OU name is #Terminated, mention it in the expression as \\#Terminated. 

DEST[IAN].distinguishedName.split(",")[0] + ",OU=\\#Terminated, DC=local"

Double-backslash must be introduced before any of the following special characters:

Comma ,
Backslash character \
Pound sign (hash sign) #
Plus sign +
Less than symbol <
Greater than symbol >
Semicolon ;
Double quote (quotation mark) "
Equal sign =
Leading or trailing spaces  

Running Orchestration

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if the correct data is updated in the Active Directory. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data, for example, employeeNumber, in the Active Directory.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Based on your analysis and considerations,
    • Translate the logical maps to the Aquera attribute maps for the fields
    • Translate the logical OU/container locations into tables and expressions
    • Translate the logical group assignment rules into Aquera rules
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in Oracle E-Business Suite.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in Oracle E-Business Suite and Active Directory before setting up the orchestration. It also provides the script to analyze the existing groups of the users in Active Directory.

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add Oracle E-Business Suite and Active Directory applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

A few sample scripts are provided here for your reference. The scripts may vary based on your use case and the attributes to be retrieved for analysis.

Analyzing the Users in Oracle E-Business Suite 

Use the below sample script in a scheduled script orchestration for generating a report from Oracle E-Business Suite for analysis of user status, department, usertype, and so on. 

log.info('OHCM ID, OHCM ACTIVE, OHCM GIVENNAME, OHCM FAMILYNAME, OHCM USERNAME, OHCM MIDDLENAME, OHCM TITLE, OHCM DEPARTMENT, OHCM DIVISION, OHCM USERTYPE');
let users = oracle.importAllUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}

 

Analyzing the Users in Active Directory

Use the below sample script in a script orchestration for generating a report from Active Directory for analysis of user domain, status, and so on.

log.info('AD ID,AD ACTIVE,AD EMPNO,AD GIVENNAME,AD FAMILYNAME,AD USERNAME,AD SAMACCOUNT,AD DN');
let users = AD.importAllUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user[IAN].sAMAccountName + '","'
+ user[IAN].distinguishedName.split(',').slice(1).join(',') + '"');
}

Analyzing the Groups in Active Directory

Use the below script to analyze the groups of existing users in the Active Directory.

let ad_users = AD.importAllUsers();
let ad_users_length = ad_users.length;
let i = 0;
let total = ad_users_length;
log.info('AD ID,GROUP NAME');
for (i = 0; i < total; i++) {
let groups = ad_users[i].groups;
if (groups) {
for (let group of groups) {
log.info(ad_users[i].userName + ',' + group.display);
}
}
