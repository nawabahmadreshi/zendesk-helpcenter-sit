Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, job requisition data is synchronized between ADP Workforce Now and iCIMS. The integration compares records from both applications, then creates, updates, or skips job requisitions based on the configured requirements.

To identify user records across both systems, the integration uses the code attribute from ADP Workforce Now and matches it to the jobnumber in iCIMS as a correlation attribute. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • iCIMS: Refer to the iCIMS Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push Job Requisition data from ADP Workforce Now to iCIMS.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to iCIMS Job Requisition Sync.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and iCIMS connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable Organization Data Sync: Select this checkbox to enable the organizational objects before running the sync.
    Note: The Manage Jobs section will be displayed only when this checkbox is selected.
  • Organization Sync List: Select the organization sync list from the dropdown menu. The available option is Job Requisition Sync.

General Settings

Configure the following general settings:

  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notifications section will be displayed only when this checkbox is selected.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.
    Note: The Debug Mode section will be displayed only when this checkbox is selected.

Manage Jobs

The Manage Jobs section enables you to customize the integration settings according to your specific use case. It has the following settings:

Correlation Settings

Configure the following correlation settings:

  • Correlation Attribute for Source: Select the ADP Workforce Now correlation attribute to identify the associated iCIMS record. For example, select code.
  • Correlation Attribute for Target: Select the iCIMS correlation attribute to identify the associated ADP Workforce Now record. For example, select response.jobnumber.

Source Filter Settings

Configure the following source filter settings:

  • Source Filter: Select the Jobs filter to filter the Job while retrieving from ADP Workforce Now.
  • Source Filter Value: Provide the value for the selected source filter. For example, Active and Inactive. 

Target Filter Settings

Configure the following target filter settings:

  • Enable Inactive Job Creation: Select this checkbox to enable inactive job creation. If this checkbox is not selected, inactive jobs are skipped, and only jobs with an active status are created.
  • Target Filter Value: Provide the value for the selected source filter. For example, Approved, Not Approved, Pending Approval, Hold, Forecasted, Canceled, Closed, Active Job Templates, and Inactive Job Templates.

Threshold

Configure the following threshold settings:

  • Create Job Limit: Specify the maximum number of jobs that can be created in the target application during a single integration run.
  • Update Job Limit: Specify the maximum number of Jobs that can be updated in the target application during a single integration run.

Debug Mode

The Debug Mode section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Select Filter: Select the Job ID filter to filter the Job ID while retrieving from ADP Workforce Now. 
  • Job ID: Enter the Job ID. For example, DIRBILRC.

Logging

The Logging section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the log.
      • Excluded Attributes: Select the attributes to be excluded from the log.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.

Reporting

The Reporting section enables you to customize the integration settings according to your specific use case. This section is used to download the integration report in CSV format from the execution timeline. It has the following settings:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.
  • Store Report Data: Select this checkbox to store report data.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.
  • Enable Integration Report Download: Select this checkbox  to allow downloading the integration report.

Notifications

The Notifications section enables you to customize the email notification settings according to your specific use case. It has the following settings:

  • Select the Notification Report Detail Level: Select the desired notification report detail level from the dropdown menu. This section is used to attach the integration report in CSV format to the email. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Attribute Filter drop-down menu with the following options:
      • Included Attributes: Select the attributes to be included in the notification report.
      • Excluded Attributes: Select the attributes to be excluded from the notification report.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details or Custom option.
  • Notify Admin Via Email: Check to send email notifications to all admins.
  • Notify Employees via Email: Check to send email notifications to all employees.

Administrator Notification Email Settings

Configure the following administrator notification email settings:

  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent
