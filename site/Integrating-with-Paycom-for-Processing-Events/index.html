Integrating with Paycom for Processing Events

← Back to index

Overview

Paycom provides support for webhooks to automatically send a data payload containing event details from the Paycom application to a third-party system. Paycom events may include creating new hires, creating employees, modifying employee data, and more.

When the Paycom webhooks are configured to send event notifications to the Aquera integration, Aquera can receive the Paycom events in real time and process them for synchronization with identity management systems and other applications.

This guide provides the details of the Paycom events and how to set up the Aquera integration to receive and process the Paycom events.

Paycom Webhooks

Paycom can send notifications via webhooks for the following events:

  • New hire created
  • New hire deleted
  • New hire self-onboarding completed 
  • Employee added
  • Employee photo added
  • Employee changes - Refer to the following section for the list of employee changes for which an event notification will be sent.

Prerequisite

You need to add and configure the Paycom version 5 connector in the Aquera Admin portal. Refer to the Paycom Configuration Guide for details.

Aquera Integration Setup

You need to set up an integration in the Aquera Admin portal to receive and process the Paycom events.

Perform the following steps to create an integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronizations on the left panel and click Create Synchronization.
  3. Select Integration.
  4. Enter a name and a description for the integration.
  5. Click Continue. A new page is displayed to configure the integration.
  6. Click Create to create the integration.

In the Configuration panel, you must add the Paycom connector to be used in the integration. Perform the following steps to add the Paycom connector:

  1. Click the + icon next to the Connectors to display the Add Connector window.
  2. Select the Paycom connector from the Connector drop-down.
  3. Enter a unique variable name for the connector in the Variable field. This name will be used to refer to the connector in the script.
  4. Click the Add button to add the Paycom connector.

You must configure a trigger to run the integration when event notifications are sent from the Paycom application using Paycom webhooks.

Perform the following steps to configure the trigger:

  1. Click the + icon next to the Trigger to display the Add Trigger window.
  2. Select the Trigger Type as Event.
  3. Select the Paycom connector from the Event Source drop-down list. A URL will be generated for the integration. This is the endpoint URL to which the configured events are sent from the Paycom application for processing.
  4. Copy the URL and paste it into the webhook setup of the Paycom application. 
  5. Click Create Token to generate a token for authentication.

  6. Copy this token by clicking the copy icon next to the generated token. The token must be securely stored and configured in Paycom's webhook setup for authentication purposes.
  7. Click Save to save the event trigger setup.

In the Script panel, you can write JavaScript to process the Paycom events and synchronize them to any identity management systems.

You can log the Paycom event request using the following format:

log.info(JSON.stringify(request));

For details on the functions supported in the script and how to work with the scripts, refer to the Working With Integration Scripts guide.
