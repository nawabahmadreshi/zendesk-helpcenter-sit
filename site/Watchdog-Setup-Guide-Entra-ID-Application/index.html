Watchdog Setup Guide: Entra ID Application

← Back to index

Overview

Aquera Watchdog is a monitoring and validation tool designed to identify data synchronization issues between integrated applications. It operates independently of the Aquera integration and analyzes application events to detect inconsistencies between source and target systems.

After an integration execution is completed, you can run the Watchdog to retrieve the event logs, analyze the activities, and report any anomalies or data discrepancies across the applications. For example, if a user is created in the source application, Watchdog checks for a corresponding user provisioning event in the target application and reports an anomaly if no matching event is found. Aquera Watchdog helps ensure guaranteed delivery of events across integrated systems by continuously monitoring and validating data synchronization between the source and target applications.

You can even run the Watchdog for a specific application. When you run the application Watchdog, a dashboard is generated with a consolidated view of the activities in the application. The dashboard can be accessed through the Insights module. 

Watchdog provides visibility into all the events happening in the application. 

This guide provides details on how to configure and run the Watchdog for the Entra ID application.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Microsoft 365 Application Connector in the Aquera Admin Portal: Ensure you add and configure the Microsoft 365 connector in the Aquera Admin portal to connect to the Entra ID application for retrieving the event logs. Refer to the Microsoft 365 Configuration Guide for details.

Watchdog Setup

This section describes how to configure the Watchdog that monitors the events in Entra ID.

Create Application Watchdog

Follow these steps to create the application watchdog:

  1. Log in to the Aquera Admin Portal.

  2. In the left panel, go to Integrations.

  3. Click Create Integration.

  4. Select Entra ID [Watchdog].

  5. Enter a name and description for the application watchdog.

  6. Choose your Microsoft 365 connector instance.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Microsoft 365 connector that you have selected while creating the application watchdog. This section enables you to search and select a different instance of the application connector, if required.

Parameters

The Parameters section enables you to customize the following Watchdog setting according to your use case:

  • Retrieve logs from: Select the date from which the event logs must be retrieved from Entra ID. These logs are used by Watchdog to analyze the events and create the dashboard.

Schedule

The application Watchdog can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details. 

Watchdog Insights

This section details the data covered in the dashboard created by the Watchdog for the Entra ID application. 

  1. Log in to the Aquera Admin portal.
  2. Navigate to Insights.
  3. Click on the application watchdog name, for example, Entra ID Watchdog. The dashboards are displayed.

The dashboard provides a comprehensive view of the events in Entra ID.
