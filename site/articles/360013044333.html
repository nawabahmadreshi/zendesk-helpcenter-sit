Overview

Duo Security is a multi-factor authentication and zero-trust access solution that helps protect applications and data from unauthorized access. It verifies user identity, device health, and security posture before granting access.

This guide helps users to create a Duo Security connector on the Aquera platform.

For further details on the Duo Security application, visit the Duo Security website and API documentation.

Supported Features

The Duo Security connector supports the following features:

Resource Import Provisioning
Users
  • Full import
  • Filter users
  • Create a user
  • Update a user
  • Delete a user
  • Activate or Deactivate a user
Groups
  • Full import
  • Create a group
  • Update a group
  • Delete a group

Prerequisite

You must have the admin credentials for your Duo Security instance.

Connector Configuration

Perform the following steps to configure the Duo Security connector:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Applications and click Add Application.
  3. Search for Duo Security and click the Duo Security icon.
  4. Enter the values for the following fields under the Configuration tab:
    • Basic Details:
      • Display Name: Enter a display name for your application.
      • Description: Enter a description for your application.
      • Tenant: Enter the API hostname from your Duo Security console. For example, if the console shows api-197d58b8.duosecurity.com, configure the API hostname as api-197d58b8.duosecurity.com. Refer to the Setting Up Duo Security Connector section for details on obtaining the API hostname.
      • Source Owner: Enter valid email IDs of the source owners.
      • Subscribe to error notification: Enable the toggle button to send notification emails to source owners for errors in the user data transactions. Click Edit Preference to select the error types for which you want to receive notifications. After you configure your preference, you will receive error notifications only for the selected error types. By default, all error types are selected.
      • Pick your protocol: Select the protocol from the dropdown list based on your requirements. SCIM is the default value.
      • Copy this URL: This field displays the base URL of the connector API endpoint. The URL is populated based on your selected protocol. Copy and paste this URL while provisioning to the identity service provider or performing OpenAPI operations, if supported.
    • Tenant Attributes: Refer to the Aquera Tenant Attributes section for details.
  5. Click Create to create the Duo Security connector.

Refer to the Getting Started with Connectors guide to learn more about connector setup, including different tabs on the connector configuration page.

Aquera Tenant Attributes

The following are the Aquera tenant attributes:

  • Delete the User on Deprovisioning: Select this checkbox to delete a user from the Duo Security application upon deprovisioning. If the checkbox is not selected, the user will not be deleted upon deactivation.
  • Hard delete users: Select this checkbox to permanently delete a user from Duo Security. If the checkbox is not selected while deleting users, the users will be moved to the trash.

Authentication

The Duo Security connector supports Basic Authentication. Provide the Integration Key as the username and the Secret Key as the password for authentication.

Refer to the Setting Up DuoSecurity Connector section for details on how to obtain the integration key and secret key.

Setting Up Duo Security Connector

To set up the Duo Security connector, log in to the Duo Admin Console and add the Admin API to the Protected Applications. After this step, you will be able to access the tenant, username, and password required for the connector setup. Before proceeding, you must send an email to support@duosecurity.com requesting that the Admin API be enabled for your tenant.

Perform the following steps to set up the Duo Security connector:

  1. Log in to the Duo Security Console.
  2. Navigate to the Applications tab and click Protect an Application.
  3. Search for Admin API and select it.

  4. Under Settings, select the Grant read resource and Grant write resource checkboxes.
    Note: If the Admin API is not available, contact the Duo Security Support Team. Refer to the Admin API documentation.

  5. Click Save.
  6. Once the Admin API is enabled, you can copy the Integration key and Secret key.
    Note: The API hostname displayed on this screen must be configured as the Tenant in the Aquera Admin Console.
