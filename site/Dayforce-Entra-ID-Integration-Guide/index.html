Dayforce / Entra ID Integration Guide

← Back to index

This guide applies to the latest public version of the integration (version 1.2.7).

Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration connects Dayforce with Microsoft Entra ID to automate end-to-end user lifecycle management.

This guide explains how to set up the Dayforce / Entra ID integration.

Key Functions

The Dayforce / Entra ID integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes employee records from Dayforce to Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.
  • Creates new users in Entra ID when they are added in Dayforce.
  • Updates existing Entra ID users when their information changes in Dayforce.
  • Deactivates Entra ID users when they are no longer active in Dayforce.
  • Generates passwords for newly created users in Entra ID.
  • Manages group assignments in Entra ID.
  • Supports integration hooks for user creation, updates, and deprovisioning.
  • Sends email notifications for provisioning events.
  • Generates and sends reports via email.
  • Supports a report-only mode for validation without making changes.
  • Supports monitored provisioning.

    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Dayforce: Ensure that you configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for details.
    • Microsoft Entra Native: Ensure that you add and configure the Entra ID connector in the Aquera Admin portal. Refer to the Microsoft Entra Native Configuration Guide for details.

Integration Setup

This section details the configuration and mapping process for integrating Dayforce with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped from Dayforce to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Entra ID in the search box next to the Explore Templates section, and select it. The Create New Integration pop-up window appears.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The Configuration tab includes a list of configurations required to set up the integration. Refer to the Integration Configurations Reference Guide for detailed information on each configuration field.

Mapping and Assignments

The Mapping and Assignments tab provides configuration options for defining user naming attributes and mapping attributes between Dayforce and Entra ID.

User Naming Attributes

In Entra ID, the user naming attributes are User Principal Name (UPN) and Work Mail. These attributes are used to identify user objects in Entra ID.

  • UPN: The User Principal Name (UPN) attribute is the user logon name (ULN) that is used to log in to a domain. By convention, UPN should map to the user's email address.
    A UPN consists of a prefix (the user account name) and a suffix (a DNS domain name), separated by the @ symbol. The UPN format is username@DomainName. For example, barbarajensen@aquera.com.
  • Mail: The Mail attribute is the user's email address, often set to match the UPN. The mail should be unique within the same domain. The mail attribute typically follows standard email formatting. For example, barbarajensen@example.com.

This integration enables you to generate unique names for these attributes while creating a user in Entra ID. The following configurations are available on this page:

  • Suffix
  • User Naming Configuration

Suffix

This section enables you to configure the domain name for UPN and work mail as follows:

  • UPN/ULN suffix: Enter the domain name to be used as the suffix in the UPN/ULN. You can configure a static domain name (for example, aquera.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional UPN/ULN Suffix checkbox and set conditional logic in the Conditional UPN/ ULN suffix field to assign domain names to users accordingly. 

    The Conditional UPN/ ULN suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate domain names during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

  • Work Mail suffix: Enter the domain name to be used as the suffix for the work email address. You can configure a static domain name (for example, aquera1.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional Work Mail Suffix checkbox and set the conditional logic in the Conditional Work Mail suffix field.


    The Conditional Work Mail suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate email domain names during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    If the work email suffix has to match the UPN suffix, select the Work Mail Suffix same as UPN/ULN Suffix checkbox.

User Naming Configuration

This section allows you to define the formats to be used to generate values for the UPN and Mail attributes of newly created users in Entra ID.

The following configuration modes are available:

  • Basic User Naming Configuration: This configuration is enabled by default. In this mode, select a naming format from the ULN/ Mail Format dropdown list that must be used for generating the UPN and Work Mail values for a new user in Entra ID. The selected format will be used to generate unique names for both attributes.

    In this mode, the UPN and Work Mail attributes are mutually dependent and are evaluated together. This means the integration will verify whether the values generated using the selected format are available for both attributes before assigning them to the new user. If a value is already in use for an attribute (that means, it is already assigned to a user in Entra ID), then the integration will append a number (starting from 1) to the attribute values and check for the availability. This process continues for up to 20 attempts or until unique values are generated for both attributes. Once unique values are found for the attributes, those values will be assigned to the new user.

  • Advanced User Naming Configuration: Select the Advanced User Naming Configuration checkbox to switch to the advanced setup mode. In this mode, you can set different formats for each of the user naming attributes.

    When you switch to the advanced mode, you can select whether the work mail should be mutually dependent with the UPN/ULN in the Mutually Dependant Attributes with UPN/ ULN field.

    If you set UPN and Work Mail as mutually dependent attributes, the integration will check for a unique value in the first format for both attributes. Both attributes are evaluated together. If a unique value is unavailable (already assigned to another user), it will proceed to the next format in sequence for both attributes. The process continues until it finds available values for both attributes to assign to a user. If the integration cannot find unique values, the integration will return an error.
    If UPN and Work Mail are not set as mutually dependent attributes, the integration evaluates each attribute separately. If the integration finds a unique value for even one of the attributes as defined by a format, that value will be assigned to the user.

    Define the user naming formats in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields.
    These fields are AI Prompt fields that use Aquera Intelligence (AI Assistant) to generate UPN and work mail values during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    When you add a format in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields, you must also specify when the specified format should be applied. You can choose to apply it On Create, On Update, or On Create & Update.

Attribute Map

Navigate to the Attribute Map section to perform attribute mapping. The Attribute Map section displays the default mapping from Dayforce Employee attributes to Entra ID user attributes.

Note: For details on the Dayforce attributes, refer to the Appendix.

Perform the following steps to add a new mapping from a Dayforce attribute to an Entra ID attribute:

  1. Click +Add Row at the end of the attribute mapping list.
  2. Click on the empty field on the source attribute side to include a Dayforce attribute.
  3. Select an attribute from the dropdown list to add it as the source attribute.
    Alternatively, you can write a JavaScript expression in the source field. Click the Enter Expression option to write the expression. The expression is evaluated at runtime and the resulting value is assigned to the corresponding target attribute.
    Note: Click the icon to expand the Expression editor area to write longer expressions.
  4. In the Attribute field on the target side, select the Entra ID attribute name from the drop-down list.
  5. Select the data type of the Entra ID attribute from the dropdown list.
  6. Select one of the following mapping options:
    • Map on Create and Update: The Entra ID attribute will be updated during the create and update operations.
    • Map on Create: The Entra ID attribute will be updated only during the create operation.
    • Map on Update: The Entra ID attribute will be updated only during the update operation.
    • Don't Map: The Entra ID attribute will never be updated and the mapping will be ignored during the integration runs.
  7. Once done, click the check mark to add the new mapping to the attribute map.

Preview Employee Data

You can preview the attribute values of a Dayforce employee that will be created or updated in Entra ID based on your mappings.

To preview the attribute values of a Dayforce employee, perform the following steps:

  1. Click the Enter objects for preview field in the top-right corner of the Attribute Map page. A pop-up window is displayed.
  2. Click the Searchable Attribute dropdown menu and select id.
  3. Enter the employee number (ID of the employee) in the Enter value field, and then click Preview.

    The employee data will be displayed corresponding to each mapping provided on the attribute mapper.

    Notes:
    • The toggle button next to the preview setup field will be enabled by default. You can disable the toggle option to hide the preview.
    • When the attributes or expressions provided in the source attributes column are invalid or do not return a value, the preview will not appear.

Group Management

The Group Management tab provides options to configure how group memberships must be handled through this integration. You can determine the group assignment, retention, and removal of users based on the update or deactivation of users in Dayforce.

Note: You can set rules for assigning users to the Entra ID groups through either the Group Rules section of the Group Management tab or the Attribute Map section of the Mapping and Assignments tab. When you use the Attribute Map functionality to set the group rules, you have to write JavaScript conditional expressions to assign the groups. 

Select the Enable Group Management checkbox to configure options for managing user groups. If this checkbox is not selected, the integration does not manage group assignments to users in Entra ID.

When you select the Enable Group Management checkbox, the following sections are displayed for further configuration:

  • Group Rules
  • Group Retention

Group Rules

In the Group Rules section, you can create conditions that control how users are added to groups. Each rule defines the conditions for assigning users to a set of groups.

Perform the following procedure to create group assignment rules:

  1. Click Choose Attribute to view the list of Dayforce attributes that you can use to set conditions.
  2. Search and select the attribute you want to use to define a condition.
  3. Enter a value for the selected attribute. You can configure multiple values for the selected attribute. If multiple values are provided, an OR condition is applied among those values. For example, in the following sample condition, a user's title must be either Manager or Developer to satisfy this group assignment rule.
  4. In the Assign To column, select the groups to which users who satisfy the specified condition must be assigned.
  5. If required to add another condition and assignment for the same selected attribute, click Add Row and provide a value. You can add any number of rows within a rule.
    Note: You can delete a row by using the delete icon at the right end of the row.
  6. [Optional] If you want to write a JavaScript expression to set the condition, click Switch to Expression. and perform the following steps:
    Note: When you switch to Expression mode or back to Condition Builder mode, all configured conditions in the rule will be cleared.
    1. Select one of the following based on your requirement:
      • For Conditions Only: Select this if you want to write a JavaScript expression only for the conditions.
      • For Conditions and Assignments: Select this if you want to write an expression for both conditions and assignments.
    2. Click Continue to configure the expression.

    3. Enter the expression based on your selection:

      • If For Conditions Only is selected, enter the expression for the conditions, and then select the group in the Assign To column.
      • If For Conditions and Assignments is selected, enter the expression that defines both the conditions and the group assignment.

      Note: Expressions can use attributes from both the source and destination applications. The configured expressions must always be evaluated to true or false. The prefix SOURCE must be used in an expression to represent an attribute in the source application, and DEST must be used to represent an attribute in the destination application. For example, SOURCE.addresses[].locality, SOURCE.addresses[].country, and DEST.group.displayName.

  7. Click Add New Rule to add another group assignment rule, as explained above.

Group Retention

The Group Retention section allows you to define how a user's group memberships must be handled when the Dayforce user is updated or deactivated.

On Deactivate / Delete

This dropdown list is used to specify what should happen to a user's group memberships when the Dayforce user is deactivated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the deactivated user satisfies some of the group assignment rules and they are not already part of those groups, they will be added to those groups; if they are already part of those groups, they will be retained in those groups.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained, even if the user does not satisfy the rules configured for the group assignments in the Group Rules section.
  • Retain in selected groups: This option has to be used when you want to retain the deactivated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained. Deactivated users will be removed from all other groups.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of users from the deactivated list of users. The group membership will be retained only for the users matching the filter condition. For the remaining set of deactivated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

On Update

This dropdown list is used to specify what should happen to a user's group memberships when the user is updated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the user satisfies the conditions defined in the Group Rules section, they may be added to new groups based on the configured group rules.
    Note: If the Update Deactivated Users configuration is enabled, a deleted or deactivated user will be added to a new group based on the group rules. However, they will not be added back to an existing group from which they have been removed during this integration run.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained. Even if the user does not satisfy a rule configured for the group assignments in the Group Rules section, they will be retained in those groups.
  • Retain in selected groups: This option has to be used when you want to retain the updated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained on user update.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of user records that were updated. The group membership will be retained only for the users matching the filter condition. For the remaining set of updated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

Limitations

  • You can save the integration even without mandatory fields configured, but the integration runs will fail.
  • It is required to download a report of all employees in the Dayforce connector (through the Data tab) to view all the Dayforce attributes in this integration.
  • When the integration creates guest users, the UPN value is automatically generated by the Entra ID in the format, emailaddress#EXT#@tenant.onmicrosoft.com. The UPN naming format configured in the integration will be ignored for guest users.
  • The Incremental Tuning configuration is designed to function only when the match expression only option is selected, as this integration does not support incremental tuning with userName.
  • Debug logging for a full run with a large number of source records can exceed back-end limits, resulting in reports that are too big to save. This will be fixed in a future release. 

  • In the User Naming Attributes tab, the Advanced mode expression does not support if-else statements. To write conditional logic in these expressions, use a conditional or ternary operator with the following syntax:

    {condition}? {expressionIfTrue} : {expressionIfFalse}
