User Integration Setup Guide - UKG Workforce Central to Okta

Overview

This guide covers the end-to-end tasks involved in setting up synchronization to sync users in UKG Workforce Central with Okta.

The following tasks are involved in setting up the synchronization from UKG Workforce Central to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running synchronization 

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access.

Configuring Applications in Aquera Admin

The following applications must be added and configured in Aquera Admin console:

  • UKG Workspace Central: Refer to the UKG Workforce Central Configuration Guide for details.
  • Okta: Refer to the Okta Configuration Guide for details.

Synchronization Setup

This section details how the User Synchronization must be configured to allow data push from UKG Workforce Central to Okta. 

It also describes the various analyses and the important points to be considered before setting up the user synchronization.

Considerations

  • Work email can be retrieved from the UKG Workforce Central system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from UKG Workforce Central with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping UKG Workforce Central user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in UKG Workforce Central as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script synchronization. For scripts, refer to the Appendix.

Building Synchronization

This document describes how to set up the synchronization using the User Synchronization method.

  • User Synchronization: In this synchronization, set UKG Workforce Central as the Source Application and Okta as the Target Application. Refer to the User Synchronization Setup Guide for details on the configurations.

You can set the following configurations in the User Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables to create lookup tables based on location and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list to limit the scope of the groups that Aquera would manage.

Refer to the User Synchronization Setup Guide for details on the configurations.

Running Synchronization

  • Enter the matching expression to match the source records with the destination records.
  • Run the synchronization in Report Only mode and determine if the correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. If errors are encountered, modify the synchronization configurations to fix them.
  • Once all users are synced, user creations and updates are based on the incremental changes in UKG Workforce Central.

This section section explains how analyses can be done using scripts. Add the UKG Workforce Central and Okta applications to this synchronization and set the corresponding variable names. The variable names must be used to refer to the applications in the script.
