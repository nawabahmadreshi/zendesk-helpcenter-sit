Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration facilitates synchronization between Dayforce and Ivanti Neurons, automating ticket creation for onboarding, offboarding, and change management based on employee lifecycle events.

The integration is triggered either on a fixed schedule or through event-based mechanisms. Once triggered, it fetches recent or upcoming employee lifecycle changes such as new hires, terminations, and updates from Dayforce, and then processes them into Ivanti Neurons tickets.

Key Functions

This integration performs the following key tasks:

  • Retrieves employee data from Dayforce, including hires, terminations, and updates.
  • Processes detailed employee profiles (contact, department, job metadata).
  • Matches employees to existing Ivanti records using correlation fields.
  • Creates or updates onboarding, offboarding, and change tickets in Ivanti.
  • Prevents duplicate ticketing by checking for existing records.

Integration Setup

This section outlines the configuration process for integrating Dayforce with Ivanti Neurons using the Aquera platform. It includes the steps for creating the integration, defining parameters, and enabling the automated processing of employee lifecycle events, such as onboarding, offboarding, and profile updates.

This integration continuously syncs employee records from Dayforce, specifically new hires, terminations, and job changes, and dynamically creates or updates corresponding tickets in Ivanti Neurons.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to the Integrations tab.
  3. Click Create Integration.
  4. Choose Dayforce to Ivanti Neurons Tickets.
  5. Name and describe your integration.
  6. Select your Dayforce and Ivanti Neurons connector instances.
  7. Click Continue. The configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Integration Scenario

  • Enable Ticket Management: Automatically creates Ivanti Neurons tickets for Dayforce hires, terminations, and updates.

Global Settings

  • Enable Report-Only Mode: Simulate the integration execution without creating or updating any Ivanti Neurons tickets. Use this mode to verify configurations and preview potential changes safely.
  • Enable Logging: Capture detailed logs of integration activities, including data changes, API interactions, and processing results, to facilitate troubleshooting and support operational audits.
  • Enable Report Generation: Generate structured CSV reports summarizing integration outcomes, employee lifecycle event processing, ticket creation results, and errors.
  • Enable Log Insights: Log transaction details from integration transactions to analytics dashboards to monitor activity trends, track performance metrics, and proactively identify and resolve issues.
  • Enable Notifications: Select to enable integration-related notifications.

 Ticket Management Settings

Configure the following Ticket management settings:

  • HR Events to Trigger Ticketing: Select one or more employee lifecycle events that should initiate ticket creation workflows.

Source Settings

  • Employee Data Retrieval: Retrieve onboarding and termination records within the next X days and update records from the previous X days. If no value is entered, the default is 30 days.

Target Settings

  • Ticket Creation: Select the employee lifecycle events, such as Onboarding, Offboarding, or Changes, to create or modify the ticket. 
  • Username: Enter your Ivanti username used to create service requests.
  • Request Template: Select the template used to create service requests.

Joiner Ticket Settings

  • Filter Onboarding Users by Status: Select employee statuses that will trigger the creation of a joiner ticket.
  • Days Before Hire to Create Ticket: Enter the number of days before the hire date to create the ticket.
  • Select the Ticket Type: Select the ticket type for new hire requests.
  • Create Limit: Enter the maximum number of tickets that can be created.

Leaver Ticket Settings

  • Filter Offboarding Users by Status: Select employee statuses that will trigger the creation of the leaver ticket.
  • Days Before Termination to Create Ticket: Enter the number of days before the termination date to create the ticket.
  • Select the Ticket Type: Select the ticket type for employee departure requests.
  • Create Limit: Enter the maximum number of tickets that can be created.

Mover Ticket Settings

  • Filter Transfer Users by Status: Select employee statuses that will trigger the creation of a mover ticket.
  • Days After Transfer to Create Ticket: Enter the number of days after the employee transfers to create the ticket.
  • Select the Ticket Type: Choose the ticket type for employee transfer requests.
  • Create Limit: Enter the maximum number of tickets that can be created.

Attribute Mapping

The Attribute Mapping tab defines how Dayforce employee fields correspond to Ivanti Neurons ticket fields during ticket creation and updates. For instance, Dayforce attributes, such as Employee Number or Work Email, can be mapped directly to Ivanti fields, like Employee ID or Contact Email. Clearly defined mappings ensure accurate and consistent data population across Ivanti tickets, facilitating seamless alignment of HR records with IT service management workflows.

Proper mapping reduces data inconsistencies, ensures that tickets contain precise employee details, and maintains data integrity throughout onboarding, offboarding, and employee change management processes.

Tables

The Tables section includes configurable key-value pairs used to standardize and translate constant values between Dayforce and Ivanti Neurons. This ensures consistency when performing integration operations. For example, you might map integration constants such as CREATE, UPDATE, or FAILED to specific status indicators in Ivanti.

Without this table configuration, the integration may incorrectly interpret statuses or produce inconsistent reporting. By clearly defining constants (e.g., mapping SUCCESS to an appropriate Ivanti status), the integration maintains accuracy.
