Filtering Data in Workflows

← Back to index

Overview

You can write expressions in workflows in the expression mode. 

You must use SCIM filter expressions to filter the attributes of the Application's Import All operation in workflows. SCIM filter expressions follow the SCIM RFC specification. 

However, you must use JavaScript expressions in the workflow elements such as If-else and While. All the JavaScript operators are supported in the JavaScript expressions of workflows.

This guide provides information on SCIM filter expressions to filter the attributes in workflows using attribute operators, logical operators, and grouping operators.

Expressions

Each expression must contain an attribute name followed by an attribute operator and an attribute value. Multiple expressions may be combined using logical operators. Furthermore, expressions can be grouped using grouping operators. 

Note: The expression will work only if the connector added in the workflows supports filtering by the specified operators and attribute names. Refer to the Supported Filters section of the respective connector configuration guides for further details.

Attribute Operators

The following table lists the attribute operators that can be used in a SCIM filter expression:

Description Behavior SCIM Filter Expression JavaScript Expression
equal

The attribute evaluation and the value must be identical to meet the filter condition.

 

Core: userName eq "bjensen"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department eq "Sales"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID eq "Q7DETWGV"

Core: userName == "bjensen"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department == "Sales"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID == "Q7DETWGV"
not equal

The attribute evaluation and the value must not be identical to meet the filter condition.

Core: userName ne "bjensen"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department ne "Sales"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID ne "Q7DETWGV"

 

Core: userName != "bjensen"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department != "Sales"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID != "Q7DETWGV"
greater than

If the attribute evaluation is greater than the value, the filter condition is met. The actual comparison is dependent on the attribute type.

For string attribute types, this is a lexicographical comparison, and for DateTimetypes, it is a chronological comparison. For integer attributes, it is a comparison by numeric value. This operator cannot be used for Boolean and Binary attributes.

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber gt 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID gt 100

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber > 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID > 100
greater than or equal to

If the attribute evaluation is greater than or equal to the value, the filter condition is met. The actual comparison is dependent on the attribute type.

For string attribute types, this is a lexicographical comparison, and for DateTimetypes, it is a chronological comparison. For integer attributes, it is a comparison by numeric value. This operator cannot be used for Boolean and Binary attributes.

 

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber ge 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID ge 100

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber >= 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID >= 100
less than

If the attribute evaluation is less than the value, the filter condition is met. The actual comparison is dependent on the attribute type.

For string attribute types, this is a lexicographical comparison, and for DateTimetypes, it is a chronological comparison. For integer attributes, it is a comparison by numeric value. This operator cannot be used for Boolean and Binary attributes.

 

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber lt 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID lt 100

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber < 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID < 100
less than or equal to

If the attribute evaluation is less than or equal to the value, the filter condition is met. The actual comparison is dependent on the attribute type.

For string attribute types, this is a lexicographical comparison, and for DateTimetypes, it is a chronological comparison. For integer attributes, it is a comparison by numeric value. This operator cannot be used for Boolean and Binary attributes.

 

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber le 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID le 100

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber <= 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID <= 100
contains

The value must be a substring of the attribute evaluation to meet the filter condition.

 

Core: userName co "@domain.com"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department co "Sale"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID co "Q7"

Core: userName includes "@domain.com" 

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:employeeNumber includes 100

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID includes 100
starts with

The value must be a substring matching at the beginning of the attribute evaluation to meet the filter condition.

 

Core: userName sw "b"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department sw "S"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID sw "Q"

Core: userName startsWith "b"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department startsWith "S"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID startsWith "Q"
ends with

The value must be a substring matching at the end of the attribute evaluation to meet the filter condition.

 

Core: userName ew "@domain.com"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department ew "s"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID ew "V"

Core: userName endsWith "@domain.com"

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department endsWith "s"

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID endsWith "V"
is defined

If the attribute has a non-empty or non-null value, or if it contains a non-empty node for complex attributes, the filter condition is met.

Core: userName pr

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department pr

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID pr

 

Core: userName != 'undefined'

Enterprise: urn:ietf:params:scim:schemas:extension:enterprise:2.0:User:department != 'undefined'

IAN: urn:ietf:params:scim:schemas:ian:2.0:User:employeeID != 'undefined'

Logical Operators

The logical operators are used to combine multiple expressions. The following table lists the logical operators that can be used in a SCIM filter expression:

Description Behavior SCIM Filter Expression JavaScript Expression
Logical AND The filter is a match only if both the expressions (criteria) are met.

 

userType ne "Guest" and urn:ietf:params:scim:schemas:ian:2.0:User:employeeID pr

userType ne "Guest" && urn:ietf:params:scim:schemas:ian:2.0:User:employeeID pr
Logical OR The filter is a match if either of the expressions (criteria) is met.

 

userType ne "Guest" or urn:ietf:params:scim:schemas:ian:2.0:User:employeeID gt 100

userType ne "Guest" || urn:ietf:params:scim:schemas:ian:2.0:User:employeeID gt 100
Logical NOT The filter is a match if the expression evaluates to false.

 

userType eq "Admin" and not (userName co "example.com" or urn:ietf:params:scim:schemas:ian:2.0:User:employeeID le 100)

userType eq "Admin" && !(userName co "example.com" or urn:ietf:params:scim:schemas:ian:2.0:User:employeeID le 100)

Note: not takes precedence over and, which takes precedence over or.
