Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the Active Directory (AD) user information with Riskonnect's internal contact records. Based on a user's status and user presence in Riskonnect, the integration will create, update, or delete the corresponding record in Riskonnect.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Connectors in the Aquera Admin Portal: You need to configure the following connectors in the Aquera Admin portal for the integration:
    • Active Directory (AD): Refer to the Active Directory (AD) Configuration Guide for details.
    • Riskonnect: Refer to the Riskonnect Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to import workers from Active Directory (AD) and provisions these users in Riskonnect.

Create an Integration

Perform the following steps to create an integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Active Directory / Riskonnect.
  5. Enter a name and a description for the integration.
  6. Select your Active Directory (AD) and Riskonnect connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Do you want to run as simulation mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable OpenSearch Logging: Select to visualize the integration-related metrics.
  • Generate CSV File: Select this checkbox to generate a CSV report detailing the data changes that would happen, but without applying them.
  • Log Data Level: Select the desired log detail level from the dropdown menu. The following are the available options: 
    • ALL: All attribute details are displayed.
    • Basic: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • None: Select this to ignore the log generation.
  • Send Email Notification: Select this checkbox to send email notifications to the email addresses configured in the Recipient's Email Addresses parameter.
  • Recipient's Email Addresses: Enter the email addresses of the recipients to whom the email notifications should be sent.
  • Delete User on Deactivation: Select this checkbox to delete users upon deactivation. If this checkbox is not selected, the users will not be deleted upon deactivation.

Tables

The Tables section includes configurable key-value pairs used to standardize and translate constant values between Active Directory (AD) and Riskonnect. This ensures consistency when performing integration operations. For example, you might map integration constants such as CREATE, UPDATE, or DELETE to specific status indicators in Riskonnect.

Without this table configuration, the integration may incorrectly interpret statuses or produce inconsistent reporting. By clearly defining constants (e.g., mapping SUCCESS to an appropriate Riskonnect status), the integration maintains accuracy.

Schedule

Integrations can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details.

Run Integration

You can set event or schedule triggers to run the script synchronization. For further details, refer to the Integration Using Scripts guide.

After the script synchronization is successfully executed, Aquera syncs the users created in Active Directory (AD) as internal contacts in Riskonnect. The imported users are staged and need admin approval to get provisioned to Riskonnect.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
