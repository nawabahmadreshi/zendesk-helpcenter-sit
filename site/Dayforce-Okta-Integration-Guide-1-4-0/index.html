Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

Key Functions

The Dayforce / Okta Integration performs the following functions:

  • Synchronizes user records from Dayforce to Okta.
  • Creates new users in Okta when they are added in Dayforce.
  • Updates existing Okta users when their information changes in Dayforce.
  • Deactivates or deletes Okta users when they are no longer active in Dayforce.
  • Generates username and email IDs for users in Okta based on configured policies.
  • Updates busniness email address and federated ID from Dayforce to Okta (writeback).
  • Supports both full synchronization and incremental synchronization to process only changed records.

Integration Setup

This section outlines how the integration must be configured to push user-related data from Dayforce / Okta.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Okta.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Okta connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following feature based on your requirements:

  • Enable Users Sync Enable user lifecycle management: Select this checkbox to synchronize employee's records from Dayforce to Okta. When enabled, the User Sync and User Life Cycle sections will be displayed for further configuration.
  • Enable Write Back: Select this checkbox to write back the user records from Okta to Dayforce. When enabled, the Write Back section will be displayed for further configuration.

Global Settings

Enable the following settings based on your requirements:

  • Enable incremental mode: Select this checkbox to execute the integration in the incremental import mode. If this checkbox is not selected, the integration will be executed in full sync.
    Note: Select this parameter only after the first integration run if you want to execute the integration in incremental import mode.
  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Generate Unique Username/Email Using AI Widget Enable AI widget to generate unique usernames and emails: Select this checkbox to generate a unique username and email using the AI widget. When enabled, the Define Manage Policy tab will be displayed for further configuration.
  • Create custom attributes in Okta: Select this checkbox to create custom attributes in Okta. When enabled, the Custom Attributes tab will be displayed for further configuration.

Feature Modules General Settings

Enable the following settings based on your requirements:

  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable log insights: Select this to visualize the integration-related metrics. When enabled, the Log Insight Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable debug mode: Select this checkbox to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.

Manage User Life Cycle

The User Life Cycle section enables you to customize the integration settings according to your specific use case. It has the following settings:

Create Settings

Configure the following settings:

  • Create Limit: Enter the maximum number of employee records that can be created from Dayforce to Okta in a single integration run. For example, entering 7 allows only 7 employee records to be created in a single integration run.
  • Create Condition*: Select a condition from the dropdown menu. Only employees who satisfy the selected condition will be created in Okta. The available options are Always Create Employees, Only Create Active Employees, Only Create Employees Based On Onboarding Date, and Custom.
    • Always Create Employees: Select this option to create all employees in Okta from Dayforce.
    • Only Create Active Employees: Select this option to create all active employees in Okta from Dayforce. When enabled, the following parameter is displayed:
      • Statuses To Be Considered As Active*: Select the statuses that must be considered as active. Only employees with these statuses will be created in Okta from Dayforce.
    • Only Create Employees Based On Onboarding Date: Select this option to create employees based on their onboarding date. When enabled, the following parameters are displayed:
      •  Attribute To Consider For Onboarding Date*: Select an attribute from the dropdown menu. The value of the selected attribute will be considered as the onboarding date. For example, if you select the Start Date attribute, its value will be considered the user’s onboarding date.
      • Onboarding Date Offset Type*: Select an onboarding date offset type from the dropdown menu. The available options are Create Before Onboarding Date, Create After Onboarding Date, and Same Day.
        • Create Before Onboarding Date: Select this option to create the user in Dayforce from Okta before the user's actual onboarding date. When selected, the following parameter is displayed:
          • Onboarding Offset days*: Enter the number of days to offset before the user’s actual onboarding date. For example, if you enter 3 and the user’s onboarding date is 06-05-2026, the user will be hired three days earlier, on 03-05-2026.
        • Create After Onboarding Date: Select this option to create the user in Dayforce from Okta after the user's actual onboarding date. When selected, the following parameter is displayed:
          • Onboarding Offset days*: Enter the number of days to offset after the user’s actual onboarding date. For example, if you enter 3 and the user’s onboarding date is 06-05-2026, the user will be hired three days later, on 09-05-2026.
        • Same Day: Select this option to create the user in Dayforce from Okta on the same day as the user's onboarding date.
    • Custom: Select this option if you want to customize the user creation condition. When selected, the following parameter is displayed:
      • Custom Create Condition: Enter an expression that defines the condition for creating users in Dayforce from Okta. Only users who satisfy this condition will be created in Dayforce from Okta.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
