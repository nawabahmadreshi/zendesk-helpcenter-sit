Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, worker records from SolarWinds Service Desk are synced to the ADP Workforce Now resource.

To identify user records across both systems, the integration uses the attribute configured in the Dayforce User Attributeparameter from Dayforce and matches it to the attribute configured in the Salesforce Contact Attribute parameter in Salesforce as a correlation attribute.

Key Functions

The SolarWinds Service Desk / ADP Workforce Now Integration performs the following functions:

  • Synchronizes worker records from SolarWinds Service Desk to ADP Workforce Now.
  • Creates new workers in ADP Workforce Now whenever new workers are added in SolarWinds Service Desk.
  • Updates existing worker when their information changes in ADP Workforce Now.
  • Deletes ADP Workforce Now worker records when they are no longer active in SolarWinds Service Desk.
  • Supports both full synchronization and incremental synchronization to process only changed records.
  • Supports filtering employees by status for creating and updating contacts in Salesforce.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • SolarWinds Service Desk: Refer to the SolarWinds Service Desk Configuration Guide for configuration details.
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from SolarWinds Service Desk / ADP Workforce Now.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select SolarWinds Service Desk / ADP Workforce Now.
  5. Enter a name and a description for the integration.
  6. Select your SolarWinds Service Desk and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable Worker Lifecycle Onboarding: Select this checkbox to synchronize worker records from SolarWinds Service Desk to ADP Workforce Now. When enabled, the Worker Management section is displayed for further configuration.

General Settings

Configure the following settings:

  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Reporting: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insightssection is displayed for further configuration.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.
  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.

Worker Management

The Manage Contacts section enables you to customize the integration settings for contacts management according to your specific use cases.

SolarWinds Service Desk Settings

Configure the following SolarWinds Service Desk settings:

  • Ticket State When Worker Is Ready to Onboard*: Select a ticket status from the dropdown menu. Only worker records with the selected status will be considered for synchronizing from SolarWinds Service Desk to ADP Workforce Now. For example, if you select New, only workers with a New status will be included.
  • Ticket State When Worker Is Onboarded*: Select a ticket status from the dropdown menu. 
  • Limit: Enter the maximum number of worker records to be processed in a single integration execution. For example, entering 10 processes only includes the first ten worker records.

ADP Workforce Now Settings

This section enables you to customize the correlation attributes of the Dayforce and Salesforce applications.

Note: All parameters marked with a red asterisk (*) are mandatory.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integrationsection in the Using Tables in Integrations guide for details.
