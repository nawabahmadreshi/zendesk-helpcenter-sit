Overview

ADP Decidium is a flexible, configurable, and easy-to-use Human Resource Management solution. It supports organizations with the management and administration of employee life cycle events in addition to planning and executing business needs.

This guide helps users to create an ADP Decidium application connector on the Aquera platform.

For further details on the ADP Decidium application, visit the ADP website.

Prerequisites

The following are the prerequisites:

  • Administrator access to the ADP Decidium instance: You must have the admin credentials for your ADP Decidium instance.
  • Bearer token: You must have a bearer token for authentication. The bearer token is provided by the Aquera Support team.

Connector Configuration

Perform the following steps to configure the ADP Decidium connector:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Applications and click Add Application.
  3. Search for ADP Decidium and click the ADP Decidium icon.
  4. Enter the values for the following fields under the Configuration tab:
    • Basic Details:
      • Display Name: Enter a display name for your application.
      • Description: Enter a description for your application.
      • Source Connection Type: Select Direct Connection.
        Note: This field is available only if you have access to Aquera's File Collection module.
      • Source Owner: Enter valid email IDs of the source owners.
      • Subscribe to error notification: Enable the toggle button to send notification emails to source owners for errors in the user data transactions. Click Edit Preference to select the error types for which you want to receive notifications. After you configure your preference, you will receive error notifications only for the selected error types. By default, all error types are selected.
      • Pick your protocol: Select the protocol from the dropdown list based on your requirement. SCIM is the default value.
      • Copy this URL: This field displays the base URL of the connector API endpoint. The URL is populated based on your selected protocol. Copy and paste this URL while provisioning to the identity service provider or performing OpenAPI operations, if supported.
    • Tenant Attributes: Refer to the Aquera Tenant Attributes section for details.
  5. Click Create to create the ADP Decidium connector.

Refer to the Getting Started with Connectors guide to learn more about connector setup including different tabs of the connector configuration page.

Aquera Tenant Attributes

The following are the tenant attributes:

  • Attributes to be excluded in the response: Enter the comma-separated list of attributes that must be excluded from the response. All other attributes will be included in the response.
    Notes:
    • The exact names of the attributes must be entered in this field. You can expand the Users list under the Schema on the Aquera Admin console to see the list of user attributes and find their exact names.
    • ADP Custom attributes can also be removed from the response. You must enter the exact names of the custom attributes as configured in the ADP UI. For example, assignment_stringFields_{codeFields.nameCode.codeValue}.
    • To exclude complex attributes of array type, such as emails, phoneNumbers, and addresses, the type value must also be entered. The format is <attributename>[type]. For example, phoneNumbers[home], emails[work].
    • The Id, userName, and active attributes cannot be removed from the response.
    • To exclude the manager attributes, the entire manager object has to be excluded by entering the value as manager in the field.

  • Attributes to be included in the response: Enter the comma-separated list of attributes that must be included in the response. Any other user attributes will be excluded from the response.
    Notes:

    • The exact names of the attributes must be entered in this field. You can expand the Users list under the Schema on the Aquera Admin console to see the list of user attributes and find their exact names.

    • ADP Custom attributes can also be removed from the response. You must enter the exact names of the custom attributes as configured in the ADP UI. For example, assignment_stringFields_{codeFields.nameCode.codeValue}.

    • To exclude complex attributes of array type, such as emails, phoneNumbers, and addresses, the type value must also be entered. The format is <attributename>[type]. For example, phoneNumbers[home], emails[work].

    • The Id, userName, and active attributes cannot be removed from the response.

    • To exclude the manager attributes, the entire manager object has to be excluded by entering the value as manager in the field.
       

    • If you set both Attributes to be included in the response and Attributes to be excluded in the response tenant attributes, a 500 error will be displayed by the connector. You must set only one of these tenant attributes.

Authentication

The ADP Decidium connector uses a Bearer token to authenticate and authorize the SCIM REST requests. The Bearer token is provided by the Aquera Support team.
