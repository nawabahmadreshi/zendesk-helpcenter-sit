Overview

The Aquera platform enables your organization to scale complex data management by automating the integration between various applications. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the ADP Vantage Leave Notification integration.

The integration helps send email notifications to designated recipients when an employee goes on leave (Leave of Absence) and returns from the leave. The integration checks the leave history of an employee using their associateOID.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Application Connector in the Aquera Admin Portal: You must have an ADP Vantage application connector configured in the Aquera Admin portal. Refer to the ADP Vantage HCM Configuration Guide for details.

Integration Setup

This section details how to set up the integration.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Vantage Leave Notification.
  5. Enter a name and a description for the integration.
  6. Select your ADP Vantage connector instance.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the application connector that is selected for the integration. This section also allows you to search and update the application connector.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Global Settings

Configure the following global settings:

  • Enable Report Generation: Select this checkbox to receive email notifications with a CSV report for all integration executions.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.

Report Settings

Configure the following global settings:

  • Report detail level: Select the level of detail to be included in the report. The following options are available:
    • Full: Select this option if you want full details in the report.
    • Basic: Select this option if you want only basic details to be included in the report.

Notification Settings

Configure the following notification settings:

  • Send email notifications on Leaves to: Enter the email addresses of the recipients to whom the email notifications on leaves will be sent.
  • Admin's Email Address: Enter the email addresses of the admins to whom the email notifications on integration execution will be sent.
  • Email heading for User's LOA: Enter the heading to be used in the email body for user's LOA status.
  • Email Heading for Users' Return From LOA: Enter the heading to be used in the email body for user's return from LOA status.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors.
  • Email Subject for Success: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • Email Subject for Failure: Enter the content that must be used as the email subject in the notifications sent for integration errors.
  • Provide Notification Email IDs for DTA Company: Enter email addresses that should receive notifications related to the DTA company.
  • Provide Notification Email IDs for DTB Company: Enter email addresses that should receive notifications related to the DTB company.
  • Provide Notification Email IDs for DTK Company: Enter email addresses that should receive notifications related to the DTK company.
  • Provide Notification Email IDs for E23 Company: Enter email addresses that should receive notifications related to the E23 company.
