Workday to ServiceNow Tickets Management Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The Workday to ServiceNow Tickets Management integration retrieves the details from Workday for users who are newly onboarded, have changed their positions or roles, or have been terminated. Based on the user lifecycle events in Workday, the integration creates tickets (catalog requests) in ServiceNow. A catalog request is created for each of the joiner, mover, and leaver records retrieved from Workday. This guide provides details on the configurations that are available in the Aquera Admin portal to set up this integration.

Key Functions

The Workday to ServiceNow Tickets Management integration performs the following functions:

  • Retrieves the data of the following workers from your Workday instance in an incremental mode.
    • New hires
    • Employees with department, title, location, manager, or any specific custom attribute changes
    • Terminated employees
  • Creates a ServiceNow ticket (catalog request) for each joiner, mover, or leaver event to trigger necessary IT or HR processes for account creation, access modification, or deprovisioning.
    • You can select the event types (joiner, mover, leaver) for which a ticket needs to be created on the integration setup page.
    • Ticket fields are updated based on the mappings provided in the Attribute Map section of the integration.
  • Sends automated, detailed notification emails to a group of people based on the configuration.
  • Generates CSV reports and logs capturing the details of the joiners, movers, and leavers, and the tickets created.
  • Enables log insights to monitor integration outcomes and for analytics.

Integration Setup

This section details the configurations required for this integration. It covers setting up the applications involved in the integration and configuring the integration parameters for creating and updating tickets in ServiceNow for the joiner, mover, and leaver events in Workday. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Workday to ServiceNow Tickets Management.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the application connectors required for the integration. This section allows you to search and select the Workday application from which the joiner, mover and leaver details need to be retrieved, and the ServiceNow application in which the tickets must be created.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

This section is used to enable the following features of the integration.

  • Enable Ticket Management: Select this checkbox to manage tickets in ServiceNow for the joiner, mover, and leaver events. This is a mandatory setting.
  • Select Event Type: Select the types of the events (Joiner, Mover, Leaver) for which you want the integration to create ServiceNow tickets. This field is displayed only if you have selected the Enable Ticket Management checkbox.
  • Enable Department Management: Select this checkbox if you want the integration to sync departments from Workday to ServiceNow. The Manage Departments configuration section is displayed to configure the department sync settings.
  • Enable Location Management: Select this checkbox if you want the integration to sync locations from Workday to ServiceNow. The Manage Locations configuration section is displayed to configure the location sync settings.
  • Enable User Management: Select this checkbox if you want the integration to sync users from Workday to ServiceNow. The Manage Users configuration section is displayed to configure the user sync settings.

Execution Features

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the updates that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed when this checkbox is selected.
  • Enable Email Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
    Note: The Log Insights section will be displayed when this checkbox is selected.
  • Debug Settings: Select this checkbox to run the integration in debug mode.
    Note: The Debug Settings section will be displayed when this checkbox is selected.

Debug Settings

Configure the following debug settings:

  • Source Filter: Used to filter a single worker record from Workday and run the integration for debugging purposes. You can filter workers by Worker ID or Worker Email. Based on the selection, another field is displayed to enter the Worker ID or the Worker Email ID.

Joiner Ticket Settings

Configure the following settings for creating joiner tickets:

  • Enable Onboarding Ticket Generation: Select this checkbox to create onboarding tickets in ServiceNow for new hires.
  • Workday Workers Retrieval Statuses: Select the statuses of the new hires who must be retrieved from Workday for creating onboarding tickets. For example, if you select Rehire, only the workers with the status as Rehire in Workday are retrieved and onboarding tickets are created in ServiceNow for only those workers.
  • Onboarding Date Type: The type of date that must be considered for retrieving new hires from Workday. For example, if you select HireDate, the date value in the Hire Date field of the workers in Workday will be considered for retrieving the workers.
  • Enable Ticket Preprocessing: Select this checkbox if you want to create tickets in advance for employees who will have a hire date in the future.
  • Create Tickets X Days Before Date: Enter the number of days before which onboarding tickets have to be created for new employees. For example, if you configure 5 in this field, onboarding tickets will be created in advance for employees whose hire date is on the date that is 5 days from the current date.
    Note: This field is displayed only if you have selected the Enable Ticket Preprocessing checkbox.
  • Include Ranges for Onboarding: Select this checkbox to create onboarding tickets for all new hires whose hire dates fall within the range between the current date and the number of days specified in the Create Tickets X Days Before Date field. For example, if you enable this option and set Create Tickets X Days Before Date to 5, onboarding tickets will be generated for all employees scheduled to be hired within the next five days from the current date.
    Note: This field is displayed only if you have selected the Enable Ticket Preprocessing checkbox.
  • ServiceNow Ticket Category: Select the category under which the onboarding tickets have to be created in ServiceNow.
  • ServiceNow Ticket Type: Select the type of ticket that needs to be created in ServiceNow for employee onboarding.
  • Enable Workday Worker ID Filter: Select this checkbox if you want to create an onboarding ticket for a particular worker only.
  • Workday Worker ID: Enter the ID of the worker for whom the onboarding ticket has to be created.
    Note: This field is displayed only if you have selected the Enable Workday Worker ID Filter checkbox.

Mover Ticket Settings

Configure the following settings for creating mover tickets:

  • Enable Mover Ticket Generation: Select this checkbox to create tickets in ServiceNow for changes in any of the following details for a worker: department, location, title, manager, or any other (custom) attribute, as per your requirement.
  • Trigger Field for Mover Ticket Creation: Select one or more field(s) whose changes should trigger the creation of a mover ticket in ServiceNow. When any of the selected fields (such as Department, Location, Title, Manager, or other Custom attributes) are updated for a worker, the integration identifies it as a mover event and automatically generates a corresponding Mover ticket. 
  • Custom Mover Field for Ticket Creation: Select the custom attribute that the integration must check for any changes for creating a mover ticket in ServiceNow. 
    Note: This field is displayed only if you have selected Custom from the Trigger Field for Mover Ticket Creation field.
  • ServiceNow Ticket Category: Select the category under which the employee mover tickets have to be created in ServiceNow.
  • ServiceNow Ticket Type: Select the type of ticket that needs to be created in ServiceNow for employee mover transactions.
  • Enable Workday Worker ID Filter: Select this checkbox if you want to create a mover ticket for a particular worker only.
  • Workday Worker ID: Enter the ID of the worker for whom the mover ticket has to be created.
    Note: This field is displayed only if you have selected the Enable Workday Worker ID Filter checkbox.

Leaver Ticket Settings

Configure the following settings for creating offboarding tickets:

  • Enable Offboarding Ticket Generation: Select this checkbox to create offboarding tickets in ServiceNow for worker terminations.
  • Workday Workers Retrieval Statuses: Select the statuses of the terminated employees who must be retrieved from Workday for creating offboarding tickets. For example, if you select Terminated, only the workers with the status as Terminated in Workday are retrieved and offboarding tickets are created in ServiceNow for only those workers.
  • Offboarding Date Type: The type of date that must be considered for retrieving terminated employees from Workday. For example, if you select Terminated Date, the date value in the Termination Date field of the workers in Workday will be considered for retrieving the workers.
  • Enable Ticket Preprocessing: Select this checkbox if you want to create tickets in advance for employees who will have a termination date in the future.
  • Create Tickets X Days Before Date: Enter the number of days before which offboarding tickets have to be created for terminated employees. For example, if you configure 5 in this field, an offboarding ticket will be created for an employee who has a termination date in the next 5 days from the current date.
    Note: This field is displayed only if you have selected the Enable Ticket Preprocessing checkbox.
  • ServiceNow Ticket Category: Select the category under which the offboarding tickets have to be created in ServiceNow.
  • ServiceNow Ticket Type: Select the type of ticket that needs to be created in ServiceNow for employee offboarding.
  • Enable Workday Worker ID Filter: Select this checkbox if you want to create an offboarding ticket for a particular worker only.
  • Workday Worker ID: Enter the ID of the worker for whom the offboarding ticket has to be created.
    Note: This field is displayed only if you have selected the Enable Workday Worker ID Filter checkbox.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: All the integration configurations are captured in the logs.
    • Full Details: All the actions performed by the integration are captured in the logs. The logs include the following details:
      • The count of joiner, mover, and leaver tickets that have been created
      • The total number of tickets that have been created successfully
      • The total number of tickets that have been updated successfully
      • The total number of ticket creation failures
      • The total number of ticket update failures.
      • The total number of workers who were skipped.

Report Settings

Configure the following report settings:

  • Generate Integration Report: Select this checkbox to generate a CSV report for integration runs. The CSV report will have the details of the tickets created for joiner, mover, and leaver transactions from Workday.
  • Report Log level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: All the integration configurations are captured in the report.
    • Full Details: All the actions performed by the integration are captured in the report. The report includes the following details:
      • The count of joiner, mover, and leaver tickets that have been created
      • The total number of tickets that have been created successfully
      • The total number of tickets that have been updated successfully
      • The total number of ticket creation failures
      • The total number of ticket update failures.
      • The total number of workers who were skipped.

Notification Settings

Configure the following settings for email notifications.

  • Recipient's Email Addresses: Enter the email addresses of the recipients to whom the email notifications must be sent.

Attribute Map

The Attribute Map enables you to set the worker attributes that must be mapped to the fields in the tickets created by the integration in ServiceNow. The data from the mapped worker attributes are populated in those ticket fields while creating tickets.
