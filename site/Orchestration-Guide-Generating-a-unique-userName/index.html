Orchestration Guide -- Generating a unique userName

← Back to index

In SCIM the userName represents a unique identifier for the user.  It can be changed, but it cannot have the same value as other the userName in other objects.  When constructing a userName for a user object in the Aquera Attribute Map there are two approaches.  Either the userName can be mapped from an existing attribute from the source that is guaranteed to be unique.  As an example, ADP holds the emails[work].value which can be mapped to the userName in the destination Active Directory.  Or the userName can be constructed in the Attribute Map from attributes of the source system.  In the case of constructing the userName if it is not unique then the user creation will fail.

A destination directory or application requires a unique userName when creating a new user.  If the source system can be the authoritative source for ensuring userName uniqueness then it is best left up to that HR system.  For example, SOURCE.emails[work].value -> DEST.userName.
