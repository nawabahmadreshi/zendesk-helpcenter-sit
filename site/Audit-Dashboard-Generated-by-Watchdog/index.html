Overview

When you set up a Watchdog in Aquera Admin Portal, you will have to setup the Access Policy. Access Policies allow you to specify the groups, applications, OUs (in case of Active Directory), and directory access that users are expected to have or are prohibited from having.

When the Watchdog runs, these policies are evaluated to identify deviations and potential access violations. These policies serve as the basis for auditing user access and detecting compliance issues. Based on the evaluation results, Watchdog generates audit dashboards and reports that provide visibility into access compliance, policy violations, and remediation status. These dashboards help administrators monitor user access, investigate anomalies, and ensure adherence to organizational access policies.

This guide explains the Audit dashboards that will be generated by the Watchdog.

Access the Audit Dashboard

  1.  
  2. Log in to the Aquera Admin portal.
  3. Click Insights on the left navigation panel. The Insights page is displayed.
  4. Navigate to the Dashboard tab.
  5. Click on the specific dashboard generated by the Watchdog. The details are displayed in the dashboard for auditing purposes. Each of these details is explained below.
