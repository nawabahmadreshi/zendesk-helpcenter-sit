Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The ADP Workforce Now to ServiceNow integration extracts the worker attributes from ADP Workforce Now and creates onboarding and offboarding tickets in ServiceNow. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal for integration:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details.

Integration Setup

This section outlines the configuration required to integrate ADP Workforce Now with ServiceNow, enabling seamless transfer of worker data from ADP Workforce Now to ServiceNow.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / ServiceNow in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Configure the following integration features:

  • Enable onboarding ticket generation: Select to create ServiceNow onboarding tickets for new hires in ADP Workforce Now.
  • Enable offboarding ticket generation: Select to create ServiceNow offboarding tickets for terminated workers in ADP Workforce Now.
  • Enable date offset: Select to enable date offset, allowing records to synchronize based on the specified date offset and time zone.

Operational Settings

Configure the following integration features:

  • Run in report-only mode: Simulate the integration run and generate results without making any changes to the target system.
  • Enable debug mode: Select to generate logs outlining data changes that would occur for debugging.
  • Run in incremental mode: Run the integration using only records that have changed since the last successful execution.
  • Enable log insights: Select to visualize the integration-related metrics.
  • Enable notifications: Select to enable integration-related notifications.

Manage Hires

This tab enables you to configure onboarding settings. 

Note: The Manage Hires tab appears when you select the Enable onboarding ticket generation checkbox in the Integration Features section of the Parameters tab.

Onboarding Tickets Generation

Configure the following onboarding ticket generation settings:

  • Onboarding condition type: Select the condition to create onboarding tickets in ServiceNow.
  • Custom onboarding condition: Enter the expression to follow when creating onboarding tickets in ServiceNow.
  • ServiceNow trigger for onboarded worker: Currently, the integration only supports the Service Catalog Item trigger, so this field cannot be configured.
  • ServiceNow catalog item for onboarded worker: Select a catalog item from the dropdown menu. This catalog item will be applicable to the onboarded worker.
  • Validate user in Okta before ticket creation: Select to ensure a ServiceNow onboarding ticket is created only after the user is verified in your selected Okta instance. If the worker is not yet found in Okta during the sync, the ticket creation will be deferred until the user is successfully identified in a subsequent run. When you select this, the Okta instance dropdown appears.
  • Okta instance: Select the Okta instance from the dropdown menu. The worker will be verified in this selected instance.
  • Validate user in Entra ID before ticket creation: Select to ensure a ServiceNow onboarding ticket is created only after the user is verified in your selected Entra ID instance. If the worker is not yet found in Entra ID during the sync, the ticket creation will be deferred until the user is successfully identified in a subsequent run. When you select this, the Entra ID instance dropdown appears.
  • Entra ID instance: Select the Entra ID instance from the dropdown menu. The worker will be verified in this selected instance.
  • Validate user in ServiceNow before ticket creation: Select to ensure a ServiceNow onboarding ticket is created only after the user is verified in ServiceNow. If the worker is not yet found in ServiceNow during the sync, the ticket creation will be deferred until the user is successfully identified in a subsequent run. 
  • Start date for onboarding ticket generation: Specify the date from which the workers' records in the ADP Workforce Now are considered for onboarding ticket creation in ServiceNow.
  • Allowed statuses for duplicate onboarding tickets: Select the RITM states that permit creating a new onboarding ticket when an existing record is already present.

Correlation Settings

Configure the following correlation settings:

  • ADP Workforce Now correlation attribute: Select an attribute that uniquely identifies a worker in ADP Workforce Now.
  • ServiceNow correlation attribute: Select an attribute in ServiceNow that matches the selected ADP Workforce Now attribute value to identify common users across ADP Workforce Now and ServiceNow.
  • Okta correlation attribute: Select an attribute in Okta that matches the selected ADP Workforce Now attribute value to identify common users across ADP Workforce Now and Okta. This field is required only when you have selected the Validate user in Okta before ticket creation checkbox.
  • Entra ID correlation attribute: Select an attribute in Entra ID that matches the selected ADP Workforce Now attribute value to identify common users across ADP Workforce Now and Entra ID. This field is required only when you have selected the Validate user in Entra ID before ticket creation checkbox.

Manage Terminations

This tab enables you to configure offboarding settings. 

Note: The Manage Terminations tab appears when you select the Enable offboarding ticket generation checkbox in the Integration Features section of the Parameters tab.

Offboarding Tickets Generation

Configure the following correlation settings for onboarding ticket creation:

  • Offboarding condition: Select the condition type to create offboarding tickets in ServiceNow.
  • Select one or more offboarding conditions: Select the offboarding condition(s) to create offboarding tickets in ServiceNow.
  • Custom offboarding condition: Enter the expression to follow when creating offboarding tickets in ServiceNow.
  • ServiceNow trigger for offboarded worker: Currently, the integration only supports the Service Catalog Item trigger, so this field cannot be configured.
  • ServiceNow catalog item for offboarded worker: Select a catalog item from the dropdown menu. This catalog item will be added for the offboarded worker.
  • Retrieve future termination date from ADP Workforce Now: Select to retrieve the upcoming termination dates for a worker or workers from ADP Workforce Now.
  • Start date for offboarding ticket generation: Specify the date from which the workers' records in the ADP Workforce Now are considered for offboarding ticket creation in ServiceNow.
  • Allowed statuses for duplicate offboarding tickets: Select the RITM states that permit creating a new offboarding ticket when an existing record is already present.

Correlation Settings

Configure the following correlation settings for offboarding ticket creation:

  • ADP Workforce Now Correlation Attribute: Select an attribute that uniquely identifies a user in ADP Workforce Now.
  • ServiceNow Correlation Attribute: Select an attribute in ServiceNow that matches the selected source attribute value to identify common users across ADP Workforce Now and ServiceNow.
  • Okta correlation attribute: Select an attribute in Okta that matches the selected ADP Workforce Now attribute value to identify common users across ADP Workforce Now and Okta. This field is required only when you have selected the Validate user in Okta before ticket creation checkbox.
  • Entra ID correlation attribute: Select an attribute in Entra ID that matches the selected ADP Workforce Now attribute value to identify common users across ADP Workforce Now and Entra ID. This field is required only when you have selected the Validate user in Entra ID before ticket creation checkbox.

Date Offset

This tab enables you to configure date offset settings. All attributes marked with a red asterisk (*) indicate required fields.

Note: The Date Offset tab appears when you select the Enable date offset checkbox in the Integration Features section of the Parameters tab.

Onboarding Date Offset

Configure the following date offset settings for onboarding:

  • Onboarding date offset type: Select an option from the dropdown menu that determines the onboarding date offset.
  • Days of offset before the onboarding date: Specify the number of days before the onboarding date to create an onboarding ticket in ServiceNow.
  • Days of offset from the onboarding date: Specify the number of days after the onboarding date to create an onboarding ticket in ServiceNow.
  • Onboarding offset time zone priority: Select the options from the dropdown menu in an order that determines the time zone for onboarding ticket creation in ServiceNow.
  • Onboarding offset time: Select the hour from the dropdown menu when the onboarding ticket is created in ServiceNow.

Offboarding Date Offset

Configure the following date offset settings for offboarding:

  • Offboarding date offset type: Select an option from the dropdown menu that determines the offboarding date offset.
  • Days of offset before offboarding: Specify the number of days before the offboarding date to create an offboarding ticket in ServiceNow.
  • Days of offset after offboarding: Specify the number of days after the offboarding date to create an offboarding ticket in ServiceNow.
  • Offboarding offset time zone priority: Select the options from the dropdown menu in an order that determines the time zone for offboarding ticket creation in ServiceNow.
  • Offboarding offset time: Select the hour from the dropdown menu when the offboarding ticket is created in ServiceNow.

Debug Settings

This tab enables you to set debug options to help troubleshoot and fix integration issues. All attributes marked with a red asterisk (*) indicate required fields.

Note: The Debug Settings tab appears when you select the Enable debug mode checkbox in the Operational Settings section of the Parameters tab.

Configure the following debug mode setting under the ADP Workforce Now Workers Debug Setting section:

  • ADP Workforce Now filter: Select the ADP Workforce Now filter to filter the workers.
  • Associate ID: Enter the associate ID to filter the workers in ADP Workforce Now.
  • Position ID: Enter the position ID to filter the workers in ADP Workforce Now.

Notifications

This tab enables you to configure the email notification settings.

Note: The Notifications tab appears when you select the Enable notifications checkbox in the Operational Settings section of the Parameters tab.

Configure the following notification settings under the Notification Settings section:

  • Admin email recipients: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Email subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Email Message: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select to attach the integration report to the email notifications for all integration runs.

Attribute Map

The ADP Workforce Now to ServiceNow integration maps the ADP Workforce Now and ServiceNow attributes. You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

You will see the following two attribute map tables:

Onboarding Map

This tab displays the attribute mapping for creating an onboarding ticket in ServiceNow.

Offboarding Map

This tab displays the attribute mapping for creating an offoarding ticket in ServiceNow.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
