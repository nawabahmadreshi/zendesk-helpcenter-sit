Overview

Aquera offers various solutions to help integrate users from any HR system to Okta. 

This guide outlines the different solutions provided by Aquera, helping you choose the best approach for your integration needs. You can choose the most suitable solution after analyzing the benefits and drawbacks of each solution.

You can either use the Okta XaaS integration by creating a custom identity source application in Okta or use the Okta API integration.

Integration Solutions

The following are the two integration solutions provided by Aquera:

  • Okta XaaS Integration
  • Okta Integration

Both integration solutions have two variants: Basic and Advanced. The Basic integration serves as a plug-and-play solution that necessitates no/minimal technical support for deployment. The Advanced integration solution provides greater flexibility by allowing you to set or modify the user attributes mapping in the Aquera Admin portal according to your requirements. You may need Aquera technical support team to set up the Advanced integration. 

Okta XaaS Integration

In this integration, you need to create a custom identity source application in your Okta tenant and then set the mapping of attributes from the application user profile to the Okta user profile. The main advantage of this integration is it can support bulk imports of a large volume of users from the identity source application.

When the integration runs, the users are created in the custom identity source application in Okta and you can then individually assign the application users to the Okta directory. 

If you are planning to perform downstream provisioning to other applications from Okta, this integration is the recommended approach.

Okta API Integration

In this integration, you can completely set the mappings and control the integration within the Aquera Admin portal. No configurations are required in your Okta tenant for this integration.

When the integration runs, the users are created directly in the Okta directory.

Parameters

Approximate Deployment Time

This is the approximate time taken to deploy the integration solution and make it work from the time of purchase of the solution.

Required Level of Technical Knowledge

This parameter indicates the level of technical knowledge that you need to setup the integration. If the required level of technical knowledge is Medium or Expert and you do not have a technical personnel to assist you, you can request Aquera Support team for the deployment.  

Require Aquera Technical Support

This parameter indicates if the solution can be deployed with or without help from the Aquera Support team. Aquera Support team can help in setting up the mapping for the user attributes from the source application to Okta.

Customization Options

This parameter indicates if customization options are available to you for the solution. For a plug-and-play solution, no customization is available. For the solutions that have customizations available, you have the option to set or modify the mapping of attributes from the source system to Okta. 

Bulk Imports of Users

This parameter indicates if the integration supports bulk user creation and bulk user updates.
