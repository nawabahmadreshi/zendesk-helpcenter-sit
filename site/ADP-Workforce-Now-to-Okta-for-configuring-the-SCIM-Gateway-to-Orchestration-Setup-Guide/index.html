Overview

This guide covers the end-to-end tasks involved in setting up the SCIM gateway to Orchestration from ADP Workforce Now to Okta.

The following tasks are involved in the ADP Workforce Now - Okta for configuring SCIM gateway to Orchestration:

  • Configure Applications in Aquera Admin
  • Setup Orchestration for provisioning users
  • Setup Orchestration for deprovisioning users
  • Setup Orchestration to synchronize existing data

Configure Applications in Aquera Admin

You must add an Okta application and configure the Okta application in the Aquera Admin console.

Refer to the Okta Configuration Guide for more details.

Setup Orchestration for provisioning users

Set up an Orchestration for provisioning users using the Synchronization orchestration method.

  • Synchronization Orchestration - In this Orchestration, set ADP Workforce Now as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

Build the Orchestration

Set the following configurations in the Synchronization orchestration:

  • Set the Admin Email List to receive email notifications for success or failures during the Orchestration run.
  • Set the Creation Distribution List to the IT admin or manager to receive user creation notification emails.
  • Enter the Create condition as SOURCE.active and Update condition as false.
  • Enter the same Match expression used in Okta for the SCIM gateway.
  • Create lookup tables based on location and department values from the source.

Create Attribute Mapping

  • Map the attributes in attribute mapping according to customer requirements.
  • Set the map criteria as Map on Create for all attributes.

Okta Setup

Once Attribute mapping is finalized, refer to the following section to set up the customer Okta environment.

Change the Source Priority for the Username in Okta

  • Log in to your Okta portal.
  • Navigate to Directory on the left menu.
  • Click Profile Editor.
  • Click User (default).
  • Navigate to the Attributes section.  
  • Click the i symbol for the attribute Username.
  • Set the parameter value for Source Priority to Inherit from Okta.
  • Click Save Attribute.

Execute the Orchestration

  • Execute the Orchestration in Report Only mode for one user.
  • Enter the Source SCIM filter to verify the user creation for one user.
  • Execute the Orchestration in Report Only mode for all users. Review the report.
  • Run a full sync (without selecting the checkbox for Incremental mode). Review the Orchestration output.
  • Select the checkbox for Incremental mode and schedule the create Orchestration according to customer requirement.
  • Verify the onboarding users with a future start date and update the number of days in the respective field for ADP configuration in the Aquera Admin console.

Setup Orchestration for deprovisioning users

Set up an Orchestration for deprovisioning users using the Synchronization orchestration method.

  • Synchronization Orchestration - In this Orchestration, set ADP Workforce Now as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

Build the Orchestration

Set the following configurations in the Synchronization orchestration:

  • Set the Admin Email List to receive email notifications for success or failures during the Orchestration run.
  • Set the Deactivation Distribution List to the IT admin or manager to receive user deactivation notification emails.
  • Enter the Create condition as false and Update condition as SOURCE.active == false && DEST.active && SOURCE[IAN].status != "Active".
  • Enter the same Match expression used in the Setup Orchestration for provisioning users.

Create Attribute Mapping

  • Map the attributes in attribute mapping according to customer requirements.
  • Set the map criteria as Map on Update for all the attributes.

Execute the Orchestration

  • Optional: Check with customer, if there are any user account available for deprovisioning for test purpose and use the Source SCIM filter to test the same.
  • Execute the orchestration in Report Only mode for all users. Review the report.
  • Run a full sync (without selecting the checkbox for Incremental mode). Review the Orchestration output.
  • Select the checkbox for Incremental mode and schedule the create orchestration according to customer requirement.

Setup Orchestration to synchronize existing data

Set up the orchestration to synchronize existing data using any of the below options:

  • Set up a new Orchestration for Update.
  • Modify the Orchestration for provisioning users for Create and Update.

Setup Orchestration for Update

Set up the orchestration for Update using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set ADP Workforce Now as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

Build the Orchestration

Set the following configurations in the Synchronization orchestration:

  • Set the Admin Email List to receive email notifications for success or failures during the Orchestration run.
  • Set the Creation Distribution List to the IT admin or manager to receive user creation notification emails. 
  • Set the Deactivation Distribution List to the IT admin or manager to receive user deactivation notification emails.
  • Enter the Create condition as false and Update condition as SOURCE.active.
  • Enter the same Match expression used in Okta for the SCIM gateway.
  • Create lookup tables based on location and department values from the source.

Create Attribute Mapping

  • Map the attributes in attribute mapping according to customer requirements.
    Note: Please verify with customer on update of EMAIL and USERNAME in Okta. Updating Username or Email will impact users access to Okta and other downstream applications.
  • Set the map criteria as Map on Update for all attributes.

Okta Setup

Once Attribute mapping is finalized, refer to the following section to set up the customer Okta environment for the all the attributes in the Update Orchestration. 

Note: Below is the sample for one attribute update in Okta, repeat the below steps for all the attributes in the Update orchestration.

Change the Source Priority for the First name in Okta

  • Log in to your Okta portal.
  • Navigate to Directory on the left menu.
  • Click Profile Editor.
  • Click User (default).
  • Navigate to the Attributes section.  
  • Click the i symbol for the attribute First name.
  • Set the parameter value for Source Priority to Inherit from Okta.
  • Click Save Attribute.

Execute the Orchestration

  • Execute the orchestration in Report Only mode for one user. 
  • Enter the Source SCIM filter to test the attribute for one user. 
  • Execute the orchestration in Report Only mode for all users. Review the report.
  • Run a full sync (without selecting the checkbox for Incremental mode). Review the Orchestration output.
  • Select the checkbox for Incremental mode and schedule the create orchestration according to customer requirement.

Modify the Orchestration for provisioning users for Create and Update 

Build the Orchestration

Select the orchestration for provisioning users and modify the Update condition as SOURCE.active.

Create Attribute Mapping

  • Map the attributes in attribute mapping according to customer requirements. 
    Note: Please verify with customer on update of EMAIL and USERNAME in Okta. Updating Username or Email will impact users access to Okta and other downstream applications.
  • Set the map criteria as Map on Create and Update for all attributes.

Okta Setup

Once Attribute mapping is finalized, refer to the following section to set up the customer Okta environment for the all the attributes in the Orchestration.
