Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration retrieves employee data from Dayforce, identifies managers, and updates a manager flag in Oracle Identity Manager (OIM). It also counts the total number of direct reportees for each manager in Dayforce and updates the number in OIM.

This guide outlines the end-to-end steps required to set up the integration that synchronizes managers' data between Dayforce and Oracle Identity Manager.

To identify records across both systems, the integration uses the employee number attribute from Dayforce and matches it to the employee number attribute in Oracle Identity Manager as a correlation attribute.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Dayforce: Refer to the Dayforce Configuration Guide for configuration details.
    • Oracle Identity Manager: Refer to the Oracle Identity Manager (OIM) REST Connector Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating Dayforce with Oracle Identity Manager. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Oracle Identity Manager. - Confirm the name
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Oracle Identity Manager connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

User Management

The User Management section enables you to customize the integration settings for your specific use case.
Note: This section appears only if the Enable User Management setting is enabled in the Parameters section.

It has the following fields:

  • Source Settings / Dayforce Settings: This section allows you to configure settings for Dayforce. The following parameters are displayed:
    • Source Correlation Attribute / Dayforce Correlation Attribute: Identifies the attribute in Dayforce used to uniquely identify and correlate users between Dayforce and OIM. By default, employeeNumber is selected.
    • Import Limit: Enter the maximum number of records to be retrieved from Dayforce during each integration execution. By default, the value is 100.
    • Sync only Active Manager: Select this checkbox to sync only active managers. When enabled, the integration processes and updates data only for managers who are currently active. Inactive managers are ignored.
  • Destination settings / Oracle Identity Manager Settings: This section allows you to configure settings for OIM. The following parameters are displayed:
    • Destination Correlation Attribute / Oracle Identity Manager Attribute: Identifies the attribute in OIM used to uniquely identify and correlate users between Dayforce and OIM. By default, employeeNumber is selected.
    • Automatically clear manager status when source indicates no longer a manager / Auto-remove Manager Status: Select this checkbox to automatically remove the manager flag in OIM and clear the direct report count if the source indicates that the user is no longer a manager.
    • Custom Field To Update Count / Attribute Name for Reportee Count: Enter the attribute name in Oracle Identity Manager where the total number of direct reportees for each manager will be stored. For example, enter customNumberofreportees if that is the attribute used in OIM to store the reportee count.
    • Is Manager / Attribute Name for Manager Flag: Enter the attribute name in Oracle Identity Manager where the manager flag should be updated. This attribute will be set to indicate whether the user is a manager based on the source data.
