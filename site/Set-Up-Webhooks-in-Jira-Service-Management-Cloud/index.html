Set Up Webhooks in Jira Service Management Cloud

← Back to index

Overview

Aquera integrations can be triggered when specific events occur in an application. An event notification will be sent from the application when data changes in the application, such as the creation of a new user, modification of an existing user, activation or deactivation of a user, addition of a user to a group, or removal of a user from a group. We need to configure webhooks in the application to receive these notifications.

To receive event notifications from JIRA, subscribe to the webhooks in the JIRA application. Once subscribed, JIRA will send notifications whenever the specified events occur. These are HTTPS REST calls containing event data in JSON format and are sent to the webhook URL configured in the integration on the Aquera Admin portal.

Aquera integration then processes these events and updates the corresponding data in a target application.

For details on how to set up an integration in the Aquera platform, refer to the Integration Using Scripts guide.

This guide explains the procedure to set up webhooks in Jira.

Setting up a Webhook

Perform the following steps to set up the webhooks in JIRA Service Management Cloud:

  1. Log in to your Jira Cloud administrator account.
  2. Navigate to Webhook Settings using one of the following methods:
    • Direct URL: Replace <your-site> with your Jira domain in the following link and open it in a browser:
      https://<your-site>/plugins/servlet/webhooks
      For example: https://aquera-test.atlassian.net/plugins/servlet/webhooks
    • JIRA UI

      1. Click the Gear icon in the top-right corner.

      2. Select System from the menu.

      3. In the left-hand panel, scroll down to Advanced and click WebHooks.

  3. Click Create a WebHook. The Create WebHook page is displayed.
  4. Enter values for the following fields:
    Field Description
    Name Enter a descriptive name for the webhook. For example, Integration Listener for Issue Events
    Status Check Enabled to activate the webhook.
    URL Enter your webhook endpoint URL. For example, https://example.com/rest/webhooks/webhook1
  5. Select the Events as per your use cases to subscribe:
    For example, to receive notifications for created, updated, and deleted issues and comments, select the following events:
    • Issue: Created, Updated, and Deleted
    • Comment: Created, Updated, and Deleted

    Note    : You can specify a JQL query or project filters to trigger the webhooks only for specific issues.
  6. Click Create to save the webhook.
