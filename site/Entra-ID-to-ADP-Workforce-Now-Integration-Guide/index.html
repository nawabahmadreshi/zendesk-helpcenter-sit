Overview

The Aquera platform enables your organization to scale complex data management by automating the integration between various applications. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the Entra ID to ADP Workforce Now integration. This integration helps synchronize users' work email addresses from Entra ID to ADP Workforce Now.

To identify user records across both systems, the integration uses the employeeId attribute from Entra ID and matches it to the workerID.idValue in the ADP Workforce Now as a correlation attribute.

This integration compares the work email addresses of users between Entra ID and ADP Workforce Now systems and performs the following actions:

  • If the work email address is available in Entra ID but is missing in the ADP Workforce Now, the integration adds the email address in ADP Workforce Now.
  • If the work email address in the ADP Workforce Now differs from the Entra ID, the integration updates the email address in ADP Workforce Now to match the record in Entra ID.
  • If the work email address is not available in Entra ID but is available in the ADP Workforce Now, the integration deletes the email address from the user record in ADP Workforce Now.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin portal for the integration:
    • Entra ID: Ensure that you have configured an Entra ID connector in the Aquera Admin portal. Refer to the Entra ID Configuration Guide for details.
    • ADP Workforce Now: Ensure that you have configured an ADP Workforce Now connector in the Aquera Admin portal. Refer to the ADP Workforce Now Configuration Guide for details.

Integration Setup

This section outlines the configuration required to integrate ADP Workforce Now with Entra ID, enabling the transfer of workers' data.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Entra ID to ADP Workforce Now.
  5. Enter a name and a description for the integration.
  6. Select your Entra ID and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit or remove these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Report Generation: Select this checkbox to generate the integration execution report. If you prefer not to generate a report, leave the checkbox unselected.
  • Select the Report Detail Level: Select the level of detail for the generated report as follows:
    • Full: Select Full if you want all details in the report.
    • Basic: Select Basic if you want only the basic details in the report.
  • Enable Dashboard Generation: Select this checkbox if you want to analyze the integration-related metrics in a dashboard.
  • Select the Analytics Detail Level: Select the level of detail to be included for analytics and dashboard reporting.
    • Full: Select Full if you want all details to be included in the integration-related metrics in a dashboard.
    • Basic: Select Basic if you want only the basic details to be included in the integration-related metrics in a dashboard.
  • Recipients' Email Addresses: Enter the comma-separated email addresses of the recipients to whom the email notifications will be sent.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors.
  • Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.

Tables

In the Tables tab, you can add tables to the integration. Refer to the Using Tables in Integrations guide for more details.

Trigger

In the Trigger tab, you can schedule the execution of the integration.

You can identify one of the following types to trigger the execution:

  • Event: You can identify an event source to trigger the execution.
  • Schedule: You can specify when the integration should run automatically. Refer to the Integration Schedule Setup Guide for further details.

Run Integration

You can set up events or schedule triggers to run the integration. For more details, refer to the Integration Using Scripts guide.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
