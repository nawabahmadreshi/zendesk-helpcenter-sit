Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration synchronizes workforce lifecycle events from an HR application to a service desk application and automatically creates, updates, or manages service requests and tickets based on workforce changes.

By using the HR application as the system of record, the integration enables organizations to automate service desk fulfillment processes throughout the workforce lifecycle, including candidate onboarding, employee onboarding, job changes, leave of absence, termination, and rehire events. The integration supports configurable lifecycle workflows, event-based request generation, automated ticket management, and multi-channel notifications, helping IT and business teams respond efficiently to workforce changes.

Key Functions

This integration automates employee lifecycle management with the following capabilities:

  • Automates workforce lifecycle processes, including onboarding, job changes, leave of absence, termination, and rehire events.
  • Creates service requests and tickets based on workforce lifecycle events.
  • Supports configurable lifecycle event timing, including before, on, or after the effective event date.
  • Automatically updates existing requests and tickets when workforce information changes.
  • Enables configurable notifications through multiple communication channels.
  • Generates detailed logs, operational reports, and execution summaries.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • HR Application Connector: You must configure a connector for your HR application in the Aquera Admin Portal. Refer to the configuration guide for the respective HR application for configuration details.
    • Service Desk Application Connector: You must configure a connector for your service desk application in the Aquera Admin Portal. Refer to the configuration guide for the respective service desk application for configuration details.

Integration Setup

This section outlines how the integration must be configured to synchronize workforce lifecycle events from an HR application to a service desk application and automate the creation and management of service requests and tickets.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Any HR Application / Any Service Desk in the search box next to the Explore Templates section and click it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select your HR Application and Service Desk Application connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Once created, the following tabs are displayed:

  • Overview
  • Features
  • Source
  • Service Desk
  • Operations

Overview

On this tab, you can view general information about the integration, including its purpose, functionality, and supported capabilities. This section provides a high-level overview of how the integration operates between connected applications.

Features

The Features tab allows you to enable or disable the major functional components of the integration. These settings determine which integration capabilities are available and which configuration tabs are activated.
Note: The following settings are mandatory for this integration and are enabled by default. As a result, they cannot be modified or disabled.

HCM Bridge

The HCM Bridge section controls workforce lifecycle processing within the integration.

  • Enable HCM bridge: Enable this option to process workforce lifecycle events from the source application, including onboarding, organizational changes, and termination events. When enabled, the Source section is displayed for further configuration.

Service Desk Bridge

The Service Desk Bridge section controls service requests and ticket automation within the integration.

  • Enable service desk bridge: Enable this option to create and manage service requests or tickets based on workforce lifecycle events. When enabled, the Service Desk section is displayed for further configuration. 
    The following option is available:
    • JML lifecycle event: Select this checkbox to create and manage service desk tickets for Joiner, Mover, and Leaver (JML) lifecycle events. When enabled, the Lifecycle Event section is displayed for further configuration

Operations

The Operations section controls the operational execution capabilities of the integration.

  • Enable operations: Enable this option to allow the integration to run scheduled sync jobs and on-demand operational workflows.

Source

The Source tab enables you to configure the HR application used by the integration and define how workforce data is retrieved and processed. It also provides advanced settings to control change detection and future-dated workforce event processing.

Source Application

The Source Application section allows you to select the HR application connector instance from which workforce data will be retrieved:

  • Source application instance: Select the HR application connector that this integration will connect to for retrieving data. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

Source Configuration

The Source Configuration section allows you to configure how the integration detects workforce lifecycle events and processes future-dated workforce records:

  • Detect changes since (days): Enter the number of days to look back for detecting new or updated records. Only records created or updated within the specified period will be considered.
  • Future hire look-ahead (days): Enter the number of days in advance to include future. Employees with a hire date within the specified period will be retrieved.

Service Desk

The Service Desk section enables you to configure the ITSM application settings used by the integration. It has the following settings:

Application Settings

Configure the following target application settings:

  •  
  • Service Desk Application: Select the ServiceNow application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

Lifecycle Event

Configure the following ticket creation settings for user lifecycle events:

  • Create Tickets for User Lifecycle Events: Enable this option to create tickets when configured user lifecycle events occur automatically. When enabled, the Event Configuration section is displayed for further configuration.

Event Configuration

Configure the following event settings:

If the List View toggle button is disabled, the following parameters are displayed:

  • Select Event Type: Select the user lifecycle event for which service desk tickets should be created. If you do not find your event type in the list, click the refresh icon available next to the field to reload the available events. The following are the available options:
    • Hire: Select this event to create tickets for new employee onboarding.
    • Change: Select this event to create tickets when employee information changes.
    • Leave: Select this event to create tickets for employee leave events.
    • Terminate: Select this event to create tickets for employee termination events.
  • Event Timing: Select when the service desk ticket should be created for the selected lifecycle event. The following are the available options:
    • When Detected: Select this option to create the ticket immediately when the event is detected.
    • Before Start Date: Select this option to create the ticket before the start date.
      • Days Before Joining Date: Enter the number of days before the employee's joining date when the service desk ticket should be created.
    • On Start Date: Select this option to create the ticket on the start date.
    • After Start Date: Select this option to create the ticket after the start date.
      • Days After Joining Date: Enter the number of days after the employee's joining date when the service desk ticket should be created.
  • Enable Conditional Event Detection: Enable this option to define a custom condition for detecting lifecycle events. When enabled, additional configuration options are displayed:
    • Custom Ticket Creation Rule: Define a custom rule to control when tickets are created for the selected lifecycle event. Use filters or expressions to specify the required conditions.
      This field uses Aquera Intelligence (AI Assistant). For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
  • Enable Ticket Creation: Enable this option to automatically create a ServiceNow ticket for configured lifecycle events. When enabled, additional configuration options are displayed:
    • Ticket Type: Select the type of ServiceNow ticket or request to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available tickets. The following are the available options:
      • Service Catalog: Select this option to create and process requests through the ServiceNow Service Catalog.
      • Ticket: Select this option to create and process standard ServiceNow tickets.
    • Ticket Category: Select the ServiceNow ticket category to assign to tickets created for the configured lifecycle event. If you do not find your ticket category in the list, click the refresh icon available next to the field to reload the available tickets. For example, select Application and Account Access.
    • Ticket Type Name: Select the name of the ServiceNow ticket type to create for the configured lifecycle event. If you do not find your ticket type in the list, click the refresh icon available next to the field to reload the available ticket types. For example, select Change Password. 
    • Enable Custom Ticket Matching: Enable this option to customize how employees from the HR system are matched with existing service desk tickets. When enabled, additional configuration options are displayed:
      • Source Correlation Attribute: Select the Oracle Cloud HCM attribute to uniquely identify and correlate users between both systems.
      • Target Correlation Attribute: Select the ServiceNow attribute to uniquely identify and correlate users between both systems.
    • Enable Ticket Auto-Update: Enable this option to automatically update ServiceNow JML tickets only when their current status matches one of the selected ticket statuses. When enabled, additional configuration options are displayed:
      • Ticket Statuses for Auto-Update Evaluation: Select the ServiceNow ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic updates. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Open.
      • Resolved Statuses for Auto-Updated Tickets: Select the service desk ticket statuses that indicate an auto-updated ticket has been completed or resolved. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses.
    • Enable Ticket Auto-Close: Enable this option to automatically close the ServiceNow JML ticket after all actions associated with the user lifecycle event have been completed successfully. When enabled, additional configuration options are displayed:
      • Ticket Statuses for Auto-Close Evaluation: Select the ServiceNow ticket statuses that should be evaluated to determine whether a ticket is eligible for automatic closure. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
      • Resolved Statuses for Auto-Closed Tickets: Select the ServiceNow status that will be assigned to the ticket when it is automatically closed. If you do not find your ticket status in the list, click the refresh icon available next to the field to reload the available ticket statuses. For example, select Closed Complete.
    • Attribute Mapper: This section allows you to control how each attribute is synchronized between Entra ID and the target application.
      Source attributes are listed on the left side and the target attributes are listed on the right side of the page. You can modify only the source attributes as needed, to align with your specific use case.
      JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
      Attributes marked with a red asterisk (*) are required for successful provisioning.
      The Search Attributes field allows you to filter the list of available attributes for mapping.
      After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.
    • Enable Ticket Creation Through Notification: Enable this option to create tickets through email notifications when the configured lifecycle event is detected. When enabled, additional configuration options are displayed:
      Note: When a configured lifecycle event is triggered, the integration sends a notification to the configured recipients. The recipient can then create the ServiceNow ticket based on the information provided in the notification.
      • Ticket Creation Email Subject: Enter the content that must be used as the email subject in the notifications sent for ticket creation.
      • Ticket Creation Email Recipients: Select one or more recipient groups to whom the email notifications will be sent for ticket creation.
      • Custom Recipient Email Addresses: Enter the email addresses that should be included as custom recipients for notifications sent for ticket creation.
      • Ticket Creation Email Template: Enter the content that must be used as the email body in the notifications sent for ticket creation.

To add another lifecycle event in this integration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the Lifecycle Event section in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row, and you will be redirected to the edit mode, where you can make changes.

Notification Configuration

This section allows you to define notification settings for provisioning events.

If the List View toggle button is disabled, the following parameters are displayed:

  • Notification Trigger Events: Select one or more integration events that should trigger an email notification. The following are the available options:
    • Success: Select this option to send an email notification when the integration completes successfully.
    • Failure: Select this option to send an email notification when the integration fails.
    • Skipped: Select this option to send an email notification when the integration is skipped.
  • Custom Recipient Email Addresses: Enter the email addresses that should be included as custom recipients for notifications sent for the triggered event.
  • Notification Email Subject: Enter the content that must be used as the email subject in the notifications sent for the triggered event.
  • Notification Email Template: Enter the content that must be used as the email body in the notifications sent for the triggered event.
  • Send Integration Report via Email: Select to receive email notifications with a CSV report for all integration runs.
    • Select Notification Report Type: Select the notification report type. This will decide the level of detail to be included in reports. The following are the available options:
      • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
      • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
      • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
      • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
      • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
      • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
    • Select Notification Report Detail Level: Select the notification report detail level. The following are the available options:
      • All: Select this to include all attribute details in reports.
      • Inclusion List: Select this if you want to choose attributes to be included in reports. When selected, the following field is displayed:
        • Fields to include in reports: Select the attributes to be included. For example, Email.
      • Exclusion List: Select this if you want to choose attributes to be excluded from the reports. When selected, the following field is displayed:
        • Fields to exclude from reports: Select the attributes to be excluded. For example, FirstName.
