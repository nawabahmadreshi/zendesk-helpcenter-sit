Bullhorn Candidates to UKG Ready Employees Integration Guide

← Back to index

Overview

The Aquera platform enables your organization to efficiently scale complex data management by automating the integration of diverse resource data across various applications and systems.

The integration synchronizes candidate data, cost centers, pay information, base compensation, job/placement records, and associated organizational structure from Bullhorn to UKG Ready. Any changes made to Bullhorn, including updates to candidates, their active placements, and connected job or company records, are automatically detected and reflected in UKG Ready.

Key Functions

The Bullhorn to UKG Ready Data integration provides the following functions:

  • Fetches candidate data, cost center assignments, pay information, base compensation, job/placement records, and company structure from Bullhorn.
  • Retrieves corresponding employee and related records from UKG Ready for comparison and processing.
  • Compares data between Bullhorn and UKG Ready using attribute mappings to identify:
    • Records to insert (exist in Bullhorn, missing in UKG Ready)
    • Records to update (exist in both, with differences)
    • Records to delete (exist in UKG Ready but missing in Bullhorn, if deletion is enabled)
  • Performs insert, update, and delete operations in UKG Ready in configurable batches.
  • Supports incremental processing by fetching and syncing only changed records since the last successful run.
  • Provides configuration options to enable or disable insert, update, and delete operations.

Integration Setup

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Bullhorn / UKG Ready.
  5. Enter a name and a description for the integration.
  6. Select your Bullhorn and UKG Ready connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Log Insights: Select to analyze integration-related metrics.
  • Enable Debug Logging: Select this option to generate logs that outline data changes for debugging purposes.
  • Enable Notifications: Select to enable the integration-related notifications.

Report Settings

Configure the following report settings:

  • Select the report detail level: Set the level of detail for the report output
    • Summary only: Attribute details are not populated.
    • Full: All attribute details are populated.
    • Custom: Includes or excludes selected attribute details.

Logging Settings

Configure the following log settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu. 
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes.
      Note: The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: Controls which event attributes are included or excluded. If you select Included Attributes from the dropdown, the Included Attributes field will appear. If you select Excluded Attributes from the dropdown, the Excluded Attributes field will appear.
        • Included Attributes: Include only selected attributes in the log.
        • Excluded Attributes: Include all attributes except the selected ones in the log.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details option.
  • Enable Debug Mode: Select this checkbox to generate logs that outline data changes, enabling debugging.

Log Insights Settings

Configure the following log insights settings:

  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu.
  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following option only appears when the log insights level is set to Custom:
      • Attribute Filter: Controls which event attributes are included or excluded. If you select Included Attributes from the dropdown, the Included Attributes field will appear. If you select Excluded Attributes from the dropdown, the Excluded Attributes field will appear.
        • Included Attributes: Select the attributes to include in the analysis.
        • Excluded Attributes: Select the attributes to exclude from the analysis. 

  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details or Custom option.

Notification Settings

Configure the following email notifications settings:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Email Body for Successful Integration Execution: Enter the content that should be used as the email body in notifications sent for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors.

Bullhorn Settings

Configure the following Bullhorn settings:

  • Candidate status: Select the status that must be applied to retrieve the Bullhorn candidates.
  • Placement Status: Select the placement status to consider the candidate placement.
  • Candidate ID: Provide the Bullhorn Candidate ID to run the integration for a single user. When this value is entered, only the candidate associated with the specified ID will be processed during the integration run. Leave this field blank to run the integration for all candidates.

UKG Ready Settings

Configure the following UKG Ready settings:

  • Update Employee Pay Information: Enable automatic update of pay rates and salary details in UKG Ready whenever changes are detected in Bullhorn placements.
  • Enable Cost Center: This setting allows the integration to create and assign cost center information for employees in UKG Ready, based on data from Bullhorn.
  • Select Cost center level to create: Choose which levels of cost center you want the integration to create or update in UKG Ready for each employee.
  • Enable Base Compensation: This setting enables the integration to fetch base compensation details, including hourly rate, salary, and compensation type for each employee.

Attribute Map

The Bullhorn to UKG Ready integration maps the Bullhorn candidates and UKG Ready Employee attributes. Modify the source attributes, if required, based on your use case.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.
