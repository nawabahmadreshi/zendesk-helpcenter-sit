Dayforce ↔ Cornerstone User and Skill Sync

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide covers the end-to-end tasks in setting up the Dayforce to Cornerstone integration to sync the Employees resource of Dayforce to the Users resource of Cornerstone.

This integration helps you sync user data from Dayforce to Cornerstone and skills data from Cornerstone to Dayforce.

To identify user records across both systems, the integration uses the Employee number, Employee ID, Login ID, or XRefCode attribute value from Dayforce. It matches it to the userName in the Cornerstone as a correlation attribute.

Key Functionalities

  • The integration first checks the type of run it should perform (full, incremental, or report-only) and which syncs are enabled (users, skills, or both).

  • It gathers data from Dayforce and downloads all employee records at once in a single export file. From Cornerstone, it retrieves the necessary user or skill data.

  • It aligns the records from both systems and identifies what is new, changed, inactive, or already matches.

  • For users, the integration creates new employees in Cornerstone if an employee exists in Dayforce but not in Cornerstone. It updates records in Cornerstone if details have changed in Dayforce. It deactivates users in Cornerstone if the employee is inactive in Dayforce. Furthermore, it skips the record if no changes are required.

  • For skills, the integration adds new skills in Dayforce if a skill exists in Cornerstone but not in Dayforce. It updates the skill in Dayforce if the skill details are different. It removes the skill from Dayforce if the skill no longer exists in Cornerstone. Furthermore, it skips the record if everything already matches.

Integration Setup

This section outlines how the integration must be configured to push Employees data from Dayforce to Cornerstone.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce ↔ Cornerstone User and Skill Sync.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Cornerstone connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

  • Enable user sync: Updates Cornerstone with employee information from Dayforce.
  • Enable skill sync: Updates Dayforce with employee skills from Cornerstone.

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Incremental Mode: Select this option to sync only the records that have changed since the last run, rather than all records.
  • Enable Logging: Select to generate logs outlining data changes.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Notifications: Select to enable the integration-related notifications.
  • Enable Log Insights: Select to visualize the integration-related metrics.

User Settings

  • Import Limit: Enter the maximum number of employee records to process in a single sync run. For example, set 500 to stop after the first 500 employees.
  • Source Filter: Select which employees to include for syncing. For example, choose All Employees to include every employee, or Specific Groups to filter by criteria.
  • Employee Numbers: Enter the specific employee numbers to include when Source Filter is set accordingly. For example, enter 2069 and 2081 to sync only those employees.
  • Create Condition: Select the rule that determines when new employees are created in Cornerstone. For example, choose Always Create to add all new employees or Only If Active to restrict creation to active employees.
  • Active Status Overrides: Enter additional employee statuses to treat as active during integration. For example, include Maternity or Leave of Absence so those employees are still synced.
  • Update Condition: Select the rule that determines when existing employees are updated in Cornerstone. For example, choose Always Update to update every matched employee each run.

User Correlation Settings

  • Dayforce Field for Correlation: Select the unique Dayforce field used to match employees with Cornerstone records. For example, use EmployeeNumber or EmployeeId.

  • Cornerstone Field for Correlation: Select the unique Cornerstone field used to match employees with Dayforce records. For example, use externalId, userName, or primaryEmail.

Reporting Settings

Configure the following settings to customize the report:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:

    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

    • Full Details: All attribute details are displayed.

    • Custom: Includes or excludes selected attributes. The following options only appear when the report detail level is set to Custom.

      • Attribute Filter: Controls which attributes are included or excluded in the report.

        • Included Attributes: Select the attributes to include in the CSV report.

        • Excluded Attributes: Select the attributes to exclude from the CSV report.

Skills Settings

  • Source Filter: Select which skills from the source system should be included. For example, choose All Skills to bring in every available skill.

  • Create Condition: Define when new skills should be created in the destination system. For example, set to Always Create Skills to add every new skill found in the source.

  • Update Condition: Define when existing skills should be updated in the destination system. For example, set to Always Update Skills to update all matching skills.

  • Import Limit: Specify the maximum number of skills to import during each sync. For example, set to 500 to import up to 500 skills per sync run.

Skills Corelation Settings

  • Dayforce Field for Correlation: Select the unique Dayforce field used to match skills with Cornerstone. For example, use transc_object_id.

    Cornerstone Field for Correlation: Select the unique Cornerstone field used to match skills from Dayforce. For example, use XRefCode.

Log Insights Settings

Configure the following log insight settings:

  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard.
    The following are the available options:

    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

    • Full Details: All attribute details are displayed.

    • Custom: Includes or excludes selected attributes. The following option only appears when the log insights level is set to Custom.

      • Attribute Filter: Controls which attribute details are included or excluded in the analysis.

        • Included Attributes: Select the attributes to include in the insights.

        • Excluded Attributes: Select the attributes to exclude from the insights.

  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details or Custom option.

Logging Settings

Configure the following logging settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: controls which event attributes are included or excluded.   
        • Included Attributes: Include only selected attributes in the log.
        • Excluded Attributes: Include all attributes except the selected ones in the log.
  • Show Attribute Values for Each Event: Select this checkbox to populate the attribute values when choosing the Full Details option.
  • Enable Debug Mode: Select this checkbox to generate logs that outline data changes, enabling debugging.
  • Display History in Logs: Shows the history of changes for each attribute in the logs.

Notification Settings

Configure the following settings for email notifications:

  • Notify Admin Via Email: Check to send email notifications to all admins.
    • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
    • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
    • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Notify Employees via Email: Check to send email notifications to all employees.
    • Employees Email Subject: Enter the content that must be used as the email subject for sending email notifications to the employees
    • Employees Email Notification Template: Enter the content that must be used as the email body for all employee email notifications.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
