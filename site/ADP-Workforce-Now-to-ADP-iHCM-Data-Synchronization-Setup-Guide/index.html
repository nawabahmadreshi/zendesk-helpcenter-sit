ADP Workforce Now to ADP iHCM Data Synchronization Setup Guide

← Back to index

This guide covers the end-to-end tasks in setting up synchronization to sync users from ADP Workforce Now to ADP iHCM.

The following tasks are involved in setting up the synchronization from ADP Workforce Now to ADP iHCM:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running Synchronization

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
  • ADP iHCM: Refer to the ADP iHCM Configuration Guide for details.

Synchronization Setup

This section details how the synchronization must be configured to allow data push from ADP Workforce Now to ADP iHCM. 

It also describes the various analyses and the important points to consider before synchronizing.

Considerations

  • Work email can be retrieved from the ADP Vantage system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from ADP Vantage with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping ADP Vantage user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in ADP Vantage as well as in Okta using one of the following ways:

  • Download the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script synchronization. For scripts, refer to the Appendix.

Building Synchronization

Perform the following steps to set up the synchronization using the Data Synchronization method:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Synchronization and click Create Synchronization. 
  3. Select Data Synchronization from the list of synchronization types.
  4. Enter a name and description for the synchronization in the Create New Synchronization pop-up window.
  5. Click Continue. A new page with the fields and tabs to configure the synchronization is displayed.
  6. Select the ADP Workforce Now connector as the Source Application.
  7. Select Employee as the Source Resource Type.
  8. Select the ADP iHCM connector as the Target Application.
  9. Select Employee as the Target Resource Type.
  10. Click Create to create the synchronization.

Note: For configuration details, refer to the Data Synchronization Setup Guide.

You can set the following configurations in the Data Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables to create lookup tables based on location and department values from the source.
  • Enable Group management if Aquera will manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping the groups, which will be managed by Aquera.

Running Synchronization

  • Enter the matching expression to match the source and destination records.
  • Run the synchronization in Report Only mode and determine if the correct data is updated in ADP iHCM. If the data is not correctly updated, change the matching expression to match users correctly and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. Modify the synchronization configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in ADP V.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow synchronization to update users, you need to disable the setting which sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting which sets Active Directory as profile master.
