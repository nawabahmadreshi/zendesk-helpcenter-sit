Overview

The Aquera platform allows your organization to efficiently scale complex data management by automating the integration of diverse resource data across different applications and systems.

The Genesys to Dayforce Approved Leaves Sync integration automates the process of synchronizing the approved leaves data of employees from Genesys to Dayforce.

To identify matching user records across both systems, the integration uses the displayName and emails attributes of users from Genesys and matches them with the displayName and emails attributes of employees in Dayforce. The matching process first compares the displayName attributes and then the emails attributes across the applications. For data to sync, the attributes in both the source and target applications must match. If even one attribute does not match, the sync will fail.

This integration is triggered based on the schedule that you set under the Schedule tab.

Key Functions

The Genesys to Dayforce Approved Leaves Sync integration provides the following functions:

  • Looks for the approved leaves data in a specified management unit in Genesys.
  • Retrieves the details of all the employee leaves approved within a specified number of past days from Genesys, during the first integration run. In the subsequent runs, only the leaves approved after the last run will be considered for the sync.
    Note: Due to a limitation of the Genesys API, only the leave details from the past 33 days can be retrieved.
  • Creates a CSV report of the changes in Dayforce based on a configuration in the integration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Genesys: Refer to the Genesys Configuration Guide for instructions on creating the Genesys application connector and setting up the authentication.
    • Dayforce: Refer to the Dayforce Configuration Guide for instructions on creating the Dayforce application connector and setting up the authentication.

Integration Setup

This section outlines how the integration must be configured to push approved leaves data from Genesys to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Genesys to Dayforce Approved Leaves Sync.
  5. Enter a name and a description for the integration.
  6. Select your Genesys and Dayforce connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
  • Enable Report Generation: Select this checkbox to generate the integration execution report. If you prefer not to generate a report, leave the checkbox unselected.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Analytics: Select this checkbox to analyze integration-related metrics in a dashboard.

Source Settings

Configure the following source settings:

  • Retrieve Approved Leaves from Past X Days for the first run: Enter the number of past days for which approved leave records must be retrieved from Dayforce during the first integration run. For example, 5. This parameter setting will be ignored in all subsequent runs.
    Note: The maximum value allowed is 33 days. Due to a limitation of the Genesys API, only the leave details from the past 33 days can be retrieved.
  • Management Unit Name(s): Enter the list of management unit names in Genesys from which employees’ approved leave records must be retrieved. If you leave this field blank, the leave data will be retrieved from all management units in Genesys.

Target Settings

Configure the following target setting:

  • Working Hours in a Day: Provide the standard number of working hours in a day to calculate the days to be deducted from the employees' leaves in Dayforce.

Notification Settings

Configure the following settings for email notifications:

  • Send Integration Report via Email: Select this checkbox to send an email with a CSV report of the integration run to the admin. When this checkbox is selected, the following parameters will be displayed:
    • Admin's Email Address: Enter the comma-separated email addresses of the admins to whom the email should be sent.
    • Email Subject: Enter the content that must be used as the email subject.
    • Email Body: Enter the content that must be used as the email body for the email.

Tables

The Tables tab enables you to create a table containing the mapping of leave types between Genesys and Dayforce. The data is provided in the table as a key-value pair, where each key represents the leave type in Genesys and the value is the corresponding leave type in Dayforce. You can populate the table data by either using the Add Row option or uploading a CSV file containing the leave type mapping information.
