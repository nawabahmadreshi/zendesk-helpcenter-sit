Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The ADP Workforce Now to Azure Synapse integration extracts Workers and related resource types such as personal contacts, government IDs, and communication details with their selected attributes from the ADP Workforce Now API Central app, storing the data in the configured Databricks environment. This data can be transferred to Databricks Volume, AWS S3, or Azure Data Lake Storage based on the chosen landing configuration. For each entity, a dedicated folder is created within the specified storage location, containing Parquet or JSON files with the data, along with Delta log files that track metadata about these files.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Databricks: Refer to the Databricks Configuration Guide for details.
    • Azure Data Lake Storage Gen2: Refer to the Azure Data Lake Storage Configuration Guide for details if you select Azure Data Lake Storage as the landing storage in Databricks settings.

Integration Setup

This section outlines how the integration must be configured to push workers data from ADP Workforce Now to Azure Synapse.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now / Azure Synapse in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Operational Settings

The Operational Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Report Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When selected, the following parameter is displayed:
    • Log Level: Select the level of detail to include in the integration logs from the dropdown menu. The following are the available options:
      • Minimum - Operations only; no GUID or fields: Include operation details in the logs. Any field names and GUIDs are excluded.
      • Metadata - Operations + field names; no GUID: Include operation details and field names in the logs. GUIDs are excluded.
      • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the logs. GUIDs are excluded.
      • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the logs.
      • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the logs.
      • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the logs.
  • Enable Reporting: Select this checkbox to generate the integration report. When selected, the following parameters are displayed:
    • Reporting Type: Select the level of detail to be included in the integration reports from the dropdown menu. The following are the available options:
      • Minimum - Operations only; no GUID or fields: Include operation details in the report. Any field names and GUIDs are excluded.
      • Metadata - Operations + field names; no GUID: Include operation details and field names in the report. GUIDs are excluded.
      • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the report. GUIDs are excluded.
      • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the report.
      • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the report.
      • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the report.
    • Report Format Type: Select the format in which the integration report should be created. The available options are CSV and Excel.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When selected, the following parameter is displayed:
    • Log Insights Level: Select the level of detail to include in the log insights dashboard from the dropdown menu. The following are the available options:
      • Minimum - Operations only; no GUID or fields: Include operation details in the log insights dashboard. Any field names and GUIDs are excluded.
      • Metadata - Operations + field names; no GUID: Include operation details and field names in the log insights dashboard. GUIDs are excluded.
      • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the log insights dashboard. GUIDs are excluded.
      • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the log insights dashboard.
      • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the log insights dashboard.
      • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the log insights dashboard.

Notification

The Notification section enables you to customize the integration settings according to your specific use case. It has the following settings:

Enable Email Notification: Select to enable email notifications for the integration.

Email Notification 

  • Select Event Type: Select an event from the dropdown menu for which the email notification must be sent. The available options are Success and Failure.
  • Select Conditional Notification: Select this checkbox to define a condition for sending the notification. When enabled, the following parameter is displayed:
    • Notification Condition:
  • Recipient Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Email Subject Line: Enter the content that must be used as the email subject in the notifications sent for the selected event in the Select Event parameter.
  • Email Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select this checkbox to send the integration report via email. When selected, the following parameter is displayed:
    • Select Report Format: Select the format in which the integration report should be sent via email. The available options are CSV and Excel.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
