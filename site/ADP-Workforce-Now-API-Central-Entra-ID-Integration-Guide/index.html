Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the configuration for the ADP Workforce Now API Central/Entra ID integration. This integration automates provisioning across ADP Workforce Now API Central and Microsoft Entra ID. It synchronizes users and candidate data, manages group assignments, and handles onboarding, updates, and terminations. It also supports automated access assignments and provides detailed notifications, logs, and reports.

To identify user records across both systems, the integration uses candidate attributes from the ADP Workforce Now API Central and matches them to user attributes in Entra ID as a correlation attribute.

Key Functions

The ADP Workforce Now API Central / Entra ID integration provides the following functions:

  • Creation of user
  • Updation of user
  • Deactivation of user
  • Deletion of user
  • Group Assignment
  • Group Revocation

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.

Integration Setup

This section outlines how the integration must be configured to push candidates' data from ADP Workforce Now API Central to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ADP Workforce Now API Central / Entra ID in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The following tabs are displayed for you to configure the integration:

Overview

This tab displays comprehensive details about the integration and its related information.

Source

This tab displays the different sections used to identify the ADP Workforce Now API Central application connector and configure the candidate-related settings.

This tab has the following configuration sections:

Application Setting

This section enables you to identify the ADP Workforce Now API Central application connector used for the integration and configure the candidate-related import settings. The following fields are displayed in this section:

  • Candidate Source Application: Select the ADP Workforce Now API Central application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Retrieve the complete manager profile for candidates: Select to retrieve the manager's profile of all candidates.
  • Retrieve Candidates with Job Applications Submitted 'X' Days Prior: Enter the number of days 
  • Candidate Status(es) to be considered for Account Activation: Select the candidate statuses for which the account will be activated.
  • Candidate Status(es) to be considered for Account De-Activation: Select the candidate statuses for which the account will be deactivated.

Directory

This section enables you to identify the Entra ID application connector used for the integration and configure the group-related settings, account correlation, and other account settings.

This tab has the following configuration sections:

Directory Settings

This section enables you to configure application provisioning. In this section, you can define how records from Microsoft Entra ID are provisioned to the target application and how attributes are mapped between the systems.

Application Configuration

This section enables you to configure the target Entra ID application connector involved in the integration. It also allows you to define how passwords are generated for new users in the directory.

It has the following settings:

  • Entra ID Application: Select the Active Directory application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
    Note: This field identifies the Active Directory application that enables provisioning and synchronization between Dayforce and Active Directory.
  • Enforce Custom Entra ID Password Policy: Select to generate passwords for new users according to the custom Entra ID policies defined in the Policy > Password Policy section. By default, the integration generates a random password for new users. When enabled, the following field is displayed:
    • Select Entra ID Password Policy: Select the password policy to be used for this integration. All password policies created in the Password Policy section are available for selection. If you do not find your password policy in the list, click the refresh icon available next to the field to reload the available connectors.

Groups Settings

This section allows you to configure group settings for users. 

Candidates Group Configuration

The following fields are available for configuration in this section:

  • Remove extra groups assigned to the user during the update: Select this checkbox to remove users from groups when the user is deleted.
  • Remove user from groups on deactivation: Select this checkbox to remove users from groups when the user is deactivated.
  • Transfer group ownership to manager on deactivation or deletion: Select this checkbox to automatically remove a user's group ownership and assign it to their manager when the user is deactivated or deleted. This applies only when the user is the only owner of the group.Entra ID Application: Select the Entra ID application connector that you have configured in the Aquera Admin portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
    Note: This field identifies the Entra ID application that enables provisioning and synchronization between Dayforce and Entra ID.
  • Generate custom passwords for users provisioned to Entra ID: Enable this option to generate passwords for new users according to the custom Entra ID policies defined in the Password Policy section. By default, the integration generates a random password for new users. When enabled, the following field is displayed:
    • Entra ID password policy: Select the password policy to be used for this integration. All password policies created in the Password Policy section are available for selection. If you do not find your password policy in the list, click the refresh icon available next to the field to reload the available connectors.
  • Entra ID enterprise application name: Select the enterprise application that you have created in your Azure portal for this integration. Click the refresh icon next to the field to retrieve all enterprise applications available in your Azure portal.
    Note: This field identifies the enterprise application created in the Azure portal to drive provisioning for this integration.
  • Target application: Select the target application connector where records will be provisioned. For example, if your use case is to provision user records from Entra ID to the Box application, select the Box connector that you have created in the Aquera Admin portal for this integration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors.
  • Customize how users are matched between Entra ID and the target application: By default, user records are matched using the User Principal Name (UPN) in Entra ID and the Username in the target application. Turn on this toggle button to use a different source and target attribute for correlation. When enabled, the following fields are displayed for further configuration:
    • Entra ID attribute for matching users: Select the correlation attribute in Entra ID that uniquely identifies a user record in your target application. For example, select employeeId if you want to use it to match user records in the target application.
    • Target application attribute for matching users: Select an attribute in the target application that matches the selected Entra ID attribute value to identify common users. For example, select ENT.employeeNumber if you want to use it to match user records in Entra ID.
  • Automatically generate password for new users: Turn on this toggle button if your use case requires generating a password for new users in the target application. When enabled, the following fields are displayed for further configuration:
    • Method for password generation: Select the method to generate passwords. This defines how passwords should be generated. The available options are as follows:
      • Generate Random Password: Select this if you want passwords to be generated automatically by the system.
      • Generate Password Using Policy: Select this if you want passwords to be generated based on a defined password policy. When selected, the following field is displayed for further configuration:
        • Policy for password generation: Select the password policy to be used for this integration. All password policies created in the Policy tab will be available for selection.
  • Attribute Mapper: This section allows you to control how each attribute is synchronized between Entra ID and the target application.
    Source attributes are listed on the left side and the target attributes are listed on the right side of the page. You can modify only the source attributes as needed, to align with your specific use case.
    JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the expand icon to write a longer expression.
    Attributes marked with a red asterisk (*) are required for successful provisioning.
    The Search Attributes field allows you to filter the list of available attributes for mapping.
    After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the configuration.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.

The following tables are available:

  • <Table Name>: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options: <Depending upon the integration use case, you can document accordingly>

    • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
    • Upload CSV: Click Upload CSV to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
      Notes:
      • The key name within a table should be unique. The key name can include a combination of letters and numbers.
      • The key name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

    Click Save to save the changes made.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
