Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration facilitates synchronization between Dayforce and Ivanti Neurons, automating ticket creation for onboarding, offboarding, and change management based on employee lifecycle events.

The integration is triggered either on a fixed schedule or through event-based mechanisms. Once triggered, it fetches recent or upcoming employee lifecycle changes such as new hires, terminations, and updates from Dayforce, and then processes them into Ivanti Neurons tickets.

Key Functions

This integration performs the following key tasks:

  • Retrieves employee data from Dayforce, including hires, terminations, and updates.
  • Processes detailed employee profiles (contact, department, job metadata).
  • Matches employees to existing Ivanti records using correlation fields.
  • Creates or updates onboarding, offboarding, and change tickets in Ivanti.
  • Prevents duplicate ticketing by checking for existing records.

Integration Setup

This section outlines the configuration process for integrating Dayforce with Ivanti Neurons using the Aquera platform. It includes the steps for creating the integration, defining parameters, and enabling the automated processing of employee lifecycle events, such as onboarding, offboarding, and profile updates.

This integration continuously syncs employee records from Dayforce, specifically new hires, terminations, and job changes, and dynamically creates or updates corresponding tickets in Ivanti Neurons.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Ivanti Neurons in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Integration Feature

Configure the following integration scenario:

  • Enable Ticket Management: Select to automatically create Ivanti Neurons tickets for Dayforce hires, terminations, and updates. When selected, the Manage Ticket section will be displayed.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes. The Logging section will be displayed only when this checkbox is selected.
  • Enable Reporting: Select to generate the integration report. The Reporting section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select to visualize the integration-related metrics. The Log Insights section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select to enable integration-related notifications. The Notifications section will be displayed only when this checkbox is selected.

Manage Ticket

Configure the following ticket management settings:

  • HR Events to Trigger Ticketing: Select one or more employee lifecycle events that should initiate ticket creation workflows. The available options from the dropdown are Joiner, Mover, and Leaver.

Target Settings

Configure the following target setting:

  • Username: Enter your Ivanti username used to create service requests.

Joiner

Configure the following joiner ticket settings:

  • Filter Onboarding Users by Status: Select employee statuses that will trigger the creation of a joiner ticket. For example, choose Active and Prestart so a joiner ticket is created when a candidate’s status changes to Active or Prestart.
  • Days Before Hire to Create Ticket: Enter the number of days before the hire date to create the ticket. For example, set 10 to create the ticket ten days before the employee’s official start date so that IT can prepare the necessary equipment.
  • Select the Ticket Type: Select the ticket type for new hire requests. For example: Select Service Request to open a new-hire service request in Ivanti.
  • Create Limit: Enter the maximum number of tickets that can be created. For example: Set 100 to cap the total number of joiner tickets generated each month.
  • Request Template: Select the template used to create service requests. For example: Choose AABC_AIM_HR Onboarding_Return 2025-08-21 to use the standard onboarding template that pre-populates workstation and application access fields.
  • Select User Pay Type to Create Service Request: Select the Dayforce user pay type to create a service request in Ivanti. For example, Select Hourly so that tickets are created only for hourly workers who require different provisioning than salaried staff.

Leaver

Configure the following leaver ticket settings:

  • Filter Offboarding Users by Status: Select the employee statuses that trigger a leaver ticket. For example, select Active and Terminated so that a leaver ticket is triggered when an employee is marked for termination.
  • Days Before Termination to Create Ticket: Enter the number of days before the termination date to create the ticket. For example, Set 7 to create the ticket a week before the termination date to schedule equipment return and access removal.
  • Select the Ticket Type: Select the ticket type for employee departure requests. For example: Select Service Request to create an offboarding service request.
  • Create Limit: Enter the maximum number of tickets that can be created. For example, Set 100 to allow no more than one hundred leaver tickets during a large layoff cycle.
  • Request Template: Select the template used to create service requests. For example, choose AABC_HR Offboarding/Leave 2025-08-26 to use the standard offboarding checklist that includes badge deactivation.
  • Select User Pay Type to Create Service Request: Select the Dayforce pay type that triggers a service request in Ivanti. For example, select Salary so that the ticket is generated only for salaried employees who require laptop retrieval.

Mover

Configure the following mover ticket settings:

  • Filter Transfer Users by Status: Select the employee statuses that trigger a mover ticket. For example, select Maternity, Paternity, Pensioner, and Inactive so a mover ticket is created when an employee moves into any of these statuses during an internal transfer.
  • Days After Transfer to Create Ticket: Enter how many days after the transfer date to create the ticket. For example: Set 100 to create the ticket one hundred days after the transfer to follow up on post-move access adjustments.
  • Select the Ticket Type: Select the ticket type for employee transfer requests. For example, choose Service Request to open a transfer-related service request.
  • Create Limit: Enter the maximum number of tickets that can be created. For example, set 1 ensures that only one mover ticket is created per employee transfer.
  • Request Template: Select the template used to create service requests. For example, choose HR_Transfer_Access_Update to request updates for access and department distribution lists.
  • Select User Pay Type to Create Service Request: Select the Dayforce pay type that triggers a service request in Ivanti. For example, choose Contractor so tickets are generated only for contractors who change departments and need new system permissions.

Logging

Configure the following logging settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: controls which event attributes are included or excluded.   
        • Included Attributes: Include only selected attributes in the log.
        • Excluded Attributes: Include all attributes except the selected ones in the log.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select this checkbox to populate the attribute values when choosing the Full Details option.
  • Enable Debug Mode: Select this checkbox to generate logs that outline data changes, enabling debugging.

Reporting

Configure the following settings to customize the report:

  • Select the Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following options only appear when the report detail level is set to Custom.
      • Attribute Filter: Controls which attributes are included or excluded in the report.
        • Included Attributes: Select the attributes to include in the CSV report.
        • Excluded Attributes: Select the attributes to exclude from the CSV report.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details or Custom option.
  • Enable Integration Report Download: Select to allow downloading the integration report.

Notifications

Configure the following settings for email notifications:

  • Admin's Email Address: Enter one or more admin email addresses that should receive notifications about integration executions, including success, failure, and CSV reports.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Email Body on Successful Integration Execution: Enter the content that must be used as the email body in the notifications sent for successful integration execution.
  • Email Body on Failed Integration Execution: Enter the content that must be used as the email body in the notifications sent for integration errors.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a report for all integration runs.
  • Notify Admin Via Email: Check this checkbox to send email notifications to all admins.

  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details or Custom option.

Attribute Mapping

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
