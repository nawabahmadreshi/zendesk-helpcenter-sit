Overview 

The Aquera platform empowers organizations to efficiently manage complex data workflows by automating integration across diverse resource types. This integration enables synchronization between Dayforce and Jira Service Management Server, automating onboarding and offboarding ticket creation for employees. 

It continuously retrieves and reconciles employee lifecycle events from Dayforce, such as hires and terminations, with Jira, ensuring timely and accurate issue creation.

This integration performs the following tasks:

  • Retrieves employee data from Dayforce, supporting incremental updates based on hire and termination date ranges.
  • Accesses and processes detailed employee records, including contact info, department, and job metadata.
  • Detects new hires and leavers, then correlates them with existing Jira tickets using employee numbers or work email addresses.
  • Automatically creates onboarding or offboarding Jira tickets using mapped field values, supporting both Jira Cloud and Jira Server environments.
  • Prevents duplicate tickets by checking existing issues in Jira and skipping already-tracked employees.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Dayforce: Refer to the Jamf Configuration Guide for instructions on creating the Jamf Pro application connector and setting up the authentication.
    • Jira : Refer to the Paylocity Configuration Guide for instructions on creating the Paylocity application connector and setting up the authentication.
