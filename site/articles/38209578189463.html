Writer Guide: How to Use ISH Documentation Template

← Back

Overview

This template helps you write Aquera documentation that looks clean, consistent, and easy to navigate. The page layout, fonts, spacing, colours, and table of contents behaviour are already built into the template. You do not need to understand web development to use it safely.

Your job is to add content using the approved content patterns shown below. If you follow these patterns, the document will look correct, and the navigation will work.

Editing Basics

You can edit the documentation by opening the project folder in VS Code and updating the main HTML file. You do not need web development knowledge. You only need to edit text, headings, links, and image paths.

  1. Open VS Code.
  2. Click File → Open Folder.
  3. Select the documentation project folder (for example, ish-template).
  4. In the left file panel, click the main HTML file (for example, index.html).
  5. Update only the content within the approved editable areas.
  6. Press Command + S to save.
  7. Open the HTML file in your browser to preview your changes.

Note: Do not edit any <style> or <script> blocks unless you are maintaining the template itself.

Preview in a Browser

After you save your changes in VS Code, you should preview the HTML file in a browser to confirm the layout, table of contents, and scrolling behaviour still work correctly.

  1. In VS Code, press Command + S to save.
  2. Locate the main HTML file (for example, index.html).
  3. Double-click the file to open it in your default browser.
  4. Scroll through the page to confirm the content displays correctly.
  5. Click a few TOC links to confirm the page jumps to the correct section.

Note: If you add new screenshots or videos, confirm the file appears inside the correct folder, and the file name matches the link exactly.

Allowed Edits

You should only edit two areas:

  • The Table of Contents (TOC) list on the left

  • The Main content in the center (sections, text, screenshots, tables, steps)

Disallowed Edits

Do not edit these parts unless you are maintaining the template itself:

  • Any <style> blocks (these control design and layout)

  • Any <script> blocks (these control TOC highlighting and scrolling behavior)

  • Major layout wrappers like the TOC column, main card, and right panel

If these are edited, the TOC can break, scrolling behavior can break, or the page may not display properly.

Navigation using IDs 

What is an ID?

An ID is a unique identifier that serves as a bookmark. The TOC uses IDs to jump to sections and subsections.

If the TOC contains a link to #reports, then the content must contain an element with id="reports".

ID rules (must follow)

Every ID must be:

  • Lowercase only

  • No spaces

  • Use hyphens instead of spaces

  • Unique across the entire page

Recommended IDs

  • overview

  • file-collection

  • create-survey

  • download-report

Not Recommended IDs

  • File Collection (spaces and capitals)

  • createSurvey (camelCase)

  • file_collection (underscore)

Add a new major section (H2)?

Add a major section when you are creating a new top-level chapter of the guide, such as Overview, Settings, Reports, or Survey Management.

Note: If  you have a question where do I add a new TOC entry?. The answer is Add it inside the TOC List, as a new tocLink.

Step 1: Add the TOC link

Add a TOC entry that points to the section ID. Example:

<a class="tocLink" href="#reports">Reports</a>

Step 2: Add the matching section in the main content

Create the section using the approved structure. Example:

<section class="docSection" id="reports">
  <div class="stickySectionHeader">
    <h2 id="reports">Reports</h2>
  </div>

  <p>
    The <strong>Reports</strong> section allows administrators to download export files
    for audits, compliance, and internal tracking.
  </p>
</section>

Note: The same ID appears in two places:

  • section id="reports"

  • h2 id="reports"

This is required. Do not remove either.

Add a subsection (H3)

Use a subsection when you want to break a major section into smaller parts, such as workflows, screen explanations, or feature details.

Subsection example (must include an ID)

<h3 id="download-report">Download a Report</h3>

<p>
  Admins can download reports as CSV files to review asset data, user responses,
  and completion status. Reports can be opened in Excel or Google Sheets.
</p>

TOC subsections (H3)

Important: Subsections do not appear automatically

If you want subsections to show under a section in the TOC, you must add them manually.

TOC example with subsection links

<a class="tocLink" href="#reports">Reports</a>

<div class="tocSub">
  <a class="tocSubLink" href="#download-report">Download a Report</a>
  <a class="tocSubLink" href="#report-statuses">Report Statuses</a>
</div>

How it behaves while scrolling

The template automatically expands and collapses the subsection group as the reader scrolls. When the reader scrolls into Reports, the TOC subsection list under Reports opens. When the reader scrolls into another section, it collapses again.

How to write step-by-step instructions (numbered steps)

Use steps when the reader must perform actions like clicking buttons, selecting menu items, entering values, uploading files, or saving settings.

Approved steps block (use exactly this)

<ol class="steps">
  <li>Navigate to the <strong>Reports</strong> tab.</li>
  <li>Locate the report you want to download.</li>
  <li>Confirm the status shows <strong>Ready</strong>.</li>
  <li>Click <strong>Download</strong>.</li>
  <li>Open the file in Excel or Google Sheets.</li>
</ol>

Steps for writing rules

  • Keep each step to one action whenever possible

  • Start steps with action words like “Click,” “Select,” “Enter,” or “Navigate”

  • Bold UI labels (buttons, tabs, menus)

How to add screenshots

Approved screenshot block (use exactly this)

<figure class="mediaBlock">
  <img class="media" src="images/reports-table.png" alt="Reports table showing download buttons" />
</figure>

Screenshot with caption (recommended)

<figure class="mediaBlock">
  <img class="media" src="images/reports-table.png" alt="Reports table showing download buttons" />
  <figcaption class="caption">
    <strong>Figure:</strong> The Reports table shows all exports and their download status.
  </figcaption>
</figure>

Small screenshots (popups/dialogs)

Use mediaSmall for small popup screenshots:

<figure class="mediaBlock">
  <img class="media mediaSmall" src="images/confirm-download.png" alt="Confirm download popup dialog" />
</figure>

ALT text rules

ALT text must describe what the image shows, e.g.:

  • “Survey dashboard showing active surveys”

  • “Confirm download popup dialog”

10. How to add tables

Table rule

Tables must use class="simpleTable".

Approved table example

<table class="simpleTable">
  <tbody>
    <tr>
      <td><strong>Status</strong></td>
      <td><strong>What it means</strong></td>
    </tr>
    <tr>
      <td>Ready</td>
      <td>The report is complete and can be downloaded immediately.</td>
    </tr>
    <tr>
      <td>Generating</td>
      <td>The report is still being prepared. The download button may be disabled.</td>
    </tr>
    <tr>
      <td>Failed</td>
      <td>The report could not be generated. Admins should retry or contact support.</td>
    </tr>
  </tbody>
</table>

How to add notes

Notes must use class="note".

Approved note example

<p class="note">
  <strong>Note:</strong> Reports may take longer to generate if the tenant contains a large number of users.
</p>

How to write nested bullets (sub → sub → sub bullets in content)

One-level bullets

<ul>
  <li>Admins can create surveys.</li>
  <li>Admins can track completion.</li>
  <li>Admins can export results.</li>
</ul>

Two-level bullets (sub-bullets)

<ul>
  <li>
    Survey statuses include:
    <ul>
      <li><strong>Pending</strong> (not started)</li>
      <li><strong>Completed</strong> (submitted)</li>
      <li><strong>Overdue</strong> (past due date)</li>
    </ul>
  </li>
</ul>

Three-level bullets (sub-sub bullets)

<ul>
  <li>
    Reports can be downloaded in multiple formats:
    <ul>
      <li>
        CSV (recommended for spreadsheets)
        <ul>
          <li>Best for Excel</li>
          <li>Best for Google Sheets</li>
          <li>Best for filtering and sorting</li>
        </ul>
      </li>
      <li>
        PDF (recommended for sharing)
        <ul>
          <li>Best for printing</li>
          <li>Best for audits</li>
        </ul>
      </li>
    </ul>
  </li>
</ul>

Nested bullets inside steps (allowed)

<ol class="steps">
  <li>
    Select the export format.
    <ul>
      <li>Choose <strong>CSV</strong> for spreadsheets.</li>
      <li>Choose <strong>PDF</strong> for printing.</li>
    </ul>
  </li>
  <li>Click <strong>Download</strong>.</li>
</ol>

Writing style rules

  • Bold UI labels so users can quickly match instructions to the product UI
    Example: Click Save, open Settings, select Weekly

  • Use the exact wording from the UI (don’t rename buttons or tabs)

  • Keep paragraphs short and clear (2–4 lines is usually best)

  • Use steps for actions, use bullets for descriptions

Final checklist before you submit your edits

TOC and IDs

  • Every TOC link matches a real ID

  • Every major section (H2) has an ID

  • Every subsection (H3) has an ID

  • No duplicate IDs exist anywhere

Correct blocks

  • Steps use <ol class="steps">

  • Tables use <table class="simpleTable">

  • Screenshots use <figure class="mediaBlock">

  • Notes use <p class="note">
