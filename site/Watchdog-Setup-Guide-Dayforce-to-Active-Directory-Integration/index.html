Watchdog Setup Guide: Dayforce to Active Directory Integration

← Back to index

Overview

Aquera Watchdog is a monitoring and validation tool designed to identify data synchronization issues between integrated applications. It operates independently of the Aquera integration and analyzes application events to detect inconsistencies between source and target systems.

After an integration execution is completed, you can run the Watchdog to retrieve the event logs, analyze the activities, and report any anomalies or data discrepancies across the applications. For example, if a user is created in the source application, Watchdog checks for a corresponding user provisioning event in the target application and reports an anomaly if no matching event is found. Aquera Watchdog helps ensure guaranteed delivery of events across integrated systems by continuously monitoring and validating data synchronization between the source and target applications.

When you run the integration Watchdog, a dashboard is generated with a consolidated view of the activities across the systems, including any detected anomalies. The dashboard can be accessed through the Insights module. Refer to the Watchdog Insights section for the details captured in the dashboard.

Watchdog ensures data consistency across systems, provides visibility into sync failures, and helps proactively identify and resolve integration issues. 

This guide provides details on how to configure and run the Watchdog for the integration that synchronizes employee data from Dayforce to user accounts in Active Directory.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal to connect to those applications for retrieving the event logs:
    • Dayforce: Ensure you add and configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for details.
    • Active Directory: Ensure you add and configure the Active Directory connector in the Aquera Admin portal. Refer to the Active Directory (AD) Configuration Guide for details.

Watchdog Setup

This section describes how to configure the Watchdog that monitors the employee sync from Dayforce to Active Directory.

Create Integration Watchdog

Follow these steps to create the integration watchdog:

  1. Log in to the Aquera Admin Portal.

  2. In the left panel, go to Integrations.

  3. Click Create Integration.

  4. Select Active Directory / Dayforce [Watchdog].

  5. Enter a name and description for the integration.

  6. Choose your Dayforce and Active Directory connector instances.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Dayforce and Active Directory connectors that you have selected while creating the integration watchdog. This section enables you to search and select a different instance of the application connectors, if required.

Parameters

The Parameters section enables you to customize the Watchdog settings according to your use case. It has the following settings:

  • Active Directory user match attribute: Select the Active Directory user attribute that must be used to uniquely identify a user record and match with a Dayforce employee record.
  • Dayforce employee match attribute: Select the corresponding Dayforce employee attribute that must be used to uniquely identify an employee record and match with the Active Directory user record.
  • Initial log lookback period: Specify the time period from which the watchdog retrieves historical logs from Dayforce and Active Directory during its initial execution. After the first run, the watchdog processes only the event logs that have been created since the last run. These logs are used by Watchdog to compare and analyze the events across both systems.

Policy

The Policy tab enables you to set the access policies of users for Organizational Units (OUs), Groups, and Directory. 

Access Policy

The Access Policy section allows you to define policies that determine where users should be placed within specific organizational units (OUs), groups, and the directory.

Organizational Unit Policy

Turn on the Enable toggle to configure the access policies for OUs. The following sections are displayed.

Required Organizational Units

Use this section to define the conditions under which users must be part of specific OUs. Users who meet these conditions are expected to be present in those OUs.

  1. Turn on the Enable toggle to configure the access policies for required OUs.
  2. Click on Choose Attribute to select a user attribute from the Dayforce application that you want to use in the condition.
  3. After selecting the attribute, specify one or more attribute values. Users who match any of the selected attribute values (OR operation) are considered eligible for the OU.
  4. If required, add more attributes and values. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. This means users who satisfy all attribute conditions in that row must be part of the corresponding OU.
  5. In the Organizational Unit field, select the organizational unit where users matching the defined conditions should be present.
  6. If required, click Add Row to add another condition within the same rule. 

  7. If required, click the + icon at the top-right corner of the rule setting section to add another rule with a set of rows/conditions for required OUs. 

    When multiple rules are set, the integration evaluates every rule for each user.

Forbidden Organizational Units

Use this section to define the conditions under which users must not be part of specific OUs. Users who meet these conditions are not expected to be present in those OUs.

The policy setup is similar to the Required Organizational Units procedure.

Group Policy

Turn on the Enable toggle to configure the access policies for groups. The following sections are displayed.

Required Groups

Use this section to define the conditions under which users must be members of specific groups. Users who meet these conditions are expected to be present in those groups.

The policy setup is similar to the Required Organizational Units procedure.

Forbidden Groups

Use this section to define the conditions under which users must not be part of specific groups. Users who meet these conditions are not expected to be present in those groups.

The policy setup is similar to the Required Organizational Units procedure.

Directory Policy

Turn on the Enable toggle to configure the directory access policies. 

You can set the conditions that define which users should and should not have access to the directory.

When you configure the policy, you must select Yes or No in the Can Access? field that indicates whether the user satisfying the condition must get access to the directory or not.

Watchdog Insights

This section details the data covered in the dashboard created by the Watchdog for the Dayforce to Active Directory integration. 

  1. Log in to the Aquera Admin portal.
  2. Navigate to Insights.
  3. Click on the integration watchdog name, for example, Dayforce to Active Directory Watchdog. The dashboards are displayed.

The dashboard provides a comprehensive view of workforce events and their synchronization across Dayforce and Active Directory. It includes the following insights:

  • Displays the count of Joiner, Mover, and Leaver events in Dayforce, along with corresponding user creations, updates, and deactivations in Active Directory. This helps identify any discrepancies in user data between the two systems.
  • A checklist that verifies whether a corresponding Active Directory user record has been created for each employee added in Dayforce.
  • A timeline view illustrating an employee's journey, from hiring in Dayforce to account creation in Active Directory, and from termination to account deactivation.
  • A dashboard showing the upcoming hires and terminations, categorized by departments
  • A trend graph that provides visibility into synchronization patterns or gaps between HR and directory events.
  • A geographical map showing the number of Entra ID user logins across various locations.
  • A list of Entra ID applications with the highest number of user logins.
  • The list of users who have been inactive for a long time.
  • A trend graph showing the most frequently accessed applications over a period of time.
  • A pie chart showing the percentage distribution of workers across different departments.
  • A horizontal bar chart showing the distribution of workers across various managers.
  • Insights into workforce distribution by geographic location, and by department and country.
  • A line chart illustrating hiring trends compared to employee terminations over time.
  • Detailed logs of events in the Dayforce application and the Active directory.
