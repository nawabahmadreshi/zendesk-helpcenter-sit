User Integration Setup Guide - ADP Vantage to Okta

← Back to index

This guide covers the end to end tasks involved in setting up a synchronization to sync users from ADP Vantage to Okta.

The following tasks are involved in setting up the synchronization from ADP Vantage to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running Synchronization

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in Aquera Admin console:

  • ADP Vantage: Refer to the ADP Vantage Configuration Guide for details.
  • Okta: Refer to the Okta Configuration Guide for details.

Synchronization Setup

This section details how the synchronization must be configured to allow data push from ADP Vantage to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the synchronization.

Considerations

  • Work email can be retrieved from the ADP Vantage system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from ADP Vantage with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping ADP Vantage user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in ADP Vantage as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script synchronization. For scripts, refer to the Appendix.

Building Synchronization

This document describes how to set up the synchronization using the User Synchronization method.

  • User Synchronization - In this synchronization, set ADP Vantage as the Source Application and Okta as the Target Application. Refer to the User Synchronization Setup Guide for details on the configurations.

You can set the following configurations in the User Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in restriction list for scoping of the groups which would be managed by Aquera.

Refer to the User Synchronization Setup Guide for details on the configurations.

Running Synchronization

  • Enter the matching expression to match the source records with the destination records.
  • Run the synchronization in Report Only mode and determine if correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. Modify the synchronization configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in ADP Vantage.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow synchronization to update users, you need to disable the setting which sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting which sets Active Directory as profile master.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in ADP Vantage and Okta before setting up the synchronization. 

This document explains how analyses can be done using a synchronization through scripts. In this synchronization, add ADP Vantage and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script synchronization, refer to the Setting Up Synchronization Using Scripts Guide.

Few sample scripts are provided here for your reference. The scripts may vary based on your use case  and the attributes to be retrieved for analysis.

Analyzing the Users in ADP Vantage

Use the following sample script in a Script Synchronization for generating a report from ADP Vantage for analysis of username, user title, work location and department. The work location and department are mapped to IAN attributes.

let adp_users = ADP.getUsers();

let i = 0;

log.info

('ADP ID,ADP STATUS,ADP EMPNO,ADP GIVENNAME,ADP FAMILYNAME,ADP WORK EMAIL,ADP LOCATION CODE,ADP JOBCODE,ADP DEPT');

for (i = 0; i < adp_users.length; i++) {

    let adp_id = adp_users[i].id;

    let adp_status = adp_users[i][IAN].status;

    let adp_emp = adp_users[i][ENT].employeeNumber;

    let adp_locationcode = adp_users[i][IAN].LocationCode || '';

    let adp_job = adp_users[i][IAN].jobCode || '';

    let adp_dept = adp_users[i][IAN].departmentCode || '';

    let adp_given = adp_users[i].name.givenName;

    let adp_family = adp_users[i].name.familyName;

    let adp_work = adp_users[i].emails && adp_users[i].emails.find(x => x.type == 'work') && adp_users[i].emails.find(x => x.type == 'work').value || '';

    log.info('"' 

    + adp_id + '","' 

    + adp_status + '","'

    + adp_emp + '","'

    + adp_given + '","' 

    + adp_family + '","'

    + adp_work + '","'

    + adp_locationcode + '","'

    + adp_job + '","'

    + adp_dept + '"');

}

 

Analyzing the Users in Okta

Use the following sample script in a Script Synchronization for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}
