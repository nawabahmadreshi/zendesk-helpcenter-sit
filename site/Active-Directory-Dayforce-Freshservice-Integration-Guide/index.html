Overview

The Aquera platform enables your organization to automate identity lifecycle management by synchronizing employee data across HR, directory, and IT service management systems.

This integration automates identity lifecycle processes between Dayforce, Active Directory, and Freshservice. Dayforce acts as the source of truth for employee data, Active Directory stores and manages user accounts, and Freshservice manages approval-driven journeys for sensitive provisioning actions.

Dayforce changes flow downstream automatically. When an action requires approval, Freshservice journeys are used to gate the Active Directory operation before the account is created, updated, or deactivated.

This integration supports user onboarding, employee updates, offboarding, name changes, group assignments, manager assignments, termination handling, logging, reporting, and report-only execution.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Retrieves employee records from Dayforce.
  • Creates new Active Directory accounts for eligible Dayforce workers.
  • Updates existing Active Directory users when Dayforce attributes change.
  • Routes onboarding, employee update, offboarding, and name change requests through Freshservice journeys when approval is required.
  • Processes approved Freshservice journeys and applies the approved changes to Active Directory.
  • Deactivates or terminates Active Directory users when workers become inactive in Dayforce.
  • Assigns and removes Active Directory groups.
  • Assigns and removes managers in Active Directory.
  • Handles duplicate users across Active Directory domains.
  • Supports report-only mode to simulate changes without applying them.
  • Generates logs, reports, and execution summaries.

Integration Setup

This section details the configuration process for integrating Dayforce, Active Directory, and Freshservice. It outlines the steps to create the integration, configure applications, define run parameters, set source filters, configure Freshservice journey approval gates, define Active Directory provisioning behavior, and configure termination handling.

Create Integration

Perform the following steps to create the integration:

  • Log in to the Aquera Admin portal.
  • Navigate to Integrations from the left side panel.
  • Click Create Integration.
  • Search for Active Directory / Dayforce / Freshservice in the search box next to the Explore Templates section.
  • Click the integration template to display the Create New Integration pop-up window.
  • Enter a name and a description for the integration.
  • Select application connector instances from the dropdown menus.
  • Click Continue. The integration configuration page is displayed to configure the integration.

Overview

This tab displays an introduction to the integration and explains how the integration is organized.

The About Overview section explains that the integration automates identity lifecycle management between Dayforce, Active Directory, and Freshservice.

Dayforce is used as the employee source of truth. Active Directory is used to manage user accounts. Freshservice is used to manage IT-driven approval workflows before sensitive Active Directory writes are performed.

The integration performs the following actions during each run:

  • Reconciles a Dayforce employee snapshot against Active Directory and Freshservice.
  • Creates Active Directory accounts for new hires or routes them through an onboarding journey.
  • Updates existing Active Directory users when Dayforce attributes change.
  • Routes employee updates, name changes, and offboarding through Freshservice journeys when configured.
  • Deactivates inactive Dayforce workers in Active Directory immediately or on the configured effective date.
  • Keeps manager assignments and group memberships synchronized.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure the following general settings:

  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable log insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable full sync: Select this checkbox to execute the integration in full sync. If this checkbox is not selected, the integration will be executed in the incremental import mode, syncing only new and updated records from Dayforce to Active Directory.
  • Enable provisioning via Freshservice: Select this checkbox to process provisioning requests through Freshservice. If this checkbox is not selected, users will be synced directly from Dayforce to Active Directory.
  • Enable Freshservice journey full sync: Select this checkbox to fetch all Journeys from Freshservice during the sync. If this checkbox is not selected, only newly created or recently updated Journeys will be fetched.

Correlation Settings

Configure the following correlation settings:

  • Dayforce correlation attribute: Select the Dayforce correlation attribute to distinctly find the associated Active Directory employee. For example, EmployeeNumber.
  • Active Directory correlation attribute: Select the Active Directory correlation attribute to distinctly find the associated Dayforce employee. For example, employeeID. 

AD Application Settings

  • AD Application List*: Select your Active Directory application from the dropdown menu. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • AD Applications in Report Only Mode: Select the Active Directory applications for Report Only mode. The integration will identify and report changes without applying them. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

Source Settings

The Source Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Full sync frequency: Enter the number of days after which a full sync should run. For example, enter 7 to run a full sync every 7 days.

  • Source filter for user processing: Select the Dayforce filter to filter the employees while retrieving from Dayforce. The following are the available options:

    • All employees: Select this option to retrieve and process all employees from Dayforce.

  • Create condition: Select a condition from the dropdown menu to determine when employees are created. The following are the available options:

    • All employees: Select this option to create all employees from Dayforce in Active Directory.
    • All active employees: Select this option to create all active employees from Dayforce in Active Directory.
    • Custom: Select this option to define a custom condition to create an employee from Dayforce in Active Directory. When selected, the following field is displayed:
      • Custom create condition: Enter the JavaScript expression to create an employee from Dayforce in Active Directory.

    • All employees: Select this option to update all employees from Dayforce in Active Directory.

User Sync Features

The User Sync Features section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Enable Group Assignments: Turn on this toggle button to assign groups to users during synchronization.
  • Enable Group Removal: Turn on this toggle button to remove extra or redundant group assignments that are no longer needed.
  • Enable Manager Assignments: Turn on this toggle button to assign managers to users during synchronization.
  • Enable Manager Removal: Turn on this toggle button to allow removing assigned managers from users when required.
  • Deactivate Duplicate Users in Other Domains: Turn on this toggle button to deactivate user accounts in other Active Directory domains when a duplicate user with the same correlation key is found. If disabled, only the primary domain account is checked to deactivate duplicate users.
  • Delete Duplicate Users from Other Domains: Turn on this toggle button to permanently delete duplicate user accounts in other Active Directory domains. This action is irreversible. If disabled, duplicate users are only deactivated.
  • Sync Title Updates Directly to AD on Worker’s Reorganization: Turn on this toggle button to update the title directly in Active Directory when a worker is reorganized, instead of routing the update through a Freshservice employee update journey.
  • Sync Department Updates Directly To AD on Worker’s Reorganization: Turn on this toggle button to update the department directly in Active Directory when a worker is reorganized, instead of routing the update through a Freshservice employee update journey.

Location-Based Termination

The Location-Based Termination section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Timezone: Select the time zone used to evaluate termination schedules. The selected time zone determines when the configured termination time is applied for users in the specified location or organizational unit.
  • Time of Day: Specify the time when the termination action should be executed. The integration uses this value along with the selected time zone to determine when the user should be deactivated in Active Directory.
  • Offset Type: Select a date offset type from the dropdown menu. This setting determines whether the termination should occur before, after, or exactly on the worker's effective termination date.
    The following are the available options:
    • X Days Before Termination: Select this option if you want the user to be terminated before their actual termination date. When selected, the following field is displayed:
      • Offset Days for Termination: Enter how many days before the termination date the user should be terminated.
    • X Days After Termination: Select this option if you want the user to be terminated after their actual termination date. When selected, the following field is displayed:
      • Offset Days for Termination: Enter how many days after the termination date the user should be terminated.
    • No Offset: Select this option if you want the user to be terminated on their actual termination date.
  • Move user to different OU on deactivation: Turn on this toggle button to move the user to a different Active Directory organizational unit when the user is deactivated. Use this option when terminated users must be moved to a dedicated OU for inactive or terminated accounts. When enabled, the following field is displayed for further configuration:
    • Termination OU distinguished name: Enter the distinguished name of the Active Directory OU where the user should be moved during deactivation.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Rules

The Rules section enables you to customize the integration settings according to your specific use case. It has the following settings:

DN Rules

Perform the following steps to define DN rules for the employees:

  1. Click Choose Attribute and then select an attribute from the list of available attributes to define the domain mapping rule.
  2. Search and select the attribute you want to use to define a condition. For example, select ActiveEmploymentStatusXRefCode.
  3. In the row below, click the dropdown icon under ActiveEmploymentStatusXRefCode column and select the attribute value to define the rule. If multiple values are selected, an OR condition is applied between them. For example, in the following sample condition, a user's employment status reason must be Active to satisfy this assignment rule.
  4. Click the dropdown icon under DN Assignments to assign the DN. If the condition matches, the user will be assigned to the selected DN group.
  5. To add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule.
  6. Click Save Changes at the top-right corner of the page to save the configuration.

Note: To delete a row, click the bin icon next to the row and select Delete Row.

Using Expressions

Follow this procedure to set the rules using JavaScript expressions. A blank row is displayed by default for configuring the first set of conditions.

  1. Click Switch to Expression. The following options are displayed:
    • For Conditions Only: Select this option to use JavaScript expressions for setting the conditions. You must select the corresponding DN Assignments field.
    • For Conditions and Assignments: Select this option to use JavaScript expressions for setting the conditions.

Note: Click Switch to Condition Builder if you want to switch out of the Expression mode.

Group Rules

Perform the following steps to define Group rules for the employees:

  1. Click Choose Attribute and then select an attribute from the list of available attributes to define the domain mapping rule.
  2. Search and select the attribute you want to use to define a condition. For example, select XRefCode for the OU name site.
  3. In the row below, click the dropdown icon under XRefCode for the OU name site column and select the attribute value to define the rule. If multiple values are selected, an OR condition is applied between them. For example, in the following sample condition, a user's OU name site must be Location 50 to satisfy this group rule.
  4. Click the dropdown icon under Group Assignments to assign the group. If the condition matches, the user will be assigned to the selected group.
  5. To add another condition and assignment for the same selected attribute, click Add Row. You can add any number of rows within a rule. 
  6. Click Save Changes at the top-right corner of the page to save the configuration.

Using Expressions

Follow this procedure to set the rules using JavaScript expressions. A blank row is displayed by default for configuring the first set of conditions.

  1. Click Switch to Expression. The following options are displayed:
    • For Conditions Only: Select this option to use JavaScript expressions for setting the conditions. You must select the corresponding Group Assignments field.
    • For Conditions and Assignments: Select this option to use JavaScript expressions for setting the conditions.

Note: Click Switch to Condition Builder if you want to switch out of the Expression mode.

Log Settings

Note: This section appears only if Enable logging is enabled in the General Settings section.

  • Select log detail level: Select the desired log detail level from the dropdown menu. The following are the available options:

Notification Settings

The Notification Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable notification is enabled in the General Settings section.

User Creation Notification Settings

Configure the following notification settings for the user creation:

  • Creation Email Recipients: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Custom Recipients — Creation: Enter the email addresses that should be included as CC recipients for notifications.
  • Creation Email Subject: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • User Creation Email Template: Enter the content that must be used as the email body in the notifications sent for successful integration execution.

User Update Notification Settings

Configure the following notification settings for the user update:

  • Update Email Recipients: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Custom Recipients — Update: Enter the email addresses that should be included as CC recipients for notifications.
  • Update Email Subject: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • User Update Email Template: Enter the content that must be used as the email body in the notifications sent for successful integration execution.

User Termination Notification Settings

Configure the following notification settings for the user termination:

  • Termination Email Recipients: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Enable send mail to manager of the deactivated user: Select this checkbox to identify whether the user’s manager exists in Dayforce or Active Directory (AD) and send a deactivation email to the manager using the termination template. If this checkbox is not selected, deactivation emails will not be sent to the manager.
  • Termination Email Subject: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • User Termination Email Template: Enter the content that must be used as the email body in the notifications sent for successful integration execution.

Successful Integration Notification Settings

Configure the following notification settings for the successful integration:

  • Recipients of Success Notifications: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject in the notifications sent for successful integration execution.
  • Admin Email Notification Template: Enter the content that must be used as the email body in the notifications sent for successful integration execution.

Failed Integration Notification Settings

Configure the following notification settings for the failed integration:

  • Recipients of Error Notifications: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Error email subject: Enter the content that must be used as the email subject in the notifications sent for failed integration execution.
  • Error email template: Enter the content that must be used as the email body in the notifications sent for failed integration execution.

Report Settings

The Report Settings section enables you to customize the integration report settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable report generation is enabled in the General Settings section.

  • Save execution report to Aquera console: Select this checkbox to save the CSV sync report to your Aquera dashboard.
  • Send Integration Report via Email: Select this checkbox to receive email notification with a CSV report for all integration runs. 
  • Select report detail level: Select the desired report detail level. The following are the available options:
    • Full details: All attribute details are displayed.
    • Inclusion list: Select this option to report only the attributes specified in the Custom inclusion list.
      • Custom inclusion list: Select the attributes to be included in the report.
    • Exclusion list: Select this option to report only the attributes specified in the Custom exclusion list.
      • Custom exclusion list: Select the attributes to be excluded from the report.
    • Attributes to Include in Report: Enter the list of attributes that should be included in the report.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
