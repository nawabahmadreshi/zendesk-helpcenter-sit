Dayforce Employees - Jumpcloud Groups to JIRA Ticketing Integration

← Back to index

Overview

The Aquera platform enables scalable, automated management of the employee lifecycle across enterprise HR and IT systems. Through seamless integrations, it helps organizations eliminate manual onboarding/offboarding processes, ensure consistent provisioning, and optimize workflows across complex employee management infrastructures.

This integration facilitates synchronization between Dayforce, JumpCloud, and Jira Service Management, automating ticket creation for employee onboarding and offboarding while maintaining up-to-date group membership information throughout the employee lifecycle.

The integration operates on scheduled triggers with configurable date windows to fetch upcoming employee changes, such as new hires and terminations, from Dayforce. It automatically creates corresponding tickets in Jira Service Management for IT provisioning workflows while simultaneously querying JumpCloud for user group memberships and updating existing tickets with current access information. The system supports both full synchronization and incremental updates, processing only tickets modified since the last execution for optimal performance.

Core Integration Tasks

This integration performs the following key tasks:

  • Retrieves employee data from Dayforce, including new hires, terminations, and employee profile details within configurable date windows.
  • Processes detailed employee profiles with contact information, employee numbers, work assignments, and organizational metadata.
  • Matches employees to existing Jira records using correlation fields such as Employee Number or Work Email Address.
  • Creates onboarding and offboarding tickets in Jira Service Management automatically based on employee lifecycle events.
  • Prevents duplicate ticketing by checking for existing tickets using correlation fields and validating the resolved status.
  • Fetches user group memberships from JumpCloud using employee identifiers to retrieve current access permissions.
  • Updates existing Jira ticket descriptions with current group membership information from JumpCloud.

Integration Setup

This section outlines the configuration process for integrating Dayforce, JumpCloud, and Jira Service Management using the Aquera platform. It includes the steps for creating the integration, defining parameters, and enabling the automated processing of employee lifecycle events, such as onboarding, offboarding, and group management updates.

This integration continuously syncs employee records from Dayforce, specifically new hires and terminations, automatically creates corresponding tickets in Jira Service Management, and updates existing tickets with current group membership information from JumpCloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to the Integrations tab.
  3. Click Create Integration.
  4. Choose Dayforce Employees — JumpCloud Groups to Jira Service Management Tickets.
  5. Name and describe your integration.
  6. Select your Dayforce, JumpCloud, and Jira Service Management connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

For further details on integrations, refer to the Integration Using Scripts guide.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to edit, remove, and refresh these application connectors.

Set Up Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

Global Settings 

Configure the following global settings:

  • Enable Report-Only Mode: Simulate the integration execution without creating or updating any Ivanti Neurons tickets. Use this mode to verify configurations and preview potential changes safely.
  • Enable Logging: Capture detailed logs of integration activities, including data changes, API interactions, and processing results, to facilitate troubleshooting and support operational audits.
  • Enable Report Generation: Generate structured CSV reports summarizing integration outcomes, employee lifecycle event processing, ticket creation results, and errors.
  • Enable Log Insights: Log transaction details from integration transactions to analytics dashboards, allowing you to monitor activity trends, track performance metrics, and proactively identify and resolve issues.
  • Enable Ticket Management: Allows the integration to automatically create, update, and manage Jira Service Management tickets for employee onboarding and offboarding workflows.
  • Enable Group Management: Allows synchronisation of JumpCloud groups and memberships to Jira Service Management, keeping ticket-related access details up to date.
  • Enable Notifications: Select to enable the integration-related notifications.

Notification Settings

Configure the following settings for email notifications:

  • Admin's Email Address: Enter one or more admin email addresses that should receive notifications about integration executions, including success, failure, and CSV reports.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications
  • Send Integration Report via Email: Select to receive email notification with a CSV report for all integration runs.

Ticket Management

Configure the following settings for managing tickets:

  • Employee Lifecycle Events: Select the employee lifecycle events that should automatically generate tickets. For example, select Joiner to create onboarding tickets when new employees are added to Dayforce, Leaver to generate offboarding tickets when employees exit the organization, or select both to handle onboarding and offboarding events simultaneously. 
  • Onboarding Ticket Action: Select the action to perform when a new hire is detected. For example, select Create to automatically generate an onboarding ticket in Jira for provisioning accounts, granting access, or setting up devices. 
  • Dayforce HR Platform: Select the Dayforce instance that stores employee data and events used for onboarding and offboarding. For example, choose Dayforce – Production to connect the integration to your production HR environment. 
  • JIRA Ticketing Platform: Select the Jira Service Management Server instance where onboarding and offboarding tickets will be created. For example, select Jira Service Management – IT Operations to send all lifecycle event tickets to your IT service desk.

Source Settings

  • New Hire Lookup Period: Enter the number of days the integration should look ahead to identify upcoming new hires. For example, enter 32 to include employees whose start date falls within the next 32 days. 
  • Lookup Employee by Employee Number: Enter the unique employee identifier used in Dayforce to locate a specific employee record. For example, enter EMP10045 to retrieve that employee’s details and trigger related actions. 
  • Maximum Employees to Process: Enter the maximum number of employee records that the integration processes in a single run. For example, enter 2 to test with two employee records at a time, or leave blank to process all eligible employees. 
  • Onboarding Match Field: Select the Dayforce field used to match onboarding tickets to the correct employee record. For example, select Employee Number if that field uniquely matches Dayforce records to Jira tickets. 
  • Select Active Employment Statuses: Enter the employment statuses considered active for onboarding and offboarding automation. For example, select Active, Maternity, and Pensioner to include employees currently active or temporarily inactive due to leave or retirement. 
    Note: Click Refresh after updating configuration values to reload the latest data. This ensures that new Active Employment Statuses appear immediately in the dropdown menu.
  • Create Condition: Select the rule that determines which employee records trigger ticket creation. For example, select All Active Employees to create tickets for all employees with active statuses automatically.

Target Settings 

  • JIRA Project: Enter the JIRA project key where lifecycle tickets are created. For example, enter onboard to send all onboarding requests to the JIRA project named “Onboard.” 
  • Resolved Ticket Status Category: Select the JIRA status category that indicates a ticket has been completed. For example, select Done or Closed to mark onboarding or offboarding tickets as finished when they reach those statuses. 
  • JIRA Match Field: Select the JIRA field used to associate tickets with the correct employee record from Dayforce. For example, select Employee Number if it matches the employee’s ID field in Dayforce. 
  • JIRA Issue Type: Specify the JIRA issue type used for onboarding or offboarding workflows. For example, select Bug or Task depending on your organization’s JIRA configuration for lifecycle automation.

    Note: Click Refresh to load the latest JIRA projects, status categories, and issue types from the connected JIRA instance.

Group Management

Configure the following settings for managing groups:

  • JIRA Project: Select the JIRA project to be used for group management activities.
  • Resolved Ticket Status Category: Select the JIRA statuses that indicate group management tickets are complete. For example, choose Done or Closed.
  • JIRA Issue Type: Specify the JIRA issue type used for onboarding workflows. For example, select Bug or Task depending on your organization’s Jira configuration for lifecycle tickets.

Attribute Mapping

The Attribute Mapping tab defines how source system fields from Dayforce and JumpCloud correspond to JIRA Service Management ticket fields across onboarding, offboarding, and group management workflows. Administrators can select the source system (Dayforce or JumpCloud) for each mapping and specify the source attribute or expression that should populate a specific JIRA target field.

For onboarding mappings, ensure that new-hire details from Dayforce, such as Employee Number, Display Name, Work Location, Position Title, and other relevant attributes, are accurately reflected in the JIRA tickets used for IT provisioning.

For offboarding termination details, including Last Working Day, Manager, Job Title, and related fields, JIRA tickets are mapped to support timely and complete deprovisioning.

For group management, JumpCloud user attributes, such as Email Address, Group Memberships, and other access-related data is synchronised into JIRA fields, including groups and description, to ensure that access information remains accurate and up to date.

Tables

The Tables section contains configurable key–value pairs used to define correlation mappings between source and target systems for onboarding and offboarding workflows. These mappings ensure the integration accurately identifies and matches records across Dayforce, JumpCloud, and JIRA Service Management.

In this integration, three correlation tables are available:

  • Source Correlation Onboard Map: Maps Dayforce employee identifiers to standard keys used for onboarding in JIRA Service Management. For example, Employee Number maps to XRefCode, Work Email Address maps to BusinessEmail, and so on.
  • Target Correlation Offboard Map: Maps JIRA ticket fields to Dayforce offboarding records. For example, Employee Number maps to Employee Number, and Work Email Address maps to Email Address, and so on.
  • Target Onboarding Correlation Map: Maps JIRA onboarding ticket fields to Dayforce records. For example, Employee Number maps to Employee ID, and Work Email Address maps to Email Address, and so on.

Without these correlation tables, the integration fails to link related records or could create duplicate tickets. You must clearly define source-to-target relationships to prevent duplication and maintain accurate record matching across workflows.
