Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration enables automatic ticket creation in ServiceNow for file upload escalation events configured through Aquera File Collection schedules.

The integration works with Aquera’s File Collection Schedule and escalation framework to trigger ServiceNow tickets whenever escalation conditions are met during scheduled file upload cycles.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
  • Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Generic Flatfile: Refer to the Generic Flatfile Configuration Guide for details on configuring the connector.
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details on configuring the connector.
  • File Collection Setup: Refer to the File Collection Setup with Schedule Groups Guide for more details.

Integration Setup

This section details the configuration and mapping process for integrating File Collection with ServiceNow. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for File Collection / ServiceNow in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section allows you to identify the table to create tickets in ServiceNow.

Table name: Select Incident in this field to allow ticket creation in ServiceNow.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target system. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

The following source attributes are required in the mapping:

  • Owner Emails: Identifies the first email address configured in the Source Owner field on your Generic Flatfile connector configuration page.
  • File Uploaders: Identifies the first email address configured in the File Uploader field on your Generic Flatfile connector configuration page.
  • Description: Identifies the description of your application configured in the Description field on the connector configuration page.
  • Short Description: Identifies the name of your application configured in the Display Name field on the connector configuration page.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Table

In the Tables tab, you can add tables to the integration. Tables are used to define custom source attributes and synchronize them with the target application. If the required data is not available in the source application, you can define it in a table and use it to update the target application. Once defined, the table can be referenced in attribute mapping expressions to set or update values automatically.

Refer to the Using Tables in Integrations guide for more details.

Event

This section enables you to configure event-based workflow triggers for the integration. When enabled, the integration can receive and process events from the selected event source to initiate workflows automatically.

Perform the following steps to configure event settings:

  1. Navigate to the Event tab.
  2. Configure the following fields:
    1. Event Trigger Enabled: Turn on this toggle button to allow the integration to execute automatically when events are received from the configured event source.
    2. Event Source: Displays the event source configured for the integration. This field is populated automatically and does not require any changes.
    3. URL: Displays the generated event endpoint URL used to receive events from the configured source application. This field is populated automatically and does not require any changes.
    4. Refresh Token: Required only if you want to regenerate the event token and update the event URL.

Execute Integration

You can execute an integration manually or schedule it to run automatically. For further details, refer to the User Action on Integrations guide.
