External Libraries Supported in Integration Scripts

← Back to index

Overview

Aquera's integration scripts are equipped with external libraries to facilitate seamless connections to various applications. This functionality allows you to perform a wide array of tasks directly on the applications, streamlining the process of data management.

By leveraging the functions from the external libraries in integration scripts, you can perform several important tasks on the applications. For instance, you can retrieve specific data that meets certain criteria. Additionally, you can update existing records, ensuring that your data remains current and accurate. You can also insert new entries into applications, which is vital for maintaining an up-to-date record of information. Furthermore, if necessary, you can delete existing records, providing you with complete control over the data in your applications.

This guide provides an overview of the external libraries available in Integration Scripts. It explains how to utilize their functions to connect to applications and perform tasks directly on those applications for better data management.

TDS (Tedious) Library

The Script Editor supports the Tedious library to connect to the Microsoft SQL Server application. You can use the functions available in the Tedious library to perform database table operations in Microsoft SQL Server.

The Tedious package implements the TDS (Tabular Data Stream) protocol, developed by Microsoft for interacting with an SQL Server instance.

To connect to the SQL Server, you must add the SQL Server application configured in your Aquera Admin tenant to the integration and use the application variable name in the script for the database connection.

The functions built in the Tedious library can be invoked in the script using the format:

tedious.<FunctionName>

The following are the Tedious library functions leveraged by Aquera for the integration:

  • Connection
  • Request

For further details, refer to the documentation for the Tedious package.

Connection

This function is used to connect to the SQL Server instance using the credentials configured in the SQL Server application connector. For this, you must add the SQL Server application configured in your Aquera tenant to the integration and assign a variable name to that application.

Format

//Create a connection
var connection = new tedious.Connection({ app: <appVariableName>, options: {}});

// Setup event handler when the connection is established
connection.on('connect', function(err) {
if(err) {
console.log('Error: ', err)
}
// If no error, then good to go
executeStatement();
});

// Initialize the connection
connection.connect();
Parameters

(mandatory) appVariableName: The SQL Server application variable name.

(mandatory) options: This object includes the database name, encryption information, the connection time-out value in milliseconds, and the TDS version to be used.
Example

const connection = new tedious.Connection({ app: sqlserver, options: {

encrypt: true,

trustServerCertificate: true,

database: "userdata",

rowCollectionOnRequestCompletion: true,

connectTimeout: 60000,

tdsVersion: tedious.TDS_VERSION.TDS_8_0

}})

connection.on("connect", function(err) {
if(err) {
if (err instanceof tedious.ConnectionError) { 
log.info(`Connection Error: ${stringify(err)}`);
} else {
log.info(`Error: ${stringify(err)}` );
}
return reject(err);
}

log.info("Connection acquired");
return resolve(connection);
})

connection.connect()

Note: The default isolation level for new connections is set to READ_COMMITED. The isolation levels can be accessed using tedious.ISOLATION_LEVEL.

Request

This function executes a request on a connection.

Format

//Create a request
var request = new tedious.Request(sql, function(err, rowCount, rows) {
...
});
connection.execSql(request);
Parameters

(mandatory) sql: The SQL statement to be executed.

(mandatory) function: This is the callback function called when the request has been completed, either successfully or with an error. 

  • err: If an error occurs, this returns error messages.
  • rowCount: The number of rows emitted from executing the SQL statement.
  • rows: Rows are the result of executing the SQL statement.
Example

const request = new tedious.Request(script, ( err, rowCount, rows) => {
if (err) {
if (err instanceof tedious.RequestError) {
log.info(`Request Error: ${stringify(err)}`);
} else {
log.info(`Error: ${stringify(err)}`);
}
reject(err);
} else {
resolve({
rowCount,
rows
});
}
});


connection.execSql(request)

node-oracledb Library

The Script Editor supports the node-oracledb library to connect to the Oracle database application. You can use the functions available in the node-oracledb library to perform database table operations in the Oracle database.

The node-oracledb is a database driver for high-performance Oracle Database applications. It connects to the Oracle database via thin client only.

To connect to the Oracle database application, you must add the Oracle database application configured in your Aquera Admin tenant to the integration and use the application variable name in the script for the database connection.

The functions built in the node-oracledb library can be invoked in the script using the format:

oracledb.<FunctionName>

The following are the node-oracledb library functions leveraged by Aquera for the integration:

  • getConnection
  • createPool

For further details, refer to the documentation for the node-oracledb driver.

getConnection

This function obtains a connection from a pool in the connection pool cache or creates a new, standalone, non-pooled connection.

Format

oracledb.getConnection([String poolAlias | Object connAttrs])
Parameters

poolAlias: The previously created pool in the connection pool cache to obtain the connection.

connAttrs: The configuration properties for the connection.
Example

for (let i = 1; i < 10; i++) {
utils.pause(200);
}

log.info("Connecting to the Oracle DB");


oracledb.outFormat = oracledb.OUT_FORMAT_OBJECT;


const connection = await oracledb.getConnection({
app: <appVariableName>
});

log.info("Connected to the Oracle DB");

const script = `SELECT userid as "id",userid as "title", givenName as "name.givenName", familyName as "name.familyName" FROM customtable_autoincrement3`;

 

const result = await connection.execute(script);

log.info(`Result : ${JSON.stringify(result, undefined, 4)}`);

 

await connection.close(); 

createPool

This function creates a pool of connections.

Format

oracledb.createPool(Object poolAttrs)
Parameters

poolAttrs: The configuration properties for the connection.
Example

for (let i = 1; i < 10; i++) {
utils.pause(200);
}

log.info("Connecting to the Oracle DB");


oracledb.outFormat = oracledb.OUT_FORMAT_OBJECT;

await oracledb.createPool({
app: <appVariableName>
})

log.info("Created Pool");

const connection = await oracledb.getConnection();

log.info("Connected to the Oracle DB");

const script = `SELECT userid as "id",userid as "title", givenName as "name.givenName", familyName as "name.familyName" FROM customtable_autoincrement3`;

const result = await connection.execute(script);

log.info(`Result : ${JSON.stringify(result, undefined, 4)}`);

 

await connection.close(); 

await oracledb.getPool().close(0);

ldapjs Library

The Script Editor supports the ldapjs library to connect to the Active Directory application. You can use the functions available in the ldapjs library to perform tasks such as searching, adding, updating, and deleting entries in the LDAP server.

To connect to the Active Directory, you must add the Active Directory application configured in your Aquera Admin tenant to the integration and use the application variable name in the script.

The functions built in the ldapjs library can be invoked in the script using the format:

ldapjs.<FunctionName>

These ldapjs library functions can be utilized when you use JavaScript for setting up an attribute map in script-based integrations. 

The following are the ldapjs library functions leveraged by Aquera for the integration:

  • createClient
  • bind
  • search
  • Change
  • modify
  • add
  • unbind

For further details, refer to the documentation for the ldapjs library.

createClient

This function creates a new client.

Format

const ldapClient = await ldapjs.createClient({
<appVariableName>,
connectTimeout: <value>
tlsOptions: <values>
});
Parameters

<appVariableName>: The Active Directory application variable name. An LDAP URL with the hostname and port will be created from the Active Directory application's configuration page in the Aquera Admin portal. 

connect timeout: Milliseconds client should wait before timing out on TCP connections.

tlsOptions: Additional options passed to TLS connection layer when connecting via ldaps:// 
Example const ldapClient = await ldapjs.createClient({
<appVariableName>, // URL from the tenant name will be passed
connectTimeout: 10000,
tlsOptions: { keepAlive: true, rejectUnauthorized: false, minVersion: "TLSv1" }
});

bind

This function performs a bind operation (simple authentication) against the LDAP server. 

Format

<ldapClient>.bind(<appVariableName>, Control controls, Function callback)
Parameters

<ldapClient>: The return value from the createClient function.
<appVariableName>: The Active Directory application variable name.
controls: Array of control objects. This is optional.
callback: A callback function that receives an error if the bind fails. Callback is of the form f(err, res).
Example ldapClient.bind(<appVariableName>, connectionError => {
if (connectionError) {
log.info(`Error while connecting to ldap: ${connectionError}`);
reject(connectionError);
}
log.info(`Bind successful`);
resolve("Bind successful");
});

search

This function performs a search operation against the LDAP server.

Format

<ldapClient>.search(string base, Object options, Control controls, Function callback)
Parameters

<ldapClient>: The return value from the createClient function.
base: The Distinguished Name in the tree to start searching at.
options: Object containing the following fields:

  • scope: {String} One of base, one, or sub. Defaults to base.
  • filter: {String} A string version of an LDAP filter or a programmatically constructed Filter object. Defaults to (objectclass=*).
  • attributes: {Array} Attributes to select and return. Defaults to the empty set, which means all attributes. You can provide a string if you want a single attribute or an array of string for one or many.
  • attrsOnly: {Boolean} Indicates whether you want the server to only return the names of the attributes, and not their values. Defaults to false.

controls: Array of control objects.
callback: Callback of the form f(err, res).
Example

ldapClient.search(BASE_DN, searchOptions, (err, res) => {
if (err) return reject(`Search error: ${err}`);

res.on('searchEntry', (entry) => {
user = entry;
});

res.on('end', () => {
resolve(user);
});

res.on('error', (err) => {
reject(`LDAP search error: ${err}`);
});
});

Change

A Change object specifies the details of the update operation to be performed. The Changes object requires you to set the operation and modification.

Format

const ldapChanges = new ldapjs.Change{
operation: value,
modification: {
type: key 
values: values
}
});
Parameters

operation: The operation to be performed. It is a string and must have one of the following values:

  • replace: This operation replaces the attribute specified in the modification. If the modification has no values, it is equivalent to a delete.
  • add: This operation adds the attribute value(s) specified in the modification. The attribute may or may not already exist.
  • delete: This operation removes the attribute (along with all associated values) specified in the modification.

modification: The modification to be performed. It is a JavaScript object that contains the type and values:

  • type: A string that defines the attribute type for the modification.
  • values: Defines the values for modification. 
Example

const ldapChanges = new ldapjs.Change({
operation: 'replace',
modification: {
"type": key,
values: Array.isArray(value) ? value : [value]
}
});

modify

This function performs a modify operation against the LDAP server. This API requires you to pass in a Change object. Note that you can pass in a single Change or an array of Change objects.

Format

<ldapClient>.modify(String name, Change <ldapChanges>, Control controls, Function callback)
Parameters

<ldapClient>: The return value from the createClient function.

name: The Distinguished Name of the entry to modify.

<ldapChanges>: The return value from the Change function.

controls: Array of control objects.

callback: Callback of the form f(err, res).
Example

ldapClient.modify(distinguishedName, ldapChanges, (err) => {
if (err) {
return reject(`Update error: ${err}`);
}
log.info(`User updated successfully: ${dn}`);
resolve(distinguishedName);
});

add

This function performs an add operation against the LDAP server.

Format

<ldapClient>.add(String name, Object entry, Control controls, Function callback)
Parameters

<ldapClient>: The return value from the createClient function.

name: The Distinguished Name of the entry to add.

entry: An array of Attributes to be added or a JS object where the values are either a plain value or an array of values.

controls: Array of control objects.

callback: Callback of the form f(err, res).
Example

ldapClient.add(distinguishedName, userData, (err) => {
if (err) {
return reject(`Create error: ${err}`);
}
log.info(`User created successfully: ${distinguishedName}`);
resolve(userData);
});

unbind

This function is used to close the connection with the LDAP server. The LDAP session will be terminated.

Format

<ldapClient>.unbind(Function callback)
Parameters

<ldapClient>: The return value from the createClient function.

callback: The function that will be invoked when the unbind operation completes.
Example

ldapClient.unbind(() => {

ldapClient._socket?.destroy();

resolve("Unbind successful");

});
