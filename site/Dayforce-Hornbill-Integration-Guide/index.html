Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide covers the end-to-end tasks involved in setting up the Dayforce / Hornbill integration, which creates joiner, mover, and leaver tickets in Hornbill based on employee creations, updates, and deletions in Dayforce.

Key Functions

The Dayforce / Hornbill integration performs the following functions:

  • Reads all the records of employees that were created, updated, and terminated in Dayforce.
    • The integration supports parameters that enable it to look for any future hires, any employment changes with a future effective date, and terminations in the future. Those employees can also be retrieved from Dayforce.
  • Creates Joiner, Mover, and Leaver service requests under specific services and catalogs in Hornbill
  • Assigns specific customer IDs and customer types for Joiner, Mover, and Leaver service requests based on the integration configuration.
  • Populates data in the service request fields based on the attribute maps configured for Joiner, Mover, and Leaver tickets
  • Creates integration reports based on the integration configuration 
  • Sends email notifications on integration runs along with the integration report based on the integration configuration.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Dayforce: Ensure that you configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for configuration details.
    • Hornbill: Ensure that you configure the Hornbill connector in the Aquera Admin portal. Refer to the Hornbill Configuration Guide for configuration details. 

Integration Setup

This section details the configurations for retrieving joiner, mover, and leaver data from Dayforce and creating tickets in Hornbill. It outlines the steps to create the integration and set up the integration preferences.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / Hornbill.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Hornbill connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the Dayforce and Hornbill connector instances used by the integration. This section allows you to search for a different instance of the application connectors and select them.

Parameters

The Parameters section enables you to customize the integration features and global settings according to your specific use case.

Integration Globals

Enable the following feature:

  • Enable Ticket Management: Select this checkbox to manage joiner, mover, and leaver service requests in Hornbill when a user is created, updated, or deleted in Dayforce. When this checkbox is selected, the Ticket Management and Ticket Flow sections are displayed for further configurations.

Global Section

Configure the following settings:

  • Enable Report Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Debug: Select this checkbox to enable debug mode for the integration. When enabled, the Debug section is displayed for further settings.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logs section is displayed for further configuration.
  • Enable Full Sync: Select this checkbox to run the integration in full sync mode. All employee records that match the Joiner, Mover, and Leaver ticket configurations are retrieved from Dayforce, and service requests are created for those employees. When this checkbox is not selected, the integration runs in incremental mode, retrieving only employee changes that have not already been processed. By default, the first execution runs in full sync mode.
  • Enable Log Insights: Select this to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.

Ticket Management

This section details the following parameters for managing ticket creations:

  • Enable Ticket Flow: Select this checkbox to define the ticket . When enabled, the the following parameters are displayed:
    • Event: Select one or more events from the dropdown menu. Based on your event selection the corresponding ticket will be updated in the Hornbill. The available options are Joiner, Mover, and Leaver.
    • Customer ID: Enter the customer ID 
    • Customer Type: Enter the customer type by which the request to be created.
  • Dayforce Time Offset: Enter the time offset from GMT to be used when synchronizing employee data from Dayforce to Hornbill. For example, if you enter -05:00, the Hornbill server time is considered 5 hours behind GMT, and the data synchronization will be performed based on this offset. This is applicable only during incremental sync.

Ticket Flow

Joiner Flow

Configure the following settings to define the Joiner 

  • Joiner service Name: Select a service name from the dropdown menu in which joiner tickets should be created.
  • Joiner Catalog Name: Select a catalog name from the dropdown menu in which joiner tickets should be created.
  • Joiner Limit: Select this checkbox to define the number of joiner tickets processed in a single integration run. When enabled, the following parameter is displayed:
    • Joiner OnBoarding Limit: Enter the maximum number of onboarding records that can be created in a single integration run. For example, entering 7 allows only 7 onboarding tickets to be created in a single integration run.
  • Enter the x Days to Look Ahead for New Hire:

Mover Flow

Configure the following settings to define the Mover 

  • Mover service Name: Select a service name from the dropdown menu in which mover tickets should be updated.
  • Mover Catalog Name:  Mover service Name: Select a catalog name from the dropdown menu in which mover tickets should be updated.
  • Mover Limit: Select this checkbox to define the number of mover tickets processed in a single integration run. When enabled, the following parameter is displayed:
    • Mover OnBoarding Limit: Enter the maximum number of employee records that can be updated in a single integration run. For example, entering 15 allows only 15 mover tickets to be updated in a single integration run.
  • Enter the X Days to Look Ahead for Updated Employees: Enter the number of days from when in the future to create a mover ticket that will be updated in future. 

Leaver Flow

Configure the following settings to define the Leaver 

  • Termination service Name: Select a service name from the dropdown menu from which leaver tickets should be removed.
  • Termination Catalog Name: Select a catalog name from the dropdown menu from which leaver tickets should be removed.
  • Leaver Limit: Select this checkbox to define the number of leaver tickets processed in a single integration run. When enabled, the following parameter is displayed:
    • Leaver OnBoarding Limit: Enter the maximum number of employee records that can be terminated in a single integration run. For example, entering 5 allows only 5 leaver tickets to be terminated in a single integration run.
  • Enter X days to Look Ahead for Term Employees:

Logs

This section enables you to customize the log generation settings. It has the following settings:

Note: This section is displayed only if you have selected the Enable Logging checkbox in the Parameters section.

  • Select the Log Type: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full Details: All attribute details are displayed.
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

Debug

This section enables you to customize the debug mode settings. It has the following settings:

  • Source Filter: Select an attribute from the dropdown to filter a user for debugging purposes. The available options are Email and EmployeeNumber.
  • Test: Enter the value for the selected attribute in the Source Filter parameter for debugging purposes.

  • Test Event: Select an event from the dropdown menu. The available options are joiner, mover, and leaver. 

    Whenever an event selected in the Test Event parameter is triggered, users matching the attribute selected in the Source Filter parameter and the value provided in the Test parameter are filtered and used for debugging purposes.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
