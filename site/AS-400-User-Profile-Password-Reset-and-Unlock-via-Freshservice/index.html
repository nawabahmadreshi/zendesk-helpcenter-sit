AS/400 User Profile Password Reset and Unlock via Freshservice

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the integration of data across resource types. 

The AS/400 User Profile Password Reset and Unlock via Freshservice integration enables you to reset the AS/400 users' passwords and unlock any locked AS/400 user accounts. This integration performs the following tasks:

  • Retrieves all the service requests that have been created in Freshservice for resetting passwords and unlocking the locked accounts of AS/400 user profiles
  • For each service request related to password reset, the integration generates a random password for the AS/400 username associated with a ticket and resets the user password in AS/400
  • For each service request related to user account unlock, the integration enables the AS/400 user account.
  • Triggers an email notification informing users of their password reset requests and account unlock activity.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Freshservice Service Requests: You must have created service requests in Freshservice for resetting user passwords as well as for unlocking user accounts in AS/400. You must have the following details handy to setup this integration to read those service requests:
    • Freshservice workspace name
    • Service catalog name
    • Service request name
    • The Freshservice attribute that stores the AS/400 user profile name
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Freshservice: Refer to the Freshservice Configuration Guide for instructions on creating the Freshservice application connector and setting up the authentication.
    • AS/400: Refer to the AS/400 Configuration Guide for instructions on creating the AS/400 application connector and setting up the authentication.

Integration Setup

This integration reads all the Freshservice service requests that have been created for resetting user passwords as well as for unlocking user accounts in AS/400.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select AS/400 User Profile Password Reset and Unlock via Freshservice.
  5. Enter a name and a description for the integration.
  6. Select your Freshservice and AS/400 connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Preferences

The Preferences section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Password Policy: Select the complexity requirements for the password to be generated by the integration. This field defines the password policy to be followed for creating strong and secure passwords. Make sure that you select the following options from this field based on the password policy configured in AS/400:
    • Include uppercase: To include uppercase letters in the password 
    • Include numbers: To include numbers in the password
    • Include special characters: To include any of the following special characters in the password to add complexity -  !@#$%^&*()_+[]{}|;:,.<>?/ </>
    • Include lowercase: To include lowercase letters in the password 
  • Email notification template for password reset requests: Enter the email body that must be used in the email notifications for password reset requests. A default text is provided in this field. You can customize it based on your requirements. 
    Note: Variables are provided within curly brackets in the email body. These variables will be replaced with the actual values when the email is sent.
  • Email notification template for unlocked accounts: Enter the email body that must be used in the email notifications for user account unlock. A default text is provided in this field. You can customize it based on your requirements. 
    Note: Variables are provided within curly brackets in the email body. These variables will be replaced with the actual values when the email is sent.
  • Send error notifications to: Enter the comma-separated list of email addresses to which integration error notifications must be sent.
    For example, ops@example.com, itadmin@example.com
  • Send reports to: Enter the comma-separated list of email addresses to which integration reports must be sent.
    For example, ops@example.com, itadmin@example.com
  • Freshservice workspace name: Enter the Freshservice workspace name where you manage the service requests. For example, IT
  • Freshservice catalog name: Enter the Freshservice catalog name where you manage the service requests. For example, Access Requests
  • Service request name: Select the service requests that should be managed through this integration. The available options are Password Reset and Account Unlock. For example, if you have selected only the Password Reset option, the integration will handle only the password reset requests and not the account unlock tasks.
  • Freshservice user profile attribute for AS/400 user profile name: Enter the name of the Freshservice service attribute that stores the unique AS/400 user profile name. This attribute value is used to uniquely identify the AS/400 user account for which the password reset requests and/or the account unlock tasks need to be performed. 
  • Admin email notification template for errors: Enter the email body that must be used in the email notifications for errors. A default text is provided in this field. You can customize it based on your requirements. 
    Note: Variables are provided within curly brackets in the email body. These variables will be replaced with the actual values when the email is sent.

Set Up Schedule

You need to set up a schedule for executing the integration automatically.

You can set up the integration to run based on a basic or an advanced schedule. Refer to the Integration Schedule Setup Guide for details on setting up a schedule.

Run the Integration

When the integration runs, the following tasks are performed based on your configuration:

  • The passwords of all the AS/400 user profiles associated with the service requests in Freshservice are reset.
  • The accounts of the AS/400 user profiles associated with the service requests in Freshservice are unlocked.

Additionally, the integration triggers an email notification to the users whose passwords have been reset and/or whose accounts have been unlocked. It may also send notifications and reports to the email addresses based on your configuration.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
