Accessing Event Requests in Workflows

← Back to index

Overview

Workflows can be triggered when specific events occur in applications such as user data changes. An event will be triggered when there is a data change in an application such as a new user creation, existing user modification, or user deactivation. The application or the synchronization sends the events that trigger the workflow. 

This guide details how to access event requests in workflows. For further details on event triggers in workflows, refer to the Workflows User Guide. 

Accessing Event Request

You can provide the attributes of the event request in the element configurations based on your requirements.

The following tables give the description, format, and examples of the event request attributes.

Synchronization Events

The workflow receives the Synchronization event when a user is created, updated, or deactivated in the target application configured in a User Synchronization. The following are the supported request object attributes for a Synchronization event:

Description Format Request Object Attribute
The standard request attributes of the Synchronization event. request.
<attributeName>
  • request.type: 
    The type of the event such as creation, update.
  • request.name:
    The Synchronization name
  • request.guid:
    The Synchronization ID
  • request.engineId:
    The Synchronization engine ID
  • request.error:
    The error details when there is a failure in the user creations/updates and the checkbox Fire Hooks on Error is selected in the configuration of the User Synchronization.
  • request.error.statusCode: 
    The error status code when there is a failure in the user creations/updates and the checkbox Fire Hooks on Error is selected in the configuration of the User Synchronization. 
  • request.error.value: 
    The error value when there is a failure in the user creations/updates and the checkbox Fire Hooks on Error is selected in the configuration of the User Synchronization. 

The values of user attributes of the Source Application schema.

 request.source.<attributeName>
  • Core attribute:
    request.source.active
  • Complex single-valued attribute:
    request.source.name.formatted
  • Complex multi-valued attribute:
    request.source.emails[work].value
  • ENT attribute:
    request.source[ENT].department
  • IAN attribute:
    request.source[IAN].hireDate

All the previous values of
user attributes of the Target Application schema.


This is applicable only for User Updated by Synchronization.

 request.target.old.<attributeName>
  • Core attribute:
    request.target.old.active
  • Complex single-valued attribute:
    request.target.old.name.formatted
  • Complex multi-valued attribute:
    request.target.old.emails[work].value
  • ENT attribute:
    request.target.old[ENT].department
  • IAN attribute:
    request.target.old[IAN].hireDate

All the mapped values of user attributes of the Target Application schema.

For User Created by Synchronization, the request object contains only the mapped attributes whose mapping option in the attribute mapping is set as "Map on Create and Update" or "Map on Create".

 

For User Updated by Synchronization, the request object contains only the mapped attributes whose mapping option in the attribute mapping is set as "Map on Create and Update" or "Map on Update".

 request.target.new.<attributeName>
  • Core attribute:
    request.target.new.active
  • Complex single-valued attribute:
    request.target.new.name.formatted
  • Complex multi-valued attribute:
    request.target.new.emails[work].value
  • ENT attribute:
    request.target.new[ENT].department
  • IAN attribute:
    request.target.new[IAN].hireDate

The changed values of user attributes of the Target Application schema.

 

The request object contains only the changed attributes of the user. Hence for unchanged user attributes, it returns undefined. 

This is applicable only for User Updated by Synchronization.

 request.target.changed.
<attributeName>
  • Core attribute:
    request.target.changed.active
  • Complex single-valued attribute:
    request.target.changed.name.formatted
  • Complex multi-valued attribute:
    request.target.changed.emails[work].value
  • ENT attribute:
    request.target.changed[ENT].department
  • IAN attribute:
    request.target.changed[IAN].hireDate
The values of metadata user attributes of the Source Application schema. request.source.meta.
<metaAttributeName>
  • request.source.meta.resourceType:
    The resource type such as user.
  • request.source.meta.lastModified:
    The last modified date of the source user.
  • request.source.meta.location:
    The reference location of the source user.
The values of metadata user attributes of the Target Application schema. request.target.old.meta.
<metaAttributeName>
  • request.target.old.meta.resourceType:
    The resource type such as user.
  • request.target.old.meta.created:
    The creation date of the target user.
  • request.target.old.meta.lastModified:
    The last modified date of the target user.
  • request.target.old.meta.location:
    The reference location of the target user.

Application Events

The workflow receives the application event when there is a data change in an application. The following are the supported request object attributes for an Application event:

Description Format Request Object Attribute

The standard request attributes of the application event.

 request.<attributeName>
  • request.type:
    The type of the event.
  • request.id:
    The ID of the event.
The event attributes of the application schema. request.<eventAttributeName>
  • request.actor
    Note: Refer to the respective Connector Configuration Guide for details on application events.
The metadata attributes of the application event. request.data.meta.
<metaAttributeName>
  • request.meta.resourceType:
    The type of resource.
  • request.meta.created:
    The creation date of the event.
  • request.meta.lastModified:
    The last modified date of the event.
  • request.meta.location:
    The reference location of the event.

Users Modified Since Event

The workflow receives the users modified since event when there is a user modification in an application. The following are the supported request object attributes for users modified since event:

Description Format Request Object Attribute
The standard request attributes of the users modified since event. request.<attributeName>
  • request.type:
    The type of the event.
  • request.id: 
    The ID of the event.
  • request.sourceType:
    The source that generated the event, such as aquera-engine.
  • request.scheduledSupported:
    Mentions if the event can be scheduled.
  • request.actor:
    The ID of the user for which the event is generated.
  • request.source:
    The ID of the application for which the event is generated.

The user attributes for which the users modified since event is generated.

 request.data.<attributeName>
  • Core attribute:
    request.data.active
  • Complex multi-valued attribute:
    request.data.emails[work].value
  • ENT attribute:
    request.data[ENT].department
  • IAN attribute:
    request.data[IAN].hireDate
The metadata attributes of the user for which the users modified since event is generated. request.data.meta.
<metaAttributeName>
  • request.data.meta.resourceType:
    The type of resource such as the user.
  • request.data.meta.created:
    The creation date of the user.
  • request.data.meta.lastModified:
    The last modified date of the user.
  • request.data.meta.location:
    The reference location of the user.

Form Events

The workflow receives the form event when a form recipient clicks on an action button mapped to the workflow. The following are the supported request object attributes for a form event:

Description Format Request Object Attribute
The standard request attributes of the form event. request.<attributeName>
  • request.formName:
    The name of the form
  • request.requestId:
    The ID of the form event
  • request.submittedBy:
    The email ID of the user who filled and submitted the form
  • request.submittedOn:
    The date and time of form submission

The attributes of the form fields.

 request.data.<fieldName>
  • Form Data Field Name:
    request.data.field1
