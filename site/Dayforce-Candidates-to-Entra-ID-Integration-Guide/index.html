Dayforce Candidates to Entra ID Integration Guide

← Back to index

Overview

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

This integration connects Dayforce with EntraID to automate end-to-end user lifecycle management. It continuously monitors candidate status changes in Dayforce, such as new hires or declined applicants, and synchronizes those changes with EntraID. When the integration is triggered, it queries Dayforce reports and retrieves candidate details. For hire-ready candidates, it creates new user accounts while verifying that no duplicate accounts are created. For declined candidates, it locates matching EntraID accounts by email and removes access to ensure they are no longer able to log in.

In this integration, the Candidate ID in Dayforce is correlated with the Employee ID stored in Entra ID (extension attribute). When a candidate is hired, the Candidate ID in Dayforce becomes the official Employee ID in Dayforce, and this remains correlated with the same Employee ID in Entra ID.

If a candidate has declined the offer, the Candidate ID in Dayforce is matched with the Employee ID in Entra ID to remove or disable the user.
In a rehire scenario, a new Candidate ID in Dayforce is again correlated with the Employee ID in Entra ID according to company policy.

Key Functions

This Dayforce to EntraID integration automates identity lifecycle management with the following primary capabilities:

  • Collects candidate information from Dayforce reports based on configured application status values. It clones, filters, and generates datasets for each status, waits for dataset completion, and retrieves candidate records while keeping only those with valid electronic contact details.

  • Collects candidate information from Dayforce reports based on selected application status values. It creates and filters data for each status, and then retrieves candidate records, keeping only those with valid contact details.

  • For hire-ready candidates, the integration checks Entra ID to prevent duplicates, flattens and cleans Dayforce data, maps attributes to the Entra ID user schema, and creates new Entra ID accounts.

  • Identifies declined candidates, verifies their presence in EntraID via email, and deletes the corresponding accounts to remove access.

  • Generates a CSV report covering created, skipped, and deleted records and applying inclusion or exclusion filters. These reports can be emailed to administrators.

Integration Setup

The Dayforce to EntraID integration automatically synchronizes candidate data from Dayforce into Entra ID.

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Entra ID in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Manage Candidates 

Configure the following settings to manage candidates:

Source Filter Settings

  • Report Name*: Enter the name of the Dayforce report used to fetch candidate data. For example, enter Example Entra Candidates, which is the custom report created for this sync.

  • Regex for Location: Filters candidates by matching their location values against a regular expression to include or exclude specific locations.

  • Default Domain Name: Appends the specified domain to the username when creating new Entra ID user accounts.

Provisioning Settings

  • Create Condition*: Select the candidate statuses used to decide when to create Entra ID user accounts. For example, select OFFERED, NEW, and INPROGRESS to show only accounts for active candidates in the hiring process.
  • Retrieve candidates whose job application status has changed in the last X days: Specify the number of past days for which the integration should retrieve candidates with job application status changes. The selected number of days is calculated backward, starting from the current date. For example, if you enter 7, the integration will retrieve all candidates whose job application status was updated within the past 7 days, including the current date. If you enter 0, it will retrieve data only for the current date.
  • Enable Account Deletion Upon Declined Status: Select to remove Entra ID users when their linked candidate's application status is set to declined. For example, enable this to automatically remove accounts when a candidate's application status is moved to DECLINED in Dayforce.
  • Deletion Condition: Select the candidate application statuses in Dayforce that determine when a user’s Microsoft Entra ID account should be deleted. A user will be removed from Entra ID only if their application is marked as Declined in Dayforce and the same user exists in Entra ID. For example, selecting All Declined Candidates will delete all users marked as declined in Dayforce from Entra ID.

Provisioning Threshold

  • Create Limit: Set the maximum number of user accounts the integration can create or attempt to create in a single run. The skipped candidates are not counted. For example, you can set the Create limit to 50, validate the configuration, then increase it once you confirm the sync is working as expected.
  • Deletion Limit: Set the maximum number of user accounts the integration can delete or attempt to delete in a single run. For example, set Deletion Limit to 25 so that even if something is misconfigured, only a limited number of accounts can be removed per execution.

Attribute Mapping

The Dayforce to EntraID integration maps candidate data from the Dayforce report to the exact fields required by Entra ID. Each mandatory attribute in the report is paired with a matching Entra ID property to create a complete user record. 

The Candidate's status controls whether an account is enabled or disabled, while the name fields are combined to populate attributes such as displayName, givenName, and surname. The primary email address is also split to generate both the mailNickname and the userPrincipalName, ensuring a unique login name. Default values, such as a temporary password with a forced reset at first sign-in, can be assigned so that every account starts with the correct security posture.

Since the mapping references the required candidate report directly, it ensures that every hire-ready candidate is provisioned in Microsoft Entra ID with validated identity details. At the same time, declined applicants are removed.

Tables

The Tables tab enables you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

In the Dayforce to Entra ID integration, tables are commonly used to derive values, such as email or UPN domains, based on source attributes like candidate location, when the source system does not directly provide the required target value.

Each table consists of key–value pairs:

  • The key (Name) represents a value from Dayforce (e.g., a location or operating company).

  • The value represents the corresponding value to apply in Entra ID (for example, an email domain).

During user creation, the integration looks up the source value in the table and applies the mapped value to the Entra ID user attribute. If no matching key is found, the integration falls back to the configured Default Domain Name.

Available Tables

Location To Domain Table

This table maps a candidate’s location or company value from Dayforce to the domain name used when generating Entra ID usernames and email addresses. For example, if the candidate’s location is Maxwell Services and the table maps this value to maxwellservices.net, the integration creates a user with a UPN such as barbara@maxwellservices.net.

You can click the table row to open the Update Table window. Each row in the table represents one location-to-domain mapping.

Managing Table Entries

You can add or update table entries using the following options, depending on the integration use case:

  1. Add Row: Click Add Row to manually add a mapping by specifying a Name (source value) and a Value (target value).
  2. Upload CSV: Click Upload CSV to bulk upload mappings using a CSV file. The CSV file must contain exactly two columns with the headers NAME and VALUE.

Notes

  1. Each key name must be unique within a table.

  2. Key names can include letters, numbers, and most special characters.
    The following characters are not supported in key names:

    • / (forward slash)
    • \ (backslash)
    • " (double quotes)
  3. Click Save to apply and store the changes.

Creating Candidate Report 

The Dayforce to Entra ID integration requires a candidate report in Dayforce that includes specific fields and configurations. This report serves as the primary data source for provisioning, updating, and deactivating users.

The report must include required fields and a manageable number of columns. The accuracy and consistency of this report are crucial for transitioning candidate statuses into user records in EntraID.

Perform the following steps to create the report:

  1. Log in to your Dayforce instance.
  2. Navigate to Reporting.
  3. Select Create > Report From Topic and Select Recruiting > Add Topic.
  4. Add the following required fields: 
    • Candidate Display Name
    • Candidate First Name
    • Candidate Last Name
    • Preferred Given Name
    • Candidate Id
    • Employee Employee Id
    • Job Id
    • Contact Method
    • Contact Type Group Reference code
    • Application Status Last Modified Timestamp
    • Application Status Group Reference Code
    • Hire Date
    • Location Name
      These fields are mandatory for the integration to work properly.
      Note: Keep the report size under 24–25 columns total to avoid performance issues.
  5. Save the Report with your preferred name.
    Note: This exact name must later be provided in the integration parameters.
  6. Test the Report by generating it to verify that it is correct.
    Note: Verify that all required fields are present and that candidate statuses are displayed accurately. Also, verify that the report updates as new candidates are added or their statuses change.
    Note: For the sample candidate report structure, see the Appendix section.
