ADP Workforce Now Worker Report Integration Setup Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process for setting up an integration to retrieve worker data from ADP Workforce Now and generate a CSV report that includes either all worker attributes or selected worker attributes. The CSV report is then shared with the specified recipients via email.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure the ADP Workforce Now application connector in the Aquera Admin portal: Refer to the ADP Workforce Now Configuration Guide for details.

Integration Setup

This section outlines the configuration required to send the generated CSV file to the designated users via email and to display the integration results on the dashboard. It covers creating the integration, configuring system preferences, scheduling the integration to run, and distributing the generated CSV to the designated users. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now Worker Report.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now connector instance.
  7. Select your CSV Template connector instance.
  8. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following tabs are displayed on the Configuration page:

Applications

The Applications tab displays the Applications section, which displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following sections:

Global Settings

Configure the following global settings:

  • Workforce Now Settings: Select to enable the ADP Workforce Now settings.
  • Enable Report Generation: Select to generate the integration report.
  • Enable Notifications: Select to enable the integration-related notifications.

Workforce Now Settings

Configure the following Workforce Now settings:

  • Workforce Now Filter: Select None, Associate ID, or Position ID from the dropdown menu to filter the workers. None is the default value and indicates that all workers will be included. If you choose Associate ID or Position ID, an additional field will appear where you can enter a specific associate or position ID to retrieve only that worker.
  • CSV Report Inclusion Filter: Select All Workers or Active Workers from the dropdown menu to include all workers or only active workers in the CSV report.

Report Settings

CSV Column Headers: Select the worker attributes from the dropdown menu to display them in the CSV report. The following are the available options:

  • Legal First Name
  • Legal Last Name
  • Worker Category
  • Hire Date
  • Status
  • Home Department
  • Reports to Name
  • Corporate Groups Location Code
    Select the Show Selected checkbox to display only the selected attributes.

Notification Settings

Configure the following notification settings:

  • Admin Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report Via Email: Select to receive email notification with a CSV report for all integration runs.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide empty or specific data.

The following tables are available:

  • Customize Column Headers in the CSV Report: Click the table row to display the Update Table pop-up window. Modify the Value column to specify custom names for the column headers that match the selected CSV column names. Click Save to retain the changes made.
  • Workforce Now Display Name to Flattened OpenAPI Response: This table contains the Workforce Now display names and their corresponding flattened OpenAPI response. The Aquera team manages this table. Do not modify this table.

Schedule

Integrations can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details.

Run Integration

You can set up an event or schedule triggers to execute the integration. For further details, refer to the Integration Using Scripts guide.

Once the integration is successfully executed, Aquera sends the CSV report via email to the specified administrators.
