Overview

The Aquera platform enables organizations to connect applications and automate data exchange across systems. This integration automates the transfer of candidate information from iCIMS to Dayforce, helping organizations streamline the transition from recruitment to employee onboarding.

iCIMS acts as the source system for candidate data, while Dayforce serves as the system of record for employee information. The integration retrieves candidate information from iCIMS and provisions eligible candidates as employees in Dayforce. By automating this process, the integration reduces manual data entry, improves data accuracy, and helps ensure that employee records are created consistently and efficiently.

Key Functions

This integration provides the following capabilities:

  • Retrieves candidate, job, applicant workflow, company, manager, and form information from iCIMS.
  • Automatically discovers new and updated fields available in iCIMS.
  • Combines information from multiple iCIMS sources into a single Candidate resource.
  • Reduces manual configuration effort by automatically adapting to changes in iCIMS.

Integration Setup

This section details the configuration process for integrating iCIMS and Dayforce. It outlines the steps to create the integration, configure the iCIMS application connector, define customer and schema settings, configure profile and iForm schema retrieval, enable optional Manager and Company schemas, and generate the Candidate schema used in downstream mappings and integrations.

Create Integration

Perform the following steps to create the integration:

  • Log in to the Aquera Admin portal.
  • Navigate to Integrations from the left side panel.
  • Click Create Integration.
  • Search for Dayforce / iCIMS in the search box next to the Explore Templates section.
  • Click the integration template to display the Create New Integration pop-up window.
  • Enter a name and a description for the integration.
  • Select application connector instances from the dropdown menus.
  • Click Continue. The integration configuration page is displayed to configure the integration.

Candidate Management

The Candidate Management section enables you to customize the integration settings according to your specific use case. It has the following settings:

Candidate Configuration

Configure the following candidate configuration settings:

  • Candidate workflow statuses*: Select one or more iCIMS workflow statuses that indicate a candidate is ready for import into Dayforce. Only candidates with the selected workflow statuses will be processed. If you do not find your workflow status in the list, click the refresh icon available next to the field to reload the available workflow status.
  • iForm names: Enter the name of the iForm used to identify candidates.
  • Workflow status field IDs: Enter the iCIMS workflow field IDs used to retrieve candidate workflow statuses
  • Map manager information: Enable this option to map the employee's manager information from iCIMS to the corresponding manager field in Dayforce.
  • Import limit: Specify the maximum number of candidate records that can be imported in the target application during a single integration run.
  • Create condition: Select a condition from the dropdown menu to determine when candidates are created. The following are the available options:
    • All candidates: Select this option to create all candidates from iCIMS in Dayforce.
    • Custom: Select this option to define a custom condition to create a candidate from iCIMS in Dayforce. When selected, the following field is displayed:
      • Custom create condition: Enter the JavaScript expression to create a candidate from iCIMS in Dayforce.
  • Update condition: Select a condition from the dropdown menu to determine when candidates are updated. The following are the available options:
    • All candidates: Select this option to update all candidates from iCIMS in Dayforce.
    • Custom: Select this option to define a custom condition to update a candidate from iCIMS in Dayforce. When selected, the following field is displayed:
      • Custom update condition: Enter the JavaScript expression to update a candidate from iCIMS in Dayforce.
  • Test iCIMS candidate flow with sample data: Enable this option to test the iCIMS candidate flow using manually provided sample data. If this checkbox is not selected, candidate data will be retrieved directly from iCIMS.
  • Sample iCIMS candidate data (JSON): Enter sample candidate data in JSON format to test the iCIMS candidate flow.

iCIMS Candidate to Dayforce Employee

The iCIMS Candidate to Dayforce Employee tab defines how data is mapped from iCIMS to Dayforce application. The default mapping of attributes is displayed on the page. The iCIMS attributes are listed on the left side, and the Dayforce attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Execution

The following sections are displayed under the Execution tab:

Advance Settings

The Advance Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Rollout Configurations 

Configure the following rollout configuration settings:

  • Enable Report-Only Mode: Enable this option to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Debug Mode: Enable this option to generate logs outlining data changes that would occur for debugging. When enabled, the Debug Mode section is displayed for further configuration.
  • Enable Source Filter: Select the iCIMS attribute to filter the employees while retrieving from iCIMS.
    Note: If Enable Source Filter is enabled, the Enable Full Sync option is not displayed.
  • Enable Logging: Enable this option to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Log Insights: Enable this option to visualize the integration-related metrics. When enabled, the Log Insight Settings section is displayed for further configuration.
  • Enable Notifications: Enable this option to enable the integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Report Generation: Enable this option to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Full Sync: Enable this option to execute the integration in full sync. If this checkbox is not selected, the integration will be executed in the incremental import mode, syncing only new and updated records from iCIMS to Dayforce.

Safeguard Limits

Configure the following safeguard limit settings:

  • Enable safeguard limits: Enable this option to define limits that restrict the number or percentage of provisioning operations. When enabled, the following fields are displayed for further configuration:
    • Safeguard limit type: Select whether the safeguard limits are defined as a number or a percentage.
    • Safeguard limit setup: Specify whether a single safeguard limit is applied across all operations or separate limits are configured per operation type. The following are the available options:
      • Single limit for all operations: Select this if you want to define a limit for all operations. When selected, the following field is displayed for configuration:
        • Safeguard limit for all operations: Enter a value to define the safeguard limit for all operations. After the limit is reached, no create, update, or terminate operations will be executed.
      • Separate limits per operation type: Select this if you want to define a separate limit for all operations. When selected, the following fields are displayed for configuration:
        • Creation: Select this option to enable safeguard limit for user creation. When enabled, the following field is displayed for further configuration:
          • Create limit: Enter a value to define the safeguard limit for creation. After the limit is reached, create operations will no longer be executed.
        • Update: Select this option to enable safeguard limit for user update. When enabled, the following field is displayed for further configuration:
          • Update limit: Enter a value to define the safeguard limit for update. After the limit is reached, update operations will no longer be executed.
        • Termination: Select this option to enable safeguard limit for user termination. When enabled, the following field is displayed for further configuration:
          • Termination limit: Enter a value to define the safeguard limit for termination. After the limit is reached, termination operations will no longer be executed.

Notification

The Notification section enables you to customize the integration settings according to your specific use case. It has the following settings:

If the List View toggle button is disabled, the following parameters are displayed:

  • Select event for notification: Select the provisioning event that will trigger a notification. The following are the available options:
    • All: Select this option to send notifications for all supported integration events.
    • Success: Select this option to send a notification when the integration run completes successfully.
    • Skipped: Select this option to send a notification when the integration run is skipped.
    • Failure: Select this option to send a notification when the integration run fails.
    • User Create: Select this option to send a notification when a user is created.
    • User Update: Select this option to send a notification when a user is updated.
    • User Terminate: Select this option to send a notification when a user is terminated.
  • Email recipient: Enter the email addresses to whom the email notifications will be sent. The following are the available options:
    • Admin: Select this option to send email notifications to the admin's email address.
    • Custom: Select this option to specify one or more custom email addresses to send notifications.
  • Email subject: Enter the content that must be used as the email subject in the notifications sent.
  • Email notification template: Enter the content that must be used as the email body in the notifications sent.
  • Send integration report via email: Select this checkbox to receive an email notification with a CSV report for all integration runs.
  • Notification Report Attachment Level: Select the desired notification report level from the dropdown menu. For example, select Employee Number with Non-PII Data.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Note: Notifications are sent based on the event type selected in the notification configuration. Each event type has a dedicated notification section that allows you to configure recipients and notification settings independently. For example, selecting Success sends notifications only when an integration completes successfully, while selecting Failure sends notifications only when an integration execution fails. Similarly, Skipped, User Create, User Update, and User Terminate notifications are triggered only when their respective events occur. If All is selected, notifications are sent for all supported event types.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
