Overview

Aquera provides an innovative platform that allows your organization to scale complex data management by automating data synchronization from any source users to Jira Service Management.

Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This hook-based integration is triggered only after the user sync integration is completed. The hook-based integration is configured during the setup of the user sync integration.

NOTE: The source application selected for this hook-based integration must be the same as the source application configured in the user sync integration. For example, if Active Directory is configured as the source application in the user sync integration, then Active Directory should also be selected as the source application in this hook-based integration to perform the automated ticket generation process for Joiner, Mover, or Leaver events in Jira Service Management.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Automatically creates Jira tickets for Joiner, Mover, and Leaver events.
  • Supports ticket generation based on any user sync integrations between connected applications.
  • Allows configuration of Jira projects, work types, and change request categories.
  • Supports attribute mapping and data transformation using JavaScript expressions.
  • Generates logs and execution details for monitoring and troubleshooting integration activities.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Applications in the Aquera Admin Portal: 
    • Any Source Connector: Ensure you add and configure the source application connector in the Aquera Admin portal. 
    • Jira Service Management Connector: Ensure you add and configure the Jira Service Management connector in the Aquera Admin portal. Refer to the Jira Service Management Cloud Configuration Guide for details. 

Integration Setup

Aquera's Any Source Application / Jira Service Management integration automatically synchronizes the user sync type from the source application and automatically creates a Jira ticket using Jira Service Management based on a configured schedule.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel.
  3. Click Create Integration.
  4. Select Any Source / Jira Service Management. The Create New Integration pop-up window is displayed.
  5. Enter a name and a description for the integration.
  6. Select the source application connector and Jira Service Management connector from the corresponding drop-down lists.
  7. Click Continue. The integration is created.

Once created, the following tabs are displayed:

  • Overview
  • Source Application
  • JIRA
  • Execution
  • Notification
  • Schedule

Overview

In this tab, you can view the integration details and integration settings to be enabled based on the use cases.

The following field is visible in this section:

  • Enable Ticket Management: Enabling this will automatically generate tickets in Jira Service Management for every Joiner, Mover, or Leaver event.

Source Application

The Source Application tab is for configuring the required source application instance in this integration. The following field is displayed in this tab:

  • Select your Source Application Instance: Select the required application instance from the dropdown in which you want to configure this integration.

JIRA

The JIRA tab allows you to configure various settings required for the integration execution. The following sections are displayed in this tab:

  • JIRA Settings
  • Ticket Generation

JIRA Settings

The JIRA Settings section is for configuring the required JIRA application instance. The following fields are displayed in this tab:

  • Select JIRA Application: Select the Jira Service Management instance that will receive and manage tickets created from any source application user sync event.
  • Enable Ticket Creation Trigger Mode: Enable this option to configure how Jira tickets are created and processed from any source application user sync events using the available execution modes. When enabled, Select the Mode field appeared.
    • Select the Mode: Define the method used to generate Jira tickets from any source application user sync events, either through scheduled runs or real-time event triggers. By default, the hood-based ticket creation is enabled.

Ticket Generation

The Ticket Generation section is used to configure how Jira tickets are created and managed for JML lifecycle events. It allows you to define the Jira project, work type, and event-specific settings required for ticket creation workflows. The following fields are displayed in this section:

  • Select JML type: Select the employee lifecycle event type from the dropdown in which it should trigger Jira ticket creation, including Joiner, Mover, or Leaver events.
  • Jira Space: Select the Jira project space where tickets for the configured JML events will be created and managed.
  • Jira Work Type: Select the Jira work type that should be used when creating tickets for the configured JML workflows.
  • Select change type for Mover: Select the change types that should trigger Jira change requests for Mover events, such as department updates or employee detail changes.
  • Attribute Map: Use this section to map source attributes to target application attributes. You can modify source values, use JavaScript expressions for transformations, and configure required fields marked with a red asterisk (*).

Execution

The Execution tab allows you to configure various settings required for the integration execution. The following sections are displayed in this tab:

Advanced settings

Enable Report-Only mode: Enabling this will execute the integration run without making any changes or updates by only generating the report.

Operational Settings

This section enables you to customize the operational settings for the integration. It has the following sub-sections for configuration:

  • Logging Settings
  • Report Settings
  • Log Insights Settings

Logging Settings

  • Select log type: Select the log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.
  • Select Log Level: Select the level of detail to include in logs.
    The following options are available in the dropdown:
    • Full: Includes all attribute details in the logs.
    • Include List: Includes only the attributes specified in the Custom Inclusion List field.
      If Include List is selected, the following field is displayed:
      • Custom Inclusion List: Enter the attributes that should be included in logs. For example, entering firstName, lastName, email ensures only these fields are captured.
    • Exclude List: Includes all attribute details except those specified in the Custom Exclusion List field.
      If Exclude List is selected, the following field is displayed:
      • Custom Exclusion List: Enter the attributes that should be excluded from logs. For example, entering salary and Location ensures these fields are not captured.

Notifications

Enable Notifications: This tab allows you to configure email notifications for integration events, such as Joiner, Mover, or Leaver, and to define the recipients and report details included in the notification emails.

If the List View toggle button is disabled, the following parameters are displayed:

  • Template Name: Provide the template a name that matches the event.
  • Select Application: Select the application from which the notification must be generated.
  • Select event for notification: Select the corresponding event to configure notifications for ticket creation activities.
  • Notification delivery channel: Allows you to pick the notification delivery medium.
  • Custom notification recipients: Enabling this will allow you to send the notification emails to custom person defined by the user. When enabled, the custom email address provider field appears.
    • Custom email address: Enter a custom person's email address or a group of people to whom you want to send these notifications.
  • Attach Report: Enabling this will attach the generated report along with the email sent.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
