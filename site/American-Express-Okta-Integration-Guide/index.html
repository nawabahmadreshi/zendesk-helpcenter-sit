Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines an integration that automates the creation and management of American Express (Amex) card applications for employees using data from Okta. The integration creates card applications for employees who meet eligibility criteria in Okta and prevents duplicate applications. It also assigns the appropriate card type [Green or Platinum] based on the employee's job title and suspends the card in Amex when the employee is terminated in Okta.

To identify user records across both systems, the integration uses the Employee Number attribute from Okta and matches it to the Employee ID in Okta as a correlation attribute.

Key Functions

The Okta to American Express @ Work integrations provide the following functions:

  • Identifies employees eligible for card application creation based on their Okta status, group membership, and job title.
  • Ensures that a new application is created only if the employee does not already have a card or an existing application at Amex.
  • Assigns the appropriate card type based on the employee's job title.
  • Suspends the employee’s card in Amex when the employee is terminated in Okta.

The following are the prerequisites:

Integration Setup

This section outlines how to configure the integration to push employee card application data from Okta to American Express @ Work.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search American Express @ Work / Okta Integration.
  5. Enter a name and a description for the integration.
  6. Select both application connector instances from the dropdown menus.
  7. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Notification Center

The Notification Center section enables you to customize the integration settings according to your specific use case. It has the following settings:

Notification Settings

Configure the following notification settings:

  • Enable Notifications: Select this option to enable the integration-related notifications
    • Notify admin via email: Select this option to send email notifications to all admins
      • Admin Notification Events: Select the events that should trigger an email notification to the admin.
      • Admin Notification Operations: Select the operations that should trigger an email notification to the admin.
      • Attach Report to Email: Select this option to attach the sync report to notification emails. When enabled, the Notification Report Settings section is displayed for further configuration.
    • Notify employees via email: Select this option to send email notifications to all employees

Notification Report Settings

Configure the following notification report settings:

  • Select Report Detail Level: Select the desired report detail level. The following are the available options:
    • Full Details: All attribute details are displayed.
    • Include List: Select this option to report only the attributes specified in the Custom Inclusion List.
      • Custom Inclusion List: Select the attributes to be included in the report.
    • Exclude List: Select this option to report only the attributes specified in the Custom Exclusion List.
      • Custom Exclusion List: Select the attributes to be excluded from the report.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
