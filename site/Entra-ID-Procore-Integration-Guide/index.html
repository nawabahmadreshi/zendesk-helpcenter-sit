Entra ID / Procore Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the Entra ID to ServiceNow integration, which synchronizes data from Entra ID to ServiceNow to support user lifecycle and ticket management.

To identify user records across both systems, the integration uses the User Principal Name attribute from Entra ID and matches it to the Email Address in Procore as a correlation attribute.

Key Functions

The Entra ID to Procore integration provides the following functions:

  • Creates new users in Procore when they do not exist.
  • Updates existing users when changes are detected.
  • Automatically assigns or updates user project memberships in Procore.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Microsoft 365: Refer to the Microsoft 365 Configuration Guide for details.
    • Procore: Refer to the Procore Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Entra ID to ServiceNow.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Entra ID / ServiceNow in the search box next to the Explore Templates section and click it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Select your Microsoft 365 and ServiceNow connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable User Lifecycle Management: Select this checkbox to synchronize users' data from Entra ID to ServiceNow. When enabled, the User Management section is displayed to configure further integration settings.
  • Enable Onboarding Ticket Management: Select to enable onboarding ticket from Entra ID to ServiceNow. When enabled, the Onboarding Ticket Settings section is displayed to configure further integration settings.
  • Enable Offboarding Ticket Management: Select to enable offboarding ticket from Entra ID to ServiceNow. When enabled, the Offboarding Ticket Settings section is displayed to configure further integration settings.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.

User Management

The User Management section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable User Lifecycle Management is enabled in the Integration Scenarios section.

User Provisioning Settings

Configure the following user provisioning settings:

  • Source Correlation Attribute: Select the Entra ID attribute to uniquely identify and correlate users between both systems.
  • Destination Correlation Attribute: Select the ServiceNow attribute to uniquely identify and correlate users between both systems.
  • Enable Source Filter: Select to enable the source filter. When enabled, the following fields are displayed:
    • Source Filter Attribute: Select the Entra ID attribute to filter the users while retrieving from Entra ID.
    • Source Filter Value: Provide the value for the selected source filter.
  • Create User: Select this checkbox to create a user from Entra ID to ServiceNow. When enabled, the Create User Settings section is displayed for further configuration.
  • Update User: Select this checkbox to update a user from Entra ID to ServiceNow. When enabled, the Update User Settings section is displayed for further configuration.
  • Deactivate User: Select this checkbox to deactivate a user from Entra ID to ServiceNow
  • Incremental Sync: Select this checkbox to run the integration in incremental mode.
  • Sync User Manager: Select this checkbox to assign the user's Entra manager as the ServiceNow manager.
  • Microsoft Entra ID Application ID: Enter the Microsoft Entra ID Application (client) ID to retrieve custom extension properties for users. 

User Provisioning Threshold Settings

Configure the following provisioning threshold settings:

  • Import Limit: Specify the maximum number of user records that can be imported in the target application during a single integration run.
  • Create Limit: Specify the maximum number of user records that can be created in the target application during a single integration run.
  • Update Limit: Specify the maximum number of user records that can be updated in the target application during a single integration run.
  • Deactivate Limit: Specify the maximum number of user records that can be deactivated from the target application during a single integration run.

Create User Settings

Configure the following create user settings:

  • Create Condition: Select a condition from the dropdown menu to determine when users are created.
    • All Employees: Select this option to create all employees from Entra ID in ServiceNow.
    • All Active Employees: Select this option to create all active employees from Entra ID in ServiceNow.
    • All Inactive Employees: Select this option to create all inactive employees from Entra ID in ServiceNow.
    • Custom: Select this option to define a custom condition to create an employee from Entra ID in ServiceNow.
      • Custom Create Condition: Enter the JavaScript expression to create an employee from Entra ID in ServiceNow.

Update User Settings

Configure the following update user settings:

  • Update Condition: Select a condition from the dropdown menu to determine when users are updated.
    • All Employees: Select this option to update all employees from Entra ID in ServiceNow.
    • All Active Employees: Select this option to update all active employees from Entra ID in ServiceNow.
    • All Inactive Employees: Select this option to update all inactive employees from Entra ID in ServiceNow.
    • Custom: Select this option to define a custom condition to update an employee from Entra ID in ServiceNow.
      • Custom Create Condition: Enter the JavaScript expression to update an employee from Entra ID in ServiceNow.

Onboarding

The Onboarding Ticket Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Onboarding Ticket Management is enabled in the Integration Scenarios section.

  • Ticket Trigger Type: Select the Ticket Trigger Type to create a ticket from the dropdown menu. 
    • Ticket Type: Select the ticket type used to onboard a user from Entra ID to ServiceNow. For example, select Incident to create tickets under Incident Management.
    • Service Catalog: Select the service catalog used to onboard a user from Entra ID to ServiceNow. For example, select Desk Phone to create tickets under Desk Phone Management.
    • Order Guide: Select the order guide used to onboard a user from Entra ID to ServiceNow. For example, select New Hire to create tickets under New Hire Management.
  • Set "Requested for" to the onboarded user: Select to update the Requested For field in the service request to the onboarded user.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
