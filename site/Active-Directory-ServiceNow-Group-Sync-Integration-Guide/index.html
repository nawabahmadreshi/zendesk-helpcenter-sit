Active Directory / ServiceNow Group Sync Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the steps required to set up an integration that synchronizes user data from ServiceNow to Active Directory and enables the writeback process from Active Directory to ServiceNow to support user lifecycle and group management.

To identify user records across both systems, the integration uses the Employee number attribute from ServiceNow and matches it to the Employee ID in the Active Directory as a correlation attribute.

Key Functions

The ServiceNow / Active Directory integrations provide the following functions:

  • User Lifecycle Management: Handles user creation, updates, deactivation, and deletion from ServiceNow to Active Directory. If the Enable ServiceNow writeback feature is enabled, the integration can create, update, or deactivate user records in ServiceNow based on changes made in Active Directory
  • Group Assignment: Assigns users to specific groups in Active Directory based on work location attributes like location, country, state, and city. Group assignments are ignored if the conditions are not met or if work location details are missing.

Integration Setup

This section describes the configuration required to integrate ServiceNow with Active Directory and enable the writeback process from Active Directory to ServiceNow. It outlines the steps to create the integration, configure integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Active Directory / ServiceNow in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Enable the following integration feature:

  • Enable Group Management: Select to enable group management from ServiceNow to Active Directory. When enabled, the Group Management section is displayed to configure further integration settings for group management.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Report Generation: Select to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Notification: Select to enable the integration-related notifications. When enabled, the Notifications section is displayed for further configuration.
  • Enable Log Insights: Select to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Ticket Reassignment For Failure: Select to automatically reassign the ticket to the configured group if the requested operation fails during the integration run. When enabled, the Ticket Reassignment section is displayed under the Group Management section.
  • Enable Automated Closure of Related Tasks: Select to automatically close all associated tasks when the parent RITM is resolved.

Group Management

The Group Management section allows you to control how users are added to, updated in, or removed from Active Directory Security Groups and Distribution Lists (DLs) based on ServiceNow tickets.

Group Management Settings

Enable the following group management settings based on your requirements:

  • Enable Security Group Management: Select to manage Active Directory security group memberships based on ServiceNow requests. When enabled, the Security Group Management Settings section is displayed for further configuration.
  • Enable Distribution List Management: Select to manage Active Directory distribution list memberships based on ServiceNow requests. When enabled, the Distribution List Settings section is displayed for further configuration.
  • Enable Group Type Match: Select to ensure that the requested group type (Security Group or Distribution List) from ServiceNow matches the actual group type in Active Directory before processing. If the types do not match, the integration will fail.

User Correlation Settings

Configure the following correlation field settings:

  • Select Source Attribute: Select an attribute in ServiceNow that matches the selected Active Directory attribute value to identify common users across ServiceNow and Active Directory. For example, select Employee number as the correlation attribute in ServiceNow.
  • Select Target Attribute: Select an attribute in Active Directory that matches the selected ServiceNow attribute value to identify common users across ServiceNow and Active Directory. For example, select Employee ID as the correlation attribute in Active Directory.

Ticket Reassignment

Configure the following settings for the reassignment of the ticket:

  • Enter Group Name For Re-assignment: Enter the group name to reassign the RITM ticket if the requested operation fails during the integration run.

Distribution List Settings

Configure the following user provisioning settings:

  • Select Catalog: Select the ServiceNow catalog that is used for distribution list requests. For example, select Create User Catalog to process only user creation-related requests.
  • Variable for user's first name: Select the ServiceNow catalog variable that identifies the user's first name.
  • Variable for user's middle name: Select the ServiceNow catalog variable that identifies the user's middle name.
  • Variable for user's last name: Select the ServiceNow catalog variable that identifies the user's last name.
  • Variable for user's manager: Select the ServiceNow catalog variable that identifies the user’s manager information.

Security Group Management Settings

Configure the following settings related to user creation:

  • Select Catalog: Select the ServiceNow service catalog that is used for security group requests. For example, select Create User Catalog to process only user creation-related requests.
  • Variable for user's first name: Select the ServiceNow catalog variable that identifies the user's first name.
  • Variable for user's middle name: Select the ServiceNow catalog variable that identifies the user's middle name.
  • Variable for user's last name: Select the ServiceNow catalog variable that identifies the user's last name.
  • Variable for user's manager: Select the ServiceNow catalog variable that identifies the user’s manager information.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
