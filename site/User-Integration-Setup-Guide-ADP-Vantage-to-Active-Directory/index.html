User Integration Setup Guide - ADP Vantage to Active Directory

Overview

This guide covers the end-to-end tasks involved in setting up a synchronization to sync users from ADP Vantage to Active Directory.

The following tasks are involved in setting up the synchronization from ADP Vantage to Active Directory:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Synchronization Setup
  • Running Synchronization

Initial Preparation

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • ADP Vantage: Refer to the ADP Vantage Configuration Guide for details.
  • Active Directory: Refer to the Active Directory Configuration Guide for details.

Synchronization Setup

This section details how the User Synchronization must be configured to allow data push from ADP Vantage to Active Directory. 

It also describes the various analyses to be performed and the important points to be considered before setting up the user synchronization.

Considerations

  • Work email can be retrieved from the ADP Vantage system or be generated by Aquera synchronization and updated back to the system.
  • The criteria for matching a user record from ADP Vantage with that in the Active Directory.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping ADP Vantage user attributes to Active Directory user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in ADP Vantage as well as in Active Directory using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script synchronization. For scripts, refer to the Appendix.

Building Synchronization

This document describes how to set up the synchronization using the User Synchronization method.

  • User Synchronization - In this synchronization, set ADP Vantage as the Source Application and Active Directory as the Target Application. Refer to the User Synchronization Setup Guide for details on the configurations.

You can set the following configurations in the User Synchronization:

  • Set the Admin Email Options to receive email notifications for success/failures during the synchronization run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the User Synchronization Setup Guide for details on the configurations.

Synchronization Scenarios

This section details the synchronization scenarios.

Creating a user of type NORMAL_ACCOUNT in the Active Directory

To create a user of type NORMAL_ACCOUNT in Active Directory, set the attributes mapping as follows:

Enter "512" as the source attribute expression and map it to the [IAN].userAccountControl attribute of AD. Set the mapping criterion as Map on Create.

Note: 512 is the decimal value corresponding to the NORMAL_ACCOUNT property flag in Active Directory. For all property flags and the corresponding decimal values, refer to the Microsoft documentation.

Prompting for Password Change at Login

To make it mandatory for the new users in Active Directory to change their password on their first login, map the Active Directory attribute pwdLastSet to "0" with the mapping option of Map on create only. If this mapping is not done and users sign into their account for the first time, it will not prompt them to change their password.

To prompt the existing users also for a password reset, set the mapping option as Map on Create and Update.

Requirements to set user password

Active Directory requires a secure LDAPS (LDAP over SSL) connection to set or change user passwords. This is an Active Directory’s security requirement. Random generation of passwords when creating new users does not work without the use of LDAPS. Also, setting the password through attribute mapping requires a secure LDAPS connection.

Note: If the connection is not secured (non-SSL), Active Directory does not allow you to update the user password. 

Active Directory imposes strength requirements on the password and the requirements vary from one instance to another.

Running Synchronization

  • Enter the matching expression to match the source records with the destination records.
  • Run the synchronization in Report Only mode and determine if the correct data is updated in Active Directory. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the synchronization logs thoroughly to ensure that no errors are encountered. Modify the synchronization configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in ADP Vantage.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in ADP Vantage and Active Directory before setting up the synchronization. 

This section explains how analyses can be done using scripts. In the synchronization using scripts, add ADP Vantage and Active Directory applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script synchronization, refer to the Synchronization Setup Using Scripts Guide.

A few sample scripts are provided here for your reference. The scripts may vary based on your usecase and the attributes to be retrieved for analysis.

Analyzing the Users in ADP Vantage

Use the below sample script in a script synchronization for generating a report from ADP Vantage for analysis of username, user title, work location, and department. The work location and department are mapped to IAN attributes.

let adp_users = ADP.getUsers();

let i = 0;

log.info

('ADP ID,ADP STATUS,ADP EMPNO,ADP GIVENNAME,ADP FAMILYNAME,ADP WORK EMAIL,ADP LOCATION CODE,ADP JOBCODE,ADP DEPT');

for (i = 0; i < adp_users.length; i++) {

    let adp_id = adp_users[i].id;

    let adp_status = adp_users[i][IAN].status;

    let adp_emp = adp_users[i][ENT].employeeNumber;

    let adp_locationcode = adp_users[i][IAN].LocationCode || '';

    let adp_job = adp_users[i][IAN].jobCode || '';

    let adp_dept = adp_users[i][IAN].departmentCode || '';

    let adp_given = adp_users[i].name.givenName;

    let adp_family = adp_users[i].name.familyName;

    let adp_work = adp_users[i].emails && adp_users[i].emails.find(x => x.type == 'work') && adp_users[i].emails.find(x => x.type == 'work').value || '';

    log.info('"' 

    + adp_id + '","' 

    + adp_status + '","'

    + adp_emp + '","'

    + adp_given + '","' 

    + adp_family + '","'

    + adp_work + '","'

    + adp_locationcode + '","'

    + adp_job + '","'

    + adp_dept + '"');

}

Analyzing the Users in Active Directory

The below sample script can be used in a script synchronization for generating a report from Active Directory for analyzing user status, department, division, user type, and so on.

log.info('AD ID,AD ACTIVE,AD EMPNO,AD GIVENNAME,AD FAMILYNAME,AD USERNAME,AD SAMACCOUNT,AD DN');

for (let user of AD.getUsers()) {

    log.info('"'

    + user.id + '","'

    + user.active + '","'

    + (user[ENT] && user[ENT].employeeNumber || '') + '","'

    + user.name.givenName + '","'

    + user.name.familyName + '","'

    + user.userName + '","'

    + user[IAN].sAMAccountName + '","'

    + user[IAN].distinguishedName.split(',').slice(1).join(',') + '"');

}
