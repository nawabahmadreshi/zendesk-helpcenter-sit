Pinpoint Candidates to ADP Vantage HCM for API Central Employees

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

This integration synchronizes candidate onboarding data from the Pinpoint HRM application to ADP Vantage HCM for API Central Employees. It extracts candidate data from Pinpoint, creates a new employee record in ADP, and generates a structured CSV export.

The integration is triggered when a candidate’s hiring process is finalized in Pinpoint (e.g., offer acceptance or onboarding initiation).

This integration performs the following tasks:

  • Receiving the candidate information from Pinpoint HRM.
  • Sends the organized candidate profile to ADP Vantage HCM for API Central Employees to create a new employee record.
  • If successful, it creates a CSV report with all candidate details.
  • Adds status columns to the report (Success or Failed) with timestamps and error messages.
  • Emails the hiring manager or admin about the result (Success/Failure) and any errors.
  • If enabled, it saves a copy of the data and results to a logging system (OpenSearch) for future review.
  • Log errors clearly for troubleshooting (e.g., “Failed to send data to ADP: Invalid date format”).

Note: You must set up webhooks in the Pinpoint application to send the candidate hire events to this Aquera integration. Refer to the Appendix for details.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Pinpoint HRM: Refer to the Pinpoint Configuration Guide for instructions on creating the Pinpoint HRM application connector and setting up the authentication.
    • ADP Vantage HCM for API Central Employees: Refer to the ADP Vantage HCM Configuration Guide for API Central Customers for instructions on creating the ADP Vantage HCM for API Central Employees application connector and setting up the authentication.

Integration Setup

This section describes the configuration process for integrating Pinpoint HRM with ADP Vantage HCM for API Central Employees. It covers creating the integration, configuring system preferences, and setting up event-driven triggers to automate the real-time transfer of new hire data from Pinpoint HRM to ADP Vantage HCM for API Central Employees.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Pinpoint employees to ADP Vantage HCM for API Central Employees.
  5. Enter a name and a description for the integration.
  6. Select your Pinpoint and ADP Vantage HCM for API Central Employees connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Preferences

  • Company Code: A unique identifier for each organizational unit.
  • Default Hours: Specifies the standard number of hours employees are expected to work in a given period, such as per week or day.
  • Recipient Email Addresses: Specifies the email addresses to which notifications are sent. It supports multiple addresses separated by commas or mailing lists.
  • Email Subject:  The subject line used in the e-mail notification sent when a user is created in the target application.
  • Email Body: The main content of the email, containing the message and formatting (plain text/HTML).
  • Debug Mode: Simulates the email-sending process without actual delivery, logging details such as recipient, subject, and body for debugging purposes.
  • Report Only: Generates a report that shows the data that would be changed without making those changes.
  • Email Body on Failed Integration:  Predefined template or content used in emails sent when the integration encounters errors or fails to execute successfully.

Set Up Event Triggers

You need to set up the event triggers to receive candidate hire events from the Pinpoint HRM application and run the integration.

  1. In the Trigger section, click Event.
  2. Select the Pinpoint HRM connector configured in your Aquera Admin tenant in the Event Source drop-down.
  3. The URL field displays the endpoint URL to which Pinpoint HRM will send the candidate hire events. This URL must be configured in the Pinpoint HRM application when configuring the webhooks. For more details on configuring webhooks, see Setting up Webhooks in Pinpoint HRM.
  4. Click Create Token to generate a token. Save this token securely.

Run the Integration

The integration is triggered when a candidate's hiring process is completed in the Pinpoint HRM application. It will process the event received from Pinpoint HRM and create an employee record in the ADP Vantage HCM for the API Central application. For more details on Events, see the Set Up Event Triggers section.

Once an employee is successfully onboarded to the ADP Vantage HCM for the API Central application, the integration triggers an email notification to the email addresses configured in your integration. It will also send messages and reports to the email addresses in case of integration errors.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.

Appendix

Setting Up Webhooks in Pinpoint HRM

This section details how to set up webhooks in the Pinpoint HRM application to send the candidate hire events to the Aquera integration for processing.

  1. Log in to Pinpoint and, using the left-hand navigation, go to Company > Profile > Webhooks.
  2. Click the New webhook button.
  3. Specify a valid URL, whether HTTP or HTTPs.

Note: You can get the URL while setting up the Event trigger. For more details, see the Set Up Event Triggers section.
