Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables seamless synchronization between Jamf Pro and Paylocity, enhancing asset management and simplifying employee-device assignments. It continuously retrieves and reconciles asset data from Jamf Pro, including iPads, MacBooks, and other Apple devices, with employee records in Paylocity, ensuring better asset tracking and accurate device allocation across your organization.

To identify user records across both systems, the integration uses the email address attribute from Jamf Pro and matches it to the email address in Paylocity as a correlation attribute.

Key Functions

The Jamf to Paylocity integration provides the following functions:

  • Retrieves asset data from Jamf Pro, supporting incremental updates and filtering by source.
  • Accesses and synchronizes employee records from Jamf Pro to Paylocity, aligning these accurately with device assets based on email addresses and device serial numbers.
    Note: Once a device has been assigned to a user along with its serial number, the same device cannot be assigned to another user.
  • Dynamically allocates or de-allocates devices within Paylocity based on up-to-date Jamf records, ensuring accuracy and compliance.
  • Handles successful events by logging details, generating CSV reports, sending notification emails, and updating log insights data, if enabled.
  • Handles failure events by logging errors, creating failure-specific CSV reports, sending failure notifications, and updating dashboard analytics, if enabled.
  • Sends automated, detailed notification emails to employees and administrators regarding device assignments, integration status, and updates.
  • Produces CSV reports capturing user data, operation results, timestamps, and error details.
  • Enables log insights to monitor integration outcomes and performance.

Integration Setup

This section details the configuration of the scheduled integration between Jamf and Paylocity. It covers creating the integration, setting up integration preferences, and configuring how device and employee lifecycle events are synchronized. The integration reads device and user data changes from Jamf, including new device enrollments, updates, and user assignments, at scheduled intervals and creates or updates corresponding records in Paylocity.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Jamf Pro to Paylocity Assets Synchronization.
  5. Enter a name and a description for the integration to identify it.
  6. Select your Jamf Pro and Paylocity connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenario

Configure the following integration scenario:

Enable Asset Management: Select this checkbox to activate the asset management feature for the company.
Note: The Asset Management settings section will be displayed only when this checkbox is selected.

General Settings

Configure the following general settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select this checkbox to generate logs outlining data changes.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics.
    Note: The Log Insights Settings section will be displayed only when this checkbox is selected.

Asset Management

Configure the following settings for the asset management:

Source Settings

Configure the following settings in Jamf Pro:

  • Company ID Attribute Name: Enter the attribute name of the company ID in Jamf Pro.
  • Device Type: Select the device type(s) for synchronization from the dropdown menu. The available options are Mobile Devices and Computer Devices.
  • Company ID Separator: Enter the special character that separates the company ID from its value. For example, if the value string is ABCDE‑12345, where ABCDE is the company ID, and 12345 is its value, then the separator is “‑”. In this case, enter “‑” as the company ID separator.
    Note: A space is not allowed as a separator (for example, ABCDE  12345 is invalid). Using a space will cause the integration to throw an error.
  • Company Name Index: Enter the numeric position (index) of the company’s name within the full value string. Indexing starts from 0, 1, 2, and so on.
    Examples:
    • If the full value is ABCDE‑12345, and ABCDE is the company name, then the index of your company name is 0.
    • If the full value is ABCDE‑12345‑EFGHI, and EFGHI is the company name, then the index of your company name is 2.
    Note: This setting becomes visible and mandatory when you have entered a separator in the Company ID Separator field.
  • Advanced Sync & Filters: Select the advanced filtering and synchronization option for the integration run from the dropdown menu.
    • Incremental Sync: Select this option to run the integration in incremental mode.
    • Full Run: Select this option to run the integration in full sync mode.
    • Source Filter: Select this option to retrieve a specific device from Jamf Pro. The following settings will be displayed when you select this option:
      • Source Filter Attribute: Select the source filter attribute to filter devices. The available options are serialNumber and modelNumber.
      • Source Filter Value: Enter the value for the selected source filter to retrieve a specific device.

Target Settings

Configure the following settings in Paylocity:

    • Company ID: Enter your company ID to enable device management for your organization.
    • Custom Target Field: Enter the custom target field in Paylocity to which the asset data will be synchronized.
      Note: Only the TextField data type is supported on the target custom field.
    • Number of Devices to Sync: Enter the number of devices to synchronize in an integration run.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Report Settings

Configure the following report settings:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Notification Settings

Configure the following settings for email notifications:

  • Notify Admin Via Email: Select this checkbox to send email notifications to all admins. When this checkbox is selected, the following settings will be displayed:
    • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
    • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
    • Admin Email Body: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select this checkbox to receive an email notification with a CSV report for all integration runs. When this checkbox is selected, the following settings will be displayed:
    • Select the Notification Report Detail Level: Select the desired notification report level from the dropdown menu. The following are the available options:
      • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
      • Full Details: All attribute details are displayed.
  • Notify Employees via Email: Select this checkbox to send email notifications to all employees. When this checkbox is selected, the following settings will be displayed:
    • Employee Email Subject: Enter the content that must be used as the email subject for sending email notifications to the employees.
    • Employee Email Body: Enter the content that must be used as the email body for sending email notifications to the employees.

Log Insights Settings

Configure the following log insights settings:

  • Select the Log Insights Detail Level: Select the desired level of log insights detail from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
  • Dashboard Name: Enter the name you want to use for the Log Insights dashboard.
