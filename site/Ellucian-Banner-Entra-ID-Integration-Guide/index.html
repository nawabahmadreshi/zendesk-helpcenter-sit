Ellucian Banner / Entra ID Integration Guide

Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This guide outlines the end-to-end steps required to set up the integration that synchronizes employees and students data between Ellucian Banner and Entra ID.

To identify records across both systems, the integration uses the Banner ID from Ellucian Banner and matches it to the Employee ID in Entra ID as a correlation attribute.

Key Functions

This integration automates identity lifecycle management between Ellucian Banner (a higher education ERP system) and Entra ID (formerly Azure AD) with the following primary capabilities:

  • Synchronizes user records from Ellucian Banner to Microsoft Entra ID.
  • Creates new users in Entra ID when they are added in Ellucian Banner.
  • Updates existing Entra ID users when their information changes in Ellucian Banner.
  • Deactivates or deletes Entra ID users when they are no longer active in Ellucian Banner.
  • Assigns users to groups in Entra ID based on configured rules.
  • Generates passwords, email IDs, and UPNs (User Principal Name) for users in Entra ID based on configured policies.
  • Updates email IDs from Entra ID to Ellucian Banner (writeback).
  • Supports both full synchronization and incremental synchronization to process only changed records.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Ellucian Banner: Refer to the Ellucian Banner Configuration Guide for configuration details.
    • Microsoft 365: Refer to the Microsoft 365 Configuration Guide for configuration details.

Integration Setup

This section details the configuration and mapping process for integrating Ellucian Banner with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Ellucian Banner / Entra ID.
  5. Enter a name and a description for the integration.
  6. Select your Ellucian Banner and Microsoft 365 connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

On the Configuration page, the following sections are displayed:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and general settings according to your specific use case.

Integration Scenarios

Enable the following feature based on your requirements:

  • Enable User Lifecycle Management: Select this checkbox to synchronize users' data from Ellucian Banner to Entra ID. When enabled, the Manage Employees section is displayed to configure further integration settings.
  • Enable Group Assignment: Select this checkbox to enable group assignments in Entra ID. When enabled, the Group Assignments section is displayed to configure further integration settings.
  • Enable Email Writeback: Select this checkbox to enable updating the email ID of users from Entra ID to Ellucian Banner.

General Settings

Configure the following settings:

  • Enable Report Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Logging section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Reporting section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights section is displayed for further configuration.
  • Enable Debug Mode: Select this checkbox to execute the integration in debug mode for debugging purposes. When enabled, the Debug Mode section is displayed for further configuration.

Manage Employees

The Manage Employees section enables you to customize the integration settings for employee lifecycle management according to your specific use case. It synchronizes employees and students data from Ellucian Banner to Entra ID.
Note: This section appears only if the Enable User Lifecycle Management setting is enabled in the Parameters section.

It has the following fields:

  • Ellucian Banner Settings: This section allows you to configure settings for Ellucian Banner. The following parameters are displayed:
    • Ellucian Banner Correlation Attribute: Identifies the Ellucian Banner attribute used to uniquely identify and correlate users between Ellucian Banner and Entra ID.
    • User Lifecycle Management: Select the user lifecycle operations that will be allowed through this integration. For example, enter Create and Update if you want all create and update operations to be managed through this integration.
  • User Type for Lifecycle Management: Select the type of users whose identities will be provisioned and managed through this integration. You can select one or both of the following options:
    • Employee: Select this to include employees.
    • Student: Select this to include students.
  • Data Sources: Select the data sources from Ellucian Banner that will be used to collect resource details for the integration.
  • Ignore Errors for Data Sources: Select the data sources for which API errors must be ignored during data processing.
  • Entra ID Settings: This section allows you to configure settings for Entra ID. The following parameters are displayed:
    • Entra ID Correlation Attribute: Identifies the Entra ID attribute used to uniquely identify and correlate users between Entra ID and Ellucian Banner.
  • Create Settings: This section allows you to configure the create conditions. The following parameters are displayed:
    • Create Condition: Select a create condition to define which new record from Ellucian Banner must be created in Entra ID. The following are the available options:
      • All: Select this if you want all new records to be created in Entra ID.
      • Custom: Select this if you want to define a custom condition for creating users in Entra ID. When selected, the following field is displayed:
        • Custom Condition: Enter an expression to define which new records from Ellucian Banner must be created in Entra ID.
  • Update Settings: This section allows you to configure the update conditions. The following parameters are displayed:
    • Update Condition: Select the update condition to define which record from Ellucian Banner must be updated in Entra ID. The following are the available options:
      • All: Select this if you want all modified records to be updated in Entra ID.
      • Custom: Select this if you want to define a custom condition for updating users in Entra ID. When selected, the following field is displayed:
        • Custom Condition: Enter an expression to define a custom condition for updating users in Entra ID.
  • Delete Settings: This section allows you to configure the delete conditions. The following parameters are displayed:
    • Delete Condition: Select a delete condition to define which record from Ellucian Banner must be deleted in Entra ID. The following are the available options:
      • All: Select this if you want all inactive records to be deleted in Entra ID.
      • Custom: Select this if you want to define a custom condition for deleting users in Entra ID. When selected, the following field is displayed:
        • Custom Condition: Enter an expression to define a custom condition for deleting users in Entra ID.
    • Delete Employee on Deactivation: Select this checkbox to automatically delete user from Entra ID when the corresponding record is deactivated in Ellucian Banner.

Manage Policy

This section enables you to configure the settings to generate password, UPN, and email ID during user creation in Entra ID.

It has the following fields:

  • Generate Password Settings: This section allows you to configure the policy for generating passwords for new users in Entra ID. The following parameters are displayed in this section:
    • Password Generation Method: Select one of the following methods to generate a password for new users in Entra ID:
      • Auto Generate Password: Select this method if you want passwords to be generated automatically by the system.
      • Configure Password Policy: Select this if you want to define a policy for generating passwords. When selected, the following fields are displayed for further configuration:
        • Password Length: Enter the number to be 
        • Allowed Characters: Select the characters that are allowed to be included in password. For example, select Include Uppercase, Include Lowercase, and Include Special Characters to allow these characters in passwords.
          When Include Special Characters is selected, the following field is displayed for further configuration:
          • Allowed Special Characters: Enter the special characters that are allowed to be included in passwords. Only the configured characters can be used when special characters are required.
  • Generate UserPrincipalName (UPN) Settings: This section allows you to configure the policy for generating UPN for users in Entra ID. The following parameters are displayed in this section:
    • Policy for Generating User's UserPrincipalName (UPN): Enter your prompt to define the format to be used while generating UPN. After defining the format, click Analyse and Preview to view the output. You can also use the Advanced Preview Options to test the preview with custom values.
      To switch to a code-based view to edit the UPN generation logic directly instead of using plain text instructions, click Code View.
  • Generate Email Settings: This section allows you to configure the policy for generating email IDs for new users in Entra ID. The following parameters are displayed in this section:
    • Email Generation Method: Select one of the following methods to generate email IDs for new users in Entra ID.
      • Configure Email ID as UPN: Select this option if you want the email ID to be set to the same value as the user’s UPN.
      • Configure Email Policy: Select this if you want to define a policy for generating email IDs. When selected, the following fields are displayed for further configuration:
        • Policy for Generating User's Email: Enter your prompt to define the format to be used while generating email IDs for users in Entra ID.
          After defining the format, click Analyse and Preview to view the output. You can also use the Advanced Preview Options to test the preview with custom values.
          To switch to a code-based view to edit the logic directly instead of using plain text instructions, click Code View.

Group Assignments

This section enables you to configure the group assignment rules that define how groups are managed through this integration.

Note: This section appears only if the Enable Group Assignment setting is enabled in the Parameters section.

It has the following fields:

  • Assign License Groups if not in Selected Groups: Select this checkbox to assign license groups to a user only if the user is not already a member of any of the selected license groups. When enabled, the following field is displayed for further configuration:
    • License Groups to Check Membership: Select the license groups that will be checked to determine whether a user already has a license group assigned. Only users who are not members of these groups will be considered for group assignment.
  • Assign License to Users: This field allows you to configure the group rules to assign license groups to users in Entra ID.
    Perform the following procedure to create the assignment rules:
    1. Click Add New Rule to create a new assignment rule.
    2. Switch to Expression mode, and select one of the following based on your requirement:
      • For Conditions Only: Select this if you want to write a JavaScript expression only for the conditions.
         
      • For Conditions and Assignments: Select this if you want to write expression for both, conditions and assignments.
    3. Click Continue to configure the expression.
    4. Enter you expression and configure the groups to be assigned then click Save Changes.
    5. If required, click Add New Rule to add another assignment rule, as explained above.
      Note: You can add any number of assignment rules. However, you can see only 5 rules on a single page. If you add more than 5 rules, navigation options will be available at the bottom of the page to navigate to other pages for configuring or viewing the remaining defined rules.

Logging

Note: This section appears only if the Enable Logging setting is enabled in the Parameters section.

It has the following settings:

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
