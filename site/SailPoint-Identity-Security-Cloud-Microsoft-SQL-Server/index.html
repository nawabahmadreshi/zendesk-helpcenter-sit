Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

The SailPoint Identity Security Cloud to Microsoft SQL Server integration extracts the accounts and identities data, along with their selected attributes, from the SailPoint Identity Security Cloud app and creates the corresponding tables in the configured SQL Server database. 

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal for integration:
    • SailPoint Identity Security Cloud: Refer to the SailPoint Identity Security Cloud Configuration Guide for details.
    • Microsoft SQL Server: Refer to the Microsoft SQL Server Configuration Guide for details.

Integration Setup

This section outlines the configuration required to integrate SailPoint Identity Security Cloud with Microsoft SQL Server, enabling seamless transfer of accounts and identities data from SailPoint Identity Security Cloud to SQL Server.

Create Integration

Perform the following steps to create the integration:

  1. Perform the following steps to create the integration:
  2. Log in to the Aquera Admin portal.
  3. Navigate to Integrations from the left side panel.
  4. Click Create Integration.
  5. Search for Microsoft SQL Server / SailPoint Identity Security Cloud in the search box next to the Explore Templates section.
  6. Click the integration template to display the Create New Integration pop-up window.
  7. Enter a name and a description for the integration.
  8. Select both application connector instances from the dropdown menus.
  9. Click Continue. The integration configuration page is displayed to configure the integration.

Database

Choose where you want to store the synced data

Configure the following database settings:

  • Choose SQL Server application: Select the configured Microsoft SQL Server app.
  • Choose the destination database: Enter the database name from the dropdown menu for the configured SQL server

  • Choose how data changes should be tracked: Select the SCD type to write data into the selected database. The available options are SCD-0, SCD-1, and SCD-2.

    SCD-0 preserves the original data and does not allow updates once a record is created. SCD-1 replaces old data with new data without keeping any history. SCD-2 retains the latest data along with the complete historical changes.

Operation

The following sections are displayed under the Operation tab:

Execution

Rollout configuration

Configure the following option to control the execution behavior:

  • Report Only Mode: Enable this toggle button to run the integration in preview mode. When enabled, the system generates a report showing the data changes that would occur without writing any data to SQL Server. This helps validate the configuration before executing the integration. When enabled, the following field is displayed:
    • Number of users to test: Enter the limit on the number of worker records processed in this run. Set a small number (e.g. 10–50) to validate field mappings and data quality before running on the full population. Set to 0 to process all records. 

Monitor

Notification settings

The Notification Settings section enables you to customize email notification settings for the integration. It has the following sections:

Enable Email Notifications: Select to enable email notifications for the integration. When enabled, the following fields are displayed to configure:

  • Select Event Type: Select which integration event triggers the notification from the dropdown menu (shown when List View is disabled). The available options are:
    • Success: Set up the following integration success email notification settings:
      • Send Conditional Notification: When enabled, the Notification Condition field is displayed, and you enter your custom condition. This notification only fires when a custom condition is met.
      • Recipient Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
      • Email Subject Line: Enter the content that must be used as the email subject.
      • Email Template - Success: Enter the content for the email body to use for notifications sent upon successful integration execution.
      • Send Integration Report via Email: Enable this toggle button to receive email notification with a report for all successful integration runs.
      • Select Report Format: Select the format of the integration report. The available options are CSV and Excel.
    • Failure: Set up the following integration failure email notification settings:
      • Send Conditional Notification: When enabled, the Notification Condition field is displayed, and you enter your custom condition. This notification only fires when a custom condition is met.
      • Recipient Email Addresses: Enter the email addresses of the recipients to whom the email notifications will be sent.
      • Email Subject Line: Enter the content that must be used as the email subject
      • Email Template - Failure: Enter the content that must be used as the email body in the notifications sent for integration errors
      • Send Integration Report via Email: Enable this toggle button to receive email notification with a report for all unsuccessful integration runs.
      • Select Report Format: Select the format of the integration report. The available options are CSV and Excel.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Operational settings

The Operational Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

  • Enable logging: Enable the Enable Logging toggle button to display the Log level dropdown menu. Enabling logging generates logs outlining data changes.
  • Log level: Select the level of detail to include in the integration logs from the dropdown menu. The following are the available options:
    • Minimum - Operations only; no GUID or fields: Include operation details in the logs. Any field names and GUIDs are excluded.
    • Metadata - Operations + field names; no GUID: Include operation details and field names in the logs. GUIDs are excluded.
    • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the logs. GUIDs are excluded.
    • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the logs.
    • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the logs.
    • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the logs.
  • Enable reporting: Enable the Enable Reporting toggle button to display the Reporting type and Report format type fields. Enabling reporting generates integration reports.
  • Reporting type: Choose the level of detail to include in the integration reports from the dropdown menu. The following are the available options:
    • Minimum - Operations only; no GUID or fields: Include operation details in the report. Any field names and GUIDs are excluded.
    • Metadata - Operations + field names; no GUID: Include operation details and field names in the report. GUIDs are excluded.
    • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the report. GUIDs are excluded.
    • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the report.
    • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the report.
    • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the report.
  • Report format type: Select the format in which the integration report is created. The available options are CSV and Excel.
  • Enable log insights: Select to display the Log Insights Level dropdown menu. Enabling log insights generates dashboards.
  • Log insights level: Enable the Log insights level toggle button to select the level of detail to include in the log insights dashboard from the dropdown menu. The following are the available options:
    • Minimum - Operations only; no GUID or fields: Include operation details in the log insights dashboard. Any field names and GUIDs are excluded.
    • Metadata - Operations + field names; no GUID: Include operation details and field names in the log insights dashboard. GUIDs are excluded.
    • Non-PII Data - Operations + field names + selected non-PII values; no GUID: Include operation details, field names, and selected non-PII values in the log insights dashboard. GUIDs are excluded.
    • GUID with Non-PII Data - GUID + operations + field names + selected non-PII values: Include GUIDs with operation details, field names, and selected non-PII values in the log insights dashboard.
    • Employee Number with Non-PIl Data - Employee # + operations + field names + selected non-PII values: Include employee numbers, operation details, field names, and selected non-PII values in the log insights dashboard.
    • Employee Number with All Data - Employee # + operations + field names + all values (PII + Non-PII): Include employee numbers, operation details, field names, and all values, including both PII and non-PII data, in the log insights dashboard.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
