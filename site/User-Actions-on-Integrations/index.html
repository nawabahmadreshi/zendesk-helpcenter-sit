User Actions on Integrations

← Back to index

Overview

You can perform actions on integrations such as execute, upgrade, copy, modify, delete, export, import, view logs, and download reports. For details on the integration listing page and setup page, refer to the Integrations Overview Guide.

This guide explains the user actions that can be performed on an integration. 

Execute Integration

You can manually execute a created integration using one of the following approaches:

  • Navigate to the Integrations module on the Aquera Admin portal. Click the Execute icon on the left side of the integration record that needs to be executed. The integration starts executing.
  • Navigate to the Integrations module on the Aquera Admin portal. Click the name of the integration record that needs to be executed. Click the vertical ellipsis at the top-right corner of the integration setup page and click Execute to start the integration. 

Note: When you start an integration execution and the Report Only mode is not enabled for a User Integration or Bulk Integration, then a confirmation pop-up will be displayed asking you to confirm the sync with the target application.

Stop Integration

You can manually stop an integration that is already executing using one of the following approaches:

  • Navigate to the Integrations module on the Aquera Admin portal. Click the Kill icon on the left side of the integration record that needs to be stopped. The integration stops its execution.
  • Navigate to the Integrations module on the Aquera Admin portal. Click the name of the integration record that needs to be stopped. Click the vertical ellipsis at the top-right corner of the integration setup page and click Stop Execution.

Note: After the integration has completed its execution, there may be a slight delay in the UI to update the option from Stop Execution to Execute.

Copy Integration

You can copy the configuration of an existing integration and create a new integration with a different name. Hence, you do not need to configure all the settings again when you want another integration set up with similar details. This approach can be followed when a new version of the source or target application is deployed and configured in the Aquera Admin portal. As required, you may need to change the source or target application in the newly created integration.

Perform the following steps to copy the details of an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Hover the mouse pointer over the integration record to be copied.
  3. Click the Copy icon on the right side of the integration record. The Copy Integration pop-up window is displayed.
  4. Enter the name of the integration to be created and click Save. The new integration is created with the same configuration details as the copied integration.

Edit Integration

You can modify an existing integration by performing the following steps:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click on the integration record that you want to edit. 
  3. Click the pencil icon at the top left of the page to edit the name and description of the integration.
  4. Make changes to the configuration fields, attribute map, tables, or group rules as required in the integration.
  5. Click Save Changes on the top right corner of the page to save the integration changes.

When the same integration is concurrently edited in different sessions (multiple tabs/windows/users) and when trying to save this integration, a conflict occurs requiring you to refresh the integration page. When you click Refresh, your changes will be lost, and the last saved changes from the concurrent integration will be loaded. 

View Logs

The logs are generated when an integration runs as per the schedule set, event trigger, or manual execution. To view the integration logs, navigate to the Logs module and click the Integration tab on the Aquera Admin portal. For further details, refer to the Integration Logs Guide.

View Latest Logs

You can view the latest logs of an integration record by performing the following steps:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the name of the integration record.
  3. Click the vertical ellipsis at the top-right corner of the integration setup page.
  4. Click Latest Logs.
    • If you have selected a user integration, the Execution Timeline panel is displayed with the execution history of the selected integration.
      1. Click View Logs next to the last execution record. The detailed logs of that execution are displayed at the bottom of the page.
      2. Click Goto Logs to navigate to the Logs > Integration page, which lists the specific log of that particular integration run.
    • If you have selected a script-based integration, the detailed logs of the latest execution are displayed at the bottom of the page.
      • Click the Expand icon at the right corner of the Logs section to navigate to the Logs > Integration page, which lists the logs of that particular integration.

 For further details, refer to the Integration Logs Guide.

Export and Import Integration

You can use the export and import options of integration to copy an integration from one tenant to another tenant.

Perform the following steps to export an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the name of the integration record to be exported.
  3. Click the vertical ellipsis at the top-right corner of the integration setup page.
  4. Click Export to download the current integration record in JSON format. This JSON file can later be used to import the integration to another tenant. 

Perform the following steps to import an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click Create Integration at the top right corner of the page.
  3. Click Import Integration to import an existing integration.
  4. Select the JSON file of the integration that you would like to import.
  5. Provide a name and description for the new integration to be created.
  6. Select the relevant applications from the dropdown list.
  7. Click Continue. The integration is imported and then if required, you can modify the integration as per your requirement.

Notes:

  • Tables, group rules, integration hooks, and event triggers are currently not supported during the import of an integration.
  • Before exporting an integration, ensure the application names are valid.
  • While importing an integration, ensure the exported integration version matches the integration version in the tenant where it will be imported.

View Execution Status and Timeline

You can view the execution status and execution timeline of integration by navigating to the Integrations module on the Aquera Admin portal. The History column in the integration listing page shows the execution status of the recent five integration runs in the form of color bars. You can click the History column to view the integration execution timeline.

For further details, refer to the Integrations Overview Guide.

Upgrade an Integration

You can upgrade an integration to a higher version to make use of the new features or enhancements available in that version. 

Note: If you are upgrading an integration that was created from a template, it can only be upgraded to the latest version of the template. However, if you have created the integration from one of the integration types (for example, User Integration), you can choose the version to which the integration has to be upgraded.

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the Ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following details:
    • The name of the integration
    • The date of creation and modification of the integration
    • The user who created or modified the integration
    • If the integration is created from a template, the name and version of the template are also displayed.
  3. Click Upgrade if a new version is available for upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • Integration name
    • Description
    • Select the version to which the integration needs to be upgraded.
      Note: If you are upgrading a User Integration, you may see the older versions also in the drop-down list. However, you must not select a version prior to the existing version.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing integration, however, with the features/enhancements that are part of the upgraded version.

Notes:

  • For User Integration or Bulk Integration, the Configuration tab of the newly created integration will be displayed where you can make any configuration changes if required.
  • When you upgrade a User Integration to version 3 or version 4, ensure to remove any double quotes in the Incremental Mode Timestamp configuration. Otherwise, an error message is displayed in the logs when you run the upgraded integration.
  • When you upgrade a User Integration to version 3 or version 4, ensure that the configuration fields have valid values; otherwise, you may be unable to save your integration.

Download Integration Report

You can download the reports generated by an integration.

Perform the following steps to download a report:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click on the History column of an integration listed on the page. The Execution Timeline panel is displayed with the details of the integration runs.
  3. Click Download Report. The list of all the reports generated as part of the integration run is displayed.
    • To download a specific report in that list, click on the report to be downloaded.
    • To download all the reports as a single zip file, click All Reports. 

Note 1: The Download Report option is available only for User Integration version 4 onwards. If the integration execution has failed or you have cancelled the execution, this option will not be available. 
Note 2: For script-based integrations, the Download Report option is available only if the option to generate the reports is enabled in the integration configuration.

Delete Integration

You can delete an integration record using one of the following approaches:

  • Navigate to the Integrations module on the Aquera Admin portal. Hover the mouse pointer over the integration record to be deleted and click the Delete icon on the right side of the integration record. The Delete Integration pop-up window is displayed. Click Confirm to confirm the deletion.
  • Navigate to the Integrations module on the Aquera Admin portal. Click the name of the integration record that needs to be deleted. Click the vertical ellipsis at the top-right corner of the integration setup page and click Delete. The Delete Integration pop-up window is displayed. Click Confirm to confirm the deletion.

View Script

You can view the script of a workflow by performing the following steps:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the name of the workflow record.
  3. Click the vertical ellipsis at the top-right corner of the workflow setup page.
  4. Click View Script. This displays the View Script pane, where you can view the script of the Workflow. This is a read-only script.

Note: This option is available only for Workflows V1 version.

Set Log Levels

You can set the log level and execution mode of a workflow by performing the following steps:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the name of the workflow record.
  3. Click the vertical ellipsis at the top-right corner of the workflow setup page.
  4. Click Settings to display the Settings pane where you can set the Log Level and the Execution Mode of the Workflow logs. After selecting the required log settings, close the Settings pane and click Save Changes.

The following are the supported log levels: 

  • Default: Prints the output of the Log function. If there is no Log function in the Workflow, then it prints only the header details in the logs.
  • Info: Prints the following in the logs:
    • The application methods, flows, functions, and variables that are added to the Workflow.
    • The output of the Log function.
  • Verbose: Prints the following in the logs:
    • The application methods, flows, functions, and variables that are added to the Workflow.
    • The output values of the application methods, flows, functions, and variables.
    • The output of the Log function.

The following are the supported execution modes: 

  • Normal: Displays the error messages in the logs, and even if an error occurs, the Workflow continues execution.
  • Exception: Displays the error messages in the logs, and if an error occurs, the Workflow stops execution.
