Dayforce / Greenhouse Recruiting Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines how to synchronize department, job, and employee details from Dayforce to Greenhouse Recruiting. The integration ensures that Greenhouse Recruiting maintains up-to-date department records that match your Dayforce, enabling accurate job postings and recruiter organization.

To identify user records across both systems, the integration uses the XRefCode attribute from Dayforce and matches it to the Name in Greenhouse Recruiting as a correlation attribute.

Key Functions

The Dayforce to Greenhouse Recruiting integration provides the following functions:

  • Synchronizes department, job, and employee details from Dayforce to Greenhouse Recruiting
  • Creates new departments, jobs, and employee records in Greenhouse Recruiting based on Dayforce data.
  • Updated new departments, jobs, and employee records in Greenhouse Recruiting based on Dayforce data.
  • Deactivates and reactivates employee records in Greenhouse Recruiting based on Dayforce data.
  • Generates detailed logs and reports.

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.

  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Dayforce: Refer to the Dayforce Configuration Guide for details.
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push Job and Employee data from Dayforce to Greenhouse Recruiting.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Greenhouse Recruiting in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable department sync: Select this checkbox to sync department data from Dayforce to Greenhouse Recruiting. When enabled, the Department Configuration section is displayed for configuring additional integration settings.
  • Enable job sync: Select this checkbox to sync job data from Dayforce to Greenhouse Recruiting. When enabled, the Job Configuration section is displayed for configuring additional integration settings.
  • Enable Employee Sync: Select this checkbox to sync employee data from Dayforce to Greenhouse Recruiting. When enabled, the Employee Configuration section is displayed for configuring additional integration settings.

General Settings

Configure the following general settings:

  • Enable report-only mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable report generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable log insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insights Settings section is displayed for further configuration.

      • Custom exclusion list: Select the attributes to be excluded from the log.
  • PII fields: Enter the field names that contain personally identifiable information (PII).
  • GUID fields: Enter the field names that contain globally unique identifiers (GUIDs).
  • Employee ID fields: Enter the field names that contain employee IDs.

Department Settings

The Department Settings section enables you to customize the integration department settings according to your specific use case. It has the following settings:

  • Dayforce Correlation Attribute: Select the Dayforce correlation attribute to distinctly find the associated Greenhouse Recruiting department. For example, XRefCode.
  • Greenhouse Recruiting Correlation Attribute: Select the Greenhouse Recruiting correlation attribute to distinctly find the associated Dayforce department. For example, Name.
  • Source Filter: Enter the XRefCode value to filter departments while retrieving from Dayforce. 

Department Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side, and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long expression.

All attributes marked with a red asterisk (*) indicate required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

Employee Configuration

The following section is displayed under the Employee Configuration tab:

Note: This section appears only if the Enable Employee Sync parameter is enabled in the Integration Scenarios section.

Employee Settings

The Employment Settings section enables you to customize the integration settings according to your specific use case. It has the following sections:

Integration Settings

Configure the following settings

  • Enable Full Sync: Select this checkbox to run the integration in full sync mode. If this checkbox is not selected, the first execution runs in full sync mode by default, and subsequent executions run in incremental sync mode.
  • Full Sync Frequency (Days): Enter the number of days between two full sync runs. For example, if you enter 5 in this field, a full sync will run every 5 days. Provide a value in this field only if the Enable Full Sync parameter is selected.

Import Filter Settings

This section allows you to define limits for synchronization from Dayforce to Greenhouse Recruiting.

  • Import Limit: Enter the maximum number of employee records to be imported into Greenhouse Recruiting during each integration execution. Once this limit is reached, no additional employee records will be processed. For example, if you enter 10, only the first ten employee records will be imported into Greenhouse Recruiting.
  • Source Filter: Enter a comma-separated list of employee numbers to include only those employee records in the synchronization. If this field is left blank, all employee records will be included in the synchronization. If this parameter and the Import Limit parameter are configured, only the employee numbers specified in this parameter will be considered for synchronization.

Correlation Settings

This section enables you to customize the correlation attributes of the Dayforce and Greenhouse Recruiting integration applications.

  • Dayforce Correlation Attribute: Select the Dayforce correlation attribute to distinctly find the associated Greenhouse Recruiting job. For example, BirthDate.
  • Greenhouse Correlation Attribute: Select the Greenhouse Recruiting correlation attribute to distinctly find the associated Dayforce job. For example, name.

Sync Operations

This section allows you to define when employees are synced from Dayforce to Greenhouse Recruiting.

  • Employee Sync Operations: Select one or more operations from the dropdown menu. The integration will be executed when the selected operations are performed in Dayforce. The available options are Creation, Updation, Deactivation, and Reactivation.
Creation Settings

This section allows you to define when employee records are created in Greenhouse Recruiting. This section is displayed only if the Creation option is selected in the Employee Sync Operations parameter:

  • Create Condition: Select one or more create conditions from the dropdown menu. Based on your selection, only employee records that meet the selected condition are synced to Greenhouse Recruiting, and the corresponding employee records are created. The available options are Based on Statuses and Based on Custom Condition.
    • Employment Statuses: Select one or more statuses from the dropdown menu, only employees with the selected statuses in Dayforce are considered for syncronization into Greenhouse Recruiting. This parameter is displayed only if the Based on Statuses option is selected in the Create Condition parameter.
    • Custom Condition Expression: Enter a JavaScript expression to define which employees in Dayforce should be considered for synchronization into Greenhouse Recruiting.
