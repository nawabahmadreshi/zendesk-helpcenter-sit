Watchdog Setup Guide: ADP Workforce Now to Okta Integration

← Back to index

Overview

Aquera Watchdog is a monitoring and validation tool designed to identify data synchronization issues between integrated applications. It operates independently of the Aquera integration and analyzes application events to detect inconsistencies between source and target systems.

After an integration execution is completed, you can run the Watchdog to retrieve the event logs, analyze the activities, and report any anomalies or data discrepancies across the applications. For example, if a user is created in the source application, Watchdog checks for a corresponding user provisioning event in the target application and reports an anomaly if no matching event is found. Aquera Watchdog helps ensure guaranteed delivery of events across integrated systems by continuously monitoring and validating data synchronization between the source and target applications.

When you run the integration Watchdog, a dashboard is generated with a consolidated view of the activities across the systems, including any detected anomalies. The dashboard can be accessed through the Insights module. Refer to the Watchdog Insights section for the details captured in the dashboard.

Watchdog ensures data consistency across systems, provides visibility into sync failures, and helps proactively identify and resolve integration issues. 

This guide provides details on how to configure and run the Watchdog for the integration that synchronizes worker data from ADP Workforce Now to user accounts in Okta.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin Portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get the Aquera tenant access. 
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You must configure the following application connectors in the Aquera Admin Portal to connect to those applications for retrieving the event logs:
    • ADP Workforce Now: Ensure you add and configure the ADP Workforce Now connector in the Aquera Admin portal. Refer to the ADP Workforce Now Configuration Guide for details.
    • Okta: Ensure you add and configure the Okta connector in the Aquera Admin portal. Refer to the Okta Configuration Guide for details.

Watchdog Setup

This section describes how to configure the Watchdog that monitors the user sync from ADP Workforce Now to Okta.

Create Integration Watchdog

Follow these steps to create the integration watchdog:

  1. Log in to the Aquera Admin Portal.

  2. In the left panel, go to Integrations.

  3. Click Create Integration.

  4. Select ADP Workforce Now / Okta [Watchdog].

  5. Enter a name and description for the integration.

  6. Choose your ADP Workforce Now and Okta connector instances.

  7. Click Continue to open the Configuration page.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the ADP Workforce Now and Okta connectors that you have selected while creating the integration watchdog. This section enables you to search and select a different instance of the application connectors, if required.

Parameters

The Parameters section enables you to customize the Watchdog settings according to your use case. It has the following settings:

  • ADP Workforce Now matching attribute: Select the ADP Workforce Now worker attribute that must be used to uniquely identify a worker record and match with an Okta user record.
  • Okta matching attribute: Select the corresponding Okta user attribute that is used to match against the selected ADP Workforce Now attribute to identify the correct Okta user record.
  • Logs lookback period (days): Specify the number of past days for which the event logs must be retrieved from ADP Workforce Now and Okta. These logs are used by Watchdog to compare and analyze the events across both systems.

Schedule

The integration Watchdog can be scheduled to run automatically based on your choice of timeframe. Refer to Integration Schedule Setup Guide for further details. 

Watchdog Insights

This section details the data covered in the dashboard created by the Watchdog for the ADP Workforce Now to Okta integration. 

  1. Log in to the Aquera Admin portal.
  2. Navigate to Insights.
  3. Click on the integration watchdog name, for example, ADP Workforce Now to Okta Watchdog. The dashboards are displayed.

The dashboard provides a comprehensive view of workforce events and their synchronization across ADP Workforce Now and Okta. It includes the following insights:

  • Displays the count of Joiner, Mover, and Leaver events in ADP Workforce Now, along with corresponding user creations, updates, and deactivations in Okta. This helps identify any discrepancies in user data between the two systems.
  • A checklist that verifies whether a corresponding Okta user record has been created for each employee added in ADP Workforce Now.
  • A timeline view illustrating a worker’s journey, from hiring in ADP Workforce Now to account creation in Okta, and from termination to account deactivation.
  • A dashboard showing the upcoming hires and terminations, categorised by departments
  • A trend graph that provides visibility into synchronization patterns or gaps between HR and directory events.
  • A pie chart showing the percentage distribution of workers across different departments.
  • A horizontal bar chart showing the distribution of workers across various managers.
  • Insights into workforce distribution by geographic location, and by department and country.
  • A line chart illustrating hiring trends compared to employee terminations over time.
  • Detailed logs of events in the ADP Workforce Now and Okta directory.
