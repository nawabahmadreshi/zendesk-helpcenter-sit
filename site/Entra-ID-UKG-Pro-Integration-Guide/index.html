Entra ID / UKG Pro Integration Guide

← Back to index

Overview

This guide applies to the latest public version of the integration (version 2.0.2).

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration connects UKG Pro with Microsoft Entra ID to automate end-to-end user lifecycle management.

This guide explains how to set up the UKG Pro / Entra ID integration.

Key Functions

The UKG Pro / Entra ID integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes employee records from UKG Pro to Entra ID.
  • Supports both full synchronization and incremental synchronization to process only changed records.
  • Creates new users in Entra ID when they are added in UKG Pro.
  • Updates existing Entra ID users when their information changes in UKG Pro.
  • Deactivates Entra ID users when they are no longer active in UKG Pro.
  • Manages user group assignments in Entra ID.
  • Supports integration hooks for user creation, updates, and deprovisioning events.
  • Sends email notifications for user creation and deactivation events when enabled.
  • Generates and sends reports via email.
  • Supports a report-only mode for validation without changing Entra ID.
  • Supports monitored provisioning.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • UKG Pro: Ensure that you configure the UKG Pro connector in the Aquera Admin portal. Refer to the UKG Pro Configuration Guide for details.
    • Microsoft Entra Native: Ensure that you add and configure the Entra ID connector in the Aquera Admin portal. Refer to the Microsoft Entra Native Configuration Guide for details.

Integration Setup

This section details the configuration and mapping process for integrating UKG Pro with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped from UKG Pro to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for UKG Pro / Entra ID in the search box next to the Explore Templates section, and select it. The Create New Integration pop-up window appears.
  5. Enter a name and a description for the integration.
  6. Select your UKG Pro and Entra ID connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The Configuration tab includes a list of configurations required to set up the integration.

Under the Source Match Expression section, you must specify the source attribute and the target attribute that must be used to match employee records from UKG Pro with the user records in Entra ID.

Note: Click Switch to Expression Mode to configure a JavaScript expression that must be used to identify the matching records.

Refer to the Integration Configurations Reference Guide for detailed information on each configuration field.

Link Analysis

The Link Analysis feature identifies duplicate or conflicting resource records across the source and target applications by comparing unique user attributes. It uses smart algorithms to classify resources as linked and unlinked, and supports cleanup between applications.

  1. In the Source Match Expression section, select the UKG Pro employee attribute and the Entra ID user attribute that are used to match employee records between UKG Pro and Entra ID.
  2. Click Run Link Analysis. 
  3. Once the analysis is completed, click View Report.
    • The Linked tab displays user records for which the values of the defined match attributes (in Source Match Expression) exactly match across the UKG Pro and Entra ID applications.
    • The Unlinked tab displays users for whom no matching UKG Pro record was found using the match expression. By default, the tab displays a system-generated Default Name Filter that helps identify potential single-match records across the applications after a successful analysis run. 
  4. Review the list of source user records corresponding to the target records, and if the records match, click Update Correlation Attribute in the Unlinked tab and then click Update to transfer the attribute value from the source application to the target application. The update process runs in the background.
    Note: The integration cannot update certain attributes of an Entra ID user that is synchronized from on-premises Active Directory via Entra Connect. To filter only the users that are created directly in Entra ID, apply the filter On Premises Sync Enabled = false. 
  5. Once the match key is updated in the target application, the corresponding user record is automatically moved to the Linked tab.

For details on the Link Analysis Report and the actions that can be performed on the report, refer to the Integration Analysis Guide.

Mapping and Assignments

The Mapping and Assignments tab provides configuration options for defining user naming attributes and mapping attributes between UKG Pro and Entra ID.

User Naming Attributes

In Entra ID, the user naming attributes are User Principal Name (UPN) and Work Mail. These attributes are used to identify user objects in Entra ID.

  • UPN: The User Principal Name (UPN) attribute is the user logon name (ULN) that is used to log in to a domain. By convention, UPN should map to the user's email address.
    A UPN consists of a prefix (the user account name) and a suffix (a DNS domain name), separated by the @ symbol. The UPN format is username@DomainName. For example, barbarajensen@aquera.com.
  • Mail: The Mail attribute is the user's email address, often set to match the UPN. The mail should be unique within the same domain. The mail attribute typically follows standard email formatting. For example, barbarajensen@example.com.

This integration enables you to generate unique names for these attributes while creating a user in Entra ID. The following configurations are available on this page:

  • Suffix
  • User Naming Configuration

Suffix

This section enables you to configure the domain name for UPN and work mail as follows:

  • UPN/ULN suffix: Enter the domain name to be used as the suffix in the UPN/ULN. You can configure a static domain name (for example, aquera.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional UPN/ULN Suffix checkbox and set conditional logic in the Conditional UPN/ULN suffix field to assign domain names to users accordingly. 

    The Conditional UPN/ ULN suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate domain names during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

  • Work Mail suffix: Enter the domain name to be used as the suffix for the work email address. You can configure a static domain name (for example, aquera1.com) as the UPN/ULN suffix or define conditions to assign a specific suffix.

    To define conditions, select the Conditional Work Mail Suffix checkbox and set the conditional logic in the Conditional Work Mail suffix field.


    The Conditional Work Mail suffix field is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate email domain names during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    If the work email suffix has to match the UPN suffix, select the Work Mail Suffix same as UPN/ULN Suffix checkbox.

User Naming Configuration

This section allows you to define the formats to be used to generate values for the UPN and Mail attributes of newly created users in Entra ID.

The following configuration modes are available:

  • Basic User Naming Configuration: This configuration is enabled by default. In this mode, select a naming format from the ULN/ Mail Format dropdown list that must be used for generating the UPN and Work Mail values for a new user in Entra ID. The selected format will be used to generate unique names for both attributes.

    In this mode, the UPN and Work Mail attributes are mutually dependent and are evaluated together. This means the integration will verify whether the values generated using the selected format are available for both attributes before assigning them to the new user. If a value is already in use for an attribute (that means, it is already assigned to a user in Entra ID), then the integration will append a number (starting from 1) to the attribute values and check for the availability. This process continues for up to 20 attempts or until unique values are generated for both attributes. Once unique values are found for the attributes, those values will be assigned to the new user.

  • Advanced User Naming Configuration: Select the Advanced User Naming Configuration checkbox to switch to the advanced setup mode. In this mode, you can set different formats for each of the user naming attributes.

    When you switch to the advanced mode, you can select whether the work mail should be mutually dependent with the UPN/ULN in the Mutually Dependant Attributes with UPN/ ULN field.

    If you set UPN and Work Mail as mutually dependent attributes, the integration will check for a unique value in the first format for both attributes. Both attributes are evaluated together. If a unique value is unavailable (already assigned to another user), it will proceed to the next format in sequence for both attributes. The process continues until it finds available values for both attributes to assign to a user. If the integration cannot find unique values, the integration will return an error.
    If UPN and Work Mail are not set as mutually dependent attributes, the integration evaluates each attribute separately. If the integration finds a unique value for even one of the attributes as defined by a format, that value will be assigned to the user.

    Define the user naming formats in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields.
    These fields are AI Prompt fields that use Aquera Intelligence (AI Assistant) to generate UPN and work mail values during integration executions. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

    When you add a format in the User Naming Format for UPN/ ULN and User Naming Format for Mail fields, you must also specify when the specified format should be applied. You can choose to apply it On Create, On Update, or On Create & Update.

Attribute Map

Navigate to the Attribute Map section to perform attribute mapping. The Attribute Map section displays the default mapping from UKG Pro Employee attributes to Entra ID user attributes.

Note: For details on the UKG Pro attributes, refer to the Appendix.

Perform the following steps to add a new mapping from an UKG Pro attribute to an Entra ID attribute:

  1. Click +Add Row at the end of the attribute mapping list.
  2. Click on the empty field on the source attribute side to include an UKG Pro attribute.
  3. Select an attribute from the dropdown list to add it as the source attribute.
    Alternatively, you can write a JavaScript expression in the source field. Click the Enter Expression option to write the expression. The expression is evaluated at runtime and the resulting value is assigned to the corresponding target attribute.
    Note: Click the icon to expand the Expression editor area to write longer expressions.
  4. In the Attribute field on the target side, select the Entra ID attribute name from the drop-down list.
  5. Select the data type of the Entra ID attribute from the dropdown list.
  6. Select one of the following mapping options:
    • Map on Create and Update: The Entra ID attribute will be updated during the create and update operations.
    • Map on Create: The Entra ID attribute will be updated only during the create operation.
    • Map on Update: The Entra ID attribute will be updated only during the update operation.
    • Don't Map: The Entra ID attribute will never be updated and the mapping will be ignored during the integration runs.
  7. Once done, click the check mark to add the new mapping to the attribute map.

Attribute Map Analysis

The Attribute Map Analysis evaluates and compares user attributes across the source and target applications to identify suitable attribute mappings for the integration. It highlights Matched and Unmatched users to guide accurate data correlation.

  1. Click Run Analysis. The following pop-up window is displayed:
  2. Select the source attribute and the target attribute that must be used for correlation. The values in these attributes must be unique for a user and must be the same across the source and target applications for identifying the matching users.
  3. Click Run analysis to run the Attribute Map analysis. The attribute map analysis will analyze all existing attributes between both applications for the eligible users and will generate a report.
  4. After the analysis has been completed, click View Report to view the Attribute Mapper Report.
    You can change the map options and the source attributes, if required.
  5. Click Export Mapped at the top right corner of the page to export and use this Attribute Map in the integration. Only the attributes that are mapped in the report are exported to the attribute map.

For details on the Attribute Map Report and the actions that can be performed, refer to the Integration Analysis Guide.

Preview Employee Data

You can preview the attribute values of a UKG Pro employee that will be created or updated in Entra ID based on your mappings.

To preview the attribute values of a UKG Pro employee, perform the following steps:

  1. Click the Enter objects for preview field in the top-right corner of the Attribute Map page. A pop-up window is displayed.
  2. Click the Searchable Attribute dropdown menu and select id.
  3. Enter the employee number (ID of the employee) in the Enter value field, and then click Preview.

    The employee data will be displayed corresponding to each mapping provided on the attribute mapper.

    Notes:
    • The toggle button next to the preview setup field will be enabled by default. You can disable the toggle option to hide the preview.
    • When the attributes or expressions provided in the source attributes column are invalid or do not return a value, the preview will not appear.

Group Management

The Group Management tab provides options to configure how group memberships must be handled through this integration. You can determine the group assignment, retention, and removal of users based on the update or deactivation of users in UKG Pro.

Note: You can set rules for assigning users to the Entra ID groups through either the Group Rules section of the Group Management tab or the Attribute Map section of the Mapping and Assignments tab. When you use the Attribute Map functionality to set the group rules, you have to write JavaScript conditional expressions to assign the groups. 

Select the Enable Group Management checkbox to configure options for managing user groups. If this checkbox is not selected, the integration does not manage group assignments to users in Entra ID.

When you select the Enable Group Management checkbox, the following sections are displayed for further configuration:

  • Group Rules
  • Group Retention

Set Up Group Rules

In the Group Rules section, you can create conditions that control how users are added to groups. Each rule defines the conditions for assigning users to a set of groups.

Perform the following procedure to create group assignment rules:

  1. Define the conditions for assignment to groups using one of the following methods:
    • Using Condition Builder
    • Using Expressions
  2. If required, click Add Row to add another condition within the same rule. When multiple rows are added within a rule, an OR condition is applied to the rows. That means, for each user, the integration evaluates the conditions in each row sequentially. If the conditions defined in a row are satisfied for a user, the user is assigned to that group and the remaining rows in the rule are not evaluated for that user.

     
  3. If required, click the + icon at the top-right corner of the rule setting section to add another rule with a set of rows/conditions for group assignments. When multiple rules are set, the integration evaluates every rule for each user.

    Note: You can add a maximum of 10 rules. 

Using Condition Builder

The Condition Builder is the default view for configuring rules. A blank row is displayed by default for configuring the first set of conditions.

  1. Click on Choose Attribute to select an UKG Pro user attribute that you want to use in the condition.


     
  2. After selecting the attribute, specify the value(s) of the attribute as comma-separated values. Any UKG Pro user that matches the specified attribute values (OR operation) will be assigned to the group when the integration runs.

     
  3. If required, add conditions with more attributes and values by clicking the + icon. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. Only users who match all the specified conditions will be assigned to the group.
     
  4. In the Assign To field, select the groups where users will be placed. 

Using Expressions

Follow this procedure to set the rules using JavaScript expressions. A blank row is displayed by default for configuring the first set of conditions.

  1. Click Switch to Expression. The following options are displayed:
    • For Conditions Only: Select this option to use JavaScript expressions for setting the conditions. You must select the corresponding group from the Assign To field.
    • For Conditions and Assignments: Select this option to use JavaScript expressions for setting the conditions and the group assignments.

You can expand the Expression editor area by clicking the icon and then write the JavaScript expressions.

Note 1: Click Switch to Condition Builder if you want to switch out of the Expression mode.

Note 2: When you switch to the Expression mode or to the Basic Condition mode, all the configured conditions of the rule will be cleared.

Note 3: Expressions can use all the employee and user attributes in the UKG Pro and Entra ID applications. The configured expressions must always be evaluated to true or false.
SOURCE must be used in an expression to represent an attribute in the source application and DEST to represent an attribute in the destination application. For example, SOURCE.EmployeeNumber, SOURCE.HomeOrganization.ShortName, DEST.EmployeeNumber, and so on.  

Set Up Group Retention Policy

The Group Retention section allows you to define how a user's group memberships must be handled when the UKG Pro user is updated or deactivated.

On Deactivate/Deletion

This drop-down list is used to specify what should happen to a user's group memberships when the UKG Pro user is deactivated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the deactivated user satisfies some of the group assignment rules and they are not already part of those groups, they will be added to those groups; if they are already part of those groups, they will be retained in those groups.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained, even if the user does not satisfy the rules configured for the group assignments in the Group Rules section.
  • Retain in selected groups: This option has to be used when you want to retain the deactivated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained. Deactivated users will be removed from all other groups.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of users from the deactivated list of users. The group membership will be retained only for the users matching the filter condition. For the remaining set of deactivated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

On Update

This drop-down list is used to specify what should happen to a user's group memberships when the user is updated. It includes the following options:

  • Remove from all assigned groups: All the group memberships currently assigned to the user will be removed. However, if the user satisfies the conditions defined in the Group Rules section, they may be added to new groups based on the configured group rules.
    Note: If the Update Deactivated Users configuration is enabled, a deleted or deactivated user will be added to a new group based on the group rules. However, they will not be added back to an existing group from which they have been removed during this integration run.
  • Retain in all assigned groups: All the group memberships currently assigned to the user will be retained. Even if the user does not satisfy a rule configured for the group assignments in the Group Rules section, they will be retained in those groups.
  • Retain in selected groups: This option has to be used when you want to retain the updated users only in specific groups. When you choose this option, the following additional fields are displayed:
    • Groups: In this field, you need to select the specific groups in which a user's membership has to be retained. For example, if a user is part of G1, G2, and G3 groups and if you have specified G3 and G4 in this field, the user's membership in G3 will be retained on user update.
    • Condition: This is an optional field to define a filter condition to apply the retention rule only to a specific set of user records that were updated. The group membership will be retained only for the users matching the filter condition. For the remaining set of updated users that do not match the filter condition, no action will be taken on their group memberships. Here is a sample filter condition:
      '(SOURCE.WorkAssigmentsPositionDepartmentXRefCode).includes("Admin") && (SOURCE.BusinessTitle).includes("Manager") && (SOURCE.BusinessTitle).includes("HR")'

Group Assignments Analysis

The Group Assignments Analysis feature enables you to resolve group assignment conflicts for users in an application. The analysis identifies specific attributes a user must have to belong to a group and generates a report of assignment rules based on those attributes. The report can then be exported as a group assignment rule and applied to the integration.

  1. Click Run Analysis. The following pop-up window is displayed:
  2. Enter the UKG Pro attribute and the Entra ID attribute that must be used for correlation. These attribute values are used for identifying the matching users.
  3. Click Run analysis to run the Group Assignments analysis. It will analyze all the users in the groups and will generate a report that contains the assignment rules based on the pattern of user data in each group.
  4. After the analysis has been completed, click View Report to view the Group Assignments Analysis Report.

    You must enable the mapping for the group assignment rules that you want to include in the integration.
  5. Click Export Mapped at the top right corner of the page to export and use these Group Assignment rules in the integration. Only the mapped rules are exported to the Group Management page.

For details on the Group Assignment Analysis Report and the actions that can be performed, refer to the Integration Analysis Guide.

Based on your requirement, you can add any number of tables in the integration.

Known Limitations and Issues

  • You can save the integration without configuring the mandatory fields; however, the integration runs will fail.
  • To view all the UKG Pro attributes in the integration, it is required to download a report of all employees from the Data tab of the UKG Pro connector configuration page.
  • When the integration creates guest users, the UPN value is automatically generated by the Entra ID in the format, emailaddress#EXT#@tenant.onmicrosoft.com. The UPN naming format configured in the integration will be ignored for guest users.
  • The Incremental Tuning configuration is designed to function only when the Match expression only option is selected. Hence, the Username option has been removed from this configuration parameter.
  • When you set up group assignment rules, the attribute values must be entered as chips. However, after you exit the Edit mode, these attribute values appear as comma-separated entries.
  • The Import option is not available on the Group Assignment setup page. Hence, group assignment rules cannot be created via a CSV import.
  • The Export option is not available on the Group Assignment setup page. Hence, group assignment rules cannot be exported as a CSV file.
  • You can save a group management rule even if some rows in the rule have groups with empty conditions. This issue will be fixed in a future release.
  • During the Group Assignments Analysis, the source and target correlation attributes must be manually entered. No drop-down list is available to select an attribute from a list of attributes.
  • In Attribute Map, when you search for a source attribute using the attribute name, duplicate attribute names may be returned in the search result. This issue will be fixed in a future release.
  • When the integration attempts to update certain attributes of an Entra ID user that is synchronized from on-premises Active Directory via Entra Connect, it may fail with the following error: “Unable to update the specified properties for on-premises mastered Directory Sync objects or objects currently undergoing migration.”
    This occurs because some user attributes are mastered in on-premises Active Directory and cannot be modified directly in Entra ID by the integration. To identify the users that are synced from Active Directory, generate the Link Analysis Report, and filter for users where On Premises Sync Enabled = true.
