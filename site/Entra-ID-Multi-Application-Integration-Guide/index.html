Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

By default, the integration uses the User Principal Name (UPN) from Entra ID and matches it with the Username in the target application to identify records, unless custom correlation attributes are configured.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • User Account in Entra ID: You must have one user account in Azure AD with permission to configure provisioning (For example, Application Administrator, Cloud Application Administrator, Application Owner, or Global Administrator).
  • Enterprise Application in Entra ID: You must have an enterprise application in Entra ID for the integration. Refer to the Creating an Enterprise Application in Entra ID section for further details.
  • Microsoft 365 Application Connector in the Aquera Admin Portal: You need to configure the Microsoft 365 application connector in the Aquera Admin Portal. Refer to the Microsoft 365 Configuration Guide for configuration details.
  • Target Application Connector in the Aquera Admin Portal: You need to configure a connector in the Aquera Admin Portal for your target application. For example, if Box is your target application, then you must have a Box application connector in the Aquera Admin Portal. Refer to the configuration guide for your application connector for configuration details.
    Note: If your use case requires multiple target applications, then you must have a connector for each target application.

Creating an Enterprise Application in Entra ID

You must have an enterprise application in Entra ID for the integration. This enterprise application will serve as a source for user records and will be used to provision users to the target applications.

Perform the following steps to create an enterprise application:

  1. Log in to your Microsoft Azure portal.
  2. Search and select Enterprise Applications.
  3. Click New application.
  4. Search for the application for which you want to create an enterprise application, and select it from the search results. For example:
  5. Enter a name for the application, and click Create.

    The enterprise application is created and will be available on the Enterprise applications page.

Integration Setup

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search and select Entra/SCIM Apps Provisioning. - This name might change. Confirm
  5. Enter a name and a description for the integration.
  6. Click Continue. The Overview page is displayed.

Overview

On the Overview page, you can view details about the integration, including its configuration and related settings.

Applications

Navigate to the Applications tab to configure the source and target settings for the integration. This page allows you to select the applications involved in the integration, choose the source and target systems, and set up their connection.

The following sections are displayed on the page:

  • Source Configuration
  • Provisioning

Source Configuration

This section enables you to identify the Microsoft Entra ID application connector used for the integration. The following field is displayed in this section:

  • Entra ID Application: Select the Entra ID connector that you have created in the Aquera Admin Portal. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.

Provisioning

This section enables you to configure application provisioning. In this section, you can define how records from Microsoft Entra ID are provisioned to the target application and how attributes are mapped between the systems.

If the List View toggle button is disabled, the following parameters are displayed:

  • Entra ID Enterprise Application Name: Click the refresh icon next to the field to retrieve all enterprise applications available in your Azure portal, then select the enterprise application that you have created for your target application from the list.

    Notes:
    • This field identifies the enterprise application created in the Azure portal to drive provisioning for this integration. This is the Entra ID application to which users or groups are assigned.
    • The enterprise applications available for selection depend on the Entra ID application connector selected in the Source Application section. - Confirm
  • Select Target Application / Target Application: Select the target application connector where records will be provisioned. For example, if your use case is to provision user records from Entra ID to the Box application, select the Box connector that you have created in the Aquera Admin portal for this integration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors.
  • Custom Lookup for User Correlation / Set Custom Correlation Attributes: By default, user records are matched using the User Principal Name (UPN) from Entra ID and the Username in the target application. Select this checkbox to use a different source or target attribute for identifying the correlation attributes. When enabled, the following fields are displayed for further configuration:
    • Entra ID Attribute Used For Lookup / Entra ID Correlation Attribute: Select the correlation attribute in Entra ID that uniquely identifies a user record in your target application. For example, select employeeId if you want to use it to match user records in the target application.
    • Target Application Attribute Used For Lookup / Target Application Correlation Attribute: Select the correlation attribute available in your target application that uniquely identifies a user record in Entra ID. For example, select userName if you want to use it to match user records in Entra ID.
  • Generate Password: Select this checkbox if your use case requires generating a password for new users in the target application. When enabled, the following fields are displayed for further configuration:
    • Password Generation Method: Select the method to generate passwords. This defines how passwords should be generated. The available options are as follows:
      • Generate Random Password: Select this if you want passwords to be generated automatically by the system.
      • Generate Password Using Policy: Select this if you want passwords to be generated based on a defined password policy. When selected, the following fields are displayed for further configuration:
        • Select Password Policy / Password Policy: Select the password policy to be used for this integration. All password policies created in the Policy tab will be available for selection.
  • Attribute Mapper: The Attribute Mapper section allows you to control how each attributes are synchronized between Entra ID and the target application.
    Source attributes are listed on the left side and the target attributes are listed on the right side of the page.
    You can modify only the source attributes as needed, to align with your specific use case.
    JavaScript expressions can be used to transform source attribute values and construct the values that are updated in the target attributes. You can enter the expression directly in the source attribute field or click the Expand icon to write a longer expression.
    Attributes marked with a red asterisk (*) are required for successful provisioning.
    The Search Attributes field allows you to filter the list of available attributes for mapping.
    After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

To add another target application in this integration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the Application Provisioning Configuration section in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Password Policy

Navigate to the Password Policy section to define how passwords are generated in the target applications. This section allows you to configure password generation rules, complexity requirements, and length constraints.

This section has the following parameters for configuration:

  • Policy Name: Enter a unique name to identify the password policy.
  • Password Policy Generation Method / Password Generation Method: Select the method to generate passwords. This defines how passwords should be generated for SCIM applications. The available options are as follows:
    • Auto Generated Password: Select this if you want passwords to be generated automatically by the system based on the defined complexity and length rules.
    • Custom Password Policy: Select this if you want passwords to be generated by the administrator.
  • Password Complexity Requirements: Select the password complexity requirements. This defines the character types that must be included in the password. The available options are as follows:
    • Require Uppercase Letters: Select this if you want passwords to include at least one uppercase character (A–Z).
    • Require Lowercase Letters: Select this if you want passwords to include at least one lowercase character (a–z).
    • Require Numbers: Select this if you want passwords to include at least one numeric character (0–9).
    • Require Special Characters: Select this if you want passwords to include at least one special character.
  • Minimum Password Length: Specify the minimum number of characters required for a password.
  • Maximum Password Length: Specify the maximum number of characters allowed in a password.
  • Allowed Special Characters: Select the special characters that are allowed to be included in passwords. Only the selected characters can be used when special characters are required.

To add another password policy, click the + icon.

Log Settings

This section enables you to customize the log settings.

Note: This section appears only if you have selected the Enable Logging setting in the General Settings section.

It has the following fields:

  • Select Log Type / Log Type: Select the log type. This will decide the level of detail to be included in logs.
    The following are the available options:
    • Minimum: Select this option to include only operation details. No GUIDs or field-level information will be included.
    • Metadata: Select this option to include operation details and field names. GUIDs are excluded.
    • Non-PII Data: Select this option to include operation details, field names, and selected non-PII values. GUIDs are excluded.
    • GUID with Non-PII Data: Select this option to include GUIDs, operation details, field names, and selected non-PII values.
    • Employee Number with Non-PII Data: Select this option to include employee numbers, operation details, field names, and selected non-PII values.
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.

      The following fields are displayed for further configuration if Metadata, Non-PII Data, GUID with Non-PII Data, or Employee Number with Non-PII Data are selected:
      • Select Log Level / Log Level: Select the log level. The following are the available options:
        • All: Select this to include all attribute details in logs.
        • Include List: Select this if you want to choose attributes to be included in logs. When selected, the following field is displayed:
          • Custom Inclusion List: Select the attributes to be included. For example, PersonNumber.
        • Exclude List: Select this if you want to choose attributes to be excluded from the logs. When selected, the following field is displayed:
          • Custom Exclusion list: Select the attributes to be excluded. For example, FirstName.
