Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.


This integration enables you to write back user principal name, business phone numbers and business email addresses of users created in Entra ID to Dayforce. 

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin Portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/.
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Dayforce: Ensure that you configure the Dayforce connector in the Aquera Admin portal. Refer to the Dayforce Configuration Guide for details.
    • Microsoft Entra Native: Ensure that you add and configure the Entra ID connector in the Aquera Admin portal. Refer to the Microsoft Entra Native Configuration Guide for details.

Integration Setup

This section details the configuration and mapping process for integrating Dayforce with Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped from Dayforce to Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Entra ID Writeback in the search box next to the Explore Templates section, and select it. The Create New Integration pop-up window appears.
  5. Enter a name and a description for the integration.
  6. Select your Dayforce and Entra ID connector instances.
  7. Click Continue.

Directory Configuration

Data Synchronization Settings

  • Enable recurring full sync: Select this checkbox to set the frequency of full synchronization from Entra ID to Dayforce. During a full synchronization, business email addresses and business phone numbers of all eligible user records in Entra ID are synchronized to the corresponding employee records in Dayforce. When the checkbox is enabled, the following parameters will be displayed to set up the full sync schedule.
    If this checkbox is not selected, only incremental synchronization will be performed based on the configured schedule. In this mode, only the business email addresses and business phone numbers that were added in Entra ID since the last synchronization run will be updated in Dayforce.
    Note: The first execution of the integration always performs a full sync.
  • Full sync schedule: Select the frequency by which the integration must run in full sync mode. For example, selecting Every 6 hours from the drop-down menu will execute the integration every 6 hours in full synchronization mode.
    For example, if the integration schedule is configured as Continuous and the Full sync schedule is configured as Every 6 hours, the integration executes continuously based on the configured schedule settings. However, one execution in every 6 hours will be in full synchronization mode. All other executions between those full synchronizations are in incremental mode, if incremental sync is enabled.
    For instance, if a full synchronization execution happens at 12 AM, the next full synchronization execution happens at 6 AM. All executions between 12 AM and 6 AM are in incremental sync mode.
  • Start schedule from: Specify when the full sync schedule must begin. 
    • Start now: Select this option for the full sync to begin from the next scheduled run based on the configured schedule.
    • From specific date: Specify the date and time from which the full sync schedule should start, in the Start date and Start time fields.
  • Sync Days: Select the days of the week when the full synchronization should run. This field is displayed only if you have selected Weekly in the Full sync schedule field.
  • Disable incremental sync: Turn on this toggle button to run the integration only in full synchronization mode and skip incremental synchronization runs. When disabled, the integration runs in both full and incremental synchronization modes based on the configured schedule settings.

Entra ID application

  • Entra Application: Select the Entra ID connector that must be used by the integration to retrieve users data from Entra ID.

WriteBack Settings

Match Employees Between Entra ID & Dayforce

Configure the following for detailed Match Employees Between Entra ID and Dayforce :

  • Entra ID user match attribute: Select the EmployeeID attribute to match with the attribute EmployeeNumber in the target(Dayforce).
  • Dayforce employee match attribute: Select EmployeeNumber to match against the EmployeeID selected in the source(Entra ID).

WriteBack Settings

Configure the following WriteBack Settings:

  • Select Attributes For WriteBack: Select the attributes from the dropdown menu for writeback. By default, the writeback process updates Business Email, Phone Number, and Federated ID / SSO.
  • Enable Custom Attribute WriteBack: Select the Enable Custom Attribute WriteBack checkbox to configure custom updates through the attribute mapping. The attribute mapping configuration writes specific user attributes from Entra ID back to Dayforce during the writeback process based on what you select.

Execution

The Execution tab allows you to configure various settings required for the integration execution. The following sections are displayed in this tab:

  • Advance Settings
  • Operational Settings

Advance Settings

This section enables you to filter employees for debugging purposes.

RollOut Configurations

The following fields are displayed for roll out configurations:

  • Report Only Mode: Enable the toggle button to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Debug Mode: Enable the toggle button to display detailed logging for troubleshooting and analysis. Captures verbose logs to diagnose issues during rule execution.
  • Filter Entra ID user: Select this checkbox to enable filtering to apply writeback only to selected records instead of all data. When enabled, the following parameters are displayed:
    • Select the Filter type: Select an attribute from the dropdown menu to filter an employee. By default, the value is Employee Id.
    • Filter Value: Enter a value for the selected attribute. For example, 4400.

Operational Settings

This section enables you to customize the operational settings.

Logging Settings

  • Select Log Type: Select the Log type. This will decide the level of detail to be included in the log. The following is the available option:
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.

Report Settings

  • Reporting Type: Select the Report type. This will decide the level of detail to be included in the report. The following is the available option:
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.

Log Insights Settings

  • Select Log Insight Type: Select the Log Insight type. This will decide the level of detail to be included in the log insights. The following is the available option:
    • Employee Number with All Data: Select this option to include employee numbers, operation details, field names, and all values, including both PII and non-PII data.

Notification

Navigate to the Notifications tab to configure notification settings for the integration. The following sections are displayed on the page:

  • Channels
  • Template Library
  • Configuration

Channels

This section enables you to customize the notification channel to be used for the integration. It has the following settings:

  • Enable Email: Select this checkbox to enable integration-related email notifications.

Template Library

This section enables you to create an email body for sending email notifications.

If the List View toggle button is disabled, the following parameters are displayed:

  • Name: Enter the unique name to identify the template.
  • Type: Select a notification type from the dropdown menu. The available option is email. For example, if you select email, this template can be used in an email.
  • Template: Click the Improve Description icon () to enter a prompt that will be used as the template body when sending notifications. Based on your prompt, the Aquera AI will generate a notification body to be sent to the user.

Once the above parameters are configured, click Save Changes to create a template. Click List View to see the newly created template.

Configuration

This section allows you to define notification settings for provisioning events.

If the List View toggle button is disabled, the following parameters are displayed:

  • Template Name: Enter a name for the email notification.
  • Select Application: Select target applications for notification configuration. If you do not find your application connector in the list, click the refresh icon next to the field to reload the available target application connectors. 
  • Select Event for Notification: Select from the dropdown menu when this notification must be triggered.
  • Notification Delivery Channel: Select the channel to be used for sending notifications. By default, the value is email.
    • Email: Select this option if you want to send notifications through email. When selected, the following fields are displayed for further configuration:
      • Custom Notification Recipients: By default, notifications are sent to the user’s work email ID. If you want to send notifications to any other recipients, enable this checkbox.
        When enabled, the following fields are displayed for further configuration:
        • Send To: Select the additional recipients to send email notifications. For example, select Supervisor if you want to send email notifications to the user's supervisor. The following field is displayed when you select Custom in this field:
          • Custom Email Address: Enter the email IDs of the users to whom you want to send email notifications.
  • Subject: Enter the content that must be used as the subject for email notifications.
  • Templates: Select from the dropdown menu which template you want to use. It has two options Success and Failure.
  • Attach Report: Select this checkbox to receive an email notification with a report for each integration execution.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.
