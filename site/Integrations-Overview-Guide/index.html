Overview

Aquera's integrations automate the execution of tasks or processes in a coordinated, streamlined manner. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications. Aquera provides numerous connectors that facilitate integrations between identity management providers and popular applications. This guide provides an overview of Aquera's integrations.

Integration Terms

The following are the integration terms:

  • Source Application: The application from which the data of users must be retrieved as part of the integration.
  • Target Application: The application to which the data of the modified users is updated as part of the integration.
  • Scheduled Trigger: The integration runs automatically based on the set time frequency.
  • Event Trigger: The integration runs when specific events occur in an application. An event notification occurs whenever there is a data change in an application, such as a new user creation, an existing user modification, a user activation, a user addition to a group, a user removal from a group, and so on.

Types of Integrations

Aquera provides the following types of integrations:

User Integration

In User Integration, every user record newly created or modified in the source application is updated to the target application per the defined integration schedule. For further information, refer to the User Integration Setup Guide.

Bulk Integration

In bulk integration, all modified or new user records are updated in the target application as a batch per the defined integration schedule. The user data is sent to the target system as an array of user records. The Bulk Integration can be configured if the target application supports bulk or batch data import. For further information, refer to the Bulk Integration Setup Guide.

Data Integration

In data integration, you can synchronize data of any resource type from a source application to any resource type of a target application. With data integration, you can set up integrations for all resource types supported by the connector. For example, you can synchronize the Candidates resource type from a source application to the Employees resource type in a target application. For further information, refer to the Data Integration Setup Guide.

Integration Templates

You can set up integrations using the templates available on the Aquera Admin portal. The templates are designed for specific use cases and can be configured to suit your requirements. You can set up the integration to run either on a specific schedule or based on the data events in an application. Refer to Integration Templates to see the various templates provided by Aquera.

Workflow

You can set up integrations using Workflows to perform a sequence of tasks on the data across applications without writing any script. The data can be users, groups, tickets, and so on, based on the resource types supported by the application. The actions that can be performed are import, create, update, delete, and so on. JavaScript is generated internally based on the Workflow you have set up, and when the Workflow runs, this script is executed. You can set up the Workflow to run either on a specific schedule or based on the data events in an application. Refer to the Integration using Workflows for further information.

Event-Based Integration

Event-based integration runs in response to data changes in an application, such as new user creation, existing user modification, user activation, user addition to a group, user removal from a group, and so on. Event-based integrations require configuration in the application and are initiated by the application outside of the Aquera platform. Whenever an event occurs in the application, the event-based integration is triggered.

Integrations Listing Page

When you navigate to the Integrations module on the Aquera Admin portal, you can view the integration listing page that displays the list of all integrations created by the users in the tenant. You can search for an integration record from the search bar. The following are the types of views supported for listing the integrations:

  1. List View: The integration records are listed in the form of a list of rows.
  2. Card View: The integration records are listed in the form of cards.

The List and Card views display the following details:

  • Execution Icon: The icon to start or stop an integration execution. You can click the Execute icon to start an integration execution. You can click the stop icon to stop an integration execution that is already running.
  • TYPE: The icon indicating scheduled or event integration and the icons for the applications configured in the integration.
  • NAME: The name of the integration. Click on the integration name to navigate to the corresponding integration.
  • STATE: The trigger state of the integration:
    • For scheduled triggers, it indicates the schedule of the integration execution.
    • For event triggers, it indicates if the event trigger is enabled.
  • HISTORY: The execution status of the most recent five integrations runs in the form of color bars. The history color bar moves from right to left for every integration run and the right-most color bar represents the most recent integration run. The following are the supported color codes:
    • Green: Indicates the integration run is a success.
    • Red: Indicates the integration run has errors.
    • Grey: Indicates the integration has never run or the integration run is canceled.
    • Blue: Indicates a currently running integration.
    • Yellow: Indicates the integration run is a failure or incomplete.
  • AUTHOR: The name of the user who created the integration.
  • DATE CREATED: The creation date of the integration.
  • Upgrade Icon: To upgrade the integration to a newer version if available.
  • Copy Icon: To create a copy of the integration.
  • Delete Icon: To delete an integration. 

Sorting Integrations

The integrations on the listing page can be sorted by one of the following columns:

  • Name: Sorted by the ascending or descending order of integration names.
  • State: Sorted by event-triggered integrations, scheduled integrations, and non-scheduled integrations.
  • Date Created: Sorted by the ascending or descending order of the integration creation dates.

Execution History

You can click the history column of an integration record to view more details about the execution history.

The execution history displays the following details:

  • Applications: The applications used in the integration.
  • Execution Timeline:
    • The date and time the integration execution started.
    • The total time taken for executing the integration.
    • The status of the integration execution.
    • Indication of whether the execution was triggered manually or automatically based on the schedule. If the execution was manually triggered, the name of the user who executed the integration is also displayed.
    • Download Report (CSV): Download the integration execution report as a zipped CSV file. For user integration, this option is displayed only if the integration version is 4 or higher and the Send CSV Report configuration is enabled.
      Note: The integration needs to be executed at least once to generate a report.
       
    • View Logs: View the integration execution logs.
