ServiceNow to Active Directory Contractor Management Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the contractor lifecycle from ServiceNow to Active Directory. This integration automates contractor account management by syncing ServiceNow tickets with Active Directory, creating or updating accounts as needed, detecting upcoming expirations, generating extension requests, and extending access only when approved, ensuring seamless and accurate contractor access.

Key Functions

The ServiceNow to Active Directory Contractor Management integration provides the following functions:

  • Correlates ServiceNow tickets with Active Directory contractor identities using configured correlation attributes to ensure a new account is created when the user does not exist in Active Directory. If the contractor account already exists, the record is updated. If there are no differences between the two systems, no changes are made.
  • Automatically detects contractor accounts in Active Directory that are approaching their configured end date based on the defined expiration threshold and creates ServiceNow contractor access extension request tickets accordingly.
  • Continuously monitors ServiceNow tickets and evaluates their status to determine contractor access extension approval, extending the contractor’s end date in Active Directory only when the corresponding ticket reaches an approved or resolved status.
  • Automatically closes the ServiceNow ticket after the contractor’s end date is successfully extended, when auto-close is enabled.
  • Generates detailed logs, notifications, and execution reports for contractor expiration detection, ticket creation, access extensions, and integration runs.

Integration Setup

This section outlines how the integration must be configured to push the contractor management ticket data from ServiceNow to Active Directory. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left-side panel.
  3. Click Create Integration.
  4. Select ServiceNow to Active Directory Contractor.
  5. Enter a name and a description for the integration.
  6. Select your ServiceNow and Active Directory connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Contractor Creation

Configure the following contractor creation settings:

Match Expression
  • Source Correlation Attribute: Select the unique identifier attribute from ServiceNow used for identity correlation and record matching with Active Directory. This attribute enables deterministic identity linking between ServiceNow contractor records and Active Directory user objects during provisioning, updates, and reconciliation operations.
  • Target Correlation Attribute: Select the unique identifier attribute from Active Directory used for identity correlation and record matching with ServiceNow. This attribute enables deterministic identity linking between ServiceNow contractor records and Active Directory user objects during provisioning, updates, and reconciliation operations.
Contractor Creation Settings
  • ITSM Workflow Trigger: Select the ITSM workflow trigger mechanism for contractor identity provisioning requests from the drop-down list. The available options are Ticket, Flow Designer Trigger, and Service Catalog. Based on your selection, the settings will change. For example,
    • If you select Service Catalog, the following settings will appear:
      • Service Catalog Category: Select the ITSM service catalog category classification for contractor onboarding requests from the drop-down list. This categorization enables proper service request routing, workflow assignment, and catalog item selection for identity provisioning operations.
      • ITSM Ticket Category: Select the appropriate ITSM ticket category for contractor onboarding requests. This categorization ensures proper routing, workflow assignment, SLA management, and accurate reporting for identity provisioning operations. Categories help organize tickets by business function and ensure assignment to the appropriate support teams.
    • If you select Ticket, the following settings will appear:
      • ITSM Ticket Type: Select the appropriate ITSM ticket type for contractor identity provisioning requests when using ticket-based triggers. This selection determines the ticket schema, workflow rules, SLA policies, and access governance controls applied to onboarding operations. Common ticket types include Incident, Change Request, HR Case, Service Catalog Request, or custom ticket types configured in the ITSM system.
  •  Trigger Ticket Statuses: Select the ITSM ticket status values that will trigger contractor identity provisioning workflows. Only tickets with these statuses will be processed for account creation and access provisioning in Active Directory. This ensures that provisioning occurs only when tickets reach the appropriate workflow stage, maintaining proper approval and validation controls.
  • Auto-Close Ticket After Provisioning: Automatically resolve and close the ITSM ticket upon successful completion of contractor identity provisioning in Active Directory. When enabled, tickets are transitioned to a closed state after account creation, ensuring ticket lifecycle management aligns with the identity lifecycle. This reduces manual ticket management overhead and maintains accurate ticket status tracking.
    • Post-Provisioning Ticket Status: Select the ITSM ticket status to apply after successful contractor identity provisioning in Active Directory. This status indicates that the provisioning workflow has been completed and that the ticket can be closed or resolved. The selected status should align with your organization's ticket lifecycle management practices.
    • Ticket Resolution Code: Select the ITSM resolution code to apply when closing tickets after successful contractor provisioning. Resolution codes provide categorization for closed tickets, enabling better reporting and analysis of completed identity lifecycle operations. This helps track provisioning success rates and identify common resolution patterns.

Execution

Configure the following execution parameter settings:

  • Run in Incremental Mode: Select this option to process only the changes since the last execution, improving performance by avoiding full synchronization of contractors from ServiceNow.
  • Report-Only Mode: Select this option to enable read-only simulation mode for testing and validation. When enabled, the integration performs all operations in dry-run mode without executing provisioning or deprovisioning actions against Active Directory. This allows for safe testing of identity lifecycle workflows and validation of business rules without impacting production directory services.
  • Import Limit: Specify the maximum number of contractor records to import from ServiceNow to Active Directory in a single execution. Leave blank for no limit.
  • Update Limit: Specify the maximum number of contractor records that can be updated in Active Directory in a single execution. Leave blank for no limit.
  • Set the Termination Limit for Contractors: Enter a numeric value to limit the number of contractor records terminated in Active Directory per execution.

Attribute Map

The ServiceNow to Active Directory Contractor Management integration maps the attributes from ServiceNow and Active Directory. The attributes displayed in this section depend on the type of synchronization the integration supports:

  • End Date Update: To extend a contractor's tenure in Active Directory, the integration utilizes ServiceNow attributes as the source and maps them to the corresponding Active Directory attributes as the target.
  • Contractor Creation: To create a contractor in Active Directory, the integration utilizes ServiceNow attributes as the source and maps them to the corresponding Active Directory attributes as the target.
  • Extension Ticket: For creating a ticket to extend the contractor's tenure, the integration uses Active Directory attributes as the source and maps them to the corresponding ServiceNow attributes as the target.

Only the source attributes can be modified, as needed, to align with your specific use case.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.

The following tables are available:

  • Active Directory OU Mapping Table: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options:

    • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
    • Upload CSV: Click Upload CSV to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
      Notes:
      • The key name within a table should be unique. The key name can include a combination of letters and numbers.
      • The key name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

    Click Save to save the changes made.
