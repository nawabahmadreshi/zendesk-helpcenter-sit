Overview

Aquera's User Integration allows the synchronization of users from a source application to a target application. In User Integration, every user record that is newly created or modified in the source application is updated to the target application, as per the defined schedule.

This user guide explains how to set up User Integration in the Aquera Admin portal.

Prerequisites

Ensure that the required source and target connectors are added and configured correctly in the Aquera Admin portal, before creating the User Integration. Refer to the corresponding configuration guides for details on configuring your required connectors.

Creating User Integration

You can create User Integration using one of the following options:

  • Create your own User Integration 
  • Create User Integration using a Template

Create your own User Integration

Create a new user integration where you need to fill in every configuration field manually.

Perform the following steps to create a new user integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel which displays the list of integrations created by the user. 
    Note: The number of integration records displayed on this page is based on the display size of the user's screen.
  3. Click Create Integration.
  4. Select User Integration.
    Note: The number of integration types displayed on this page is based on the display size of the user's screen. If you do not see User Integration, scroll right to view this integration type and then select it.
  5. Enter a name and a description for the integration.
  6. Click Continue. A new page is displayed with the fields and tabs to configure the integration.
  7. Enter the details on the Configuration tab.
  8. Click Create to create the user integration.

Create User Integration using a Template

Templates are pre-configured integrations created by Aquera for specific use cases. You can use the templates as-is or modify them to match your organization's needs. You can view the templates if Aquera has enabled templates in your tenant. 

Perform the following steps to create user integration using a template:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations on the left panel which displays the list of integrations created by the user. 
    Note: The number of integration records displayed on this page is based on the display size of the user's screen.
  3. Click Create Integration.
  4. Select a pre-built user template based on your requirements.
  5. Enter a name and a description for the integration.
  6. Select a source connector in the Source Application dropdown field.
  7. Select a target connector in the Target Application dropdown field.
  8. Click Continue. A new page is displayed with the fields and tabs to configure the Integration.
  9. You can update the configurations if required.
  10. Click Create to create the user integration.

Integration Email Report

The following will be the details mentioned in the content of an integration email report:

  • For a successful integration run, the content of the email report will contain the following messages: 
    • For each user creation:
      {userName}: created
    • For each user update:
      {userName}: updated   
    • For each user added to a group:
      {userName}: added to: groupName   
    • For each user removed from a group:
      {userName}: removed from: groupName  
  • For a failure integration  run, the content of the email report will be the same as the integration logs. 
  • When Report Only mode is enabled, the content of the email will be the same as the integration logs and is based on the Report Mode selected. For information on the details that are captured in the logs for each of the report modes, refer to the guide here. 
  • When the Destination Modification Safeguard is triggered, the content of the email will contain one of the following messages for each user:
    • {userName}: safeguard triggered, not created  
    • {userName}: safeguard triggered, not updated  
  • All the integration email report will contain the integration parameters and their values. For details on the integration parameters, refer to the guide here. 

Destination Modification Safeguard Scenarios

Following are a few samples that explain the Destination Modification Safeguard feature:

1) For full import, suppose there are 100 new records to be created in the target application and no updates to the existing records in the target application, the Max Destination Modification Percentage set is 25%, and the source records matched is 0, then the destination modification safeguard calculation will be:
(target records that will be created (100) + target records that will be updated (0) ) / ( target records that will be created (100) + source records matched (0) ) ) = 100%

In this scenario, 100% > safeguard (25%), hence the destination modification safeguard will be triggered.

2) For full import, suppose there are 2 new records to be created in the target application and 4 updates to the existing records in the target application, the Max Destination Modification Percentage set is 25%, and the source records matched is 100, then the destination modification safeguard calculation will be:
(target records that will be created (2) + target records that will be updated (4) ) / ( target records that will be created (2) + source records matched  (100) ) ) = 6%

In this scenario, 6% < safeguard (25%), hence the destination modification safeguard will not be triggered.

3) For incremental import, suppose there is 1 new record to be created in the target application and 3 updates to the existing records in the target application, the Max Destination Modification Percentage set is 25%, and the total records matched in the previous runs is 3, then the destination modification safeguard calculation will be:
(target records that will be created (1) + target records that will be updated (3) ) / ( target records that will be created (1) + total records matched (3) ) ) = 100%

In this scenario, 100% > safeguard (25%), hence the destination modification safeguard will be triggered.

4) For incremental import, suppose there is 1 new record to be created in the target application and 1 update to the existing records in the target application, the Max Destination Modification Percentage set is 50%, and the total records matched in the previous runs is 4, then the destination modification safeguard calculation will be:
(target records that will be created (1) + target records that will be updated (1) ) / ( target records that will be created (1) + total records matched (4) ) ) = 40%

In this scenario, 40% < safeguard (50%), hence the destination modification safeguard will not be triggered.

5) When the Report Only checkbox is selected, there will be no creations nor updates to the records in the target application. If the Max Destination Modification Percentage set is 25%, and the source records matched is 100, then the destination modification safeguard calculation will be:
(target records that will be created (0) + target records that will be updated (0) ) / ( target records that will be created (0) + source records matched (100) ) ) = 0%

In this scenario, 0% < safeguard (25%), hence the destination modification safeguard will not be triggered. 

6) If there are no creations nor updates to the records in the target application and the source records matched is 0, then the destination modification safeguard calculation will be: 
(target records that will be created (0) + target records that will be updated (0) ) / ( target records that will be created (0) + source records matched (0) ) ) = NaN%

Since 0/0 is neither 0 nor infinity, it is NaN (Not a Number). In this scenario, the destination modification safeguard will not be triggered.

Mapping of Attributes

You can map the source SCIM attributes to the target SCIM attributes by navigating to the Attribute Map tab. This is required to be performed to synchronize the data from the source application to the target application. You can also write expressions to modify the value of the source attribute before updating the value in the target attribute.

For details, refer to the Attribute Mapping in Integrations guide.

Adding and Using Tables

Tables define a name-value pair to create custom attributes. Tables are added in integration to create custom source attributes, and POST (sync) them to the target application. Tables are used when an application attribute is empty or particular data is not sent from an application, and you want to update a value for that attribute in another application.

For details on how to add tables and use them in integrations, refer to the Using Tables in Integrations guide.

Setting Group Rules

You can set up rules to manage groups in the target application. Group rules can be created using basic conditions or by writing JavaScript expressions. One group rule must be defined for each managed group.

For details on how to set up group rules, refer to the Managing Groups in User Integrations guide.

Scheduling Integration Run

To schedule an integration, navigate to the Schedule tab of the integration. You can set a time frequency to run the integrations automatically. Refer to the Integration Schedule Setup Guide for further details. 

User Actions on Integrations

You can perform actions on an integration such as execute, upgrade, copy, modify, delete, export, import, view logs, and download reports. For further details, refer to the User Actions on Integrations guide.
