ADP Workforce Now to Jira Ticket Management via Cloud Integration Guide

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide outlines the process of synchronizing the user's lifecycle from ADP Workforce Now to Jira for the selected work type. This integration ensures that all user lifecycle events in ADP Workforce Now are accurately tracked and managed within the Jira system for seamless operational workflows.

Key Functions

The ADP Workforce Now to JIRA Ticket Management Integration provides the following functions:

  • Whenever an employee is onboarded or transferred in the ADP Workforce Now application, the integration automatically creates a ticket in the Jira application.
  • If a ticket is already available for an employee in Jira, the integration updates the details of the ticket based on the integration settings.
  • When an employee is rehired, a ticket will be created.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for details.
    • Jira Service Management Cloud: Refer to the Jira Service Management Cloud Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user-related data from ADP Workforce Now to Jira Service Management Cloud.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to Jira Ticket Management.
  5. Enter a name and a description for the integration.
  6. Select your ADP Workforce Now and Jira Service Management Cloud connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following features based on your requirements:

  • Enable Ticket Management: Select this checkbox to enable ticket management in your Jira instance for the preferred scenarios.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Dashboard Generation: Select this checkbox to enable operational search and dashboard capabilities for querying onboarding data. When enabled, the Log Insight Settings section is displayed for further configuration.

Ticket Management

The Ticket Management section enables you to customize the user integration settings according to your specific use case. It has the following settings:

Source Application (WorkforceNow) Settings

Configure the following source application settings:

  • Enable Source Filter: Select this checkbox to filter ADP Workforce Now workers.
  • Workforce Now Filter*: Select a Workforce Now filter to filter employees while retrieving from Workforce Now. The available options are Associate ID and Position ID.
    • Associate ID: Select this option to filter employees by their Associate ID.
      • Associate ID (Filter) Value: Enter the Associate ID to filter employees by this value.
    • Position ID: Select this option to filter employees by their Position ID.
      • Associate ID Value: Enter the Position ID to filter employees by this value.

Target Application (JIRA) Settings

Configure the following target application settings:

  • HR Events to Trigger Ticketing*: Select the JM operations that must be supported by the integration for creating or updating a Jira ticket. The available options are Joiner and Mover.
  • JIRA Destination: Select the destination application in which JIRA tickets should be automatically created whenever a joiner or mover operation is triggered.

Joiner Management

The Joiner Management section enables you to customize the integration settings for joiner management according to your specific use case.
Note: This section appears only if you have selected the Joiner in the Ticket Management section.

It has the following settings:

  • Jira Project Key for Onboarding Tickets*: Select a project key that should be assigned to the onboarding tickets.
  • Jira Issue Type for Onboarding Tickets*: Select an issue type that should be assigned to the onboarding tickets.
  • Onboarding Tickets for Future Hires: Enter the number of days in advance that onboarding tickets should be created for future hires. For example, if you enter 5, JIRA tickets will be generated for employees scheduled to be hired within the next five days.
  • Onboarding Tickets for Past Hires: Enter the number of days to create onboarding tickets for past hires. For example, if you enter 5, JIRA tickets will be created for employees who were hired within the past five days.
  • Worker Fetch Limit: Enter the maximum number of workers that should be created during a single integration run.
  • Use Different Project or Work Type for Joiner Modification (Project and Work Type for Joiner Modification): Select this checkbox to create an onboarding JIRA ticket with a new project key or work type for existing employees whose attribute values have been modified.
    • Onboard Project Key for Joiner Change: Select a project key that should be assigned to the onboarding tickets.
      This parameter will be displayed only when the Use Different Project or Work Type for Joiner Modification parameter is selected.
    •  Onboard Work Issue Type for Joiner Change: Select a work type that should be assigned to the onboarding tickets.
      This parameter will be displayed only when the Use Different Project or Work Type for Joiner Modification parameter is selected.

Mover Settings

Configure the following Joiner settings:

  • Jira Project Key for Mover Tickets*: Select a project key that should be assigned to the mover tickets. The available options are Joiner, Mover and My Software Team.
  • Jira Work Type for Mover Tickets*: Select an work type that should be assigned to the mover tickets. The available options are Task, Bug, Story, and Epic.
  • Employee Mover Ticket Limit: Enter the maximum number of mover tickets that can be modified in a single integration run.

Notification Settings

This section enables you to customize the notification settings.

Note: This section appears only if you have enabled the Enable Notifications setting in the Parameters section.

It has the following settings:

  • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
  • Domain: Enter the domain name whose users should receive email notifications. If you configure a domain in this field, only users belonging to that domain will receive notifications. If you leave this field blank, all email addresses configured in the Admin's Email Addresses parameter will receive the email notifications.
  • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
  • Admin Email Notification Template: Enter the content that must be used as the email body for all admin email notifications.
  • Send Integration Report via Email: Select to receive email notification with a CSV report for all integration runs. When this checkbox is selected, the following parameters are displayed:
    • Notification Level: Select the desired notification report detail level from the dropdown menu. This section is used to attach the integration report in CSV format to the email. The following are the available options:
      • Summary: Attribute details are not displayed. Only the count of CRUD operations is shown.
      • Full: All attribute details are displayed.
      • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Custom Notification Level* drop-down menu with the following options:
        • Notification Inclusion List: Select the attributes to be included in the notification report. The available options are Summary, Issue Type, Reporter, Component/s, Description, Fix Version/s, Priority, Labels, Time Tracking, Due Date, Linked Issues, Epic Link, Email Address, Employee ID, Employee ID String, Assignee, and Project.
        • Notification Exclusion List: Select the attributes to be excluded from the notification report. The available options are Summary, Issue Type, Reporter, Component/s, Description, Fix Version/s, Priority, Labels, Time Tracking, Due Date, Linked Issues, Epic Link, Email Address, Employee ID, Employee ID String, Assignee, and Project.
  • Alert Aquera Support on Integration Failures: Select this checkbox to send an alert notification for the Aquera support team on integration failures.

Log Settings

This section enables you to customize the log generation settings.

Note: This section appears only if you have selected the Enable Logging setting in the Advance Settings section.

It has the following settings:

  • Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Custom Log Level drop-down menu with the following options:
      • Inclusion Log List: Enter the attributes to be included in the log.
      • Exclusion List: Enter the attributes to be excluded from the log.
  • Consent to Log with Values: Select this checkbox to enable logging that includes the corresponding values for each event.

Report Settings

This section enables you to customize the report generation settings.

Note: This section appears only if you have enabled the Enable Report Generation setting in the Parameters section.

It has the following settings:

  • Report Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details. Selecting the Custom option will display the Custom Report Level drop-down menu with the following options:
      • Inclusion Report List: Enter the attributes to be included in the report.
      • Exclusion Report List: Enter the attributes to be excluded from the report.
  • Consent to Report with Values: Select this checkbox to generate a report along with their corresponding values.

Attribute Map

The Attribute Map tab in Entra ID to Donesafe Synchronization defines the default ADP Workforce Now attributes that are mapped to the Jira Service Management Cloud attributes.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.
