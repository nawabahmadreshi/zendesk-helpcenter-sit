Flo Recruit / Workday Integration Guide

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

In this integration, any candidates created in Flo Recruit are automatically created in Workday based on the defined schedule.

To identify candidate records across both systems from Flo Recruit to Workday, the integration uses the attribute configured in the Source Application Correlation Attribute parameter in Flo Recruit and matches it to the attribute configured in the Target Application Correlation Attribute parameter in Workday as a correlation attribute.

Key Function

Creates new candidate records in Workday when they are added in Flo Recruit.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Flo Recruit: Refer to the Flo Recruit Configuration Guide for configuration details.
    • Workday: Refer to the Workday Configuration Guide for configuration details.

Integration Setup

This section outlines how the integration must be configured to push candidate-related data from Flo Recruit / Workday.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Flo Recruit / Workday.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Configuration page is displayed to configure the integration.

Candidate Management

The following sections are displayed under the Candidate Management tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Manage Candidate

The Manage Candidate section enables you to customize the integration settings according to your specific use case. It has the following settings:

Filter Candidate

Enable the following feature based on your requirements:

  • Customize Candidate Status Processing: Turn on this toggle button to filter candidates. If this toggle button is not turned on, only candidates with the Onboarding Complete status are synchronized from Flo Recruit to Workday. When enabled, the following parameters are displayed:
    • Source Candidate Filter: Select a filter type from the dropdown menu. Based on your selection, only candidate records that satisfy the selected filter type and included for synchronization into Workday from Flo Recruit. The available options are Candidate filter by statuses and Custom filter by ID.
    • Fetch the Candidates by Application Status: Select one or more candidate statuses from the dropdown menu. Only candidates with these statuses will be filtered and included for synchronization into Workday from Flo Recruit. This parameter is displayed only if you have selected Candidate filter by statuses option in the Source Candidate Filter parameter.
    • Delete Users With Application Status: Select one or more candidate statuses from the dropdown menu. Only candidates with these statuses will be filtered and deleted in Workday when they are deleted in Flo Recruit. This parameter is displayed only if you have selected Candidate filter by statuses option in the Source Candidate Filter parameter.
    • Candidate Id Filter: Enter a candidate ID to filter a specific candidate for synchronization from Flo Recruit. This parameter is displayed only if you have selected Candidate filter by ID in the Source Candidate Filter parameter.
    • Defaults to last 60 days. Enable to set custom number of days: Turn on this toggle button to filter candidate records updated within the last X days in Flo Recruit. Only those candidates will be synchronized to Workday. By default, candidates from the past 60 days are included in synchronization. This parameter applies only during full synchronization. The integration performs a full sync during the first execution and then switches to incremental import mode. When enabled, the following parameter is displayed:
      • Retrieve Candidates Whose Job Applications Were Updated 'X' Days Prior: Enter the number of days to filter candidate records for synchronization from Flo Recruit to Workday. For example, if you enter 80, only records updated within the last 80 days in Flo Recruit will be included.

Match Expression

Configure the following correlation settings based on your requirements:

  • Enable Custom Lookup for User Correlation: Turn on this toggle button to define the correlation attributes in Flo Recruit and Workday. When enabled, the following parameters are displayed: 
    • Source Application Correlation Attribute: Select the Flo Recruit attribute to distinctly find the associated Workday candidate record. For example, select Candidate Id.
    • Target Application Correlation Attribute: Select the Workday attribute to distinctly find the associated Flo Recruit candidate record. For example, select Employee ID.

Create Settings

Configure the following parameters to define the conditions for creating Flo Recruit candidates in Workday:

  • Create Condition for Candidate: Select a condition from the dropdown menu. Only candidate records that satisfy the selected condition are created in Workday from Flo Recruit. The following are the available options:
    • Create All Hired Candidates: Select this option to create all hired candidates in Workday from Flo Recruit.
    • Define Custom Based Create Condition: Select this option to define a condition for creating the candidates in Workday from Flo Recruit. When selected, the following parameter is displayed:
      • Custom Create Condition: Enter a prompt that defines the condition for creating candidates in Workday from Flo Recruit.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.

      • Custom Exclusion list: Enter the attributes to be excluded from the log. This parameter is available only if you select Exclude List in the Select Log Level parameter.

Advance Settings

Configure the following advance settings:

  • Report Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Debug Mode: Select to generate logs outlining data changes that would occur for debugging. When enabled, the following parameters are displayed:
    • Debug Using Filter By: Select an attribute from the dropdown to filter candidates for debugging purposes. The available option is Candidate ID.
    • Provide Attribute Value: Enter the candidate ID to filter for debugging.

Notification Center

Navigate to the Notifications tab to configure notification settings for the integration.

Notifications

This section allows you to define notification settings for provisioning events.

If the List View toggle button is disabled, the following parameters are displayed:

  • Select Event for Notification: Select the provisioning event that will trigger a notification. For example, select Account Provisioned to get email notifications every time accounts are provisioned to the target application.
  • Custom Condition to Send Notification: Turn on this parameter to customize conditions for sending notifications. When enabled the following parameter is displayed:
    • Custom Condition: Enter a prompt to define the condition for sending notifications. 
  • Notification Delivery Channel: Select the channel to be used for sending notifications. By default, the value is email.
    • Email: Select this option if you want to send notifications through email. When selected, the following fields are displayed for further configuration:
  • Send To: Select the additional recipients to send email notifications. For example, select Supervisor if you want to send email notifications to the user's supervisor. The following field is displayed when you select Custom in this field:
  • Subject: Enter the content that must be used as the subject for email notifications.
  • Template: Click the Improve Description icon () to create a new template for the email body. Enter a prompt to generate the email body for sending email notifications.
  • Attach Report: Turn on this toggle button to receive an email notification with a report for each integration execution.

To add another configuration, click the + icon adjacent to the List View toggle button.

To navigate to multiple configurations, use the arrow icons (< or >).

To delete a configuration, click the delete icon.

To view the notification settings in list view, enable the List View toggle button.

In List View, only configured values are displayed. If you want to modify the configuration, click any value in the row and you will be redirected to the edit mode, where you can make changes.

Tables

Tables can be added to create custom source attributes and POST (sync) them to the target application. Tables are used when an application attribute is empty or a specific data is not sent from the Jira Service Management Cloud application, and you want to update a value for that attribute in Active Directory. Based on your requirement, you can add any number of tables in the integration.

The table setup and functionality are the same as that in the User Integration. Refer to the Adding Tables in Integration section in the Using Tables in Integrations guide for details.
