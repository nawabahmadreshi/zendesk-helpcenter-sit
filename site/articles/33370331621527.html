Overview

As organizations grow, they continuously adopt new software assets to improve efficiency, enhance collaboration, and drive overall business performance. However, it is common for employees to use multiple assets under different names across departments, making it difficult to track and manage these assets effectively.
Aquera’s Identity Survey Hub solution addresses this challenge by providing a centralized repository for efficient asset discovery and onboarding. It simplifies and accelerates integration with Identity Governance and Administration (IGA) platforms by tackling key challenges related to asset visibility, governance, and compliance. The solution empowers IT teams with centralized insight and control over digital assets, ensuring secure and efficient access management. By streamlining asset data collection, it enhances security posture, reduces operational costs, and supports scalable organizational growth.
This solution also streamlines asset inventory management, facilitates collaborative requirement gathering, and automates asset onboarding into IGA systems. The platform encourages collaboration among IAM administrators, asset owners, and other stakeholders to collect asset details, from basic information to comprehensive onboarding data tailored to organizational needs.
By leveraging Aquera as the central platform, organizations can effectively manage their asset ecosystem with improved flexibility, stronger security, and greater usability. Detailed reporting across the asset lifecycle, along with usage insights, supports better decision-making and optimizes operational efficiency.

Asset Definition

An asset refers to any application, system, platform, or resource used by the organization that must be tracked, governed, or managed. This includes, but is not limited to:

  • SaaS applications
  • On-premises enterprise applications
  • Cloud platforms and resources
  • Databases, servers, and virtual infrastructure
  • Shared accounts and non-human identities

Each asset entry in the catalog contains standardized metadata to enable classification, governance, and lifecycle management.

IGA Configuration

The IGA Configuration tab allows you to correlate data uploaded in the application with the list of users obtained from a file downloaded from an Identity Governance and Administration (IGA) system, such as SailPoint or Okta.

The IGA Configuration tab enables you to upload an IGA file and configure the schedule for uploading the file. It also provides an option to enable correlation checks, which automatically correlate the data for every file upload.

When the correlation check is enabled, a report showing the number of mismatched records is generated during each data file upload through the Generic Flatfile connector.

Enable Correlation Checks and Set Upload Schedule

Perform the following steps to enable the correlation check and set the IGA file upload schedule:

  1. Navigate to the IGA Configuration tab.
  2. Click Edit Configuration. The IGA Global Configuration pop-up window appears.
  3. In the IGA Global Configuration pop-up window, enable the Global Correlation Check toggle. 
    Note: If this setting is disabled, the correlation check will not be performed during the data file uploads.
  4. Set the following configuration under the Schedule section:
    • File Upload Frequency: Specifies how often the IGA file must be uploaded. The dropdown options are: Weekly, Monthly, Quarterly, and Yearly.
    • Next File Upload Date: The due date for the next IGA file upload. For example, if the File Upload Frequency is set to Weekly and the Next File Upload Date is 16-January-2026 (a Friday), the file upload will be due every Friday.
    • Timezone: The timezone that should be followed for the file upload schedule.
  5. Click Save Configuration to apply your settings.

An email reminder for the IGA file upload is sent to the corresponding asset owners and admin users on the due date.

File Retention Settings

The File Retention Settings tab allows you to configure how long uploaded files and their history are retained. Once the retention period for a file has passed, both the uploaded file and its upload history record will be deleted.

Perform the following steps to configure file retention settings:

  1. Navigate to the File Retention Settings tab.
  2. By default, the File History Retention toggle is enabled. The Retention Duration field is visible only when this toggle is enabled.
  3. Select the desired duration for retaining file upload history in the Retention Duration field. Available retention duration options are:
    • Last 6 Months
    • Last 1 Year
    • Last 2 Years
  4. The retention settings are saved automatically as soon as you select a duration.

Upload history records and files older than the selected duration (calculated from the current date) will be deleted weekly.

  • Deletion is scheduled to run every Sunday.
  • For example, if Last 6 Months is selected, all upload history records and associated files older than six months from the current date will be deleted weekly.

Note: If a file was not uploaded on the scheduled date, the file history will show a record with the status Missed. These records will not be deleted and will remain in history.

Lifecycle of the Asset

The Asset Lifecycle consists of several key stages that represent the journey of an IT asset from identification to full cataloging:

  1. Survey: The initial phase involves administrators sending simple questionnaires to asset owners or relevant stakeholders to collect basic information about IT assets.
  2. Staging: In this intermediate phase, the collected survey data undergoes validation and enrichment. This ensures the information is accurate and comprehensive, facilitating deeper discovery and preparation for subsequent onboarding activities.
  3. Discovery: This phase involves a collection of detailed data about IT assets through extended questionnaires. The gathered information is then validated and enhanced as necessary to support the final onboarding process.
  4. Inventory: The final stage involves transferring fully validated and enriched asset data into the official asset catalog and onboarding it to the IGA/SSO platform. This ensures accurate tracking, compliance, and effective management of IT assets throughout their lifecycle.

Survey

The initial phase involves collecting data through surveys distributed to asset owners or relevant stakeholders. These surveys aim to gather preliminary information about IT assets.

Admin Workflow: Creating and Managing Asset Surveys

As an Admin, perform the following steps to create and manage a survey:

Create Survey

  1. Log in to the Identity Survey Hub portal and navigate to the Survey tab located under Collection at the top left corner of the screen.
  2. Click Create Survey at the top right of the screen, and the Asset Request Survey screen appears.
  3. Enter the following survey details on the Asset Request Survey screen:
    • Basic Details
      • Survey Name: Enter a name for your survey.
      • Survey Request Questionnaire: Choose the appropriate questionnaire from the dropdown list. For example, select Asset List Survey. Click the Preview option located above the dropdown to view the selected questionnaire form in the Preview panel.
      • Recipients: Enter the recipients’ email addresses or select them from the list that appears when you click the field.
      • Start Date: Set the start date for the survey.
      • Due Date: Set the deadline for survey completion.
    • Email Templates
      • Survey Email Template: Choose the email template to be used for the survey from the dropdown list. Click the Preview option located above the dropdown to view the selected email template in the Preview panel.

  4. Click Create Survey to distribute the survey to the intended recipients after completing all required fields and configurations.

Staging

The Staging is the intermediate phase where IT assets are cataloged using surveys and structured questionnaires to build a reliable inventory foundation. During this stage, assets are primarily populated from survey responses, although a few can be added manually or imported directly from IGA applications. The collected information is then validated and enriched to create a comprehensive asset inventory that supports subsequent discovery and onboarding activities.

Assets are grouped into collection groups based on asset types. Administrators create and configure these groups, while Asset Owners and Collaborators supply the required information. This role-based coordination ensures a reliable foundation for the organization's IT asset catalog, supporting identity and access management and setting the stage for further discovery and onboarding activities.

Collection

A Collection groups related assets together, making it easier for administrators to manage the inventory by handling one set at a time. This approach enables them to validate information, send reminders, or escalate issues without being overwhelmed by the entire catalog. Because each Collection is clearly defined, administrators, asset owners, and collaborators know exactly which assets require attention and understand their status in the process. Once every asset in a collection has been reviewed and approved, those assets can move forward to the next phase.

Discovery

This phase involves the comprehensive collection of detailed IT asset data through extended questionnaires. The information gathered is then validated and enriched with the technical, security, operational, and access-model details needed to support the final onboarding process. This phase transforms the validated but high-level asset profile (from Staging) into a complete and authoritative system definition suitable for governance, compliance, and provisioning operations.

Assets are organized into collection groups based on asset type. Administrators create and configure these groups, while Asset Owners and Collaborators provide the necessary details. Once all information has been verified for accuracy, the corresponding assets can advance to the final inventory phase.

Questionnaire

The Questionnaire section enables administrators to create and manage questionnaires used during the Staging, Discovery, Inventory, and Onboarding phases. It centralizes all questionnaire templates for easy access and management.

This section includes the following two subsections:

  • Questionnaire Library: This section stores all questionnaire templates in one centralized location for easy access and reuse.
  • Questionnaires: This section allows you to create, organize, and manage individual questionnaires.

Create and Manage Questionnaires

Perform the following steps to create and manage a questionnaire:

  1. Log in to the Identity Survey Hub portal and navigate to the Questionnaire tab located in the top menu.
  2. Click the Questionnaires tab under the Manage Questionnaire section.
  3. Click Create Questionnaire, and the Create New Questionnaire screen appears.
  4. Enter the following details:
    • Questionnaire Name: Enter a name for the questionnaire.
    • Phase Type: Select the phase type from the dropdown list. Available options include Staging, Discovery, Inventory, and Onboarding.
    • Asset Type: Select the asset type from the dropdown list. Available options include Application, Server, Virtual Group, etc.
    • Description: Provide a description for the questionnaire.
  5. Click Continue. The Add New Categories screen appears.
  6. Select the checkboxes for the categories you want to include from the Question Library.
  7. Click Add to include the selected categories in the questionnaire.
  8. Enable the Mandatory toggle if you want to make the selected categories and their questions mandatory.
  9. Click Add Category to include additional categories. The Add New Categories screen appears again.
  10. Select more categories from the Question Library as needed and click Add.
  11. To add a custom category, click Add Custom Category. The Add New Category screen appears.
  12. Enter a name in the Category Name field and click Save.
  13. Use the Edit option to modify existing questions, add custom questions to existing categories, or add questions to newly created categories. Click Edit to open the questionnaire's edit page. On the edit page, you can:
    • Edit questions and change their question types or categories.
    • Delete a category using the Delete icon at the category level.
    • Delete individual questions using the Delete option at the question level.
    • Click Add More to add additional answer options based on the selected question type.
    • Configure question logic, where selecting specific answers reveals additional related questions.
    • Enable or disable the Mandatory toggle at the question level.
    • Use the Add New Question to add custom questions to existing or newly created categories.
  14. Click Add New Question. The Add New Question screen appears.
  15. On the Add New Question screen, enter the following details:
    • Question: Enter the question text.
    • Question Type: Select the question type. Available options include:
      • Single Text Box: Allows a short text response.
      • Textarea: Allows a longer, multi-line response.
      • Radio: Allows selection of only one option from a list.
      • Select: Displays a dropdown list with a single selectable option.
      • Multi Select: Allows selection of multiple options.
      • File Upload: Allows users to upload a file as a response.
    • Question Category: Select the category for the question. 
  16. Click Save. The question is added to the selected category.
  17. Click Save & Apply in the top-right corner of the screen to finalize the questionnaire after reviewing all required information. The questionnaire appears in the Questionnaires tab under the Manage Questionnaire section.
  18. Use the Search Questionnaire search bar to locate a specific questionnaire.
  19. Click View Details for any questionnaire to view its categories and questions. Click Edit to modify the questionnaire as needed. The following are the two options available in the questionnaire edit screen:
    • Save & Apply: Use this option to save your changes directly to the same questionnaire.
      Note: The Save & Apply option is not available for the global questionnaires.
    • Save as Copy: Use this option if you want to apply changes to a new copy of the questionnaire, leaving the original unchanged.
  20. Click the vertical ellipsis (⋮) next to a questionnaire to display the Edit, Delete, and Create Copy options.
    • Edit: Click this option to edit the questionnaire as per your requirement.
      Note: The Edit and Delete options are not available for global questionnaires. They are non‑editable by design, but you can use them to create a new questionnaire.
    • Delete: Click this option to delete the corresponding questionnaire.
    • Create Copy: Click this option to create a duplicate of the questionnaire with a different name. Modify and use the copied questionnaire without affecting the original.

Managing Global Questionnaire

Perform the following steps to manage a global questionnaire:

  1. Click View Details for a global questionnaire to view its categories and questions.
  2. Click Edit to modify the questionnaire as per your requirement.
  3. Click Save as Copy to apply changes to a new copy of the questionnaire.
  4. Alternatively, click the vertical ellipsis (⋮) next to the questionnaire name, select Create Copy to create a duplicate of the questionnaire with a different name. Modify and use the copied questionnaire without affecting the original.

Task

Task Management provides a centralized workspace for tracking all Identity Survey Hub activities assigned to users. Tasks are generated during the Survey, Staging, Discovery, and Inventory Maintenance phases, and ensure that asset data is captured, validated, and reviewed in a timely manner.

Accessing the Tasks Tab

Log in to the Identity Survey Hub portal and navigate to the Task tab located in the main screen. The page displays the summary of task metrics at the top, search and filtering options, along with the task creation option, and a list of created tasks in a table format.

Task Dashboard Overview

The task dashboard provides a quick snapshot of all tasks in the system. At the top of the page, key metrics give real-time visibility into task progress, priorities, and due dates.

Task Status Summary

  • Total Tasks: Total number of tasks in the system.
  • Open: Number of tasks not yet started.
  • In Progress: Number of tasks currently being worked on.
  • Done: Number of tasks completed.
  • Closed: Number of tasks that are formally closed.

Priority Summary

  • Critical Priority: Number of tasks with the highest urgency.
  • High Priority: Number of high-priority tasks.
  • Medium Priority: Number of medium-priority tasks.
  • Low Priority: Number of low-priority tasks.

Due Date Alert

  • Past Due Date: Number of tasks that have crossed their due date.

Search and Filter

Search Task

Use the Search Task bar to quickly locate specific tasks by entering relevant keywords. The search works across task fields such as the task ID and Summary, and the results update dynamically as you type, making it easy to find the exact task you are looking for.

Filter Tasks

Click Filter to narrow down tasks using the following criteria:

  • Assignee
  • Task Type
  • Task Status
  • Reporter
  • Priority
  • Created By
  • Approval Status

You can apply one or multiple filters at the same time to refine your results.

Clear All Filters

The Clear All option removes all active filters and restores the task list to its default view.

Task List Table

Tasks are displayed in a tabular layout. Each row represents a single task.

Create Task

You can use the Create Task button to create a task for yourself as an Admin or assign it to the appropriate asset owner.

  1. Click the Create Task button to open the Create Task screen.
  2. Enter the following data in the Create Task screen:
    • Summary: Enter a short title describing the task.
    • Description: Provide detailed information about the task.
    • Status: Select the current status from the dropdown list. For example, Open, In Progress.
    • Priority: Select the priority level from the dropdown list. For example, Low, Medium, High.
    • Reporter: Select the reporter who requested the asset data change from the dropdown list.
    • Assignee: Select the person responsible for completing the task from the dropdown list.
    • Asset Name: Select the asset associated with the task from the dropdown list.
    • Phase: Enter the phase in which the task is being created.
    • Due Date: Enter the expected completion date or pick a date from the calendar.
  3. After entering all the required information, click Create Task. The task will be created, and notifications will be sent to both the reporter and the assignee.
  4. The created task will appear under the Task tab, where you can track all tasks in one place.

Task Management

To manage a task from the task list, perform the following steps:

  1. Click anywhere on a task row in the task list to view its detailed information.
  2. On the task details page, you can view the following information:
    • Task ID, Summary, Priority level, and Status (displayed at the top)
    • Task Type
    • Due date
    • Assignee
    • Reporter
    • Phase
    • Description

  3. Update the task status using the Task Status dropdown menu based on the progress. The available options are:

    • Open: Task is created but not started
    • In Progress: Task is actively being worked on
    • Done: Task is completed

    After selecting the current status, click Save to finalize your update.

  4. Use the Comments section to communicate updates, questions, or clarifications. To add and manage a comment:
    1. Click in the Add a comment text box under the Comments section.
    2. Type your message and click Add. The comment will appear under the Comments section with a timestamp.
    3. Click Edit to update your comment and Save to finalize your changes.
    4. Click Delete to remove the corresponding comment.
  5. If the task meets all requirements, the Admin or the assignee of the task will click Complete. The task status will then be changed to Closed status.
  6. The Activity Log automatically records all important changes to a task, from creation to completion. This helps track the task’s complete history. Examples of logged actions include:
    • Task creation
    • Status changes
    • Adding comments
    • Task completion

Bulk Update of Task Status

Perform the following steps to update the status of multiple tasks at once:

  1. Use the checkboxes to the left of the task ID column in the task list table to select one or more tasks. Select the checkbox in the column header to select all tasks.
  2. Once tasks are selected, the Bulk Update option appears above the task list table.
  3. Click Bulk Update. The Bulk Update window appears.
  4. On the Bulk Update window, choose the following options:
    • Select Task Status from the Attribute drop-down menu.
    • Select the appropriate status (Open, In Progress, or Done) from the New Status drop-down menu.
  5. Click Update. The status of the selected tasks will be changed to the status you selected.
