Greenhouse Recruiting Candidates to Freshservice User Onboarding Ticket

← Back to index

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

The integration from Greenhouse Recruiting to Freshservice creates a new employee onboarding ticket in Freshservice whenever a candidate’s status changes to Hired in Greenhouse Recruiting. This ensures that all new hires are promptly set up with the necessary resources and support.
This integration performs the following tasks:

  • Retrieves the details of the hired candidate from Greenhouse Recruiting.
  • Converts Greenhouse office locations into corresponding Freshservice location IDs, creating new locations if they do not already exist.
  • Maps personal details by combining first and last names, primary email, and contact information; synchronizes job-related data such as department, start date, and job title; and creates a new user onboarding ticket in Freshservice using the mapped information.
  • Sets the onboarding request status to either In-Progress or Complete.
  • Sends confirmation of ticket creation name and other details to HR or managers.
  • Alert administrators about errors such as invalid locations or duplicate emails, providing diagnostic details for troubleshooting.
  • Logs all actions, including successes and failures, in OpenSearch with timestamps and payload information for auditing and dashboard creation.
  • Generates a CSV compliance report summarizing user onboarding ticket creation, errors encountered, and timestamps, supporting governance and review.

Note: You must set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to this Aquera integration. Refer to the Appendix for details.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for instructions on creating the Greenhouse Recruiting application connector and setting up the authentication.
    • Freshservice: Refer to the Freshservice Configuration Guide for instructions on creating the Freshservice application connector and setting up the authentication.
  • Permissions to set up Webhooks in Greenhouse Recruiting: Ensure you have permission to set up Webhooks in Greenhouse Recruiting to send the candidate hire events to the Aquera integration. 

Integration Setup

This section details the configuration process for integrating Greenhouse Recruiting with Fresh service. It covers creating the integration, configuring system preferences, and setting up event-driven triggers to automate the real-time transfer of new hire data from Greenhouse recruiting to Freshservice user onboarding tickets.

This integration reads the data of a newly hired candidate from the Greenhouse Recruiting application and creates a corresponding user onboarding ticket in the Freshservice application.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Freshservice / Greenhouse Recruiting.
  5. Enter a name and a description for the integration.
  6. Select your Greenhouse Recruiting and Freshservice connector instances.
  7. Click Continue. The configuration appears to set up the integration.

Set Up Preferences

The Preferences section contains essential parameters that determine how the integration operates. Each setting is crucial in customizing the integration's behavior to meet your organization's needs.

  • Admin Email Address: The designated email address that receives all system notifications, alerts, and reports related to the integration.
  • Agent Email Address: The email address of the agent responsible for this integration.
  • Email Subject: The title or summary displayed in the recipient’s inbox that describes the content of the email notification.
  • Email Body on Successful Integration Execution: Predefined template or content used in emails sent upon successfully completing the integration process.
  • Email Body on Failed Integration: Predefined template or content used in emails sent when the integration encounters errors or fails to execute successfully.
  • Enable Integration Report Download: Allows generation of integration results in CSV format, providing structured, downloadable data for analysis or import into other systems.
  • Enable Dashboard Generation:  Monitors integration performance, analyzes trends, and quickly identifies issues through customizable graphical dashboards.
  • Trigger Event: Defines the specific events in Greenhouse Recruiting that will activate or trigger the integration.
  • Report Only: Simulates the integration process by creating detailed reports of all planned actions without making any actual changes or sending notifications.
  • Debug Mode: Enables detailed logging and monitoring to assist in troubleshooting by providing comprehensive information about each step of the integration process.

Set Up Event Triggers

You need to set up the event triggers to receive candidate hire events from the Greenhouse Recruiting application and run the integration.

  1. In the Trigger section, click Event.
  2. In the Event Source drop-down, select the Greenhouse Recruiting connector that you have configured in your Aquera Admin tenant.
    The URL field displays the endpoint URL to which Greenhouse Recruiting must send the candidate hire events. This URL must be configured in the Greenhouse Recruiting application when configuring the webhooks. Refer to the Setting Up Webhooks in Greenhouse Recruiting section in the Appendix for details.
  3. Click Create Token to generate a token. Save this token securely.

Run the Integration

The integration runs when a candidate in the Greenhouse Recruiting application is hired. It processes the event received from Greenhouse Recruiting and creates a user onboarding ticket in the Freshservice application.

When a new employee is successfully onboarded, the integration sends confirmation emails to the HR/IT teams and creates a support ticket for the employee's equipment and access needs. Suppose any issues occur during the setup (like missing information or system errors). In that case, it retries to send urgent alerts about the problem.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.

Appendix

Setting Up Webhooks in Greenhouse Recruiting

This section details how to set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to the Aquera integration for processing.

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center on the left navigation pane and click Webhooks.
  4. Enter the following fields to configure your new webhook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for. For example, Aquera Integration
    • When: Select the event, Candidate has been hired, from this drop-down list. 
    • Endpoint URL: Copy the URL from the Trigger section on the Greenhouse Recruiting Candidates to the Freshservice integration setup page of the Aquera Admin portal and paste it in this field. Refer to the Set Up Event Triggers section for details.
    • Secret key: Enter the token generated in the Trigger section on the Greenhouse Recruiting Candidates to Freshservice integration setup page of the Aquera Admin portal. Refer to the Set Up Event Triggers section for details.
    • Deactivated?: Make sure that the No option is selected.
  5. Click Advanced Settings and then provide the Basic Authorization Credentials, where the token generated in the Aquera Admin portal must be provided as the password for authentication. The username can be any random string. Refer to the Set Up Event Triggers section for details on obtaining the token.
  6. Click Create webhook to complete the webhook setup.

When you create a new webhook, Greenhouse Recruiting will ping the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the webhook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the webhook.

Note: If the webhook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Webhook section and edit the webhook configuration to fix the error. After you have fixed the configuration issue, make sure that you set the Deactivated? option to No.
