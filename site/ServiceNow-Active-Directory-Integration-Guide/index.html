Integration Setup

This section outlines how the integration must be configured to push <resource type> data from <source application> to <target application>.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for <Integration Template Name> in the search box next to the Explore Templates section.
  5. Click the integration template to display the Create New Integration pop-up window.
  6. Enter a name and a description for the integration.
  7. Select both application connector instances from the dropdown menus.
  8. Click Continue. The integration configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Features

Enable the following features based on your requirements:

  • Enable user lifecycle management: Select this checkbox to synchronize users' data from Paylocity to Brivo. When enabled, the Manage Users section is displayed to configure further integration settings.
  • Enable ServiceNow writeback: Select this checkbox to enable writeback of user records from Active Directory to ServiceNow. When enabled, the system can create, update, or delete user records in ServiceNow based on changes made in Active Directory. In this configuration, Active Directory acts as the source application, and ServiceNow acts as the target application.

Manage Users

The Manage Users section allows you to configure user integration settings based on your specific use case. This section enables synchronization of user data from ServiceNow to Active Directory. It has the following settings:

JML Settings

Configure the following Joiner–Mover–Leaver (JML) settings:

  • Enable user creation: Select this checkbox to create new user records in Active Directory based on requests from the ServiceNow catalog. When enabled, the User Creation Settings section will appear for further configuration.
  • Enable user updation: Select this checkbox to update existing user records in Active Directory based on changes made in the ServiceNow catalog. When enabled, the User Updation Settings will appear for further configuration.
  • Enable user deactivation: Select this checkbox to deactivate user records in Active Directory based on requests from the ServiceNow catalog. When enabled, the User Deactivation Settings will appear for further configuration.

User Creation Settings

Configure the following settings related to user creation:

ServiceNow Catalog Managemnet

Configure the following ServiceNow catalog settings:

  • Service catalog category: Select the ServiceNow service catalog category that is used for user creation requests. For example, select Create User Catalog to process only user creation-related requests.
ServiceNow Field Management

Configure the following ServiceNow field settings:

  • Select variable for user's first name: Select the ServiceNow catalog variable that stores the user's first name.
  • Select variable for user's middle name: Select the ServiceNow catalog variable that stores the user's middle name.
  • Select variable for user's last name: Select the ServiceNow catalog variable that stores the user's last name.

User Updation Settings

Configure the following settings related to user updation:

ServiceNow Catalog Managemnet

Configure the following ServiceNow catalog settings:

  • Service catalog category: Select the ServiceNow service catalog category that is used for user updation requests. For example, select Update User Catalog to process only user updation-related requests.
ServiceNow Field Management

Configure the following ServiceNow field settings:

  • Select variable for user from update catalog: Select the ServiceNow catalog variable that identifies the user whose record needs to be updated.

User Deactivation Settings

Configure the following settings related to user deactivation:

ServiceNow Catalog Managemnet

Configure the following ServiceNow catalog settings:

  • Service catalog category: Select the ServiceNow service catalog category that is used for user deactivation requests. For example, select Deactivation of user to process only user deactivation-related requests.
ServiceNow Field Management

Configure the following ServiceNow field settings:

  • Select variable for user from deactivation catalog: Select the ServiceNow catalog variable that identifies the user whose account needs to be deactivated.

User Provisioning Settings

Configure the following user provisioning settings:

  • Enable manager mapping: Select this checkbox to map the user's manager. When enabled, the following field will appear for further configuration.
    • Select variable for user manager: Select the ServiceNow catalog variable that stores the user’s manager information.
  • Enable process future hire: Select this checkbox to enable processing users who have a future hire date. When enabled, such users will be created in Active Directory in a disabled state. These accounts can be automatically activated prior to the user's start date based on your configuration. When this option is enabled, additional fields will appear for further configuration.
    • Select variable for user activation date: Select the ServiceNow catalog variable that stores the user’s hire date or start date.
    • Activate user before X days from start date: Specify the number of days prior to the user’s start date when the account should be automatically activated in Active Directory. For example, if you enter 3, the system will activate the user account three days before the configured start date.
  • Enable InActive User Scan: Scan for users and create a request for those who haven't logged in for x days. When enabled, the following field will appear for further configuration.
    • Scan Lookback Window: Number of days to look back from today for users who haven't logged in . Example: Enter 90 to create a request for all users who haven't logged in within the last 90 days.

Correlation Field Settings

Configure the following correlation field settings:

  • Select correlation attribute in AD: Select the unique identifier attribute from Active Directory that is used for correlation and user record matching with ServiceNow. For example, select employeeId as the correlation attribute.
  • Select correlation attribute in ServiceNow: Select the corresponding unique identifier attribute from ServiceNow that is used for correlation and user record matching with Active Directory. For example, select Employee number as the correlation attribute.

Ticket Management

Configure the following post-provisioning ticket settings:

Ticket Closer Status: Select the ServiceNow ticket state to which the request should transition after successful user creation, updation, or deactivation in Active Directory. For example, you may select Closed Complete to indicate that the requested operation was completed successfully.

Tables

Tables enable you to update specific data of a resource in a target application during integration runs when that data is missing in the source application from which the resource data is retrieved.

You can add tables in integrations to sync the values in tables to another application. Tables are used to update an attribute value in another application when the application attribute does not provide specific data.

The following tables are available:

  • <Table Name>: Click the table row to display the Update Table pop-up window. Each row in the table corresponds to a key name and key value pair. You can add rows to the table using one of the following options: <Depending upon the integration use case, you can document accordingly>

    • Add Row: Click Add Row to manually add a row by providing a key name and a key value.
    • Upload CSV: Click Upload CSV to upload rows to the table using a CSV file. The CSV file should have two columns with headings 'NAME' and 'VALUE.'
      Notes:
      • The key name within a table should be unique. The key name can include a combination of letters and numbers.
      • The key name supports all the special characters except the following:
        • / (forward slash)
        • \ (backslash)
        • " " (double quotes)

    Click Save to save the changes made.

Update the Integration to the Latest Version

You can upgrade an integration built from a template to the latest version to access new features and enhancements. 

Perform the following steps to upgrade an integration:

  1. Navigate to the Integrations module on the Aquera Admin portal.
  2. Click the vertical ellipsis icon next to an integration and click View Details. The Details panel is displayed with the following sections:
    • INTEGRATION DETAILS: Displays the integration name, description, creation date, last update date, and the user who created or last updated it.
    • TEMPLATE DETAILS: Displays the current version number and its features under CURRENT VERSION. If a newer version is available, the version number and the changes are shown under NEW VERSION AVAILABLE with an Upgrade button.
  3. Click Upgrade. The Upgrade Integration pop-up window is displayed.
  4. Provide the following details for the new integration to be created after the upgrade:
    • New Integration Name: Enter the name of the new integration.
    • Description: Provide a description for the new integration.
      The current version number and the upgraded version number are displayed.
  5. Click Upgrade. A new integration is created with the same configuration details as the existing one, but with features or enhancements from the upgraded version.
