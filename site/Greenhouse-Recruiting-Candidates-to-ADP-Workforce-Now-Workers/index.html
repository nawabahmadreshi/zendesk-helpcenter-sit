Greenhouse Recruiting Candidates to ADP Workforce Now Workers

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating the integration of data across resource types. 

The Greenhouse Recruiting Candidates to ADP Workforce Now Workers integration automates the process of onboarding candidates from the Greenhouse Recruiting application to the ADP Workforce Now application. This integration is triggered when a candidate in your Greenhouse Recruiting application is moved to the hired status.

This integration performs the following tasks:

  • Retrieves the details of the candidates whose status has been changed to hired in the Greenhouse Recruiting application. Aquera integration receives an event when a candidate is hired.
  • Creates a worker record in the ADP Workforce Now application for a newly hired candidate in Greenhouse Recruiting
  • Synchronizes the details such as candidate information, job details, and offer details from Greenhouse Recruiting to the new worker records in the ADP Workforce Now application. The mapping of attributes and the worker data synchronization will be performed based on the template that you have configured in this integration.
  • Sets the onboarding status of the newly hired worker in the ADP Workforce Now application to In progress or Complete based on the configuration that you have set in this integration
  • Sends an email notification with the applicant ID and Associate OID details of a worker to the configured list of email addresses, on successful onboarding of the worker
  • Sends an email notification to the configured list of email addresses, if an error is encountered, for example, when you have configured an incorrect onboarding template name in the integration.

Note: You must set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to this Aquera integration. Refer to the Appendix for details.

Prerequisites

The following are the prerequisites:

  • Tenant Account in the Aquera Admin portal: You must have a tenant account with admin access in the Aquera Admin portal. Visit https://admin.aquera.io and register to get a tenant account. Contact your Aquera account manager to get the Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure application connectors in the Aquera Admin portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Greenhouse Recruiting: Refer to the Greenhouse Recruiting Configuration Guide for instructions on creating the Greenhouse Recruiting application connector and setting up the authentication.
    • ADP Workforce Now: Refer to the ADP Workforce Now Configuration Guide for instructions on creating the ADP Workforce Now application connector and setting up the authentication.
  • Permissions to configure webhooks: Make sure that the Greenhouse Recruiting user has enough permissions to configure webhooks.

Integration Setup

This integration reads the data of a newly hired candidate from the Greenhouse Recrutiing application and creates a corresponding worker record in the ADP Workforce Now application.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Greenhouse Recruiting Candidates to ADP Workforce Now Workers.
  5. Enter a name and a description for the integration.
  6. Select your Greenhouse Recruiting and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Set Up Preferences

The Preferences section enables you to customize the integration settings according to your specific use case. It has the following configuration parameters:

  • Applicant onboarding template name: Enter the name of the ADP onboarding template that must be applied for the new hires. This is a mandatory setting. If you provide an incorrect template name, the integration fails and an error notification is sent to the email addresses configured in the Send email notifications to field. 
  • Applicant onboarding status: Select the onboarding status that must be assigned to the new hires in ADP Workforce Now. The valid values are In progress and Complete. Depending on the onboarding status, the set of attributes that will be updated for the employee will vary.
  • Send email notifications to: Enter the comma-separated list of the email addresses of the admins to whom email notifications must be sent on successful employee onboarding or any integration errors.

Set Up Event Triggers

You need to set up the event triggers for receiving candidate hire events from the Greenhouse Recruiting application and running the integration.

  1. In the Trigger section, click Event.
  2. In the Event Source drop-down, select the Greenhouse Recruiting connector that you have configured in your Aquera Admin tenant.
    The URL field displays the endpoint URL to which Greenhouse Recruiting will have to send the candidate hire events. This URL needs to be configured in the Greenhouse Recruiting application when you configure the webhooks. Refer to the Setting Up Webhooks in Greenhouse Recruiting section in Appendix for details.
  3. Click Create Token to generate a token. Save this token securely.

Run the Integration

The integration runs when a candidate in the Greenhouse Recruiting application is hired. The integration will process the event received from Greenhouse Recruiting and create an employee record in the ADP Workforce Now application.

On successful onboarding of an employee to the ADP Workforce Now application, the integration triggers an email notification to the email addresses configured in your integration. It will also send notifications and reports to the email addresses in case of any integration errors.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.

Appendix

Setting Up Webhooks in Greenhouse Recruiting

This section details how to set up webhooks in the Greenhouse Recruiting application to send the candidate hire events to the Aquera integration for processing.

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center on the left navigation pane and click Webhooks.
  4. Enter the following fields to configure your new webhook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for. For example, Aquera Integration
    • When: Select the event, Candidate has been hired, from this drop-down list. 
    • Endpoint URL: Copy the URL from the Trigger section on the Greenhouse Recruiting Candidates to ADP Workforce Now Workers integration setup page of the Aquera Admin portal and paste it in this field. Refer to the Set Up Event Triggers section for details.
    • Secret key: Enter the token generated in the Trigger section on the Greenhouse Recruiting Candidates to ADP Workforce Now Workers integration setup page of the Aquera Admin portal. Refer to the Set Up Event Triggers section for details.
    • Deactivated?: Make sure that the No option is selected.
  5. Click Advanced Settings and then provide the Basic Authorization Credentials where in the token generated in the Aquera Admin portal must be provided as the password for authentication. The username can be any random string. Refer to the Set Up Event Triggers section for details on obtaining the token.
  6. Click Create webhook to complete the webhook setup.

When you create a new webhook, Greenhouse Recruiting will send a ping to the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the webhook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the webhook.

Note: If the webhook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Web Hook section and edit the webhook configuration to fix the error. Make sure that you set the Deactivated? option to No after you have fixed the configuration issue.
