Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables seamless synchronization between Jamf and Paylocity, enhancing asset management and simplifying employee-device assignments. It continuously retrieves and reconciles asset data from Jamf, including iPads, MacBooks, and other Apple devices, with employee records in Paylocity, ensuring better asset tracking and accurate device allocation across your organization.

Key Functions

The Jamf to Paylocity integration provides the following functions:

  • Retrieves asset data from Jamf, supporting incremental updates and filtering by source.
  • Accesses and synchronizes employee records from Paylocity, aligning these accurately with device assets based on email addresses and device serial numbers.
  • Dynamically allocates or de-allocates devices within Paylocity based on up-to-date Jamf records, ensuring accuracy and compliance.
  • Handles successful events by logging details, generating CSV reports, sending notification emails, and updating dashboard data, if enabled.
  • Handles failure events by logging errors, creating failure-specific CSV reports, sending failure notifications, and updating dashboard analytics, if enabled.
  • Sends automated, detailed notification emails to employees and administrators regarding device assignments, integration status, and updates.
  • Produces CSV reports capturing user data, operation results, timestamps, and error details.
  • Enables dashboard analytics to monitor integration outcomes and performance.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin portal:
    • Jamf: Refer to the Jamf Configuration Guide for instructions on creating the Jamf application connector and setting up the authentication.
    • Paylocity: Refer to the Paylocity Configuration Guide for instructions on creating the Paylocity application connector and setting up the authentication.

Integration Setup

This section details the configuration of the scheduled integration between Jamf and Paylocity. It covers creating the integration, setting up integration preferences, and configuring how device and employee lifecycle events are synchronized. The integration reads device and user data changes from Jamf, including new device enrollments, updates, and user assignments, at scheduled intervals and creates or updates corresponding records in Paylocity.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Jamf to Paylocity Assets Synchronization.
  5. Enter a name and a description for the integration to identify it.
  6. Select your Jamf and Paylocity connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Global Settings

Configure the following global settings:

  • Enable Report-Only Mode: Select this checkbox to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Debug Mode: Select this checkbox to generate logs outlining data changes that would occur for debugging.
    Note: The Log Settings section will be displayed only when this checkbox is selected.
  • Enable Notifications: Select this checkbox to enable the integration-related notifications.
    Note: The Notification Settings section will be displayed only when this checkbox is selected.
  • Enable Report Generation: Select this checkbox to generate the integration report.
    Note: The Report Settings section will be displayed only when this checkbox is selected.
  • Enable Analytics: Select this checkbox to analyze integration-related metrics in a dashboard.
    Note: The Analytics Settings section will be displayed only when this checkbox is selected.
  • Enable Asset Management: Select this checkbox to activate the asset management feature for the company.
    Note: The Asset Management Settings section will be displayed only when this checkbox is selected.

Log Settings

Configure the following log settings:

  • Select the Log Detail Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Notification Settings

Configure the following settings for email notifications:

  • Notify Admin Via Email: Select this checkbox to send email notifications to all admins. When this checkbox is selected, the following settings will be displayed:
    • Admin's Email Addresses: Enter the email addresses of the admins to whom the email notifications will be sent.
    • Admin Email Subject: Enter the content that must be used as the email subject for sending email notifications to the admins.
    • Admin Email Body: Enter the content that must be used as the email body for all admin email notifications.
  • Notify Employees via Email: Select this checkbox to send email notifications to all employees. When this checkbox is selected, the following settings will be displayed:
    • Employee Email Subject: Enter the content that must be used as the email subject for sending email notifications to the employees.
    • Employee Email Body: Enter the content that must be used as the email body for sending email notifications to the employees.
  • Send Integration Report via Email: Select this checkbox to receive email notification with a CSV report for all integration runs. When this checkbox is selected, the following settings will be displayed:
    • Select the Notification Report Detail Level: Select the desired notification report level from the dropdown menu. The following are the available options:
      • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
      • Full Details: All attribute details are displayed.

Report Settings

Configure the following report settings:

  • Select the Report Detail Level: Select the desired report detail level from the dropdown menu. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attribute details.
      • Included Attributes: Select the attributes to be included in the report.
      • Excluded Attributes: Select the attributes to be excluded from the report.

Analytics Settings

Configure the following analytics settings:

  • Select the Analytics Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard. The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.

Asset Management Settings

Configure the following settings for the asset management:

  • Source Settings
    Configure the following settings in the source application:
    • Device Type: Select the device type(s) for synchronization.
    • Number of Devices to Sync: Enter the number of devices to synchronize in an integration run.
    • Advanced Sync & Filters: Select the advanced filtering and synchronization option for the integration run.
    • Days for Initial Incremental Sync: Enter the number of previous days to consider for data sync during the first incremental run.
  • Target Settings:
    Configure the following settings in the target application:
    • Company ID: Enter your company ID to enable device management for your organization.
    • Target Application Category: Select the category in the target application where devices will be synchronized.

Attribute Map

The Jamf to Paylocity Assets Synchronization integration maps the Jamf and Paylocity attributes. Only the source attributes can be modified, as needed, to align with your specific use case.
