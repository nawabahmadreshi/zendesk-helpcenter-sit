Null Values Sync Integration Guide

← Back to index

Overview

Microsoft Azure AD can facilitate changes in attribute values, transitioning them from one value to another. However, Azure AD does not support transitioning some attribute values to empty. If an attribute is set to empty for a user object within the Azure AD application, Azure AD will omit that attribute during outbound provisioning to the target application. For details, refer to the Microsoft documentation.

To resolve this issue, Aquera provides a workaround script to sync attributes with null values from Azure AD to the target application. The script compares the target and source users to check for any updates, and then those attributes are sent in a patch call. The script will fetch the target users based on the attribute name mentioned in the variable targetExtAttributeNameToMatch. If targetExtAttributeNameToMatch is left empty, by default, the script will use email to match the target users.

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize the following cost center attributes from ADP Workforce Now to ADP iHCM: Cost center code value, Cost center short name, and Effective date.

To identify cost center records across both systems, the integration uses the Cost Center Code attribute from ADP Workforce Now and matches it to the Cost Center Code available in the ADP iHCM.

Key Functions

The ADP iHCM / ADP Workforce Now Cost Center Sync integration performs the following functions:

  • Reads cost center data from the Cost Center or the Department validation table in ADP Workforce Now, based on the integration configuration.
  • Reads data from a specific company code in ADP Workforce Now.
  • Creates new cost centers in ADP iHCM when new cost centers are added in ADP Workforce Now.
  • Updates cost center details in ADP iHCM when any changes are made to the existing cost centers in ADP Workforce Now.
  • Retrieves, creates, and updates cost center records based on the limits set in the integration.
  • Performs full synchronization of cost centers on each integration run.

Integration Setup

This section outlines how the integration must be configured to synchronize cost center data from ADP Workforce Now to ADP iHCM.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select ADP iHCM / ADP Workforce Now Cost Center Sync.
  5. Enter a name and a description for the integration.
  6. Select your ADP iHCM and ADP Workforce Now connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source (ADP Workforce Now) and target (ADP iHCM) application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and settings according to your use case. It has the following settings:

Integration Features

Enable the following feature based on your requirements:

  • Enable Cost Center Life Cycle Management: Turn on this toggle to synchronize the cost center data from ADP Workforce Now to ADP iHCM. When turned on, the Manage Cost Centers section is displayed for further configuration.
    Note: This setting is mandatory for the integration.

General Settings

Configure the following general settings:

  • Report only: Turn on this toggle to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable logging: Turn on this toggle to generate logs outlining data changes. When the toggle is on, the Log Settings section is displayed for further configuration.
  • Enable notifications: Turn on this toggle to enable integration-related notifications. When the toggle is on, the Notifications section is displayed for further configuration.
  • Enable report generation: Turn on this toggle to generate the integration report. When the toggle is on, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Turn on this toggle to visualize the integration-related metrics. When the toggle is on, the Log Insights section is displayed for further configuration.
  • Enable debug mode: Turn on this toggle to generate logs outlining data changes that would occur, for debugging. When the toggle is on, the Debug Settings section is displayed for further configuration.

Manage Cost Centers

The Manage Cost Centers section allows you to customize the settings to manage cost center synchronizations based on your use case.

Note: This section appears only if you have turned on the Enable Cost Center Life Cycle Management toggle in the Parameters section.

Cost Center Settings

This section enables you to configure the settings related to the cost center data and the company code.

  • Select validation table: Select the validation table in ADP Workforce Now that must be used by the integration to read the cost center data. 
    • If you select Cost Centers, the cost center data will be retrieved from the Cost Center validation table in your ADP Workforce Now instance.
    • If you select Departments, the cost center data will be retrieved from the Department validation table in your ADP Workforce Now instance.
  • Select company code: Select a company code in ADP Workforce Now from which the cost center data must be retrieved.

Correlation Settings

This section shows the source and target attributes that are used for identifying the matching cost center records across the source and target applications. 

  • Source correlation attribute: Displays the cost center attribute that will be used to identify a unique cost center record in ADP Workforce Now. 
  • Target correlation attribute: Displays the cost center attribute that will be used to identify the corresponding cost center record in ADP iHCM. 

Provisioning Settings

This section provides the settings to enable creation and update of cost center records.

  • Create cost centers: Turn on this toggle to allow the integration to create cost centers in ADP iHCM. When the toggle is off, the integration will not create cost centers in ADP iHCM.
  • Update cost centers: Turn on this toggle to allow the integration to update cost centers in ADP iHCM. When the toggle is off, the integration will not update cost centers in ADP iHCM.

Provisioning Threshold

This section provides the settings to limit the number of cost center records that will be retrieved from Workforce Now and that will be created or updated in ADP iHCM.

  • Fetch limit mode: Select whether you want to limit the number of cost center records that must be retrieved from ADP Workforce Now in a single integration run. 
    • If you select Unlimited, all cost center data will be retrieved.
    • If you select Value, you can set a limit to the number of cost center records that will be retrieved.
  • Fetch limit: Enter the maximum number of cost center records that will be retrieved from ADP Workforce Now in a single integration run. This field is displayed only if you have selected Value in the Fetch limit mode drop-down.
  • Create limit mode: Select whether you want to limit the number of cost center records that will be created in iHCM in a single integration run. This field is displayed only if the Create cost centers toggle is turned on.
    • If you select Unlimited, all the new cost centers from Workforce Now will be created in iHCM.
    • If you select Value, you can set a limit to the number of cost center records that will be created.
  • Create limit: Enter the maximum number of cost center records that are allowed to be created in ADP iHCM in a single integration run. This field is displayed only if you have selected Value in the Create limit mode drop-down.
  • Update limit mode: Select whether you want to limit the number of cost center records that will be updated in iHCM in a single integration run. This field is displayed only if the Update cost centers toggle is turned on.
    • If you select Unlimited, all the modified cost center data will be updated in iHCM.
    • If you select Value, you can set a limit to the number of cost center records that will be updated.
  • Update limit: Enter the maximum number of cost center records that are allowed to be updated in ADP iHCM in a single integration run. This field is displayed only if you have selected Value in the Update limit mode drop-down.

Notification Template Settings

Configure the following settings to customize notifications:

  • Send email notifications for success: Turn on this toggle to send an email notification on successful cost center syncs in an integration run. When enabled, the Success Notifications section is displayed for further configuration.
  • Send email notifications for failures: Turn on this toggle to send an email notification on failed cost center syncs in an integration run. When enabled, the Failure Notifications section is displayed for further configuration.
