User Integration Setup Guide - Paylocity to Okta

← Back to index

Overview

This guide covers the end-to-end tasks involved in setting up an orchestration to sync users in Paylocity with Okta. 

Prerequisites

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in the Aquera Admin console:

  • Paylocity. Refer to the Paylocity Configuration Guide for details.
  • Okta. Refer to the Okta Configuration Guide for details.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from Paylocity to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Considerations

  • Work email can be retrieved from the Paylocity system or be generated by Aquera orchestration and updated back to the system.
  • The criteria for matching a user record from Paylocity with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping Paylocity user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in Paylocity as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script orchestration. For scripts, refer to the Appendix.

Building Orchestration

This document describes how to set up the orchestration using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set Paylocity as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during the orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, and department values from the source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in the restriction list for scoping of the groups which would be managed by Aquera.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Running Orchestration

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if the correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in Paylocity.

Note: If Active Directory is set as profile master in Okta, the user update operation will fail. To allow orchestration to update users, you need to disable the setting which sets Active Directory as profile master. Perform the following steps to disable the setting:

  1. Go to the Active Directory application in your Okta instance.
  2. Go to the Provisioning tab and click To Okta.
  3. Go to Profile & Lifecycle Sourcing and disable the setting which sets Active Directory as profile master.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in Paylocity and Okta before setting up the orchestration. 

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add Paylocity and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

A few sample scripts are provided here for your reference. The scripts may vary based on your use case and the attributes to be retrieved for analysis.

Analyzing the Users in Paylocity 

Use the below sample script in a scheduled script orchestration for generating a report from Paylocity for analysis of username, user title, work location, and department. The work location and department are mapped to IAN attributes.

const ENT = "urn:ietf:params:scim:schemas:extension:enterprise:2.0:User";
let employees = payloc.getUsers();
for (let employee of employees) 
{
    log.info(employee.userName + ',' + employee.active + ',' + 
    employee.title + ',' + employee[IAN].worklocation + ',' + 
    employee[IAN].costCenter1);
}

Analyzing the Users in Okta

Use the below sample script in a script orchestration for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}
