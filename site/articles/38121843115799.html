Overview

This guide provides details on the changes across various versions of User Integration.

V5

New Feature

Incremental Cutoff Limit

The User Integration setup now includes support for configuring the Incremental Cutoff Limit. This setting allows the integration to automatically switch from incremental mode to full sync mode when the number of source records to be created or updated in the target exceeds the specified limit. 
Note: This feature is introduced in the V5.3 version and is rolled out only for a limited set of customers.

Enhancements

Additional Configurations in Weekly Schedule Setup

The weekly schedule setup has the following two additional configurations:

  • Start Date: Select the date from which the weekly schedule becomes effective. The integration will first run automatically on or after this date, depending on the selected weekdays and the week frequency set in the Repeat every field.
  • Repeat every: Specify how frequently the integration must run in terms of weeks. This option enables you to schedule the integration to run weekly, bi-weekly, once in 3 weeks, or once in 4 weeks. For example, if you set 2 in this field, the integration will run every 2 weeks on the selected weekdays.

Support for Truncated Logs

When the content in an integration log report exceeds 100 KB, the word TRUNCATED appears in the centre of the log report.

Note: To view the complete logs without truncation, you must download the integration log report.

Trigger Integration Hooks on User Group Changes

Integration hooks are now triggered when any group changes occur to a user record during user integrations, if Update User Integration Hook is configured. Earlier, for group changes to a user, the user update integration hooks were not triggered.

Delete Condition

A new configuraton parameter, Delete Condition, has been added to set the condition for deleting users in the target application. This optional configuration is effective only if you have set the Termination Delay configuration. When the number of days is configured in the Termination Delay parameter, you can optionally set a condition for deleting the users. For example, SOURCE.active==false.

Note: This feature is introduced in the V5.2 version and is rolled out only for a limited set of customers.

V4

Enhancements

Logs

  • The summary section in the User Integration logs is now improved for better accounting.
    • The following parameters have been added:
      • source records collision
      • source duplicate records
      • create errors
      • create condition met
      • update errors
      • update condition met
      • terminate errors
      • group update failures
    • The following parameters have been renamed:
      • ignored renamed as source records ignored
      • deleted renamed as terminated
      • pending delete renamed as pending terminate
      • inactive renamed as inactive records processed
  • The integration logs in debug mode are now improved with the following details:
    • For expression failures, the logs mention the failure reason.
    • When the expressions include %ATTEMPT, %UATTEMPT, or %USEQ, the logs mention the target attribute to which the expression is mapped.
    • The logs mention the attempt value after evaluating the attempt logic.
  • The user data transaction errors that occur during the integration runs are now shown in the Integration Logs > RESPONSE DATA column as Error stats. Earlier, such transaction errors were reported as failures. 

Weekly Schedule Enhancement

  • The Weekly Synchronization schedule now supports options to set the minutes or hours and set an end time.

Integration Expressions

  • %UATTEMPT keyword is now supported in the attribute mapping. %UATTEMPT uses the same logic as %ATTEMPT; however, %UATTEMPT operates on each attribute containing the keyword independently rather than as a set. %UATTEMPT can be used to achieve the uniqueness of all the attributes in the same integration. This keyword operates like %ATTEMPT but settles on a unique value regardless of other attributes. %ATTEMPT enforces the same attempt value for multiple attributes. However, %UATTEMPT may use different attempt values for multiple attributes.
  • %USEQ is now supported in incremental mode.
  • Attribute names with special characters are now supported in attribute mapping expressions.
  • Variables are now supported in TABLE expressions when accessing the tables.
  • Advanced expressions are now supported in the Match Expression configuration in incremental mode. This includes support of JavaScript functions for the source attributes, logical operators, and array type attributes.

Performance Improvements

  • Integrations now load faster when the Enable Group Management configuration is disabled.

Modifications

Reports

  • The maximum size of the zip file in the integration email report is now increased from 10 MB to 40 MB.
