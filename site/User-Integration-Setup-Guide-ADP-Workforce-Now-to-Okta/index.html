User Integration Setup Guide - ADP Workforce Now to Okta

← Back to index

Overview

This guide covers the end to end tasks involved in setting up an orchestration to sync users in ADP Workforce Now with Okta.

The following tasks are involved in setting up the orchestration from ADP Workforce Now to Okta:

  • Initial Preparation
  • Configuring Applications in Aquera Admin
  • Orchestration Setup
  • Running Orchestration

Initial Preparation

ADP Workforce Now

  1. Log in to the ADP Marketplace as an ADP Admin.
  2. Purchase Aquera's orchestration connector for ADP Workforce Now® edition from the ADP Marketplace. The connector to be selected from this page must be based on the number of employees (users).

  3. After you purchase the connector app on the ADP Marketplace, you must give appropriate permissions to the app so that the app will have authorized access to your organization's data. Aquera generates a token to make the connection.

    • To authorize Aquera for the data access to your ADP instance, navigate to My Apps > Aquera in the Marketplace.
    • For details on the required permissions and how to set them up, refer to the Setting  up Permissions and the Permissions Required sections in the ADP Workforce Now Configuration Guide. 

    The following screen is displayed in the ADP Marketplace when the authorization is successful:

Aquera Admin Console

  1. Your IT admin must register as an Aquera Admin.
  2. Once registered successfully, Aquera creates a tenant for the IT Admin and provides access to it.

Configuring Applications in Aquera Admin

The following applications must be added and configured in Aquera Admin console:

  • ADP Workforce Now. Refer to the ADP Workforce Now Configuration Guide for details.
  • Okta. Refer to the Okta Configuration Guide for details.

Orchestration Setup

This section details how the orchestration must be configured to allow data push from ADP Workforce Now to Okta. 

It also describes the various analyses to be performed and the important points to be considered before setting up the orchestration.

Considerations

  • Work email can be retrieved from the ADP Workforce Now system or be generated by Aquera orchestration and updated back to the system.
  • The criteria for matching a user record from ADP Workforce Now with that in the Okta.

General Analysis Steps

  • Fill out the mapping template attached with this guide for mapping ADP Workforce Now user attributes to Okta user attributes. The most common attributes are mapped in the attached template. You can add more attribute maps to the sheet as per your requirement.

You can analyze the users in ADP Workforce Now as well as in Okta using one of the following ways:

  • Downloading the CSV file containing the list of all users through the Data tab on the Application page. For details, refer to the User Reports Guide.
  • Using Script orchestration. For scripts, refer to the Appendix.

Building Orchestration

This document describes how to set up the orchestration using the Synchronization orchestration method.

  • Synchronization Orchestration - In this orchestration, set ADP Workforce Now as the Source Application and Okta as the Target Application. Refer to the Synchronization Orchestration Guide for details on the configurations.

You can set the following configurations in the Synchronization orchestration:

  • Set the Admin Email Options to receive email notifications for success/failures during orchestration run.
  • Set the email list of the IT admin or manager to receive user creation notification emails and user deactivation notification emails.
  • Use tables for creating lookup tables based on location, department values from source.
  • Enable Group management if Aquera has to manage the groups and group assignment rules based on your group analysis. Set the group names in restriction list for scoping of the groups which would be managed by Aquera.
    • To retrieve only the supervisors/managers from ADP and add them to a group, you can create a group rule with the basic condition as follows:
      SOURCE[IAN].managementPositionIndicator Equals true

      Make sure that the group name is configured in the Group Management Restriction List in the Configuration tab.

Refer to the Synchronization Orchestration Guide for details on Synchronization orchestration configurations.

Orchestration Expressions 

Expressions are necessary for complex data transformations. Aquera provides extensions to JavaScript expressions to support full JavaScript syntax, and the extensions are designed to make it easier to work with SCIM attributes.

Expressions are used in some configuration fields, the Attribute Map tab, and the Group Rules tab, when you setup orchestrations. For more details on the keywords, inbuilt methods and functions provided by Aquera, refer to Orchestration - Expressions.

Expression to change the date format from source to target application

The following is a sample expression to change the date format of an attribute value from source application to target application.

Example: If the date format received from source is "YYYY-MM-DD", then to change the date format to "MM/DD/YYYY" in the target application, use the below Javascript expression:

let hireDt = SOURCE[IAN].originalHireDate.split("-");

hireDt[1]+"/"+hireDt[2]+"/"+hireDt[0];

Running Orchestration

  • Enter the matching expression to match the source records with the destination records.
  • Run the orchestration in Report Only mode and determine if the correct data is updated in Okta. If the data is not correctly updated, change the matching expression to correctly match users and/or update the data.
  • Increment the Update Limit (maximum number of records to be updated in a single run) for each run in Report Only mode to ensure that the records are updated correctly when the records count is more.
  • Modify the attributes mapping and repeat the process until the data is correctly updated.
  • Monitor the orchestration logs thoroughly to ensure that no errors are encountered. Modify the orchestration configurations to fix any errors.
  • Once all users are synced, user creations and updates are based on the incremental changes in ADP Workforce Now.
  • ADP Workforce Now does not return an event when a user's manager is removed. This is a limitation of the ADP Workforce Now.  Hence during incremental imports, when a user’s manager is removed from the ADP Workforce Now, the manager attribute values will not be removed from the target even if the Map Manager checkbox is selected. However, during full imports, the user’s manager will be removed from the target.

Appendix

This section provides the scripts that may be used by the Customer Support team to analyze the existing users in ADP Workforce Now and Okta before setting up the orchestration. 

This document explains how analyses can be done using a scheduled script orchestration. In this orchestration, add ADP Workforce Now and Okta applications and set the corresponding variable names. In the script, the variable names must be used to refer to the applications.
For details on configuring the script orchestration, refer to the Custom Script Orchestration Guide.

Few sample scripts are provided here for your reference. The scripts may vary based on your use case  and the attributes to be retrieved for analysis.

Analyzing the Users in ADP Workforce Now 

Use the below sample script in a scheduled script orchestration for generating a report from ADP Workforce Now for analysis of user status, location, department, and so on.

let adp_users = ADP.getUsers();

let i = 0;

log.info('ADP ID,ADP STATUS,ADP EMPNO,ADP GIVENNAME,ADP FAMILYNAME,ADP WORK EMAIL,ADP LOCATION CODE,ADP JOBCODE,ADP DEPT');

for (i = 0; i < adp_users.length; i++) {

    let adp_id = adp_users[i].id;

    let adp_status = adp_users[i][IAN].status;

    let adp_emp = adp_users[i][ENT].employeeNumber;

    let adp_locationcode = adp_users[i][IAN].LocationCode || '';

    let adp_job = adp_users[i][IAN].jobCode || '';

    let adp_dept = adp_users[i][IAN].departmentCode || '';

    let adp_given = adp_users[i].name.givenName;

    let adp_family = adp_users[i].name.familyName;

    let adp_work = adp_users[i].emails && adp_users[i].emails.find(x => x.type == 'work') && adp_users[i].emails.find(x => x.type == 'work').value || '';

    log.info('"' 

    + adp_id + '","' 

    + adp_status + '","'

    + adp_emp + '","'

    + adp_given + '","' 

    + adp_family + '","'

    + adp_work + '","'

    + adp_locationcode + '","'

    + adp_job + '","'

    + adp_dept + '"');

}

Analyzing the Users in Okta

Use the below sample script in a script orchestration for generating a report from Okta for analysis of user status, department, division, user type, and so on.

log.info('OKTA ID, OKTA ACTIVE, OKTA EMPLOYEENUMBER, OKTA GIVENNAME, OKTA FAMILYNAME, OKTA USERNAME, OKTA MIDDLENAME, OKTA NICKNAME, OKTA TITLE, OKTA DEPARTMENT, OKTA DIVISION, OKTA USERTYPE');
let users = okta.getUsers();
for (let user of users) {
log.info('"'
+ user.id + '","'
+ user.active + '","'
+ (user[ENT] && user[ENT].employeeNumber || '') + '","'
+ user.name.givenName + '","'
+ user.name.familyName + '","'
+ user.userName + '","'
+ user.name.middleName + '","' 
+ user.name.nickName + '","'
+ user.title + '","'
+ user.department + '","'
+ user.division + '","'
+ user.userType +  '"');
}
