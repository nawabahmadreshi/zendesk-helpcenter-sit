Set Up Webhooks in Paycor

← Back to index

Overview

When a specific set of events occurs in an application or system, Aquera integrations can process those events and update those changes in a different system or application. The events can be specific data changes in the application, such as creating a new user, creating a new hire, and updating an employee's data. For the application to send those events, you must configure webhooks in the application and specify the set of events for which the notifications have to be sent. When you configure webhooks in that application, you must also specify Aquera's event integration URL to which you want to send those event notifications.

This guide explains the webhook setup in the Paycor application to enable the application to send events to Aquera. After the webhooks setup is completed in Paycor, when a specific event occurs, Paycor sends a JSON payload containing the event details to the Aquera event integration URL configured in the webhooks. Depending on the type of event, the payload data vary.

Aquera integration then processes these events and updates the corresponding data in a target application.

Prerequisite

You must set up an integration in the Aquera Admin portal to process the Paycor events. For details, refer to the Integration Using Scripts guide.

Set Up Webhooks in Paycor

Perform the following steps to set up Webhooks in Paycor:

  1. Log in to your Paycor instance using the admin credentials.
  2. Click Applications on the top menu bar.
  3. Click the application name in the Applications section.
  4. Go to the Event Subscription tab and enter the following details:
    • Event Notification URL: Paste the URL that has been copied from the integration in the Aquera Admin portal.
    • Event Secret: Paste the refresh token that has been copied from the integration in the Aquera Admin portal.

      Note: Make sure the events are subscribed before you activate the application. If events are subscribed after the application activation process, you must re-activate the application.
  5. Enable the required events to trigger an action when the corresponding fields are updated in the Paycor application, then click Save.

    Whenever any of the enabled events occur, Paycor sends a JSON payload with the event details to the Aquera event integration URL. Based on the scopes enabled in the Data Access tab, the events will be listed in the Events Subscription tab.

    For example, to capture updates to an employee’s termination, you must enable the Employee Information Updated event.

    Refer to Event Triggers for more details about the specific events.
