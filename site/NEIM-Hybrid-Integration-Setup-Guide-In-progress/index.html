Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize contractors from NEIM to Active Directory and Entra ID. 

Key Functions

The NEIM / Hybrid integration performs the following functions:

  • Retrieves the data of contractors from NEIM and provisions them as users in Active Directory, which are then synced to Entra ID, based on the configured integration settings.
    • For new users, the integration generates a unique sAMAccountName, Common Name (CN), User Principal Name (UPN), and email address, depending on the configuration.
    • Generates a random password or a password based on a configured password policy for each new users.
    • Assigns users to Entra ID groups and Active Directory groups and Organizational Units (OUs) based on the assignment rules configured in the integration.
    • Assigns Entra ID licenses to users based on the assignment rules configured in the integration.
  • Updates or deactivates users in Active Directory and Entra ID based on the conditions configured in the integration.
  • Removes users from Active Directory and Entra ID groups based on the configured rules.
  • Removes license from Entra ID users based on the configured rules.
  • Generates reports and logs, and sends notification emails based on the integration settings.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • Aquera Contractor: For configuration details, refer to the Aquera Contractor Configuration Guide.
    • Active Directory:  For configuration details, refer to the Active Directory (AD) Version 8 Configuration Guide.
    • Entra ID: For configuration details, refer to the Microsoft 365 Configuration Guide or the Microsoft 365 App Registration Configuration Guide, depending on the type of authentication to be used for the connector.

Integration Setup

This section outlines how the integration must be configured to synchronize contractors from NEIM to Active Directory and Entra ID.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select NEIM / Hybrid.
  5. Enter a name and a description for the integration.
  6. Click Continue. The Overview page is displayed to start configuring the integration.

Overview

The Overview tab provides a summary of the capabilities of the integration and also allows you to enable the core integration features. It has the following sections:

Features

The Features section provides options to enable or disable the following integration features:

  • Enable Entra ID for NEI Management: Enable this toggle for the integration to support synchronizing contractors from NEIM to Entra ID.
  • Enable Active Directory for NEI Management: Enable this toggle for the integration to support synchronizing contractors from NEIM to Active Directory.
  • Enable Notification Management: Enable this toggle for the integration to send email notifications on integration runs.

NEIM

The NEIM tab provides options to select the NEIM application and filter the contractors.

  • Select your NEI application instance: Select the Aquera Contractor application connector configured in the Aquera Admin portal.
  • Enable filtering contractors: Enable this toggle to allow filtering contractors. When enabled, the following two settings are displayed.
  • Filter contractors by: Select Status to filter contractors by their status in NEIM. Currently, the integration supports filtering contractors only by their status.
  • Contractor Statuses: Select the contractor statuses that must be considered for synchronizing to Active Directory and Entra ID. Only the candidates with the selected statuses will be synced to the directories.

Directory

The Directory tab provides options to select your Active Directory and Entra ID application instances and to set the rules for filtering the contractors to be considered for provisioning to the directories.

Directory Settings

The Directory Settings section enables you to configure the settings for selecting the directory instances, selecting the password policies, and setting the directory assignment rules.

Application Configuration

  • Active Directory: Select the Active Directory application connector instance to be used for creating users in Active Directory.
  • Enforce custom password policy for AD users: Enable this toggle to use a custom password policy that is configured in this integration for generating passwords for the users that will be created in Active Directory. When enabled, the Select password policy drop-down is displayed. When disabled, a random password will be generated by the integration, and the password does not follow any specific password policy.
  • Select password policy: Select the password policy that will be used by the integration to generate the password for Active Directory users.
  • Entra ID: Select the Microsoft 365 application connector to be used for creating users in Entra ID.
  • Enforce custom password policy for Entra ID users: Enable this toggle to use a custom password policy that is configured in this integration for generating passwords for the users that will be created in Entra ID. When enabled, the Select password policy drop-down is displayed. When disabled, a random password will be generated by the integration, and the password does not follow any specific password policy.
  • Select password policy: Select the password policy that will be used by the integration to generate the password for Entra ID users.

Directory Assignment Rule

  • Enable directory assignment rules: Enable this toggle to set rules for evaluating the eligibility of contractors that must be created in the directories. When disabled, all contractors satisfying the filter condition will be provisioned to the directories.
  • Directory assignment rules: Set the rules for identifying the contractors to be provisioned to the directories. Currently, only Hybrid directory provisioning is supported; that is, the eligible contractors will be created in Active Directory, which will be synced to Entra ID via Microsoft Entra Connect. Refer to the following Setting Directory Assignment Rules section for details.
Setting Directory Assignment Rules

Follow this procedure to set up directory assignment rules for contractors based on the attribute values received from NEIM:

  1. Click on Choose Attribute to select an NEIM contractor attribute that you want to use in the condition.
  2. After selecting the attribute, select the value(s) of the attribute. Any NEIM contractor that matches the specified attribute values (OR operation) will be added to the directory when the integration runs.
  3. If required, add conditions with more attributes and values by clicking Add Attribute. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. Only contractors who match all the specified conditions will be added to the directory.
  4. In the Directory field, select Hybrid Directory Provisioning.
  5. If required, click Add Row to add another condition within the same rule. When multiple rows are added within a rule, an OR condition is applied to the rows. That means, for each contractor, the integration evaluates the conditions in each row sequentially. If the conditions defined in a row are satisfied for a contractor, the contractor is added to the directory and the remaining rows in the rule are not evaluated for that contractor.
  6. If required, click the + icon to add another rule with a set of rows/conditions. When multiple rules are set, the integration evaluates every rule for each contractor.

Active Directory Sync Settings

This section enables you to configure the NEIM contractor to Active Directory user correlation settings, and the mapping from NEIM contractor attributes to Active Directory user attributes. It also provides options to enable group assignment rules and OU assignment rules.

Account Correlation Settings

  • Enable user match settings: Enable this toggle to set conditions for matching contractors with users in the directory. The following option is displayed.
  • Matching condition: Select an option that indicates the NEIM and Active Directory attributes that must be considered to find the directory user that matches with a contractor.
    • If you select Match users using custom attributes, the NEIM matching attribute and the Active Directory matching attribute fields are displayed.
    • If you select Use AI expression builder, the Match expression field is displayed.
  • NEIM matching attribute: Select the contractor attribute that is used for finding a matching user record in the directory.
  • Active Directory matching attribute: Select the corresponding user attribute in Active Directory that must be used for finding the matching user record.
  • Match expression: Enter the prompt for matching contractors with users. This is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to form the matching expression. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.

Account Creation Settings

  • Groups / OUs Assignment Policy: Select whether you want to enable policies for assigning groups and OUs for Active Directory users.

Attribute Mapping

Policy

The Policy tab enables you to set the OU assignment rules, group assignment rules, license assignment rules, unique user naming policy, and password policy.

Assignment Rules

Default OU Assignment Rules

  • Assign Users to Default OU: Enable this option to assign all users created in Active Directory to a default OU. When enabled, the New users default OU field is displayed.
  • New users default OU: Select the default OU to which all new users must be assigned.
  • Assign terminated users to default OU: Enable this option to assign all users terminated in NEIM to a default OU. When enabled, the Terminated users default OU field is displayed.
  • Terminated users default OU: Select the default OU to which all terminated NEIM users must be assigned.

OU Assignment Rules

  • OU assignment rules: Set the rules for identifying the contractors to be provisioned to the OUs. Refer to the following Setting OU Assignment Rules section for details.
Setting OU Assignment Rules

Follow this procedure to set up OU assignment rules for contractors based on the attribute values received from NEIM:

  1. Click on Choose Attribute to select an NEIM contractor attribute that you want to use in the condition.
  2. After selecting the attribute, select the value(s) of the attribute. Any NEIM contractor that matches the specified attribute values (OR operation) will be added to the OU when the integration runs.
  3. If required, add conditions with more attributes and values by clicking Add Attribute. When multiple attributes are added within the same row, those attribute conditions are evaluated using an AND operation. Only contractors who match all the specified conditions will be added to the OU.
  4. In the Organizational Unit (OU) field, select the OU to which users matching the specified conditions must be assigned.
  5. If required, click Add Row to add another condition within the same rule. When multiple rows are added within a rule, an OR condition is applied to the rows. That means, for each contractor, the integration evaluates the conditions in each row sequentially. If the conditions defined in a row are satisfied for a contractor, the contractor is added to that OU and the remaining rows in the rule are not evaluated for that contractor.
  6. If required, click the + icon to add another rule with a set of rows/conditions. When multiple rules are set, the integration evaluates every rule for each contractor.

Entra ID Group and License Assignment Rules

Setting Group and License Assignment Rules

Follow this procedure to set up group and license assignment rules for contractors based on the attribute values received from NEIM:

  1. Click on Choose Attribute to select an NEIM contractor attribute that you want to use in the condition.
  2. After selecting the attribute, select the value(s) of the attribute. Any NEIM contractor that matches the specified attribute values (OR operation) will be added to the Entra ID group and license when the integration runs.

If the conditions defined in a row are satisfied for a contractor, the contractor is assigned that group and license, and the remaining rows in the rule are not evaluated for that contractor.

Naming Policy

This section details the settings for generating unique SAM Account Name, User Principal Name (UPN), Common Name (CN), and email while creating users in Active Directory and Entra ID:

Naming Policy Settings

  • Generate SAM Account Name: Enable this toggle for the integration to generate the SAM Account Name for a new user in Active Directory. When enabled, the SAM Account Name Generation Settings section is displayed for further configuration.
  • Generate Common Name: Enable this toggle for the integration to generate the Common Name (CN) for a new user in Active Directory. When enabled, the CN Generation Settings section is displayed for further configuration.
  • Generate User Principal Name (UPN): Enable this toggle for the integration to generate the UPN for a new user in Active Directory. This UPN will be synced to Entra ID via Entra Connect. When enabled, the UPN Generation Settings section is displayed for further configuration.
  • Generate email:  Enable this toggle for the integration to generate the email address for a new user in Active Directory. This email address will be synced to Entra ID via Entra Connect. When enabled, the Email Generation Settings section is displayed for further configuration.

SAM Account Name Generation Settings

  • Unique SAM Account Name generation pattern: Select the pattern that must be followed for generating a unique SAM Account Name for new users created in Active Directory. If a different pattern has to be followed for the SAM Account Name, select Custom from the list. 
  • Custom pattern for SAM Account Name: Enter the prompt for generating a unique SAM Account Name for Active Directory users. This field is displayed only if you have selected Custom from the Unique SAM Account Name generation pattern field. This is an AI Prompt field that uses Aquera Intelligence (AI Assistant) to generate SAM Account Names during integration runs. For details on how to set up an AI prompt, review and analyze the output generated by the AI Assistant, refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide.
