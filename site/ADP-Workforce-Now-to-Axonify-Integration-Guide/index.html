Overview 

The Aquera platform enables scalable, automated data management across enterprise applications. Through seamless integrations, it helps organizations eliminate manual processes, ensure consistency, and optimize workflows across complex IT infrastructures.

The ADP Workforce Now to Axonify integration extracts worker data from ADP Workforce Now via a CSV file hosted on an FTP server. The integration then processes and transforms this data, and publishes the updated file to another FTP location. The Axonify application retrieves this file on a scheduled basis and updates worker information within the system.

Key Functions

The ADP Workforce Now to Axonify integration performs the following key functions:

  • Reads ADP worker data from a CSV file stored in an FTP location.
  • Transforms the data into the format required by the Axonify application.
  • Writes the CSV file with the transformed data into another FTP location. This file will be later accessed by Axonify on a scheduled basis to update the worker details.
  • Send an email notification for each integration run.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU or AU region, use the following URL to register:
    • https://admin.eu.aquera.io for EU region
    • https://admin.au.aquera.io for AU region
  • Create Worker reports in ADP Workforce Now: Create a custom report in ADP Workforce Now containing the worker fields required for this integration use case. Refer to the Appendix section of the guide for details on creating the report and exporting it as a CSV file.
    • Make sure that the file is in CSV format. 
    • If the data in the file contains any special characters, it is recommended to convert the CSV file to a UTF-8 encoded format.
  • Schedule the report: Setup a schedule in ADP to push the report data to an FTP location. 
    • You can either use your own FTP server or use Aquera's FTP server to store the ADP worker data.
      Note: Contact your ADP representative to set up publishing the report to an FTP server on a scheduled basis.
  • Configure FTP Connectors in the Aquera Admin Portal: You need to configure two FTP generic connectors in the Aquera Admin Portal - one for retrieving the ADP worker data from an FTP server and another one for writing the transformed data to an FTP location for the Axonify application. Refer to the FTP Generic Configuration Guide for details. 
    • If you want to use your own FTP server:
      • Configure the hostname of your FTP server as the Tenant.
      • For authentication, provide the admin username of your FTP server and enter either the password of your FTP server admin or the private key of the SSH key pair. To generate the SSH key pair, refer to the Generating SSH Key Pair guide.
      • If you are using the SSH key-pair authentication, share the public key with the administrator of your FTP server to configure it in the FTP server
    • If you want to use Aquera's FTP server:
      • Configure sftp.aquera.io as the Tenant.
      • For authentication, provide the admin username of the Aquera SFTP server and enter the private key of the SSH key pair, which you have to generate. 
        • Contact the Aquera Support personnel to obtain the username. 
        • To generate the SSH key pair, refer to the Generating SSH Key Pair guide.
      • Share the public key (of the SSH key pair) with the Aquera Support personnel.

Integration Setup

This section describes the configuration steps required to set up Workday <> Microsoft Entra ID User Photo Sync integration in Aquera.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.
  2. Navigate to Integrations from the left navigation panel.
  3. Click Create Integration.
  4. Select ADP Workforce Now to Axonify Workers Sync.
  5. Enter a name and description for the integration.
  6. Click Continue. The Configuration page is displayed.

Appendix

Creating a Custom Report in ADP Workforce Now

You must create a report in ADP that contains the worker fields you want to sync with Axonify. This section details the procedure to create the custom report.

  1. Log in to your ADP Workforce Now instance.
  2. Click the Reports & Analytics tab.
  3. Click Set Up New under the Custom Reports section.

    The Set Up New Report page is displayed.
  4. Provide a title for the report, select the folder to save the report, and a short description of the report, if required.
  5. Click Select Fields to start adding the required worker data fields to the report.
  6. Search for a worker data field to be added to the report, and then click the Add icon.

    The added field is displayed in the Selected Fields section of the page.
  7. Continue adding all the required fields.
  8. After you have added all the required fields, click Save + Run. The Runtime Settings pop-up screen is displayed.
  9. Click Run.
  10. Once the report is created successfully, click Export and select CSV to export the report as a CSV file.
  11. To schedule the automatic generation of reports, navigate to the Reports & Analytics tab > All Custom Reports and then perform the following:
    1. Click the ellipsis icon next to the report that you have generated.
    2. Select Schedule to Run.
    3. Set the schedule for report generation and click Apply.
  12. To automatically publish the report to an FTP server on a scheduled basis, contact your ADP representative.
