Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This guide describes the integration between ServiceNow, Microsoft Entra ID, and Okta. The integration synchronizes user data from ServiceNow to Entra ID and Okta to support contractor onboarding and offboarding based on requests raised in ServiceNow. It retrieves Requested Items (RITMs) from ServiceNow and performs the corresponding user provisioning and deprovisioning in Entra ID and Okta.

To identify user records across both systems, the integration uses the attribute selected in the Unique Identity as the correlation attribute to match user records between the two systems.

Key Functions

The ServiceNow to Entra ID and Okta integration provides the following functions:

  • Fetches RITM records from ServiceNow to create a contractor.
  • Creates user accounts in Entra ID and Okta.
  • Fetches RITM records from ServiceNow to terminate a contractor.
  • Deactivates the user accounts in Entra ID and Okta.

Prerequisites

The following are the prerequisites:

  • Aquera Tenant Account: You must have a tenant account in the Aquera Admin portal. Visit https://admin.aquera.io to register for a tenant account. Contact your Aquera account manager to get Aquera tenant access.
    Note: If your organization is in the EU region, use the following URL to register: https://admin.eu.aquera.io/
  • Configure Application Connectors in the Aquera Admin Portal: You need to configure the following application connectors in the Aquera Admin Portal:
    • ServiceNow: Refer to the ServiceNow Configuration Guide for details.
    • Microsoft 365: Refer to the Microsoft 365 Configuration Guide for details.
    • Okta: Refer to the Okta Configuration Guide for details.

Integration Setup

This section outlines how the integration must be configured to push user data from Entra ID to ServiceNow.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for ServiceNow / Entra ID / Okta integration.
  5. Enter a name and a description for the integration.
  6. Select your ServiceNow, Microsoft 365, and Okta connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source and target application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

Integration Scenarios

Configure the following integration scenarios:

  • Enable Contractor Onboarding: Select this checkbox to create a contractor account in Entra ID and/or Okta. When enabled, the Onboarding, Unique Identity, and Password Policy sections appear, allowing you to configure additional integration settings.
  • Enable Contractor Offboarding: Select this checkbox to terminate a contractor account in Entra ID and/or Okta. When enabled, the Offboarding section appears, allowing you to configure additional integration settings.

General Settings

Configure the following settings:

  • Enable Report-Only Mode: Select this checkbox to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable Logging: Select this checkbox to generate logs outlining data changes. When enabled, the Log Settings section is displayed for further configuration.
  • Enable Report Generation: Select this checkbox to generate the integration report. When enabled, the Report Settings section is displayed for further configuration.
  • Enable Notifications: Select this checkbox to enable integration-related notifications. When enabled, the Notification Settings section is displayed for further configuration.
  • Enable Log Insights: Select this checkbox to visualize the integration-related metrics. When enabled, the Log Insight Settings section is displayed for further configuration.
  • Enable Full Sync: Select this checkbox to run the integration in full sync mode.
  • Full Sync Frequency: Enter the frequency of full sync from ServiceNow to Entra ID and/or Okta. For example, enter 2 to run the integration in full-sync mode every 2 days.

Onboarding

The Onboarding Ticket Settings section enables you to customize the integration settings according to your specific use case. It has the following settings:

Note: This section appears only if the Enable Contractor Onboarding is enabled in the Integration Scenarios section.

Create Contractor Settings

Configure the following create contractor settings:

  • Onboarding Use Case: Select a use case from the dropdown to specify where the contractor should be created. The following are the available options:
    Note: If both options are selected, the contractor is created in both Entra ID and Okta.
    • ServiceNow to Entra ID: Select this option to create a contractor in Entra ID.
    • ServiceNow to Okta: Select this option to create a contractor in Okta. 
  • Service Catalog Type*: Select the service catalog type used to onboard a contractor from the dropdown menu. For example, Contractor Onboarding.
  • Short Description for Catalog Task (RITM)*: Enter the short description to be added to the catalog task created in ServiceNow for the contractor onboarding RITM.
  • Catalog Task Status (RITM): Select the status to update the catalog task associated with the onboarding RITM in ServiceNow from the dropdown. For example, Work in progress.
  • Okta User Type: Select the user type from the dropdown to create the user in Okta.
  • Source Filter: Select the ServiceNow filter to filter the users while retrieving from ServiceNow. For example, RITM Number / Ticket Number. 
  • Source Filter Value: Provide the value for the selected ServiceNow filter.
  • Email Writeback: Select this checkbox to enable email writeback. When enabled, additional configuration options appear below.
    • ServiceNow Email Writeback Attribute: Select the ServiceNow attribute where the user’s email address will be written back. For example, Email.
  • Status Writeback: Select this checkbox to enable status writeback for onboarding. When enabled, additional configuration options appear below.
    •  Catalog Task Closing Status: Select the status to set when closing the onboarding catalog task associated with the RITM in ServiceNow. For example, Work in progress.

Unique Identity

Note: This section appears only if the Enable Contractor Onboarding is enabled in the Integration Scenarios section.

Unique Identity Settings

Configure the following unique identity settings:

  • Unique Email Generation: Select this checkbox to generate a unique email. When enabled, the following setting appears to define your email.
    • Enter Your Prompt to Create an Email: Enter your prompt to create an email for new users using Aquera Intelligence. Refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide for more details.
  • Use Email as Username: Select this checkbox to use email as username. 
  • Unique Username Generation: Select this checkbox to generate a unique username. When enabled, the following setting appears to define your email.
    • Enter Your Prompt to Create a Username: Enter your prompt to create a username for new users using Aquera Intelligence. Refer to the Using Aquera AI Assistant for Generating Unique Values in Integrations guide for more details.
  • ServiceNow Correlation Attribute: Select the ServiceNow correlation attribute to distinctly find the associated Entra / Okta user. For example, Email.
  • Entra ID Correlation Attribute: Select the Entra ID correlation attribute to distinctly find the associated ServiceNow user. For example, Mail.
  • Okta Correlation Attribute: Select the Okta correlation attribute to distinctly find the associated ServiceNow user. For example, Primary Email.

Attribute Map

The Attribute Map tab defines how data is mapped from the source to the target application. The default mapping of attributes is displayed on the page. The source attributes are listed on the left side and the target attributes are listed on the right side of the page.

You can modify only the source attributes as needed, to align with your specific use case.

You can write JavaScript expressions to modify the value of the source attribute and construct a value to be updated in the target attribute. Write the expression directly in the source attribute field or click the Expand icon in the source attribute field to write a long-expression.

All attributes marked with a red asterisk (*) indicates required fields for successful provisioning.

After the source-to-target attribute mapping is completed, click Save Changes at the top-right corner of the page to save the integration.
