Setting Up Web Hooks

Overview

This guide covers information on how a script orchestration can be set up in the Aquera Admin portal to use the Greenhouse Recruiting web hooks. It also details the webhook setup to be done in the Greenhouse Recruiting system to send events to Aquera.

When an event occurs in Greenhouse Recruiting – for example, when a candidate is hired – and a webhook is configured for this event, data about the event is sent to the Aquera endpoint. The events for which this orchestration needs to be triggered have to be configured in the Greenhouse Recruiting application.

Once the setup is complete, when an event configured in the web hook setup occurs in the Greenhouse Recruiting system, it will be sent to Aquera and the script orchestration is triggered for processing that event.

The following procedures are involved in the Webhook setup:

  • Setting Up Script Orchestration in Aquera
  • Setting Up Web Hooks in Greenhouse Recruiting

Setting Up Script Orchestration in Aquera

Prerequisite

Make sure that the Greenhouse Recruiting application connector is configured successfully in the Aquera Admin portal. For details on how to configure the connector, refer to the Greenhouse Recruiting Configuration Guide.

Configuration and Trigger Setup

  1. Log in to the Aquera Admin portal.
  2. Navigate to the Orchestrations module, and click Create Orchestration.
  3. Enter the following details on the Add Orchestration pop-up window and click Done:
    • Provide a name for the orchestration.
    • Select Script from the list of orchestration types drop-down.

      The Configuration tab is displayed.
  4. Perform the following setup in the Configuration tab: 
    • In the Connections section, click Add Application and then select the name of the Greenhouse Recruiting application connector from the Applications drop-down.
    • Enter the Variable name that is used to refer to the Greenhouse Recruiting application in the script. For example, greenhouse
  5. Click Create at the top-right corner of the page.
  6. Set the trigger for the orchestration as follows:
    • Click the Trigger tab and then click Event in the Add Trigger section.
    • Select the name of the Greenhouse Recruiting application connector from the Select Source drop-down. A URL is displayed as follows:

      This is the Aquera endpoint to which events will be sent from the Greenhouse Recruiting application.

    • Click Create Token to generate a token.

  7. Copy the token for future reference and close the Generate Token pop-up window.

  8. Click Save Changes at the top-right corner of the page.

Write Script to Process Events from Greenhouse Recruiting

This section details how to access Greenhouse Recruiting events in scripts. For guidance on writing scripts in the script editor area of the script orchestration, refer to the Script Orchestration: Working with Scripts guide.

Greenhouse sends a payload of JSON data containing the details of an event when the event occurs in Greenhouse Recruiting application. To know the list of common attributes that are included in the payload for all types of events, refer to the Greenhouse Recruiting Web Hooks documentation. To know the specific set of attributes that are sent for any of the following events, refer to those specific sections in the Greenhouse Recruiting documentation:

  • Application events
  • Candidate events
  • Interview events
  • Job events
  • Organization events

You can use the following format in the script to access any of the attributes sent via the webhook events: request.<requestAttribute>

Note: To print all the data received from an event, enter the following in the script area:
log.info(JSON.stringify(request));

The following is a sample script:

const action=request.action

log.info("The following event is received via webhooks: " +action)

log.info("The candidate ID is " +request.payload.application.id)

log.info(JSON.stringify(request));

Setting Up Web Hooks in Greenhouse Recruiting

Prerequisite

Make sure that the Greenhouse Recruiting application user has permissions to manage and configure web hooks.

Setting Up Web Hooks

  1. Log in to the Greenhouse Recruiting application.
  2. Click the Configure icon on the navigation bar.
  3. Click Dev Center > Web Hooks on the left navigation pane.
  4. Click Web Hooks. The Create A New Web Hook page is displayed.
  5. Enter the following fields to configure your new web hook:
    • Name this webhook: Enter the name that specifies what the webhook is being used for.
    • When: Select the event in Greenhouse Recruiting that causes the webhook to activate.
    • Endpoint URL: Copy the URL from the Trigger tab in the Aquera Admin Portal and paste it in this field. Refer to the Setting Up Script Orchestration in Aquera section for details.
    • Secret key: Enter the token generated in the Trigger tab on the Aquera Admin Portal. Refer to the Configuration and Trigger Setup  section for details on the token.
  6. Click Advanced settings and then provide the Basic Authorisation Credentials where in the token generated from the Trigger tab in the Aquera Admin Portal must be provided as the password for authentication. The username can be any random string. Refer to the Configuration and Trigger Setup section for details on the token.
  7. Click Create web hook to complete the webhook setup. A confirmation message, Webhook created successfully, is displayed.

When you create a new web hook, Greenhouse Recruiting will send a ping to the Aquera endpoint to verify the connection is working. If the ping is accepted successfully, the web hook will be created. If the ping is unsuccessful, Greenhouse Recruiting will save the configuration but deactivate the web hook.

Note: If the web hook is deactivated automatically because the endpoint ping failed, scroll down to the Edit Web Hook section and edit the web hook configuration to fix the error. Ensure that you have selected No under Deactivated.
