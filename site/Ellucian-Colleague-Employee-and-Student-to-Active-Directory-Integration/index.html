Ellucian Colleague (Employee and Student) to Active Directory Integration

← Back to index

Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration synchronizes user accounts between Ellucian Colleague or Banner and Microsoft Active Directory. It uses employee and student data from Ellucian to create new accounts, update existing accounts, and disable users who are no longer active, ensuring Active Directory always reflects the current state of Ellucian.

The integration matches users by linking the Banner ID or Colleague Person ID from Ellucian to the employeeID field in Active Directory. On each run, it automatically decides whether to process all users or only recent changes. The integration links to Active Directory through a secure connection (LDAP) and allows for report-only runs, along with logging, reports, analytics, and email notifications.

The integration matches the Banner ID or Colleague Person ID from Ellucian to the employeeID attribute in Active Directory. This match allows the integration to create, update, or disable the correct user account in Active Directory when Ellucian data changes.

Key Functions

The Ellucian Colleague or Banner to Microsoft Active Directory integration provides the following functions:

  • Retrieves employee and student records from Ellucian by reading identity data, including active status, department, job role, campus, and manager information, used for account management.

  • Matches Ellucian users to Active Directory accounts by linking the Banner ID or Colleague Person ID from Ellucian to the employeeID attribute in Active Directory to accurately identify users.

  • Creates new Active Directory user accounts when a matching account does not already exist, and the user meets configured lifecycle conditions.

  • Updates existing Active Directory user accounts when user details change in Ellucian, applying only the changed attributes to keep accounts current.

  • Disables Active Directory user accounts when a user is no longer active in Ellucian, with optional removal of group access and deactivation email notifications.

  • Manages passwords and group memberships by setting or resetting passwords and adding or removing users from groups, in accordance with defined rules and configuration.

  • Supports full and partial synchronization by automatically processing either all users or only users with recent changes, improving performance and reducing system load.

  • Provides reporting and visibility through report-only runs, logging, reports, analytics, and email notifications to help administrators review and validate changes.

Integration Setup

This section outlines how the integration must be configured to synchronize user accounts in Microsoft Active Directory using employee and student data from Ellucian Colleague or Banner.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin Portal.

  2. Navigate to Integrations from the left-side panel.

  3. Click Create Integration.

  4. Select Ellucian Colleague or Banner to Microsoft Active Directory User Lifecycle.

  5. Enter a name and description for the integration.

  6. Select your Ellucian Colleague or Banner and Microsoft Active Directory connector instances.

  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the Ellucian Colleague or Banner and Microsoft Active Directory application connectors used for the integration. You can review, replace, or update the application connector instances used by the integration, if required.

Parameters

The Parameters section enables you to customize the integration settings according to your specific use case. It has the following settings:

General Settings

Configure these settings to define how the integration manages user accounts and group access in Active Directory.

  • Enable User Lifecycle Management: Select this option to allow the integration to manage user accounts in Active Directory. When enabled, the integration can create new users, update existing users, and disable users based on Ellucian data.
  • Enable Group Management: Select this option to allow the integration to manage group memberships in Active Directory. When enabled, users can be added to or removed from groups based on the configured rules.

Execution Features

  • Enable Report-Only Mode: Select to generate a report detailing the data changes that would happen, but without applying them.
  • Enable Logging: Select to generate logs outlining data changes. The Logging section appears only when this option is enabled.
  • Enable Notifications: Select to enable email notifications for the integration. The Notifications section appears only when this option is enabled.
  • Enable Report Generation: Select to generate the integration report. The Reporting section appears only when this option is enabled.
  • Enable Dashboard Generation: Select this option to view integration metrics and execution details in a dashboard for monitoring and analysis.
  • Enable Debug Mode: Select this option to capture additional technical details for troubleshooting. This option is typically used only during testing or issue investigation.

Log Settings

Configure the following logging settings:

  • Select the Log Detail Level:  Select the desired log detail level from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following option only appears when the dashboard detail level is set to Custom.
      • Attribute Filter: controls which event attributes are included or excluded.   
        • Included Attributes: Include only selected attributes in the log.
        • Excluded Attributes: Include all attributes except the selected ones in the log.
  • Show Attribute Values for Each Event: Select to populate the attribute values when choosing the Full Details option. This option is hidden when Summary Only is selected.

Report Settings

Configure the following settings to customize the report:

  • Select Report Detail Level: Select the desired level of report detail from the dropdown menu.
    The following are the available options:
    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.
    • Full Details: All attribute details are displayed.
    • Custom: Includes or excludes selected attributes. The following options only appear when the report detail level is set to Custom.
      • Attribute Filter: Controls which attributes are included or excluded in the report.
        • Included Attributes: Select the attributes to include in the CSV report.
        • Excluded Attributes: Select the attributes to exclude from the CSV report.
  • Integration Report Download: Select to allow downloading the integration report.
  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details or Custom option.

Log Insights Settings

Configure the following log insight settings:

  • Select the Log Insights Detail Level: Select the desired level of detail from the dropdown menu on the analytics dashboard.
    The following are the available options:

    • Summary Only: Attribute details are not displayed. Only the count of CRUD operations is shown.

    • Full Details: All attribute details are displayed.

    • Custom: Includes or excludes selected attributes. The following option only appears when the log insights level is set to Custom.

      • Attribute Filter: Controls which attribute details are included or excluded in the analysis.

        • Included Attributes: Select the attributes to include in the insights.

        • Excluded Attributes: Select the attributes to exclude from the insights.

  • Show Attribute Values for Each Event (requires consent to store event data temporarily): Select to populate the attribute values when choosing the Full Details or Custom option.

Notification Settings

Defines how the integration sends email notifications and reports.

Welcome Email Notifications: Controls email notifications sent to users when their Active Directory account is created.

  • Send Welcome Email to User: Enables sending a welcome email when a new Active Directory account is created.

  • Email Subject: Specifies the subject line used for the welcome email sent to the user.

  • Email Body: Specifies the content of the welcome email. This field supports dynamic placeholders such as username, email address, and temporary password.

Deactivation Email Notifications

Controls email notifications sent when a user account is disabled.

  • Send Deactivation Email to Supervisor/Manager: Enables sending an email notification to the user’s supervisor or manager when the user account is deactivated.

  • Deactivation Email Subject: Specifies the subject line used for the deactivation email.

  • Deactivation Email Body: Specifies the content of the deactivation email sent to the supervisor or manager.

Notification Report Settings

Defines report-related email notifications generated after each integration run.

  • Send Integration Report via Email:
    Enables sending an email notification with the integration execution report after each run.

  • Notification Report Detail Level:
    Specifies the level of detail included in the emailed integration report.

  • Show Attribute Values for Each Event:
    Includes attribute-level values for each create, update, or disable event in the report.

Success Notifications

Controls email notifications sent when the integration completes successfully.

  • Admin Email Address (Success Notification): Specifies the administrator's email address to receive success notifications.

  • Email Subject: Specifies the subject line used for the success notification email.

  • Email Notification Template: Specifies the content of the email sent when the integration completes successfully.

Failure Notifications

Controls email notifications sent when the integration encounters an error.

  • Admin Email Address (Failure Notification): Specifies the administrator's email address to receive failure notifications.

  • Email Subject: Specifies the subject line used for the failure notification email.

  • Email Notification Template: Specifies the content of the email sent when the integration fails.

Manage Employees

  • User Type for Lifecycle Management: Select the type of users to include in lifecycle processing. For example, select Employee to manage employee accounts only.

  • Ellucian Data Sources: Select the Ellucian data sources used to build user profiles. For example, select Employees, Persons, Institution Jobs, and Institution Positions to collect identity and job details.

  • Ignore Errors for Data Sources: Select the reference data sources for which processing errors should be ignored. For example, select Pay Classes or Academic Levels to skip errors from optional reference data.

  • Enable Manager Mapping: Enable automatic creation or mapping of manager accounts in Active Directory. For example, enable this option to create a manager account if it does not already exist.

Ellucian Colleague Settings

  • Ellucian Correlation Attribute: Select the Ellucian attribute used to uniquely identify users. For example, select Colleague Person ID to match users across systems.

  • Enabled Lifecycle Actions: Select the lifecycle actions that the integration should perform. For example, select Create, Update, and Delete to enable full lifecycle management.

Active Directory Settings

  • Active Directory Correlation Attribute: Select the Active Directory attribute used for user matching. For example, select Employee ID to correlate with the Ellucian identifier.

Employee Creation Settings

  • Employee Creation Rule: Select the rule that determines when new employee accounts are created. For example, select All Active Employees to create accounts for all active employees.

  • Custom Employee Creation Rule: Select this option and enter a custom expression to control when employee accounts are created. For example, create accounts only for staff with a valid role.

Employee Updation Settings

  • Employee Update Rule: Select the rule that determines when existing employee accounts are updated. For example, select All Active Employees to update all active accounts.

  • Custom Employee Update Rule: Select this option and enter a custom expression to control when employee accounts are updated. For example, update accounts only when key attributes change.

Employee Deactivation Settings

  • Employee Deactivation Rule: Select the rule that determines when employee accounts are deactivated or deleted. For example, select All Inactive Employees to disable accounts after termination.

  • Custom Employee Deactivation Rule: Select this option and enter a custom expression to control when employee accounts are deactivated. For example, deactivate accounts after a termination date is reached.

Manage Policy 

Configure the rules and manage password policy:

  • Password Policy Generation Method: Select how passwords are generated. 

    • Auto Generated Password: Select to generate passwords automatically. 

    • Custom Password Policy: Select to define your own rules.

      • Password Complexity Rules: Select the required complexity for generated passwords. For example, select uppercase, lowercase, numbers, and special characters to enforce strong passwords.

      • Allowed Special Characters: Select which special characters can be used in generated passwords. For example, allow characters such as @, #, or $.

      • Minimum Character Types Required: Select the minimum number of character categories required in each password. For example, require at least three different character types.

Generate User Principal Name (UPN)

Configure the rules and logic to generate unique User Principal Names for users.

  • UPN Generation Prompt: Select and define the rules used to generate UPNs. For example, generate the UPN using the first letter of the first name, the full last name, and a numeric suffix to ensure uniqueness.

    • Code View (Toggle): Enable Code View to edit the UPN generation logic directly as text.

    • Analyze and Preview: Select Analyze and Preview to validate the prompt and review generated UPN values before saving.

  • Advanced Preview Options (Toggle): Enable Advanced Preview Options to control how many UPN variations are generated and reviewed.

  • UPN Preview: Select to review sample UPN values and confirm format and uniqueness before applying the configuration.

Generate User Email

Configure how user email addresses are generated, validated, and previewed before assignment.

  • Email Generation Technique: Select how email addresses are generated. For example, select Configure Email Policy to define a custom format or select Use UPN as Email to reuse the generated UPN.

  • Email Generation Prompt: Select and define the rules used to generate email addresses. For example, generate emails using first name and last name, append the domain, and add numbers if duplicates exist.

    • Code View (Toggle): Enable Code View to edit the email generation logic directly as text. 

    • Analyze and Preview: Select Analyze and Preview to validate the prompt and review generated email addresses before saving.

  • Advanced Preview Options (Toggle): Enable Advanced Preview Options to control preview behavior.

  • Email Preview: Select to review sample email addresses to ensure correctness and compliance with organizational standards.

Manage Groups

  • Enable Group Assignment After User Creation: Select to automatically assign groups to users as soon as their Active Directory accounts are created. For example, assign default access groups to all new users.

  • Manage Groups Based on Department Change: Select to manage group memberships when a user’s department changes. For example, move users to department-specific groups when they transfer roles.                                                                              Select the option to automatically remove users from groups when they no longer meet the rule conditions. For example, remove access when a user leaves a department.

Employee Group Rules

  • Assign Group After User Creation: Select to configure group rules that apply during user creation. For example, assign administrative or baseline access groups to new users.

    • Custom Expression: Select and define a condition that must evaluate to true for the rule to apply. For example, apply the rule to all users by using a condition that always evaluates to true.

    • Groups to Be Assigned: Select one or more groups to assign when the condition is met. For example, assign users to the Administrators group.

  • Add New Rule: Select to create additional group assignment rules.

Department-Based Group Rules

  • Department-Based Group Rule: Select this section to configure group assignments based on department information.

    • Custom Expression: Select and define a condition that evaluates department data. For example, match users whose primary job belongs to a specific department.

    • Department-Based Group: Select the group that should be assigned when the department condition is met. For example, assign users in IT to the RDS Remote Access group.

  • Switch to Condition Builder: Select this option to build conditions using a guided interface instead of writing expressions manually.

  •  Add New Rule: Select to create additional department-based group rules to support multiple departments.

Note: Click the refresh button to reload the latest group list from Active Directory. 

Debugging Settings

  • Import Limit: Set the maximum number of records the integration should process during a debug run. Use this field to limit the number of users processed by the integration. This is useful when testing changes or troubleshooting issues. For example, setting the import limit to 1 processes only a single user record.
  • Employee Filter: Select the criteria used to filter employees retrieved from Ellucian during the debug run. For example, select Filter by Colleague Person ID to target a specific employee record.
    • Employee Filter Value: Enter the exact value corresponding to the selected filter. For example, if filtering by Colleague Person ID, enter the specific person ID to process only that employee.
