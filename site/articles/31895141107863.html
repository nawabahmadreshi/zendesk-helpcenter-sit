Overview

The Integration Analysis module featured in the Aquera Admin portal helps you to analyze and manage user attributes across multiple applications. It enables you to identify and resolve conflicts, ensuring clean and consistent data across systems. You can use the Integration Analysis to perform the following:

  • Record Linking Analysis: Resolve conflicts in user attributes
  • Group Membership Analysis: Resolve conflicts in group assignments.
  • Organizational Unit Analysis: Resolve conflicts in organizational unit assignments.
  • Attribute Map Analysis: Assist with attribute mapping during integrations.

Email Notification Configuration

Before creating an analysis, it is recommended to configure email notifications. This ensures that automated alerts are sent when an analysis completes successfully or encounters errors, helping you stay informed about its status without the need for manual monitoring. The Email Notification Configuration allows you to set up email alerts for all types of analysis.

Perform the following steps to configure the email notification settings:

  1. Log in to the Aquera Admin portal and navigate to Integration Analysis on the left panel.
  2. On the Integration Analysis page, click the vertical ellipsis (⋮) located at the top right corner of the screen to display the Settings option.
  3. Click Settings. The Email Notification Configuration pop-up window is displayed.
  4. In the pop-up window, click the Analysis Type dropdown menu and select the analysis type(s) for which you want to configure email notifications.
  5. Click the Recipient List dropdown menu and select the user(s) who should receive the notification emails.
    Note: Only users added to your Aquera instance will be available for selection.
  6. Select the Send the notification emails for any errors checkbox to receive email notifications when an error occurs during analysis execution.
  7. Select the Send the notification emails upon analysis completion checkbox to receive email notifications when the analysis completes successfully.
  8. Click Done to save your changes. The saved configuration will apply to all selected analysis types.
  9. Repeat steps 2 through 8, and modify the configuration as needed to change the email notification settings.

Record Linking Analysis

Record Linking Analysis identifies duplicate or conflicting resource records across two applications by comparing unique people attributes. It uses smart algorithms to classify resources as linked and unlinked, and supports cleanup between applications.

Using Record Linking Analysis to Resolve People Resource Attribute Conflict

You can use Record Linking Analysis to identify and resolve user attribute conflicts across two applications.

Suppose your organization uses two applications to manage users, and you want to ensure there are no duplicate user records. You need to compare unique user attributes between a source application and a target application. In some cases, the target application may not have values for the selected attribute, or the values may be null.

Record Linking Analysis allows you to select a distinct user attribute that uniquely identifies each user across both applications. If the target application does not contain values for that attribute, the system compares other available attributes to identify potential matches.

For example, assume your organization is using Active Directory (AD) and ADP applications for all users, and it needs to confirm that there aren't any duplicate user records across the two applications.

The employeeNumber in ADP is assigned to a worker when first entered into ADP and will remain consistent even when that worker changes his position. Therefore, the employee number attribute of a user can be used for Record Linking Analysis, but the attribute value is null for all existing users in AD.
Record Linking Analysis helps you copy all users' employee number attribute values from ADP to AD. You need to create a user analysis across ADP and AD by selecting the employee number attributes (which act as a matching key) in both applications. Once the matching key is created, you can execute the Analyze operation. You will then see a list of linked users whose match keys are exact matches between the source and target applications.

For users whose employee number attribute doesn't have any value in AD, you will see them listed under unlinked users. You can review the users from the unlinked user list, and if confirmed, you can copy the employee number attribute from ADP to AD for these users. 

Once the cleanup is done, you can execute the Analyze operation again to confirm that all resources appear in the linked list.

Create Record Linking Analysis

Perform the following steps to create a Record Linking Analysis:

  1. Log in to the Aquera Admin portal and navigate to Integration Analysis on the left panel.
  2. Refer to the Email Notification Configuration to set up automated email alerts for the analysis.
  3. Click the + Add New button at the top-right corner of the screen to launch the Add a New Analysis pop-up window.
  4. Select the Record Linking Analysis under the Select the Analysis Type column. The related configuration fields are displayed.
  5. Enter a name for the analysis in the Analysis Name text box.

  6. Choose one of the following options:

    • Select Correlation: Choose this option to reuse an existing correlation key.
      • Search for an existing correlation by entering the application or attribute name in the Search correlation text box.
      • Select the required correlation for the source and target applications.
    • Create Correlation: Choose this option to create a new correlation.
      • Choose the source and target applications, resources, and their respective attributes to define a new correlation.

    Note: This attribute pair will be referred to as the "match key" hereafter.

  7. Select the Refresh Application Data checkbox to refresh data from the selected applications before running the analysis.
  8. Click Run Analysis. You will be redirected to the Integration Analysis page, where the analysis appears with the status In Progress. If you want to stop the analysis while it is running, click the Stop Analysis button. The analysis process will stop, and the status will change to Stopped.

    The following statuses indicate the current state of the record linking analysis:
    • In Progress: The analysis is currently running.
    • Stopped: The analysis was manually stopped by clicking Stop Analysis.
    • Completed: The analysis finished successfully.
    • Error: The analysis failed due to a technical issue.
    • Not Generated: The analysis did not run because the selected applications have no correlated users.

Group Membership Analysis

Group Membership Analysis uses smart algorithms to generate rules based on specific user attributes required for assignment to a group. It helps automate group assignment using defined rules and highlights conflicts for manual review and correction.

Using Group Membership Analysis to Resolve the Group Assignment Conflict

You can use group membership analysis to resolve the group assignment conflict for users in an application.

Let's say your application has multiple groups, and numerous users might be incorrectly assigned to any of those groups. Instead of scanning every group and its members individually, you can create a group membership analysis for an application.

When you run the analysis, the system’s smart algorithm defines rules based on specific attributes a user must have to belong to a group. It then identifies the Same User and Different User records for that group. You can also use the generated rules to automatically assign any new member to that group via integration. For example, suppose there is a "Sales Technology" group in Active Directory, where almost all users have the title "Sales Engineer" and belong to the "Sales" department. The system’s algorithm detects this pattern and defines it as a rule. Any user who meets both criteria is considered a Same User, while users who do not meet the criteria are considered Different User for that group.

For the different users, you can view the conflict details to understand the issue that needs to be corrected.

The analysis can be performed in two ways: 

  • By selecting only the target application: In this method, only the target application and its groups and attributes are selected for analysis, and the group membership rules are generated accordingly.

Organizational Unit Analysis Structured

Organizational Unit Analysis uses smart algorithms to generate rules based on specific user attributes required for assignment to an organizational unit (org unit). It identifies matched and unmatched user assignments across org units, helping to automate accurate user placement and highlighting conflicts that need correction.

Using Organizational Unit Analysis to Resolve the Organization Unit Assignment Conflict

You can use Organizational Unit Analysis to resolve user assignment conflicts across organizational units within an application.

Manually reviewing each org unit and its members for assignment accuracy can be time-consuming, especially in large applications. Instead, running an Organizational Unit Analysis automates this process.

When you run the analysis, the system’s smart algorithm defines rules based on specific attributes a user must have to belong to an org unit. It then identifies matched and unmatched user records for that org unit. You can also use the generated rules to automatically assign any new member to that organization unit via integration. For example, suppose there is a "Sales Technology" org unit in Active Directory, where almost all users have the title "Sales Engineer" and belong to the "Sales" department. The system’s algorithm detects this pattern and defines it as a rule. Any user who meets both criteria is considered Matched, while users who do not meet one or both are considered Unmatched for that org unit.

For the unmatched users, you can view the conflict details to understand the issue that needs to be corrected.

Attribute Map Analysis evaluates and compares user attributes across two applications to identify suitable attribute mappings for integration. It highlights Matched and Unmatched users to guide accurate data correlation.

Using Attribute Map Analysis to Assist in the Integration

You can use attribute map analysis to evaluate the attributes suitable for performing an integration between two applications.

To perform an integration, you must know the attributes to be mapped. For example, you need to perform an integration between ADP and AD applications. You will create a correlation rule to ensure that a unique identifier, such as an attribute whose value is the same across both the ADP and AD. You will create a correlation between the employee number attribute of both applications and use this correlation to run the attribute map analysis. The attribute map analysis will compare the employee number attribute value of all users in the AD with employee number attribute values in the ADP application to calculate the users eligible for analysis. The attribute map analysis will analyze all existing attributes between both applications for the eligible users and will retrieve results on each attribute mapping pair, i.e., whether the attribute values are matched or unmatched.

The final report will provide you with all the attributes suitable for performing an integration.

View Analysis Report

This section provides a summary of the attribute map analysis for users whose correlation match key is matched between the source and target applications. 

Perform the following steps to view the analysis report summary:

  1. Locate the attribute map analysis that you recently created or the one you want to review.
  2. Click the analysis name to view the analysis report summary.

After clicking the analysis name, the analysis report is displayed. It shows the total number of matched users under Total Analyzed Records and presents detailed findings organized across the following tabs:

All

The All tab displays all source attributes and their corresponding target attributes that are automatically generated for all matched users from the attribute map analysis, regardless of the mapping status. The results are grouped by mapping status, with mapped attributes shown first, followed by unmapped and then excluded attributes.

You can search for a specific source or target application attribute by entering the attribute name in the Search attribute search box.
Notes:

  • The Default Map label indicates that the source attribute was automatically generated by the system based on its match with the corresponding target attribute. If the user edits the attribute’s expression, its status changes from Default Map to Edited.
  • You can edit system-generated attributes, but you cannot delete them. Instead, you can reset them. If you create an attribute by defining the entire expression yourself and save it as a new attribute, that attribute can be deleted.
  • All source attributes and their corresponding target attributes are listed as Unmapped or Excluded by default. To use a particular expression for integration, you must map it using the Map action button.
  • If no source attribute exists for a corresponding target attribute, it is automatically moved to the Excluded tab by the system. In this case, the source attribute section displays No Rules Generated, the matched/unmatched counts are not shown, and you cannot perform the Map or Unmap actions until it is included in the analysis using the Excluded/Included button.

Matched Records

You can view all user records where the selected attribute value is the same in both source and target applications. By default, this section displays the correlation match attribute for the source and target applications in the table, along with the selected attribute.

The following features are available in the Matched Records tab:

  • Add Columns

    Click the Add Column icon () to select the additional user attributes for both source and target applications that will appear in the report. Select the Show selected checkbox to view only the selected attributes. Select the Clear all checkbox to clear your selection.

    You can view a maximum of four attribute columns in total, combining both source and target applications. If you select more than four attributes, navigate through them using the left and right arrows () available in both the source and target application views.

  • Resizable Attribute Columns

    You can resize the attributes column in the table by dragging its border left or right.

  • Pagination

    Pagination controls at the bottom of the page allow you to navigate between pages and set the number of user records displayed per page.

  • Access User Profile

    Click on any attribute value of the user to view their complete profile.

  • Sort and Filter

    Click the dropdown arrow next to any attribute header column to use the Sort and Filter features to organize your data in a specific order, making it easier to view and analyze.

    • Sort: This feature allows you to organize your data in a specific order for easier viewing and analysis.
      • Ascending: Sorts the data from A to Z (for text) or from smallest to largest (for numbers).
      • Descending: Sorts the data from Z to A (for text) or from largest to smallest (for numbers).
    • Filter: Use the Search box to filter for specific user records.
    • After sorting and filtering based on your requirements, click Apply. Your data will then be adjusted accordingly. Once applied, the Sort and Filter indicator appears next to the corresponding attribute header’s dropdown, showing that sorting or filtering is active.
      Note: Sorting and filtering can be applied to only one attribute column at a time. If you apply sorting or filtering to another attribute column, the previous sorting and filtering will be reset.

  • Calculated Value

    The Calculated Value option controls whether the system displays the final value produced by a source expression. When enabled, it shows the result of evaluating the source expression, even if that expression combines multiple attributes, allowing you to see the actual value that will be used. 
    For example, if name.familyName = Smith and name.givenName = John, and the source expression is name.familyName + name.givenName + "@example.com", the Calculated Value would display SmithJohn@example.com. 
    If the source expression references only a single attribute, the calculated value is identical to the source attribute value. By default, this toggle is turned on. When turned off, the Calculated Value column disappears, and the calculated values for the selected attributes are no longer displayed.

Unmatched Records

You can view all user records for which the selected attribute value is different or missing in either the source or the target application. Refer to the Matched Records section to learn more about adding columns, sorting and filtering attributes, resizing the table attributes column, using pagination controls, viewing the complete user profile, and calculating values for the source expression.

Exclusion of Unmatched Users from the Analysis
Perform the following steps to exclude the unmatched users from the analysis:
  1. Select the required checkboxes to the left of the attribute column to exclude specific members. To select all users on the current page, select the checkbox in the attribute column header. To select all users across all pages, click Select All Records. Click Unselect All to clear all selections.
  2. Click Exclude, and the Confirm Exclusion pop-up window will appear. 
  3. Click Exclude to confirm. The selected members are moved to the Excluded Records tab.

Note: Once an unmatched user record is moved to the Excluded Records tab, the unmatched count is automatically refreshed and updated on the main analysis page.

Excluded Records

You can view all members who were excluded from the Unmatched Records section. Refer to the Matched Records section to learn more about adding columns, sorting and filtering attributes, resizing the table attributes column, using pagination controls, viewing the complete user profile, and calculating values for the source expression.

Inclusion of Excluded Records in the Analysis
Perform the following steps to include the excluded records back into the analysis:
  • Select the required checkboxes to the left of the attribute column to include specific members. To select all users on the current page, select the checkbox in the attribute column header. To select all users across all pages, click Select All Records. Click Unselect All to clear all selections.
  • Click Include, and the Confirm Inclusion pop-up window will appear.
  • Click Include to confirm. The selected members are moved back to the Unmatched Records tab.

Note: Once an excluded user record is moved to the Unmatched Records tab, the unmatched count is automatically refreshed and updated on the main analysis page.

Editing a Source Attribute

If required, you can manually edit a source attribute to match the target attribute based on your use case.

To edit the source attribute, perform the following steps:

  1. Hover your cursor over the attribute you want to edit. The Edit icon will appear. 
  2. If the source attribute expression uses a table, a Table icon appears next to the Edit icon. Click the Table icon to view the table details. You can modify the existing data as needed. For more information, refer to the Manage Tables section.
  3. Click the Edit icon to open the Edit expression screen.

    If the expression uses a table, the Edit Expression screen displays the Table icon. Click the Table icon to view the table details. You can modify the existing data as needed. For more information, refer to the Manage Tables section.
  4. Define your expression so that it matches your target attribute.
  5. Click Save. Two options will appear:
    • Save: Finalizes your changes in the current system-generated attribute. Its status changes from Default Map to Edited. You can edit system-generated attributes, but you cannot delete them. Instead, you can reset them. 
      Perform the following steps to re-edit or reset the modified system-generated attribute:
      1. Hover your cursor over the attribute, and click the Edit icon again.
      2. Modify your expression so that it matches your target attribute, and click Save or Save as new based on your case.
      3. Click Reset in the Edit expression screen if you want to restore the system-generated attribute.
      4. Click the Refresh icon at the top-right of the screen to update the matched and unmatched counts, and enable the Map action button. Its status changes from Edited to Default Map again.
        Note: If your source attribute and its corresponding target attribute are in a Mapped status and you reset your recent edit, the status will automatically change from Mapped to Unmapped.
    • Save as new: Finalizes your changes as a separate entry without modifying the system-generated attribute.
      • The new attribute is created as a subset of the main attribute.
      • The user-created attributes do not display any label initially. Once this is edited again, it will appear as Edited. The user-created entries can be deleted.
        Perform the following steps to delete or re-edit a user-created attribute:
        1. Hover your cursor over the attribute, and click the Edit icon again.
        2. Modify your expression so that it matches your target attribute, and click Save or Save as new as needed.
        3. To delete the user-created attribute entry, click Delete in the Edit expression panel.
          Note: If your source attribute and its corresponding target attribute are in a Mapped status and you delete your new user-created entry, the status will automatically change from Mapped to Unmapped.
  6. After editing or creating a new attribute, the count of matched and unmatched attributes will temporarily not appear, and the Map action button will be disabled.
  7. Click the Refresh icon at the top-right of the screen to update the matched and unmatched count, and enable the Map action button.
  8. Click the button under the Map column to view the options Map on Create and Update, Map on Create, Map on Update, and Don’t Map. Select any one of these options and map your attributes for integration based on your use case. Once mapped, the attribute moves from the Unmapped tab to the Mapped tab. Refer to the Export Mapped Attribute to the Integration section to export the mapped attributes to the integration.
    Note: For unmapped attributes, the Don’t Map option is disabled.

Exclude or Include Attributes for the Analysis

Perform the following steps to include/exclude attributes for the analysis:

  1. Click Excluded/Included.
  2. Select the attributes you want to exclude (you can also search for attributes using the Search attribute textbox).
  3. Click the > icon to move selected attributes to the Exclude side. Their status will change to Excluded, and they will appear in the Excluded tab.
  4. To include attributes back, select the required attributes from the Exclude side.
  5. Click the < icon to move them back to the Include side.

Mapped

The Mapped tab displays only the source attributes and their corresponding target attributes that are currently mapped.

  • Click the button under the Map column to view the options Map on Create and Update, Map on Create, Map on Update, and Don’t Map. Select any one of these options to update the mapping setting that you previously configured during attribute mapping. Refer to the Unmapped section for more details on mapping unmapped attributes.
  • Select Don’t Map to remove an existing mapping. Once unmapped, the attribute pair moves from the Mapped tab to the Unmapped tab.
  • You can search for a specific source or target application attribute by entering the attribute name in the Search attribute search box.
  • Pagination controls at the bottom of the page allow you to navigate between pages and set the number of user records displayed per page.

Export Mapped Attributes to the Integration

Perform the following steps to export the mapped attributes to the integration:

  1. Click Export Mapped to open the Export To Integration panel.
  2. Select the integration where you want to export the mapped attributes.
  3. Click Export to send the attributes to your integration. You can view the exported attributes in the Attribute Map tab in that integration.

Excluded

The Excluded tab displays the source attributes and their corresponding target attributes that are not currently included in the analysis.

  • If a target attribute does not have a corresponding source attribute, the system automatically moves it to the Excluded tab. In such cases, the source attribute section shows No Rules Generated, the matched and unmatched counts are not displayed, and the map and unmap actions are disabled.
  • To include these attributes in the analysis, use the Excluded/Included option. Once included, the attributes become available for mapping and unmapping. For more information, refer to the Exclude or Include Attributes for the Analysis section to learn how to include attributes in the analysis.
  • You can locate a specific source or target application attribute by entering the attribute name in the Search attribute search box.
  • Pagination controls at the bottom of the page allow you to navigate between pages and set the number of user records displayed per page.

Table

This section explains how tables are created, used, and managed within Attribute Map Analysis. 

Tables are generated either automatically by the system or manually by users. They store key–name and key–value pairs that define custom attributes and are used in source expressions during attribute map analysis.

  • System-generated tables are automatically applied in the source attribute expression.
  • User-generated tables must be manually added to the source attribute expression.

Refer to the Editing a Source Attribute section for more details.

View Tables

This section describes how to view all available tables in the system.

To view all tables, click the Table button located in the top-right corner of the main analysis screen. The Table View screen appears, displaying a list of both system-generated and user-generated tables.

Create a Table

This section explains how to create a new table manually.

Perform the following steps to add a table:

  1. Click Add Table on the Table View screen. The Add Table screen appears.
  2. On the Add Table screen, enter the following details:
    • Table Name: Enter a unique name for your table.
    • Description: Enter a description for your table.
    • Key: Enter a key name.
    • Key Value: Enter a key value
  3. Click + Add Row to add additional key–value pairs.
  4. Click the Delete icon () to remove a specific key–value pair.
  5. Click Save Table to create the table. Once saved, the new table appears in the Table View screen.

Create a Table Using CSV

This section explains how to create a table by uploading a CSV file.

Note: The CSV file must follow a key–value format.

Perform the following steps to create a table by uploading a CSV file:

  1. Click Add Table on the Table View screen. The Add Table screen appears.
  2. On the Add Table screen, enter the following details:
    • Table Name: Enter a unique name for your table.
    • Description: Enter a description for your table.
  3. Click Upload as CSV on the Add Table screen.
  4. Browse and select the CSV file, then click Open. The table is automatically populated from the uploaded file.
  5. Click + Add Row to add additional key–value pairs.
  6. Click the Delete icon () to remove a specific key–value pair.
  7. Click Save Table to create the table. Once saved, the new table appears in the Table View screen.

Export Table

This section explains how to export table data for integrations. Perform the following steps to export a table as a CSV file:

  1. Click Add Table on the Table View screen. The Add Table screen appears.
  2. Click the Export option on the Add Table screen. The table is downloaded as a CSV file, which can be used in integrations.

Manage Tables

This section explains how to edit or delete existing tables.

Perform the following steps to manage a table:

  1. Click the table name or the Edit icon () in the Table View screen.
  2. The table details appear, where you can:
    • Edit the table name or description.
    • Modify existing key–value pairs.
    • Click + Add Row to add new entries.
    • Click Delete to remove a specific key–value pair.
  3. Click Save Table to apply changes.
  4. Click the Delete icon () at the end of the table row in the Table View screen to permanently remove the table.
