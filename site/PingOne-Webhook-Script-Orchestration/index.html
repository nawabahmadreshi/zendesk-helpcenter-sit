Overview

'PingOne Webhook Script' is a type of orchestration that helps update User Events such as Creating, Deleting, Moving, and Updating a User and reflecting the updates in the relevant channels. By using this orchestration, the User Events can be updated in one go.  

This step-by-step demonstration will help Users executePingOne Webhook Script orchestration from Aquera to PingOne. Once this orchestration is completed, whenever the User Event at PingOne is changed, it will be reflected in Aquera's application automatically. 

Target Creation

  1. On the Aquera Admin Home page, navigate to the Orchestrations module and click Create Orchestration.
  2. Enter the following details on the Add Orchestration page:
    • Enter a Name for the orchestration.
    • Select PingOne Webhook Script from the orchestration type drop-down list and click Done.
  3. Select an Application that supports the orchestration from the dropdown in the Application field. 
  4. Enter the Java script that suits your requirement under Script and click on Create on the top right. 
  5. Go to Event Detail to see the URL. Click Token to view it. As this token appears only once, make a copy of it for future reference. 
  6. Click Save Changes on the top right, so that the token will be assigned to the desired Event. 

Message Listener Orchestration

  1. Login to PingOne with username and password and click Administrators.
  2. Click Connections from the left menu.
  3. Navigate to Webhooks.
  4. Click Add Webhook and provide a Name.
  5. Copy URL from the Aquera's Event Detail page (refer to Step-4 of this guide) and paste the same in the box prescribed for the URL. 
  6. Select Json as the format.
  7. Click Add Headers
  8. Copy the token from the Aquera's Event Detail page and paste the same in the two boxes prescribed for Basic Authentication.
  9. Click the Event Type dropdown and select User Events.
  10. Check the User Access Events
  11. Ensure that User Created, User Deleted, User Moved, and User Updated are selected
  12. Click SaveUser can see the newly created Webhook
  13. Click the dropdown next to the newly created Webhook
  14. The new Webhook can be edited and updated. The changes will be reflected on Aquera's page automatically.
