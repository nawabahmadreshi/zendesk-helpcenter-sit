Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types.

This integration automates creating and updating Freshservice tickets based on employee lifecycle events in PrismHR (onboarding, offboarding, employee transfers (moves). The integration runs on a scheduled basis, checking PrismHR for employee changes (new hires, terminations, and transfers), creating Freshservice incidents, and sending reports to administrators.

This integration performs the following tasks:

  • Monitors employee onboarding, offboarding, and transfers in PrismHR and runs the integration process based on configured schedules or manual execution to process such events.
  • Extracts key employee data from PrismHR, including the employee’s email (work email primarily, with fallback to contact email if enabled) for identification.
  • Maps the employee’s email from PrismHR to Freshservice to uniquely identify and create or update tickets. 
  • Includes additional employee attributes (first name, last name, department, location, job code, reports to, last status change date, and last job change date) in the incident description of Freshservice tickets to provide support agents with detailed context about the employee’s status and organizational information.
  • Supports filtering employee data based on either an absolute date (YYYY-MM-DD) or a relative duration (e.g., 5d for the last 5 days), allowing the integration to process only recent or relevant changes.
  • Creates or updates Freshservice tickets for each qualifying employee event, applying preferences such as ticket priority and status. The ticket description also includes all selected additional fields.
  • Maintains detailed logs of all operations, capturing both successes and failures for auditing and troubleshooting purposes.
  • Generates CSV reports summarizing ticket creation and update results, with options for full or basic reports as configured in preferences.
  • Sends summary emails to administrators containing operation results and optionally attaches CSV reports, formatted using customizable email templates.
  • Handles errors, logs error details, and notifies administrators via email.

Integration Setup

This section details the configuration of the scheduled integration between PrismHR and Freshservice.  It covers creating an integration, setting up integration preferences, and configuring how employee lifecycle events are processed. The integration reads employee data changes from PrismHR, including onboarding, offboarding, and transfers, at scheduled intervals and creates or updates the corresponding incident in Freshservice. 

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select PrismHR Employees to Freshservice Tickets.
  5. Enter a name and a description for the integration.
  6. Select your PrismHR and Freshservice connector instances.
  7. Click Continue. The configuration appears to set up the integration.

Set Up Preferences

The Preferences section contains essential parameters that determine how the integration operates. Each setting is crucial in customizing the integration's behavior to meet your organization's needs.

  • Report Only: Generates a report outlining the data changes detected by the integration without making any updates or changes to the system.
  • Enable Integration Report Download: Allows users to download the integration report in CSV format for further analysis or record-keeping.
  • Report Type: Specifies the level of detail included in the generated report.
  • Enable Dashboard Generation: Creates a dashboard to visually analyze the results of the integration.
  • Recipients' Email Addresses: Comma-separated list of email addresses that will receive email notifications from the integration.
  • Email Body on Successful Integration Execution: Customizable content that will be used as the body of email notifications sent when the integration runs successfully.
  • Email Body on Failed Integration: Customizable content that will be used as the body of email notifications sent when the integration encounters errors.
  • Debug Mode: Enables detailed logging to help identify and troubleshoot issues during integration execution.
  • Ticket Priority: Sets the default priority level for tickets created by the integration. (Default: Medium)
  • Ticket Status: Sets the default status for tickets created by the integration. (Default: Open)
  • Additional Employee Fields for Onboarding: Lets you select extra employee details to include in the onboarding description field. Email and employee ID are always included by default.
  • Additional Employee Fields for Offboarding: Lets you select extra employee details to include in the offboarding description field. Email and employee ID are always included by default.
  • Additional Employee Fields for Transfers: Lets you select extra employee details to include in the transfer description field. Email and employee ID are always included by default.
  • Enable Email Fallback: Allows use of the contact email if the work email is unavailable for the employee.
  • PrismHR Client ID: The unique client ID provided by PrismHR, used to authenticate and enable the integration.
  • Employee Data Filter by Date/Duration: Specify a date (YYYY-MM-DD) or duration (e.g., "2d" for 2 days) to retrieve and create tickets for employee data changes that have occurred since then.
    Note:The Employee Data Filter by Date/Duration preference helps you control which employee changes are to be processed, so you only handle the most relevant updates. You can enter either a specific date (like 2025-05-20) or a duration (like 2d for the last two days). The integration will then only retrieve and create tickets for employee data changes that happened on or after this specified time.
    • If you enter a date (e.g., 2025-05-20), only changes from that date forward are included.
    • If you enter a duration (e.g., 2d), only changes from the last two days are included.

Attribute Mapping

The integration of PrismHR to Freshservice incident creation maps the following attributes:

PrismHR Employee Attributes Freshservice Attributes
workEmail or contactEmail Work Email

This integration requires only email mapping to identify the employees. The employee’s work email is used primarily, with an option to fall back to the contact email if the work email is unavailable and the fallback preference is enabled.

In addition to email, the following fields are included in the incident description for onboarding, offboarding, and transfer tickets to provide context:

PrismHR Field  Freshservice Incident Description 
firstName  First Name
lastName Last Name
worker.client ID Employee ID
lastStatusChangeDate Last Status Change Date
department Department
location Location
jobCode Job Code
reportsTo Reports To
lastJobChangeDate Last Job Change Date

Run the Integration

The integration can be scheduled to run automatically at customizable intervals. Available frequencies include 15 minutes, 30 minutes, 1 hour, 4 hours, 6 hours, 12 hours, or 24 hours. You can also enable recurring executions to run the integration continuously on your chosen schedule.

On successful execution, the CSV file will be sent to the specified recipients. In case of integration errors, notifications and reports will also be sent to the email addresses.

Download CSV Report

You can download a report of an integration run in CSV format by clicking the execution status indicators in the History column and navigating to the Execution Timeline panel.
