Overview

This guide provides information on the audit logs. The Audit tab in the Logs module displays all audit logs.

An audit log is a record of events and changes in your system. It can be a specific activity or a series of activities. 

The audit log entries capture events by recording who performed an activity, what activity was performed, and when it was performed. Audit logs document nearly all system changes, offering a comprehensive history of your users' actions.

Audit Log Entries

The following operations in the Users, Agent, Applications, Synchronization, and Partner Organization modules are recorded in audit logs: 

  • The creation, modification, and deletion of Aquera Admin users. When a user is created in the tenant, the audit log displays the role of the user also.
  • Switching into the current tenant and switching out to another tenant by a user.
  • The Aquera Agent download, creation, update, and deletion of Agent groups, Agent token refresh, and addition of notes in the Agent Details column.
  • The creation, modification, and deletion of application connectors.
  • The report downloads and the deletion of report templates through the Data tab > Reports in Applications.
  • The creation, modification, and deletion of Aquera integrations. The audit logs also capture the following details:
    • Manual execution/stop of integrations.
    • Schedule changes
    • Disabling or enabling integrations

      Note: When a basic user executes the Dayforce to Active Directory integration through the wizard provided by Aquera, two entries will be created in the audit logs - one for saving the integration and another for executing it.
  • The modifications to the file upload retention settings in the File Collections module.
  • The scheduled deletions of file upload history records in the File Collections module. When you click on this audit log record to view the details, the username who triggered the deletion is displayed as Aquera-Console, as the deletion is automatically triggered and not triggered manually by a user.
  • The user logs out of the tenant manually. The logout is recorded, not the timeout.

Note: The retention period of audit logs is 1 year; the audit logs will be automatically deleted after a year.

For file-based application connectors, the file upload activity is captured in the audit logs.

Filtering Logs

You can filter the audit logs by username, action, module name, and console. A dropdown at the top of the page allows you to filter logs by time range, including All, 1 day, 3 days, 7 days, or a Custom range. Click Custom to filter the logs within a specific date and time.

Note: The default values 00:00:00 and 23:59:59 are automatically set for the start and end times, respectively.

For further details, refer to the Custom Time Period Settings section.

The date and time of each log are displayed under the Date column on the page. Click button to choose if you want to display the log time in your local time or UTC. The date and time will be displayed based on the selection. 

Custom Time Period Settings

The Custom option enables you to set the time period as per your requirements for filtering audit logs. Perform the following steps to set the custom time period:

  1. Select Custom from the dropdown at the top right corner of the page. A Calendar window is launched.

  2. Select a start date & time and an end date & time from the displayed calendar.

  3. Click Apply. The logs are filtered based on the selected date and time range.

Basic Filter

The Switch to basic filter option enables you to filter logs by specifying the username, action, module name, and console value in the Enter value field. Based on the specified value, the logs are displayed in tabular format. 

Attribute Filter

The Switch to the attribute filter option enables you to set conditions for filtering audit logs. You can filter logs by selecting the username, action, module name, or console from the Select column dropdown list. Then, select the required condition from the Select condition dropdown list and specify the corresponding value in the Enter value field. Based on the condition, the logs are displayed in tabular format.

Notes:

  • After applying a custom filter, switching to the Basic or Attribute filter mode clears the selected custom filter dates.
  • If no log data is available for the specified filter criteria, the message No Audit Logs is displayed.
  • You can switch back to the Basic Filter at any time from the Attribute Filter option

Download Audit Logs

You can download the audit logs as a CSV file to your local system by clicking Download in the top-right corner of the Audit Logs page. The exported CSV file contains audit log entries based on the filters currently applied on the page.

Notes: 

  • The Download button will be disabled if the time period filter is set to all.
  • The Audit Log page does not support pagination. All logs are displayed in a single, continuous list, and you must scroll to view the entire log history. When downloaded, the CSV file includes the entire audit log, not just the entries currently visible on the screen.
