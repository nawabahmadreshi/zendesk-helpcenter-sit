Overview

The Aquera platform enables your organization to scale complex data management by automating data integration across various resource types. Aquera's integrations facilitate automatic provisioning, ensuring consistency across systems. It enables organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately enhancing efficiency in managing complex IT infrastructures and applications.

This integration automates provisioning between Dayforce, Active Directory, and Microsoft Entra ID. It synchronizes employee and candidate data, manages group assignments, and handles onboarding, updates, and terminations. It also supports automated access assignments and provides detailed notifications, logs, and reports.

This guide covers the end-to-end tasks in setting up the integration to sync employee and candidate data from Dayforce to the directory.

Key Functions

This integration automates identity lifecycle management with the following primary capabilities:

  • Synchronizes employee and candidate data from Dayforce to Active Directory and Microsoft Entra ID.
  • Generates password for new users in Active Directory and Microsoft Entra ID.
  • Manages group assignments.
  • Sends notifications about provisioning events.
  • Generates detailed logs and reports.

Integration Setup

This section details the configuration and mapping process for integrating Dayforce with Active Directory and Microsoft Entra ID. It outlines the steps to create the integration, set up integration preferences, and define how data is mapped.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Search for Dayforce / Active Directory / Microsoft Entra ID in the search box next to the Explore Templates section, and then select it to create a new integration.
  5. Enter a name and a description for the integration.
  6. Click Continue.

Once created, the following tabs are displayed:

  • Overview
  • Source
  • Directory
  • Policy
  • Execution
  • Notification Center
  • Tables
  • Schedule

Features List

This section lists the features supported by the integration. You can enable or disable each feature based on your requirements.

The following are the supported features:

  • Enable Employee Management in Entra ID: Enable this option to synchronize employee data with Microsoft Entra ID. When enabled, the integration provisions new user accounts, updates user attributes based on changes, and deprovisions accounts according to employee lifecycle events and data changes in Dayforce.
  • Enable Candidate Management in Entra ID: Enable this option to synchronize candidate data with Microsoft Entra ID. When enabled, candidate accounts can be provisioned and managed in Microsoft Entra ID.
  • Enable Employee Management in Active Directory: Enable this option to synchronize employee data with Active Directory. When enabled, the integration creates, updates, and deprovisions user accounts in Active Directory based on employee lifecycle events and data changes in Dayforce.
  • Enable Candidate Management in Active Directory: Enable this option to synchronize candidate data with Active Directory. When enabled, candidate accounts can be provisioned and managed in Active Directory.

Application Settings

This section allows you to identify the Dayforce application connector for employee and candidate management and define onboarding and offboarding criteria.

The following sub-sections are displayed on this page for configuration:

  • Employee Source Configuration
  • Candidate Source Configuration

Employee Source Configuration

The following fields are displayed:

  • Source Application: Select the Dayforce application connector that you have configured in the Aquera Admin portal for employee management. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
  • Employee onboarding/offboarding criteria: Select one of the following options to define how employee accounts from Dayforce are evaluated for creation and termination in the directory:
    • Based on status: Select this option to evaluate account creation and termination based on the employee’s status in Dayforce. When selected, the integration uses the configured statuses to determine which users are eligible for account creation or termination in the directory. The following parameters are available for further configuration:
      • Status(es) to be considered for account activation in the target application: Select the statuses that should be considered active. Employees with these statuses in Dayforce are considered for account creation in the directory.
      • Status(es) to be considered for account deactivation in the target application: Select the statuses that should be considered inactive. Employees with these statuses in Dayforce are considered for account termination in the directory.
    • Based on date: Select this option to evaluate account creation and termination using a date attribute in Dayforce. When selected, the integration uses the configured date attribute to determine when employee accounts are considered for creation or termination in the directory. The following parameters are available for further configuration:
      • Date to be considered for account activation in the target application: Select a date attribute in Dayforce to identify employees for creation in the directory.
      • Date to be considered for account deactivation in the target application: Select a date attribute in Dayforce to identify employees for termination in the directory.

Candidate Source Configuration

The following fields are displayed:

  • Source Application: Select the Dayforce application connector that you have configured in the Aquera Admin portal for candidate management. If you do not find your application connector in the list, click the refresh icon available next to the field to reload the available connectors.
    Note: The source application can be the same for employee and candidate management.
  • Candidate onboarding/offboarding criteria: Select one of the following options to define how candidate accounts from Dayforce are evaluated for creation and termination in the directory:
    • Based on status: Select this option to evaluate account creation and termination based on the candidate’s status in Dayforce. When selected, the integration uses the configured statuses to determine which candidates are eligible for account creation or termination in the directory. The following parameters are available for further configuration:
      • Status(es) to be considered for account activation in the target application: Select the statuses that should be considered active. Candidates with these statuses in Dayforce are considered for account creation in the directory.
      • Status(es) to be considered for account deactivation in the target application: Select the statuses that should be considered inactive. Candidates with these statuses in Dayforce are considered for account termination in the directory.
    • Based on the date: Select this option to evaluate account creation and termination using a date attribute in Dayforce. When selected, the integration uses the configured date attribute to determine when candidate accounts are considered for creation or termination in the directory. The following parameters are available for further configuration:
      • Date to be considered for account activation in the target application: Select a date attribute in Dayforce to identify candidates for creation in the directory.
      • Date to be considered for account deactivation in the target application: Select a date attribute in Dayforce to identify candidates for termination in the directory.
  • Dayforce-generated report name to retrieve candidates: Enter the name of the Dayforce report that contains candidate data. The integration retrieves candidate information from the specified report.
    Note: You can view a report in your Dayforce instance by navigating to Reporting and Analytics > Reporting > Reports.
  • Report column to retrieve candidate status: Enter the column name in the report that contains candidate status information. This field is used by the integration to evaluate candidate status.
  • Report column to retrieve the candidate's last modified info: Enter the column name in the report that contains the last modified timestamp information. This field is used by the integration to identify updates to candidate records.

Time Based Settings

This section allows you to define the timezone and schedule for onboarding and offboarding processes.

The following fields are displayed on this page:

  • Applicable time zone for provisioning: Select the timezone to be used for onboarding and offboarding. The following are the available options:
    • Co-Ordinated Universal Timezone (UTC): Select this option to onboard or offboard users using Coordinated Universal Time (UTC), regardless of the user's local time.
    • Attribute Based Timezone: Select this option to onboard and offboard users based on their work location’s timezone. This allows processing to align with each user’s local timezone. When selected, the following field is displayed for further configuration:
      • Attribute for the time zone calculation: Select a Dayforce attribute to determine the employee's timezone.
  • Onboarding time: Enter the time for the onboarding in 24-hour format (HH:MM). For example, 03:40.
  • Offboarding time: Enter the time for the offboarding in 24-hour format (HH:MM). For example, 03:40.

Mailbox Settings

This section allows you to configure and manage settings related to a user’s mailbox. The following fields are available for configuration in this section:

  • Forward emails sent to deactivated user to manager / Forward emails of a deactivated user to their manager: Select this checkbox to automatically forward all incoming emails of a deactivated user to their manager's email address. This ensures that important communications are not missed and can be handled appropriately after the user is deactivated.
  • Disable Exchange ActiveSync (mobile) on deactivation / Disable Exchange ActiveSync on deactivation: Select this checkbox to disable Exchange ActiveSync for users upon deactivation. If not selected, the deactivated user may continue to access emails on previously configured mobile devices. Enabling this option prevents mobile email access after deactivation.
    Note: Exchange ActiveSync enables users to access email, calendar, contacts, and tasks on mobile devices, and allows them to continue using this data even when offline.
  • Remove recurring calendar meetings / Remove recurring meetings from calendar on deactivation: Select this checkbox to automatically remove all recurring meetings organized by the user when the user is deactivated.
  • Convert user mailbox to shared mailbox / Convert Users regular mailbox to shared mailbox on deactivation: Select this checkbox to convert the private (regular) mailbox of a user to a shared mailbox on user deactivation.
  • Remove shared mailbox delegations / Remove shared mailbox delegations on user deactivation: Select this checkbox to remove the shared mailbox delegations on user deactivation. If this checkbox is selected, all delegated access to other users’ shared mailboxes will be removed from the user on deactivation.
  • Add shared mailbox permissions to manager / Add shared mailbox permission to manager on deactivation: Select this checkbox to add the shared mailbox permission to the user's manager when the user is deactivated.
    Note: Deactivating and converting the shared mailbox and assigning permission to the manager changes will be visible in the Exchange Online admin portal.
  • Send auto-response after deactivation / Send auto-reply message after the user is deactivated: Select this checkbox to set an automatic out-of-office reply in a user's mailbox upon deactivation. When enabled, the following fields are displayed for further configuration:
    • Out-of-office / auto-reply message configuration / Auto-reply message configuration: Enter the message that must be sent as an automatic out-of-office reply on behalf of a user when the user is deactivated. - This field must be hidden when the parent field is disabled.
    • Auto-reply end date (in days): Enter the number of days after which you want to disable the automatic out-of-office reply from a deactivated user's mailbox. For example, if you configure 90 in this field, the automatic out-of-office reply will not be sent after 90 days of deactivation of a user account. - This field must be hidden when the parent field is disabled.

Note: The Exchange Administrator role is required for Exchange-related features.

This section allows you to configure users’ group settings. The following fields are available for configuration in this section:

  • Reassign group ownership to manager if user is sole owner / Transfer group ownership to manager on deactivation or deletion: Select this checkbox to automatically remove a user's group ownership and assign it to their manager when the user is deactivated or deleted. This applies only when the user is the only owner of the group.

WriteBack Settings / Writeback Settings

This section allows you to configure settings for updating data back to the source system. The following fields are available in this section:

  • Enable HR WriteBack / Enable Writeback: Select this checkbox to enable updating attributes in Dayforce (source system) based on changes in the directory.
  • Write Back to / Select attributes for writeback: Select which attributes should be updated in Dayforce when their values change.
  • WriteBack Source Attribute / Source application attribute: Select the correlation attribute available in the directory that uniquely identifies a user record in Dayforce. For example, select employeeId if you want to use it to match user records in Dayforce.
  • WriteBack Target Attribute / Target Application Attribute: Select the correlation attribute available in Dayforce that uniquely identifies a user record in the directory. For example, select EmployeeNumber.

Provisioning Methods

This section describes the provisioning methods supported by the integration. It explains how employee and candidate records from Dayforce are created, updated, and deactivated across different directory environments based on the selection of a provisioning method.

The integration supports the following provisioning methods:

  • Active Directory Provisioning: In this provisioning method, employee and candidate records from Dayforce are created, updated, and deactivated in Active Directory. The integration synchronizes data from Dayforce to Active Directory and applies the required changes, including account creation, attribute updates, password management, Organizational Unit (OU) assignments, and group assignments. You can use this method if your use case requires automated provisioning from Dayforce to Active Directory.
  • Entra ID Directory Provisioning: In this provisioning method, employee and candidate records from Dayforce are created, updated, and deactivated in Microsoft Entra ID. The integration manages the user lifecycle by synchronizing user attributes and handling access assignments in Entra ID.
  • Hybrid Directory Provisioning: This provisioning method combines both Active Directory and Microsoft Entra ID by provisioning users in Active Directory, and then the records are synchronized to Entra ID using the Microsoft Entra Connect. Employee and candidate records from Dayforce are initially created and managed in Active Directory.
    Note: Microsoft Entra Connect (Azure AD Connect) is a Microsoft tool that integrates your on-premises Active Directory with Microsoft Entra ID. It syncs users and access information from Active Directory to Entra ID.
  • Dual Directory Provisioning: This provisioning method enables independent provisioning to both Active Directory and Microsoft Entra ID. In this method, employee and candidate records from Dayforce are created, updated, and deactivated in both Active Directory and Microsoft Entra ID separately.
