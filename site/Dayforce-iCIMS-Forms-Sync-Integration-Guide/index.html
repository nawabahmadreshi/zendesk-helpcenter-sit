Overview

The Aquera platform allows your organization to scale complex data management by automating data integration across resource types. Aquera's integrations facilitate the automatic provisioning, ensuring consistency across systems. It allows organizations to streamline processes, dynamically scale resources, and automate repetitive tasks, ultimately improving efficiency in managing intricate IT infrastructures and applications.

This integration enables you to synchronize iForms from iCIMS to the forms in Dayforce. When an iForm is signed or completed for an employee in iCIMS, the form is added to the corresponding user record in Dayforce. This integration is triggered when the status of an iForm changes in iCIMS.

To identify matching employee records across both systems, the integration uses the SSN attribute value.

Key Functions

The Dayforce / iCIMS Forms Sync integration performs the following functions:

  • Processes the event that is sent from iCIMS when an iForm is completed or signed, based on the integration setting.
  • Converts the form data received from iCIMS to a PDF format and adds it to the corresponding user record in Dayforce.

Integration Setup

This section outlines how the integration must be configured to synchronize iForms of users from iCIMS to Dayforce.

Create Integration

Perform the following steps to create the integration:

  1. Log in to the Aquera Admin portal.
  2. Navigate to Integrations from the left side panel.
  3. Click Create Integration.
  4. Select Dayforce / iCIMS Forms Sync.
  5. Enter a name and a description for the integration.
  6. Select your iCIMS and Dayforce connector instances.
  7. Click Continue. The Configuration page is displayed to configure the integration.

Configuration

The following sections are displayed under the Configuration tab:

Applications

The Applications section displays the source (iCIMS) and target (Dayforce) application connectors for the integration. This section allows you to search and update these application connectors.

Parameters

The Parameters section enables you to customize the integration features and settings according to your use case. It has the following settings:

Integration Scenarios

Enable the following feature based on your requirements:

  • Enable Form Sync: Turn on this toggle to synchronize iForms from iCIMS to Dayforce. When turned on, the  Forms Sync Settings section is displayed for further configuration.
    Note: This setting is mandatory for the integration.

General Settings

Configure the following general settings:

  • Report only: Turn on this toggle to execute the integration in the report-only mode. The report-only mode generates a report to show the data that would be changed without actually making those changes.
  • Enable debug mode: Turn on this toggle to generate logs outlining data changes that would occur, for debugging. When the toggle is on, the Debug Settings section is displayed for further configuration.
  • Enable logging: Turn on this toggle to generate logs outlining data changes. When the toggle is on, the Log Settings section is displayed for further configuration.
  • Enable notifications: Turn on this toggle to enable integration-related notifications. When the toggle is on, the Notifications section is displayed for further configuration.
  • Enable report generation: Turn on this toggle to generate the integration report. When the toggle is on, the Report Settings section is displayed for further configuration.
  • Enable Log Insights: Turn on this toggle to visualize the integration-related metrics. When the toggle is on, the Log Insights section is displayed for further configuration.

Forms Sync Settings

The Forms Sync Settings section allows you to view the source and target attributes that are used by the integration for identifying the matching user records across the source and target applications. It also provides options to customize the settings for synchronizing the forms based on your use case.

Note: This section appears only if you have turned on the Enable Form Sync toggle in the Parameters section.

Correlation Settings

This section shows the source and target attributes that are used for identifying the matching user records across the source and target applications. It also provides options to customize the form sync settings based on your use case.

  • Source correlation attribute: Displays the cost center attribute that will be used to identify a unique cost center record in ADP Workforce Now. 
  • Target correlation attribute: Displays the cost center attribute that will be used to identify the corresponding cost center record in ADP iHCM. 

Provisioning Settings

This section provides the settings to enable creation, update, and deletion of cost center records.

  • Create cost centers: Turn on this toggle to allow the integration to create cost centers in ADP iHCM. When the toggle is off, the integration will not create cost centers in ADP iHCM.
  • Update cost centers: Turn on this toggle to allow the integration to update cost centers in ADP iHCM. When the toggle is off, the integration will not update cost centers in ADP iHCM.
  • Delete cost centers: Turn on this toggle to allow the integration to delete cost centers in ADP iHCM. When the toggle is off, the integration will not delete cost centers in ADP iHCM.
    Note: When the Fetch Limit is configured, the cost center deletion operations will not be performed. Since the number of records retrieved from the source is limited, the integration cannot identify the actual number of records that are missing in the source.

Provisioning Threshold

This section provides the settings to limit the number of cost center records that will be created, updated, or deleted in ADP iHCM.

  • Fetch limit: Enter the maximum number of cost center records that must be retrieved from ADP Workforce Now in a single integration run. 
  • Create limit: Enter the maximum number of cost center records that are allowed to be created in ADP iHCM in a single integration run. If you enter 0, no cost center records will be created. To allow unlimited creations of cost centers, enter -1
    Note: This field is displayed only if the Create cost centers toggle is turned on.
  • Update limit: Enter the maximum number of cost center records that are allowed to be updated in ADP iHCM in a single integration run. If you enter 0, no cost center records will be updated. To allow unlimited updates of cost centers, enter -1
    Note: This field is displayed only if the Update cost centers toggle is turned on.
  • Delete limit: Enter the maximum number of cost center records that are allowed to be deleted in ADP iHCM in a single integration run. If you enter 0, no cost center records will be deleted. To allow unlimited deletions of cost centers, enter -1
    Note: This field is displayed only if the Delete cost centers toggle is turned on.

Sensitive Fields

This section enables you to set any sensitive (PII -Personally Identifiable Information) fields processed by the integration. Currently, the integration does not handle any PII data, so you can skip this section.

Log Settings

This section enables you to customize the log generation settings. It has the following settings:

Note: This section appears only if you have selected the Enable logging parameter in the Parameters section.

  • Select Log Type: Select a log type. This will decide the level of detail to be included in logs.
    The following are the available options:

    • Minimum: Select this option to include only operation details.
    • Metadata: Select this option to include operation details and attribute names.
      Note: You may ignore the other options in the drop-down list.
  •  Select Log Level: Select the desired log detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the log.
    • Include List: Select this option to include specific attributes in the log.
      • Custom Inclusion List: Enter the attributes to be included in the log. This parameter is available only if you select Include List in the Select Log Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the log.
      • Custom Exclusion list: Enter the attributes to be excluded from the log. This parameter is available only if you select Exclude List in the Select Log Level parameter.

This section enables you to customize the report generation settings. It has the following settings:

Note: This section appears only if you have enabled the Enable report generation parameter in the Parameters section.

  • Select Report Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the report.
    • Include List: Select this option to include specific attributes in the report.
      • Custom Inclusion List: Enter the attributes to be included in the report. This parameter is available only if you select Include List in the Select Report Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the report.
      • Custom Exclusion List: Enter the attributes to be excluded from the report. This parameter is available only if you select Exclude List in the Select Report Level parameter.

Notifications

This section enables you to customize the notification settings. It has the following settings:

Note: This section appears only if you have enabled the Enable notifications setting in the Parameters section.

Notification Template Settings

Configure the following settings to customize notifications:

  • Send email notifications for success: Turn on this toggle to send an email notification on successful cost center syncs in an integration run. When enabled, the Success Notifications section is displayed for further configuration.
  • Send email notifications for failures: Turn on this toggle to send an email notification on failed cost center syncs in an integration run. When enabled, the Failure Notifications section is displayed for further configuration.
  • Attach the report to email: Turn on this toggle to receive an email notification with an attached report for each integration execution.
  • Merge all reports: Turn on this toggle to merge all the reports (records with successful sync and failed sync) in a single integration notification email. When this toggle is turned off, separate reports will be generated for successful records and failed records.
    Note: This toggle is displayed only if you have turned on the Attach the report to email toggle.

Success Notifications

Configure the following settings for successful cost center sync notifications:

  • Send To: Enter the email addresses of the admins to whom the email notifications will be sent upon successful sync of cost center records.
  • Cc: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the subject in email notifications.
  • Email Body: Enter the content that must be used as the email body for notifications sent upon successful sync of cost center records.

Failure Notifications

Configure the following settings for failed cost center sync notifications:

  • Send To: Enter the email addresses of the admins to whom the email notifications will be sent upon failed cost center records sync.
  • Cc: Enter the email addresses of additional recipients who should receive a copy of the email notification.
  • Email Subject: Enter the content that must be used as the subject in email notifications.
  • Email Body: Enter the content that must be used as the email body for notifications sent upon failed cost center sync.

    • Metadata: Select this option to include operation details and attribute names.
      Note: You may ignore the other options in the drop-down list.
  • Select Report Level: Select the report detail level from the dropdown menu. The following are the available options:
    • Full: Select this option to include all attributes in the notification report.
    • Include List: Select this option to include specific attributes in the notification report.
      • Custom Inclusion List: Enter the attributes to be included in the notification report. This parameter is available only if you select Include List in the Select Report Level parameter.
    • Exclude List: Select this option to exclude specific attributes from the notification report.
      • Custom Exclusion List: Enter the attributes to be excluded from the notification report. This parameter is available only if you select Exclude List in the Select Report Level parameter.

Debug Mode

This section enables you to customize the settings for debugging the integration setup.

Note: This section appears only if you have turned on the Enable Debug Mode toggle in the Parameters section. 

Source Filter

  • Enable source filter: Turn on this toggle to filter a specific cost center record and then run the integration for debugging purposes.
  • Filter query: Enter the expression that must be used to filter a cost center record from ADP Workforce Now. For example: nameCode.code eq "01234SE"
    Note: This setting is effective only if you have turned on the Enable source filter toggle.
